Release notes

1.0.0

  • Initial version available with Glims 8.7

1.1.0

  • Correction in the description of field Q.14.1.10 of the Query message - Request information record:
    The field value it may contain is 'REPLYERROR', not  'ERRORREPLY' as previously stated.
  • Introduced new heading: Field syntax
  • Add query argument 'Query specimen container cap control'
  • Correction in Result message - Patient information record:
    • Correction in title (R -> P)
    • Physician field: use correct reference (P.14 instead of P.15) in element description
    • Result status type field: typo in element description
  • Added field H.9.1.1 in Header record of Query and Result message

1.2.0

  • Field specimen type (O.16.1.1) in result message now also used in specimen search.
  • Added fields specimen receipt time (O.15.1.1) and specimen status (O.15.1.2)
  • The default target of a result comment or isolation comment in a result message is now configurable on the station in Glims.
  • Added field Microbiology station field in test order record of result message

1.3.0

  • Support of cancel status in isolation in a result message
  • Provide alternate solution of sending microbiology test information in result message; via M-record. See: Result message, microbiology result block, microbiology test record

1.4.0

  • Introduction of entering device records in query and result message
  • Add query argument segment 'PCarOffsTm': pathology carrier creation time offset

1.5.0

  • Introduce and use additional manufacturer fields in result QC-record.
    1. Field M.4.1.1: Target value
    2. Field M.5.1.1: Deviation value
  • Introduction of new fields to get the "next isolation (id/sequence)" to use in a result message for a new isolation found for a specimen.
    The following fields will be added in the one specimen query:
    1. Field Q.14.1.12: NEXTISOLATION
    2. Field Q.5.1.4: TestID for the microbiology test.
    3. Field Q.5.1.5: Microbiology procedure code.
    4. Field Q.5.1.6: MicroOrganism code (analogous to the O-record ASTM-standard)
  • Introduce and use additional manufacturer fields in work order carrier record.
    1. Field M.4.1.4: Parent carrier number (analoguous to M.4.1.1: Carrier number)
    2. Field M.4.1.5: Parent carrier external id (analoguous to M.4.1.2: Carrier external id)
  • Introduction of new "negative comment code" field (M.5.1.1) in the Microbiology test record of the result message.
  • Introduction of new "microbiology test comment" field (M.6.1.1) in the Microbiology test record of the result message.
  • Introduction of new "Specimen Carrier Observation" record (manufacturer type) in the result message to report any observed changes (like grafting and growth) on a carrier.
  • Introduce and use additional manufacturer fields in the carrier related result records.
    1. Field M.5.1.2: Lot number (of the microbiology reagent lot the carrier belongs to)
    2. Field M.5.1.3: Expiration date (of the microbiology reagent lot the carrier belongs to)
  • Introduce new "Carrier Observation record" for a corresponding specimen carrier record and corresponding isolation carrier record of a result message.

1.6.0

  • Renamed patient information record field P.7.1.1 "spouse last name" into "alternative name" to indicate that it is not just for the spouse last name.
  • Add new query specific tag TestDepartment to have the department responsible to execute a test returned in the work order message field O.25.1.3 Station Department.
  • Introduced a new value 'C' for the action code of a test order record O.12.1.1. In case of a ONEOREDER query, the (non-QC) tests for the same station will now be grouped according to that value. This means that a ONEORDER reply can have 2 O-segments for tests on a specimen (on the same station): one for tests with action code 'N' and one for tests with 'C'.
  • Introduced a new identification field for a manually entered (specimen) ID in the result message test order record, which is accessible as "ManualId" in the unknown specimen trigger.
