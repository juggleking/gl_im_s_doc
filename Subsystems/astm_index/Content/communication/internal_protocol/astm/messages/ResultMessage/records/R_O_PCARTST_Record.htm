Result message - Pathology carrier test record (M)

Usage

This record contains a pathology carrier test related to the pathology carrier identified in the preceding pathology carrier record.

Element Descriptions

Pathology carrier test sequence number (M.4.1.1)

This field contains the pathology carrier test sequence number value.

Medium test code (M.4.1.2)

This field contains the code of the pathology carrier medium test. Glims searches for a medium test with a mnemonic matching the code.

Glims allows to map the medium test code by defining a code set for the medium test table on the station's coding system. If the code set is available but the medium test code is not defined, the carrier test record is ignored.

Carrier test value (M.5.1.1)

The field contains the value of the pathology carrier test.

Action code (M.6.1.1)

This field contains the action code. The supported values are 'N' (new) or 'U' (update).

When the action code is a 'N', Glims will try to create the carrier test. When the action code is a 'U', Glims will update an existing carrier test or create a new one if the carrier test doesn't exist.

Test value user ID (M.7.1.1)

This field identifies the user which is responsible of setting the pathology carrier test value. Glims will search for the user with a login name or initials matching the value user ID.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the value user ID. When no match is found, the pathology carrier record and related records following it are not processed by Glims.

Note: In case this field is left empty, Glims will use the pathology carrier status user ID.

Test date (M.8.1.1)

This field is used to select the pathology carrier test in Glims based on the test date of that carrier. When the pathology carrier test record concerns a creation (Action code = 'N'), this field value is used to set the carrier date in Glims. If no date is specified upon creation, today's date is assumed.
