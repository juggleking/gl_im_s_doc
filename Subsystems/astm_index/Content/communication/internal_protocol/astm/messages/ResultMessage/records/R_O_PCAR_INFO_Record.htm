Element Descriptions

Pathology carrier status (M.4.1.1)

This field allows to update the status of the pathology carrier in Glims: When set to 'Processed', Glims will mark the carrier as being processed (the grafting time is set to the time when this record is being received by Glims)

Pathology carrier status user ID (M.4.1.2)

This field identifies the user which is responsible of setting the pathology carrier status. Glims will search for the user with a login name or initials matching the status user ID.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the status user ID. When no match is found, the pathology carrier record and related records following it are not processed by Glims.

Pathology carrier comment (M.4.1.2)

This field contains the carrier comment text. Glims will store the text in the comment field of the carrier.

Action code (M.4.1.3)

This field contains the action code. The supported values are 'R' (replace) or 'A' (add).

When the action code is a 'R', Glims will try to replace the carrier comment. When the action code is a 'A', Glims will add the comment to the one already available.

Comment user ID (M.5.1.1)

This field identifies the user which is responsible of setting the pathology carrier comment. Glims will search for the user with a login name or initials matching the status user ID.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the comment user ID. When no match is found, the pathology carrier record and related records following it are not processed by Glims.

Notes:

1) In case this field is left empty, Glims will use the pathology carrier status user ID.

2) When multiple comment records are received for the same pathology carrier, Glims will only consider the user Id of the first comment record or any replacing comment record (Action code = 'R')
