Element Descriptions

Specimen or QC Lot ID (O.3.1.1)

This field contains the specimen ID or QC Lot ID. Glims search for a specimen or a QC lot (in that order). The station settings in Glims determine how specimens are to be searched or created.  If no specimen or QC lot is not found, the test order record and following result records are ignored. Refer to the Glims manual for more details about retrieval and auto creation of specimens.

This field is accessible by the unknown specimen trigger.

Blood bag ID code (O.3.1.2)

This field contains the ID of a blood bag (or blood unit). The station settings in Glims determine how blood bags are to be searched. If the blood bag is not found, the test order record and following result records are skipped.

Manual ID (O.4.1.1)

In some use-cases, the Specimen.ID is not scanned by the instrument, but entered manually on the instrument. To differentiate this use-case from the normal routine a separate field is available, which is one of the accessible fields in the unknown specimen trigger, i.e. "ManualId".

Specimen location rack, position and type (O.4.1.2, O.4.1.3, O.4.1.5)

These fields are used for sampling tracking in Glims. The location type field determines how Glims treats the data:

  • When specimen location type is a 'R' (Result), rack and position are stored in the specimen home location (only the first time) and specimen last location.
     
  • When specimen location type is a 'A' (Archive), rack and position are stored in the specimen last location and specimen storage location. Glims will also try to register the specimen in the specimen archive module.
  • When specimen location type is a  'H' (Home), rack and position are stored in the specimen home location and specimen last location.
     
  • When specimen location type is 'RS' (Result - Sort), rack and position are stored in the specimen last sorting location.
    The instrument event record and sort result records that follow the test order record are processed.
     
  • When specimen location type is 'RSA' (Result - Sort - Archive), rack and position are stored in the specimen last sorting location.
    Glims will also try to register the specimen in the specimen archive module. The instrument event record and sort result records that follow the test order record are processed.

A empty or unrecognized location type is considered as a type 'R'.

The fields are accessible by the unknown specimen trigger.

Specimen Cup ID (O.4.1.4)

Glims interprets this field as the cup position number on the instrument's carrousel, represented in Glims as the entry on a station's active work list.

This field contains the ID of the test. The table below shows the usage in Glims:

 Glims uses the assessment method inbound id to identify test ID's, but also allows to configure alternate codes when dealing with concentrator and distributor stations. See Glims manual for more details.

Microbiology procedure (O.5.1.6)

This field contains the microbiology procedure name. Glims uses this field to find the microbiology action of the specimen being uploaded.

This information can also be transmitted in form of a M record: See Microbiology test record

Microbiology test (O.5.1.7)

This field contains a test ID representing the microbiology procedure. Glims uses this field to find a grafting or processing property which is related to the microbiology action of the specimen being uploaded.

This information can also be transmitted in form of a M record: See Microbiology test record

Specimen type (O.16.1.1)

Specimen material code is used in Glims as additional information in the search for a specimen instance. When station's configurations allows it, the material code can also be used to create unsolicited specimens.

This field is accessible by the unknown specimen trigger.

Physician ID (O.17.1.1)

Glims doesn't process the content of this field. It is accessible by the unknown specimen trigger.

Laboratory field 1 (O.21.1.1)

This field is used when the value of the report types (O.26.1.1) is a 'X'.

Instrument ID (O.25.1.1)

The field contains the ID of the result sending instrument. In Glims, this field is known as the station remote ID and used to identify the client station of a concentrator or distributor.

This field is accessible by the unknown specimen trigger.

Specimen department (O.25.1.3)

When this field is filled in, Glims will try to find a matching department.

  • If the department is not found, the specimen's availability will not be confirmed.
     
  • If the department is found, Glims verifies if work is planned on the specimen for that department. If so, the specimen's availability is confirmed.

If the result receiving station has a coding system with a code set on table department, Glims will use the code set to find the department. If no code set is defined, the department is searched by mnemonic.

This field is accessible by the unknown specimen trigger.

Microbiology station (O.25.1.4)

This field is used when uploading microbiology results to Glims. It identifies the station to which the microbiology action is assigned.

When the field is filled in, Glims will search for a station with the same mnemonic as in O.25.1.4. If a coding system is defined on the result uploading station with a code set for stations, Glims will search the station in that code set (and only there). Once the station is found, Glims will search for a microbiology action of the uploaded specimen and assigned to the identified station.

Report types (O.26.1.1)

This field contains the report types.

If the field value is a 'X', Glims will:

  • Store the result value 'X-NORESULT' for the test ID in O.5.1.4. If in addition, laboratory field 1 (O.21.1.1) contains the value 'F', the result value will be 'F-FOLLOWUP'.
  • Skip the result records following this test order record.

Other values in this field are not processed. This field is accessible by the unknown specimen trigger.

Microbiology flag (O.27.1.1)

When this field contains the value 'MICRO', Glims treats this test order segment and all the segments related this test order segment as microbiology related information.
This field is required in order to upload microbiology related results (like isolations, antibiotic results, etc.).

Pathology flag (O.27.1.3)

When this field contains the value 'PATH', Glims treats this test order segment and all the segments related this test order segment as pathology examination related information.
This field is required in order to upload pathology examination related results (like pathology carrier comment, pathology carrier test results, etc.).

Location ward (O.28.1.1)

Glims doesn't process the content of this field. It is accessible by the unknown specimen trigger.

Hospital service (O.30.1.1)

Glims doesn't process the content of this field. It is accessible by the unknown specimen trigger.
