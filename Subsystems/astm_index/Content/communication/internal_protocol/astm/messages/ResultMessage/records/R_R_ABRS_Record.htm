Result message - Antibiotic result record (R)

Usage

These records contain antibiotic result values for the isolation identified by the preceding isolation result record.

Field R.3.1.5  determines the type of antibiotic result value.

Element Descriptions

Antibiotic code (R.3.1.4)

This field contains the antibiotic code.

Glims uses the antibiotic table to find an antibiotic with a mnemonic matching the code. If the antibiotic is not found, the antibiotic result record is rejected.

Glims allows to map the antibiotic code by defining a code set for the antibiotic table on the station's coding system. If the code set is available but the antibiotic code is not defined, the antibiotic result record is rejected.

Report status (R.9.1.2)

When this field contains the value 'H' (Hide), Glims will set the reportable field of the antibiotic result to false.

Other values in this field are ignored.

Operator (R.11.1.1)

This field contains the ID of the operator who performed the antibiotic test.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the operator ID. The user is displayed in the audit trail of the antibiotic result. When no match is found, the antibiotic result record is rejected.

Antibiotic RIS result value (R.4.1.1)

This field contains the antibiotic RIS result value. Glims stores this value in the RIS raw value field of the antibiotic result.

Antibiotic RIS report result value (R.4.1.1)

This field contains the antibiotic RIS report result value. Glims stores this value in the RIS report value field of the antibiotic result.

Antibiotic MIC result value (R.4.1.1)

This field contains the antibiotic MIC result value. Glims stores this value in the MIC raw value field of the antibiotic result.

Antibiotic agar diffusion result value (R.4.1.1)

This field contains the antibiotic agar diffusion result value. Glims stores this value in the agar diffusion raw value field of the antibiotic result.

Antibiotic E-test result value (R.4.1.1)

This field contains the antibiotic E-test result value. Glims stores this value in the E-test raw value field of the antibiotic result.

Antibiotic comment (R.4.1.1)

This field contains antibiotic result comment.

Glims will verify if the value of this field consists of 3 parts, being: a comment prefix, the equal sign ('=') and a comment text. Glims converts the comment prefix and text to a tagged list entry ( \ comment prefix '=' comment text ), and stores the entry in the instrument comment field of the antibiotic result. If the equal sign is not available in the R.4.1.1, Glims takes the value 'COMMENT' as the default prefix.
