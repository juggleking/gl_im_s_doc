Element Descriptions

Isolation sequence number (R.3.1.5)

This field contains the sequence number of the isolation. In Glims, this field corresponds with either an isolation internal or external sequencer field.

An isolation sequence number is essential for Glims in the search or creation of isolations in the database. Glims will reject a isolation result record and the related records when no sequence number is available. Please refer to the Glims manual for more details.

Extra organism flag (R.3.1.6)

When this field contains the value 'EXTRA', the isolation record is recognized as an informational record.

Glims stores the information in this 'extra' record in the internal comment of another isolation record related to the same test order record and with the same isolation sequence number.

Organism code (R.4.1.1)

This field contains the isolation organism code.

Glims uses the micro organism table to find an organism with a mnemonic matching the code. If the organism is not found, the isolation result record and related records are rejected.

Glims allows to map the organism code by defining a code set for the micro organism table on the station's coding system. If the code set is available but the organism code is not defined, the isolation result record and related records are rejected.

Organism name (R.4.1.2)

This field contains the isolation organism name. It is not processed by Glims.

Isolation appraisal code (R.4.1.3)

This field contains the isolation appraisal value. Glims searches for an appraisal with a mnemonic matching the code. If the appraisal is not found, the isolation result record and related records are rejected.

Result status type (R.9.1.1)

When this field contains a 'X', Glims will delete the isolation.

Operator (R.11.1.1)

This field contains the ID of the operator who performed the examination on the isolation.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the operator ID. The user is displayed in the audit trail of the isolation. When no match is found, the isolation result record and related records are rejected.
