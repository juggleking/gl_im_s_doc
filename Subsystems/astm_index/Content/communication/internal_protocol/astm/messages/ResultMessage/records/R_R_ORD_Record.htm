Element Descriptions

Test ID (R.3.1.4)

This field contains the test ID. Glims uses the assessment method inbound id to identify test ID's.

Glims allows to configure alternate codes when dealing with concentrator and distributor stations. See Glims manual for more details.

Dilution condition (R.3.1.5)

This field contains the dilution condition for this result. In Glims, this field is known as the remote condition field of the station's dilution code table.

Dilution factor (R.3.1.6)

This field contains the dilution factor for this result. In Glims, this field is known as the remote factor field of the station's dilution code table.

Result type (R.3.1.8)

This field is ignored when it has the value 'F'.

If the field is not empty and has a value different from 'F', the value is used to compose the Glims Test ID.

Result value (R.4.1.1)

This field contains the result value stored in the Glims result raw value.

Second result value (R.4.1.2)

This field contains the second result value stored in the Glims result raw value, for a composed test id.

Third result value (R.4.1.3)

This field contains the third result value stored in a Glims result raw value, for a composed test id.

Fourth result value (R.4.1.4)

This field contains the fourth result value stored in a Glims result raw value, for a composed test id.

Fifth result value (R.4.1.5)

This field contains the fifth result value stored in a Glims result raw value, for a composed test id.

Result flag (R.7.1.{n})

This field contains a result flag, known in Glims as the flag field of the station flag table.

For almost all instrument connections, Glims will receive a single component in R.7. For compatibility reasons, Glims supports repeated fields (multiple flags) in R.7. An alternate manner is uploading (additional) flags with the result comment flag record.

Result status type (R.9.1.1)

When this field has the value 'X', result components that have a value are overruled with a generic value 'X-NORESULT'. If no single result component has a value, the first result component in R.4.1.1 gets the generic value.

Other values in this field are ignored.

Operator (R.11.1.1)

This field contains the ID of the operator who performed the test.

When the field 'store operator ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the operator ID. The user is displayed in the audit trail of the result (E.g.: 'Operated on instrument by ...'). When no match is found, the result record is ignored and no result is stored in Glims.

When this result record is the first record following the test order record, the field is accessible by the unknown specimen trigger.

Verifier (R.11.1.2)

This field contains the ID of the verifier who verified the test.

When the field 'store verifier ID' is enabled on the result receiving station, Glims will search for the user with a login name or initials matching the verifier ID. The user is displayed in the audit trail of the result (E.g.: 'Verified on instrument by ...'). When no match is found, the result record is ignored and no result is stored in Glims.

When this result record is the first record following the test order record, it is accessible by the unknown specimen trigger.

Date time test started (R.12.1.1)

This field contains the date and time the instrument started the test results being reported. The value is stored in the Glims field measurement started of the result tabel.

When this result record is the first record following the test order record, it is accessible by the unknown specimen trigger.

Date time test completed (R.13.1.1)

This field contains the date and time the instrument completed the test results being reported.  The value is stored in the Glims field measurement completed of the result tabel.

When this result record is the first record following the test order record, it is accessible by the unknown specimen trigger.
