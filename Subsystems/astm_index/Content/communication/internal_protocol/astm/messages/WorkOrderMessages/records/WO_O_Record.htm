Element Descriptions

Specimen or QC Lot ID (O.3.1.1)

This field contains the ID of the specimen or QC Lot.

In case the ID explicitly requested by field Q.3.1.2 of the query message, the value of this field is copied from that query field.

In case the ID is not explicitly requested (for instance an ALL-query), Glims is responsible for returning the ID currently valid for that specimen or QC Lot.

Work list entry sequence number (O.3.1.4)

This field contains the sequence number (SeqNo) of a work list entry, in case Glims searched for specimens or QC lots on a work list.

The field is omitted in the following cases:

  • the query message is an aliquotter query (see field Q.14.1.9)
  • the query message contains the query specific tag 'TestSelectionUpdate'
  • the field send complete selection is enabled on the inquired station

Blood bag ID code (O.3.1.5)

This field contains the ID code of the blood bag and is retrieved by Glims when the query message is requesting cross match tests (see field Q.14.1.8).

Glims uses the field 'blood bag Id' on the inquired station to retrieve the blood bag ID code.

Material type (O.5.{n}.5)

When the query message is requesting for it (see field Q.14.1.2), this field will contain the assessment method remote material id.

Microbiology procedure code (O.5.{n}.6)

When the specimen concerns a microbiology specimen, the field contains the microbiology procedure mnemonic .

If the inquired station has a coding system with a code set on table microbiology procedure, the value for this field is retrieved from the code value. If no code value is defined, the field remains empty.

By default, this field is omitted when isolation result records are following the test order record. It is not omitted when the query specific tag 'MicroIsolWithExam' is available in the query message.

Dilution factor code (O.5.{n}.6)

This field contains the dilution factor code, defined in Glims as the remote factor field in the dilution code table.

Dilution condition code (O.5.{n}.7)

This field contains the dilution condition code, defined in Glims as the remote condition field in the dilution code table.

Property output consumption (O.5.{n}.8)

This field is filled in when the query specific tag 'TestConsumption' is available in the query message. It contains the property output consumption. The consumption value is reported in the unit of the specimen's material, or alternatively, the specimen size unit of the inquired station.

Test repeat method code (O.5.{n}.9)

When the field 'send repeat info' is enabled on the inquired station and the has been repeated, this field contains the repeat method.

The possible values are 'DIL' (repeat with request for dilution) or 'REP' (ordinary repeat).

Primary test indication (O.5.{n}.10)

When the query message is an aliquotter query (see field Q.14.1.9), this field contains the value 'PTT' (primary tube test) if it concerns a test for a parent specimen.

Informational test indication  (O.5.1.12)

This field has the value 'I' when the query specific tag 'SynchronizeResults' is available in the query message and the test concerns a synchronized procedure data result. See work order result synchronization message.

Urgency level (O.6.1.1)

This field contains the urgency level of a specimen. By default, all urgency levels are reported as 'S' (Stat) and routine specimens as 'R'.

Glims allows to map urgency levels by defining a list of 'urgency level codes' on the inquired station. See the Glims manual for more details.

Specimen size (O.10.1.5)

This field contains the specimen's size. The size is reported in the unit of the material, or alternatively, the specimen size unit of the inquired station.

This field is filled in when the:

  • the query specific tag 'SpecimenSize'  is available in the query message
  • the query specific tag 'TestConsumption' is available in the query message
  • the query message is an 'aliquotter' query (Q.14.1.9) and the query specific tag 'SpecimenData' is available in the query message

Total specimen distribution size (O.10.1.6)

This fields contains the total size required for the specimen when it is to be distributed in sub specimens. The total size is calculated when the specimen was queried in a specimen sort query message (Q.14.1.9).

The size is reported in the unit of the material, or alternatively, the specimen size unit of the inquired station.

This field is filled in when the:

  • the query specific tag 'SpecimenSize'  is available in the query message
  • the query specific tag 'TestConsumption' is available in the query message


Specimen dead distribution size (O.10.1.7)

This fields contains the dead distribution size required for the specimen when it is to be distributed in sub specimens. It is the required volume that should remain in a root specimen when a distributor system has to sort it into sub specimens. The size is calculated when the specimen was queried in a specimen sort query message (Q.14.1.9).

The size is reported in the unit of the material, or alternatively, the specimen size unit of the inquired station.

This field is filled in when the:

  • the query specific tag 'SpecimenSize'  is available in the query message
  • the query specific tag 'TestConsumption' is available in the query message

Action code (O.12.1.1)

If the ID in O.3.1.1 concerns a QC lot, this field contains the value 'Q'. If it concerns a specimen, it contains the value 'C' for any cancelled test, while in any other case it will be 'N'.

Special case: in case of a ONEORDER query reply, all non-QC tests (on the same station) with the same action code will be grouped together in one O-segment. This means that a ONEORDER reply can have 2 O-segments for tests on a specimen (on the same station): one for tests with action code 'N' and one for tests with 'C'.

Remote material ID (O.16.1.1)

This field contains the assessment method remote material id.

When this record contains multiple test ID's , this field contains the remote material id related to the first test.

Specimen source (O.16.1.2)

This field is filled in when the query specific tag 'SpecimenData' is available in the query message. By default, it contains the material variable choice mnemonic of the specimen variable. If multiple specimen variables are available, the one with the lowest material variable sequence number is taken.

The fields 'specimen variable' and 'source body site name' of the inquired station have influence on the content of this field. See the Glims manual for more details.

Specimen's material code (O.16.1.3)

This field is filled in when the query specific tag 'SpecimenData' is available in the query message. It contains the specimen's material mnemonic.

If the inquired station has a coding system with a code set on table material, the value for this field is retrieved from the code value. If no code value is defined, the field remains empty.

Specimen source extra info (O.16.1.4)

This field is filled in when the query specific tag 'SpecimenData' is available in the query message. By default, it contains the value of the specimen variable's extra info. If multiple specimen variables are available, the one with the lowest material variable choice sequence number is taken.

The fields 'specimen variable' and 'source body site name' of the inquired station have influence on the content of this field. See the Glims manual for more details.

Specimen's material short name (O.16.1.5)

This field is filled in when the query specific tag 'SpecimenData' is available in the query message. It contains the specimen's material short name.

Station department (O.25.1.3)

This field contains the code of the department responsible to execute the current test on, when queried for with the query specific tag TestDepartment is provided in the query.

Microbiology test station code (O.25.1.4)

This field refers to the station the microbiology test is currently assigned to. It contains the mnemonic of that station.

If the inquired station has a coding system with a code set on table station, the value for this field is retrieved from the code value. If no code value is defined, the field remains empty.

Report type (O.26.1.1)

By default, this field contains the value 'Q'. The field contains the value 'Z' when no tests are available for the work order.

Microbiology specimen indication (O.27.1.1)

This field contains the value 'MICRO' when the test order record concerns a microbiology test.

Parent or child specimen indication (O.27.1.2)

When the query message is an aliquotter query (see field Q.14.1.9) or sorting query (see same field Q.14.1.9), this field contains the value 'PT' (primary tube) if the specimen is a parent specimen. It contains the value 'ST' (secondary tube) when the specimen is a child specimen.

Pathology examination test indication (O.27.1.3)

This field contains the value 'PATH' when the test order record concerns a pathology examination test.

Sampling ward code (O.28.1.1)

This field is filled in when the query specific tag 'SamplingWard' is available in the query message. It contains the internal ID of an agent or issuer related to the specimen's order, on the condition that they are wards. The agent has precedence on the issuer.

The station's alternate order data function allows to put an alternate value in this field.

Order ward code (O.28.1.2)

This field is filled in when the query specific tag 'OrderWard' is available in the query message. It contains the order's encounter ward mnemonic.

The station's alternate order data function allows to put an alternate value in this field. Returning a ward code with this function overrules the search of a ward code as described above and does not require the 'OrderWard' tag.

Order ward room number (O.28.1.3)

This field is filled in when the query specific tag 'OrderWard' is available in the query message. It contains the stay's room number of the order's encounter ward.

The station's alternate order data function allows to put an alternate value in the order ward code field (see above). If an alternate ward code is returned, this field will be left empty.

Order encounter's type (O.28.1.4)

This field is filled in when the query specific tags 'OrderWard' and 'EncounterData' are available in the query message. It contains the type of the order's encounter. The type value is retrieved from the encounter type code field.

The station's alternate order data function allows to put an alternate value in the order ward code field (see above). If an alternate ward code is returned, this field will be left empty.

Order encounter's external ID (O.28.1.5)

This field is filled in when the query specific tags 'OrderWard' and 'EncounterData' are available in the query message. It contains the external ID of the order's encounter.

The station's alternate order data function allows to put an alternate value in the order ward code field (see above). If an alternate ward code is returned, this field will be left empty.

Order encounter's start time (O.28.1.6)

This field is filled in when the query specific tags 'OrderWard' and 'EncounterData' are available in the query message. It contains the start time of the order's encounter.

The station's alternate order data function allows to put an alternate value in the order ward code field (see above). If an alternate ward code is returned, this field will be left empty.

Order encounter's institution code (O.28.1.7)

This field is filled in when the query specific tags 'OrderWard' and 'EncounterData' are available in the query message. It contains the institution mnemonic of the order's encounter.

The station's alternate order data function allows to put an alternate value in the order ward code field (see above). If an alternate ward code is returned, this field will be left empty.
