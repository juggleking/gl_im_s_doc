Work order message - Sort request record (M)

Usage

A sort request record contains information Glims gathered when processing the specimen that received with the query message.

Element Descriptions

Received event code (M.4.1.2)

This field is send in a sort request record of type 'ReceivedEvent' and is a copy of the value received in field M.4.1.1 of an instrument event record transmitted in the query message.

Event code (M.4.1.2)

This field is send in a sort request record of type 'Event' and contains a code of the event Glims returns when processing the sort request information received with the query message.

Event codes can be defined in Glims by means of the station event code table.

Sorting zone code (M.4.1.2)

This field is send in a sort request record of type 'Zone' and contains the code of the zone Glims sorted the specimen to.

Sorting zone codes are defined in Glims in the sorting zone table. A sorting zone refers to the distributor station.

Sorting zone type (M.4.1.3)

This field is send in a sort request record of type 'Zone' and contains the sorting zone type as defined in the sorting zone table of Glims.

Possible values are: Routine,Error,Archive,External,Urgent'

Required volume (M.4.1.2)

This field is send in a sort request record of type 'RequiredVolume' and contains the volume required when sorting to a zone.

Required volume unit (M.4.1.3)

This field is send in a sort request record of type 'RequiredVolume' and contains the unit name of the volume required when sorting to a zone.

Added remaining volume  (M.4.1.2)

This field is send in a sort request record of type 'AddedVolume' and contains the amount of remaining volume from the primary queried specimen that has been added to the amount of required volume in a sort request of a secondary specimen.

Added remaining volume unit (M.4.1.3)

This field is send in a sort request record of type 'AddedVolume' and contains the unit applied with the added volume.

Decap flag  (M.4.1.1)

A sort request record of type 'Decap' indicates Glims decided that the tube is to be decapped.

Recap flag  (M.4.1.1)

A sort request record of type 'Recap' indicates Glims decided that the specimen container is to be Recapped. Field M.4.1.2 described below contains information about the closing cap.

Closing cap code (M.4.1.2)

In a sort request record of type 'Recap', this field contains the code of the cap that should close the specimen container.

By default, Glims uses the name field of the specimen container cap table to identify the cap.

Glims allows to map the container cap code by defining a code set for the specimen container cap table on the station's coding system. If the code set is available but the container cap code is not defined, this field will be left empty.

Centrifuge flag (M.4.1.1)

A sort request record of type 'Centrifuge' indicates Glims decided that the specimen is to be centrifuged.

Store remaining volume flag  (M.4.1.1)

A sort request record of type 'StoreRemainingVolume'' indicates Glims decided that the specimen should keep the remaining volume.

Dilution factor code (M.4.1.2)

This field is send in a sort request record of type 'Dilute' and contains the code of the factor with which the specimen is to be diluted according to Glims.

This value in this field is defined in Glims as the remote factor field in the dilution code table.

Dilution condition code (M.4.1.3)

This field is send in a sort request record of type 'Dilute' and contains the code of the condition with which the specimen is to be diluted according to Glims.

This value in this field is defined in Glims as the remote factor field in the dilution code table.
