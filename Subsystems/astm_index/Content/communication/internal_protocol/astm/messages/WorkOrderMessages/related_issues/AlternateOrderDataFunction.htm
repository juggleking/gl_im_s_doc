Fields filled with station's alternate order data function

Usage

The station's field 'alternate order data function' allows specifying an order-based MISPL function returning a string value. The string value should be a tagged list with a back slash as delimiter (\). Glims will use the tag values to fill dedicated fields in the ASTM work order message.

Note that returning a value for some of the tags will omit other fields (in order to prevent sending conflicting data).

Tag ASTM Field Default ASTM Field name Omitted fields
WARD1 P.26.1.1 Current ward code P.26.1.2, P.33.1.1, P.34.1.2, P.34.1.3
WARD2 O.28.1.1 Sampling ward code  
WARD3 O.28.1.2 Order ward code O.28.1.3, O.28.1.4, O.28.1.5, O.28.1.6, O.28.1.7
ISSUER O.17.1.1 Issuer code  

Example

STRING List;
STRING IssuerId;
STRING WardId;

IssuerId := .Issuer.Identification("ISRPROV");
WardId := "DEP100"; /* Default ward */

List := PutTag("Issuer", IssuerId, List, "\");
List := PutTag("Ward1", WardId, List, "\"); 

RETURN List;
