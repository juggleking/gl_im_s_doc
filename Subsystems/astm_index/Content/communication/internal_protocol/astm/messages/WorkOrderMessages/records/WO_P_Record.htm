Element Descriptions

Patient ID (P.3.1.1)

This field contains the patient ID. The Glims field patient identification of the inquired station determines the content of the field:

Station patient identification set to: Retrieved ID in Glims Comment
Person identification code The patient ID is the identification code of the person valid at the time the query message was received.

If in addition, the identification source is known by the service, Glims restricts the search for the identifications assigned by it. The identification source is supposed to be an institution.

 These station patient identification settings are only meaningful if the object is a person.

The identification source known by the service is the default PIN provider of the service batch user.
Order encounter external id The patient ID is the external ID of the order's encounter.
Current encounter external id The patient ID is the external ID of the person's encounter valid at the time the query message was received.

If in addition, the identification source is known by the service, and the identification source is a institution, Glims restricts the search for the encounters related to it.
Object internal id Object internal ID The setting 'Object internal id' guarantees a value for any type of object
National number The patient ID is the identification code of the person valid at the time the query message was received and assigned by the National Pin Provider (if defined in Glims)  
Object id Object ID The setting 'Object id' guarantees a value for any type of object

Second patient ID (P.4.1.1)

This field contains the object internal ID.

Third patient ID (P.5.1.1)

This field contains the patient's national number

Last name (P.6.1.1)

This content of this field depends of the object type. The table below shows the types and the corresponding last name field value:

Object type Last name field value Comment
Person Person's last name When Glims is configured to use combined names, this field is a composed value.
Lot Lot external ID  
QC Lot QC lot's material description  
Animal Animal's name  
Unknown - Field remains empty

Combined names

When Glims is configured to use combined names (the environment variable 'CombinedName' is set to 'yes'), Glims will try to compose the field on condition that:

  • The person's spouse last name is known
  • The person's sex is female

The field is composed as follows: person's spouse last name + '-' + person's last name.

First name (P.6.1.2)

This field contains the person's first name. It is omitted when the object is not a person.

Middle name (P.6.1.3)

This field contains the person's middle name. It is omitted when the object is not a person.

Title (P.6.1.5)

This field contains the person's title. It is omitted when the object is not a person.

Alternative name (P.7.1.1)

This field contains an alternative name for a person. It is omitted when the object is not a person.

Object template ID or name (P.15.1.1)

This field is filled when the query specific tag 'Species' is available in the query message. It contains the object's template (if available) ID or name (whatever is filled in).

Number of days pregnant (P.19.1.1)

Field returns the number of days the patient is pregnant.

Current encounter's type (P.25.1.1)

This field is filled when the query specific tag 'EncounterData' is available in the query message. It contains the type of the patient's (person's) encounter being valid at the time Glims received the query message. The type value is retrieved from the encounter type code field.

If in addition, the identification source is known by the service, and the identification source is a institution, Glims restricts the search for the encounters related to it.

The station's alternate order data function allows to put an alternate value in the current ward code field (see below). If an alternate ward code is returned, this field will be left empty.

Current ward code (P.26.1.1)

This field contains the ward's mnemonic the patient (person) currently stays (Currently is the time Glims received the query message).

The station's alternate order data function allows to put an alternate value in this field. The default behavior is then ignored.

Current ward room number (P.26.1.2)

This field contains the room number were the patient (person) currently stays (Currently is the time Glims received the query message).

The station's alternate order data function allows to put an alternate value in the current ward code field (see above). If an alternate ward code is returned, this field will be left empty.
