Element Descriptions

Query for patient (Q.3.1.1)

Glims will search for objects matching this patient ID. The Glims field patient identification of the inquired station determines the search path Glims uses:

Station patient identification set to: Search path in Glims Comment
Person identification code Glims searches for an identification with a matching code and valid at the time the query message was received.

If in addition, the identification source is known by the service, Glims restricts the search for the identifications assigned by it. The identification source is supposed to be an institution.

 The identification source known by the service is the default PIN provider of the service batch user.
Current encounter external id

or

Order encounter external id

 Glims searches for an encounter with a matching external ID and valid at the time the query message was received.

If in addition, the identification source is known by the service, and the identification source is a institution, Glims restricts the search for the encounters related to it.
Object internal id Glims searches for an object with a matching internal ID  
National number Glims searches for an identification assigned by the National PIN provider (if defined), with a matching code and valid at the time the query message was received.  
Object id Glims searches for an object with a matching ID (record ID)  

Query for specimen or QC Lot (Q.3.1.2)

When value is not 'ALL', Glims searches for a specimen or a QC Lot with a matching 'code' (in that order).

'Code' is either a specimen internal ID, specimen external ID, specimen identification code, QC lot code or QC material code.

ALL query (Q.3.1.2)

Glims searches for ALL specimens (Specimens, QC Lots when on a work list) that match the criteria of the query message and the inquired station.

Specimen location rack, position and type (Q.3.1.3, Q.3.1.4, Q.3.1.6)

These fields are used for sampling tracking in Glims. The location type field determines how Glims treats the data:

  • When specimen location type is a 'Q' (Query), rack and position are stored in the specimen home location (only the first time) and specimen last location
     
  • When specimen location type is a 'A' (Archive), rack and position are stored in the specimen last location and specimen storage location. Glims will also try to register the specimen in the specimen archive module.
  • When specimen location type is a  'H' (Home), rack and position are stored in the specimen home location and specimen last location.

With each specimen location instance, Glims will add an entry in the specimen audit trail (as a scan or archive event).

Specimen cup ID (Q.3.1.5)

Glims interprets this field as the cup position number on the instrument's carrousel, represented in Glims as the entry on a station's active work list.

Specimen material code (Q.3.1.7)

Specimen material code is used in Glims as additional information in the search for a specimen instance. When station's configurations allows it, the material code can also be used to create unsolicited specimens.

Query reply format (Q.4.1.1)

When this fields contains the value 'ONEORDER', Glims will group multiple tests in one test order record of a work order message.

The 'ONEORDER' tag is ignored in the following cases:

  • the query is an aliquotter (see Q.14.1.9) or a specimen sorting query (see Q.14.1.9)
  • cross match tests are requested (see Q.14.1.8)
  • result synchronization is requested (see query specific tag 'SynchronizeResults')

Query for a range of specimens (Q.4.1.2)

This fields defines the end in the specimen range query. The start ID is defined in Q.3.1.1.

Glims will search for specimens with a 'code' that is in the range of Q.3.1.1 and Q.4.1.2, borders inclusive.

Microbiology test (Q.5.1.4)

This field contains a test ID representing the microbiology procedure. Glims uses this field to find a grafting or processing property which is related to the microbiology action of the specimen being queried.

This field is only evaluated for a microbiologyone specimen query requesting for the next isolation.

Omitting this field and microbiology procedure code when requesting for a next isolation on a specimen results in Glims assuming that there is only one microbiology action for the specimen and will try to find it. Upon failure to find the correct microbiology action, Glims rejects the next isolation request.

Microbiology procedure code (Q.5.1.5)

This field contains the microbiology procedure mnemonic . If the inquired station has a coding system with a code set on table microbiology procedure, the value for this field is retrieved from the code value.

This field has precedence over the microbiology test field in the case where both are provided.

This field is only evaluated for a microbiologyone specimen query requesting for the next isolation.

Omitting this field and microbiology test when requesting for a next isolation on a specimen results in Glims assuming that there is only one microbiology action for the specimen and will try to find it. Upon failure to find the correct microbiology action, Glims rejects the next isolation request.

Organism code (Q.5.1.6)

This field contains the isolation organism code. It's only evaluated for a microbiologyone specimen query requesting for the next isolation.

Glims uses the microorganism table to find an organism with a mnemonic matching the code. If the organism is not found, the get next isolation request is rejected.

Glims allows to map the organism code by defining a code set for the micro organism table on the station's coding system. If the code set is available, but the organism code is not defined, the next isolation request is rejected.

Omitting this field when requesting for a next isolation on a specimen results in Glims using the "initial organism" that is configured on the found microbiology procedure (cfr. microbiology test). When none is configured, then the next isolation is request is rejected.

Query by work list name (Q.11.1.1)

Glims uses this field to search for specimens and QC lots on the 'current' work list. The value Glims expects in this field a fixed syntax, being the prefix "Current:" followed by the name of a work list.

The current work list is the latest work list with a matching name and not older than 1 hour.

Department related to the queried specimen (Q.12.1.3)

When this field is filled in and the query concerns a single specimen query, Glims will try to find a matching department.

  • If the department is not found, the specimen's availability will not be confirmed.
     
  • If the department is found, Glims verifies if work is planned on the specimen for that department. If so, the specimen's availability is confirmed.

If the inquired station has a coding system with a code set on table department, Glims will use the code set to find the department. If no code set is defined, the department is searched by mnemonic.

Query with station remote ID (Q.12.1.1)

By specifying a remote ID in a query for a station of type concentrator, Glims will restrict the search for specimens on the clients of the concentrator station to the one with a matching remote ID field.

No verification on download status (Q.14.1.1)

When this field contains the value 'NOCHECK', Glims will no verify if a test was sent in a work order download message earlier. It overrules the station's setting 'check download status' and is mainly for debugging purposes.

Only work order tests with matching remote material ID (Q.14.1.2)

Glims will restrict the download of tests in the work order message by those that have a matching remote material id field on the assessment method.

Additionally, Glims will add the value of the assessment method's remote material id field in work order message.

Add remote material ID in work order message (Q.14.1.2)

When this field contains the value 'RemoteMaterialId', Glims will add the value of the assessment method's remote material id field in work order message.

Add comment in work order message (Q.14.1.3)

When this field contains the value 'AddComment', Glims will add comment segments on the test order record of the work order message. The comments Glims will add are the request internal comments and the order descriptions.

Get previous results (Q.14.1.4)

If this field contains a positive integer value, Glims will add historical results for each test it will send in the work order message.

The assessment method's field 'send previous results' determines if sending historical result is applicable for the test and the integer value in Q.14.1.4 indicates how many results Glims should retrieve.

Add procedure data results in the work order message (Q.14.1.5)

When this field contains the value 'GetProcedureData', Glims will add results related to the procedure's test in the work order message. The relation with the test is defined by instances of procedure data for which the history index value equals zero.

Query for microbiology specimens (Q.14.1.6)

When this field contains the value 'Micro', Glims will add microbiology related information in the work order message.

Get entries from a report work list (Q.14.1.7)

When this field contains the value 'REPORT', Glims will use the station's report work list settings to search specimens on work lists. The report work list selection can be used to report micro-organisms from closed and positive microbiology actions.

Query for cross match tests (Q.14.1.8)

When this field contains the value 'BTMCrossMatch', Glims will add cross match test related information (especially blood bags) in the work order message. Requesting for cross match tests also implies that tests are not grouped in one test order record.

Query for parent and child specimens (Q.14.1.9)

When this field contains the value 'Aliquotter', Glims will search for parent and child specimens. Both parent and child specimen are sent in the work order message. If the field contains the value 'PT', Glims will do the same search but will send all tests with the parent specimen, child specimens are not in the message.

Get sorting information for the queried specimen  (Q.14.1.9)

When this field contains 'Sort', 'SortPT' and 'SortST', indicating that specimen sorting information is requested, and no test id information. With 'SortPT' and 'SortST' we also indicate that the specimen being queried is seen by the distributor system as a root specimen and sub specimen respectively.

Reporting error information in work order message (Q.14.1.10)

When this field contains 'REPLYERROR', Glims will report errors occurring during the search for data. The error code and message is reported in a comment segment of the work order message.

Query for pathology specimens (Q.14.1.11)

When this field contains the value 'Path', Glims will add pathology related information in the work order message.

Query for the next isolation of a specimen (Q.14.1.12)

When this field contains the value 'NEXTISOLATION', Glims will create a new isolation on the provided specimen..

Query specific tags (Q.15.{n}.1)

This repeated field may contain a list of tags specifying which additional information is to be sent in the work order message.  See here for a list of query specific tags.
