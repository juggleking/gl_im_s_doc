Query specific tags

Usage

Query specific tags can be added in the request information record of a query message. They are used to send additional information with the work order message. Below is listed the tags and there purpose.

Tags

DaysPregnant

Send number of days the patient is pregnant in field P.19.1.1.

SamplingWard

Send a ward code in O.28.1.1

OrderWard

Send the order's encounter ward mnemonic in O.28.1.2 and the related stay's room number in O.28.1.3

EncounterData

Send extra encounter related data. The extra data is for the current encounter (fields P.25.1.1, P.33.1.1, P.34.1.2, P.34.1.3) and the order's encounter (fields O.28.1.4, O.28.1.5, O.28.1.6, O.28.1.7)

SpecimenData

Send specimen related data fields: O.16.1.2, O.16.1.3, O.16.1.4 and O.16.1.5.

Additionally, when the inquired station field 'specimen variables' is set to 'ALL', send all specimen variables in specimen variable records.

SpecimenSize

Send the specimen size and related fields: size in O.10.1.5, minimal size in O.10.1.1, size unit name in O.10.1.2, maximal size in O.10.1.4.

SpecimenComment

Send the specimen external comment in O.14.1.1.

ObjectComment

Send the specimen's object comment in a patient information comment record.

SpecimenDiscriminator

Send the specimen discriminator comment in O.16.1.6.

Species

Send the object's template (if available) external ID or name in P.15.1.1

IssuerName

Send the order's issuer correspondent data: name in O.17.1.2, phone in O.17.1.3, address in O.17.1.4, municipality in O.17.1.5 and postal code in O.17.1.6.

ArchiveData

Send an 'archive' result in an archive result record.

OrderDescription

Send the order description in a test order comment record.

OrderQuestion

Send the order question in a test order comment record.

OrderReportInfo

Send the order report info in a test order comment record.

OrderIssuerInfo

Send the order issuer info in a test order comment record.

OrderPhone

Send the order phone in a test order comment record.

OrderExternalId

Send the order external ID in O.19.1.2.

StudyData

Send order study data: study mnemonic in O.19.1.3, study name in O.19.1.4.

PatientRelations

Send patient's relations in person relation records.

TestConsumption

Send the property output consumption in O.5.{n}.8. Also send the fields related to the 'SpecimenSize' tag (see above).

FinishedTests

Send tests of confirmed or validated results in the test order record. In addition, the station for which the results are scheduled should have the field 'download finished tests' enabled.

CancelledTests

Send tests of discontinued results in the test order record. In addition, the station for which the results are scheduled should have the field 'download finished tests' enabled.

TestStatus

Send test status information in O.5.{n}.11

TestSelectionUpdate

Send tests for specimens that have at least one test downloaded earlier to instrument.

TestDepartment

Send the department responsible to execute a test in the work order message field O.25.1.3 Station Department.

NOTE: This tag overrules the Query reply format (Q.4.1.1) ONEORDER-tag in case of any incompatibility is encountered. Meaning that not all tests of an order will necessarily have the same station department.

LabelInfo

Send specimen label information in specimen label records. See the Glims manual for more details.

SynchronizeResults

Send result synchronization work orders. See the Glims manual for more details.

ResultStatusInfo or StatusInfo

Send information concerning the status of:

- a synchronized (procedure data) result in a result status info record

- a microbiology test in a microbiology test status info record

OldResultTimeMode

Send result start (R.12.1.1) and end time (R.13.1.1) values in a backward compatible mode.

Applicable for any work order related result record. In old mode, the values are the availability time and confirmation time. When old result time mode is not set, the values are the object time and availability time.

MicroIsolWithExam

Send the microbiology procedure in O.5.{n}.6 when isolations are sent in the work order message.

Carriers

Send carriers related to specimens or isolations in carrier records, on the condition that the query is requesting microbiology specimens (see Q.14.1.6).

DonorInfo

Send information of the blood bag's donor in donor info records.

CarrierWithIsolation

Send carrier information related to isolation record preceding it.

IsolationTests

Send isolation tests when there was a request for microbiology carriers (See tag specific 'Carriers'). For each carrier in the work order message, send the isolations and isolation tests related to the carrier

CompleteSelection

When a test is found which is not sent earlier, send all the tests related to that specimen regardless if they were sent earlier.

This field overrules the Glims station field 'send complete selection'.

In case Glims is configured for microbiology test synchronization and the query message is requesting synchronization, this specific tag assures that data for which no synchronization event was triggered is sent along with a microbiology test for which Glims detected at least one synchronization event.

SelectAll

In a query for a single specimen, send all tests regardless if they were sent earlier.

SynchronizeNew

In case Glims is configured for microbiology test synchronization and the query message is requesting synchronization, then Glims only synchronizes tests which have been sent earlier. With the this query specific tag, Glims will also synchronize microbiology test which were not sent earlier.

PathologyCarrierTests

Send pathology carrier tests along with the pathology carriers of a work order message, , on the condition that the query is requesting pathology examinations (see Q.14.1.11).
