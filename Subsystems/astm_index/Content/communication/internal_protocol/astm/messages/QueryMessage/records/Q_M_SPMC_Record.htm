Query message - Specimen characteristics record (M)

Usage

These records contain information about the characteristics of a specimen and it's container when it was scanned and inspected on the sorting system. It is received by Glims in a query message.

Element Descriptions

Date time of processing (M.4.1.2)

This field contains the date and time the specimen was scanned and inspected in order to determine the characteristics of the specimen and it's container.

The processing time is accessible in Glims by the event evaluation function on the station.

Container type code (M.4.1.2)

This field contains the code of the specimen's container type.

Glims will search for a specimen container type with a name matching the code. When found, Glims will verify the container type with the one defined on the specimen's material. Please refer to the Glims manual for more details.

Glims allows to map the container type code by defining a code set for the specimen container type table on the station's coding system. If the code set is available but the container type code is not defined, the specimen container type is considered as unknown.

The container type code is accessible in Glims by the event evaluation function on the station.

Container type name (M.4.1.3)

This field contains the name of the specimen's container type.

The container type name is accessible in Glims by the event evaluation function on the station.

Container height (M.4.1.2)

This field contains the height of the specimen's container type.

The container type height is accessible in Glims by the event evaluation function on the station.

Container height unit (M.4.1.3)

This field contains the height unit of the specimen's container type.

The container type height unit is accessible in Glims by the event evaluation function on the station.

Container width (M.4.1.2)

This field contains the width of the specimen's container type.

The container width is accessible in Glims by the event evaluation function on the station.

Container width unit (M.4.1.3)

This field contains the width unit of the specimen's container type.

The container width unit is accessible in Glims by the event evaluation function on the station.

Container cap code (M.4.1.2)

This field contains the code of the specimen's container cap.

Glims will search for a specimen container cap with a name matching the code. When found, Glims will verify the container cap with the one defined on the specimen's material. Please refer to the Glims manual for more details.

Glims allows to map the container cap code by defining a code set for the specimen container cap table on the station's coding system. If the code set is available but the container cap code is not defined, the specimen container type is considered as unknown.

The container cap code is accessible in Glims by the event evaluation function on the station.

Container cap name (M.4.1.3)

This field contains the name of the specimen's container cap.

The container cap name is accessible in Glims by the event evaluation function on the station.

Specimen's measured volume (M.4.1.2)

This field contains the volume of the specimen.

In Glims, the value is stored in the specimen measured size field,  on condition that field M.4.1.3 contains a valid unit code.

The specimen's measured volume is accessible in Glims by the event evaluation function on the station.

Specimen's measured volume unit (M.4.1.3)

This field contains the volume unit of the specimen.

Glims searches in a unit with a name that matches the code. If no unit is found, the measured volume is ignored.

The unit is used to convert the volume in M.4.1.2 to the unit of the specimen's material. The converted value is stored in the specimen measured size field.

Glims allows to map the unit code by defining a code set for unit table on the station's coding system. If the code set is available but the unit code is not defined, the remaining volume is ignored.

The specimen's measured volume unit is accessible in Glims by the event evaluation function on the station.

Specimen quality code (M.4.1.2)

This field contains a code representing a quality aspect of a specimen. Glims stores the code in the quality info field of the specimen.

The specimen quality code is accessible in Glims by the event evaluation function on the station.

Specimen quality name (M.4.1.3)

This field contains a the name of a quality aspect of a specimen. Typical quality aspects are: 'good', 'haemolytic', 'lipaemic', 'icteric', ...

The specimen quality name is accessible in Glims by the event evaluation function on the station.
