Element Descriptions

Maximal distribution count value (M.5.1.1)

The maximal distribution count is used to specify the maximal number of sub specimens a distributor system can handle when being distributed from the queried root specimen. It can be specified in specimen sort query message (See Q.14.1.9).

Sampling offset time (M.5.1.1)

This field is used in Glims in for selecting pathology examinations (See Q.14.1.11) based on the sampling time of the examination. Only examinations with a sampling time starting from 'Sampling offset time' are selected.

Minimal pathology carrier status (M.5.1.1)

This field is used in Glims in for selecting pathology carriers (See Q.14.1.11) based on the status of the carrier.

Possible status values are listed in the table below:

Pathology carrier status Description Comment
'Initial' The carrier has not been processed. This implies the grafting time of the carrier record is unknown
'Processed' The carrier has not been processed. This implies the grafting time of the carrier record is known
'Deleted' The carrier has been deleted Not used

Carrier print status (M.5.1.1)

This field is used in Glims in for selecting pathology carriers (See Q.14.1.11) based on the print status of the carrier.

Possible status values are listed in the table below:

Pathology print carrier status Description Comment
'All' Select pathology carriers regardless if the carrier label was printed.  
'Printed' Select a pathology carrier for which labels have been printed  
'NotPrinted' Select a pathology carrier for which no labels have been printed  

Pathology carrier medium type (M.5.1.1)

This field is used in Glims in for selecting pathology carriers (See Q.14.1.11) based on the type of carrier medium

Possible status values are listed in the table below:

Pathology carrier medium type Description Comment
'Block' Select pathology carriers for which the medium is a block  
'Slide' Select a pathology carrier for which the medium is a slide  

Note that multiple instances of this type of query argument record allows to select both types.

Pathology carrier ID code (M.5.1.1)

This field is used in Glims in for selecting pathology carriers (See Q.14.1.11) based on the internal ID of the carrier.

Note that multiple instances of this type of query argument record allows to select multiple carriers with the same internal id.

Pathology carrier creation  time offset (M.5.1.1)

This field is used in Glims in for selecting carriers of a pathology examination (See Q.14.1.11) based on the creation time of the carriers. Only carriers with a creation time starting from the offset time are selected.

Secondary tube order (M.5.1.1)

This field can be specified in a specimen sort query message (See Q.14.1.9).

The secondary tube order is used to specify the order in which sub specimens should be transmitted in the work order message. Possible values are:

Secondary tube order Description Comment
'ByTarget' Secondary tubes are transmitted in order of obtained sorting. The logical order is:

- First, the sequence number of the specimen sorting target (an unknown sequence number comes last)
- Second, the sequence number of the sorting zone  (an unknown sequence number comes last)
- Third, the sorting zone code

Note that this order is the default sorting order (when this query argument is not specified)
'BySampleId' Secondary tubes are transmitted in order of the code representing the specimen id The specimen id code is available in field O.3.1.1 of the test order record.

Query specimen container cap control (M.5.1.1)

This logical field can be specified in a specimen sort query message (See Q.14.1.9).

By default, Glims will use the specimen container cap information in a sort query message to determine if a work order sort request message may contain sort request 'decap' and 'recap' records.  Setting the logical value to false prevents Glims from doing this verification.
