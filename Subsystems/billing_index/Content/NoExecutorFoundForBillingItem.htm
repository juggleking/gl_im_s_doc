No Executor found for Billing item

Verifying the log

Information found in the error:

  • Order set label (1)
  • Object external ID (2)
  • Billing code (3)
  • Price list (4)
  • Lab and Department where the Request linked to the Billing item (Billing code) was executed (5)

This kind of error can have different causes. Here, we list the most common causes that you can check.

Getting the list of Provisions that are available for the Executing class

Navigate to the Billing code. On the Billing code the Tariffs can be viewed.

In order to know which Tariff is picked up, the Price list (4) has to be checked in the error. In the example, this is Price list (4) RIZ_MA.

From the specific Tariff, navigate to the Executing class.

 

On the Executing class, the Provisions can be viewed.

By using the Provisions, it is possible to link an Executor to a specific Executing class.

By filling in the Department and/or Lab on the Provision, the Executor can also be limited to a specific Department or Lab.

Checking the Provisions

A Provision is missing

Example:

In the error there can be found:

  • Billing code (3): 127050
  • Price list (4): RIZ_MA
  • Lab and Department where the Request linked to the Billing item (Billing code) was executed (5): L_MOL_HH (lab) - D_MOL_HH (department)

The Billing code 127050 in combination with the Price list RIZ_MA gives the used Tariff:

On the Tariff the Executing class can be found. On the Executing class the Provisions can be viewed.

The Provision needed, is one for Executing class Niet_RIA, Executing Lab L_MOLL_HH and Executing Department D_MOLL_HH.

If the Provision isn't found in the list, a new Provision needs to be added.

No Provision is missing

The specific Provision that should be picked up, needs to be checked.

1. Check if the Order (LowestObjectTime) still falls within the Valid from - Valid until period. If a Provision is Valid from 01/06/2011 00:00:00, then it won't be applicable for an Order placed on 01/05/2011.

2. Check if the Executor filled in on the Provision is active.

3. Check if you can find an Executing class on the Tariff that is picked up. Without an Executing class on the Tariff, the application doesn't know which Executor may be picked up.

4. On the Tariff you can find the Executing class for which the Tariff can be used. On the assignment of the Billing code to the Request definition, you can also find the Executing class. Normally, the Executing class isn't filled in on the assignment, only in specific cases. If the Executing class on the Billing code assignment isn't filled in, there is no need for adjustments. If the Executing class is filled in on the Billing code assignment, it needs to correspond to the one filled in on the Tariff in order for the Tariff to be available for this specific Billing code.

Checking the Provisions when the Billing Code is assigned to a Material

When a Billing code is linked to a Material instead of a normal Request, the Executor is picked based on the Procedure that uses this Material as input.

On the Procedure, the Department can be set. The Provision is only applicable if it contains this specific Department.
