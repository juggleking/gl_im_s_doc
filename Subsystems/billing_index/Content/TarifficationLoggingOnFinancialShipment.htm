Tariffication logging on Financial shipment

A basic configuration consists of 5 tariffication commands.

TAR 0: Checking the Order set label

  • Function: ReEvaluateOrderSetDefaults
  • Checks the Order set label and adjusts it when needed
  • Linked to a Task, set for running daily

TAR 1: Tariffication of the Order set

  • Function: Tariff
  • Sets the Order set status to Fixed and generates the Invoices

TAR 2: Creation of Invoice summary

  • Function: InvoiceSummaryCreation
  • Sets the status of the Invoices to Fixed and generates the Invoice summaries

TAR 3: Creation of Financial shipment

  • Function: PlaceOnFinancialShipment
  • Sets the status of the Invoice summaries to Fixed and generates the Financial shipment

TAR 4: Sending of Financial Shipment

  • Function: Send
  • Sets the status of the Financial shipment to Fixed

It's in this final step, when the TAR 4 command is run, that GLIMS will do a check of the Financial shipment and the Invoices that it contains.

When all requirements are met, GLIMS will execute the TAR 4 command.

As a result of this, the Financial shipment will be promoted to status Fixed and the Shipment date on the Financial shipment will be filled in.

When there are flaws in the Financial shipment (incorrect statuses, missing elements,...), it won't be sent by the command.

As a result of this, the Financial shipment will stay in its original status (Numbered or Initial) and the Shipment date won't be filled in.

In this case, the logging on the Financial shipment can give you an indication of why the command wasn't carried out.

REMARK

In order for GLIMS to check the Financial shipment and save the logging, the Log type Tariffication needs to be enabled. The can be done via: Start - System management - Logging - Log types.

Checking the Tariffication logging starting from the Financial shipment

On the Financial shipment, the logging can be checked by clicking on Audit - Log entries.

This will give a screen that will display all the log entries on that Financial shipment. Using 'F6' the content of each line is displayed.

Checking the Tariffication logging starting from the Start menu

Using the general menu, navigate to Log types.

In the list, the Log type Tariffication, can be found. Opening the Logs of this Log type will give the logging of those Financial shipments that haven't been sent.

By clicking on Verify, each logging can be checked.
