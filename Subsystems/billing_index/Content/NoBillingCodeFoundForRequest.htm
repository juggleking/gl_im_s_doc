No Billing code found for Request

Verifying the log

Information found in the error:

  • Order set label (1)
  • Object external ID (2)
  • Order internal ID (3)
  • Request definition of Property, Panel, Material, Microbiology procedure,... for which the billing code is missing (4)

This kind of error can have different causes. Here, we list the most common causes that you can check.

Checking the configuration of a Property

Navigate to the Request definition of the Property and open its configuration editor.

Here you should find that To charge is checked.

If so, there should be at least one billing code present in the Assigned billing codes.

If this is not the case, a Billing code needs to be assigned.

To assign a Billing code, use the 'INSERT' key in the table of the Assigned billing codes.

Information to be filled in whilst assigning a Billing code :

  • The Billing code (1) that needs to be linked to the Request definition.
  • 'Valid from...' (2) to set the start date for the assignment of the Billing code. The code may be picked up for this Request definition as of this date.
  • 'Valid until...' (2) to set the end date for the assignment of the Billing code. The code may be picked up for this Request definition until this date.
  • The Executing class (3) which can be found on the Tariff of the Billing code that is chosen.

The Executing class can be found on the Tariff of the Billing code that is chosen.

REMARK

One should always check if Eligible count is filled in on the Billing code assignment. If so, it is always worth checking out the MISPL that is used. It can be possible that this MISPL returns a "0" at one point. As a result, the Billing Code won't be charged. In order to charge a billing code when using the Eligible count, the MISPL should return at least a "1".

Checking the configuration of a Panel

Navigate to the Request definition of the Panel and open its configuration editor.

Here, you should find that To charge is checked.

If To charge is checked, the billing configuration can be done on different levels, depending on the configuration of the field Charge (1).

This field can have different values: Members only (a), Panel only (b), Panel and members (c), Members or Panel (d). Depending on this choice, the assignment of the Billing codes needs to be done on panel-level (b), member-level (a) or both (c) (d).

a) Charge: Members only

Right click on the panel and choose As Panel requestable - Show all... - Panel members to view all the linked Properties on the Panel.

Check the configuration of each Panel member's Request definition.

If To charge is set on the Request definition of the panel member, there should be at least one Billing code that is assigned.

If To charge is not set on the Request definition of the panel member, then this test isn't charged and no Billing codes need to be present.

To assign a Billing code, use the 'INSERT' key in the table of Assigned billing codes.

Information to be filled in whilst assigning a Billing code :

  • The Billing code (1) that needs to be linked to the Request definition.
  • 'Valid from...' (2) to set the start date for the assignment of the Billing code. The code may be picked up for this Request definition as of this date.
  • 'Valid until...' (2) to set the end date for the assignment of the Billing code. The code may be picked up for this Request definition until this date.
  • The Executing class (3) which can be found on the Tariff of the Billing code that is chosen.

REMARK

1) One should always check if Condition(s) is filled in on the Billing code assignment. If so, it is always worth checking out the MISPL that is used. It can be possible that this MISPL returns a "NO" at one point. As a result, the Billing Code won't be picked up and can't be charged. In order to charge a billing code when using the Condition(s), the MISPL should return "TRUE".

2) One should always check if Eligible count is filled in on the Billing code assignment. If so, it is always worth checking out the MISPL that is used. It can be possible that this MISPL returns a "0" at one point. As a result, the Billing Code won't be charged. In order to charge a billing code when using the Eligible count, the MISPL should return at least a "1".

b) Charge: Panel only

Check the configuration of the Request definition of the Panel.

If To charge is set on the Request definition of the Panel, there should be at least one Billing code that is assigned.

If To charge is not set on the Request definition of the panel member, then this test isn't charged and no Billing codes need to be present.

To assign a Billing code, use the 'INSERT' key in the table of Assigned billing codes.

Here, there need to be Billing codes assigned on a panel level and on a member level (combo (a) + (b)
