No Tariff found for Billing item

Verifying the log

Information found in the error:

  • Order set label (1)
  • Object external ID (2)
  • Billing code (3)
  • Lab, Department and Date of request that is linked to the Billing code (4)
  • Price list (5)

This kind of error can have different causes. Here, we list the most common causes that you can check.

Checking the Tariffs on the Billing Code

Are all needed Tariffs configured for the Billing code?

The Tariffs need to be checked on the Billing code that is given in the error.

This gives a list of the Tariffs available for the Billing code.

Depending on the Encounter Type of the Order and if the Order is registered as 'Subcontracting' (Agent filled in on Order: yes/no), a specific Tariff is picked up.

The difference between these Tariffs lies in the fact that different Price Lists and Price codes are used.

There should be a Tariff present for each of the following situations.

1) Encounter type = Ambulatory - Subcontracting = no

  • Price list: RIZ_MA
  • Price code: MA**

2) Encounter type = Ambulatory - Subcontracting = yes

  • Price list: RIZ_SA
  • Price code: SA**

3) Encounter type = Hospitalized - Subcontracting = no

  • Price list: RIZ_MH
  • Price code: MH**

4) Encounter type = Hospitalized - Subcontracting = yes

  • Price list: RIZ_SH
  • Price code: SH**

Example:

We are dealing wit an ambulatory Order without 'Subcontracting'. This means we will need a Tariff with a Price code from the Price list RIS_MA.

Looking at the list of Tariffs on the Billing code, we won't find a Tariff with this Price list, so this Tariff needs to be created.

In this case, we have other Tariffs where the Price code is filled in. Looking at them, we know we need to pick the MA06 as the Price code for the new Tariff.

REMARK:

1) Make sure To charge is set on the Tariff .

Does the Order LowestObjectTime fall within the limits of the Validity of the Tariff?

If the Order LowestObjectTime doesn't fall within those boundaries, the Tariff shall not be available.

For this example, the 'Until' isn't filled in which means that the Tariff is available from 01/01/1995 for an unlimited time.

2) The Nomenclature code field is filled in by the official codes that are provided by the RIZIV. The codes differ depending on the Encounter type (Ambulatory/Hospitalized).

Does the Executing class configured on the Tariff correspond to the Executing class configured on the Billing code assignment?

On Tariff level, you can find the Executing class for which the Tariff can be used.

On the assignment of the Billing code to the Request definition, you can also find the Executing class.

Normally the Executing class isn't filled in on the assignment, only in specific cases.

If the Executing class on the Billing code assignment isn't filled in, there is no need for adjustments.

If the Executing class is filled in on the Billing code assignment, this needs to correspond to the one filled in on the Tariff in order for the Tariff to be available for this specific Billing code.
