Terminal servers: Lantronix

Lantronix

Setting up the hardware

When you set up the terminal server for the first time, it does not have an IP address yet, so you can't connect to it using the network. A serial connection must be made with the terminal adapter to configure the network parameters.

  1. You should connect the terminal server to a serial port (a.k.a. COM-port, RS-232 interface) of a PC or laptop to assign an IP number to it. The cable to do this is shipped with the terminal server. At the side of the terminal server, you should plug it in port 1, at the side of the computer, you should plug it in one of the serial ports.
  2. Connect the terminal server to the network, by plugging in a standard network cable into the network connector (don't confuse this with the serial ports!)
  3. Turn on the power of the terminal server.
  4. Start HyperTerminal. This is a serial terminal emulation program, shipped with any Windows NT. You can find this in the Start-menu -> Programs -> Accessories -> Hyperterminal.
  5. Answer "No" when asked to configure a modem
  6. Give your session a name (e.g. "Terminal Server") and hit OK.
  7. Change the "Connect using" drop-down box to the serial port you have plugged the terminal server into (COM1/COM2) and hit OK.
  8. Enter the following settings, and hit the OK button.
    • Bits per second: 9600
    • Data bits: 8
    • Parity: None
    • Stop bits: 1
    • Flow control: none
  9. hit the enter key
  10. A Username> prompt will be displayed. After you enter a username, you will get the Local> prompt.
  11. Type SET PRIVILEGED (+ enter)
  12. Type system (not echoed)
  13. Type DEFINE SERVER IPADDRESS nnn.nnn.nnn.nnn (substitute nnn.nnn.nnn.nnn with a unique IP number e.g. 192.168.1.204. This number should be assigned according to the network numbering scheme in the lab/hospital.)
  14. Type logout
  15. Close HyperTerminal. You can save your current session for later reconfiguration. You can answer yes when asked to disconnect.
  16. Now the initial terminal server setup is finished. The rest of the configuration can be done using a telnet connection now. The serial cable can be disconnected now if necessary.

Configuring ports

Analyzer communication

For each analyzer to be connected to the terminal server, you will have to configure a port x for it.  (substitute x with the port number on the terminal server the analyzer is connected to)

  1. Start telnet from any workstation in the network to the terminal server. You can do this by going to the Start-menu -> Run..., and type telnet + OK.
  2. Go to the Connect -> Remote System menu item
  3. Fill in the following parameters and hit the Connect button
    • Host Name: The IP number of your terminal server
    • Port: telnet
    • TermType: vt100
  4. Hit enter once
  5. Type set priv (password: system)
  6. Type show port x
  7. Type define port x access remote
  8. Type define port x autoprompt disabled
  9. Type define port x broadcast disabled
  10. Type define port x loss disabled
  11. Type define port x verif disabled
  12. Type define port x remote disabled
  13. Type define port x telnet pad disabled
  14. Type define port x flow control none
  15. Type define port x autostart disabled 
  16. Type define port x break disabled
  17. Type define port x inactivity logout disabled
  18. Have a look at the communication parameters of the analyzer. You should take note of the port speed, the number of data bits, stop bits and the parity. Now enter the same settings in the terminal server:
  19. Type define port x speed speed (substitute speed by the value you have seen in the analyzer setting; this should be 57600 or less)
  20. Type define port x character databits (substitute databits by the value you have seen in the analyzer setting; this should be 7 or 8)
  21. Type define port x parity parity (substitute parity by the value you have seen in the analyzer setting; this can be none (usually), odd or even)
  22. logout port x
  23. logout
  24. Close telnet.
