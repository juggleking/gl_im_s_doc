Terminal servers: Annex

Nortel Networks (Bay Networks in earlier days).

Setting up the hardware

When you set up the terminal server for the first time, it does not have an IP address yet, so you can't connect to it using the network. A serial connection must be made with the terminal adapter to configure the network parameters.

  1. You should connect the terminal server to a serial port (a.k.a. COM-port, RS-232 interface) of a PC or laptop to assign an IP number to it. The cable to do this is shipped with the terminal server. At the side of the terminal server, you should plug it in port 1, at the side of the computer, you should plug it in one of the serial ports, using the shipped adapter.
  2. Connect the terminal server to the network, by plugging in a standard network cable into the network connector (don't confuse this with the serial ports!)
  3. Turn on the power of the terminal server.
  4. Press the Test-button on the front panel of the terminal server within 5 seconds after powering up the Annex. This will bring the annex in configuration mode.
    You can also enter configuration mode later, when the Annex is already running. Be careful with this, because all active connections will be disconnected
    • Hold down the test button until the Power-LED blinks rapidly
    • Release the test button
    • Press it again within 5 seconds
  5. Wait until the LEDs Power, Unit and Net are green. The Attn LED should be amber. This might take a few seconds. The Active led might flash from time to time.
  6. Start HyperTerminal. This is a serial terminal emulation program, shipped with any Windows NT. You can find this in the Start-menu -> Programs -> Accessories -> Hyperterminal.
  7. Answer "No" when asked to configure a modem
  8. Give your session a name (e.g. "Terminal Server") and hit OK.
  9. Change the "Connect using" drop-down box to the serial port you have plugged the terminal server into (COM1/COM2) and hit OK.
  10. Enter the following settings, and hit the OK button.
    • Bits per second: 9600
    • Data bits: 8
    • Parity: None
    • Stop bits: 1
    • Flow control: none
  11. You should see the "monitor::" prompt now. If not, hit the enter key a few times.
  12. Type addr (+ enter)
  13. Fill in the following parameters, and hit enter after each parameter:
  14. Enter Internet address: Give it a unique IP number (e.g. 192.168.1.204). This number should be assigned according to the network numbering scheme in the lab/hospital.
  15. Enter Subnet mask: This number will also depend of the network configuration in the lab/hospital. In many cases this will be 255.255.255.0
  16. Take the default value (just hit enter) for all other parameters.
  17. Type boot (+ enter)
  18. Wait until the Load LED is out.
  19. The LEDs Power, Unit and Net should be green now. The Active led might flash from time to time. All other LEDs should be out. If the Test LED is still amber, you can press the Test button to turn it off.
  20. You should see the "annex:" prompt now. If not, hit the enter key a few times.
  21. Type su (+ enter)
  22. The annex asks for a password. Enter the IP number you have entered in step 12 (+ enter).
    1. Hint: access
  23. You should see the "annex#" prompt now.
  24. Type passwd (+ enter)
  25. Enter the IP number again (old password) (+ enter)
  26. Choose a new password and give it in twice (+ enter).
  27. Close HyperTerminal. You can save your current session for later reconfiguration. You can answer yes when asked to disconnect.
  28. Now the initial terminal server setup is finished. The rest of the configuration can be done using a telnet connection now. The serial cable can be disconnected now if necessary.

Configuring ports

Analyzer communication

For each analyzer to be connected to the terminal server, you will have to configure a port for it.

  1. Connect the analyzer using the appropriate cable (DTE Crossover Terminal Cable). You can get this cable from Nortel Networks. If you want to make one yourself, you can use the wiring scheme in the Micro Annex Hardware Installation Guide, page B-2. Plug in the cable in any port of the terminal server, and connect it to the serial interface of the analyzer.
  2. Start telnet from any workstation in the network to the terminal server. You can do this by going to the Start-menu -> Run..., and type telnet + OK.
  3. Go to the Connect -> Remote System menu item
  4. Fill in the following parameters and hit the Connect button
    • Host Name: The IP number of your terminal server
    • Port: telnet
    • TermType: vt100
  5. Hit enter once
  6. You should see "Enter Annex port name or number:" now.
  7. Type cli (+ enter)
  8. You should see the "annex:" prompt now.
  9. Type su (+ enter)
  10. Enter the password you have chosen during the hardware installation (+ enter)
    1. Hint: access
  11. You should see the "annex#" prompt now.
  12. Type admin (+ enter)
  13. You should see the "admin :" prompt now.
  14. Give in the following configuration commands. Each command should be followed by enter:
  15. Type set port=x mode slave (substitute x with the port number on the terminal server the analyzer is connected to)
  16. Ignore all messages saying that the port or annex must be rebooted
  17. Type set port=x control_lines none
  18. Type set port=x input_flow_control none
  19. Type set port=x output_flow_control none
  20. Have a look at the communication parameters of the analyzer. You should take note of the port speed, the number of data bits, stop bits and the parity. Now enter the same settings in the terminal server:
  21. Type set port=x speed speed (substitute speed by the value you have seen in the analyzer setting; this should be 57600 or less)
  22. Type set port=x data_bits databits (substitute databits by the value you have seen in the analyzer setting; this should be 7 or 8)
  23. Type set port=x stop_bits stopbits (substitute stopbits by the value you have seen in the analyzer setting; usually this is 1)
  24. Type set port=x parity parity (substitute parity by the value you have seen in the analyzer setting; this can be none (usually), odd or even)
  25. Type reset port=x or reset x
  26. You can configure more ports now, by repeating steps 14 to 17, with a different x.
  27. Type quit
  28. You should see the "annex#" prompt now.
  29. Close telnet.
