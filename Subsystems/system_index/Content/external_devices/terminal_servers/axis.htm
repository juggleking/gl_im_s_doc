Setting up the hardware

When you set up the terminal server for the first time, it does not have an IP address yet, so you can't connect to it using the network. A serial connection must be made with the terminal adapter to configure the network parameters.

  1. Connect the terminal server to the network, by plugging in a standard network cable into the network connector.
  2. Using a standard DB-9 connector, connect the analyzer to the COM1 port.
  3. Turn on the power of the terminal server.
  4. Connect the axis power supply.
  5. Acquire an unused Internet Address from your Network Administrator and choose a unique name for your AXIS 560.
  6. Add your Internet address and host name to your system host table by appending them to the hosts file.
  7. perform the following commands to download the Internet Address and verify correct internet communication
    • arp -s 'ip address' 'node-address'
    • ping 'ip address'
  8. Now the initial terminal server setup is finished.
  • Please note that you need UNIX root privileges, or administrator privileges on a Windows NT server for this configuration procedure.
  • The Node address is based upon the serial number of the axis (the node address must be a 12 digt number with syntaxis:  XX-XX-XX-XX-XX-XX   e.g. 00-40-8C-42-7D-1C)

Configuring ports

Analyzer communication

For the analyzer to be connected to the terminal server, you will have to configure a port for it.

  1. enter the Internet adress or the host name of the AXIS as a URL in your Web browser as follows:
    • http://'ip-address' or
    • http://'host'
  2. following screen appears

  3. choose admin


  4. fill in the characteristics of the communication corresponding to the settings on the on-line you want to connect
  5. go back to the main menu
  6. choose network settings detailed view


  7. fill in the ip-address
  8. fill in the pr1 tcp port number: 3001

physical connection

The Axis Network Print Server serial port uses a standard male 9-pin D-sub connector as shown below.

Figure C-3 Serial connector pins as seen from outside the print server

The table below shows the pin-out configuration of the serial printer port (COM1):

Pin no.  Signal
1 Not used
2 Receive Data (RXD)
3 Transmit Data (TXD)
4,7 Request to Send (RTS)
5 Signal Ground (GND)
6 Clear to Send (CTS)
8, 9 Not used
