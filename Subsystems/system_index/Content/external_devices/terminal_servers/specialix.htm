Terminal servers: Specialix

PORT CONFIGURATION

This document describes the configuration of a Specialix terminal server to make it possible to connect analyzers via a server port.
This server doesn't know the LAT protocol. We always have to configure it via TCP/IP, especially via sockets.

Each port of the server can be assigned a TCP/IP port-address. This address or socket port number is used to connect to the server port.

Following commands let us define a TCP/IP port-address for a server port:

First you have to login onto the server. Or you have a direct port or you can do it via telnet to access the server.
Via telnet you have to say:

# telnet 'server-name'
  e.g. # telnet mts1
  where mts1 is a host name defined in /etc/hosts.

Hereafter you receive a login prompt, type 'admin' and press <return>. At the password prompt, just press <return>. This is the default admin user password.

login:admin
passwd: (none)

The command line prompt (MTS#) will be displayed. Now you are logged in as the system administrator. To use Full Screen mode (menus), you must first set your terminal type. Use the 'set line' command as follows:

MTS# set line . termtype vt100

Now to enter Full screen mode, type screen and press <return>. The main
menu will be displayed.

MTS# screen

Select the option 'Server Configuration'

Specify the following items :

  • Servername : the familiar name for your server e.g. mts1
  • Internet Address : e.g. 195.49.144.4
  • Broadcast Address: e.g. 195.49.144.255
  • Subnet Mask : 255.255.255.0
  • Domain Name

After completion, press <return> to exit. The Quit form will be displayed to save the form.

Select the option 'Network Configuration Menu'

  • Select the Host table  (hostnames and internet addresses of the hosts)
  • Select the Change Host to have an overview of the already known hosts.
  • Select Add Host to add a new one.

Typically for analyzers specify:

  • Hostname : the name of the host
  • Type: rev raw: this line type enables TCP/IP hosts to establish connections on external machines via MTS ports. The MTS is effectively transparent. Reverse raw connections are typically used to access printers or dialout modems.
  • Host Port: 23
  • MTS Port : the socket port number

MTS# ping hostname

LOGGING OUT OF THE PORT

  • Logout

REBOOT THE SERVER

Whenever you set or change the Server Configuration, you must reboot the server. This will broadcast the information across the network.

    MTS# reboot server

or from the Network Configuration menu - option Reboot.
