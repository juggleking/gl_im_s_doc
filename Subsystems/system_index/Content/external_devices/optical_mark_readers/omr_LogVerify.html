OMR log verify

Howto:

  • Log creation:
    Start all OMR translators with the '-p' command-option enabled. This option will set the used LogType::Name. Each time the OMR translator is started, a new log will be created. The log::Process will always be the translatorname. If needed, a stop sheet can be used to create a new log under the used log type.

  • Log verification:

    • Via LogTypes::Logs:

      • Select the concerning log type.

      • Open log browser of selected log type.

      • Select all logs which need to be checked.

      • Run Verify() method.

      • Start log verification.

      • Remark: the log type and log process are read-only fields. They will show the correct values but won't be editable. Open the Log::Verify() method via the menu when the above fields are expected to be editable.

    • Via menu:

      • Version 5.4 only:

        • The function Start\Configuration\SharedSpecimenGroupsByName will be used to call the Log::Verify() method. However, the item's place in the menu as well as its name  can be changed easily.
          HowTo:
          Select table gp_Site - Start\System Management\Tables\gp_Site. Edit gp_Site's menu. Rename method 'SharedSpecimenGroupsByName' to something more apllicable. Put the method under the Start\Routine if wanted.

      • From version 5.4 on:

        • The Log::Verify() method can be called directly from within the menu. Open it.

        • Select the log type. All log processes will be calculated.
          Remark: only logs created at a date no longer than after 2 days ago will be shown. Currently this is hardcoded.

        • Select the log process to verify.

        • The title bart will now show how many logs there are to be verified.

        • The browser will show the current log's entries.

        • Start log verification.

    • Log::Verify() user interface behaviour:

      • Only logs with at least one containing log entry with the NeedsChecking flag set, will be shown.

      • Only the log's log entries with the NeedsChecking flag will be shown.

      • Editing a log entry or opening a log entrie's Subject will unset the entrie's NeedsChecking flag.

      • When the NeedsChecking flag is unset, the entry will be shown in black. A red entry has the NeedsChecking flag set.

      • To set the NeedsChecking flag of an entry, open the entry for edit (F6) and set the flag.

      • The Previous and Next buttons can be used to go to the previous or next log.

  • Log entry message layout:

    • The first line will show the sheet sequence number.

    • Error messages follows, than all warning messages and last all informational messages.
