OMR specific parameters

  • -g <used OMR category name list>
    Specify the name of the OM categories used for import.
    Remark: Multiple categories can be specified here (comma separated). be sure not to use blanks in the Categroy names. If a form being imported checks out for more then one Category, an error will be reported. In this case, the parametrization is ambiguous and should be checked.
  • [-m <mark-map file - supported marks, format: \"'Side' 'Clock' 'Track'<NL>\"]
    When using a markmap, specify its full path here.
    File format:
    SS CC TT<NL>
    SS CC TT<NL>
    ...
    When specified, only marks set in this file will be taken into account for further processing.
  • [-o <auto feed mode (0: no, 1: yes; default no)>
    When set to 1, the translator will auto-feed the connected OMR reader.
  •     [-i <when auto feed: error sleep timers \"'time','increment','maximum'\">]]
    Sleep timers when in auto-feed mode.
    e.g.  3,1.60 - which are the standard values 3,1,60
    If the OMR does not respond to a 'feed', the next feed will be sent after a 3 s sleep time. If the OMR still fails to respond, the sleep time till the next feed is incremented by 1 s each time until a maximum value of 60 s is reached. Thus, the sleep time does not exceed 60 s. On successful processing of a feed by the OMR, the sleep time is reset to the initial value (3 s).
  • [-x <STX character (as decimal code), standard CTRL-B>]
    Start of Text character.
  • [-b <EOB character (as decimal code), standard CTRL-M (carriage return)>]
    End of Text character - e.g. 13 for carriage-return.
  • [-e <ETX character (as decimal code), standard CTRL-C>]
    End of Text character - e.g. 13 for carriage-return.
  • [-n <do not use STX synchronization>]
    When set, the translator does not expect a STX character in front of the sheet data stream.
  • [-p <specific database log process type name>
    Log type name of log type to be created in the database. This logging will contain all OM sheet parsing errors and communication errors as well as the errors reported at the database data import phase.
  •     [-l <when enabled: minimal logging level 'Info,Warning,Error'>]]
    Database log minimal log level.
  • [-v verbosity enabled - extra logging e.g. mark coordinate translation -]
    More logging. The Hex to Coordinate translatotion will be logged.

[]: Non-mandatory
