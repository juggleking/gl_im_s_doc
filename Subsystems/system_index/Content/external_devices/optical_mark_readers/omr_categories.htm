GLIMS categories, forms and form recognition

Categories

Name

Must be unique.

Identifier

An optical mark category specifies a set of forms sharing the same identification ’r;zone’. Which means: the identification marks of these forms are positioned within the same ’r;zone’ or ’r;box’ for each form.

Such an identification box can be specified using the following syntax:

"LeftUpperPoint:RightLowerPoint", a point is made up: "Sxx,Cyy,Tzz"
S = Side, C = Clock and T = Track.

eg. S1,C1,T36:S1,C3,T34

Which means: the identification ’r;zone’ is positioned at side 1, from clock 1 to clock 2 and from track 34 to track 36.

Forms

Each form has a unique name and a description.

Also a total side, clock, track, barcode and prompt count must be specified. These are used for validating the input given in the tree editor (e.g. the clock of a mark may not exceed the total clock count of the form at hand, identical for track and side). Barcode Count specifies the maximum ammount of barcodes that can be on the OM sheet (can be zero). Currently, the prompt subtype is not yet supported.

The form can be enabled or disabled. If disabled, it will not be used for OMR sheet processing.

The Form Identification must be filled for form recognition usage. As I mentioned before, a category specifies a set of forms with the same identification ’r;box’.

e.g. S1,C1,T36:S1,C3,T34  Category Identifier

e.g. 100010001   Form Identification filled in form left-to-right upper-to-lower. 1 = mark is set on form, 0 = mark is not set on form.

e.g. Marks read by reader:

Mark1: (Side 1, Clock 1, Track 34)
Mark2: (Side 1, Clock 2, Track 35)
Mark3: (Side 1, Clock 3, Track 36)

Each form also has a prefix, suffix and delimiter character (see chapter on Optical mark parameterization and parsing).

Copying OMR form configuration

It is possible to copy OMR form configuration independently of categories. A form contains the configuration of a physical OM sheet. A category can contain a number of  forms which have an identical identification zone.

It is possible to copy a form both to the same or to another category.

In order to do so, select the menu option Duplicate on any OMR category. 

First of all, you will have to specify a source form of which the configuration is to be copied. By default, the selected category will be specified as Target category, but as said before, it is also possible to copy the configuration of a form to another category.  The Target form allows to enter the name of a new form to which the configuration should be copied.
