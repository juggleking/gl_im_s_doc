Progress configuration files: Progress.ini (Windows) and .glimsrc (UNIX)

Contents: Introduction - MA Section - Startup section - Colors section - Fonts section - Keys section

Introduction

By default, the progress.ini file is located in the 'Bin' directory of your GLIMS installation (unless defined otherwise in the GLIMS shortcut by mean of the -ini name parameter).  The .glimsrc file can be found in $GLIMS.

All sections except the MA (MIPS Application) section are defined by Progress software.

MA Section

MA_HOST=GLIMS
MA_SERVICE=glimstasksv

See the chapter "Environment variables" for additional settings.

Startup section

The Startup section in the Progress.ini file contains Progress specific environment characteristics to run GLIMS.

AllowDesignMode=(yes|no)
Should be set to "no".
DefaultFont=<TypeFace>[, size=<PointSize>]
Lets you specify the default font. Default: "MS Sans Serif, size=8".
DefaultFixedFont=<TypeFace>[, size=<PointSize>]
Lets you specify the default fixed-width font. Default: "MS Sans Serif, size=8".
DLC=<Path>
Path to Progress installation folder. On the server, this is an absolute path (e.g. c:\dlc91b) and on the client, this is a UNC path (e.g. \\glimsserv\dlc91b), both pointing to the same physical directory.
EndMsgText=<Text>
Title for dialog box shown on attempted Windows shutdown during GLIMS session. Default: " Windows is exiting, are you sure to leave?".
EndMsgTitle=<Text>
Title for dialog box shown on attempted Windows shutdown during GLIMS session. Default: " GLIMS".
PROSTARTUP=<Path>
The relative path of the Progress default startup parameter file (.pf file). This file is read by all Progress modules at startup; it must exist for Progress to execute properly. Default: "Dat\Session.pf".
PROPATH=<Directory>{,<Directory>}
A list of directory paths separated by commas. Progress will search these directories in the specified sequence in order to locate a needed Progress routine. A Progress Library (.pl file) will be searched like any other real directory. Default: " ., LIB\PATCH.PL, LIB\APPL.PL, LIB\DSP.PL, LIB\TRG.PL, LIMG, IMG, LIB\MISC.PL, LIB\VAL.PL, LIB\BRW.PL, LIB\NAV.PL, LIB\EDT.PL, LIB\ZAP.PL, LIB\ADM.PL, LIB\IMP.PL, WRX, LIB\OBS.PL, c:\dlc, c:\dlc\gui".
nosplash=(yes|no)
Progress supplies a splash screen "Powered by Progress" at the startup of a GUI client and WebClient. Setting the variable to yes will disable this splash screen.

More information about these and other Progress specific environment variables can be found in the Client Deployment Guide of Progress.

Colors section

The colors section of the Progress.ini file define the colors that make up the color table of the graphical user interface.

Each entry in this section performs two semantic functions: it defines a color, and it maps that color to an integer in the range 0 to 255. GLIMS uses this mapping for making color assignments to widgets.

A color is being specified using the following syntax:

colorn= { R, G, B | colorname }

Where...
  • n: An integer from 0 to 255 that specifies the color table entry
  • R: An integer that specifies the amount of red present in the color
  • G: An integer that specifies the amount of green present in the color
  • B: An integer that specifies the amount of blue present in the color
  • colorname: Any of the following color names, mapped to the colors defined in the Windows Control Panel:
      
    COLOR-SCROLLBARCOLOR-BACKGROUNDCOLOR-ACTIVECAPTION
    COLOR-INACTIVECAPTIONCOLOR-INACTIVECAPTIONTEXTCOLOR-MENU
    COLOR-WINDOWCOLOR-WINDOWFRAMECOLOR-MENUTEXT
    COLOR-WINDOWTEXTCOLOR-CAPTIONTEXTCOLOR-ACTIVEBORDER
    COLOR-INACTIVEBORDERCOLOR-APPWORKSPACECOLOR-HIGHLIGHT
    COLOR-HIGHLIGHTTEXTCOLOR-BTNFACECOLOR-BTNHIGHLIGHT
    COLOR-SHADOWCOLOR-GRAYTEXTCOLOR-BTNTEXT

All entries in the color list must be sequential. If not, some colors will be considered as undefined.

Colors 0 to 15 a reserved by Progress. Don't change these values. 16-59 have a symbolic color reference (Windows color scheme) and will be reset to these defaults whenever you restart GLIMS.  So there is no use in changing these values.

The most relevant colors that can be changed are:

  • color 88 and 89: used for the microbiology work screen
  • color 15, 74 and 75: used for layout of browser master externalizations

We recommend not to change any other colors.

Fonts section

Next to the default system fonts defined in the Startup section, additional fonts that are in use by GLIMS are defined in this section.

Each entry in this section performs two semantic functions: it names a font that exists in the Windows environment and it maps that font to an integer in the range 0 to 255.  GLIMS uses this mapping when making a font assignment to a widget.

A font is being specified using the following syntax:

fontn= typeface, [ size=point-size, keyword, ... ]

Where...
  • n: An integer from 0 to 255 that specifies the font table entry
  • typeface: A valid typeface name, such as MS Sans Serif, Times or Courier.
  • point-size: The size of the font, in points. If you omit the point size, Progress uses the default point size as defined by the typeface.

Fonts 0 to 8 are reserved by Progress. Don't change these values. Fonts 16 to 256 are defined by MIPS, and can be edited if needed.
