System manual -> Environment variables

The application startup file (.glimsrc on UNIX or progress.ini on Windows) defines some additional environment variables required to run the application. It contains defaults that allow to run the application without changing them. Only when deviating from a standard configuration, this file will need to be edited.

RequestByRange
When the environment variable RequestByRange has been activated, the order entry program will now also accept a syntax like: ‘code1’ or ‘code1-code2’, where ‘code1’ and ‘code2’ are numeric and have at least 3 digits.
Under these circumstances, all requestables having a mnemonic starting with all possible numbers lying between ‘code1’ and ‘code2’ will be selected.
OrderEntryEncounterCreationConfirmation
Yes/No.  Creating or updating an encounter via the order entry program generates a confirmation popup screen in order to avoid erroneous creation of double encounters.  For the creation of encounters, this can be switched off via an environment variable 'OrderEntryEncounterCreationConfirmation' with possible values YES or NO.  When not specified, the confirmation is asked for sites with a HIS query possibility in the order entry screen.
OrderInternalIdOnMainPage
By setting the environment symbol OrderInternalIdOnMainPage=yes, it is possible to put the Order.InternaId field in the header part of the order entry screen.  The user will be forced to manually enter the order internal id before he can create a new order.  When this field is typed in, the program checks whether another order with the same internal id already exists in the database and displays an error messages if this is true. As soon as an acceptable internal id has been typed in, the field is hidden again.
OrderShortIdOnMainPage
By setting the environment symbol OrderShortIdOnMainPage=yes, it is possible to put the Order.ShortId field in the header part of the order entry screen.  The user will be forced to manually enter the order short id before he can create a new order.  When this field is typed in, the program checks whether another order with the same short id already exists in the database and displays an error messages if this is true. As soon as an acceptable short id has been typed in, the field is hidden again.
OrderShortIdValidity
The validity of an order short id can be defined via an environment variable OrderShortIdValidity=<x> where <x> is a number of days. Both the manual order entry program and the order manager will take this into account to determine whether an order with the given short id and a receipt time not longer than this <validity> ago already exists.  So when specifying "30", it will not be allowed to reuse the same order short id within a period of 30 days.
Note: when this variable is not set, the default value will make sure that an order short id can never be reused !
SpecimenExternalIdValidity
The validity of a specimen external id can be defined via an environment variable SpecimenExternalIdValidity=<x> where <x> is a number of days. The environment variable allows you to specify a period in time during which the external ID as barcode (scanned via specimen scan or queried by an analyzer) can be found unambiguously. If this variable is not found, the default period taken into account by GLIMS is 30 days.
Note: the GLIMS database will accept doubles for specimens referring to the same external ID !
WardIsDefaultIssuer
This variable (logical) is used in order entry in order to indicate whether the ward should be the default issuer.  Since GLIMS 5.4G, you can set the default issuer in the order entry options.
WardIsDefaultAgent
This variable (logical) is used in order entry in order to indicate whether the ward should be the default agent.  If set, entering an encounter type manually in the order entry screen will copy the already specified agent to the ward when the ward is not yet specified and when the agent is a ward.
Since GLIMS 5.4G, you can set the default issuer in the order entry options.
OrderEntryObjectTimeConfirmation
Yes/No. If set to Yes, (and a ZIS connection is active) during order entry, the user will be asked to confirm the object time before the latter is used in other references (e.g. as start time of the encounter). If set to No, no confirmation will be asked.
MA_CustomizeKey
This variable should be set in order to customize the order entry screen.
Example: MA_CustomizeKey=LAB
MA_NextPageKey
By defining an environment variable MA_NextPageKey, the user can now choose an explicit key to jump from field to field in the dialogs which have an active WidgetSequence list.

The only program that uses this so far is the order entry screen (settable via the order entry options button). When such a key is defined, pressing it will bring the user to the next non-visited field from the preconfigured list. Also, the F2 (or Do) key or selecting the OK button will then immediately save the current screen without going to the next page. When such a key is not defined, the F2 or Do keys keep their behaviour as before. Currently, only F8, F9, F11, F12 and SHIFT-F7 are supported as "Next Page" key.
MA_Shielding
We advise not to use this variable any longer.  It could be used in previous Glims versions to activate Shielding.
MA_OrderShielding
We advise not to use this variable any longer.  It could be used in previous Glims versions to activate Shielding.
MA_PersonShielding
We advise not to use this variable any longer.  It could be used in previous Glims versions to activate Shielding.
CombinedName
See ASTM documentation (Work order message - Patient information record (P)).
MA_ExitEditorViaFind
The proces of zooming into a record from a browser, editing and saving the record and then selecting a new record from the browser to zoom in, may take some time. To speed up this action, this environment variable can be used. Check out an example and more detailed information.
MA_ExitEditorViaStandBy
This environment variable is used for the same purpose as MA_ExitEditorViaFind. The only difference is that the editor is no longer displayed when leaving it. It is, however, saved on the background.
ExportDirectory
This environment variable is used for the batch-executed import/export program. It allows to specify the directory where the tables to be exported should be saved.
ExportIndex
This environment variable is used for the batch-executed import/export program. It allows to specify the file which should contain the index of all tables to be exported.
ImportDirectory
This environment variable is used for the batch-executed import/export program. It allows to specify the directory from where the tables should be imported.
ImportIndex
This environment variable is used for the batch-executed import/export program. It allows to specify the file which contains the index of all tables to be imported.
ImportByMnemonic
YES/NO. This environment variable is used for the batch-executed import/export program. It allows to indicate if the tables to be imported are specified by mnemonic or by auto id.
ImportUpdateExistingRecords
YES/NO. This environment variable is used for the batch-executed import/export program. It allows to specify whether existing records should be updated during the import.
MA_Name
This variable is used to indicate the name of the application being run.
Example: MA_NAME=GLIMS.
GLIMSRemoteCalls
Yes/No. If this variable is set to Yes, the order entry options Action scheduling mode, Postscheduling mode and Report scheduling mode apply. If GLIMSRemoteCalls is set to No, then the action scheduling, post scheduling and report scheduling will be executed locally, regardless of the order entry options. In order to use GLIMSRemoteCalls, you will have to specify MA_HOST and MA_SERVICE (see below).
GLIMSRemoteQuerying
YES/NO. It is possible to perform some queries on a remote machine.  Please check the chapter on remote querying for a full description of this functionality.
MA_HOST
This variable is used to indicate the server on which GLIMS has been installed.  It must be set on all the clients and the server.  On Unix systems, you can check the server name with "uname -a".
Example: MA_HOST=sun450
MA_DEFAULTSERVICEHOST
This variable is used to indicate the computer where Services have to run if their Host is not specified. This variable is used for Windows only. It should thus only be specified in progress.ini. Example : MA_DEFAULTSERVICEHOST=theNT
MA_SERVICE
This variable is used to indicate the service used by GLIMS.
Example: MA_SERVICE=2505
MA_LC_CTYPE
Country-specific characters are now allowed in mnemonics. From now on, all printable characters are allowed in mnemonics. Blank space is still not accepted. Determining which characters are printable can now be specified per country via the environment variable MA_LC_CTYPE which should be set to an appropriate value on both the Glims database server and all PC clients.

Property mnemonics, however, still have additional restrictions:

  • The mnemonic should contains at least one alphabetic character
  • Only alphabetical characters (settable via the MA_LC_CTYPE variable), digits and delimiters from the set ‘ - _ + . , ’ (minus, underscore, plus, period, comma). We recommend you not to use the comma character if possible.
  • Note that setting the MA_LC_CTYPE environment variable also allows the usage of country specific characters (e.g. ‘é’, ‘ü’, …) correctly in MISPL site functions or text definitions !
MA_TIMESOURCE
Is used to determine which time is used (time stamps etc).
  • If not set, the current date of the server is used (more specifically the server that contains the genrw-database)
  • If set to "LOCAL", the current date of the local PC is used (which we do not recommend unless you have serious performance issues).  In order to secure the synchronization with the server time, dedicated software tools as e.g. Dimension 4 can be used.
  • You can also specify a database name.  The current date of the server containing that database will be assumed.
This also influences the MISPL functions Today() and Now().

MA_ShowListWhenAmbiguous (list of table short names)

Fields that allow choosing a record by typing in its key or that allow opening a selection browser by hitting CTRL-F or with a double mouse click are used in many screens.  Typically, when typing in an ambiguous key, they give a warning beep when leaving the field. When trying to commit the screen, they usually issue a warning message on screen stating that the chosen record is ambiguous.

These fields also offer the possibility to automatically display a selection browser on the screen, showing all the records matching the specified key value.

This feature is only active for a given field if the short name of the corresponding table is specified in the environment variable MA_ShowListWhenAmbiguous. 

Example: MA_ShowListWhenAmbiguous=mncp,enct,stn

If you want to activate this feature for all tables at once, put MA_ShowListWhenAmbiguous=* in your Progress.ini file.

Note that this feature does not work when the corresponding key is made up of more than one field. Examples of these: Person (key is Lastname + FirstName) and PaymentAgreement (Matriculation + Discriminator).

Currently, this feature is only supported for the following tables:  Antibiotic, Animal, BloodBag, BloodProduct, ChoiceList, Correspondent, Department, Encounter, Identification, IsolationAppraisal, Lot, material, MaterialVariable, MicrobiologyProcedure, Medium, Municipality, MicroOrganism, PathologyProcedure, QCMaterial, Reagent, Station, User and Ward.

WindowsPrintingSupportedExtensions
This variable allows specifying a comma-separated list of file extensions.  Files with these extensions can then be printed to a Windows printer.  See 'Using Windows printers'.
RenameOrderOnDiscontinue (YES/NO) 
This environment variable allows to rename the order internal id when an order is discontinued.  If the variable is not set then the order internal id will not be renamed. If set, and an order is discontinued, the order internal id will be renamed so that it can be reused. A date stamp (year, month, day, hour, minutes, seconds) will be appended to the external id. The order short id will also be cleared.

We strongly suggest not to use the RenameOrderOnDiscontinue environment variable in combination with HIS communication as the new internal id will not be acknowledged and the link with the original id will be lost.

RenameSpecimenOnDiscontinue (YES/NO)
Analogously to the order internal id, the specimen internal id will only be renamed if the variable is set to YES.
ExitConfirmation (YES/NO)
If you do not want confirmation upon exiting GLIMS ( "Are you sure to leave ?"), you can skip this message by setting ExitConfirmation=NO
ImageServerDir
Allows to specify a directory in which all your image results are stored.  For more information, please check the chapter on Graphs and Image results.
CheckBillingRules (YES/NO)
If set to Yes, GLIMS will check the billing code of each requestable during order entry and display the associated billing rules at the bottom of the screen.  The default setting is NO.
BrowserSupportedExtensions
Comma-separated list of file extensions.  This variable allows to decide which file type should be previewed using a browser like Internet Explorer instead of the normal built-in text editor.  In order to use the browser for previewing reports for instance, the variable could contain 'prn' to specify regular text report output file extensions.  The default value for this variable is "htm,html".
AllowInteractiveShMemConnnections
With a "Self-servicing client", we mean a progress process which is connected to the database through shared memory, that is, without -S .. -H network parameters. If such a process is killed with a Unix kill -9 command or with the windows task manager, there is a possibility that the process had a lock on a buffer. 
That is, the database is being shut down.  To avoid this, for interactive self-servicing clients, a connection with -S .. -H is realised instead of the shared memory connection (even if no -S is not specified in the *.cpf files). A kill -9 of this progress process in that case will not cause the database to shut down (the remote client server process will perform the cleanup).

By defining the environment variable "AllowInteractiveShMemConnnections=yes", you can overrule this mechanism, thus allowing interactive shared memory connections.

CronRestartInterval

A decimal time span value which indicates the number of days after which a task scheduler should autonomously restart.

LogInteractiveSessionErrors

Determines if interactive session errors will be logged.

MaxDBWritesPerSec

Used for limiting disk access of log purge. A reasonable value is 0.3.

MA_EnableSmartMenus (Yes/No)

Glims offers smart popup menus.  This implies that popup menus will only be active if they are applicable to the selected record.  Example: if no printer is defined for a default report, then the option "Printer" will be grayed out in the popup menu for that default report. Default value is "Yes".

MA_AllowClosedEncounters (Yes/No)

If set to YES, the default restrictions in the order entry screen to avoid selection of closed encounters will be disabled.

UseAutonomousTransaction

When specifying "Yes" for this variable (MA Section), the report's NeedsChecking flag will be set using an autonomous transaction. Autonomous transactions were introduced to avoid locking problems. The setting is only relevant when used with an Oracle database.

Warning

Setting this option can cause a performance degradation (maximum number of open cursors reached) and should only be used when strictly needed.
