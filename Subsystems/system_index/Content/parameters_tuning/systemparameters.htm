System parameters

Important system parameters

Parameter Specific parameter names Meaning Progress optimal setting Reference value *
SEMMNI
  • HP-UX: Semmni
  • SunOS & SOLARIS: semsys:seminfo_semmni
  • OSF Digital Alpha: sem-mni
  • AIX: not configurable
  • LINUX FOURTH NUMBER IN SEM (/proc/sys/kernel/sem)
 Number of semaphore (SEM) IDs; each represents an array of SEMs. 1 per active multi-user database. 100
SEMMSL
  • HP-UX: Semmsl
  • SunOS & SOLARIS: semsys:seminfo_semmsl
  • OSF Digital Alpha: sem-msl
  • AIX: not configurable
  • LINUX FIRST NUMBER IN SEM (/proc/sys/kernel/sem)
 Maximum number of semaphores per SEM ID. Max number of local users + Max number of servers /  DB + 4 500
SEMMNS
  • HP-UX: Semmns
  • SunOS & SOLARIS: semsys:seminfo_semmns
  • OSF Digital Alpha: Num-of-sems (not taken into account)
  • LINUX second NUMBER IN SEM (/proc/sys/kernel/sem)
 Total semaphores in the system. (SEMMSL x #active databases) 1500-3000
SEMMNU
  • HP-UX: Semmnu
  • SunOS & SOLARIS: semsys:seminfo_semmnu
  • OSF Digital Alpha: sem-mnu
  • AIX: not configurable
 Number of semaphore undo structures. Same value as SEMMNS. 1500-3000
SHMMAX
  • HP-UX: Shmmax
  • SunOS & SOLARIS: shmsys:shminfo_shmmax
  • OSF Digital Alpha: shm-max
  • Linux: shmmax (/proc/sys/kernel/shmmax)

 

 Maximum SHM segment size. System default; increase if you get Progress error 1135. Minimum 1/3 of available memory and relative to -B parameter.  ((Sum of all -B values * blocksize) + 10%)

Example: 1 GB system -> SHMMAX= 400
SHMMNI
  • HP-UX: Shmmni
  • SunOS & SOLARIS: shmsys:shminfo_shmmni
  • OSF Digital Alpha: shm-mni
  • Linux: shmmni (/proc/sys/kernel/shmmni)

     

 Maximum number of shared memory (SHM) identifiers. 2 per active multi-user database;

if (#users > 50), add 1 per 25 users; if (-B > 400), add 1 per 200 buffers.

 200
SHMSEG
  • HP-UX: Shmseg
  • SunOS & SOLARIS: shmsys:shminfo_shmseg
  • OSF Digital Alpha: shm-seg
 Maximum number of SHM segments a single process can attach. 4-8 120
NFILES
  • HP-UX: Nfile
  • SunOS & SOLARIS: Not applicable
  • OSF Digital Alpha: Not applicable
 Number of files open.   32.000
NPROC
  • HP-UX: Nproc
  • SunOS & SOLARIS: Not applicable
  • OSF Digital Alpha: task-max
  • Set process limit with ulimit -u 16384
 Number of processes.   1024
MAXUPROC
  • HP-UX: Maxuprc
  • SunOS & SOLARIS: Maxup
  • OSF Digital Alpha: max-proc-per-user
 Maximum number of processes per user.   512
FILEDESCSOFT
  • HP-UX: Maxfiles
  • SunOS & SOLARIS: rlim_fd_cur
  • OSF Digital Alpha: open-max-soft
  • LINUX file-max (/proc/sys/fs/file-max)
 Number of file descriptors, soft limit.   2048
FILEDESCHARD
  • HP-UX: Maxfiles_lim
  • SunOS & SOLARIS: Rlim_fd_max
  • OSF Digital Alpha: open-max-hard
 Number of file descriptors, hard limit.   2048

* These values are merely default reference values.  The actual values can vary significantly according to your hardware and software configuration.    

How to configure ulimit on UNIX systems

AIX:

             /etc/security/limits

 mips  fsize = -1

LINUX

            /etc/security/limits.conf

SOLARIS

            /etc/default/login

Swap space size: We recommend to set the swap space size to at least twice the size of system memory
