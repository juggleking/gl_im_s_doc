Introduction

Session.pf is a Progress parameter file that can be used to change the default startup parameters. This parameter file is specified in the progress.ini file, in the section Startup, variable Prostartup. The file used by MIPS is located on the GLIMS Installation Root Directory, and called Dat\Session.pf.

Automated update of session parameters

The session.pf file can be updated with parameters, specified in a site attribute "SessionParameters" on gp_site. Parameters must be comma-separated; example: -q,-zod.  The program tries to check if the session file is writable.  If not, startup will continue. Startup will not continue if PROSTARTUP is not set.

Parameters can be removed by putting '!' in front of a parameter. 

Example: '!-noshvarfix,-TM 32' will remove '-noshvarfix' from session.pf (if present) and append '-TM 32' if -TM not present or specifying a value other than '32'.  The original -TM value remains, the -TM 32 appended overrides the previous setting.

Parameters

-d <DateFormat>
Lets you specify the date format used in your GLIMS session. For a valid <DateFormat>, use a three-character string of d, m, y in any order.
-E
Use European number format. If not specified, American number format is used.
-nb <Integer>
Sets the maximum number (soft limit) of nested procedure blocks.  We advise to set this parameter to 240.
  
-yy <Integer>
Specify the Century Year Offset to determine the start of the 100-year period in which a date with a two-digit year is valid. GLIMS uses the year 1950.
-s <Integer>
We advise to set this parameter to at least 120.

Lets you change the size of the stack (an internal memory area used by Progress program modules). The value must be increased when one of the following messages appears:
  • SYSTEM ERROR: stkpush: stack overflow. Increase -s parameter.
  • SYSTEM ERROR: stkditem: stack overflow. Increase -s parameter.
-T <Path>
Lets you specify the temporary directory. If you do not use this parameter, Progress puts the temporary files in the current working directory. On UNIX and Windows, these filenames begin with a prefix of lbi, dbi, pge, srt, or trp, and end with the process ID of the user's Progress session.

An integer that specifies the number of blocks in the buffer for temporary tables.

Use Buffer Size for Temporary Tables (-Bt) to specify the size of the buffer pool for blocks/records from the temporary table. Progress uses the value you specify for temporary databases used during the session the same way it uses Blocks in Database Buffers (-B) for permanent databases. The default value is 10.

-TB <Integer>
<Integer> represents the block size in kilobytes, to allocate when sorting records for reports and when rebuilding indexes. The default block size is 2K but can be reset anywhere from 1KB to 31KB. GLIMS uses a value of 16. Use Speed Sort primarily to improve sort performance, particularly during index rebuild operations. If you increase the block size beyond 2KB, Progress sorts records faster but uses more memory and disk space in the process. If system memory is severely limited, you might actually choose to lower the block size. Sort space is allocated in the SRT file, a temporary session file having a system-generated unique name with the prefix srt. Progress uses the SRT file to store session-compile modules and as temporary work space during sorting (including rebuilding indexes). Use -TB to set the SRT file block size-the increments by which the SRT file grows. 
-TM <Integer>
<Integer> represents the number of blocks or streams to be simultaneously merged during the sort process. Use merge number to increase the speed of the merge phase of the sort process (at the cost of increased temporary memory usage). See also the description of the Speed Sort (-TB) startup parameter. GLIMS uses a value of 32.
-q
Check if the -q parameter is set in session.pf or GLIMS shortcut on each client + server.

Sets Quick Request mode, telling Progress to search PROPATH directories only on the first use of a procedure. Usually in a Progress procedure, when the RUN statement is used to run a subprocedure, Progress searches the directories named by the PROPATH environment variable, looking for a procedure of the same name with a .r extension. If it finds a file with a .r extension (an r-code file), it checks to make sure the r-code file has not changed since that r-code file was created.  With Quick Request (-q), after the initial search, if the procedure still resides in memory or in the local session-compiled file, Progress uses that version of the procedure rather than searching the directories again. However, Progress always checks whether Data Dictionary definitions related to a procedure were modified. If they were modified, Progress displays an error when it tries to retrieve the procedure. 
- noshvarfix
While the usage of the session parameter -noshvarfix could have a minor performance gain, it may also cause errors in some cases, like this one: ** Shared variable <varname> has not yet been created (392). Therefore, we recommend not to use it. If you get an error like the one above, check session.pf and verify that -noshvarfix is not active (the string '-noshvarfix' should not appear in session.pf).
- debugalert
See chapter on debug alert

-rereadnolock

To avoid inconsistent action and order statuses, we strongly recommended to activate the "-rereadnolock" Progress session parameter. This should happen on the server and ALL clients.

-expandbrow

Sets the EXPANDABLE and FIT-LAST-COLUMN option to on for all browsers created in the current session. Influences BROWSE look. Do not change.

-zod

This parameter optimizes sorting in queries with an Oracle database by eliminating unnecessary BY-clauses. Necessary for an Oracle db to obtain reasonable performance. No effect on a progress db.

-yr4def

Format all dates by default with 4 digits in the year specification. Important when doing ascii dump & load

-mmax

Initial size of .r code execution buffer. Will by default be set to a value of 65534. At application start up, this setting will automatically be changed to the default value, if needed.

-cpstream 1252 # iso latin-9 /-cpinternal 1252 # iso latin-9

Set to "1252" on WINDOWS. Set to "iso8859-15" on UNIX.

-cpcoll basic

Determines how character strings are sorted. It specifies the collation table used.

-D 512

Internal .r code directory size. Leave at 512.

-logthreshold <Integer>

Sets the maximal size of interactive session error log files to the specified number of bytes.

-numlogfiles <Integer>

Sets the maximal number of interactive session error log files that will be retained.
