System manual -> The parameters in glimsdb.pf, genrwdb.pf and genrodb.pf

The glimsdb.pf, genrwdb.pf and genrodb.pf parameter files are located in the "Dat" directory of your GLIMS installation.  These parameter files are used when modifying the bi-block size and bi-cluster size:

E.g.  proutil glims -C truncate -pf glimsdb.pf

The parameter files contain two parameters:

-bi

Specifies the size of the cluster in kilobytes. The number must be a multiple of 16 ranging from 16 to 262,128 (16K to 256MB). The default cluster size is 128K. If you use a value that is not a multiple of 16, PROGRESS rounds the value up to next multiple of 16.

-biblocksize

Specifies the size of the BI blocks in each buffer in kilobytes. The valid values are 0, 1, 2, 4, 8, and 16. A value of 0 tells PROGRESS to use the default block size.  You cannot use a value of 1 on 2K database block size machines, such as Sequent. The block size cannot be smaller than the database block size.

Attention:

As of version 5.5E, GLIMS forces default values for these parameters:#

<dbname>db.pf

  • Unix: -bi 8192 -biblocksize 8  -  Windows -biblocksize 4)
  • (-bi 4096 for genrw)
