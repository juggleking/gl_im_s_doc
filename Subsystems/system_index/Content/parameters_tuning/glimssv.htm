System manual -> The parameters in glimssv.pf, genrosv.pf and genrwsv.pf

The glimssv.pf, genrosv.pf and genrwsv.pf parameter files are located in the "Dat" directory of your GLIMS installation.  

-n

Max number of licensed users

Interactive users + batch users + 5

-Mn

Max number of servers

Proposed value: -Mn = -n/Ma

-Ma

Max number of clients per server

Keep this as low as possible to gain performance. Proposed value = 4.
-Mi Min number of clients per server Suggested value: 2 for genro/genrw and 1 for glims

-L

Max Lock Table Amount

Total size of record locking table.  Set to 32768 for genrw and glims. 

-B

Max Block size in Memory

Total amount of memory available to load Data of the DB.

Value *Block size of the Db is the actual chunk of memory allocated.  For GLIMS 1/3 of total available Memory

See remark below

-spin

Processor count parameter

This parameter should only be activated in case of multi-processors and in case there is an Enterprise license.

Recommended value:  20000 * #processors (if #processors > 1).
-semsets Semaphore sets Unix only.  Use a value of 3.
-bibufs Before image buffers Proposed value = 25 for genrw, 50 for glims

-Mpb

Number of servers for client connections

The -Mpb parameter limits the number of servers to be spawned via the database broker. Setting the -Mpb to 1 allows only 1 server process to be spawned per the database broker. The one server spawned is currently handling its allowed client connections. If the "-Mpb" parameter is set to 1 then only the number of clients specified with the "-Ma" parameter are allowed to connect regardless of what the "-Mn" parameter is set to.

-s

Stack size

The  default value  is -s 40 , Progress recommend to use  minimum -s 128

Case: when the content of a text module becomes too long, it may occur that the mnemonic of the text module can no longer be changed.  If so, increase the -s parameter in genrwsv.pf.

As of version 5.5E, GLIMS forces a number of default values:

In <dbname>sv.pf:

  • -semsets 3 (unix only)
  • -bibufs 25

A sample parameter file

# GLIMS GLIMS Server Startup Options
# ----------------------------------
-n 32                   # allow enough users
-cpstream iso8859-1     # stream I/O code page (= ISO Latin-1)
-bibufs 50              # before image buffers; only relevant for probiw
-B 1024                 # buffers; default is 8 * nr of users; 
                        # 1024 * 4K blocksize = 4096 is 8MB 
-Mn 8                   # maximum nr of servers; default is 4
-L 1024                 # lock table entries; default is 500 
#-spin 1                # on hp-systems with 1 processor 
                        # other systems: comment it out
                        # if multi-processor (any system) -spin 40000
-Ma 4                   # maximum 4 client / remote client server
