Oracle Database Server installation

 

Deploying an Oracle Database Server environment requires Oracle database administrator (DBA) skills. This installation guide is only intended as a guideline.

Deploying a GLIMS server environment must always be done by a MIPS representative in close cooperation with the Oracle DBA.

This installation guide was written for a Windows platform. While the individual steps are very similar on Linux and UNIX systems in general, specific platforms often require specific additional steps, such as installing additional operating system packages, creating operating system users, kernel parameter tuning, running additional installation steps under the root account etc. Check the Oracle installation documentation for the specific platform.

The screenshots below show an Oracle 12.2 Database Server installation. The installation procedure for other supported Oracle versions is similar to this version. Check the Oracle installation documentation of the specific version for more details.

 

Oracle Database Server software installation

 

Run setup.exe (Windows) or runInstaller (UNIX) from the Oracle Database Server installation files.

 

 

In the Configure Security Updates, you can opt-in to be informed of security issues and to receive security updates in an automated way. Fill this in according to the security to the security standards of the site.

 

 

If you decide not to register for the security updates, the warning below will be shown. Click Yes to continue.

 

 

In the Installation Option dialog, you will typically choose the Install database software only option, as the DBA usually creates the database manually or uses the GLIMS database installer.

 

 

In the Database Installation Options dialog, you will typically choose the Single instance database installation option.  Check the Oracle installation documentation for more information about Oracle Real Application Clusters (RAC) when required.

 

 

In the Database Edition dialog, choose the appropriate Oracle Database edition in function of your Oracle licenses.

The Standard Edition is sufficient, but the Enterprise Edition + Diagnostic Pack offer better means for performance troubleshooting. When using the Standard Edition, MIPS cannot perform detailed performance troubleshooting at the database level.

 

 

In the Oracle Home User Selection dialog, the Windows user that owns the Oracle software installation can be chosen. Deploying the software under the "Windows Built-in Account" is the easiest solution, but from a security perspective it could be better to use a dedicated Windows user account. This setting should be chosen based on the security policies of the site. This topic should be discussed with the Oracle database administrator.

 

 

The installer will issue an extra security warning if the Windows Built-in Account was chosen. Click Yes to continue.

 

 

In the Specify Installation Location dialog, the installation folders for the Oracle software can be chosen.

 

  • The Oracle base is the common root folder for all Oracle products and versions.
  • The Software location is a subfolder of the Oracle base in which this specific version of Oracle Database software will be installed.

 

GLIMS automatically detects the installation folders. The folders can be chosen according to the installation policy of the site. When in doubt, it is often best to change the Oracle base to a meaningful location (e.g. d:\oracle), while keeping the default for the Software location.

 

 

During the Perform Prerequisite Checks step, the installer verifies if all system requirements are met. If the server meets the GLIMS system requirements, no warnings or errors should appear.

 

 

Reread everything in the Summary dialog and click the Install button to proceed.

 

 

Wait for the Install Product dialog to complete.

 

 

The Finish dialog will appear. Click Close to finish the installation.

 

 

Oracle Listener configuration

 

Before GLIMS can be installed, an Oracle Listener needs to be available.

 

Run the Net Configuration Assistant from the Start Menu, or launch netca from the command line.

 

If you are running multiple Oracle software installations on the same machine, for example a side-by-side server and client installation, or when multiple Oracle software versions are installed on the same machine, you should make sure that you take Net Configuration Assistant from the right Oracle Database Server version.

 

In the main menu, choose the Listener configuration option.

 

 

Choose to Add a listener.

 

 

Give the listener a meaningful name or accept the default name.

 

 

GLIMS only supports a TCP network. Other protocols should not be selected.

 

 

Usually choosing Use the standard port number of 1521 is the best choice. Only in exceptional cases, for instance if you want to run multiple listeners from different Oracle Database versions on the same machine, a non-default port number should be chosen.

 

 

There is no need to configure a second listener, so choose No.

 

 

Wait for the listener configuration to complete and click Next.
