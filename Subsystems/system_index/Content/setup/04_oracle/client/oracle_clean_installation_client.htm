Oracle Database Client installation

 

GLIMS requires a 32 bit Oracle Database Client. In a single server model, as of Oracle 12.2, the 32 bit client can be installed next to a 64 bit Oracle Database Server installation on the same machine without any problems. With erlier versions, this could be somewhat problematic. Check the Oracle installation documentation of the specific version for more details.
The screenshots below show an Oracle 12.2 Database Client installation. The installation procedure for other supported Oracle versions is similar to this version. Check the Oracle installation documentation of the specific version for more details.

 

Run setup.exe from the Oracle Database Client installation files.

 

Wait for the requirements check to finish.

 

 

In the Select Installation Type dialog, the appropriate installation type should be chosen in function of the role of the machine on which the Oracle software is installed:

The InstantClient is not supported for GLIMS.

 

 

In the Oracle Home User Selection dialog, the Windows user that owns the Oracle software installation can be chosen. Deploying the software under the "Windows Built-in Account" is the easiest solution, but from a security perspective it could be better to use a dedicated Windows user account. This setting should be chosen based on the security policies of the site. This topic should be discussed with the Oracle database administrator.

 

 

In the Specify Installation Location dialog, the installation folders for the Oracle software can be chosen.

 

  • The Oracle base is the common root folder for all Oracle products and versions.
  • The Software location is a subfolder of the Oracle base in which this specific version of Oracle Database Client software will be installed.

 

GLIMS automatically detects the installation folders. The folders can be chosen according to the installation policy of the site. When in doubt, it is often best to change the Oracle base to a meaningful location (e.g. D:\oracle), while keeping the default for the Software location.

 

 

During the Perform Prerequisite Checks step, the installer verifies if all system requirements are met. If the server meets the GLIMS system requirements, no warnings or errors should appear.

 

 

Reread everything in the Summary dialog and click the Install button to proceed.

 

 

Wait for the Install Product dialog to complete.

 

 

The Finish dialog will appear. Click Close to finish the installation.

 

 

It is not necessary to configure the TNS layer (Oracle network settings in the tnsnames.ora file) manually on the client, as this will be done by the GLIMS software installers. However, if this is required by the Oracle deployment policy of the site, GLIMS can use a preexisting or alternative TNS configuration.
