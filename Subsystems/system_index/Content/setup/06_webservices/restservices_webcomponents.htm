Initial installation

The following components will be installed by the web installer:

  • The Tomcat Web server with

    • specific web components for the extended Genetics module:

      • APvisuals: needed to display the visual representation of approach plans and approaches;
      • JSPedigree: needed to display and draw pedigrees;

    • REST components
  • the GLIMS 9 REST AppServer;
  • a certificate wizard that will help you obtain the certificate required to use the web components.

To install these components, execute the web installer that is available on the MIPS Support website. Choose the version of the web installer that corresponds to your GLIMS version or - if no web installer is available for that specific version - choose the version of the web installer that is closest to your GLIMS version.

Example

Suppose a clean install of GLIMS 9.9.4 is performed and the customer wants to use the new Genetics module.

The available web installers are:

  • GLIMS 9.9.3
  • GLIMS 9.9.2 (Windows only)

As no web installer is available for GLIMS 9.9.4, you should execute the web installer of which the version number is closest to this GLIMS version. In this case, the GLIMS 9.9.3 web installer can be executed for the initial installation. However, when the customer decides to upgrade to a higher GLIMS 9.9 version, the web installer that corresponds to that specific GLIMS version must be used during the upgrade phase (see below).

Upgrade

When a new GLIMS version is released, it may contain corrections or extensions of one of the REST services or web components. Therefore, a web installer will be created with a version number that matches the new GLIMS version. When upgrading to that specific GLIMS version, the new web installer must be executed as well.

Example

Suppose a customer was using GLIMS 9.9.2 with the Genetics module or CyberTrack and decides to upgrade to GLIMS 9.9.3.

The available web installers are:

  • GLIMS 9.9.3
  • GLIMS 9.9.2 (Windows only)

Since this is an upgrade and a web installer is available with a version number that matches the new GLIMS version, this web installer needs to be executed as well.

If no web installer is available with a matching version number, no additional actions are required.

 

Before launching the web installer, make sure to:

  • stop the web server
    • On Windows: Stop the Windows service named "Glims9 web service"
    • On UNIX: Run "$GLIMS/tomcat/bin/catalina.sh stop"
  • stop the Glims9rest AppServer. This can be done via the OpenEdge management console or using the command "asbman -name Glims9rest -stop".

 

On Windows, the web installer allows the user to start the "Glims9 web service" automatically. It is recommended that you enable this setting.

On Linux, the web server can be started by running "$GLIMS/tomcat/bin/catalina.sh start"

 

When the execution of the web installer is completed, make sure that:

  • the web server is running
    • check the Windows service named "Glims9 web service"
    • on UNIX you need to check if the tomcat process exists with "ps -eaf | grep tomcat"
  • the AppServer Glims9rest should is running
    • this can be verified via the OpenEdge management console or via the command "asbman -name Glims9rest -query".

     

Web wizard for Windows

Download the WebWizard executable to a directory of your choice and execute it.

The web wizard looks as follows. Fill in the fields.

Application server definition

Installation of the GLIMS 9 REST AppServer. By default, the port number is 4209 but it may be modified.

Web services configuration

Configuration of the Tomcat Web server. By default, the port number is 8443 but it may be modified.
"FQDN" stands for Fully Qualified Domain Name.
 

Web wizard for UNIX

Download the UNIX installation files and extract them to an installation folder of your choice, for instance in a folder next to the GLIMS installation files (e.g. if the GLIMS installation files are in /install/glims9, consider putting the Web wizard in /install/wsglims9 or similar). Do not extract the files directly into $GLIMS!

Run webwizard.sh from this installation folder.

You can choose a web server port and you will be asked to fill in the information that will appear on the server certificate, just like on Windows.

 

Certificate wizard

Warning

MIPS only provides the functionality to generate a certificate signing request and to import the signed certificate.

The certificate signing, expiration monitoring and renewal are the responsibilities of the customer.

Once the web wizard has finished, you find a certificate wizard file in the Tomcat "conf" folder (CertWizard.exe on Windows, certwizard.sh on UNIX). Execute this file and select Generate a Certificate Signing Request:

Subject alternative name configuration

Name that will be used by the client to connect to the server.

 

Note for UNIX users: the dialogs on UNIX contain essentially the same information as on Windows.
