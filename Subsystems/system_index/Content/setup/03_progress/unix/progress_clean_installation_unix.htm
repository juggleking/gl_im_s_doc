Progress OpenEdge installation on UNIX

There is no need to install a base version (e.g. 11.7), followed by a service pack (e.g. 11.7.5) anymore.  As of Progress OpenEdge 11.7, the minor versions have become full standalone installers.
Progress OpenEdge needs to be installed as root.
All other GLIMS components need to be installed with a dedicated MIPS user.

 

Log in to the server as user root.

Extract the Progress OpenEdge installation files in a folder of you choice. In the example below, we use /install/progress.

Launch the Progress OpenEdge installer proinst from outside the directory containing the installation files. In the example below, we change to /tmp first.

 

 

 

Answer "N" (=NO) when asked if you would like to apply the service pack to an existing installation.

  • Press the Enter key to continue.

Done Configuration Data Confirmation dialog

 

 

  • Answer "Y"(=YES) to continue.

 

Enable OpenEdge Explorer Confirmation dialog

 

 

  • Answer "Y"(=YES) to continue. This will install the Progress OpenEdge web interface.

 

Install Type and Destination dialog

 

 

  • Go into the Select Type of Installation submenu.



    • Select Complete Install.

  • Go into the Select Destination Pathname submenu.




    • Fill in the Destination Path. Preferably install Progress OpenEdge in /<any root folder>/dlc117. Other directories can be used for the installation when needed.
    • Fill in the Work Directory Path. This can be any temporary folder like /tmp.
    • Press Enter to continue.

DLC stands for "Data Link Corporation", as Progress Software was called in the old days. It's still a convention in the Progress world to use "DLC" to refer to the Progress OpenEdge installation folders.


  • Go into the Select Management Pathname submenu.

 

    • Fill in the Management Path. Preferably install the management console in /<any root folder>/dlc117/OEMgmt. Installing the management console in the OEMgmt subfolder of the OpenEdge installation folder results in a cleaner directory structure and allows easy patching when needed. This path is case sensitive!

    • Fill in the Work Directory Path. This can be any temporary folder like /tmp.

    • Press Enter to continue.

  • Select Continue with Installation.

 

Select Server Engine dialog

 

 

  • Accept the default settings. Select Continue with Install.
