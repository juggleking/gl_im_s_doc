Progress OpenEdge installation on Windows

 

There is no need to install a base version (e.g. 11.7), followed by a service pack (e.g. 11.7.5) anymore.  As of Progress OpenEdge 11.7, the minor versions have become full standalone installers.

 

Run Setup.exe from the Progress OpenEdge installation files.

 

Click the Next button on the Welcome screen.

 

 

In the Serial & Control Numbers dialog, click the Load... button and select the license file provided by MIPS.

This will load all your available Progress OpenEdge licenses. The required licenses depend on the installation scenario as listed below. Highlight the unnecessary licenses for the given scenario and click the Remove button to remove them.

In the Choose Destination And Working Path Directories dialog, set the Destination Directory to <Drive:>\dlc117 and the Working Directory to<Drive:>\Temp as seen in the example below. Other directories can be used for the installation when needed, but avoid spaces in their names.

DLC stands for "Data Link Corporation", as Progress Software was called in the old days. It's still a convention in the Progress world to use "DLC" to refer to the Progress OpenEdge installation folders.

 

 

In the Choose Management Destination and Working Path dialog, set the Destination Directory to <Drive:>\dlc117\oemgmt and the Working Directory to <Drive:>\Temp as seen in the example below. Other directories can be used for the installation when needed, but avoid spaces in their names. Installing the management console in the oemgmt subfolder of the OpenEdge installation folder results in a cleaner directory structure and allows easy patching when needed.

 

 

In the Setup type dialog, choose Complete. The name of the option is somewhat misleading, as only the components related to the given licenses will be installed.

 

 

In the Configuring/Installing Components dialog

  1. Deselect WebSpeed with Local Web Server.
  2. Deselect OpenEdge Adapter for Sonic ESB.
  3. Select OpenEdge Explorer.

  4. Microsoft .Net Framework 4.6 must be selected.  It will be grayed out if already installed. This is usually the case on more recent operating systems.

    Note

    If the Microsoft .Net Framework 4.6 does not install correctly during the Progress installation, it can be installed manually from the "3party\dotnetfx\eng" folder in the Progress installation files

 

 

In the Select Program Folder dialog, keep the default program folder.

 

 

In the Select Server Engine dialog, the selection of Install the SQL Server Engine depends on the installation scenario as listed below.

  Server installation Client installation
Using a Progress database

Using an Oracle database

 

 

In the Language Choice dialog, choose English - American as the default language. It is not necessary to install other languages. The GLIMS user interface language is independent on the Progress OpenEdge database language chosen here.

 

 

 

In the International Settings dialog, set the

  1. Character Set, Collation and Case to ISO8859-15 Basic-Basic.
  2. Date Format to dmy.
  3. Number Format to 1,234.56 (comma, period).

These settings are being used by the database tools only. GLIMS has its own international settings.

 

Don't change anything in the Web Services Adapter Settings dialog.

 

 

 

Don't change anything in the AdminServer Authorization Options dialog.

 

 

A dialog showing a Summary of your choices appears. Click Next to continue.

 

 

 

The installation continues for some minutes.

 

When the installation is ready press Finish in the Complete Setup Done.
