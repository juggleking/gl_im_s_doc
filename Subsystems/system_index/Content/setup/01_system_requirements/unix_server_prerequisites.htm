Prerequisites for a Unix server

- Provide/create a user on the system with non-root rights.

- Make sure gcc-c++ and ksh are available on the system.

- Create the following symbolic links:

  • ln -s /bin/ksh /usr/bin/ksh
  • ln -s /usr/bin/gzip /usr/bin/compress
  • ln -s /usr/bin/gunzip /usr/bin/uncompress

- Add the following lines to /etc/profile (update the paths where needed):

  • export DLC=/mips/dlc116
  • export GLIMS=/mips/glims9
  • export INSTALL_DIR=/mips/install/glims/install
  • export PATH=$DLC/bin:$GLIMS/bin:$PATH

Background:

  • Compress and uncompress are used by pinstall to install Glims, so we link to gzip and gunzip
  • The install scripts make use of KornShell (ksh), and expect ksh under /usr/bin/ksh

- If the file /etc/magic contains no entry’s except comments, rename the file to /etc/magic.bkp

  • The file may not be empty for the GLIMS installer to work, if the file has entry’s besides the comments keep the file in place.

- Add services to /etc/services

  • glimsgenrwsv 15000/tcp
  • glimsglimssv 15001/tcp
  • glimsgenrosv 15002/tcp

- Untar the OpenEdge setup

  • tar -xvf PROGRESS_OE_11.6_LNX_64.tar.gz

- Untar the OpenEdge ServicePack

  • tar -xvf PROGRESS_OE_11.3.6_LNX_64.tar.gz

- Uncompress the GLIMS 9.8 installer

  • uncompress Glims_9.8.0_Progress_Unix_cdrom_pid1.tar.Z

- Untar the GLIMS 9.8 installer

  • tar -xvf Glims_9.3.0_Progress_Unix_cdrom_pid1.tar
