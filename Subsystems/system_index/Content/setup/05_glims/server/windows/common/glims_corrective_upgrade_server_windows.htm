GLIMS - Corrective upgrade on a Windows server

Upgrade procedure

  1. If you are not on GLIMS 9.9 yet, please switch to the minor upgrade procedure.
  2. If you are already on a GLIMS 9.9 version, take the following steps:
    1. Make sure all clients are disconnected
    2. Make sure the application system is fully stopped (translators, services and appservers): mam glims stop
    3. Run the corrective version self extractor (e.g. Glims9.8.2Progress116_pid3.exe)
    4. Make sure that the "Extract to:" path points to your GLIMS installation folder
    5. Click the "Extract" button and wait for the extraction to complete (e.g. c:\glims9)

    6. Restart the application system: mam glims start -n -a

 

Do not copy individual files (e.g. corr.pl) from one system to another. Always extract the full service pack into the GLIMS folder. Some files are dependent on other files in the service pack.
