GLIMS - Clean installation on a Windows server with an Oracle database

This procedure should only be executed by a MIPS representative.

Prerequisites

Oracle Database software needs to be installed before any of the GLIMS installers can be executed.

In a single server environment, a side-by-side installation of the 64 bit Oracle Database Server software and the 32 bit Oracle Database Client software (type Administrator) needs to be available.

In a multi server environment, also known as a three tier environment, in which the database and the application server run on different machines, the database machine should carry the 64 bit Oracle Database Server software, while the application server carries the 32 bit Oracle Database Client software (type Administrator).

A quick overview of the Oracle Database software installation can be found here:

  • Oracle Database Server installation
  • Oracle Database Client installation

More detailed information can be found in the Oracle Database installation manual.

 

This installation procedure should be executed after creating the GLIMS database. An automated installation procedure is available for Windows as described in the Oracle DB Creation manual.

 

A listener must be available on the database server. More information can be found about this in the Oracle Database Server installation guide.

 

The procedure below needs to be executed on the machine running the GLIMS application server.

 

Installation

Extract the installation files you have downloaded from the MIPS website in an installation folder of your choice.

Start the InstallShield setup by running setup.exe from the installation folder.

 

Wait for the initialization to complete.

 

A welcome dialog is shown. Click Next to proceed.

 

The setup will show a warning screen when the system requirements are not fully met.
Warning messages are not blocking the setup.
Fatal messages stop the installer from continuing.

 

Optionally you can press the Open logfile button to get more details.

 

Please accept the License Agreement.

 

Choose the application root folder in which GLIMS 9 will be installed.
E.g.: C:\glims9

 

Most setups can be performed with the Typical option.

 

However, if you want to install rather rare components like the CPage interface, the Carte Vitale drivers or the German billing data files, you can choose for a Custom install. In this case the following screen will be shown.

 

Select the Oracle home folder to be used. An Oracle home folder is the installation folder of an Oracle Database Client software installation. You can choose between multiple home folders if multiple Oracle Database Client software versions reside on the same machine. Note that the application server installer only detects 32 bit versions of the Oracle Database Client software.

 

The Net Service Name or TNS Name is the client side identification of the database. This name can be different from the database name as defined on the server. The Net Service Name is defined in the tnsnames.ora file and points to the name and listener port of the server.

 

If the Net Service Name is not known on the client yet, the installer will ask for the Oracle Database Service Name. A database can have multiple Service Names, but usually the Global Database name is being used, which is the database name, as known on the server, followed by the domain name. However some customers have different configurations. Check the right value to specify here with DBA.

 

Enter the database server name. If the database and the GLIMS application server are running on the same machine, you can just specify localhost here.

 

Enter Oracle TNS Listenerport for the database.

 

The temporary folder is typically located under the GLIMS root folder.
E.g.: C:\glims9\tmp

 

Specify the name server listening port. In most cases, the proposed default value can be used.

 

Specify a log directory for the name server.

This directory should never be a subdirectory of an existing application, including the installation folder of the ongoing GLIMS installation.

 

Specify the application server listening ports. In most cases, the proposed default values can be used.

Most of the GLIMS clients will use the Production application server. For debugging purposes, a client can switch to the Debug application server, which generates more logging and which can be reconfigured or restarted independently from the production environment.

 

Select the required Date Format option. The default option is read from the Windows settings.

 

Select the required Number Format option.

For compatibility reasons with external software, MIPS strongly recommends using the American number format.

 

Click Install to begin the installation.

 

Wait while GLIMS is being installed.

 

When all GLIMS components are installed, the Mips Application Management Scripts installer will be started. Wait for the initialization to complete.

 

A welcome dialog is shown. Click Next to proceed.

 

Select the installation folder for the scripts.

As the scripts are independent of the GLIMS version, they should always be installed in a dedicated folder. This folder should never be a subdirectory of an existing application.

 

Press Install to begin the MAM Scripts installation.

 

Press Finish when the MAM Scripts setup is ready.

 

Optionally you can review the GLIMS installation log file.

 

Press Finish to close the GLIMS setup.

 

A reboot is recommended.

 

Starting the first GLIMS session

When starting the GLIMS Server for the first time an automatic version update from version 9.1 to the target GLIMS version will take place.

No user interaction is required.
