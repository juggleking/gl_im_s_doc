GLIMS - Clean installation on a Windows server with a Progress database

Introduction

This document describes the installation of a GLIMS 9 server for use with a Progress database.

This procedure should only be executed by a MIPS representative.

Installation

Extract the installation files you have downloaded from the MIPS website in an installation folder of your choice.

Start the InstallShield setup by running setup.exe from the installation folder.

 

Wait for the initialization to complete.

 

A welcome dialog is shown. Click Next to proceed.

 

The setup will show a warning screen when the system requirements are not fully met.
Warning messages are not blocking the setup.
Fatal messages stop the installer from continuing.

 

Optionally you can press the Open logfile button to get more details.

 

Please accept the License Agreement.

 

Choose the application root folder in which GLIMS 9 will be installed.
E.g.: C:\glims9

 

Most setups can be performed with the Typical option.

 

However, if you want to install rather rare components like the CPage interface, the Carte Vitale drivers or the German billing data files, you can choose for a Custom install. In this case the following screen will be shown.

 

Choose the database destination folder.

 

Set the initial database size for the glims and genrw databases.

 

For most customers, 4 GB is a good starting point for the genrw database.

For the glims database, the following rule of thumb can be applied:

  • Start with the number of orders the lab processes per day.
  • Multiply this with 25k. This is the average size of an order in Glims.
  • Multiply this with 365 days per year.
  • Multiply this with the number of years the lab wants to keep data online

 

Example:

A lab processes 3000 order per day and wants to keep 5 years online.

3000 orders/day * 25k per order * 365 days * 5 years = 136.875.000k = +/- 138 Gb

Note that a database size can always be extended. There is no need to assign the full database size from the early beginning.

 

The installer will configure the database connection parameters based on the number of simultaneous GLIMS clients (GUI) and the number of analyzer connections.

 

In the next screen, you can choose a service name and a port number for the genro database. Check the Progress OpenEdge licenses for the existence of a "OE Personal RDBMS" license. If this license is available and installed during the Progress OpenEdge documentation, these fields can be left blank. If no such license is available, a random free port number should be chosen here.

 

Provide the service name and port number for the genrw database. In most cases, the default value can be used.

 

Provide the service name and port number for the glims database. In most cases, the default value can be used.

 

The temporary folder is typically located under the GLIMS root folder.
E.g.: C:\glims9\tmp

 

Specify the name server listening port. In most cases, the proposed default value can be used.

 

Specify a log directory for the name server.

This directory should never be a subdirectory of an existing application, including the installation folder of the ongoing GLIMS installation.

 

Specify the application server listening ports. In most cases, the proposed default values can be used.

Most of the GLIMS clients will use the Production application server. For debugging purposes, a client can switch to the Debug application server, which generates more logging and which can be reconfigured or restarted independently from the production environment.

 

Set the required language. This setting allows the correct configuration of the internationalization options for your GLIMS installation. Note that the chosen language is not the default language that will be used in GLIMS.

 

Select the required Date Format option. The default option is read from the Windows settings.

 

Select the required Number Format option.

For compatibility reasons with external software, MIPS strongly recommends using the American number format.

 

Click Install to begin the installation.

 

Wait while GLIMS is being installed:

 

  • Files are being copied.

 

  • Databases are being created.

 

  • The database scheme is being loaded

 

When all GLIMS components are installed, the Mips Application Management Scripts installer will be started. Wait for the initialization to complete.

 

A welcome dialog is shown. Click Next to proceed.

 

Select the installation folder for the scripts.

As the scripts are independent of the GLIMS version, they should always be installed in a dedicated folder. This folder should never be a subdirectory of an existing application.

 

Choose the disaster recovery role for this machine and press Next.

There are 3 options to choose from.

  • Production system: Configures a backup, enables after imaging and sets the database as replication source.
  • Standby system: Configures the database as replication target.
  • None: Does not configure backups nor after imaging or replication.

 

Select a location for the backup files. This must be another physical location then the location where the GLIMS database is stored.

 

Select a location for the after image files. This should preferably be another physical location then the location where the GLIMS database is stored.

The after-image files can be stored together with the backup.

 

Set the time when the backup should run.

 

Set the interval between 2 after image archiving operations. This defines the maximum amount of data a lab can loose in case of a database crash.

 

Press Install to begin the MAM Scripts installation.

 

Wait while the MAM Scripts are being installed.

Depending on the chosen installation type, backup or restore operations will take place to enable after imaging and replication.

 

Press Finish when the MAM Scripts setup is ready.

 

Optionally you can review the GLIMS installation log file.

 

Press Finish to close the GLIMS setup.

 

A reboot is recommended.

 

Starting the first GLIMS session

When starting the GLIMS Server for the first time an automatic version update from version 9.1 to the target GLIMS version will take place.

No user interaction is required.
