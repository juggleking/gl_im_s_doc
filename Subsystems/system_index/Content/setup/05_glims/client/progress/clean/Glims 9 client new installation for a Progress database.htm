GLIMS - Clean installation on a client with a Progress database

Introduction

This document describes the installation of a GLIMS 9 client for use with a Progress database.

Installation

Extract the installation files you have downloaded from the MIPS website in an installation folder of your choice.

Start the InstallShield setup by running setup.exe from the installation folder.

 

Wait for the initialization to complete.

 

A welcome dialog is shown. Click Next to proceed.

 

The setup will show a warning screen when the system requirements are not fully met.
Warning messages are not blocking the setup.
Fatal messages stop the installer from continuing.

 

Optionally you can press the Open logfile button to get more details.

 

Please accept the License Agreement.

 

Choose the application root folder in which GLIMS 9 will be installed.
E.g.: C:\glims9

 

Most setups can be performed with the Typical option.

 

However, if you want to install rather rare components like the Java Communcation Engine, the Carte Vitale drivers or the German billing data files, you can choose for a Custom install. In this case the following screen will be shown.

 

The temporary folder is typically located under the GLIMS root folder.
E.g.: C:\glims9\tmp

 

Provide the servername or IP address of the GLIMS server.

 

In the next screen, you can choose a service name and a port number for the genro database. Check the Progress OpenEdge licenses for the existence of a "OE Personal RDBMS" license. If this license is available and installed during the Progress OpenEdge documentation, these fields can be left blank. If no such license is available, the same values that were given during the server installation should be specified here. When in doubt, the name and value can be found in the Windows services file on the server (c:\windows\system32\drivers\etc\services).

 

Provide the service name and port number for the genrw database as configured on the GLIMS server. When in doubt, the name and value can be found in the Windows services file on the server (c:\windows\system32\drivers\etc\services). In most cases, the default value is being used.

 

Provide the service name and port number for the glims database as configured on the GLIMS server. When in doubt, the name and value can be found in the Windows services file on the server (c:\windows\system32\drivers\etc\services). In most cases, the default value is being used.

 

Specify the name server listening port as chosen on the GLIMS server. In most cases, the proposed default value is being used.

 

Set the required language. This setting allows the correct configuration of the internationalization options for your GLIMS installation. Note that the chosen language is not the default language that will be used in GLIMS.

 

Select the required Date Format option. The default option is read from the Windows settings.

 

Select the required Number Format option. For compatibility reasons with external software, MIPS strongly recommends using the American number format.

 

Click Install to begin the installation.

 

Wait for the installation to finish.

 

Choose Finish when the setup is ready.

 

Starting the first GLIMS session

 

When starting the GLIMS Client for the first time an automatic version update might take place. This merely depends on the versions being installed.

 

 

Next the GLIMS login screen is shown.
