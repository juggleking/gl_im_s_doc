OCX registration

The default location of an ocx file supplied with your application is the ocx sudirectory of the application. Examples: D:\GLIMS7\ocx\MIPSGraph.ocx, D:\GLIMS7\ocx\mipsimageview.ocx.  This is also the place where the program searches for the ocx file i when the application tries to do a registration of this ocx.

New functional or corrective versions may replace the ocx file on this directory with newer versions.  In this case, no re-registration should be required.  If you suspect that your installation is using the wrong ocx version, do the following (supposing the application is installed under D:\GLIMS7).

  1.  quit all GLIMS applications on this computer

  2. open a DOS (or command) box

  3. type regsvr32 D:\GLIMS7\ocx\mipsgraph.ocx (use correct application path!)

Make sure the following ocx files no longer reside in one of the Windows or WinNT directory trees. The correct location is now in the application ocx subdirectory. Remove the versions which can be found in the system directories.
