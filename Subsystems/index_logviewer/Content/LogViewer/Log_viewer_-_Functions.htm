Functions

Open a file

You can open a log file from the file system or via FTP.

File system

There are different ways to open a log file:

  • Use the Open file button.

  • Drag a file from Windows Explorer e.g. and drop it into the log viewer.

    Note: LogViewer also supports opening an entire folder through drag and drop. Instead of having to open every file one by one in a folder it is possible to simply drag an entire folder. The tool will then open up all the files in the directory. Please note that the tool will NOT go through subdirectories.

  • Select one of the options View log or View log of translator and service in the application.

FTP

This option allows you to open a log file by means of a FTP connection on the server.

Note

In order to open a file via FTP, the configuration of FTP users is necessary. For more information about the configuration, please click here.

Open a log file via FTP

  1. Click the Open file button and select FTP. A window to enter your password will open.

  2. Enter your password. A window to select a log file will open.

  3. Select the log file by browsing to the specific directory. The FTP explorer window contains the following information:

Home directory

Directory to which you have initial access. It is basically the starting point for from where all browsing needs to occur. This field is not editable.

Current directory

Current directory relative to the Home directory. This field is not editable.

Display only files with extension 'log'

Allows you to indicate if you only want to view files with extension ‘log’. This is the default extension of MIPS log files.

D/F

Directory or file.

Name

Name of the directory or file.

File size

Size of the file in bytes.

Permissions

Permissions (read, write, execute) you have on the file.

Date

Date when the file was last changed.

Owner

Owner of the file.

Group

Group that owns the file.

Remark

It is possible that the FTP explorer does not display any information about the content of the FTP directory.

The FTP explorer is able to retrieve information from the FTP server but does not succeed in parsing it. The parsing of this information is done by using a regular expression. In this case a possibility has been built into the Log viewer to provide an additional site-specific regular expression. In order to use this ability the following needs to be done:

  • Activate debugging. Click here to know how to activate debugging.

  • Attempt to read the content of a FTP directory.

  • Close the Log viewer.

  • Provide the output of the DEBUG file to MIPS.

  • MIPS will provide a file FTP_NLIST_REGEX.TXT allowing to correctly parse the content of the data sent by the FTP server.

Details of a log entry

When you double-click a log entry, you will have a detailed view of this entry. The entire message, which is sometimes not completely visible in the main screen, is displayed and also the source, date, time, action and channel.

You can also decide in the detail screen whether you want to show <CR> and <LF> in your message and/or to display XML messages as plain text.

When the button is enabled, it is possible to open several detail windows. When the button is enabled, it will only be possible to open a detail window of one log entry. Once a log entry detail screen has been opened, you can browse through the lines in the general screen and the detail screen will be synchronized with the selected entry.

Set filter

On behalf of this option, you can set a filter on the log file which allows you to view only certain log entries.

View the entries of a specific log type

  1. Click the Set filter button.

  2. Select a log type. Additionally you can refine your selection by adding the date and time, the type of source...

  3. Finally click OK. You should now only see the log entries of the type you selected.

View the entries based on a specific message

 Open the Set filter window and enter a text in the Message field. It is not necessary to enter the complete text of a message, you can also filter the entries based on a word.

View the log entries of two different log types

Select a specific log type (e.g. error) in the first tab page and another one (e.g. warning) in the second tab page. The Operator should be set to OR. You will now have a list of all the error and warning entries.

Set colors

The Set colors screen is composed the same way as the Set filter en Search screen. In this screen it is possible to modify the colors of the log entries. An error needs to attract attention and should be displayed in red for example.

  1. Click the Set colors button.

  2. Select a log type.

  3. Change the foreground and/or background color.

  4. Click OK. The log type of which you modified the color should now be displayed in the color you selected.

In the additional tab pages you can modify the color of the other log types.

Search

The Search screen allows you to search quickly for a specific log entry. You can search based on a log type but also on the message of a log entry.

  1. Click the Search button or press CTRL+F.

  2. Select a log type or enter part of the message you want to search on.

  3. Click OK. The log entry matching the specified criteria will be selected. To go to the next log entry that matches the specified criteria, click the button or press CTRL+F.

Go to line

To go to a specific line in the log file, press the key combination CTRL+G. A new window will open allowing you to enter the line number. To confirm the inputted line, press ENTER. To exit this screen, press ESC.

Please note that this functionality will not work if you have set a filter on the log file and the line number does not belong to your selection. In that case first remove the filter and then search for the specific line.

Follow mode

When this button is activated, the log viewer will scroll down (up) automatically to the last (first) line as lines are added to the file.

Stop current tasks

When a progress bar is displayed then it is possible to halt whatever operation has caused the progress bar to be displayed.

Synchronize the log file view

This functionality is only supported by the Dual log viewer. It provides the ability to auto-synchronize opened log files based on the date time entries in the log file. When this functionality is enabled, the follow mode is deactivated and the sorting order is set to ascending.

When clicking an entry in one view, the other view will be synchronized based on date and time. The entry selected in the other view, is the first entry that matches the date and time of the selected entry in the following way:

  • The date and time entry must have a valid value.

  • The date and time entry must have a value equal or lower than the one selected.

  • If several entries match the above criteria then the first entry found, will be selected.

You can stop the synchronization by clicking the button. To switch back to the follow mode, click the according button.

Copy to the clip board

This functionality allows you to copy (CTRL+C) the selected log entries and to paste (CTRL+V) them in another application.
