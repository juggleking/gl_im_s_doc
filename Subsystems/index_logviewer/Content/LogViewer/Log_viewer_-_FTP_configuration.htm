FTP configuration

Configure FTP users

Add a user

  1. Click the Configure FTP users button. A new window will open.

  2. Click the Add button and enter the following information:

Description

General description to describe the connection in order to allow easy browsing in a possible list.

Login

The login that needs to be used when establishing the FTP connection.

Note

This user must be given the appropriate access on the FTP server. If the targeted FTP server is case sensitive, then this must also be taken into account when entering the login.

Host

IP address or name of the computer acting as FTP server.

The Log viewer will check its validity when entered. The Log viewer will attempt to send a "ping” to the server. If no appropriate reply is received then the name/IP address entered, will be considered invalid and will thus be rejected.

Refresh

The rate, in seconds, at which the Log viewer will check if the file being viewed is updated or not. The minimum imposed value is 10 seconds.

Local folder

Local folder where a copy of the file is stored.

When closing this window, the information will be saved.

Additional information

  • A user created in Multi log viewer is also created in Dual log viewer. There is thus no need to do the configuration twice.

  • The information entered in this screen, is stored in a XML file (FTPusers.xml) which can be found in the directory where Log viewer is installed.

Delete a user

  1. Click the Configure FTP users button. A new window will open.

  2. Click the Delete button.

Override configuration of users

It is possible to override the need to configure FTP users and to provide a FTP password.

How?

Put the file FTP_AUTOLOGIN.TXT (is case-sensitive) in the installation directory of the Log viewer. This file must contain the following lines separated with <enters> without any indication to their content.

<FTPServer>

<FTPUser>

<FTPPassword>

<RefreshRate>

<LocalDownloadfolder>

Note

MIPS sees this as a possible security risk as the file needs to hold the data in an unencrypted form. This means that the FTP credentials, including the FTP password, are humanly readable !!!

This possibility has been provided in case a specific FTP user is created at customer site who would only be used to access the appropriate top-level logging directories. Simply distributing this file to all parties who need to be able to perform FTP based logging, avoids the need to configure this manually on a long list of PC’s.

Activate debugging

It is possible to enable a detailed logging of the Log viewer when using FTP as an intermediate tool to consult log files. This detailed logging is mainly meant for possible debugging with end users.

In order to enable this logging, put the file FTP_DEBUG.TXT (is case sensitive) in the installation directory of the Log viewer. This file can be empty as it is mainly intended to indicate whether or not logging is active.

Note

Putting this file in place requires restarting all active FTP connections on the PC.

All logging will be sent to the file FTP_DEBUG_<SomeString>.<SomeString>.LOG. This file will be placed in the installation directory of the Log viewer. For every Log viewer session, one file will be created.

If there are any problems with the FTP explorer or any of the FTP functionalities, this file enables you to provide the required data to MIPS for debugging purposes.

Note

These files are not cleaned up automatically and the Log viewer applications will continue to create these files until the disk space is used up.

Do not activate this option unless MIPS tells you to do so and remember to clean up when debugging is no longer necessary.
