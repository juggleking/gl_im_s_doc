Release notes v6.0.0 - WatchdogDaemon - ServiceController interaction (WatchDog-00241)

An issue was reported where a timeout in the execution of a Start/Stop request from the Watchdog Dashboard could lead to a cascade of alerts that services might be disconnected, although they were regularly sending their 'Alive' messages.

The underlying cause was that while Watchdog Daemon was handling the Start/Stop request from the Watchdog Dashboard, all 'Alive' messages were queued, and the processing of these 'Alive' messages was delayed until the Start/Stop request was finished. In the meantime, separate timer threads were checking the status and last update time of the services, and concluded that they hadn't sent an 'Alive' message.

This has been corrected, and the handling of Start/Stop requests from the Watchdog Dashboard no longer delays the processing of 'Alive' messages.
