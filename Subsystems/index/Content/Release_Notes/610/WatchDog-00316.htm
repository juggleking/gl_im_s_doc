Release notes v6.1.0 - Extend Service Controller threshold for Undetermined status (WatchDog-00316)

Services of type Service Controller are expected to send a 'ServiceControllerAlive' message to Watchdog Daemon every 10 seconds.

Watchdog Daemon regularly checks that it has not timed out, and if the Daemon hasn't received an alive message in over a minute, the status is changed to Undetermined. After another minute, the status is changed to Disconnected.

However, if it takes a Service Controller a very long time to handle a request from the Daemon, then we may get a false alert that its status has changed to Undetermined - simply because the Service Controller could not send an alive message while it was processing the request.

To prevent this type of false alert, we have extended the threshold where we consider the Service controller to be Undetermined / Disconnected.
