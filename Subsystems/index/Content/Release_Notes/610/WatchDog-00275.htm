Release notes v6.1.0 - Accept multiple incoming connections in Daemon (WatchDog-00275)

In previous versions, the Watchdog Daemon main loop only accepted one connection at a time, read the message, and put it on an internal queue for further processing in a separate thread. While this ensured temporal linearity, it constituted a bottleneck for the application processes which wanted to send their data to WatchDog Daemon.

We have revised the way Watchdog Daemon handles incoming connections, in such a way that multiple application processes can submit messages at the same time, while preserving the order and thus the model status integrity.
