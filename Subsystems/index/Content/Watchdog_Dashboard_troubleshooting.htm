Watchdog/Dashboard troubleshooting

I cannot start/stop a service/translator from the dashboard.

  • Make sure that the Windows user corresponds to an application user having sufficient privileges to start/stop services/translators.
  • Make sure you have correctly defined the XML control service

My changes in the application to be monitored do not have any effect.

Such changes often require a restart of the Watchdog service.

Changing preferences in Dashboard does not have any effect.

  • You need to click the Apply button at the bottom of the screen before the preferences are actually used.
  • Such changes often require to close and restart the Dashboard.

The daemon.log says "DaemonApplication - Argument <port> must be an integer (ERROR)"

Do not register the Watchdog service using an alias to the etc/services file.  The port needs to be numeric !
