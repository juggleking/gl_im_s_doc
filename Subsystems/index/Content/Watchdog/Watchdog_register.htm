Registering the Watchdog service/daemon

When configuring Watchdog for the first time, you need to:

  1. Define an XML control service.
  2. Register Watchdog (see below).
  3. Start Watchdog.

Register the Watchdog service/daemon

To register Watchdog:

  1. Go to the Register window (System Management > Services > Watchdog > Register).
  2. Fill out the fields as appropriate (see below).
  3. Click OK.

Note

Make sure you register WatchDog on the main application server. See also Watchdog in a multi-server environment.

Fields

Service name

Service name is the name of the Windows service to be registered. Unless a Windows service with that name is already registered (e.g. when you have multiple application instances running on the same machine), it can be left to the default value.

Folder

Folder is a reference to the folder where Watchdog is installed. It defaults to the "Watchdog" subfolder of the application root folder.

Server

Server contains the name or IP address of the server.

Port

The TCP/IP port where Watchdog is listening.  

Warning

Do not register the Watchdog service using an alias to the etc/services file.  The port needs to be numeric ! If not, the daemon log file will contain the following error message: "DaemonApplication - Argument <port> must be an integer".

User login name

User login name is a Windows user account that will be used to run the service. This account does not need administrator privileges but it needs the user right Log on as a service.

If running in a domain, use <Domain name>\<Login name>. Else, use <Computer name>\<Login name>.

User password

User password is the user's Windows password.
