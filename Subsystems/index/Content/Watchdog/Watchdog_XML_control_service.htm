Define the XML control service

When configuring Watchdog for the first time, you need to:

  1. Define an XML control service (see below).
  2. Register Watchdog.
  3. Start Watchdog.

Define an XML control service

An XML control service is required to:

  1. start /stop services and translators from within the dashboard
  2. have Watchdog send alerts via internal application mail
  3. exchange database status information

To define an control service:

  1. Go to the Services window (System Management -> Services -> Services).
  2. Press Insert to add a new service.
  3. In the Name field, enter a name for the service.
  4. Set the Internal language to XML.
  5. Set the Type to Service controller.
  6. Enter a Port.
  7. Enter the Service group.
  8. In the Seq no field, enter a sequence number.
  9. All other fields can be left unchanged.

Which XML control service is used

When starting/stopping a service/translator from the Dashboard, Watchdog looks at the Host of the Service / Translator the user is requesting to start/stop, tries to find a matching Control service running on that host, and then forwards the request to that Control Service.

If Watchdog cannot find a match (based on the IP address of the host), the request is forwarded to the Watchdog Control service (Service specified as Control service when starting WatchDog).
