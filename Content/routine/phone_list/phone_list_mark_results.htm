Phone list - Mark results as "to be phoned"

Contents: Order entry - Result.ToBePhoned - Result editor - Via a severity - Via a phone urgency - Via MISPL - Exclude a property - Remarks

Introduction

In order for a result to appear on the phone list, it must be marked as To be phoned.

GLIMS offers several ways to accomplish this.

Order entry

Phone results

Enable the Phone results option in the Extra tab page of the Order entry screen and all explicitly requested properties of the order will be marked as To be phoned.

Notes

  • When disabling this option afterward, the phone status of the properties to be phoned (which have not yet been phoned through) will be reset. Note that this does not apply to results which have automatically been marked as to be phoned (via the phone severity or phone urgency).
  • When adding requests to an existing order, the additional results will always take this setting into account.

  • Restrictions:

    • the property should be externally visible (Property.Visible = Yes)
    • the property should have a report format different from Hidden (Property.ReportFormat <> Hidden).
    • the option On phone list should be activated.

Phone number

The Phone field in the Extra tab page of the Order entry screen allows entering the phone number to be used. Note that you can still select another phone number when actually phoning the results.

Phone status

The Phone status field in the Extra tab page of the Order entry screen provides a summary of the phone status of all results in the order.

No

There are no results to be phoned.

Yes

There is at least 1 result that still is to be phoned.

Done

All results are phoned.

Contextual function "To be phoned"

You can select one or more results and choose the contextual menu / ribbon item To be phoned.

Result editor

In the Result editor, you can manually change the Phone status.

Via a severity

Phone list severity of a result

A result can have a phone list severity which is used to decide whether a result should be marked as To be phoned.

When the value of a patient result is outside the norm range, its phone list severity will inherit the Severity defined for the norm if the Norm option For phone is enabled.

Phone severity limit

In the assessment method, you can specify a Phone severity. The Phone severity defined in the Assessment method is compared with the maximum of the result's phone list severity, delta norm severity and manual severity to decide whether the result should be marked as To be phoned.

Notes

  • The phone severity is not checked when the Property option Visible is set to Internal

  • The phone status of a result which was automatically marked as To be phoned via the AssessmentMethod.PhoneSeverity is automatically cleared when the result is repeated, modified or recalculated.

Via a phone urgency

Phone urgency range

The general options (Start -> System management -> Customize -> GLIMS General) allow to specify a Phone urgency range. Results with an urgency within this range will automatically be marked as To be phoned.

Notes

When only the From field is specified (maximal urgency = ?), all results with an urgency equal or higher than this minimal phone urgency will be marked as To be phoned.

Via MISPL

Result.ToBePhoned

Allows to mark an individual result as To be phoned. Can e.g. be used in a Property trigger.

Order.ToBePhoned

Allows to mark a complete order as To be phoned. The effect is the same as if the order was marked manually at order entry.

Notes

These MISPL functions have some built-in restrictions:

  • They will not mark Discontinued results.
  • They will not mark results where .Property.ReportFormat = Hidden.
  • They will only mark results for which at least one property classification node can be found.

Exclude a property from the phone list

Property.OnPhoneList

If the Property option On Phone list is disabled, results for this property will never be shown on a phone list. By default, this option is enabled.

Remarks

Track how phone status was reached

The field Result.PhoneMarkMethod keeps track of how the current phone status of the result was reached. This field is used to decide whether to activate or deactivate the current phone status when the value, severity or urgency of a result changes.

Its possible values are:

Manual

Automatic

Example: via AssessmentMethod.PhoneSeverity.

Urgency

Example: via SpecificSite.MinimalPhoneUrgency.

Reset automatically assigned phone status

If you want to reset the phone status of results that have automatically been marked as To be phoned (e.g. by means of the phone severity or phone urgency), you must do this explicitly by using the phone list program.You can not do this by disabling the Order.PhoneResults field.
