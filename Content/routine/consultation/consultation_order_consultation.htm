The order consultation program

Contents: Introduction - Query - Browser

Introduction

The order consultation query is accessible via the main menu (Start -> Routine -> Orders -> Consultation query) or via the Orders of object browser.

Note

The Patient and Study query options cannot be combined ! You can only select a study / study episode/ study object or a patient, not both at the same time.

Query

Note

The Patient/Animal option at the bottom of the screen allows to search orders either by patient or animal characteristics.

Patient

Single patient

The order consultation query allows to look up all orders of a given patient.  Available query options are:

PIN / Assigned by

Allows the user to query the orders of the patient with this PIN.  You can double-click to get a full list of PIN's (identifications).  If a PIN provider is specified in the assigned by field, only the PIN's assigned by the specified provider will be shown in the list.

Internal id

Encounter

Name (Last name and first name)

Birth date

Spouse name

Externalization

This field shows details of the selected patient.  In order to clear the patient data, enter a '?' in the Name field.

Tips

  • If activated, you can use the advanced person search screen.
  • GLIMS allows using the Belgian electronic identity card (eID) for patient selection during order entry.

Multiple patients matching a pattern

It is possible that GLIMS has orders for several patients with the same/similar name. To ease the consultation in such cases, the order consultation program allows to look up all orders of ALL patients matching a specified pattern.

  1. At the bottom of the Order consultation query screen, an option Support object pattern search is available.
  2. If this option is enabled, a search box will appear at the top of the screen that allows to perform a wild card search on the object externalization.  All data in the externalization can be used.

Animal

The order consultation query allows to look up all orders of a given animal.  Available query options are:

Owner

Animal name

Object template

Study

This tab page allows to search orders for a specific study episode and study object. 

Study

Allows searching for orders of a specific study. This can be especially useful in department who do a lot of study-related work (e.g. Genetics).

Study episode

When selecting a study, all study episodes of that study will be available for selection. 

Study object

When a study is selected, the study object field allows to select a patient that participated in the selected study.

Correspondent

Correspondent

Any correspondent can be selected, along with its function in the order: as issuer, as agent, or as report receiver. 

Note

On selection of Report receiver, any order with a report targeted to the specified correspondent will be selected.  This option may slow down the query response time for correspondents with a large amount of reports. Use it with care!

Identification / assigned by

Allows the user to select a correspondent via his identification.  You can also select the identification provider.

For all group practice members

This option becomes active only when the specified correspondent (of type 'Health care provider') is a member of a group practice. When activated, the name of the practice is shown in the widget label.  If this option is checked, the order query will be executed for each member of the group practice.

Other

Department

Allows the user to search on to the Order.Department field.

The order department is set as follows:

Manual order entry

Department of the logged in user that created the order.

Electronic order entry

Department of the batch user linked to the service that processes the order messages.  This department can be overruled if a department is specified in the order message.

Executing department (group)

 Allows the user to search for orders that have at least one action that is executed in the specified executing department (group).  This is determined by the procedure's department.

Order status option

All

Query orders of any status.

Active

Query orders <> pending.

Pending

Query pending orders (see Electronic Order Scheduler).

Short id

Only orders with this short id.

Received since-until

Allows the user to specify a minimal and maximal order receipt date.  To limit the execution time and the result list of the query, the "Received since" option is mandatory. When searching very old orders, it is suggested to search via the patient name rather than moving the receipt date range back in time.

Filter

Optional MISPL filter to select orders that satisfy the specified MISPL expression.  Users of type "Guest" are not allowed to change the filter function.

Note

Some cases may yield a slow query, especially when using the Report receiver option.  Always try to combine the "Correspondent... as report receiver" option with another one (short id, patient). When not possible, try to limit the result list by choosing a minimal receipt date which is not too far in the past.

Browser

Lower left window

The lower left window is customizable by means of a text module called "OrderConsultationSummary".  If this text module should not exist, a default MISPL will be used: .Attribute("PropertyList")

Update and insert orders

It is possible to update and insert orders in the consultation browser if the user type is different from type 'Guest'. Deleting orders is not allowed.

When double-clicking a pending order, the order activation screen will be opened.  When double-clicking an active order, the order entry screen will be opened.
