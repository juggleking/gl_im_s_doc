Patient history graph

Introduction

GLIMS allows to show a result history graph for a patient that shows the results for multiple properties over a given period of time.

The graph is visualized with the MIPSGraph tool.

Tip

Color coding is used to show reference ranges (background), delta check violations (lines) and QC violations (dots).  The tool tip on the result shows more detailed information (property, result availability time, value, result flag, norm).

Access

Select an Object and choose the contextual function History graph. The following options are available.

Since - until

Date range for the results to be shown.

Property

Select the properties to be included in the graph.  You can display multiple properties simultaneously.

Logarithmic scale

When checked, a logarithmic scale will be used at redraw.

Logarithmic labels will always be powers of 10: 0.1, 1, 10, 100 etc. To use a logarithmic scale, the minimum Y values should be strictly positive (> 0).

Pre-configuration

When pre-configuring the History graph function as a Tool, the following options are available:

Since - until

Date range for the results to be shown.

Property

Select the properties to be included in the graph.  You can display multiple properties simultaneously.

Logarithmic scale

When checked, a logarithmic scale will be used at redraw.

Logarithmic labels will always be powers of 10: 0.1, 1, 10, 100 etc. To use a logarithmic scale, the minimum Y values should be strictly positive (> 0).

Ref. line / value

Allows to draw two reference lines on the graph.  Specify a label and a value.

Y-min / y-max

Minimum / maximum Y values: allow setting up a fixed scaling.

HTML version

GLIMS also offers a HTML version of the history graph. Use the Object function HTML history graph to generate a HTML document with a graph and a value table.
