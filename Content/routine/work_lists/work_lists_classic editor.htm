Work list - Classic editor

Warning

When creating and editing work lists in the context of the extended Genetics module, please use the advanced editor.

Layout of the classic work list grid

Most layout options can be defined in the work list template used to generate the work list.

If the option Use advanced editor was not activated in the Generate work list screen when the work list was generated, the work list grid opens in classic format:

Rows

The rows in the work list grid correspond to actions. An action is the realization of a procedure.

An action appears on a work list because the procedure is linked to the work list template via a work list template entry.

Columns

The columns in the work list grid correspond to action outputs, which are typically properties.

Cells

The actual grid cells show the results that are output of the actions.

Row labels

The default row label can be customized by a site function defined in:

  1. The work list template field Action label.

  2. If no Action label is specified in the work list template, the Action label specified in the general settings (Start -> System management -> Customize -> GLIMS General) will be used.

Colors

The color scheme defined for results is applied to the cells.

Note

The color scheme is not applied to results from procedure data.

Notation

Notation Explanation
@ Results which have not been requested (unsolicited and auto-prompt results) are prefixed with a @ character.
* Results with an internal or external comment are prefixed with a * character. Part of the comment will be visible via the tool tip by holding the mouse cursor over a result cell.
() Procedure data results that are available are enclosed in round brackets.
{} Procedure data results that are confirmed are enclosed in curly brackets.
  Procedure data results that are validated are not enclosed in brackets.
<Calc> A calculated result (i.e. a result where the PropertyOutput.Value contains a formula AND where the action has at least one input result) is shown as "<Calc>", as long as it does not have a value. This should warn the users that they are not supposed to enter a value for this result.  Calculated results are automatically refreshed when another result is modified or confirmed.
[] Used to display discontinued results without a value.

Notes

  • When a result is shown as 'data' (i.e. as procedure input or as procedure data), it is suppressed when it is also present as a normal procedure output.
  • Procedure data results that have been discontinued are not displayed in the worklist grid.

Customization

A number of options are available to customize the grid.  Please check the chapter User interface: grid options.

Functions available in the grid

Result entry

  • Result cells are enabled for input until the result is confirmed.
  • When positioned in a work list cell, use CTRL-F to select a choice or result code.

Row and column selection

Rows and columns can be selected together to speed up result confirmation, for instance:

  • Double click the row label to select an entire row.
  • Double click the column header to select an entire column.
  • Double click in the upper left corner (first cell or first column) to select all rows and columns.

Refresh

The Refresh button at the bottom of the screen updates the active work list. If a work list still expects results from an analyzer, refresh the work list before you close it, to see if they already appear on the work list.

Show details

When positioned in a work list cell, you can:

  • Zoom directly into the result hitting F6.

  • Call a number of functions by hitting CTRL-F6.
