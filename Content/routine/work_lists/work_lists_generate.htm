Basic routine -> Work lists -> Work list generation

Introduction

The manual work list generation function can be called from the main menu. There are two ways to generate a work list:

Using a work list template

Name

Allows to generate a work list based on one particular work list template. 

Group name

Allows generating work lists for all work list templates belonging to a given work list template group. 

Station

When selecting a single work list template (by using the Name field), the Station field below will indicate to which station this template applies.

Selecting procedures linked to a specified station

Select a station and then choose one or more procedures linked to this station. By hitting the Enter key, the procedures will be added in the window below.

Other fields

Name

Enter the name of the work list which your are about to generate.

Specimen

If set, you can scan or manually enter a specimen internal id or external id. Hit the return key to add the specimen in the window above. 

Each scanned bar code is first compared with SpecimenIdentification.Code and then with Specimen.InternalId and Specimen.ExternalId. This allows the usage of both the Specimen Identification barcode as the Internal ID or External ID barcode to identify a specimen. It is possible to select multiple specimen records by double clicking the specimen entry field and opening the SpecimensByInternalId browser. If the Specimen option is activated but the user selects Generate with no specimen specified, the Specimen selection screen is displayed, prompting the user to select at least one specimen record. The user can delete specimens one at a time by using the Delete key, or clear the list of the displayed specimens by using the Clear button on the screen.

When scanning a patient sample, the actions of the specimen are immediately subjected to ALL the options as specified in the generation setup screen, just like during the actual generation of the work list. This allows early detection of samples which do not belong on this work list.  

When a maximal entry count is specified, a warning is issued when trying to scan more specimens than this number.

Print

If set, the work list will also be printed after generation.  A print dialogue will first be displayed:

Specific options

Layout type

By specifying an unknown layout type, you can indicate that the layout should be taken from the template. If there is no template available, layout type "Grid" will be chosen.

Copies

Number of copies when printing to a printer.

Append

When exporting to a file ("Output to file" option), using the same file name, the context will be appended to the existing file.  This option is only supported when "Format" is set to "Text".

Default file name

Only supported when using this function in a command.

Output options

Print

Must be enabled to print the work list to a printer.  Note that "Print" and "Output to file" cannot be combined.

Printer

When the Print option is enabled, you can select a printer.

Output to file

Must be enabled to save the work list to a file.  Note that "Output to file" and "Print" cannot be combined.

Directory

When saving the work list to a file, this field indicates the directory in which the file should be saved.  Supports the variable {TempDir}.

Name

When saving the work list to a file, this field indicates the file name (and extension).  Supports the variables {Date} and # (sequence number).

Format

PDF and XML are only supported for work lists of type "Outline".  PDF work lists are printed with GLIMS Report Builder.

Preview

Shows a preview of the print-out.

Labels

In case of a regular work list: this toggle box allows to print the specimen input labels.
In case of a sampling list: this toggle box allows to print the specimen output labels.

Edit

If set, you will be able to edit the work list on screen immediately after the generation.

Include QC

If activated, it is also possible to scan QC samples when generating a scanned work list. When recognized, a QC action will be created for all QC populations of the corresponding QC lot and the assessment methods present in the work list template.  Any existing QC Schedule records will be ignored as soon as one scanned QC sample is detected.

Confirm specimens

When this option is set, all output specimens are confirmed when the work list is generated. This can be used in instances where during printing of sampling lists, we can assume that all the printed samples will be retrieved.

Activate

Sets the generated work list as active work list for the referred station.  For more information, please see the chapter Active work lists.

Use advanced editor

If this option is activated, the work list is generated in the advanced format. If it is not activated, it is generated in the classic format.

Note

Once a work list is generated in one format or the other, it will always open in this format. If you want to change the format, you need to (de)activate the option Use advanced editor and then regenerate the work list.

Type

It is possible to append actions to an existing work list, so it can grow during the day. The Type field allows this to be accomplished. there are four work list types:

'Complete' work list

All actions are placed on the work list.

'Only new' work list

Only actions that are not already on a work list (of the same template) are placed on the new work list.

'Continuation'

Only actions that are not on a work list of today are placed on the new work list. The numbering of the new work list starts with the last number of the previous work list of today + 1.

'Continuation - reset'

Same as previous but numbering starts from 1.

For continuation work lists (with continued numbering), any unfinished QC actions at the end of the last work list of the same day are deleted. Next, the QC schedule of the new work list is added as if the new entries were really added to the previous work list, i.e. there is no start QC, and the QC cycles continue at the same position where the previous work list ended.

Entries from - to

Minimum number of entries

If a minimum count is specified, no work list will be generated as long as the minimum amount of actions isn't reached.  The rationale behind this, is that some tests are that expensive that they shouldn't be started as long as the rack or carrousel isn't completely filled. 

Maximum number of entries

The number of entries on the work list should not exceed this maximum.  When you do not specify a maximum, GLIMS will first check if a run capacity is specified in the station.  If so, this run capacity will be applied as maximum.  To avoid accidental creation of huge work lists, causing all kinds of problems, there is an absolute maximum of 10.000 entries per work list. This maximum cannot be changed.

Include criterion

A MISPL function allows to specify further criteria for this work list. This filter overrides the include criterion specified in the work list template.
