Work list browser

Introduction

When accessing work lists via the main menu (Start -> Routine -> Work lists -> Work lists) or via the Result entry program, a browser is used. Before the work list browser is displayed a query options screen appears.

Query criteria

Template

Only select work lists with the specified template.

Name pattern

A pattern is a character expression that you want to match with the string. The pattern can contain wild cards: a period (.) in a particular position indicates that any single character is accepted in that position; an asterisk (*) indicates that any group of characters is acceptable.

Versions

For work lists referring to a template, the version is established according to the template. For work lists not referring to a template, the version is establish according to its name.

The 'Latest version' is determined using one of 2 possible criteria:

  1. When the work lists are selected via work list template(s), work list template group, station or work place, the most recent work list of a group of work lists referring to the same work list template is considered to be the 'latest version'.
  2. When the work lists are selected purely based on the name pattern (or no criterion at all), the most recent work list of a group of work lists having the same actual name is considered to be the 'latest version'.

Updatable

This field is used in combination with the shielding mechanisms in GLIMS. If set, shielded work lists will not be selected.

A work list is shielded when all procedures of the work list template are shielded for the current user. Shielding is always related to departments. The department of a procedure is defined by the field Procedure.Department. The department of a result is defined by the field Result.Department. GLIMS will check the "Result" shielding access level for the concerned department.

Note

A work list may not be shielded entirely: some procedures are shielded, others are not. In that case, the work list will be selected, but when opening that work list in grid layout, GLIMS will check the access rights for each individual action on the work list.

All departments

If this option is set, the work lists for all the departments will be shown. If the toggle box is unchecked, only the work lists for the user's department will be displayed.

Active only

Only show work lists currently active on a station. For more information, please see the chapter Active work lists.

Work list browser

Sort order

By default, work lists are sorted by name and then in descending order of creation time (i.e. the most recent one first).

Columns

The columns of the browser include the work list's name and its template name (when known and different than the work list name), the creation user, the creation time, and the number of entries.

Note

The work list browser will reuse the last settings on each run, also when it is used in selection mode (e.g. from the 'Result entry' program). This should allow to specify query parameters like All departments or Versions only once.

Tip

To speed up this browser, it is highly recommended that work lists are not kept on disk longer than necessary. The number of days after which a work list can be deleted can be specified in the work list template.
