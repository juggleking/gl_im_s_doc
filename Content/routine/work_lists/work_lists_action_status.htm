Basic routine -> Work lists -> Work lists and action states

A work list is a simple collection of different actions. One of the criteria to generate a work list is the status of the actions. The different states allow the customer to generate different work list:

  • Query all actions from Pending to Outputting: generates a list of actions which could theoretically be started, but which have not yet been executed.
  • Query all actions from Outputting to Outputting: generates a list of all the actions that still have to be started.
  • Query all action from Outputting to Partial: generates a list of actions that are started but not all results are available yet.
  • Query all actions from Realized to Realized: generates a list of actions that have been realized but not all results have been confirmed yet.

If a work list is set cumulative, GLIMS will not take earlier generations and will include actions which have already appeared on other work lists. If it is set non-cumulative the work list will only include actions which are not yet linked to a work list entry.
