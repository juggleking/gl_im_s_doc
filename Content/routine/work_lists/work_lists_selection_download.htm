Work list selection download

It is possible to download a work list selection to an instrument. This can be achieved by means of the Station fields described below. In case of an ALL-query from the analyzer, it is sufficient to enter a value for the fields described below and the work list selection mechanism will be applied. However, in case of a sample query (single sample or sample range), the Station field Sample query by work list has to be enabled as well.

Active work list name / Active work list creation time

Please check the chapter Active work lists.

Worklist name pattern

The pattern of a work list name can contain wildcard characters: a period (.) in a particular position indicates that any single character is acceptable in that position; an asterisk (*) indicates that any group of characters is acceptable, including a null group of characters.

Example

hitachi* will result in all work lists with names beginning with hitachi, such as hitachi747, hitachi911, ...

Warning

When a site function is specified in the general option Worklist name to define how automatically created work list names should look like, the Station field 'Worklist name pattern' must contain an actual pattern. When a fixed value, matching the worklist template name or part of it, is specified as 'Worklist name pattern', GLIMS will not find any work lists.

For instance:

  • the Station field 'Worklist name pattern' is set to 'MyWorkList'
  • the work list template name is MyWorkList

  • the GLIMS General setting 'Worklist name' generates a work list name that looks like 'MyWorklist <year-worklist_identifier>'

    string InternalId;

    InternalId := Identifier("Worklist", DateToString(Today(),"%Y-"), 5);

    RETURN ifknownstring( .Name ) + " "+ Internalid;

    Depending on the year and the identifier, GLIMS then generates the work list name 'MyWorkList 2021-0001'.

However, with these settings, a work list query for this station will not return any work lists because GLIMS will not find any work lists named 'MyWorkList'. If 'Worklist name pattern' were set to 'MyWorkList*', GLIMS would find the work list named 'MyWorkList 2021-0001'.

When using this Worklist name pattern, you MUST also specify a Worklist age (see below).

Worklist age

Maximal age in hours of a work list. Work lists that are older than the given number of hours since the time of the station inquiry are not selected.

Worklist versions

Possible values: Latest / All.

Allows you to indicate if only the latest version or all versions of a work list (with the same name) should be selected.

The work list download status field (hidden field) will be updated whenever the work list is selected by a station. The contextual function View download status  shows a list with stations or station download groups for which the work list has been download.

Upon the next station inquiry, the work list will be selected again, except if the option Check downloaded actions is enabled for the inquiring station.  You can clear the work list download status using the contextual function Clear download status. For more information, please check the chapter Analyzer download status.

Example

This functionality allows you to send the same test selection to two or more identical analyzers.

Exclusive worklist selection

This field allows you to specify that a worklist selection configuration (a worklist selection configuration implies filling in the Station fields Worklist name pattern, Worklist age, Worklist versions) is only applicable to the station being queried for work orders.

By default, this field is disabled. If enabled, GLIMS will ignore the worklist selection configuration when:

  • The querying station is a concentrator and the worklist selection configuration is on a client station of the concentrator.
  • The querying station is a distributor and the worklist selection configuration is on another distributor station in the same download group as the querying station.

Before Exclusive worklist selection was available and work orders needed to be transmitted for a concentrator with a worklist selection configuration on a concentrator client station, GLIMS always selected actions of the client station via the work lists of the selection. However, this is not always the desired behaviour. The worklist selection configuration on the client station could be intended for the download of work orders requested by a translator for the client station only (and not for the concentrator).

Note

If the configuration described in the example below is in place and you want to make sure that the corresponding action of a work list entry that is scheduled on a station that is not a client station of the concentrator is included in the work order message sent in reply to the query received by the concentrator, then the work list selection configuration should not be on the concentrator (this means that the fields Worklist name pattern and Worklist age should be set to ? and the options Exclusive work list selection, Worklist items and Sample query by worklist should be disabled) but on its client stations. Moreover, Exclusive worklist selection needs to be disabled on the clients stations as well.

Example

Configuration

A concentrator "conc1" has two client stations "stat1" and "stat2". Another station, "stat3", which is not a client station of "conc1", is in the same download group as "stat1" and "stat2".

Routine

An action is scheduled on "stat3" for a property, "testX", which is an entry of a work list that can be selected by "conc1".

Subsequent to an ALL query received by "conc1", the work list in question is selected.

The action of "testX" needs to be included in the work order reply message.

Worklist items

This field is interpreted in combination with the above-mentioned work list selection parameters.

If enabled, procedure outputs are selected only if they are defined as an item on the work list template of the work list being selected. If disabled, all outputs of a work list entry procedure are selected, no matter if work list template items are defined.

Note

  1. If a selected work list has no items, all result output will be selected, no matter the setting of Worklist items.
  2. The Station field Send complete selection has no effect on work list selection queries.

  3. The Station field Check downloaded actions has another effect when used with work list selection queries:

    • For regular queries, the Check downloaded actions setting has effect on the selection of result outputs.
    • For work list selection queries, the Check downloaded actions setting has effect on the work list selection: A work list that is queried by a station is marked by that station. If Check downloaded actions is set, only work list that are not marked are candidate for the work list selection.
  4. To activate the work list selection query functionality, you should fill in at least the fields Worklist name pattern and Worklist age. Only ALL-queries received from a translator can result in a work list selection query.
