Result: Repeat

This function allows to discontinue and repeat one or several selected results:

Options

Justification

A justification reason must be specified. Enter a free text or use 'Insert' to select a Result-based text module. The justification reason can be consulted in the 'Log' page of the result detail.

Restrict validation

Allows the user to specify that the new result may only be validated by the logged in user. This restricted validation can also be obtained by executing the 'Restrict validation' method on a result record. From then on, only the current user can validate the result. In the 'Order review' screen, such results will not be pre-selected and even when selecting them manually, they will not be validated.

Note

A user should have the required privileges for both Result.Validate() and Result.RestrictValidation() before this option can be used, either from the contextual menu or the discontinue user interface.

Note

When discontinuing a result that is calculated from other results, these other results are also discontinued when they are not requested explicitly (Result.Request.Requestable = ?) and when no other non-discontinued result needs the same result as input.

Required privileges

When repeating manually confirmed or validated results, GLIMS checks whether the user has result confirmation or validation privileges respectively. Users without the required privileges can only repeat automatically confirmed/validated results.

Note

Users must always have "Result modify" privileges to repeat a result.
