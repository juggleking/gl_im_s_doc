Result.SelectValue

Introduction

In GLIMS, it is possible to select a final value for results which have been measured several times. 

  1. This function allows to select one of the previous results and restore it as the final result. 

  2. You can also select several previous results, in which case the mean value of these results will be restored as final result.

Note that you can also use the result function Use value from previous run to quickly revert to the previous assessments of the selected results.

Usage

General features

When this function is applied to one or more results, the following takes place:

  • The predecessor chain of the result is searched to find at least 2 values which have not been manually modified (i.e. the candidate's value should not have been replaced via Result.Modify()).
  • If  these previous values are found, they are shown in a browser.
  • This browser allows to select one or more assessment values for each final result. Selecting numeric values leads to the automatic calculation of their mean value and coefficient of variation, and the new value is proposed as final value. When a string result is selected, all other selected values are deselected and the newly selected value is proposed as final result.
  • By clicking a value column header, it is possible to indicate that for each property, the value of a specific column should be used, without having to select this for each row separately.  Clicking it again will deselect the whole column.  
  • It is also possible to manually type in a final value.
  • When pressing "F6" in result value cell, the result details are shown.

Upon saving the screen, all thus selected values are stored as the 'selected value' for the result. If necessary, a new result record will be created to store this new value. All previous assessments from the corresponding result assessment chain are discontinued if necessary.  The discontinuation trigger is also executed on the selected result (if configured on the property).

When the selected result was output of a certain action, the new result which is created will also be output of an action (re-scheduled).

Customization

Mean value of previous results

The Select value function allows selecting several previous results in which case the mean value of these results will be stored as final result.  

GLIMS allows to deactivate this feature by disabling the field Allow selecting multiple results. in the general options (Specific site).  If this option is disabled, the calculation of the average of the selected previous results no longer occurs and the VC column is no longer shown in the browser.

Sorting of previous results

GLIMS allows to customize the sorting of the previous results.  By default, the most recent result is shown first (Reversed chronological).

This sorting order can be reversed by enabling the field Select Value Result Sort in the general options (Specific site).  If this option is set to Chronological, the oldest result will be shown first.

Notes

  • The result browsers can detect that the displayed result value is too wide to fit in its column. This may lead to misinterpretation. Therefore, this is made clear to the user by replacing the last part of the value with the '@'-sign.  Note that such a situation may sometimes quickly be resolved by double-clicking the frame title in order to maximize the browse window.

  • When a previous value is selected, and this result is repeated thereafter, the selected value will not appear in the list of previous values. Only actually measured values are displayed in the select value browser.

Auto-confirmation and auto-validation rules applied to selected result

Auto-confirmation

  • When using the Result.SelectValue() method to obtain a new value from previously measured values, the new value is automatically confirmed.  If configured on the property, the confirmation trigger will also be executed .

Auto-validation

  • Auto-validation is performed when each of the selected results could have been auto-validated according to the configuration of the assessment method that was used for that specific result.  If all selected results meet the required auto-validation criteria, then the new result will also be auto-validated.
  • Glims also checks if auto-validation is not prohibited for the new result: if manual validation is required (can be set in the Extra page of the order entry screen) or if Restrict validation is set while discontinue-repeating a result, auto-validation will not be performed.

Access to originating result

When selecting a previous value, a site attribute that contains a reference to the originating result is created. Via the MISPL function GetSiteAttribute with third parameter "_SelectedResult.<fieldname>", the referenced result can then be accessed to retrieve its fields e.g. external and internal comments, station severity etc.

Example

GetSiteAttribute("Result",.Id,"_SelectedResult.Externalcomment");
