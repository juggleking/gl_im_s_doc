Image results

Property configuration

Properties that have images as result values should have Data type set to Image.

Storing images as result values

Some analyzers can send images as result values.

Location of images

The images should be stored in a central directory accessible by GLIMS.

Result value

It is recommended to store only the file name of the image as result value, and not the full path to the file.

If all your images reside in a central directory, you can specify that directory in the ImageServerDir environment variable (MA Section in progress.ini).  From then on, you can specify your image files relative to that directory.

Example

Result image file: \\servername\images\cytogram.bmp

[MA Section]

ImageServerDir=\\servername\images\

Result value:  cytogram.bmp

Warning

We recommend using relative paths in Result.Value in combination with the ImageServerDir variable. This way, image files can be moved to another location, if required. Only the directory in the ImageServerDir variable would then need to be updated.

This is not possible when using absolute file paths in Result.Value! If used, absolute UNC paths must be used.

Relative paths cannot start with "/" or "\". Paths starting with "/" or "\" are considered to be absolute paths.

Manually entering an image result

Associating an image with a result can be done manually.  In most screens an image result will be represented as 'Image', but in the Result Modify screen, you can see the full file name of the image file.  Here you can type a new file name or select an image file by pressing Ctrl-F:

Image viewer

How to view images

Image results can be viewed using the contextual function View image on Result.

Other ways to view an image result are:

  • Pressing 'Zoom' (F6) on the result in the Result -> Modify screen.
  • Pressing the 'View image' button in the Result -> Modify screen.

Note

If the image cannot be opened by the built-in image viewer, the image can be opened by an external application. This depends on the application that is associated with the file extension (as defined in Windows).

About the image viewer

The image viewer has four options which are remembered in the user preferences:

The Stay on top button.  It  indicates that the image viewer should always bring itself to the foreground when another record is chosen in the calling program.
The Stay on screen button.  In its pressed state, the image viewer will synchronize with the program from which its is called. When navigating to another record, the viewer will refresh synchronously. In its released state, the viewer will automatically disappear when navigating to another record.

The Fit to frame button.   When pressed, the image will be reduced or enlarged so that it exactly fits into the frame's current size. If the file type doesn't allow to be resized, the button is disabled.

Will open the image file externally in whatever program is associated to the image file type on Windows level. This is a flexible way to have end-users use a viewer of their own choice for a particular image file type.

In the image viewer, you can

  • zoom: CTRL-scroll
  • drag and drop an image file
  • press Insert or CTRL-F to open the Windows explorer.

Tool to change path of image results

As mentioned above, it is recommended to store only the file name of the image as result value, and not the full path to the file.

If the result value contains the full path to an image anyway, and you want to change the location of the images, a tool is available to change the image path of all image results in GLIMS. It is available from the main menu via Start -> System management -> Tools -> Update image results.

Property

Only image results (of which the image path matches the specified image path) of the specified property will be modified. If no property is specified, the image results (of which the image path matches the specified image path) of all properties will be modified.

Old image path

Original directory where the images are stored.

New image path

New directory where the images are stored.

Verify image file existence

If enabled, GLIMS will check (for each image result) if the image file actually exists in the new directory.

Note

This information will be shown in the log file. If the image file cannot be found in the new directory, the image result will be updated anyway.

The log file can be used to check if all images are actually available in the new directory.

Test mode

If enabled, all information will be stored in the log file, but the actual image results will not be changed.

Conversion of image result values

This tool looks up all results for properties with data type Image where the raw value matches the specified Old image path and replaces the raw value with the specified New image path (when not in test mode).

All changes are saved in a log file with a unique file name in the session's temp folder (also in test mode).
