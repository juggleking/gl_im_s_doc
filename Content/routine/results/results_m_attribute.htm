Result.Attribute(AttributeName)

Introduction

The Result.Attribute MISPL function fetches an attribute of the current result in string format.

String Attribute (String AttributeName)

Supported values for AttributeName

Value

Returns the value string.  The Result.Attribute("Value") function allows a format specification by means of a ":<ViewFormat>" modifier. The modifier determines in which format the result value string should be returned. Below is a table of the supported values and their effect:

ViewFormat Raw Unit Decimals Compact Multiling
"BrowseFormat" Property.ReportRawUnit YES YES NO
"OnlineFormat" NO YES NO NO
"RawFormat" YES NO NO NO
"ReportFormat" Property.ReportRawUnit YES NO YES
"StandardFormat"  NO NO YES NO
"WorkFormat"  Property.ReportRawUnit NO YES NO

Legend

Raw unit

If set, the result is reported in the unit of assessment.

Decimals

Format according to Property.ReportedDecimals and Property.DecimalThreshold. Ignored for QC results.

Compact

Applies to culture and antibiogram results only.

Multiling

If the result unit is different from the property unit, the unit is added to the result string. For a format with the "Multiling" indication, the multi-lingual unit description will be used instead of the unit name.

Example

When entering a new request within a property's blocking period, the request is typically not effectuated, but the original result is reported instead.  In order to report the original result in the same way as it would be reported in a standard report column (i.e. taking ReportedDecimals, DecimalThreshold,... into account),  this can be accomplished as follows:  .Original.Attribute("Value:ReportFormat")

Note

Both .Attribute("Value") and .Attribute("Value:ReportFormat") MISPL functions return the plain text version attribute value (limited to 20 000 characters) for rich text results.

EncounterInfo

If the object is a person with a valid encounter at the result's object time, the external encounter info is returned (Encounter.ExternalInfo).

ExpandedValue

Behaves identically to the "Value" attribute, but additionally expands the result value if it would contain a symbolic text reference.  Example: where .Attribute("Value") might return "{<TB}", the .Attribute("ExpandedValue") will return the expansion "Tube broken".

ReferenceValue

Shows the reference values of a result. If no reference values are defined for the result an empty string is returned.

ReagentUsage

Returns a string containing information about the reagent and reagent lot (and its expiration date) used for the result. The information is returned in the following format:

<ReagentCode1>,<LotNumber1>,<ExpirationDate1>\<ReagentCode2>,<LotNumber2>,<ExpirationDate2>\...

This information could e.g. be added in the result's external comment via the property's availability, confirmation or validation trigger.  This allows including this information on reports.
