Result: Responsible

Introduction

For each individual result, GLIMS stores a responsible. This can be used for:

  • Printing the result responsibles on result reports.
  • Determining the report responsible.
  • Determining the billing item's executor during tariffication.

About Responsibilities and Availabilities

GLIMS can determine the responsible via the Responsibility and HC provider availability configuration.

Defining responsibilities

From the main menu, choose Start -> Correspondents -> HC providers -> Responsibilities.

Sequence

Allows defining the sequence in which the responsibilities are evaluated by GLIMS.

Property domain

Specified

The HC providers assigned to this responsibility are only responsible for results for properties that belong to this property domain.

Not specified

The HC providers assigned to this responsibility are responsible for the results for all properties, regardless of the property domain to which they belong.

Property classification

Specified

The HC providers assigned to this responsibility are only responsible for results for properties that belong to this property classification.

Not specified

The HC providers assigned to this responsibility are responsible for the results for all properties, regardless of the property classification to which they belong.

Department

Specified

The HC providers assigned to this responsibility are only responsible for results executed by the specified department.

Not specified

The HC providers assigned to this responsibility are responsible for results executed by any department.

Lab

Specified

The HC providers assigned to this responsibility are only responsible for results executed by the specified lab.

Note

GLIMS will first search the executing department (see note above) and then check the lab specified in the Department.Lab field.

Not specified

The HC providers assigned to this responsibility are responsible for results executed by any lab.

Assigning HC providers to a responsibility

To assign HC providers to a responsibility:

  1. Select a responsibility.
  2. Right-click and choose Show all -> Responsibility HC providers.
  3. Hit the Insert key.
  4. Choose a HC provider and click OK.

Defining availabilities

After having defined the responsibilities, you also need to define when the responsible HC provider is available. From the main menu, choose Start -> Correspondents -> HC providers -> Availabilities.

HC provider

Choose a HC provider.

Duration

Start and end time within the day. The day starts at 00:00 and ends at 23:59.

Tip

If a HC provider is available from e.g. Monday 20:00 till Tuesday 04:00, you will have to define two Availability records. One for Monday (20:00 - 23:59) and one for Tuesday (00:00 till 04:00).

Recurrence pattern

Days of the week at which the HC provider is available. Allows to exclude holidays.

Range of recurrence

Start date and end date. End date is not mandatory.

Determination of the result responsible using responsibilities and HC provider availabilities

The following logic is applied to look for a valid responsibility and HC provider availability:

  1. GLIMS searches for a valid Responsibility record.
    1. GLIMS evaluates the responsibilities in the order of their Sequence.
    2. If specified in the Responsibility, the property domain, property classification, department and lab must match with the result.
    3. As soon as a valid Responsibility is found, GLIMS checks the HC provider availability (see below)
  2. If a valid responsibility is found, GLIMS checks if the linked HC providers are available.
    1. GLIMS checks whether the linked HC providers have an availability record that matches the result's validation time. Only the HC providers which are available at the time of result validation will be retained.
    2. If multiple HC providers are available, please check the diagram above to see which one will be used to set the result responsible.
    3. If none of the linked HC providers are available, GLIMS will try with the next valid Responsibility record.

Tip

It is possible to guarantee that a result responsible is always found by defining a default responsibility. This is a responsibility with no criteria specified and with a responsible HC provider linked to it who is always available. The default responsibility should have the highest sequence number of all responsibilities.

Responsibility delegation

A Responsibility can be delegated, which can be useful e.g. if the person in charge of a responsibility is absent and cannot validate results. Check the page Result validation by responsibility delegation for more information.

Warning

Defining a Responsibility delegation may affect the determination of the Result.Responsible.
