Extra tools -> Cell counter: usage

Contents: Introduction - Cell counter screen

Introduction

Once you have completed your cell counter configuration, you can now start the cell counter program. The cell counter program can be accessed in several ways.

Directly from the main menu

Click Start -> Routine -> Action -> Cell counter.

From an action context

Action browser

The main application menu offers an action query/browser to provide quick access to the cell counter program. It is available via Start -> Routine -> Actions -> Browse actions.

  1. It offers the same query options as the standard action query (as used in the Confirm by action program). The Sort by option is not available, as this action browser can be sorted by clicking column headers (see below).

  2. The action browser can be sorted on any column (e.g. Station) by clicking a column header.

  3. When pressing Enter or double-clicking, the cell counter will immediately be started for the selected action.

  4. Pressing OK/Page Down or Page Up in the cell counter will navigate to the next or previous action respectively.

Work list grid

You can also start the cell counter program from a work list grid by selecting the contextual menu function Action -> Cell counter.

From a specimen context

  1. The cell counter can also be started from a specimen context, e.g. the Specimens of order browser. Use the Specimen contextual menu function Open cell counter.

  2. Pressing OK/Page Down or Page Up in the cell counter will navigate to the next or previous specimen respectively.

From a result context

  1. The cell counter can also be started from a result context, e.g. the "Results of order" browser. Use the new Result contextual menu function Open cell counter.

  2. Pressing OK/Page Down or Page Up in the cell counter will navigate to the next or previous result respectively.

Screen synchronization

Example:

  1. Select an Order from an Order browser.

  2. Right-click Request -> Specimens.

  3. On a Specimen, right-click Open cell counter.

  4. When navigating to another order, the specimen browser and cell counter will both be synchronized.

Cell counter screen

General information

When started from the main menu

When starting the cell counter from the main menu, the user should select a Station, a Specimen, and a Procedure. The station is obviously the manual count station. When entering a bar code, the program will try to find the corresponding specimen, using the count station's identification settings. When a specimen is found, the program will generate a list of procedures of this station which could be used to produce results for at least one of the existing results of this specimen. Note that this may imply selecting the station's manual count procedure for use on a result which was originally scheduled for another procedure of another station, but which can also be counted via this procedure.

If more than one procedure is found, the user will have to select the desired procedure manually. This can occur when this station is able to perform several count procedures and the count procedures are both requested for the current specimen. If only one such procedure is found, this procedure is selected automatically. In the other case, the user has to select the correct procedure.

The user is then able to start the counting process. After committing the screen, he can enter a new bar code without having to restart the program.

When started from the action context

When started from the action context (e.g. from a work list), the station, specimen and procedure will already be selected.

Remarks

  • No counting is allowed when one of the cells belonging to the formula has a status 'Confirmed' or higher.
  • During the counting process, the standard specimen contextual menu is available.

Left frame

In the left frame, the properties to be counted are listed (i.e. all property outputs for which a counter key label is defined in the assessment method). You can count the cells by using the keys defined in the assessment methods. The key for each property is displayed in the first column. The Customize button allows to enter new key codes in the key code column of the browser. These new key codes are user-specific. Upon committing the screen, these preferences will be stored. It is also possible to increment or decrement any counter by hitting the right arrow or left arrow when the browser with the count cells is positioned on the cell whose counter you want to change. This allows for corrections after or during the counting process. The Reset button allows to clear all entered counts. The user-defined result colors are also used in this cell count program.

When the count limit is reached, the word 'Ready' will be displayed at the bottom of the screen. If you want a sound to be played when you reach the count limit, simply copy a wave file called "CountFinished.wav" in the "snd" sub-directory of GLIMS. When a cell count cycle is specified, the user will also get an audible signal (snd/CountCycle.wav) each time this number of cells have been counted.

When cell count property has been defined, the cell counter will show an extra column in the counter browser, displaying the absolute values of the counted fractions (i.e. fraction (in %) * Procedure.CellCountProperty result value / 100). These values are only shown to check the results of the counting process. They are not stored in the database.

The last result values of the formula are shown in a separate column in the count browser. These 'last known results' may very well be obtained via another procedure (e.g. the automated cell counting via analyzer).

Upper right frame

In the upper right frame, the properties of the procedure which do not have to be counted are displayed, i.e. all property outputs for which no counter key label is defined in the assessment method.

For these results, the last known value of each result will be proposed as new value. When one of these last results had already received a value, committing a new count will always result in discontinuing the old results and creating a new action for the manual count procedure.

Result code look up is also possible. When using the keystroke CTRL-F for a non-countable result, a selection list pops up with all possible choices. The list contains all applicable result codes and, if the property is enumerated, the choices.

Lower right frame

In the lower right frame, you will see the information requested in your procedure data configuration. These values are accompanied by their object time if different from the object time of the action being processed. Any property input results of the action are also added to the already shown property data result.

Buttons

Reset

Allows to clear all entered counts.

Customize

Allows to enter new key codes in the key code column of the browser. These new key codes are user-specific. Upon committing the screen, these preferences will be stored.

Specimen

Opens a browser with all specimens for which you performed a counting during this session.

Bottom of the screen

At the bottom of the Cell counter screen a trail will be displayed with your latest actions.

Image results

Available results of data type 'Image' automatically pop-up in one single frame. The frame automatically appears as images are available, and disappears if there are none.

This is accomplished by means of an embedded Internet Explorer window. It can be resized as desired, in which case the images will automatically re-arrange. Images can be viewed full-size by clicking on the initial image. The latter may require the following Internet Explorer option to be set: Tools > Internet Options... Advanced > Security > Allow active content to run in files on My Computer. The HTML content is hard-coded and cannot be configured.

The images are collected and sorted as follows:

QUERY

  • Procedure data
  • Result outputs of the previous (analyzer) count action

SORTING:

  • ProcedureData.SeqNo
  • ProcedureOutput.SeqNo
