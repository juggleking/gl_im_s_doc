Automated creation of reports: report scheduling

Report scheduling

When an order is written to the database, the issuer's report records are created automatically as well. These report records are configured following the models of the issuer's associated default reports. If the issuer has no associated default reports, then those of the default organization, as indicated in the GLIMS configuration screen, will be used.

Each default report's eligibility will then be evaluated to see whether it should be used to generate a new report record. Examination always starts with default reports of type Exclusive. If any of those is actually used, then no other reports will be further generated. This feature avoids having to negate in the subsequent defaults any special eligibility condition of the exclusive default. Check here for more information on default reports.

The automatic report scheduling which takes place after an order has been created or modified via the order entry screen is no longer executed for orders which are more than 1 month old. This avoids unwanted reprinting of reports when changing something small to an old order for which the reports were already sent and deleted.

No report scheduling for auto-added / unsolicited results

GLIMS will NOT automatically schedule new reports when:

  1. A property is automatically added to an order via the MISPL function Order.AddRequest.
  2. A property is automatically added to an order via the MISPL function ResultOutput.CascadeRequest.
  3. A station uploads an unsolicited result.

Note

Should automatic report scheduling still be required, you can use the Order.ScheduleReports() function in a MISPL expression.

Sending new reports for old orders

The contextual menu function Order.ScheduleReports() allows to force the rescheduling of the order's default reports. This method allows the re-scheduling of the reports for one or more orders. Scheduling reports by using the Order.ScheduleReports() function in a command is possible. This allows scheduling reports for orders some time after the order was created.

This can be helpful when the default report setting has been changed so that a given issuer now receives a new type of report. By selecting this issuer's order (via the standard Order Query browser), selecting the appropriate order from the list, and calling the Order.ScheduleReports() method, the new report type is added for the selected orders.

Please check the chapter Adding reports manually or by MISPL for more information on alternative methods.
