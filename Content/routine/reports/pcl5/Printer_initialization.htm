Print job initialization

This document explains how you can fully initialize a print job with PCL5 coding ( = Esc)

%-12345XE&l1X&l0S&l1T&l26A&l0O&a0P&l1L&a0L&l0E&l5D&k2G
(0N)0N(s0p16h12v0s0b16901T)s1p10h16v1s3b16901T&s1C

When adding this initialization, the logical page will start at the top of the physical page.  As a result, the page will have more lines to print on.

In addition, the PCL5 coding above also contain the initialization of the character code set to iso-latin which corresponds to the GLIMS-Progress standard ( see <esc>(0N ).  This allows to print special characters as | é è ç %.

Therefore, in order to initialize a print job correctly, you always need to start with "%-12345XE".  

Note

This coding should never be used in a second page header !!  It should only be included in the beginning of the file.

Tip

In GLIMS, you could create a text module named 'HPINI' on table gp_Site.  This allows referring the text module from any other text module, from any other context using {<HPINI}.  Examples could be the report first page header, work list first page header, etc.
