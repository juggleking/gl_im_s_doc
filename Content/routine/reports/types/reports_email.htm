Basic routine -> Reports -> Email reports

Contents: Introduction - Configuration

Introduction

GLIMS allows to send text or HTML reports using external email.  

Configuration

Setup

Steps to generate email reports:

  • Specify the mail server and mail port (System management -> Customize -> MATE).
  • Specify the email address of the report target correspondent.
  • Specify the email address of the default organization (this will be the E-mail sender) (System Management -> Customize -> GLIMS).
  • Set the report medium in the Default Report records (or of already existing Report records) to "E-mail".
  • Generate the report(s).

During the generation of a text report, the fact that Report.Medium = E-Mail and that the target correspondent has an email address will send the report to this address via mail in stead of to the printer.  

Note

If an explicit printer is selected in the report generation setup screen, the reports will always be printed to this printer!

Customization

  1. The subject of an e-mailreport can be configured via the ReportTemplate.EmailSubject field, referring to a Report-based dynamic text.  This text should not contain new lines, as this is not supported in all email clients (e.g. MS Outlook).
  2. The body of an e-mail report can be configured via the ReportTemplate.EmailBody field, referring to a Report-based dynamic text. When such a body is specified, then the report will always be sent as attachment (even when it is a text or HTML report).
