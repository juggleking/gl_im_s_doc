Hardware requirements

  • 1 PC NT4.0 workstation (or Windows 2000 Professional)
  • 1 internal/external modem
  • 1 phone line (preferably a direct external line)
  • 1 network card
  • Hard disk : 5 Gb
  • Memory: 128 Mb

Concept of the fax module

First of all, a fax program, Zetafax, has to be installed on a PC, which will fax all incoming reports from GLIMS. In order to automatically fax reports created in GLIMS, these reports have to be "printed" to a dedicated directory on the fax server which will continuously be checked by Zetafax for new files. If you want the faxes to be sent automatically, you will have to purchase the Zetafax add-on ZF222 Application Programmers interface (API).

The Zetafax client program allows addressing instructions to be embedded into documents using FaxMerge commands.

External applications' documents may contain commands to indicate where a fax should be sent, together with a wide range of settings such as the time of sending, priority, resolution, coversheet and letterhead to use. If these commands are used in a document, they must be typed using a special syntax. The most important command is obviously the fax number to which the document should be sent. 

For more detailed information on the available commands, please read the Zetafax user guide or visit the Zetafax website.

GLIMS configuration for the fax module

As explained in the previous chapter, documents to be faxed can contain embedded commands. In GLIMS, these commands will have to be specified in the report template.

So, a separate report template has to be created in GLIMS for the reports to be faxed. The Report medium field of the default report can be set to 'Fax'.  This merely marks the report to be a 'fax' report, but has no further influence on the reporting process. 

Warning

Make sure that the File extension field in the report template is set to ".prn", else Zetafax will not process the fax reports.

In the Header field of the Texts tab folder, you can specify the fax number to which the reports should be faxed, respecting the Zetafax syntax.

Example

Example of report template header for fax reports:

%%[MESSAGE]
COVERSHEET:
FORMAT: Epson
To: {=.Name}
FAX: {=Entry(1,.Fax,"-")+Entry(2,.Fax,"-")}
%%[TEXT]
%%[F O N T:Arial,8]

Notes for this example

  • COVERSHEET: this is a mandatory field in Zetafax.  In this example an empty cover sheet will be printed.
  • FAX: the example MISPL removes the "-" from the target fax number if necessary .
  • FORMAT: if you would like to use another font, you will have to specify the format
  • F O N T:  Note that you need to specify a space between the characters

Note

If this header is specified in the report template header, Zetafax will fax each individual report.  It might be a good idea to specify this header in the field Header page of the report generator.  This allows to fax a set of reports instead of each single report.

When generating reports with the 'fax' template, you will have to "print" the files to a special directory (field: Directory path), which will continuously be checked by the Zetafax program.  On detection of a new file, the report will then be faxed to the specified fax number.
