Introduction

From each order, you can call a quick report to the screen by selecting the contextual menu function Quick report.  In order to be able to do so, you first have to specify a default report with Usage set to 'Screen'. 

When a screen report is asked, the Default Report settings of the default organization are searched for the first eligible default report where Usage = 'Screen'. When found, its settings are used for the generation of the screen report. This allows for an extensive control over the different settings of such a report, and even allows for showing another type of report depending on the order or other criteria.

Quick report for a single specimen

It is possible to have a quick report of one single specimen.  This can be achieved using the existing function Quick report on Specimen, but by configuring a default report with usage "Screen" and scope "Specimen" (instead of "Order").

The screen report program will first search for the appropriate screen report with scope "Specimen". If such a default report has been configured, it will show the screen report containing only the focused specimen, not the complete order.  If it cannot find such a screen report with scope "Specimen", then it will show the normal screen report (with scope "Order").

The same behavior also applies to:

  • Result -> Quick report
  • Microbiology action -> Quick report
  • Pathology exam -> Quick report

The quick report screen

 The Stay on top button.  It indicates that the screen should always bring itself to the foreground when another record is chosen in the calling program.
The Stay on screen button.  In its pressed state, the screen will synchronize with the program from which its is called. When navigating to another record, the screen will refresh synchronously. In its released state, the screen will automatically disappear when navigating to another record.

Validate

If you have validation privileges and the order contains results to be validated, then a Validate button will be available.  When the screen report is called from the 'Order Review' program, the button takes the same role (and label) as the Validation/Confirmation button of the review program.

Skip

When called from the 'Order Review' program, the Skip button allows to skip the current order (just as in the review screen itself).

Template

The Template button allows you to show the current quick report using another template. A popup list of available templates will be offered.

Print

When hitting the Print button (or CTRL-P), a contextual menu allows to select one of 3 choices:

Print the report as currently in display

Generate an already scheduled report

Allows the user to choose an already scheduled report and to generate this report as originally intended (i.e. a text report will be printed, a HL-7 report will be sent to file or over the network, ...). This is functionally the same as going to the 'Reports of Order' browser and executing the 'Generate' method on one of the reports.

Generate a new (unscheduled) report

Allows the user to create a NEW (unscheduled) text report and to print it immediately. The Order.AddReport() function will be used for this purpose.  The thus created report record is kept in the database for logging purposes. 

Tips and tricks

  • From the quick report screen, you can access the order contextual menu.  Note that the focus must be set to the quick report frame and not the body content.  Check the chapter 'Using menu accelerators' on how you could use this functionality. 
  • Properties with Property.Visible = "Internal" are shown on the screen report when the user type is 'User', 'System manager' or 'Developer'. However, when using the print button to print the current report, the internal properties are not printed, even when using the option 'Print the report as currently in display'.
  • The quick report does not show orders of which the object time lies more than 1 hour in the future. This will avoid showing only empty result columns.
  • Via the environment variable BrowserSupportedExtensions, the user can decide which file type should be previewed using a browser like Internet Explorer instead of the normal built-in text editor.  This variable can contain a list of file extensions for which the browser has the necessary viewing capabilities.  In order to use the browser for previewing reports for instance, the variable could contain 'doc,prn' to specify MS-Word (deprecated) and regular text report output file extensions.
