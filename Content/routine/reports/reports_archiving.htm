Basic routine -> Reports -> How to configure report archiving ?

It is possible to add a special footer/header message to a report page, if the content of a page is final. This page can then be archived.  There are two ways do so:

Using the ReportArchiveMessage functionality

In order to use this ReportArchiveMessage functionality, a number of criteria must be met:

The default report must be of type encounter, object, period or year AND the option Previous results should be set to None. If the report belongs to one of these types, then the footer/header message content will be displayed if:

  • The report has been closed (e.g. year report older than 1 year, encounter has been closed,...).
  • The maximum number of columns that can be displayed on 1 page has been reached.
  • The report is complete.

Configuration

  • In the header or footer (report template), specify the string &ReportArchiveMessage.
  • Create a text module named RepArchiveReport (start table Report). This text will be displayed.
  • If you do not create the text module, a default text ("Report to archive") will be displayed.

2. Using the {ReportToArchive} variable

A report variable {ReportToArchive} is available during expansion of a report header, trailer or footer.

It is a logical value which contains YES when the report obeys the 'archive report' rules (e.g. a section of an encounter report where all columns are used and are complete).

Example

Example of a report header:

Start of report header...
{= IF {ReportToArchive} THEN "Archive this report" ELSE "" }
....
Rest of report header...

The advantage of this second option is that you can include a different text per individual report.
