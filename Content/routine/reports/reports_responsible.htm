Reports: responsible

Introduction

For each individual report, GLIMS stores a responsible. The report responsible is saved in the Report field Responsible HC provider.

The information is:

  • Exported in the XML data for GLIMS Report Builder.
  • Included in online reports. Check the applicable driver documentation to see if the driver (e.g. j_dmp_ccr) can process this information.

How is the responsible of a report determined

The diagram below shows how GLIMS determines the responsible of a report.

  • If the report has been explicitly approved by a HC provider, the latter will be the report responsible.
  • Else the responsible of the last validated result within the report will be the report responsible.

When is the report responsible determined

The report responsible is set during report approval.

Note

For customers that do not use the report approval concept: reports that do not require report approval are automatically / implicitly approved during report generation. In this case, the report responsible is set during report generation.
