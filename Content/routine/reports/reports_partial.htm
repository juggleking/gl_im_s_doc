What about partial reports ?

In the Default report editor, it is possible to specify a trigger status. If a result reaches this status, the Needs checking field in the Report is automatically enabled and the report becomes eligible for further inspection.

There are two ways to select (or not select) partial or incomplete reports:

The field Partial report hold of the Default report:
The Partial report hold is the number of days to hold (prevent) output of partial reports. It is a fractional: 1 hour is approx. 0.04 days.
The field Minimal interval in the Report query (report generation):
Indicates the reprint delay (in days) for incomplete reports.

Example

If the Partial report hold is set to 0 and the Minimal interval is set to 3, then partial reports will be taken into account during the first selection but the next selection of this report will only be after 3 days. This avoids unnecessary generation of reports.

Note

During the report query, you will off course have to allow incomplete reports (Report status).
