Introduction

GLIMS offers several ways to add a report for an existing order in addition to the reports that were originally scheduled during order entry. For all methods, you need to specify a Code in the Default report.

Adding a report

Using the Order.AddReport menu function

When using the contextual function Add report on Order, a list of all the organization's default reports having a known code are retrieved. You will then be offered a choice between these codes.

Parameters

Report code

After selecting a code, all eligible default reports (either issuer-specific or organization-default, see standard report scheduling) having this code are scheduled, regardless of their Usage setting (except when set to Screen). This allows adding a specific type of report (e.g. a fax report) simply by choosing the appropriate code from the list.

Note

When the 'Forced' option is set, and no eligible report is found, the program still creates only the first default report (else the risk exists that too many reports are created). Also note that the above is also true when adding a specific report via MISPL.

Target

This function also allows to specify an explicit Target for the new report. During the search for the most suitable default report, the program reacts as if the specified target is the issuer of the order (much the same as for Order.ReportCopyReceiver).

Forced

If the Forced option is set, the DefaultReport.Eligibility settings are ignored (if necessary) when no report could be created due to these eligibility checks. This option is by default enabled.

Generate now

The Generate now option allows to immediately generate the desired report after having created it. If enabled, the report generation program will immediately be called for all the created (or reused) reports.

Parameter set

The field Parameter set can be filled with a reference to an existing parameter set for the function Report.Generate.

  • If this reference is filled, the report generation program will be called with the parameter settings as defined in this parameter set, without further user interaction.
  • If it is not filled, the report generation program will first show its own parameter setup screen in order to allow specifying the necessary parameters (like printer,...).

Medium info

The field Medium info can be used to specify e.g. an alternative fax number which will be used during report generation. It is stored in the report.

Pre-configuration

The Order.AddReport function can be implemented as a pre-configured menu/ribbon item.

Notes

Note: The <Code> parameter is mandatory. However, there is a workaround by first selecting an existing code during the parameter set setup, and afterwards deleting or emptying the <Code> parameter by means of ParameterSet.Parameters().

Using the Order.CreateReport MISPL function

The same rules apply when adding a report to an order via the Order.CreateReport MISPL function. This function takes a default report code as single argument and simply schedules all eligible default reports having this value in their Code field.

Adding a report for a specific report medium

Using the Order.AddMediumReport menu function

This function is similar to the Order.AddReport function (see above), but offers some additional functionality.

Parameters

Report medium

Allows to filter on report medium type. Possible values: Fax, Phone, Modem, E-mail, Courier, Communication, File. The value of this field determines which default report codes will be selectable in the Report code field (see below).

Report code

Lists all default report codes with the same report medium type as the one selected in the Report medium field (see above). After selecting a code, all eligible default reports having this code and the specified report medium are scheduled.

Target

Lists the most obvious targets of which the user can select one:

  • Patient related to the order
  • Issuer related to the order
  • Agent related to the order
  • "Explicit": allows to explicitly specify a target.
Notes

Note: when multiple orders have been selected, only the targets that are shared by all orders will be displayed.

Medium info / Fax number / E-mail address

This field can have different labels depending on the report medium that has been chosen. It contains additional information (e.g. fax number, phone number, ...) that is linked to the report medium and that is displayed (if available) when the target is selected. The displayed information can only be overruled for the report medium Fax. The information will be used during report generation and is stored in the report.

Tip

The field Fax number validity in the general options (Start -> System management -> Customize -> GLIMS general) allows to define a MISPL expression that checks the validity of fax numbers in GLIMS.

Forced

If the Forced option is set, the DefaultReport.Eligibility settings are ignored (if necessary) when no report could be created due to these eligibility checks.

Note: when the Forced option is set, and no eligible report is found, the program still creates only the first default report (else the risk exists that too many reports are created). Also note that the above is also true when adding a specific report via MISPL.

Generate now

The Generate now option allows to immediately generate the desired report after having created it. If enabled, the report generation program will immediately be called. A Generation parameter set can be specified when this option is set (see below).

Generation parameter set

Can only be specified when the Generate now option is enabled. The field Generation parameter set can be filled with a reference to an existing parameter set for the function Report.Generate.

  • If this reference is filled, the report generation program will be called with the parameter settings as defined in this parameter set, without further user interaction.
  • If it is not filled, the report generation program will first show its own parameter setup screen in order to allow specifying the necessary parameters (like printer,...)

Generate only once

Allows to indicate that the requested report has to be generated only once. This means any possible subsequent triggering of the report is blocked and does not retrigger a generation of the report. In the report editor (from an Order, select Reports > Reports), it will be indicated whether or not the report was requested as 'Generate only once'. If the 'Generate only once' option is enabled and the report has been generated, the 'Needs checking' option of the report will be disabled and will remain disabled so that the report will not be generated again when the order changes.

Pre-configuration

The Order.AddMediumReport function can be implemented as a pre-configured menu/ribbon item.

Using the Order.CreateMediumReport MISPL function

When adding a report via the Order.CreateMediumReport MISPL function, the following parameters can be supplied:

Code

Allows to filter on the Code field of the default reports that will be taken into consideration when selecting default reports.

Medium

Allows to filter on the Report medium field of the default reports that will be taken into consideration when selecting default reports. Possible values are: Courier, Fax, Phone, Modem, E-mail, Communication, File.

Target

Internal id of the report's target.

Forced

If the Forced option is set, the DefaultReport.Eligibility settings are ignored (if necessary) when no report could be created due to these eligibility checks.

GenerationParameterSet

This field can be filled with a reference to an existing parameter set for the function Report.Generate.

  • If this reference is filled, the report generation program will be called with the parameter settings as defined in this parameter set, without further user interaction.
  • If it is not filled, the report is scheduled, but not generated.

MediumInfo

Can be used to specify e.g. an alternative fax number which will be used during report generation. It is stored in the report.

In the report page of the order entry screen

You can also manually add reports for an order in the 'Report' page of the order entry screen.
