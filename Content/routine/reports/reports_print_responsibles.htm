How to print the result responsibles on a report ?

Determination of the result responsible

For each individual result, GLIMS stores a responsible. For more information, please check the chapter Results: responsible.

How to display the result responsibles on the report

There are several ways to display the result responsibles on the report.

Using a responsible column

Specifying the character B in the field Result section layout of a report template will generate a column with the initials of the result responsible.

In case of cumulative reports (several result value columns), this responsible column will only be displayed for the column with the most recent results.

Using a responsible row

The Report template field ValidatorRowLabel (Rows > Text reports > Row labels > Validator) allows specifying if a responsible row has to be generated at the end of the report.

The text specified in this field will be used as row label. As soon as a value is specified, each column of this row will be filled with the initials of the responsible of the results in this column.

Using the MISPL function Order.Attribute

The Order.Attribute MISPL function offers different possibilities to display the responsibles on the report.  This could for instance be incorporated in the footer text of a report.

"ValidatorInitialsList"

Returns a comma-separated list of the initials of the responsible of the results of this order. Responsibles of both manually and automatically validated results are included.

"ValidatorSignatureList"

Returns a comma-separated list of the signatures of the responsible of the results of this order. Responsibles of both manually and automatically validated results are included.

"ValidatorInitialsSignatureList"

Returns a comma-separated list of <initials><signature> of the responsible of the results of this order. Responsibles of both manually and automatically validated results are included.

"ManualValidatorInitialsList"

Returns a comma-separated list of the initials of the responsible of the results of this order. Only the responsibles of manually validated results are included.

"ManualValidatorSignatureList"

Returns a comma-separated list of the signatures of the responsible of the results of this order. Only the responsibles of manually validated results are included.

"ManualValidatorInitialsSignatureList"

Returns a comma-separated list of <initials><signature> of the responsible of the results of this order. Only the responsibles of manually validated results are included.

Examples

Example: Trailer text module based on the Report table containing:

{=IfKnownString ("Responsible: " + .Order.Attribute("ValidatorSignatureList"))}

Using variables in the report headers, footers or other dynamic text

Two variables are available during the expansion of report headers, footers or other dynamic text: 

  • ValidatorSignatureList
  • ValidatorInitialsList

They are comparable to the Order.Attribute("ValidatorInitialsList") and Order.Attribute("ValidatorSignatureList) MISPL functions as described above but only show the responsibles of the results shown on this report and the first responsible in the list is guaranteed to be the most recent responsible of the results.

Examples

Example: Example of use in a report header:

Responsible: {ValidatorSignatureList}

or, more complex:

Signature: {< SIG-{= ENTRY(1, "{ValidatorInitialsList}", ",")} }

where SIG-XX hold the signature of responsible XX, and where XX is the most recent responsible.
