Report approval

Contents: Introduction - Report approval - Report responsible

Introduction

In some countries, regulation stipulates that result reports should be approved before they are sent to the physician or an external application. Additionally, the approved report should mention a responsible.

Report approval

Configuration

In the Default report configuration, enable the option Approval required to indicate that reports scheduled via this default report must be explicitly approved. The value ? means No.

Warning

Reports that require approval cannot be generated if they have not been explicitly approved yet.

Approval

Approval is required

When the default report configuration indicates that approval is required, the reports needs explicit approval.

Manual approval

Report query

The report query allows to select all reports that require explicit approval. Set Approval required to Yes and Approved to No.

The report browser shows all reports to be approved. The Approval status column will show To do.

Approve function

Once you have selected the reports that require explicit approval, use the contextual menu / ribbon item Approve. Optionally, an Approval comment can be specified.

Report's result preview

Once you have selected the reports that require explicit approval, you can use the contextual menu or ribbon item Result preview, which offers an alternative way of approving reports.

The result preview screen shows all results of the report. Click the Approve button, to approve the report.

Automated approval

A command / task can be defined using the report query and report function Approve.

Approval is not required

When the default report configuration does not indicate that approval is required, the reports does not need explicit approval. However, reports that do not require explicit approval can still be approved.

Explicitly

The same approval methods can be used as for reports that require explicit approval (see above).

Implicitly

When the report has not been approved explicitly, GLIMS will implicitly approve the report during report generation. This happens automatically.

Approval fields

All approval data can be consulted in the Status tab page of the Report editor.

User

User who approved the report.

Explicit approval

Interactive user who manually approved the report or batch user in case of automation via a command.

Implicit approval

In case of implicit approval during report generation, this will be:

  1. User who last validated a result of the report.
  2. User who last confirmed a result of the report.
  3. User who last entered a result of the report.

Time

Time at which the report was approved.

Method

Explicit

Report has been explicitly approved, either manually or via a command.

Implicit

Reports can be implicitly approved in the following cases:

  • When a report (that does not require approval) has not been approved explicitly, GLIMS will implicitly approve it during report generation.
  • When adding reports manually via the Add report or Add medium report function, GLIMS will implicitly approve the report if the Default report option Approval required is enabled.
  • When adding reports via the Order-based MISPL function CreateReport or CreateMediumReport, GLIMS will implicitly approve the report if the Default report option Approval required is enabled.

Comment

Explicit approval

Comment specified during explicit report approval.

Implicit approval

Fixed text indicating when the report was approved (e.g. during report generation).

Identification method

Will currently always be set to Login. Will be extended in future versions for eID authentication.

Status

Not applicable

This report does not require explicit approval; i.e. the Default report option Approval required was not enabled when scheduling this report.

To do

This report requires explicit approval, but has not been approved yet. The Default report option Approval required was enabled when scheduling this report.

Done

This report requires explicit approval and has been approved. The Default report option Approval required was enabled when scheduling this report.

Undo approval

GLIMS allows to undo the approval of a report. All approval fields (see above) will be cleared. If the report has status Done, it will be reset to status To do.

Manually

To manually undo the approval of a report, select it and use the contextual menu / ribbon item Undo approval.

Automatically

In case of important changes to the report, the approval will automatically be undone. Examples: a new result was added, an existing result was updated, the external comment of a result was changed, etc.

Logging

All approval actions are logged in a Log type with name Report processing.

Report responsible

The chapter above describes how you can approve a report and how the approval user is set. However, the approval user is not necessarily the final responsible for the report.

For more information, please check the chapter Reports: Responsible.
