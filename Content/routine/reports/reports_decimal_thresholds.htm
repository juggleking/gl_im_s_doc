Decimal thresholds

The accurate rounding of results depends on Reported decimals and Decimal threshold. The Reported decimals combine two possible methods: either fixed number of decimals, or one extra decimal per decade the result is lower than the Decimal threshold.
Reported Decimals Decimal Threshold Reported
?   as entered
n ? Fixed n decimals
n t n decimals, augmented or lowered by the number of decades the result deviates from t (see below)

The following examples will best illustrate the above table.

Example 1

Reported Decimals = 0, Decimal Threshold = 55:

Result Decimals Example
…    
0.55 <= value < 5.5 2 1.13, 5.00
5.5 <= value < 55 1 7.6, 43.2
55 <= value < 550 0 63, 543
550 <= value < 5500 -1 (rounding to decade) 560, 4630
5500 <= value < 55000 -2 (rounding to hundred) 5800
…    

If the last digit you drop is 5, the program will always round up or down to the nearest even digit (because of statistical accuracy purposes).

Example

23.35 -> 23.4, 23.45 -> 23.4, 23.451 -> 23.5, 23.3499999 -> 23.3, 23.547 -> 23.5

Example 2

ReportedDecimals = 2, DecimalThreshold = 8:

Values between 0,08 and 0,8 will have 4 decimals (e.g. 0,712345 -> 0,7123)
Values between 0.8 and 8 will have 3 decimals (e.g. 7.12345 -> 7,123)
Values between 8 and 80 will have 2 decimals (e.g. 71.23456 -> 71,23)
Values between 80 and 800 will have 1 decimals (e.g. 712.3456 -> 712,3)
Values between 800 and 8000 will have 0 decimals (e.g. 7123,456 -> 7123)
Values between 8000 and 80000. A zero will be added (e.g. 71234,56 -> 71230)

Unit conversion

If a result is converted to another unit, i.e. raw unit or alternate unit, the program checks how many decades the converted result deviates from the original result. Reported decimals and Decimal threshold are adapted accordingly.

Example

Property Result Decimals Threshold Conversion factor Reported
Cholesterol 4.2 mmol/l 1 ? 0.0259 160 mg/100ml
HDL 1.43 2 ? 0.0259 55 mg/100ml

Note

The Unit used to report results is determined as following:

  • The Report raw unit field has priority. If the check box is marked, the result will be reported in the unit of result entry or as sent by the analyzer.
  • If check box is off, the Alternate units prevails (which on its turn is copied from DefaultReport.UseAlternateUnits). This field reports results in their alternate unit. Make sure AlternateUnit is filled in and the base unit is convertible to this alternate unit (from 4.8 on, validation on Property).
  • In all other cases results are reported in the unit of the property.

Non-numerical results can also have a unit.
