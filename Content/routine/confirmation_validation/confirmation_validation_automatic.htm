Automatic confirmation and validation

Severities for automatic confirmation and validation

The automatic confirmation and validation of results in GLIMS is based on severities.

In the Assessment method, you can specify an Auto-confirm severity and Auto-validate severity.  Results with a severity within these limits are auto-confirmed / auto-validated.

Confirmation / Validation user

The Confirmation user / Validation user of an automatically confirmed/validated result is determined as follows:

  1. The Responsible defined in the applicable Procedure.

  2. If not defined, the Batch user of the Service that uploaded the result.

Modifying automatically confirmed/validated results

By default, when a result is confirmed automatically, a user without confirmation privileges can modify the result. The same is true for automatically validated results. 

However, if the option SpecificSite.StrictStatusCheck is enabled, automatically confirmed or validated results can no longer be modified by users without the necessary confirmation or validation privileges.
