Order query options

The Order review (gp_Site.OrderReview()) function can be activated from the main menu via Start -> Routine -> Order review and first produces an order query:

Objective

Three options are available:

Confirm

Orders with 'Available' results will be selected and the eligible results will be pre-selected for confirmation.

Validate

Orders with 'Confirmed' results will be selected and the eligible results will be pre-selected for validation.

Confirm + validate

Orders with 'Available' and/or 'Confirmed' results will be selected and the eligible results will be pre-selected for confirmation and validation.

Created / Received / Sampled from - until

You can specify fixed dates or use relative dates. It is possible to specify e.g. "Created since: Today" instead of having to enter an absolute date value.

Internal id from - to

Order internal id range

Minimal urgency

Only orders with urgency at least as specified.

Issuer

Orders only issued by the specified correspondent.  If a specific issuer is specified, the 'Correspondent group' and 'Tour prefix' options are ignored. 

Correspondent group

Orders only issued by members of this correspondent group.

Tour prefix

When a tour prefix is specified, only orders for issuers belonging to at least one of the matching tours will be selected. It is the Tour.Mnemonic field that is compared against the entered prefix.

Agent

Only orders having the specified correspondent as agent.

Department

Allows for searching in to the Order.Department field.

The order's department is set as follows:

Manual order entry

Department of the logged in user that created the order.

Electronic order entry

Department of the batch user linked to the service that processes the order messages.  This department can be overruled if a department is specified in the order message.

Executing department (group)

 Allows to search for orders that have at least one action that is executed in the specified executing department (group).  This is determined by the procedure's department.

Escalated

For more information, see Validation in phases: result escalation.

Classification

Only orders with results for properties from this property classification.

Domains

Please see the chapter Validation by property domain.

Valab (expert system) status

This field will only be available if the specific site field 'Expert system' is not set to '?'.  Possible values:

Evaluated

Only select orders which have been evaluated by the expert system, regardless of the appraisal.

Not evaluated

Only select orders which have not yet been evaluated by the expert system.

Approved

Only select orders which have been evaluated and approved by the expert system.

Rejected

Only select orders which have been evaluated and rejected by the expert system.

To be evaluated

Select orders that are not evaluated by Valab yet or orders that are rejected or approved by valab but should be evaluated again.
 

Filter

Allows for the specification of a MISPL expression to filter the orders.

Sort function

Orders will be sorted according to the value of this MISPL expression.  By default the orders are sorted by creation time. 

Only complete

Only 'complete' orders, i.e. where all results from the current classification are confirmed.

Note

During the order query, a progress indication appears. It shows the amount of candidate orders for confirmation or validation, versus the amount of actually selected ones.  This should give some indication about the progress, and signal the user about configuration problems if the candidate order count is too large and grows day by day. Note that this progress indicator cannot be shown when making use of the remote query mechanism.

Order review screen

After having selected the order query options, the order review screen will be displayed:

General features

This is a screen with dual navigation possibilities: orders at the master level (navigation via the Page Up / Page Down shortcut keys), and results at the detail level (navigation via the CTRL+Page Up / CTRL + Page Down shortcut keys). Initially, the property column is focused. Entering the value cell is assumed to be an update signal. When the validation browser is in update mode, a lookup (CTRL-F) from the result value cell activates a result code & enumeration choice selection list. Click here for more information on result updates and comments.

Hit the confirmation/validation button to confirm and/or validate the selected results.  The label of this button varies between Confirm and Validate depending of the 'Objective' field of the order review query.  If a user is not registered as an employee (no employee assignments) or does not have confirmation/validation privileges, the confirmation/validation buttons will be disabled.  The tool tip of these buttons will explain why they are disabled.

If at least one of the results has a severity higher than or equal to the warning severity level (see result query options below), an image will be displayed and a sound will be played. This also applies to procedure data records in the confirmation program, provided their history index is 0 (= of the same object time). The warning image that is searched for should be named "warning.bmp" and located somewhere in the PROPATH. Its expected dimensions are 32 x 32. If you have a sound card, and "warning.wav" is found, the sound is played simultaneously. Simply remove the wave file if you want to get rid of the noise.

The validation program keeps track of which results are already shown to the user. Only if all the results to be validated (i.e. the ones that are pre-selected) have been viewed at, the corresponding button is enabled. So, if not all results fit into the browser, you will have to scroll down before you are able to confirm/validate. This should avoid inattentive processing of hidden results.

Other columns

For more information on specials marks, please see the chapter Result browsers.

Previous run / material

The Previous run column offers a cycle button in its header which allows to show the material instead of the previous run.

Previous run

Shows the most recent predecessor value of the result (if any). A predecessor value is the previous result of the same test that has been discontinued and repeated.  So the status of a predecessor value is always discontinued.  As a consequence, this value is always shown even if the result status range in the options does not include discontinued results.  If you want to see all discontinued results in the browser, you must include the Discontinued status in the result status range of the options screen.

All predecessor values are also shown in the comment box at the bottom left-hand side of the screen. They are displayed as a comma separated list, in descending order of assessment (i.e. the most recent first), and without their unit.

Material

Shows the material and, if available, the primary material variable.

You can hide this Previous run / material column by disabling the option Show previous run/material in the options screen.

Previous values

In the Options screen you can indicate if (and how many) previous results are to be shown.  Contrary to Predecessor values, previous results are results for the same test, the same patient but a different sample.

Image results

If the option Show image results (see below) is enabled, available results of data type 'Image' automatically pop-up in one single frame. The frame automatically appears as images are available, and disappears if there are none. 

This is accomplished by means of an embedded Internet Explorer window. It can be resized as desired, in which case the images will automatically re-arrange. Images can be viewed full-size by clicking on the initial image. The latter may require the following Internet Explorer option to be set: Tools > Internet Options... Advanced > Security > Allow active content to run in files on My Computer.  The HTML content is hard-coded and cannot be configured.

The images are collected and sorted as follows:

QUERY

  • Results of order.

SORTING

  • PropertyClassification sequence.

Result query options

The Options button in the order review screen allows specification of result selection criteria.  

General options

Results status from-to

These fields allow the user to specify a result status range.

Severity from-to

These fields allows the user to specify a result severity range.

Warning severity

When one of the results shown during confirmation/validation reaches a given severity level, it is possible to produce an extra visible/audible alert. To activate this feature, supply a severity level in the "Warning severity" field of the options dialog. On return of the dialog, the header will make room for an image at the top right-hand side of the window.  The warning image that is searched for should be named "warning.bmp" and located somewhere in the PROPATH. Its expected dimensions are 32 x 32. If you have a sound card, and "warning.wav" is found, the sound is played simultaneously. Simply remove the wave file if you want to get rid of the noise.

Conclusion property

If a conclusion property is specified, an extra window will be integrated in the validation screen which allows the user to enter a conclusion text.  A conclusion result will automatically be created for the order.  Shielded conclusions will be displayed as "XXX". When update of the conclusion property is denied, the widget will be displayed with a gray background.

Alternate classification

Allows the user to specify any 'display' classification, i.e. a property classification which will be used to display the results.   Eligible records are only pre-selected when belonging to the initial 'selection' classification (as specified in the query).

Warning

Make sure that the 'selection' classification is always a subset of the display classification. If not, you risk that results are hidden unnoticed. This may cause the amount of unconfirmed/unvalidated results and incomplete orders to accumulate.

Therefore:

  • The title bar contains extra information about the applied classifications in the format "(SelectionClassification[/DisplayClassification])".
  • Watch the selected order count that is displayed during the initial query. A continuously growing amount of eligible orders (right hand side of the count) may be an indication of a structural configuration problem. Classifications that do not completely overlap will probably be the reason.

Show previous run/material

If set, a "Previous run / material" column will be shown.  Using the cycle button in the column header, you can choose which column should be shown.

Show image results

See Image results.

Preview calculations

  • If disabled, calculated results are only shown if their input results are confirmed/validated.
  • If enabled, calculated results will be shown between parentheses if their input results are not yet confirmed/validated.

Warning

The site function specified in the PropertyOutput field Value (PropertyOutput.Value) is evaluated when previewing calculated values during order review or specimen review.

Therefore, the site function should never ever contain 'active' MISPL functions such as .AddRequest(), .CascadeRequest(), .Cancel(), .Discontinue(), .SetSiteAttribute(), etc. During the preview, the site function might be executed on premature, unavailable or unreliable inputs, thus triggering incorrect decisions based on unexpected results!

Only use PropertyOutput.Value for its intended purpose.

Hide unsolicited

If set, unsolicited results (i.e. results that were originally not requested) will not be shown.

Previous results options

Count

This option indicates the number of previous results to be displayed.  The dynamic time discriminator (dynamic panels) of a result is also taken into account when searching for a previous value for this result.

Minimal status

Minimal status for previous values to be displayed.

Hide unsolicited

The Hide unsolicited toggle-box allows determining whether unsolicited previous results should be displayed or not.

Restricted to the same material

Allows the restricting of the previous value columns to results assessed on the same material as the result being confirmed/validated. When the primary material variable is mandatory, the previous result will be checked to verify if it has the same specimen variable as the original result.

Relative time indication

This option determines the delta time indication that will be used for previous results. When checked, times will be shown as a lapse, with a unit depending on the time difference. When unchecked, an absolute date will be displayed; or when both results are of the same day, the time is shown instead.

The value of these options is saved in the user's personal settings.

Order locking during validation

When an order is shown in the Order review browser, it is marked (or locked) as being under validation for the current property classification. This lock is only removed when the user leaves the validation browser completely or re-executes the query.

When moving through the query, the next order is checked whether it has already been marked for the same property classification by another user. If so, the order will be skipped.  This process should minimize the risk of having two biologists validating the same order simultaneously, or even validating an order which was already reviewed by another biologist in his current validation session.

To avoid locking orders too long, upon leaving the validation program, all locks created by any user more than 30 minutes ago are automatically removed.
