Confirm by action

Contents: Introduction - Action query - Confirmation browser

Introduction

This program is available from the main menu by clicking Start > Routine > Actions > Confirm by action.

Action query

Activating this function first produces an action query dialog, as shown in the figure below.

Station

Allows for the narrowing of the search to actions of the station's procedures. 

Workplace

Only the actions performed at this workplace

Work list template (group)

Only actions belonging to this work list template or group of work list templates.  When this work list template has work list template item records specifying that only a limited number of outputs of the procedures should be shown on the work list, the program will take these into account.  Only actions where at least one of the 'restricted' outputs meets the specified status criteria are selected and only the restricted outputs are shown in the confirmation browser.

Action status range

Result status range

Minimal urgency

Results with an urgency below the one specified here will not be displayed.

Available since

Results older than this date/time will not be displayed.

Executing department (group)

Only orders having at least one result executed in the specified executing department (group)

Filter

Allows for the specifying of a site function to define additional query criteria. 

Sort by

It is also possible to change the order in which actions are presented to the user in the action confirmation program. The available choices are:

Time

Sort by availability time

Specimen

Sort by input specimen internal id

Location

Sort by input specimen last location

In case of the last 2 choice criteria, when multiple actions have the same sort value (e.g. same specimen), they are further sorted by availability time.

Confirmation browser

Upon committing the selection criteria, an expander dialog appears, as shown in the figure below.

General features

This dialog shows one action at the time. It also shows different results delivered by that action. The expander can navigate both at the master (action) and at the detail (result) level. The navigation scope of the expander is visible on the status bar. It can be explicitly switched between master and detail from the generic menu. Toggling the navigation scope affects all navigation commands. Use the Confirm button to confirm all pre-selected results.  The screen will automatically proceed with the next action.

Initially, the property column is focused.  Entering the value cell is assumed to be a result update signal.  A lookup (CTRL-F) from the result value cell activates a result code & enumeration choice selection list.  Click here for more information on result updates and comments.  If the result is updateable, the Result.Modify() dialog is invoked on RETURN or F6. Else, the standard result editor is called in view mode.  

The confirmation program keeps track of which results are already shown to the user. Only if all confirmable results (i.e. the ones that are pre-selected) have been viewed at, the corresponding button is enabled. So, if not all results fit into the browser, you will have to scroll down before you are able to confirm. This should avoid inattentive processing of hidden results.

Info window

On top of the action review dialog is a MISPL-configurable text area, determined by the Action text 'ActionExternalization'. It might contain something like:

{=.Procedure.Name} {=DateToString(DateTimeToDate(.ObjectTime), "%x")}
{=TimeToString(DateTimeToTime(.ObjectTime), "%X")}
{=.Object.PersonData("LastName,FirstName,BirthDate,Sex")}

Check the chapter on browser externalization zones for more information.

Other columns

For more information on specials marks, please check the chapter Result browsers.

Previous run

This column shows the most recent predecessor value of the result (if any). A predecessor value is the previous result of the same test that has been discontinued and repeated.  So the status of a predecessor value is always discontinued.  As a consequence, this value is always shown even if the result status range in the options does not include discontinued results.  If you want to see all discontinued results in the browser, you must include the "Discontinued" status in the result status range of the options screen.

All predecessor values are also shown in the comment box at the bottom left-hand side of the screen. They are displayed as a comma separated list, in descending order of assessment (i.e. the most recent first), and without their unit.

Previous results

The previous results are displayed in the right-hand side cells. The dynamic time discriminator (dynamic panels) of a result is also taken into account when searching for a previous value for this result.

Image results

If the option Show image results (see below) is set, available results of data type 'Image' automatically pop-up in one single frame. The frame automatically appears as images are available, and disappears if there are none. 

This is accomplished by means of an embedded Internet Explorer window. It can be resized as desired, in which case the images will automatically re-arrange. Images can be viewed full-size by clicking on the initial image. The latter may require the following Internet Explorer option to be set: Tools > Internet Options... Advanced > Security > Allow active content to run in files on My Computer.  The HTML content is hard-coded and cannot be configured.

The images are collected and sorted as follows:

QUERY

  • Result outputs
  • Procedure data

SORTING

  • ProcedureOutput.SeqNo
  • ProcedureData.SeqNo

Options

The Options button allows specification of result selection criteria (independent of the navigation scope) in a result query dialog, as shown in the figure below.

General options

Result status from-to

Allows for the specification of a status range.  

Severity from-to

Allows for the specification of a severity range.  

Warning severity

When one of the results shown during confirmation reaches a given severity level, it is possible to produce an extra visible/audible alert. To activate this feature, supply a severity level in the "Warning severity" field of the options dialog. On return of the dialog, the header will make room for an image at the top right-hand side of the window.  If at least one of the results has a severity higher than or equal to the warning severity level, an image will be displayed and a sound will be played. This also applies to procedure data records in the confirmation program, provided their history index is 0 (= of the same object time). The warning image that is searched for should be named "warning.bmp" and located somewhere in the PROPATH. Its expected dimensions are 32 x 32. If you have a sound card, and "warning.wav" is found, the sound is played simultaneously. Simply remove the wave file if you want to get rid of the noise.  

Show image results

See Image results.

Previous results options

Minimal result status

Minimal status for previous results to be shown in the confirmation screen.

Hide unsolicited

Allows for the determination of whether unsolicited results should be displayed or not. The value of this option is saved along with the other options in the user's personal settings.  This option is not taken into account for QC results, since all QC results are unsolicited.

Restricted to the same material

Allows for the restriction of the previous value columns to results assessed on the same material as the result being confirmed/validated. When the primary material variable is mandatory, the previous result will be checked to verify if it has the same specimen variable as the original result.

Relative time indication

Determines the delta time indication that will be used for previous results. When checked, times will be shown as a lapse, with a unit depending on the time difference. When unchecked, an absolute date will be displayed; or when both results are of the same day, the time is shown instead.
