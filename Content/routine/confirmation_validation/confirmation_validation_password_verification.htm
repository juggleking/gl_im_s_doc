Password verification for result validation

Introduction

Glims offers an option which will require the user to enter his/her password when validating a result.  This option allows to specify a password time-out. 

Configuration

In the Department configuration screen, you can specify a Validation password timeout (expressed in minutes) for result validation.  

This allows specific behavior per department.

Usage

  1. If a time-out is specified, the user will have to specify his/her password during the first result validation.  
  2. After the first result validation, the time-out will be applied.  Within that time-out, the user can continue validating results without password specification.  
  3. When the time-out is exceeded in between two validation actions, the user will have to enter his/her password again.
  4. The time-out will be reset each time a result is validated
