Specimen identifications

Contents: Introduction - Specimen internal id - Specimen external id - Specimen identification

Introduction

In GLIMS, a specimen can be recognized by different identifications (internal id, external id and specimen identification).  This document describes how and when these identifications are assigned and how they are used.

Specimen internal id

When manually entering an order in GLIMS and specifying the requests, GLIMS will verify which materials are needed and create the necessary specimens.  The latter will always receive an internal id. This id can be assigned by GLIMS using the MISPL functions SpecimenInternalId and SpecimenInternalIdOnInPut.  If the order is created in GLIMS via electronic order communication, this internal id can also be assigned by an external party (electronic order communication).

For information on how to (re)calculate specimen internal ids on the basis of work ids, please read the chapters Order.GetIdentifier and Order.RecalculateSpecimen.

Specimen external id

It is also possible that a specimen already has a barcode. This barcode can be entered, scanned or sent electronically and will typically be stored in the specimen external id. 

By default, the specimen external id has to be unique within the workflow. If your organization uses recurring external IDs, it is recommended to configure a specimen identification.

Alternatively, the environment variable "SpecimenExternalIdValidity" allows you to specify a period in time during which the external ID as barcode (scanned via specimen scan or queried by an analyzer) can be found unambiguously. If this variable is not found, the default period taken into account by GLIMS is 30 days.

Specimen identification

As the internal and external specimen id's may be too long to fit a barcode, it is possible to convert them to a specimen identification which can be used for analyzer communication.  This conversion can be configured by means of specimen identification types, which determine how the specimen identification is composed (e.g. based on the internal and external id).  

During order entry, GLIMS will check (via the procedures) on which station(s) the specimen(s) will be put to execute the requested tests.  For each station, you can indicate which specimen identification type should be used.  The corresponding identification(s) will automatically be created for the specimen(s).

Specimen identification types can be accessed from the main menu via Start -> Configuration -> Specimen identification types:

Mnemonic

Description

Code

Refers a specimen-based string-valued MISPL method which returns the actual specimen identification.  Example: RETURN (substr(.InternalId,4,?));

Validity

Number of days that this specimen identification is valid.

Evaluation

This option indicates when the specimen identification should be calculated

Once

The specimen identification will only be calculated once, i.e. during the creation of the specimen in the GLIMS database.

Always

The specimen identification will be re-calculated whenever the external id or the sampling time of the specimen changes.  This is typically used when a specimen identification is based on the specimen external id which is assigned by an external party (e.g. electronic order communication).  As the specimen identification is calculated as soon as the specimen is created in GLIMS, the external id is not yet available in GLIMS.  Therefore, you should choose this option to re-calculate the specimen identification as soon as the external id changes.

Potential problem: if the specimen identification MISPL contains a sequencer, a double increment may occur because the MISPL may be executed twice.  

Manual

The specimen identification is never calculated automatically. This option can be used in combination with the specimen reception scan program which can save tube barcodes directly as specimen identification (pre-printed sample barcodes).

Early receipt days

GLIMS allows to find specimen identification codes (barcodes) that arrive in the lab earlier than the validity start date of the identification code. This is useful when orders are planned for the future and it might happen that a specimen arrives earlier in the lab.

This option will have effect on the specimen receipt program and on station instruments that are querying or sending results for a barcode.  Take for instance a specimen that has been planned for 3 days further than today. When setting this field to "2", GLIMS will 'recognize' the barcode from tomorrow on.

Notes

Each time the specimen sampling date or receipt date is changed, the start date of a specimen's identification records is changed to the lowest value of the receipt date and the sampling date.  Also when the specimen identification is created, this lowest date is used as start date.  This allows for specimens that were scheduled to be sampled in the future to be received today, and still allow recognizing the identification codes when these specimens are put on an analyzer, after they have been properly confirmed, e.g. via Order.SpecimensConfirm().
