Basic routine -> Specimen -> Double specimen handling

Contents: Introduction - Tools

Introduction

Sometimes the lab receives more specimens than required or than calculated by the action scheduler in GLIMS. This may be intentional in order to supply extra material to the lab for long term storage or when the possibility exists that a single specimen will not be sufficient.

In such a case, the need sometimes arises to continue working with such an extra specimen instead of the specimen the actions where originally scheduled for. This may be because the original specimen is empty or destroyed.

Tools

To cope with such situations, GLIMS offers the following functionality:

  1. The Order.Specimens() browser shows the Specimen.LastLocation field of the specimens. This helps in finding a specific specimen.
       
  2. The Specimen.Replace function allows showing all specimens of an order containing the same material as the original specimen.  This browser can be called both from the specimen contextual menu and from the main routine menu.  After determining the original specimen, all specimens of the same material within the same order are shown.   

Note

The Specimen.Replace function cannot be used for specimens of microbiology actions.

The browser shows the internal id, status, material, sampling time, discriminator and the home location, last location or storage location (by clicking the cycle icon in the location column header).

The Specimen.Replace function allows to move non-finished actions from a specific specimen to another specimen of the same material in the same order.  When selecting a specimen and hitting the Replace button (or double-clicking the target specimen), you will first be presented with a browser of all the unfinished actions on this specimen, with all the actions preselected.  If you want to move all the actions, simply hit the Ok button or F2 key. You can deselect or select actions as you wish. Selecting all procedures via Ctrl-A or deselecting all procedures via Ctrl-N is supported. The Repl column indicates if the procedure field Allows replacement is enabled, but currently this field is not actively checked.
