Specimen info screen

Content: Introduction - Collection information - Results of order - Material variables - Other results - MISPL function

Introduction

The Specimen info screen

  • provides information regarding the collection of a selected specimen.
  • allows you to enter results for this specimen in the order.
  • displays the results for this specimen used in other orders.

This screen can be opened

  • from the general Order entry screen -> Specimens menu/ribbon item -> select a specimen in the embedded browser -> Press F6 or click the Specimen info menu/ribbon item.
  • from the Genetic exam screen -> Select a specimen in the Specimens or Other specimens pane ->
    • press F6.
    • Specimen info menu/ribbon item (-> if the specimen is used in more than one non-discontinued order, a Choose order window opens -> select an order).
  • From the Specimens of order browser -> Specimen info menu/ribbon item (-> if the specimen is used in more than one non-discontinued order, a Choose order window opens -> select an order).

Clicking on the Detail button at the bottom of the screen opens the specimen's editor.

The values of the Specimen info fields are stored in the table SpecimenCollectionInfo and can be retrieved using MISPL.

Collection information

Root specimens

This section describes the fields displayed when the Specimen info screen is open for a root specimen, i.e. a specimen which has no parent specimen in the current order.

Internal id - External id - Material - Status - Discriminator - Sampling time - Received on

These fields contain the same information as the corresponding fields in the Specimen editor.

Note

If you update a field manually in the Specimen info screen, you need to press F5 for the corresponding field in the Specimen editor to be updated accordingly.

Differently, if you update a field manually in the Specimen editor, you need to close and reopen the Specimen info screen for the update to be visible in this screen.

Sampler

HC provider who sampled the specimen (in the first order to which the specimen was linked).
The first field displays the sampler's internal id. The second field displays a string, such as the sampler's name.
The button "..." allows for the selection of a HC provider.

Sampler info

This read-only field contains the name of the sampler if
  • the sampler is set on the basis of information received from an electronic order message, and
  • the sampler is not known in GLIMS as a HC provider.

Note

If the electronic message contains several samplers for a given specimen, the first sampler of the SpecimenList is saved in the Sampler field (if known in GLIMS as HC provider) and the other samplers are saved in the Sampler info field.

Sampling location

The possible values are the same as for the Sampling location field in the Order entry screen. Series is however not part of the choice list.
When a specimen is added in the Genetic exam screen, its sampling location is set automatically to that of :
  • the order, if the specimen is the first one added to the genetic exam,
  • the most recently added specimen, if the specimen is not the first one in the genetic exam.

However, the sampling location can be updated manually for each specimen and is independent of the sampling location indicated in the Order entry screen.

Sampling address

This field is only displayed when the field Sampling location is set to Specific.

Container usage

Choice list of all the available material container usages. If the container definition of the chosen container usage is linked to an image, this image is displayed.

Received quality

Choice list of items belonging to the Quality choice list of the material. This field is only editable on the first order of the specimen.

Received quantity

Total amount collected for the specimen. The unit is that defined in the Size unit field of the Material editor.

This field is

  • only displayed if the option Specimens can have a quantity is activated in the Material editor.
  • only editable on the first order of the specimen.

Container count

Number of containers in which the specimen is placed.

This field is

  • only displayed if the option Specimens can have a container count is activated in the Material editor.
  • only editable on the first order of the specimen

Prenatal

This option indicates whether the root specimen was collected on a foetus. When it is activated, you can

  • select an existing Person record for the foetus in the choice list. This list contains all the non-deceased foetuses of the mother's current and former pregnancies.
  • modify the chosen Person record using the "pencil" icon,
  • create a new Person object record for the foetus using the green icon:

Warning

In practice, specimens should only be marked as Prenatal if the order's object (the mother) has sex Female and is registered as pregnant. However, GLIMS does not check that this is indeed the case.

Pregnancy no

One or two digits to identify the pregnancy. This number is stored on the foetus Person record.

Set by default to "1". If the order's object (the mother) contains finished pregnancies, the pregnancy number for the new pregnancy is incremented automatically.

Note

To retrieve the pregnancy number, you can use the following MISPL function:

Specimen.CollectionInfo().Foetus.PregnancyNo

Genotypic gender

The following values are available:
  • ?: unknown
  • XX: female (normal)
  • XY: male (normal)
  • XO: female (Turner)
  • XXY: male (Klinefelter)
  • XYY: male (XYY)
  • XXX: female (triple-X)

Twin

Activated if there is more than one foetus in the pregnancy.

Placenta

The following values are available:
  • ?: unknown
  • Dichorial: each twin has its own placenta
  • Monochorial: single placenta shared by the twins
  • Other: e.g. conjoined twins

Discriminator

Mandatory field when Twin is activated. One or two characters, such as "F1" or "F2", to distinguish the foetus from the other foetuses of the same pregnancy.

This can be useful for work lists.

Note

To retrieve the foetus discriminator, and for instance add it to the specimens' internal id, you can use the following MISPL function:

Specimen.CollectionInfo().Foetus.FoetusDiscriminator

Twin type

Mandatory field when Twin is activated. The following values are available:
  • Unspecified: unknown
  • Monozygotic: identical twins
  • Dizygotic: fraternal twins
  • Other: e.g. sesquizygotic

Pregnancy information

This fields are only activated if the foetus' mother is the order's object.
First indicate
  • either the number of days/weeks/months since the beginning of the pregnancy, or
  • the estimated start/end date of the pregnancy.
The information
  • is saved in the "Pregnancy" object attribute field on the mother's Person record.
  • can be modified in the Pregnancy row of the Attributes tab in the Order entry screen.

When these fields are filled in,

  • a Person record is automatically created: the Mother field of that record is filled in automatically with the order's object (the foetus' mother) and the option Foetus on creation is activated.
  • the information is displayed in the Foetus pane of the Family screen.

Material variables

If material variables are defined on the specimen's material, they are displayed in the Specimen info screen. Their Variable and info fields can be edited directly in this screen.

Note

If mandatory material variables are defined on the specimen's material, the corresponding specimen variables have to be indicated when the specimen is added in the Genetic exam screen. However, in the Specimen info screen, these variables may be cleared. In other words, in the Specimen info screen, GLIMS does not check whether the material variables configured as mandatory are specified.

Other results of specimen in other orders

Specimens can be reused. When the Specimen info screen is open for a root or derived specimen that has already been used in another order of the patient, a grid appears at the bottom of the screen. This grid shows the same properties as in the Results of order part but which were assessed in other orders.

If there is more than one result for a property, the results are shown in separate columns.

The grid contains the following elements:

Property column

Property short name and unit inside brackets.

Date columns

The headers of these columns contain the date of the results' availability times, starting with the most recent result on the left. If there is more than one result with the same availability date for the same property, the one with the most recent time (i.e. same date but later hour) is used.
