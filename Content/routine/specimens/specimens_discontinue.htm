Specimen.Discontinue

This function allows to discontinue the selected specimen. A typical example could be a broken tube.

When a specimen is not received, everything derived from that specimen is to be discontinued. However, it might still be desirable to send a report anyway.  This can be accomplished in GLIMS by discontinuing the specimen and all its sub-specimens, and by putting a result replacing code in the result values.  The Specimen.Discontinue() function allows to do so by means of the Cascade toggle-box and the Result code field.

Justification

This field can contain text modules.  The justification will be copied to the internal comment of all processed results.

Cascade and Result code

If the Cascade toggle box is selected, the Result code field is mandatory. Instead of simply discontinuing all results, the specified result code will be applied to all derived results that are not yet available.  When during processing, results with status >= Available are found, the user will get a question whether or not he also wants to process these results.

Both Cascade and Result code are remembered in the user preferences. By means of the result code, the desired action can be configured. The results get a result replacing code and can be confirmed, validated or discontinued. A result code with action 'Repeat' or 'Dilute' will not be accepted. 
Take care of the ResultCode.ToCharge field. Make sure it is set to 'No' if you do not want the requests to be tariffed!

Note

  • The specimen discontinuation program only allows selection of a general result code. However, during the processing of the results, a property-specific result code with the same mnemonic has precedence over the general result code (just as is the case during manual result entry). Special care should be taken for properties who do not allow general result codes: when the result code specified (as general result code!) in the specimen discontinue screen does not have a property-specific counterpart for this property, the mnemonic of the result code will silently be stored as result value, but the actions attached to the result code (e.g. Validate, Discontinue, ...) will NOT be executed.   
  • All parameters except the justification are stored in the user's preferences and reused the next time the function is used.  
  • The Specimen.Discontinue() function has pre-configuration capabilities: it can e.g. be pre-configured as a menu option with a fixed rejection reason.   
  • The Specimen.Discontinue() function can also called from batch.
