Confirmation of specimen availability

Introduction

Depending on the use case, GLIMS offers several programs to confirm the availability of specimens.  Each program has its specific advantages, depending on the workflow within the lab.

Different specimen confirmation programs

  1. Specimen scan
  2. Specimen reception scan
  3. Confirm specimens by order
  4. Process specimens by number
  5. Confirm specimens by sampling list

Encounter evaluation during confirmation of specimen availability

Encounter re-evaluation

When confirming the first specimen in an order (no other specimen in that order is already confirmed), the encounter of the order will be re-evaluated. This only applies when the general setting Encounter selection is not set to None.  This mechanism can come in handy when e.g. orders are created beforehand in CyberLab and no encounter is available yet at that time.

This option can be set in the general settings (Start -> System management -> Customize -> GLIMS general).

Options:

It is possible to have a popup selection browser allowing manual selection of the correct encounter. If needed, the browser appears as soon as a person is selected.

Manual

The browser always appears, even when there is only one open encounter.

Manual When Ambiguous

The browser appears as soon as there is more than 1 open encounter for this person or if the most recent encounter has no External Id.

Automatic

The manual selection browser only appears when the encounter that would be automatically selected has no External Id.

Automatic when hospitalized

Same as automatic, but non-hospitalized encounters are ignored in the search for an encounter that can be selected.

User defined

This option offers a customizable mechanism to automatically select the appropriate encounter for electronic orders and orders placed in the future.  Glims can automatically select an encounter based on a calculated score for each encounter.  The encounter with the highest score will be selected. 

The encounter score is calculated by a MISPL expression which is defined in the  Encounter selection trigger field in specific site (Start -> System management -> Customize -> GLIMS general).  The MISPL expression has base table "Order" and should return an integer which will be the encounter score.

Encounter selection

  • The selection of the best suitable encounter is based on the time of arrival of that specimen (the actual time that this happens).  This will make sure that an encounter is chosen that is open at the time when the specimen is received.

  • The receipt time of the order can be automatically updated to the actual time at which the first specimen was received.

In light of the Electronic Order Scheduling (EOS) module, where the receipt time is also updated to the time of order activation, this feature cannot be generalized, so it can be activated via an option in the GLIMS general settings, labeled as "Sync receipt time when first specimen is confirmed".

This will make sure that the order's receipt time will fall in the period of the encounter that was selected.

Unconfirm specimen availability

In order to be able to indicate that one or more specimens for a particular order are not yet available, take the following steps:

  • Use the contextual menu Requests -> Specimens on Order. A browser will be displayed with the specimens for the selected order.
  • Select the specimens which are not yet available and use the contextual menu function Unconfirm availability to mark them as such.

Note

If the availability of the specimens has been confirmed automatically (order entry options: automatic sample login=YES), they will never be displayed on a sampling list, even after unconfirming the specimens (see above).  GLIMS does not create actions for sampling procedures when the automatic sample login is enabled.
