Specimens - Change sampling time

Introduction

The Specimen-based function Change sampling time allows to actually change the sampling time of a specimen in much the same way as the Order.ChangeObjectTime function.

The object time of the results derived from this specimen and of the actions related to this specimen will also be updated.

Fields

New time

The new sampling date and time for the specimen. The Now button allows to set the current date and time (rounded to the minute) as new sampling time.

Discriminator

New specimen discriminator.

Sampling time confirmed

Allows to explicitly mark a specimen as Sampled. As long as a specimen is not marked as Sampled, its Sampling time is considered to be a scheduled sampling time.

Note

A specimen can also be marked as Sampled through communication with an external system (e.g. CyberLab).

Notes

  • Changing the sampling time of a specimen does not trigger recalculation of its Discriminator field when the specimen has a related dynamic result: you have to change the Specimen.Discriminator manually. 
  • When the sampling time of a specimen is changed using this function, the corresponding reports where Report.Medium = Communication are marked with Report.NeedsChecking=YES.  This will allow automatic resending of these reports (providing the NewResultCheck report generation parameter is set to 'Ignore').
  • Changing a specimen's sampling time triggers the recalculation of the order's lowest object time, except for specimens used in more than one order.
