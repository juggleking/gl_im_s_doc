Specimen reception scan

Contents: Introduction - Usage - User interface - Options - Remarks

Introduction

This program allows sample reception via scanning. It can be accessed from the main menu via Start -> Routine -> Specimens -> Reception scan.

Usage

The specimen reception scan program has many possible uses:

  • It can be used for simple specimen confirmation. In this case the program simply finds the specimen and confirms it. In this mode, the program tries to determine the specimen using the standard search mechanism. The selected station determines if the search should be done via a specimen identification, the external id or the internal id.
  • It can be used to assign the correct bar code to the sample. If the Station field Search via order is set and the Material is specified in the scan program, then the program searches an unambiguous specimen of this material in the order determined by the SpecificSite.BarCodeParser site function. Via the Allow selection option, the program can even allow the user to choose between equivalent specimens of this order. If the specimen is found, then the scanned bar code is assigned to its specimen identification or external id, depending on the Station settings.

  • When the Station fields Create specimen if necessary and Create order if necessary are set and the scan option Allow creation is set, the program can even create an empty order and/or an unsolicited specimen when the previous steps did not yield an existing specimen. This mode allows automatic creation of specimens not expected for this order, or even the reception of specimens PRIOR to order entry.

    Warning

    This option should be used with the utmost care.

User interface

General fields

Bar code

Allows scanning a sample barcode.

Station

Enter a dummy station that is used as "reception" station.

Material

When scanning a specimen that is not yet known in GLIMS, you will have to specify the material of that specimen. You can do this manually or use the "Material button screen" functionality (see option Material buttons described below). If the specimen is already known in GLIMS, the material is not to be specified.

Selected button

Shows the name of the selected material button. For more information, see Show button screen on start.

Single scan mode

When used in single scan mode (i.e. the option Use buffered mode is not enabled (see below)), each scanned specimen will be processed immediately. In the History tab page, the last processed specimen is shown and a log can be consulted of the scanned specimens. The button Show specimens allows reviewing the last scanned specimens.

Buffered scan mode

Instead of processing each scanned specimen immediately, you can also choose to work in buffered mode (by enabling the option Use buffered mode, see below). This allows scanning multiple specimens (which will be listed in a browser) and then processing these specimens in one action.

Buffered mode page

P

Indicates if the specimen has been processed (after clicking the Save all button).

N

Indicates that the scanned specimen is a new specimen (i.e. not yet known in GLIMS).

Options

The icon in the lower left corner gives access to the options screen:

Only root specimens

If enabled, when looking for a matching specimen in GLIMS, only root specimens will be considered.

Allow selection

The reception scan program can also be used to assign the correct bar code to a sample. If the Station field Search via order is set and the Material is specified in the scan program, then the program searches an unambiguous specimen of this material in the order determined by the SpecificSite.BarCodeParser site function.

Via the Allow selection option, the program can even allow the user to choose between equivalent specimens of this order. In that selection screen, GLIMS shows the property codes. If the property code is unknown, the short name is shown. If the short name is unknown, the mnemonic is shown.

Allow creation

When the Station fields Create specimen if necessary and Create order if necessary are set and the scan option Allow creation is set, the program can even create an empty order and/or an unsolicited specimen when the specimen could not be found. This mode allows automatic creation of specimens not expected for this order, or even the reception of specimens PRIOR to order entry.

This option should be used with the utmost care!

Synchronize object times

Synchronize future object times

When a specimen or order is processed while this option is active, its sampling time will be compared with the current time. When the sampling time is higher than the current time (i.e. it was intended for a sampling time in the future), the object time of the order is synchronized. For simple orders (only one object time) this implies that the object time of the order and all its specimens and results is set to the current time.

Synchronize past object times

Also allows synchronizing the object times of orders where the object time lies in the PAST. This is useful for labs where the actual specimen receipt time is a better indication for the real sampling time than the one introduced during order entry.

Notes

  • In both directions (synchronize forward and backward), the program tries to ensure that the largest sampling time in the order is synchronized with the current time, so that all sampling times of the different specimens in the order are smaller or equal to the current time.
  • An order object time change is logged in the order log.
  • The synchronization options of the specimen scan and specimen receipt programs have priority over the settings of the selected station in these programs. Please consult the Mark as synchronized option in the Order entry documentation to disable unwanted backward / forward synchronization and to check the Synchronized flag of the order.

Use buffered mode

Instead of processing each scanned specimen immediately, you can also choose to work in buffered mode. This allows scanning multiple specimens (which will be listed in a browser) and then processing these specimens in one action.

Reschedule at close

If this option is activated, rescheduling is postponed until the scanning screen is closed. If it is deactivated, rescheduling takes place every time a specimen is scanned and becomes available.

Asynchronous specimen confirmation

Indicates whether the specimen confirmation (and thus the specimen rescheduling if Reschedule when specimen becomes available is active) should be delegated to the AppServer.

Note

The Asynchronous specimen confirmation option only has effect when the option Reschedule at close is not active.

Skip setup order activation

If this option is set, GLIMS will activate pending orders without showing the order activation screen to the user.

Asynchronous barcode processing

If this option is set, the barcode processing will be done asynchronously on the AppServer in order to allow the user to continue scanning specimens while the AppServer is processing the barcode. This results in a considerable performance benefit.

Warning

Asynchronous barcode processing cannot be used if any user input (for instance: specimen selection, interactive MISPL functions, user actions during order activation, etc.) is required during the barcode scanning.

New internal id

Allows choosing a Specimen-based, string-valued site function which will be executed on the scanned specimen in order to determine a new internal id. Note that it is the responsibility of the site function to determine whether this specimen requires a new internal id. When changing a specimen's internal id for the first time in each session, the user is asked for confirmation.

Material buttons

With this field, you can define a screen with material buttons. This allows choosing a material after having scanned a new specimen (not yet known in Glims) via a screen with buttons where each button represents a material. This can be more efficient for the user than filling out the material field each time, especially when using a touch screen.

The field actually contains the mnemonic of a gp_Site-based dynamic text describing the contents and size of the button screen, together with the position, size, label and return value of each button.

A screen definition text should now comply with the following format:

"<Screen title>" <Width in pixels> <Height in pixels>
. (separator)
"<Tab page name>" <Tab page position>
. (separator)
"<Button Label>" <X position> <Y Position> <Width> <Height> <BorderWidth> <Font> <Bordercolor> "<Return value>" "<Tooltip>" "<Image>" <Tab page position>

The first line contains a global screen definition:

"<Screen title>" <Width in pixels> <Height in pixels>

The next line contains only a period (.) and serves as separator

The line(s) after the first separator line allows defining tab pages. Format:

"<Tab page name>" <Tab page position>

If no tab pages are needed, simply specify a period (.).

The following lines contain the button definitions, one button per line, using the syntax

"<Button Label>" <X position> <Y Position> <Width> <Height> <BorderWidth> <Font> <Bordercolor> "<Return value>" "<Tooltip>" "<Image>" <Tab page position>

X, Y, Width and Height are all in pixels. The only mandatory fields are <Button Label> and <Return value>. Do not put any comments in the text !

The configured font should refer to a font that is defined in the progress.ini file (Fonts section).

Once a button is chosen, its <Return value> will be used as material mnemonic for the scanned specimen. You can also specify a request code as <Return value>. If so, GLIMS will add the request definition variables to the specimen (e.g. sampling origin or method). The format for the <Return value> is "Rqcd\" + <request code>

If the reception scan program should ask for a discriminator, use the following syntax:

"<button label>" ... "Rqcd/<Request code>/<Discriminator>" ...

So, there are 3 entries, separated by a forward slash. This implies that the discriminator functionality is only available for request codes, not for materials! The inputted value will be used to create/update a specimen with discriminator (Specimen.Discriminator).

Example

Example of a dynamic text for a button screen:

"Choose a material" 900 400
.
"Page 1" 1
"Page 2" 2
.
"Serum" 8 8 100 63 0 6 ? "Serum" "Serum" ? 1
"Req. Ur-MidStream" 110 9 147 104 0 ? ? "Rqcd/Ur-MidStream" "Add request code Ur-MidStream" ? 1
"DryBlood" 8 8 100 63 0 6 ? "Serum" "Serum" ? 2

Once displayed, the screen can be customized by hitting the <F12> function key and following on-screen instructions. It is also possible to customize the label, tool tip and return value of a button by simply double clicking it.

Show button screen on start

When this option is enabled and a material button screen is selected (see option Material buttons described above), the selected material button screen will be opened when opening the specimen reception scan program and will stay open as long as the specimen reception scan screen is open. Instead of having the material button screen pop up at each scan, the user can now click once on a material button in order for the selected button to be taken into account for several scans. The selected button will also be visible in the specimen reception scan screen (see Selected button described above). If a Material is specified in the specimen reception scan screen, the currently activated button will be ignored.

Remarks

  • Most of these functions are also available when a station performs a specimen query or result download, or when a work list is created using specimen scanning or when scanning a bar code in the original specimen scan program. This way, any station (e.g. a regular blood analyzer, a specimen sorter or an aliquotter) can confirm existing specimens, assign bar codes to specimens or even create new specimens.
  • Disable automatic specimen confirmation upon scan: please consult the description of the GLIMS general setting Confirm specimen on scan.
