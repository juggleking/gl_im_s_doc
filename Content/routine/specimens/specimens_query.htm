Specimen query

The specimen query is available from the main menu via Start -> Routine -> Specimens -> Query. The specimen query screen looks as follows:

Sampled

Allows selecting only already Sampled specimens, optionally within a desired date range. This status can be achieved by activating the Sampling time confirmed option on a specimen.

Type of specimens

Allows selecting only Mother or Work specimens, or Both.

All actions ready

If set to Yes, only the specimens for which all actions are finished will be selected. If set to No, only specimens with at least one unfinished action will be selected.

Executing department (group)

A specimen will be selected if:
  • Any of the departments under Specimen -> Specimen input -> Action -> Procedure -> Department matches any of the following executing departments:
    • the department specified in the field Executing Department, OR
    • any of the executing departments specified in the field Executing department group
  • OR if this is true for any of the sub-specimens.

Specimen browser

The specimens that meet your selection criteria are displayed in the specimen browser from which all specimen-related functions are available (discontinue, unconfirm availability, etc.).

'Specimens of order' browser

From each order, you can use the contextual menu/ribbon item Requests -> Specimens to open a browser with all specimens needed by this order. The same browser is also available from the main menu via Start -> Routine -> Specimens -> By order.

Note

When scanning/entering an order in the Specimens of order screen, and the program is not set in 'auto' mode, the barcode field stays insensitive until the user has explicitly processed or cleared the already scanned order.

This browser is a dynamic browser. This implies that clicking on a column header changes the sort order of the records. An arrow indicator in the column header shows that the browser has this functionality and indicates the current sort column and order. Note that the current sort column also becomes the column used for fast positioning.

Barcode

This field is only available if this specimen browser is opened from the main menu (Start -> Routine -> Specimens -> By order). It allows entering an order internal id or order short id. The specimens for that order are shown in the embedded browser.

Browser columns

The first column may contain an urgency indication (!). Clicking the Column icon in the upper left corner of the embedded browser allows choosing which columns should be displayed.

Print

Prints the specimen labels

Process

When clicking the Process button, all actions defined in the Options screen will be executed.

Options

Status from - to

Only show specimens of which the status falls within this range.

Show root / work specimens

Only show root/work specimens or both.

Preselect root specimens

If set, the root specimens will be pre-selected in the embedded browser.

Preselect work specimens

If set, the work specimens will be pre-selected in the embedded browser.

All departments

Is enabled by default. When disabled, only specimens which are input to at least one action for the current user's department will be shown.

Filter

Allows specifying a Specimen-based filter site function to fine-tune the specimen query.

Confirm

If set, the availability of the selected specimens will be confirmed when hitting the Process button.

Synchronize future object times

When a specimen or order is processed while this option is active, its sampling time will be compared with the current time. If the sampling time is higher than the current time (i.e. it was intended for a sampling time in the future), the object time of the order is synchronized. For simple orders (only one object time), this implies that the object time of the order and all its specimens and results is set to the current time.

Synchronize past object times

Allows synchronizing the object times of orders where the object time lies in the PAST. This is useful for labs where the actual specimen receipt time is a better indication for the real sampling time than the one introduced during order entry.

Notes

  • In both directions (synchronize forward and backward), the program tries to ensure that the largest sampling time in the order is synchronized with the current time, so that all sampling times of the different specimens in the order are smaller or equal to the current time.
  • An order object time change is logged in the order log.
  • The synchronization options of the specimen scan and specimen receipt programs have priority over the settings of the selected station in these programs. Please consult the Mark as synchronized option in the Order entry documentation to disable unwanted backward / forward synchronization and to check the Synchronized flag of the order.

Own labels, Sublabels

If set, the own and/or sublabels are be printed when hitting the Process button.

Carrier labels

If set, the carrier labels are printed when hitting the Process button.

Labels at end

Allows postponing the actual printing of the generated labels to the end of the session, i.e. when the browser is closed or when the Print button is pressed. This allows ensuring that the requested labels are printed in a single block, without being interspersed with labels from other users. The labels are sorted by Order.InternalId + Specimen.InternalId.

Automatic

Allows the execution of the selected actions as soon as an order barcode has been entered, i.e. without having to press the Process button. This option only functions in 'standalone' mode.
