Specimen scan: how does it work?

Contents: Introduction - Fields and buttons - Moving actions - Confirm specimen when scanned

Introduction

The specimen scan program is available from the main menu (Start -> Routine -> Specimen -> Scan). This program scans the specimen identification (or the specimen internal id). The specimen scan program will always try to find a specimen where Specimen.InternalId is equal to the scanned bar code, even when the selected station uses specimen identifications.  The practical usage of this program is described here.

Fields and buttons

General

When scanning, the Last location of the specimen is automatically updated. Optionally the Storage location or Home location is updated. The current station must be selected. 

The Rack capacity and Next position fields are used for specimen archiving.  The currently entered rack or position can be cleared explicitly using 2 buttons located next to the rack and position entry fields. This allows selecting any rack or position in a rack (including a rack for which no additional scanning is currently allowed) and clear the contents of that rack or that position. 

Notes

  • During the deletion of an ItemStorage record, the Specimen.StorageLocation field is either replaced by the location stored in the most recent alternative ItemStorage record of this specimen, or is cleared when no other item storage record can be found.
  • The scan program checks privileges defined for the function "Rack.ClearRack" and hides the "X" button (clear full content of rack) if the user does not have the appropriate privilege.  Note that the button to clear only 1 position of the rack is not protected by this privilege.

Archive (sub) reason

These fields are used for the specimen archive module. They only become active when the field Save into is set to Storage location.  These two fields are filled with gp_text mnemonics starting with "ARCH-" and "ARCH-S-".  The latter two prefixes are stripped when displayed in the specimen scan program (cf. example: "text1" and "text3"). The actual text modules must be based on the Specimen table.   
  
If the Automatic option is enabled, you will have to select the archive (sub) reason before entering the barcode.  It is also possible to query on these two specimen scan fields in order to obtain a list of specimens archived with a specific code.

Last scanned

For the scanned specimen the following data are displayed by default:

Current position

Specimen internal id

Barcode

Order, issuer and object

Available volume

Previous location

Home location and storage location

The workflow (actions with status, station, volume needed and result properties) and general status

Patient info

The object field shows the object externalization of the last scanned specimen's object.

The highest urgency of the actions related to the specimen, in the colors defined via the order urgency color scheme.

The text in this info box can be customized via a specimen-based dynamic text specified in the options page (specimen info).

Options

Allow creation of specimen or order

Allow creation of specimen or order when permitted by the station.  

One rack per day

Is used for the archive module. Allows to suppress the warning message which is shown when scanning the code of a rack that already contains non-expired specimens of another day.  

Show details

If set, a detailed description of the workflow will be displayed.  

Show actions of current department only

When enabled, only the actions of the current user's department will be listed for the scanned specimen.

Note

When the specimen scan option Check if specimen is ready is also enabled and the actions related to the current user's department are in status 'Ready', a message will inform the user that the specimen is ready (even though other actions for other departments might not be).

Show actions with status 'Ready'

If set, actions with status 'Ready' will also be displayed.  

All levels

Show only the first level actions (i.e. the actions where the scanned specimen is input) or all action levels (i.e. also the actions on the subspecimens).

Show sub-actions

If set, instead of showing a list of the actions, the procedures scheduled on the specimen are shown. Each procedure is represented by a box with a color as defined in the Station.Color field.  When the status action is 'Realized' the background color of the actions will be changed to grey. When the action is 'Read', the color will be changed to light gray.

The box contains the station mnemonic, the procedure code or name, the required volume and a status indication of the actions. Note that when repeat actions have been created, all actions of the same procedure are represented by the same procedure box.  Each action's urgency is also shown in the station box.  

Reschedule at close

If this option is activated, rescheduling is postponed until the scanning screen is closed. If it is deactivated, rescheduling takes place every time a specimen is scanned and becomes available.

Asynchronous specimen confirmation

Indicates whether the specimen confirmation (and thus the specimen rescheduling if Reschedule when specimen becomes available is active) should be delegated to the AppServer.

Note

The Asynchronous specimen confirmation option only has effect when the option Reschedule at close is not active.

Double storage action

Allow: when a specimen already has a location, the new location is created without warning.

Confirm: when the specimen already has a location, the user is asked if he is sure to add this new location. Upon a negative answer, the new location is not created.

Refuse: the program does not allow scanning a new location for a specimen which already has a location

Replace: when the specimen already has a location, the user is asked if he wants to replace the existing location(s) of the specimen with the new location. Upon a negative answer, the new location is not stored.  

Specimen info text

Allows specifying a specimen-based dynamic text which determines the contents of the specimen info box in the 'Last scanned' page of the specimen scan program.

The height of the zone used to display specimen info text can be easily changed on the fly by dragging the separator between the specimen info and the work flow widget up or down.

To allow a more permanent setting of this height, the SpecimenInfo widget also supports settings its height via the "<!--height=xx-->" syntax in html text definitions, in exactly the same way as is already supported when using a html header text for specific browsers.  This way, the height of the specimen info widget is determined by the text definition itself. 

New internal Id function

Allows choosing a specimen-based, string-valued site function which will be executed on the scanned specimen in order to determine a new internal id.  Note that it is the responsibility of the site function to determine whether this specimen requires a new internal id.  When changing a specimen's internal id for the first time in each session, the user is asked for confirmation.

Actions

Automatic

If set, no confirmation will be required.  

Own labels / sublabels

If the Own labels option is set, GLIMS will print the 'own labels' of the scanned specimen. If the Sublabels option is set, GLIMS will print the 'own labels' of the actual, physical subspecimens (if any) AND the sublabels of the scanned specimen (material input labels with MaterialInput.IsSubLabel=YES).  For more information, check the chapter on specimen labels.  

Carrier labels

If set the carrier labels will be printed.

Isolation test labels

Are printed in the sequence defined by Procedure / MicroOrganism / MicroOrganismTest SeqNo to provide a fixed sorting. The isolation test label printer defined for the current user is used.

Check order completeness

If set, the program will automatically check if all specimens of an order are scanned (without having to press the Check order button. This check will take place when the first specimen of a new order is scanned (only then the program can check the previous order)   

Check if specimen is ready

Allows prohibiting the assignment of the rack position to the specimen when not all of the specimen's actions have attained status 'Ready'.

Synchronize future object times

When a specimen or order is processed while this option is active, its sampling time will be compared with the current time. When the sampling time is higher than the current time (i.e. it was intended for a sampling time in the future), the object time of the order is synchronized.  For simple orders (only one object time) this implies that the object time of the order and all its specimens and results is set to the current time.  

Synchronize past object times

Also allows synchronizing the object times of orders where the object time lies in the PAST. This is useful for labs where the actual specimen receipt time is a better indication for the real sampling time than the one introduced during order entry.

Note

Remarks about object time synchronization:

  • In both directions (synchronize forward and backward), the program tries to ensure that the largest sampling time in the order is synchronized with the current time, so that all sampling times of the different specimens in the order are smaller or equal to the current time.
  • An order object time change is logged in the order log.
  • The synchronization options of the specimen scan and specimen receipt programs have priority over the settings of the selected station in these programs. Please consult the Mark as synchronized option in the Order entry documentation to disable unwanted backward / forward synchronization and to check the Synchronized flag of the order.

Buttons

All specimens of the current order are checked and a list of the unscanned root specimens is displayed.
Opens the order entry screen for the specimen's order, which allows viewing more information regarding the specimen or even modifying some data.
Opens the specimen editor, which allows viewing more information regarding the specimen or even modifying some data.
Print own and sub specimen labels for the current specimen.
Allows to mark the scanned specimen with Specimen.Distributed = YES. When the scanned specimen is already marked as such, the button is not sensitive.

When a user uses this button to mark a specimen as being distributed manually, the program will search all non-ready actions having this specimen as input and which belong to a 'sample distribution' station (i.e. where Procedure.Station.Type = "Distributor").  When such actions are found, any unavailable output results are discontinued, while any specimen output record referring to unavailable specimens are deleted.

The logic is based on the assumption that when the user states that the specimen has been distributed manually into all its required sub-samples, we can assume that the specimens which were scheduled to be derived from this specimen by the distributor no longer have to be created by the distributor station.  

So, whether the distribution procedure makes use of (dummy) properties (which typically receive the created aliquot position as result) or submaterials as outputs, if a query for the original bar code is sent from the distributor, Glims will reply that no further subsamples are required.  In the latter case (where the distributor procedure has specimen records as outputs), the resulting specimens still have to confirmed as being available (e.g. by scanning them in the same specimen scan program).

Clear

Ignore the new home and storage location.

OK

Accept the new home and storage location.

Moving actions

The specimen scan program will move actions to the scanned specimen whenever required and possible. Click here for more information about the conditions for moving actions.

When one or more actions have been moved after the specimen was scanned, a message is shown (automatically disappears after 1 second) with a bell sound.

The tool tip of the specimen information box shows the actions being moved.

Confirm specimen when scanned

Normally, when a specimen is scanned, it is automatically confirmed if this was not yet the case. However, the GLIMS general setting Confirm specimen on scan allow you to limit the automatic confirmation to specific situations.
