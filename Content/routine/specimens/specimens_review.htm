Specimen review

Contents: Introduction - Main specimen browser

Introduction

The standard menu location of the Specimen Review program is Routine > Specimens > Review. 

This program is essentially an elaborated specimen query, which can also be used in batch commands. When starting the program interactively, two browsers appear simultaneously: a specimen browser on the left-hand side, and a linked Results of specimen browser on the right-hand side. Both browsers are synchronized with each other in both directions. 

Performance of the program is (just like is the case for many other programs) strongly influenced by unfinished work that remains untouched in the database. It is strongly recommended to check and clean-up old non-confirmed specimens and non-validated results on a regular base.

Main specimen browser

This is a single column browser showing the specimen Internal id by default. Using the cycle button of the column header, its contents can be changed. Available alternatives are: External id, Identification, Last location and Home location. The Identification possibility is only available when a specimen identification type is specified on the Specimen page of the options screen, as explained below. Unless pre-parameterized, all selected options are remembered in the user’s preferences.

Because of the numerous amount of options, they were spread over 4 pages.

Main page

Objective

Pull-down selection list with 4 choices: Review, Confirm, Validate, Confirm + Validate.  The Review option is intended for a kind of work status query, allowing to query for specimens which are not yet available, or for which there are no or lacking results.  The effect of the other options is obvious. They will cause the appropriate specimens to be selected, and the eligible results to be pre-selected for confirmation, validation or both.

Sort by

Allows to choose how the returned specimens should be sorted. Available options are: Internal id, External id, Identification, Object time, Receipt time, Last location, Home location or Storage location.

"Specimen" page

Internal id from… to…

Allows specifying a specimen internal id range. Suppose specimen internal id’s are composed of the date followed by some sequence number. In that case all specimens of September 2005 could be queried for by specifying "from 0509 to 0510".

External id from… to…

Allows specifying a specimen external id range. These fields are coupled with the "Station" field on the "Procedure" page. If no station is specified, they are always enabled. Else, two conditions should be satisfied in order for these fields to be active: the station should have NO "Identification type" and "Use external id" should be checked for that station.

Identification from… to… type…

Allows specifying a specimen identification range. If a specimen identification type is supplied, only identifications of that type will be considered.
This field is also coupled with the "Station" field on the "Procedure" page. If a station is specified, the type field is always disabled and set to the identification type as configured for that station. If the station does not have an identification type, then also the identification range fields will be disabled.

Material

Only specimens of the specified material will be returned.

Scope

Wildcard pattern which the Specimen.Scope field should match. A period (".") in a particular position indicates that any single character is acceptable in that position; an asterisk ("*") indicates that any group of characters is acceptable, including a null group of characters.

Sampled

Read here, this field is coupled with the Objective field. If the program was started for confirmation or validation, it is obvious that the specimen must be sampled.

Receipt time from… to…

Allows specifying a receipt time range. These fields are only enabled when the "Yes" option is selected for the "Available" field.

Executing department (group)

A specimen will be selected if:

Any of these departments:

  • Specimen -> SpecimenInput -> Action -> Procedure -> department.

Matches any of the following executing departments:

  • the specified "Executing Department".
  • OR any of the executing departments in the specified Executing department group.

OR if this is true for any of the sub-specimens.

Filter

General purpose MISPL filter allowing additional filtering in case the existing options would still be insufficient. Its usage is discouraged for performance reasons.

"Procedure" page

All options on this page: Station, Download group, Work place, Work list template and Work list template group are internally reduced to one single selection on Procedure. From the selected options, a list of procedures is deduced, for which corresponding actions should be found. In case of work list selection, any work list template item or include criterion is additionally taken into account.

"Result" page

Allows specification of result restrictions. If any of these options is supplied, at least one result assessed on the specimen should satisfy the criteria in order for the specimen to be included.

Classification

Only specimens with at least one result appearing in the specified classification will be returned.

Result status from… to…

The available statuses depend on the selected objective.  Validated results cannot be queried for as these represent 99.9 % of all results in the database. When trying to do so, a warning will be issued, unless this option is combined with one of the main options on the Specimen page.

Minimal urgency

Only specimens with at least one result having the specified urgency will be returned.

Minimal severity

Only specimens with at least one result having the specified severity will be returned.

Expert system status

At least one result should have the specified expert system status.

Note that if the expert system is set to Valab, the label will change into Valab status.
Possible choices are: Not evaluated, Evaluated, Approved and Rejected (the last option To be evaluated is not yet supported).

"Order" page

All options on this page allow imposing order restrictions. They are self-explaining and analogous to the order review program. As specimens may be re-used among different orders, it is the results that are verified against their corresponding order restrictions.

The currently supported options are: Issuer, Correspondent group, Agent and Department.

Detail result browser

This browser is very similar to the order review result browser. At the top, it shows specimen details in the information box. The default content of the information box is composed of 3 lines:

  1. Specimen information (internal id, identification, size, material, material variables, sampling time).
  2. Patient externalization.
  3. Specimen location information (last location, home location).

If this default is insufficient, it can be customized by creating a text module with the fixed mnemonic "SpecimenExternalization" on the Specimen table. In such a text module, the {DefaultExternalization} variable is available and will be replaced by the default externalization as described above.

Example

<html> <body> <font size=1.6 face=Verdana><b> {DefaultExternalization} </b> </font> </body> </html>

A toolbar on the right-hand side provides fast access to some commonly used functions:

  1. Result.Order.QuickReport()
  2. Result.Order.ViewImage()
  3. Result.Order()
  4. Result.Object.ResultOverview()
  5. Result.Specimen.PrintLabel()

The toolbar is hard-coded and hence cannot be customized. Due to space limitations, it only appears if the screen resolution is larger than 800 x 600.

Note

If any of the results displayed in the detail result browser is used as input for a formula, the calculated output(s) will automatically be added to the browser. If the calculation didn’t take place because not all inputs are confirmed yet, still a preview of the calculated results will be shown. In that case, the calculated value is shown in a normal font (instead of bold) and between braces ("()") to indicate that it concerns a provisional value.  On in-line or via F6 update of any result value, calculations are automatically updated.

Image results

If the option Show image results (see below) is enabled, available results of data type 'Image' automatically pop-up in one single frame. The frame automatically appears as images are available, and disappears if there are none. 

This is accomplished by means of an embedded Internet Explorer window. It can be resized as desired, in which case the images will automatically re-arrange. Images can be viewed full-size by clicking on the initial image. The latter may require the following Internet Explorer option to be set: Tools > Internet Options... Advanced > Security > Allow active content to run in files on My Computer

The HTML content is hard-coded and cannot be configured.

The images are collected and sorted as follows:

QUERY

Results directly assessed on the specimen (Specimen > SpecimenInput > Action > ResultOutput > Result).

SORTING

PropertyClassification sequence.

Options

For the moment, the options are identically to the order review result browser.

The property classification option is inherited from the parent specimen query, together with the Objective option. Results are always pre-selected according to the selected Objective and the button label varies between invisible, Confirm or Validate accordingly. No matter which Alternate classification is selected, only results belonging to the selection classification of the specimen query will be pre-selected. The Alternate classification only determines which results should be shown. Therefore, take care not to ‘hide’ any result, as this may cause the amount of non-validated results to increase unnoticed !

When the maximal result status is set to Discontinued, all discontinued results will be shown, but only iff they are linked to a corresponding action. This means that only actually measured results will be shown, and not the ones that were replaced because of a manual value or comments update. Any predecessors of that kind will still be shown in the information box in the bottom left corner.
