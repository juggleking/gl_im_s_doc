Order documents - Create CD

Description

GLIMS allows to scan order forms while creating or updating an order and store them as images.  For more information, please check the chapters Order/Patient documents - Twain scanning and Order documents - Electronic order entry.

In some countries (e.g. French private labs using FSE), it is required to regularly provide a CD that contains a copy of all originally scanned order forms.  

This chapter describes how GLIMS can prepare this CD.

License

This feature requires a separate license "CD Financial Export".

Preparing the image CD

Introduction

To gather all scanned order form images in a directory, the popup function Prepare FSE prescriptions CD on a set of financial shipments must be used.  In GLIMS version 8, this function must be added manually using the RunProcedure mechanism.

Select a set of financial shipments

  1. Click Start -> Billing -> Financial shipments -> Query.

  2. Select a date range (external date), status range (fixed-fixed).

  3. You can now either create a CD for a single destination or use a MISPL filter to create a CD for multiple destinations.

Storage of image file and index file

Images

  1. The image copy has the following file name: "order internal id" + "original extension" (independent of the Order.Image file name).  The order is retrieved as follows: Financial shipment -> Invoice summaries -> Invoice -> Order.

  2. The images are stored in the specified Target root directory and are grouped in the following sub-structure:

    [Lab nr/]  Destinataire / Year /Month /Day / LotNr

    Lab nr: lab identification code or mnemonic.  If several labs (shipment -> firm -> lab) are involved, additional subdirectories will be created. 

  3. If the destination has more than 2000 orders, a separate CD (sub directory and index.htm) is created directory name "LotNo" + "_CDn"where n=2".

    The maximum number of order images and CD size can optionally be configured in the lab (of firm) site attributes _FSE_MaxOrdersPerCD and _FSE_MaxCDSize (expressed in bytes).

  4. When the order image is missing, this is indicated in the index.html file (see below) and a warning is logged (LogType "FSE Prescriptions CD").

Index file

GLIMS also generates an overview in HTML format with the order details and links to the images.

  1. An index file is generated for each destination.

  2. The order details can be customized by specifying a text module (base table "Invoice") in the field "Detail text" (see above).

    Example of text expression 

            Numéro de lot: {=TRIM(.InvoiceSummary.FinancialTransaction.MostRecentShipment(?, ?, NO).ExternalId, "PX")}
                Numéro de facture: {=TRIM(.DocNo, "PXC") /* remove pyxbio extension */}
                Date d'exécution des actes: {=DateToString(.ExternalDate, "%d/%m/%Y")}
                Bénéficiaire: {=.Order.Object.Externalization }
                Numérod'immatriculationassuré: {=.PaymentAgreement.Matriculation}       

  3. An HTML preview is shown at the end of the process (only if a single CD is created) .
