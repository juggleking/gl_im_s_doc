Introduction

The urgency monitor shows a continuously updated browser with urgent orders which have not been completely processed yet.

The urgency monitor can be activated from the main menu (Start -> Routine -> Orders -> Urgency monitor).

Browser

The Urgency monitor browser looks as follows:

Depending on the order due time, the order is shown in a different color.

The browser can be dynamically sorted by clicking on the column headers. The user can perform any standard action on this browser (e.g. call screen report, examine order contents, ...) in the normal way.

To display columns with more data, such as object externalizations and receipt times, use the field chooser in the upper left corner of the browser.

The content of the information box under the browser can be customized by creating a (or modifying the) text-module with mnemonic OrderInfoSummary on the Order table.

Notes

  • Orders in the future are never shown in the urgency monitor.
  • The result column will not show results for a specimen that is not available.

Due time definition

In the urgency monitor, due time refers to different times, depending on the situation:

  1. If at least one result of the order is linked to a specimen:

    The due time of the order is calculated on the basis of the Receipt time (i.e AvailabilityTime) or Sampling time of the specimen. That is,

    • Order due time = Current time - Specimen receipt time

      OR

    • Order due time = Current time - Specimen sampling time

    The choice between the two options is to be made using the Due time start setting of the Urgent orders - query options screen.

    Note

    If the result creation time is more recent than the specimen receipt time or specimen sampling time, then the order due time is calculated based on the result creation time.

  2. If in the order two results are linked to a specimen and have different Receipt/Sampling times, then the oldest of the two Receipt/Sampling times is used to calculate the order due time.
  3. If no result of the order is linked to a specimen:
    • if the order's lowest object time lies after the order's creation time, the object time is used to calculate the order's due time.
    • if order's lowest object time lies before the order's creation time, the creation time is used to calculate the order's due time.

Query settings

The query settings can be accessed via the option icon in the upper left corner of the urgent order browser or via menu pre-configuration.

Minimal urgency

The urgency monitor allows the selection of orders starting from the lowest (default) urgency level. This can be important for small or night shift labs. Keep in mind that this option may yield a long query when used in normal routine in a large laboratory.

Maximal result status

Allows you to search orders with at least one result which is not yet available, for instance.

Issuer

Agent

Department

Allows you to search on the Order.Department field.
The order department is set as follows:
  • Manual order entry: department of the logged in user that created the order.
  • Electronic order entry: department of the batch user linked to the service that processes the order messages. This department can be overruled if a department is specified in the order message.

Executing department (group)

Only orders having at least one result executed in the specified executing department (group).

Classification

Only select orders having results for properties in the specified classification. For the selected orders, the urgency monitor will only show results for properties in that classification.

Waiting period

No longer used.

Due time start

Indicates whether the due time of the orders to be displayed should be calculated on the basis of the Receipt time or Sampling time of the specimen.

MISPL filter

Allows the definition of extra order-based selection criteria to exclude certain orders (e.g. object time, creation date etc).

Available specimens only

If enabled, the urgency monitor will only show orders that have at least 1 available specimen.

Display settings

To permanently change the settings, configure the menu option via a function parameter set. The urgency monitor will always re-use its last settings when not pre-configured via a menu or ribbon item.

Refresh after

Time in minutes before the browser is automatically refreshed (keep this as large as possible to avoid excessive use of system resources).

Color change after

Time in minutes before an order is displayed in the next color from the color list.

Color function

Order-based, integer-valued MISPL function. If specified, this function is used to determine the color number (between 1 and the number of colors), and the value of the 'Color change after' field is ignored.

Note

The Order.Urgency of the order passed to this function is the highest urgency of the unfinished results ! This way, it is possible that an order with original Urgency=3, but where all results with urgency 3 are completed, has Urgency=2 when evaluating it with MISPL (when a level 2 urgency result is still unfinished).

Color sequence

Colors of the color list (on TTY: use arrows up/down, on Windows: use mouse double-click or CTRL-F to change a color).
