Order log

Introduction

The Order context function Log > Order log gives an overview of all major events that occurred during the order's lifetime.

Note

The Order functions .DiscontinuationUser(), .ArchiveUser() and .ClosureUser() from the 'Log' sub-menu are redirected to the .OrderLog() function. It is advised to keep only one of them and update its menu label.

Information shown

It includes information from different sources:

  • Log entries for the Order

  • Log entries for the Object

  • Request information

  • Result audit trail

  • Specimen audit trail

  • Order.Log field

    The following events are stored in the Order.Log field. The creation user and time, the completion time, the closure user and time, the reopen user, time and justification, the archive user and time and finally the discontinuation user, time and reason.

Tip

When a site attribute with name _CreationTiming is defined on the Order table, GLIMS will log detailed timing information about each step of the order entry process.

If the site attribute is defined, you can find the timing information in the order log.
