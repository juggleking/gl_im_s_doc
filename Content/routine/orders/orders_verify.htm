Order verification

Contents: Introduction - Usage

Introduction

The order verification program is available from the main menu (Start > Routine > Orders > Verify entry) and as an item in the contextual menu / ribbon of each order. This program has been developed for clinical verification and therefore checks the explicitly requested properties, materials, microbiology procedures and pathology examinations and not the request definitions.

Note

Note that this program does not provide support for:

  • Issuer-specific request definitions.
  • The PanelMember.BillingMarks field (which allows specifying that the panel member should only be requested when the request has a billing mark which appears in the specified list).
  • Requests added via MISPL.

Usage

  1. When opening the order verification screen via the Order.Verify() contextual menu function, GLIMS allows processing the selected order(s).
  2. When opening the order verification screen via the main menu, the verification screen will be opened immediately, allowing the user to either enter/scan an order number or push the query button to fill in the query parameters for his selection.  The user's last settings for the query are saved and restored the next time the query is used.

In the top window, the order to be verified is displayed. 

Entering request definitions

Requests fields

The requests field allows to enter the requests which this order should contain.  This program also allows the use of "RequestByRange" (environment variable).

Request form

Allows using a request form to add/remove requests.  The default request form used is the one linked to the user's department.  If the department option Req. form is fixed is enabled, no other form can be selected.  If no department specific request form is defined, any form can be selected and the first found is used as default.

Also check urgencies

Allows checking the urgency.

Ignore materials

Allows ignoring the materials.

When hitting the OK button, the order will be verified.  If differences have been found, the following message will be displayed.

If you want to modify the order, click No.  The next screen will now clearly indicate (in red) the differences in the order. Request definitions marked with a "+" have been requested in this order, but were not entered during verification. Reques definitions marked with a "-" were entered during verification, but are not requested in this order.

Select the Correct button to zoom into the order and make you changes.

When an order is verified , this is now marked in the order by setting Order.Verified = YES.  When an order is marked as reviewed, this is added to the order log.  Note that the standard order query allows querying on Order.Verified.

Note

If the order to be verified contains nested panels and one of the members of a panel is discontinued, the Verify function may not notice that a panel member was discontinued.
