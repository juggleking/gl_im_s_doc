Order.GenerateDocument

Introduction

This function generates a document based on multilingual text expressions. It is mainly used to generate documents with incomplete (order/patient) data.

Fields

Header

References a MISPL text expression to be used as header of the generated document.

Body

References a MISPL text expression to be used as body text of the generated document.

Footer

References a MISPL text expression to be used as footer text of the generated document.

Page skip

By order

Skips pages by order

By issuer

Skips pages by issuer

Never

Allows generating a more general-purposed order listing document

Language type

The language of the document printout is adjustable to several language types: the language of the issuer, the object (patient), the user language and the site language. The site language is the default whenever a language of one instance is unknown.

Page height

Number of lines in one page

Number of copies

Printer

Preview

Output to file

Calculate budgets

Determines if budget calculation must be triggered by the document generation. Note that budget calculation does not automatically include printing of the budget calculation results on the document.
