Order.Purge

Introduction

The Order.Purge() contextual menu function allows complete deletion of an order, including billing data. Only orders with status Discontinued or Closed can be purged with this function. Orders in other statuses can be purged using the Purge old routine data program.

Options

Maximal object date

Offers an extra protection to avoid purging recent orders. Orders with an object time that is more recent than the maximal object time cannot be purged.

Note

If the order's lowest object time differs less than 2 days from the specified object date, the order will not be purged and a warning message "Object date too recent" will be shown.

Delete results

If not set, results will not be deleted. Note that these results will no longer refer to an order. They will still refer to the patient which allows to see them in result history overviews.

Delete invoices

If set, the order's invoices will also be deleted. Invoiced orders can not be deleted without enabling the option Delete invoices. The invoice summaries and payments are not deleted (unlike the Purge routine data program). Instead, the invoice summaries will be demoted to status Purged (and keep the amount).

Delete log entries

If enabled, the log entries linked to the purged data will also be deleted (i.e. log entries for Order, Order set, Invoice, Request, Action, Blood Selection, Result, Isolation, Carrier, Microbiology action and Pathology exam.
