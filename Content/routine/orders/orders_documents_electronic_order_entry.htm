Order documents - Electronic order entry

In case of electronic order communication (e.g. Mediaform), it is possible that the scanned order document is also sent to GLIMS.  This image file will be stored in the Order.Image field.  This image can be viewed with the Order.ViewImage function which is available from the Order contextual menu via Documents -> View order document.

GLIMS also allows to prepare a CD that contains a copy of all originally scanned order forms.  

Note

You can also use the Order.SetImage() MISPL function to specify a value for the Order.Image field.

Syntax

Logical .SetImage(String FileName)

This MISPL function sets the Order.Image field for communication with an external document imaging system. Allows specifying e.g. an URL based on the order internal id during the post commit phase of an order.  This eliminates the need for misuse of dummy properties. 

Note

The post-commit MISPL expression is also executed after each order update.  If the Order.SetImage function is used in this context, it may be run for each order update.  This will probably not cause any problems (unless a date is used in the image file name), but can be avoided by checking if the order image has already been set (e.g. IF .Image <> ? THEN .SetImage() )

The image file name is visualized and can be updated in the order image view screen.  Additionally, this order image view screen supports the standard order contextual menu.
