Basic routine -> Orders -> Issuer combinations

Issuer combinations can be used to establish an automatic link between an issuer correspondent and an agent correspondent.  This link will be used during order entry.

Starting from the issuer correspondent, the issuer combination can be set via the contextual menu function Correspondent -> Issuer -> Issuer combinations -> As issuer.

If the chosen issuer only has one IssuerCombination with a specified IssuerCombination.Agent member, it is used to automatically fill in the Order.Agent member.  If multiple choices exist, a popup browser is displayed allowing the user to pick the combination valid for the current order.

The issuer combinations of the issuer are also checked after scanning an OMR sheet in the interactive order entry program, just as when the issuer would have been typed in manually.

Agent

This field contains the agent which will be automatically linked to the issuer during order entry.

External code

This allows for the printing of an external code on bills or reports for orders entered using this issuer combination.
