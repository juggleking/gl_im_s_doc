Order entry - Executing department imposed by external system

Introduction

GLIMS offers support for external systems that impose the department in which requests should be executed.

Electronic order communication

A driver named "J_ORB_OFPOM" is available that allows to send an executing department for each request code in the order import message.  The executing department is specified in the field "MSH.6".

When a department is specified in the order import message, but GLIMS cannot find a matching procedure linked to that department, the request will be created but will not be scheduled.  An error message will be returned to the external system.

GLIMS will try and schedule all requests, even if this failed for some requests in the order.

Action scheduling

The action scheduling mechanism supports that an external system imposes the department is which a request should be executed.  This influences the procedure to be selected during electronic order entry.

When adding a request to an order, the electronic order import program can specify the department where it must be scheduled. The department where the request should be executed, is stored in the request record (field "Department to execute").

If a department has been specified for the request, then the action that delivers that request as output MUST be an action for a procedure defined in that department.  If no such procedure can be found, the request will not be scheduled.

If multiple suitable procedures can be found in the specified department, the existing action scheduling rules will be applied to choose the most appropriate procedure amongst those.

Note

The specified department is not copied further onto the requests (results/specimens) that are needed as input for the procedure. So these “induced” requests (inputs for the action) will be scheduled in the normal way, without restriction to the department.

The specified department is only applied for the last action, the action that directly delivers the request.
