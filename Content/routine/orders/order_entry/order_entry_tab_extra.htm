Order entry: Extra

Due time

Shows the date and time on which the order is due.  Check the chapter Urgencies and due times for more info.

Context

Three order context fields allow indicating why the order was requested (Sickness, Sickness local mode, Maternity, Industrial accident, Prevention of disease, Accident, Child birth tests on child, Child birth tests on mother).

You can also supply a context date and info. In case of 'Industrial accident', the corresponding Context date can be considered as the accident date and the Context info allows entering the accident code. In case of Maternity, the context datum will be the delivery date. Can be used in France for specifying the 'Nature d'assurance'.

Phone results

If set, the results which meet the following conditions are marked with Result.PhoneStatus = YES:

  • Result.PhoneStatus = YES
  • Request.Unsolicited = NO
  • Result.Request.Explicit = YES (i.e. the result is explicitly requested in the order entry program)
  • Result.Property.Visible = YES
  • Result.Property.ReportFormat <> "Hidden"
  • There is at least one property classification node for this property

See the documentation on the phone list program for more details.

Include in budget

Allows indicating that this order is not to be taken into account for the issuer budget calculations.  So far, it is only possible to specify this field, either via the order entry screen or via the electronic order import using the URL protocol.

Exon. Code

Used in France only.  Please also check the chapter Carte Vitale.

Order label 1/2 printed

These fields will automatically be enabled when the order labels have been printed to avoid that these order labels are reprinted upon an order update. The order entry option "Check if printed" for order labels determines whether GLIMS should check if order labels have been printed before.

Order identifiers

The integrated browser that is available if custom order identifier types are defined allows the user to specify the order identifiers for these order identifier types.

The External id and Work id fields are available in the Order tab page of the order entry screen.

Note

For more detailed information on the other fields, see the documentation on the Order table in the reference guide.
