Basic routine -> Orders -> Order entry: Reports

This page allows inserting manual reports (i.e. reports that are not scheduled by default). Other possibilities are explicit specification of the report copy count, a specific fax number for fax reports or the report target, or even the deletion of unwanted manual reports.

It is also possible to specify only a report target (without entering a report code). This will generate all the reports that would be generated if this target were the issuer. This is exactly the same as specifying the target in the Order.ReportCopyReceiver field in the 'Order' page.

The option Show all allows the user to choose to see either all the reports (including the specific reports) or only the specific reports.

When the Manual marker in the report column is red, this indicates that the report is not yet stored on disk.

When the report target is shown in gray, this means that this is a 'calculated target' which may change when e.g. the issuer changes, while a black color indicates a fixed report target, which will not be changed when the order issuer changes.

Remarks

Just like in the Order.AddReport() function, a copy report with an explicit target (receiver) added manually via this page will use the default report settings (when specified) of the report target instead of the default reports of the order issuer.  Also, during the evaluation of the DefaultReport.Eligible MISPL function, the order issuer will be set to the report target instead of the original issuer (just as in the AddReport() function or as for the Order.ReportCopyReceiver).  This makes it possible to take the report receiver preferences into account when scheduling a manual report copy for this receiver.  A typical example would be to select the proper format for an electronic report copy.
