Urgencies and due times

Introduction

When entering an order in GLIMS, you can set the urgency level of the requests.  The available urgency levels can be defined in GLIMS.  Moreover, you can link a due time to each urgency level, i.e. when should an action with a given urgency be completed.

Urgencies

Click Start -> System management -> Scheme -> Site domains:

  • Enter a site domain with the name Urgency.
  • The data type will be enumerated.
  • Enter the different urgency level in the List of values field. Make sure that you indicate the short name of the urgency level (see figure above).

Due times

Due times can be specified in the Specific site preference set.  Click Start -> System management -> Customize -> GLIMS:

In the More tab page you can enter the times in the field Due Times (see figure above). Each time indication corresponds to an urgency level.

There are two possibilities:

1. You specify a time lapse preceded by a + sign, (see figure above).

Urgency level 1        48hours
Urgency level 2        4 hours
Etc.

The due time will then be calculated by adding this time lapse to the object time.

2. You enter a fixed time. (e.g. 08:30).

Note

The due time of an order will be the highest due time of all requests within that same order.

Due time calculation in chained procedures + turnaround times

If you have specified a due time for an urgency, this will either be added to the object time of the order (time lapse) or will be set as the due time of the request (fixed time). Note that the turnaround time of the procedure is also taken into account when determining the due time of an action.

Example

You have two urgency levels: "Normal" and "Urgent".  In the due time configuration, you have specified "2,1".  This implies that normal request should be ready within 2 days, urgent requests within 1 day.

Assume the following situation:

  • You now enter a test "X" on 13/03/2006 at 15h00 with urgency level "Normal"
  • To measure test "X" you have defined a sampling procedure, a centrifuge procedure and an analysis procedure
  • The analysis procedure has a turnaround time of 1 day, the centrifuge procedure 6 hours (0.25 days)

Due time are then calculated as follows:

  • Due time of the analysis action: 13/03/06 15:00 + 2 days (Due time) - 1 day (turnaround time) = 14/03/06 15:00
  • Due time of the centrifuge action: 14/03/06 15:00 - 0.25 days (Due time) = 14/03/06 09:00
  • Due time of the sampling action: 14/03/06 09:00
