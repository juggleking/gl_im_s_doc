Consult registration (former features)

Content: Introduction - Configuration - Usage

Introduction

GLIMS allows for the registration of "consult" orders.  A consult request indicates that a lab biologist has conferred with the patient's health care provider.  This consult request may be charged just like any other.  Note that the consult registration program requires a separate license.

Tip

In order to register a consult in more detail, you can use the Consult registration module.

Configuration

The only configuration needed to register a consult order is the definition of consult request codes.  These request codes can e.g. be used to distinguish between a "short" and a "long" consult, which may be subject to different tariffs.  The consult request codes must be property request codes and can be configured by using the "Options" button of the consult registration (see below).

Usage

Access to the registration screen

The consult registration screen is accessible in 3 different ways:

Via the result phone screen

When phoning through results, the left-most button (with the facing faces) allows to register a consult. When clicking this button, the selected results will be phoned through and a phone log will be added.  Then the consult registration screen will be opened. The comment specified in the result phone screen will be copied to the Comment field of the consult registration field where you can adapt it if needed. Note: when the option Create phone log is enabled in the consult registration screen, a second phone log will be created (containing the comment specified in the consult registration screen).

Via the main menu

Click Start > Routine > Orders > Register consult.

contextual menu function on Object / Person

The consult registration screen is also available as contextual menu function from an object or person.

General functionality of the registration screen

The consult registration screen allows to add a consult request to an existing order or create a new order with a consult request.

Patient

When opening the consult registration program via the result phone screen or via the object / person contextual menu, the patient information will already be specified.  When opening the consult registration via the main menu, you will have to specify for which patient you want to register a consult.

Order

When opening the consult registration via the result phone screen, the original order will be reused, unless when the order was closed. In all other cases, a new order is created.  Issuer and Comment are mandatory.

The consult registration screen allows to choose between different consult request codes. These codes can e.g. be used to distinguish between a "short" and a "long" consult, which may be subject to different tariffs.  The consult request codes must be property request codes and can be configured by using the Options button, and are remembered via the user's preferences. You must specify a comma-separated list of request code mnemonics.

The mandatory comment allows entering some information regarding the consult and is stored as the result value of the consult request.

A phone log will always be created containing the comment specified in the consult registration screen.

Multiple consult requests can be added to the same order, but they must have a different object time (in order to avoid problems with electronic reporting).
