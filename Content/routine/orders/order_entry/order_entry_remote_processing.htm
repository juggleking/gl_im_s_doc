Remote order processing

Introduction

It is possible to have GLIMS execute some phases of the order processing remotely on the server instead of locally on the client PC. This implies that the data needed to execute a task is not transferred from the server to the client and back, but that the task is executed on the server itself. This may increase performance and makes it possible to create a new order without having to wait for order completion. However, in case of many clients using the remote processing mechanism simultaneously, this may also cause a delay as the service responsible for the remote execution on the server can only handle 1 task at the time.

Order entry options

The order entry options Action scheduling mode / Postscheduling mode / Report scheduling mode allow the user to determine for each phase of the order post-processing if it should be executed locally or on the server.

GLIMSRemoteCalls

The GLIMSRemoteCalls environment variable needs to be set to YES. If GLIMSRemoteCalls is set to NO, then the action scheduling, post scheduling and report scheduling will be executed locally, regardless of the Action scheduling mode, Postscheduling mode and Report scheduling mode.

Note

In order to use GLIMSRemoteCalls you must also set the environment variables "MA_Service" and "MA_Host".

Order entry evaluation functions

The remote order processing mechanism also has an influence on the implementation of the order entry option Evaluation after scheduling.
