General

Issuer

When an identification provider is known, it is possible to search the issuer via his identification.

At order entry, a number of default issuers are proposed by GLIMS:

  1. When entering the encounter external id, the physician of the encounter will be the default issuer.
  2. If not available, the family doctor of the person linked to the encounter will be the default issuer.

  3. If not available, GLIMS will check if there already exists an order for this person. If so, the issuer of the person's last order will be the default issuer.

    Note

    When selecting a patient for which a family doctor is specified (in the person configuration), an extra button will be displayed which allows specifying this family doctor as the order's issuer. When no family doctor is defined for the patient or when the patient's family doctor is already specified as issuer, the button is hidden.

However, it is also possible to specify the environment variable WardIsDefaultIssuer. If set to Yes, the ward linked to the stay of the encounter (or the ward that was manually entered) will be used as default issuer.

Note

When an order form is scanned via the interactive OMR program, and no issuer was detected on the form, the above mentioned default issuers (2) and (3) are not applied.

Note that the general correspondent lookup mechanism is also applied here.

Agent

When an identification provider is known, it is possible to search the agent via his identification.

Default agent: by default, the environment variable WardIsDefaultAgent is set to Yes. Therefore, the agent will always be the ward, unless you set WardIsDefaultAgent to No.

Note

The general correspondent lookup mechanism is also applied here.

Receipt time / Prescription time

Sampling location

The sampling location will be used in the billing module to determine whether a supplement is to be charged for the sampling process. A default value can be set in the order entry options.

Central

The sample has been taken in the same location of the laboratory.

Decentral

The sample has been taken in another location.

Object site

The sample has been taken in the patient's home.

Specific

If set to Specific, the sampling address is to be specified.

Series

This is actually the same as Specific, but indicates that the rest of the orders created in the same order entry session will also have the same specific sampling address. The value of Order.SamplingLocation after committing the order will automatically be changed from Series to Specific. This implies that the actual intention of this value is to facilitate the creation of a number of orders for the same specific sampling address.

To allow the specification and lookup of German HC provider identification numbers, the LANRs and BSNRs, the Identification and HCP code fields of Issuers and Agents can be replaced by an LANR and a BSNR field.

To this effect,

  • the field Use LANR and BSNR as issuer identification source in the order entry options should be enabled, and

  • two fields should be filled in under System management -> Customize -> GLIMS General -> Identification:
    • Provider for LANR: this field must be filled in in order for the LANR and BSNR fields to be visible in order entry and correspondent search screens.

    • Provider for BSNR: this field must be filled in to make the specification and lookup of BSNRs possible in order entry and correspondent search screens. The provider indicated in this field is the provider of BSNRs for both executing departments (and thus for executing HC providers) and requesting HC providers, i.e. issuers.

      Note

      • The provider indicated in Provider for BSNR need not be identical to the one indicated in HC code provider, which is only used for fund related identifications.

      • When entering an order study, if the order's patient and the site's default PIN provider have not been indicated,

        • the Identification field appears instead of the LANR and BSNR fields.

        • double-clicking in this field allows you to search for identifications. The selectable identifications are those provided by the study correspondent, study issuer or the study responsible.

        • In the Correspondent search screen the LANR identification field is still displayed.

The searched and displayed LANRs and BSNRs are those that are valid on the date of the Order lowest object time, if this date is known. If the order lowest object time is not known, the searched and displayed LANRs and BSNRs are the codes that are valid on the day the functionality is being used.

Automatic fill-in of the Issuer field when using Ticket books

Concept

Certain German labs distribute ticket books to their issuers. These ticket books contain labels to be stuck on paper orders. Each label carries a barcode belonging to a certain number range, called ticket book range.

Example

Example

Ticket book A contains 500 labels numbered from 0001 to 0500. The ticket book range of ticket book A is 0001-0500.

Ticket book B contains 500 labels numbered from 0501 to 1000. The ticket book range of ticket book B is 0501-1000.

Each issuer receives a ticket book. Therefore, a specific ticket book range corresponds to a specific issuer. By looking in which number range a barcode is situated, it is therefore possible to identify the issuer of the order and thus to fill in the Issuer field of the Order entry screen automatically.

Configuration

Ticket book ranges and issuers must be associated in the database. This is to be done in the Ticket book range editor, which can be open in two ways:

  • From the Ticket book range browser: Start -> Ticket book ranges -> press Insert key.
  • From the Correspondent browser, select a specific correspondent record, right-click Show all -> Ticket books, and press the Insert key.

First number - Last number

First and last number of the ticket book range.

Valid from - Valid until

The Valid from field is mandatory, the Valid until field is not.

The validity period is checked against the Object date. If the Object date is not yet known, the date of the day is used.

Once the validity period has expired, the ticket book is not valid anymore and the number range can be reused for another ticket book.

Routine

The barcodes on the ticket book labels are directly related to the order short IDs.

When

  • The barcode on the ticket book label:
    • Is entered manually in GLIMS, or
    • Is scanned in the Specimen reception scan program.
  • An electronic message containing an order short ID is received,

the logic of the barcode parser determines the order short ID and identifies the issuer of the order. The issuer is automatically added to the Issuer field of the Order entry screen.

Note

The issuer information is not updated when the Object date or Ticket book range is modified.
