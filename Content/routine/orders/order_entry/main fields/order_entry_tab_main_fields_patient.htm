Order entry: Main fields - Patient info

These fields will only be shown when an order is created for an object of type "Person".

Encounter

The encounter selection mechanism used depends on the order entry option Encounter selection.

Encounter search in local GLIMS database

When entering an encounter number, GLIMS will look for any matching encounter in the default institution for encounters (if specified in the user preferences).

Encounter search in external system (e.g. HIS)

If the identification source specified in the Default identification source field of the user preferences refers to a translator (which is active), a Query button will be shown which allows looking up the encounter in an external system.  The translator specified will handle the query to the external system.  If no identification source is specified in the user preferences, GLIMS will check if a person identification source is specified on SpecificSite level (Start -> System management -> Customize -> GLIMS general).

The Identification source access in the main page of the order entry options screen will determine if the user can see and/or access the source used for the external query.

When one or more patients are found in the external system, GLIMS will show a selection screen.  For more information about this screen, please check the chapter "Querying persons in an external system."

Note

  • If both the PIN and encounter are specified, two query buttons will be displayed that allow to query the patient either by PIN or by Encounter.
  • F8 will launch a query by PIN if the field "PIN" has focus. 
  • F8 will launch a query by encounter if the field "Encounter" has focus.

Encounter external id

The encounter's Ext id field matches the patient's admission number in the hospital. An external id query can be launched in the external application pressing function key F8 (same as Pin query). If the external application finds a match, it returns detailed patient demographic data in a separate window. In case of communication with an external hospital system the encounter's external id is assigned automatically. For private laboratories it is entered manually by the laboratory technician.

Encounter type

Specifies the type of encounter (an outpatient clinic, for example).  Don't confuse it with the residence type, which is an enumerated value ambulant or hospitalized.

The field 'Ward.DefaultEncounterType' allows setting the default value for the Encounter Type field of the order entry program and is set as soon as this ward is chosen in the 'Ward' field.  

Institution

Can have a default value set in the user preferences.

Ward

Specifies the location in the hospital where the patient stays. When clearing the ward, it is assumed that the stay is to be closed. The stay end time will be set to the lowest object time of the order - 1 minute.

Room

References the number of the room where the patient stays.

Info

Provides external information about the encounter (for example inpatient, outpatient, external, ...).

Name

The name field consists of several sub fields: last name, first name, middle name and title.

Name search in local GLIMS database

GLIMS offers two main local name search methods.  The applicable search method depends on a setting in the user preferences:

  • If set to Default and you type ctrl-f, GLIMS will show a standard person browser from which you can select the patient.
  • If set to "Via search screen", an advanced search screen will be displayed when typing in one of the patient fields.  You can disable the person search screens in order entry by using the order entry option "Suppress automatic popup of person search screen".  If enabled, GLIMS will not use the person search screen in order entry, regardless of the user setting.

    Click here for a full description of this special search screen.

Name search in external system (e.g. HIS)

If the identification source specified in the "Default identification source" field of the user preferences refers to a translator (which is active), a "Name query" will be shown which allows looking up the name in an external system. The translator specified will handle the query to the external system.  If no default identification source is specified in the user preferences, GLIMS will check if a person identification source is specified on SpecificSite level (Start -> System management -> Customize -> GLIMS general).

When one or more patients are found in the external system, GLIMS will show a selection screen.  For more information about this screen, please check the chapter "Querying persons in an external system."

Search via eID

GLIMS allows using the Belgian electronic identity card (eID) for patient selection during order entry.

Birth date

Allows selecting a person by its birth date.  Identical to the name fields, you can use the advanced search screen when typing ctrl-f.  Click here for a full description of this special search screen.

Note

When typing in a birth date and selecting the person's sex, GLIMS will create a person even if the last name / first name is unknown.  You can still lookup an existing person by using "CTRL-F" our double-clicking in the "Birth date" field.

When updating the newly created person (e.g. in order entry) by specifying a "Last name", the "Person comparator" MISPL function will be evaluated to see if a matching person can be found.  If so, a confirmation will be asked to update the existing person record.

Age

Person's age at sampling time.

Sex

Person's sex

Spouse

Select the patient by spouse name, which is used to enable separate storage of maiden names (in the Name member) and spouse names for female patients.  Identical to the other name fields, you can use the advanced search screen when typing ctrl-f.  Click here for a full description of this special search screen.
