Basic routine -> Orders -> Error handling during order entry

This chapter describes the error handling when encountering problems during the scheduling phase of order entry process. When a specimen could not properly be created (e.g. because of a duplicate internal id), a proper error message is shown and the program takes this error into account during the rest of the scheduling.

Additionally, error generated during the order entry phase are also logged in the log tables of the database, using a log type of "Order processing". This log type is created automatically. A log record is opened and closed on a monthly basis. The created log entry records refer to the order which produced the result. This implies that the error records of an order can be found via the "View->Log entries" menu option.

The fact that the errors are stored in the database can be especially helpful when the scheduling is performed on a remote task server. In this case, the errors can now also be verified from a client PC.
