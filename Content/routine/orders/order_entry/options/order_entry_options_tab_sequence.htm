Order entry options: Sequence

Introduction

The Sequence tab of the Order entry options screen displays a browser containing all fields of the order entry screen.

Sequence: Ins - Upd

With the Ins and Upd columns, you can indicate in which order the fields of the order entry screen should be visited each time the user clicks the OK button or presses the GO function key.

By entering a sequence number next to the appropriate fields in the browser, you indicate that upon each GO event, the program should position itself on the next field. The sequence can be defined separately for order creation (Ins) and order updating (Upd). After the last field in the list has been visited, the order is stored. 

Note

By defining an environment variable MA_NextPageKey (e.g. MA_NextPageKey=F9) in .glimsrc or progress.ini, you can choose an explicit key to jump from field to field according to the sequence specified in the browser. Currently, only F8, F9, F11, F12 and SHIFT-F7 are supported as "Next Page" key.

  • When such a key is defined, pressing it will bring the user to the next non-visited field from the preconfigured list. Also, the F2 key (or Do key for TTY) or selecting the OK button will then immediately save the current screen without going to the next field.
  • When such a key is not defined, the F2 key, the OK button or the Do key (TTY) keep their behavior as before.

Note

To define the sequence for the FSE field "Nature d'assurance", you should use the field "OrderContextString".

Emphasis: Emph Ins - Emph Upd

The Emph Ins and Emph Upd columns allow you to indicate whether the field should be emphasized during order creation (Ins) or order update (Upd).

An emphasized field is shown with a different background color.
