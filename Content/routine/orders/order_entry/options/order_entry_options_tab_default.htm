Order entry options: Default

Object type

Enables to specify a default object type to be proposed each time an order is created. Possible values are person, animal or lot. In a clinical environment, this will mostly be person.

Issuer type

Enables to specify a default issuer type, as for instance HC Provider, Lab, Organization, etc. The default issuer type is only set automatically when creating a new order or editing an existing order without issuer.

Agent type

Enables to specify a default agent type to be proposed each time an order is created.

Sampling location

Is used to set the default sampling location. The sampling location will be used in the billing module to determine whether a supplement is to be charged for the sampling process.

Central

The sample has been taken in the same location of the laboratory.

Decentral

The sample has been taken in another location.

Object site

The sample has been taken in the patient's home.

Specific

If set to 'Specific', the sampling address is to be specified in the field Sampling address.

Series

This is actually the same as Specific, but indicates that the rest of the orders created in the same order entry session will also have the same specific sampling address. After committing the order, the Series value will automatically be changed to Specific. The intention of the Series value is to facilitate the creation of a number of orders for the same specific sampling address.

Sampling address

When the option Sampling location is set to Specific or Series, you can also specify a default sampling address.

Issuer

Ward

Previous

The same issuer as for the previous order (of the same person)

Family doctor

Attending physician

None

Dynamic

  • For inpatients (= encounter of type hospitalized): use attending physician.
  • For outpatients (= encounter of type ambulatory), use attending physician, or if unknown, previous issuer.
  • For external patients (= no encounter): use family doctor.

Specialism

The Specialism of the stay will be copied to the Issuer field of the order. This allows sending a report or a bill to the specialism, or classify medical costs by specialism in statistics programs.

Fixed

The issuer will remain the same until changed manually. When inserting an order for the first time in an order entry session, the issuer is empty. Once an issuer is chosen, it will remain active during that order entry session until changed explicitly by the user. This option can be helpful when entering a series of order for the same issuer. Note that the chosen issuer is not remembered in the user's preferences.

Agent

None

No default value.

Ward

The default agent sub type will also be set to ward.

Specialism

The Specialism field of the stay will be copied to the Agent field of the order. This allows sending a report or a bill to the specialism, or classify medical costs by specialism in statistics programs.

Fixed

The agent will remain the same until changed manually. When inserting an order for the first time in an order entry session, the agent is empty. Once an agent is chosen, it will remain active during that order entry session until changed explicitly by the user. This option can be helpful when entering a series of order for the same agent. Note that the chosen agent is not remembered in the user's preferences.

Note

The initial values in the order entry options will be filled according to the environment variables WardIsDefaultIssuer and WardIsDefaultAgent defined in progress.ini. Afterwards, these environment variables will be ignored.

Study

None

Do not use a default study.

Previous

When creating a new order for an object, the program will check if there is already an order for that object + department. If so and the 'Study' field is specified, its value will be used in the new order.

Fixed

The study will remain the same until changed manually. When inserting an order for the first time in an order entry session, the study is empty. Once a study is chosen, it will remain active during that order entry session until changed explicitly by the user. This option can be helpful when entering a series of order for the same study. Note that the chosen study is not remembered in the user's preferences.

Institution

Allows to set an initial value for the institution of the encounter of a new order. Note that setting 'Default institution' also indicates that you want to choose or create an encounter for each order, since the program will try to create an encounter if necessary in order to save the selected institution. If not specified, Glims will use the default institution specified in the user's preferences.

Department / Department access

It is possible to specify the department when filling out an order (in the order page of the order entry screen). These fields allow to set the default department and the department access mode. The following accesses are possible:

Hidden

The department is hidden.

Read-only

The default department is shown but can't be changed.

Update

The department can be changed in order entry.

Urgency

Allows defining the default urgency which will be used for entering new requests.
