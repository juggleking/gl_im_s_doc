Introduction

In some organizations, the order entry process is divided into two phases. First all orders are created with the requested tests. In a second phase, all patient data is added to the created orders. GLIMS offers two features that allow easy update of the created orders.

Via the "Order update" program

The order entry program can be called from the main menu in Update mode (Start -> Routine -> Orders -> Update).

In this mode, the order update program starts by asking an internal id via the find interface. When entering a valid order number, the order is shown in the order editor. When leaving the order editor, a new number is asked, an so on.

Internal id

Specify the internal id of the order you wish to update

Aut. increment

Each time the screen pops up, the last number is increased and proposed as default number. When processing several orders with increasing numbers, this may be helpful.

Prefix

A manually entered prefix will be used to look up the order.

Use default prefix

If enabled, the program searches for the existence of a gp_Site-based, string-valued site function with the name OrderInternalIdPrefix. When such a site function is found, it is executed in order to calculate the prefix in function of the current user, the current date, etc...

Note

When called from the order browser (see below), the order find screen inherits the prefix settings from the browser, on condition that the Use prefix as positioner order browser option has been set.

Via the environment variable "MA_ExitEditorViaFind"

The same functionality can be obtained by using the environment variable "MA_ExitEditorViaFind". In theory, this environment variable can be used for any browser and its corresponding editor. However, this variable has been designed for usage in order entry.

Example 

MA_ExitEditorViaFind=ed_ord:br_ord6
  • Ed_ord is the name of the order entry editor.
  • Br_ord6 is the name of the OrdersByInternalId browser (Br_ord8 is the name of the OrdersByShortId browser)

Open the OrdersByInternalId browser and zoom into a record. When hitting the OK button, in the order entry screen, the Find Order box (as documented above) will be displayed, allowing to select a new order from the OrdersByInternalId browser. This way, the editor is never closed and does not have to be opened each time you select a new record from the browser.
