Orders - Change object

Content: Introduction - Change object - Resend electronic reports - Changing the object of multiple orders at once

Introduction

The Order-based function Change object can help to solve issues with duplicate patients.

Change object

Select an order and choose the contextual menu / ribbon item Change object.

Update its identity

The object editor will be started, in which you can update the identity of the current person, animal, lot...

Replace by another object

This disconnects the object from the order and starts the order entry program where you can assign a new object.

Replace by and merge with another person

This option will only be available if the object is of type Person and the user has access rights to the Person.Merge function.  The function allows replacing a temporary patient by its final version.

The Replace by another object option has the disadvantage that the temporary patient is only disconnected, and remains in the database where it serves of no more purpose and only pollutes the database.

With this third option, the person search screen appears, from which you can select the right patient. On selection, the original (temporary) patient is merged with the selected one, and hence disappears from the database. So, you end up with the order linked to the right patient and the temporary patient is gone.

Note

  • The Replace by and merge with another person option is typically used to replace a temporary version of a person record by its final one. Most often, the temporary version was created manually because the registration by the HIS wasn't received yet at the time the order was created. This implies that no encounter external id was available either.

    Therefore it is logical to re-determine the encounter for that order. So, after assigning the new patient to the order, the encounter will be determined as if the order was originally entered.

    After that, the order entry program appears. When pressing OK, the updated encounter information is stored. When pressing cancel, it is undone.

  • This action is logged and can be viewed for both the Order and the Person via the Log entries.

Resend electronic reports

When changing the object of an order (e.g. address change via Order entry screen), GLIMS will examine the object's orders of the last month and resend any electronic reports for these orders.

When editing the object of an order

All orders of the object of the last month will be examined in order to determine if any electronic reports are defined for these orders.

If this is the case then these electronic reports will be marked with Needs checking if they have been sent before. This will cause the reports to be resent.

When changing the object of an order

This will cause all reports for the specific order to be checked for electronic reports which have been sent before. If any electronic reports are found then these reports will be resent.

When merging persons / objects

All orders of the last month of both the bad and good object will be examined in order to determine if any electronic reports are defined for these orders.

If this is the case then these electronic reports will be marked with Needs checking if they have been sent before. This will cause the reports to be resent.

Tip

If certain electronic reports should not be marked as Needs checking when an order's object or encounter is updated, activate the option Only result changes in their templates.

Changing the object of multiple orders at once

This function also works when selecting multiple orders, to allow changing the object for multiple orders of the same patient at the same time. The following remarks apply in this case:

  • Only multiple orders of the same object are accepted 
  • Since the Replace by and merge with another person option already moves all the orders of the bad patient to the good patient, selecting multiple orders makes no sense if you intend to use this merge option. 

  • When choosing the Replace by another object option, the selected orders will be disconnected from the original object and shown in the order entry screen one after the other, in order to allow choosing the correct patient.

    From the second order on, the patient selected for the first order will already be filled in, so in many cases the order entry screen for these orders an simply be closed. The order entry screen is still opened for the second and subsequent orders to allow choosing the correct encounter.
