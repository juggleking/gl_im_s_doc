Default reports: use cases

Common setup

  1. Issuers H1_A and H1_B belong to Issuer Group "Hospital 1" and do not share a common health care provider practice.
  2. Issuers H2_A and H2_B belong to Issuer Group "Hospital 2" and share a common health care provider practice.
  3. A patient is treated in "Hospital 1" first by Issuer H1_A and afterwards by Issuer H1_B.
  4. The same patient is treated in "Hospital 2" first by Issuer H2_A and afterwards by Issuer H2_B.

Case 1

Report scheduling

Two reports will be scheduled for the object.  One for Issuer H1_ A and one for Issuer H1_B. Both reports will be marked with "Needs checking", which implies evaluation during report generation.

Report generation

Both reports will contain the same orders and results, i.e. those requested by Issuers H1_A and H1_B.

Case 2

Report scheduling

Two reports will be scheduled for the object.  One for Issuer H1_ A and one for Issuer H1_B.  Only the report of the order that was last issued (issuer H1_B) will be marked with "Needs checking". In this case, no report will be sent to issuer H1_A only to issuer H1_B.

Note

If the order of Issuer H1_A is updated afterwards, the report scheduling mechanism will mark the report with target H1_A with Needs checking and the report with target H1_B will no longer be marked with "Needs checking".

Report generation

The report will contain the orders and results requested by Issuers H1_A and H1_B.

Case 3

Report scheduling

Two reports will be scheduled for the object.  One for Issuer H1_A and one for Issuer H1_B. Both reports will be marked with "Needs checking", which implies evaluation during report generation.

Report generation

The report targeted to issuer H1_A only contains the orders and results requested by issuer H1_A.

The report targeted to issuer H1_B only contains the orders and results requested by issuer H1_B.

Case 4

Report scheduling

A report will be scheduled for the order's issuer.

Report generation

The report will contain all orders and results requested by issuers belonging to issuer group "Hospital 1".

Case 5

Report scheduling

A report will be scheduled for the order's issuer.

Report generation

The report will only contain orders and results requested by the order's issuer.

Case 6

Report scheduling

A report will be scheduled for the order's issuer.

Report generation

The report will only contain orders and results requested by the issuers H2_A and H2_B as they share a common health care provider practice.

Case 7

Report scheduling

A report will be scheduled for the order's patient.

Report generation

The report will contain all orders for this patient.

Case 8

Report scheduling

A report will be scheduled for the order's patient.

Report generation

The report will contain all orders requested by issuers belonging to the issuer group Hospital 1.

Difference between issuer group and correspondent group (in eligibility MISPL)

Issuer group:

  1. To whom: all issuers belonging to the specified issuer group (possibly combined with target specific, see above)

  2. Content: only orders issued by issuers from that issuer group

Correspondent group (in eligibility MISPL)

  1. To whom: all issuers belonging to the specified correspondent group

  2. Content: is always identical.  E.g. used to send electronic reports to CyberLab users
