Services - Start

Contents: Introduction - Parameters

Introduction

To start a service, select a Service record and choose the contextual menu / ribbon item Start.

Notes

  • On Windows, services first need to be registered as a Windows service.
  • Services can only be started from the server on which they are running, not from a client PC.
  • When logged is as a user having a role with User type set to Developer, the service's Specific options will be displayed. 
  • Services can be started from the command prompt: mam glims run -b  -P [ap_svbk|ap_cron]> -N <ServiceName> 

Tip

  • GLIMS also offers a general Startup services function designed for a full start of all services.
  • GLIMS also offers a Monitor services function designed for automatic restart of services and translators that unintentionally stopped.

Parameters

Start depending translators

Determines if any depending translators will be started immediately after the service is started.

None

None of the associated translators will be started.

Autostart

The associated translators for which the option Autostart is enabled will be started.

All

All associated translators will be started.
