Services - Start/Stop profiler

About service profiling

Profiling generates a report with data which may be useful for pinpointing performance bottlenecks in the code. Developers can load these reports into the Progress profile info viewer for further analysis.

Note

  • This function is only supported for services with Internal language set to URL

Start profiling

To start profiling, select a Service record and choose the contextual menu / ribbon item Start profiler.

Note

  • This function can only be executed by users having a role with User type set to Developer.

Scope

Session

One single report (in file <LogFileName>_<Date>_<Time>_S<SeqNo>.out) will be generated, scoped to the entire process.

Detail

One report (in <LogFileName>_<Date>_<Time>_D<SeqNo>.out) will be generated per message processed (for translator servers) or task executed (for task schedulers).

Minimal processing time

Expressed in seconds. It is only taken in account when the Scope option is set to Detail.

It constitutes a criterion for messages or tasks to be eligible for report generation. If they require less than the specified amount of time, report generation is skipped.

Stop profiling

To stop profiling, select the Service record and choose the contextual menu / ribbon item Stop profiler.
