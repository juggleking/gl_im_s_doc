Services - Register As Windows Service

Contents: Introduction - Parameters

Introduction

Services running on a Windows server need to be registered as a Windows service before they can be started.

  1. From a Service record, use the contextual menu / ribbon item Register as windows service.
  2. You need to register a service only once. When successfully registered, the service will be visible in the list of Windows services.

Parameters

User login name

The user account that will be used to run the service. There are two options:

LocalSystem

If the service only needs access to the server on which it is running, you can specify LocalSystem.

Domain user

If the service needs access to other systems in the network domain, a domain user with the necessary access rights needs to be specified in the format <DomainName>\<LoginName.

Note

The specified user does not need Administrator privileges but needs the user right Log on as a service. The procedure to grant this user right varies according to the Windows version.

Example:

  1. Go to Start -> Programs -> Administrative Tools -> Local Security Policy
  2. In the left pane, go down to Security Setting -> Local Policies -> User Rights Assignments
  3. In the right pane, double click on Log on as a service
  4. Click the Add button
  5. Add the user account you want to use to run the service to the list
  6. Click OK twice and close the Security Policy window

User password

Must be set to

  • LocalSystem when user login name is set to LocalSystem.
  • the matching Windows password when you are running the service using a real user account.
