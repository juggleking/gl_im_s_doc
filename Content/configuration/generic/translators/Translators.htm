Translators

Contents: Introduction - Fields - Beta translator versions

Introduction

The translator's job is to translate messages from an external application's language into the GLIMS internal language, and vice versa. Translators are available from the main menu via Start -> System management -> Services -> Translators.

Fields

Background and Description Page

Name

Should not contain any characters other than letters, digits and underscores. The first character must be a letter or an underscore.

Internal language

Should be matched by the internal language of the associated service. For GLIMS, this should always be set to URL.

Autostart

When enabled, and the Autostart option of the associated Service is also enabled, the translator will be started by the Startup services function.

Note

On Unix: use the parameter -a: e.g. mam glims start -a -n.

Seq no

Determines the sequence in which translators are started and stopped by the Startup services and Shutdown services functions, respectively. Shutdown uses the inverted order.

Time out

The number of seconds that the associated service is willing to wait for an answer from the translator.  The service sends a message to the translator and expects the answer within this amount of seconds.

Note

The translator forwards the translated message to the external application and expects the answer within a number of seconds as specified by the External interface time out (see below). If the latter exceeds the translator Time out, it is possible that when the translator tries to forward the translated response back to the service, the latter is no longer listening. That is when the translator logs Invalid I/O message channel index...

Server

Reference to a Service of type Translator server.

Driver type

Should be set to Java for Java-based translators. For Java translators, a Communication engine must be specified.

Note

  • Contrary to Translators with Classic drivers, Translators with Java drivers cannot be registered as Windows services.
  • A Java translator inherits the Service group from its Communication engine.

Communication engine

For translators with Driver type set to Java, a communication engine must be specified.

Driver

Name of the actual driver used to communicate with the external application.

Tip

The version of the translator's driver can be extracted through MISPL. 

Example:  "RETURN .DriverVersion;"

Specific options

A list of options passed to the driver. The format is { -<Name> [<Value>] }. These driver-specific options are documented in the driver documentation.

Restart failure

Not yet supported.

Service group

The service group to which this translator belongs.

Tip

To easily move multiple Translators from one Service group to another, use the Translator-based function Change service group.

Description

Free text description.

Logging Page

Verbosity

Defines the amount of logging detail. Info, warning and error messages are always logged, independent from the log verbosity.

Low

TCP/IP communication is not logged.

Medium (recommended)

TCP/IP communication is fully logged.

High

Byte per byte logging.

Note

Use this for debugging purposes only !

File refresh, File refresh unit

Period after which open log files should be closed and a new log file should be opened.The actual log file names will be different for each new version by suffixing them with a creation time stamp.

File keep, File keep unit

Period after which log files should be automatically deleted. If not specified, old log files have to be deleted manually.  You cannot delete a log file of a running translator.

File max size

The maximal size in MB of the log file.

File refresh on max size

Determines what happens if the File max size is reached.

  • If enabled, a new log file is created.
  • If disabled, the translator exits to prevent disk overflow. In such case, the associated service is informed which alerts all users having a role with User type set to System manager by email.

External interface page

Host, Port

Informs the translator where to find the external application it needs to communicate with.

Time out

Expressed in seconds. Can usually be left to its default value.

Status page

Host

As specified in the service group to which the translator belongs.

Port, Process id

Are set automatically when the translator is started.

Advanced page

Internal options

Stores the option list sent by some translators upon registration (e.g. HOML).

Translator log files

Translator log files are located in automatically created folders.

Default location / file name

<log root>/trl/<translator name>/<translator name>_<time stamp>.log where <log root> is <application root>/log by default.

Custom location

<log root> is <application root>/log by default, but can be overridden by

  1. the Log directory of the service group to which the service belongs
  2. the Log folder in the general options (Start -> System management -> Customize -> MATE).

Log file names with time stamp

Due to automatic refreshing, multiple log files may be generated. The actual log file names will be different for each new version by suffixing them with a creation time stamp.

Log file names without time stamp

There is also a single log file without a time stamp suffix in its name.

This file captures any messages generated either before the logging environment of the process is fully set up or by default low-level error handling.

Tip

The presence of such a log file may be an indication of unexpected errors.

Beta translator versions

To allow on-site testing of unreleased communication drivers, the internal service will allow translators using a non-validated driver version to register without checking for a valid license.

In such case:

  • Users having a role with User type set to System manager will receive a warning message by mail. 
  • Use of non-validated driver versions is marked in the Translator record as long as it is running.
  • Use of non-validated driver versions is logged in the Service log.
