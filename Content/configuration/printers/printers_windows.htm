Printers

Introduction

In GLIMS, printers must be defined via Start -> System management -> Printers.

Fields

Name

Free to choose name for the printer

Printer paths

Windows path, Unix path

Should be printername for local printers or \\server\printername for remote printers.

Tip

In case of Windows printers, you can double-click to select an available printer. GLIMS will then use the actual Windows printer driver.

Text reporting

Top limit

Distance from top of paper to printable page.

Left limit

Distance from left edge of paper to printable page.

Print command

System command to print file (specify '&1' for file name).

Process print command locally

Please check the chapter Report Builder - Raw print commands.

Preamble, Postamble

The name of a file that will be printed, using the print command, before/after the actual print job. Can be used to execute additional print commands.

Note

If the file to be printed has an extension included in the WindowsPrintingSupportedExtensions list (see below), the Preamble/Postamble will not be applied.

Report builder

Printer sides, Job sheets, Media tray, Print quality, Sheet collate, Chromaticity

Optionally allows to override the default setup of the Windows printer driver or UNIX printer queue.

Tip

Equivalent print options are available in the Report template configuration. If the Report template option is set to Default, the Printer option is used. Else, the Report template option takes precedence over the Printer option.

Please check the Report template documentation for a full description of these options.

Report builder service

Please check the chapter Report builder service.

Which files types can be printed on a Windows printer ?

By default, only MS Word documents are printed with Windows printers, but you can add other file extensions that should be printed directly to a windows printer. These file extensions must be added to the 'WindowsPrintingSupportedExtensions' list in the MA_SECTION of Progress.ini (comma separated list).

Define the file type in Windows

To print text files on a windows printer, like for instance '.prn' files (default GLIMS text reports), you have to define the .prn file type in Windows. To do so, open the Windows Explorer and select View->Options->File Types.  and create a new file type. 

Description, Associated extension

Should be set to prn

Content type (MIME)

leave this field blank

Make a new action for the file type

Action

print

Application used to perform action

c:\winnt\notepad.exe /p

 

 GLIMS should now be able to print .prn files on a Windows printer.

Alternative way of using Windows printers

Files that have an extension that is not included in the "Windowsprintingsupportedextensions" environment variable can still be printed on a Windows printer:

  • Select a valid printer from the printer list by pressing CTRL-F on the 'Windows path' field.
  • As print command, specify: '<Glims directory>\bin\print.bat &1 &2 &3'.  
    (&1 = file name, &2 = copy count, &3 = printer name.  These values will automatically be substituted by GLIMS).

The print.bat contains two lines :

q:\appl\glims\bin\Printfile.exe %1 %2 %3
del %1

If you do not yet have Printfile.exe, you can download it here.

It can be freely adapted. If, for instance, you do not want the printed files to be deleted, you can remove the second line.

Note

If you want to use this method for e.g. .prn files, you still have to define the .prn file type in Windows as described above.
