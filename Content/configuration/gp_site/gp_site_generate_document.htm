Generate document

Contents: Introduction - Options - Example

Introduction

In addition to the Order.GenerateDocument and Correspondent.GenerateDocument functions, GLIMS also offers a general document generation tool.  

It is available from the main menu via Start -> Routine -> Generate document.

Parameters

Base table

The selected base table (Site, Correspondent, CorrespondentGroup and Tour) determines:

  • for which type of record the document can be generated (see Record parameter).
  • the dynamic text table for title page, first page header, page header and footer.
  • the dynamic text table of the body is gp_Site if Site is selected, Correspondent in all other cases.

Record

For which record should the document be generated ?

Title page, First page header, Page header

Dynamic text based on the base table you selected.

Body

Dynamic text based on table

  • gp_Site if Base table is set to Site
  • Correspondent if Base table is set to Correspondent, CorrespondentGroup or Tour

Notes

  • The body is repeated as many times as determined by the range From seq no / To seq no.
  • The current seq no is passed to the body text as variable {SeqNo}.
 

From seq no, To seq no, Extra parameter

These options are passed to all dynamic texts (title page, first page header, etc...) and can be displayed in reports by using the following variables: {FromSeqNo}, {ToSeqNo}, {ExtraParameter} (see example below).

Footer

Dynamic text based on the base table you selected.

Language

An explicit language can be chosen. If not specified, the report language is set to:

  • The correspondent's language if Base table is set to Correspondent (otherwise user/site language)
  • User language or site language, if user language not set.

Other options

The other options are self-explaining.
