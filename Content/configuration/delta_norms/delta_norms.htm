Delta norms

Content: Introduction - Fields - Examples - Use baseline value for studies

Introduction

Delta norms set reference values for the difference between consecutive assessments of the same property on the same object.  

The search for previous results against which delta checks are executed takes the following criteria into account :

  1. Unsolicited results are not taken into account, because they might be unreliable.
  2. The minimal previous result status should at least be Confirmed, not Available, for the same reason.

Select the property for which you want to define a delta norm and select the contextual menu/ribbon item Delta norms. 

Fields

Severity

Severity applied to results for which this delta check fails.  For more information, please check the chapter Severities.

Discriminator

If specified, this delta norm will only be applied when the property was requested with the specified discriminator. 

Note

When GLIMS evaluates which delta norms are applicable, the following logic is applied: as soon as 1 delta norm for the result's discriminator is found, only delta norms matching the result discriminator will be considered.  Other delta norms will no longer be taken into account.

Object template

If specified, this delta norm will only be applied for objects of the specified template. (e.g. delta norm only applicable for objects of object template 'horse')

Discriminator restricted

Indicates that the base result used to compare the new result with, should have the same discriminator as the new result.

Material restricted

Indicate that the delta norm is restricted to results of the same material.

Enabled

Can be used to disable the delta norm without deleting it. Not deleting a delta norm allows it to be included with values assessed earlier, when the delta norm was still enabled.

Eligibility

Allows specifying a Result-based, Logical-valued MISPL expression.

It is evaluated as a final criterion for each delta norm fitting the result context.

  • If it evaluates to False, the delta norm is not eligible.
  • If it evaluates to True, the delta norm is eligible.  

Note

When executing the specified MISPL function, the updated result record is not yet written to the database.  This implies that start table methods like .NumericValue will NOT work!

The only Result fields that can be relied upon in this MISPL field are .RawValue and .AvailabilityUser

Type

Linear

In case of Linear delta norms, it is assumed that the new result is allowed to deviate from the previous result by the same number PER Lapse denominator (see below).

Example

When the allowed deviation is 10% per 2 days (Lapse denominator), then the allowed deviation after 4 days is 20% of the original value. 

Exponential

In case of Exponential delta norms, it is assumed that the new result is allowed to deviate from the previous result by the same percentage per Lapse denominator and that this is a cumulative process.

Example

When the allowed deviation is -10% per 2 days (Lapse denominator) and the original value is 100, then the allowed value is 90 after 2 days, 81 after 4 days, 72.9 after 6 days, ect.

The formula used is:

X = X0 * EXP(1 + <p>/100, <time> / <Lapse denominator>)

where <p> is expressed in percent and typically is negative.

Lapse denominator

A fractional day count. Check the description of the Type field for an example of its usage.

Example

Assume a linear delta norm:

  • Low limit= -10,
  • High limit= 20
  • Lapse denominator= 14.  

Result:

  • After 7 days, the high limit=10%
  • After 14 days, the high limit=20%,
  • After 28 days, the high limit=40%, etc.  

Note:

  • The lapse denominator is valid for both relative (%) and absolute (see Pivot field) value shifts.

Minimum lapse

A fractional day count. If the time lapse since the previous result does not exceed this minimum, the delta norm is not applied.  

Typical usage: avoid using an exponential delta norm when the time difference is too small.

Maximum lapse

A fractional day count. If the time lapse since the previous result exceeds this maximum, the delta norm is not applied.

Low limit

The acceptable value shift since the previous result, expressed in percentage.  If the value shift is lower than the Low limit, the delta norm will be applied.

High limit

The acceptable value shift since the previous result, expressed in percentage.  If the value shift exceeds the high limit, the delta norm will be applied.  

Pivot

It is possible that the allowed shift value is calculated by applying the specified limits to the Pivot value.  This implies that a fixed limit range will be used instead of a relative one.  

When will this fixed range be applied:

Linear delta norms

If the previous result value is lower than the pivot value.

Exponential delta norms

If the time between two assessments is shorter than the pivot value, which in this case is a time indication in fractional days.  This prevents the allowed difference from approaching zero when the time difference between the old and the new value becomes too small.

Example

Assume a linear delta norm:

  • Low limit=-20
  • High limit=10
  • Pivot=15
  • Previous value = prev 

Result

  • Previous result equal to or higher than 15 -> relative shift value -> acceptance range : prev -20% - prev +10%
  • Previous result lower than 15 -> fixed shift value -> acceptance range:  prev -3 - prev +1.5

Examples of delta norm settings

In terms of percentages

The result of a property ranges between 100 - 500 (norms).  If you want the delta check always to be executed in terms of percentages, then you should specify a very low pivot.

E.g.

  • Pivot = -10
  • Low limit = -10
  • High limit = 20
  • Previous result = 110

If current result = 120 (above the pivot value), then a relative check will be performed =>
acceptable value shift = (110 + 110*-0.1=) 99 - 132 (=110+110*0.2)

All results between 99-132 will not be flagged for delta check violation.

In terms of fixed shifts

The result of a property ranges between 100 - 500 (norms).  If you want the delta check always to be executed with fixed value shifts, then you will have to specify a very high pivot.

E.g.

  • Pivot = 1000
  • Low limit = -3
  • High limit = 5
  • Previous result = 110

If current result = 100 (below the pivot value), then an absolute check will be performed =>
acceptable value shift = (110+1000*-0.03=)  80 - 160 (=110+1000*0.05)

All results between 80-160 will not be flagged for delta check violation.

Use baseline value for studies

Introduction

Regular delta norms in GLIMS define reference values for the difference between consecutive assessments of the same property on the same object.  When executing a delta norm check for a result, GLIMS will lookup the most recent previous and confirmed result for value comparison.

For clinical trials (studies), it is possible to perform the delta norm check against a result that is manually marked as "baseline value" instead of using the most recent previous result.  

Note

This feature can only be used for results that belong to a study.

Features

Mark a result as being a base line value

A result can be (un)marked as a base line value for delta norms as follows:

  • From a Result, select the contextual menu / ribbon item Mark as base line
  • Using the  Result-based MISPL function SetAsBaseLine(Yes/No).  

The Result option Is base line value will be checked.

When to use the base line value

The Delta norm option Use base line indicates how the base line feature needs to be applied. Possible options:

No base line

Base line values will never be used, the regular delta check will be performed (using the most recent previous result).

Preference for base line

If a base line value can be found, it will be used for the delta check, else the regular delta check will be performed.

Only base line

If a base line value can be found, it will be used for the delta check, else NO delta check will be performed.

Note

When using base line values, GLIMS will look for the most recent result (of the same patient with the same study) for which the field Is base line value is enabled.
