Procedures - Organization

In order to smoothly organize your procedures, it is important that you configure the different related tables in the right order:

The following tables will have to be configured:

  1. Properties

    Properties are the tests executed in the laboratory.  They will mainly be defined as procedure outputs.   

  2. Stations  

    A station can either be a place where tests are executed manually or an analyzer.

  3. Assessment method

    Assessment methods create a link between stations and properties. You can open the Assessment method browser using the popup of properties or stations.  

  4. Material

    Materials will be mainly be used as procedure inputs.  

  5. Procedure

    The procedures relate all previously configured tables (Properties, Stations, Assessment methods, Materials, etc.)

  6. Procedure input (Material input or Property input)

    Each procedure needs an input (usually a material).  Procedures inputs can be entered by using the contextual menu on procedures. You can either choose a material (most common) or properties (calculations) as input for this procedure  

  7. Procedure output (Material output or property output)

    Each procedure needs an output (usually one or more properties). Procedures outputs can be entered by using the contextual menu on procedures.  You can either choose a property (most common) or a material as output for this procedure.
