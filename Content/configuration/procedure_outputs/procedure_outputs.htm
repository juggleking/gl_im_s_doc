Procedure outputs

Contents: Introduction - General - Property - Material page - Microorganism test page

Introduction

Procedure outputs can be accessed:

  • By selecting a procedure and choosing the contextual menu / ribbon item Outputs.
  • In the Station - Edit procedures screen.

General

Procedure

Procedure to which this output applies.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Automatically generated by GLIMS.

Seq no

The relative order of this output within the procedure.  Among other, this is used to sort the outputs on work lists.

Enabled

Must be enabled if GLIMS should take this procedure output into account during action scheduling.

Shareable

Indicates if the same action can produce several of these outputs. If the same output needs to be realized multiple times, the action scheduler will only reuse the same action if the Shareable option is enabled.

Type

A procedure output can be a Property, Material or Microorganism test.

Depending on the type, additional fields will be shown (see below).

Property

Method station

Automatic reference to the station (of the procedure).

Method property

Property that is output of the procedure.

Consumption

Please check the chapter Specimen size calculation.

Schedule preference

Used by the action scheduler in case the procedure selection is ambiguous. The available property outputs for a given property will be tested for eligibility in the order of their schedule preference.

Auto prompt

If enabled, no request is required for the property to appear on work lists.

Example

Let's assume you want to perform a microscopic examination for which you do not know in advance what exactly you will find. In that case you could define a procedure with:

  • A procedure output referring to a 'microscopy examination' property that will be requested during order entry (Auto prompt is disabled).
  • 15 procedure outputs for each property (element) that might possibly be detected during the examination. For these procedure outputs, the Auto prompt option must be enabled.

When generating a work list, all 16 properties will be shown for the procedure's action. If you now enter results for e.g. 3 of the 15 Auto prompt outputs, only these 3 properties will be requested (automatically as soon as you enter a result).

Value

This field is used for standard results and formulas (calculations). It allows to specify an Action-based site function. The specified site function is evaluated when the corresponding Action reaches the status Outputting. The site function yields a fractional for numerical properties, else a string. The return value will be the result of the output property.

Warning

The specified site function is evaluated when previewing calculated values (see Preview calculations) during order review or specimen review.

Therefore, the site function should never ever contain 'active' MISPL functions such as .AddRequest(), .CascadeRequest(), .Cancel(), .Discontinue(), .SetSiteAttribute(), etc. During the preview, the site function might be executed on premature, unavailable or unreliable inputs, thus triggering incorrect decisions based on unexpected results!

Only use PropertyOutput.Value for its intended purpose. Configure 'active' MISPL functions in the Property Triggers.

When the SendMail() function is used in the Property output Value field during a preview, the MISPL code will NOT be executed, and the user will see an information message explaining why no MISPL was run. Other MISPL functions will be executed during the preview, but their changes will NOT be saved.

Note

In the calculation, you can refer to the input results by specifying the Property mnemonic embraced by a Space character.

Examples

Calculation-GFR ("CREAT" as input)

STRING value; 
FRACTIONAL fValue, age, sex; 

age := .Object.AgeInYears(DateTimeToDate(.0bjectTime)); 

IF .Object <> ? THEN 

   IF age > 18.0 THEN 

      IF .Object.Person().Sex = 2 THEN /* FEMALE */ 
         sex := 0.742; 
      ELSE 
         sex := 1.0; 
      ENDIF; 

      IF CREAT <> ? THEN
         fValue := 175 * Exp( CREAT , -1.154 ) * Exp( age , -0.203 ) * sex; 
         value := FractionalToString(fValue,"%4.2f"); 
      ELSE 
         value := "Not calculated"; 
      ENDIF; 

   ELSE 
      value := "Not calculated: age < 18 years"; 
   ENDIF; 

ELSE
  value := "Not calculated: Object unknown at calculation time"; 

ENDIF; 

RETURN value;

Calculation-CIT-THROMB ("THR.CITR" as input)

STRING value; 
FRACTIONAL fValue; 

IF THR.CITR <> ? THEN 
   fwraarde := 1.1 * THR.CITR ;
   value := FractionalToString(fValue,"%4.2f"); 
ELSE 
   value := "Not calculated"; 
ENDIF; 

RETURN value;

Auto request when starting activity

If this option is activated, a result for the output property is added to the order automatically when the approach activity starts.

Material

Material

Material that is output of the procedure (e.g. for sampling procedures).

Production

Please check the chapter Specimen size calculation.

Auto login

If enabled, the status of the specimen (that is output of the action) will automatically be promoted from Expected to Available.

Number of specimens to be auto requested

This field allows indicating how many identical specimens of the material output should be added to the order when the approach activity of type Procedure is started. The default value is "1". If the value is higher than 1, the option Shareable must be activated. If the field contains "?", the material output is not added to the order automatically.

Microorganism test

Method station

Automatic reference to the station (of the procedure).

Method microorganism test

Micro-organism test that is output of the procedure.

Auto prompt

If enabled, no request is required for the microorganism test to appear on work lists.
