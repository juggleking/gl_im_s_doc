Coding systems

Introduction

GLIMS features a general coding system functionality which allows assigning external codes to virtually any configuration table record in your database.  These coding systems are used to map GLIMS configuration data with data in an external system in order to ensure correct communication between the two. 

Coding system

Coding systems are available from the main menu via Start -> Configuration -> Coding systems.

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

The mnemonic will also be exported for reports that require adding the coding system along with the external code (e.g. HL-7).

Name

Free-to-choose name

Usage

Allows to find specific coding systems for specific programs (e.g. the Valab program needs a coding system with Usage set to Valab), or allows the program to check for the correctness of codes (e.g. 'Codes must be numeric').

The currently allowed values are Biorad, Valab, QCToday, Iris

Oid

Object id: unique coding system identifier used in communication with external applications.

Code sets

Embedded browser showing the code sets defined for this coding system.

Code set

A code set groups the codes assigned to records of a single table, within the context of a coding system. Only template tables with a Mnemonic or Name field in their primary index can have a code set.

Code sets can be defined for each coding system by selecting the contextual menu / ribbon item Code sets.

Fields

Coding system

Coding system to which the code set belongs

Table

The table being coded.

Note

  • Code records can refer to theoretically any record in any table. The configuration allows to choose any table (with the exception that code sets are only allowed for template tables).  This may lead to a situation where the record can not be properly externalized in the Code editor.  In such cases, the record will either be externalized via its internal identifier (e.g. StationFlag.Id) or via a special format (e.g. <Station Mnemonic>/<Property mnemonic> for assessment methods). The special format used for e.g. AssessmentMethod even allows to manually create code records by actually typing in something like 'H717/Na' in the externalization field of the code editor.
  • Although the externalization may be limited, this allows creating Code records for any template table via the option View -> Codes in the contextual ribbon.

Code function

A <table>-based site function for automatic creation of codes for the records of <table>.  Check the chapter Filling a code set for more information.

Code check

A <table>-based site function that is executed to check the validity of a newly defined code value

Original coding system

If defined, this field refers to the coding system in which the actual code set is defined. This allows to create coding systems which make use of code sets actually residing in another coding system without having to copy the code sets (and update several code sets afterward).   It allows to define a "common" code set used by several coding systems.

Example

A Medar coding system which makes use of the Property code set of the Medidoc coding system. 

Unique value

Indicates that a code's value should be unique within this code set. If disabled, a discriminator can be specified in the Code configuration.

Unique record

Indicates that a code's record should be unique within this code set

Genetics

The fields are described in the documentation of the Genetics module.

Code

The code is the actual external code for a specific record of <table> within the coding system. Codes can be defined by selecting contextual menu / ribbon item Codes on a given code set.

Fields

Record

Refers to the original record in <table>

Value

The external code value for this record

Description

Free text description

Sequence

Used:

  • to evaluate the eligible codes (see also: Coding systems: LOINC)
  • as sorting order when displaying multiple codes for 1 record (e.g. in browsers)

Eligible

MISPL function that allows to define additional criteria that should be met before the code is valid. Can e.g. be used to specify a Result-based MISPL function for properties.

Discriminator

Only shown when the code set option Unique value is disabled. Multiple codes (with a different value) for the same record can then be distinguished using a discriminator.

Direct access to all codes

To provide a full overview of all codes of a specific configuration record, you can use the contextual ribbon item View -> Codes.

Example

  1. Open a Property browser
  2. Select a property
  3. In the contextual ribbon, click View -> Codes
  4. All codes (possibly from different coding systems) are displayed.

This browser also allows inserting new codes for a specified coding system.

Copying a code set

Using the contextual menu / ribbon item Copy on a Code set, you can copy an entire code set to another (new or existing) coding system.

Filling a code set

The contextual menu /ribbon item Fill on a Code set allows to automatically fill a code set by creating Code records for all records of the code set's source table.

The Code values are generated by the function (based on the source table and returning a string) specified in the Code function field of the Code set.  

When selecting this function, you will be offered the choice to overwrite already existing code records or not.

MISPL access

GLIMS offers a Result-based MISPL function GetCode which allows to lookup a code in a specific coding system. The lookup logic is identical to the one described in the chapter Coding systems: LOINC.

Synopsis:

StringGetCode (MnemonicCodingSystemMnemonic)

Note

A general GetCode function is available as well.
