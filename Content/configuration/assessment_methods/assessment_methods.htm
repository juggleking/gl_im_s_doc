Assessment methods

Contents: Introduction - General - Setup - Status - Limits - QC - Counter - More

Introduction

Assessment methods create a link between Stations and Properties. You can open the Assessment method browser using the context menu/ribbon of properties or stations.

General

Station

Station linked to the assessment method.

Property

Property linked to the assessment method.

Setup page

Inbound id and Outbound id

These are identifications which indicate a channel number for communication between GLIMS and the analyzer.

Inbound id

A remote property identification used on the result uploading from the instrument.

Outbound id

A remote property identification used on request downloading to the instrument.

During result upload, GLIMS will look for a case-sensitive match between the instrument test id and the assessment method inbound id.  GLIMS will ignore the case-sensitivity check when a case-insensitive unique match is found.

Example

Assessment method inbound id's are a, A and b.

  • GLIMS will upload a result with instrument test id 'a' for inbound id 'a'
  • GLIMS will upload a result with instrument test id 'A' for inbound id 'A'
  • GLIMS will upload a result with instrument test id 'b' or 'B' for inbound id 'b'

Store unsolicited

Determines whether unsolicited results (i.e. results sent by the station but which were not requested) are stored. If this option is checked, a result is created without a request. This implies that it will not be reported nor charged.

Unit

This field contains the default unit for result upload in case of numerical results.  For more information, please check the chapter "Units and dimensions".

Convert if manual

  • If enabled, then also manually typed in results will be assumed to be in the assessment method unit and therefore be converted if needed.
  • If disabled, the default unit for a manually entered result will be the property's unit, not the assessment method's unit.

This allows to connect an analyzer which sends its result in another unit, while still being able to review and enter results in the property units (e.g. via the work list). For more information, please check the chapter "Units and dimensions".

Remote material id

A minority of analyzers also request a description of the material (as known by the analyzer) on which a test has to be executed.  If specified, this info will also be downloaded from GLIMS to the analyzer when sending tests.

Send previous result

If checked, the most recent result is sent to the instrument upon order downloading.

It may happen that, along with the order, GLIMS sends results to an analyzer (e.g. previous result for delta checking). If so, the start date of this result will be the Result.AvailabilityTime and the end time will be the Result.ConfirmationTime.

Conversion

This is a result based MISPL expression that allows converting incoming results.  Note that this field has been developed only for conversion of raw data and not to perform e.g. unit conversions (for which explicit support is available in GLIMS).

Examples

"+++" to "> 100"
"red" to "positive"

The actual conversion is performed BEFORE the result is stored in the database, which implies that when using Result table functions, you will be using OLD data !  The only fields that can be used in this MISPL expression are:  
  • RawValue
  • AvailabilityUser
  • StationSeverity
  • ManualSeverity
  • InternalComment
  • ExternalComment

Precision

This field specifies the number of significant digits stored in a numerical result.

Note

This field also determines the number of significant digits stored in the QC population's Mean value if the QC precision is not specified. Click here for more information.

QC precision

This field specifies the number of significant digits stored in a numerical QC result. If unknown, the Precision field will be used for QC results as well.

Note

This field also determines the number of significant digits stored in the QC population's Mean value. If unknown, the Precision field will be used. Click here for more information.

Status page

Status

This field can only be set manually to one of the following values:

Disabled

Indicates that this channel should no longer be used.

Enabled

If you want to make full use of an assessment method, its status should be set to Enabled.

Closing

Indicates that this channel will no longer be used for scheduling, but result upload from the analyzer to GLIMS is still possible.

Locked

Indicates that this channel is still used for scheduling, but GLIMS will not download orders via this channel.  This can for instance be used to buffer all tests requiring an expensive reagent.  When all tests requiring this reagent are scheduled, you can reset the status to Enabled and all tests will be downloaded to the analyzer.

Status comment

Allows to enter comments on the status of the assessment method (e.g. reason why it has been locked).

Quality

The quality status of an assessment method can automatically be set to Unreliable when the QC alarm severity is exceeded.  For more information, please check the chapter "Deviation actions" in the QC tutorials.

Quality comment

Comments on the quality status of the assessment method. For more information, please check the chapter "Deviation actions" in the QC tutorials.

Limits page

Low reject

Numerical values below this limit are rejected. This value is expressed in the unit of the property.

High reject

Numerical values above this limit are rejected. This value is expressed in the unit of the property.

Low truncate

Numerical values below this limit are truncated to " <x "

High truncate

Numerical values above this limit are truncated to " >x "

Auto-confirm severity

The results with a severity up to this level will be auto-confirmed.

If the Result.Severity (calculated as the maximum of Result.NormSeverity, Result.TechnicalSeverity, Result.DeltaNormSeverity, Result.StationSeverity, Result.QCSeverity and Result.ManualSeverity) exceeds this limit, the result will not be auto-confirmed.

Auto-validate severity

The results with a severity up to this level will be auto-validated.

If the Result.Severity (calculated as the maximum of Result.NormSeverity, Result.TechnicalSeverity, Result.DeltaNormSeverity, Result.StationSeverity, Result.QCSeverity and Result.ManualSeverity) exceeds this limit, the result will not be auto-validated.

Auto-confirm if manual

If this option is NOT checked, manually entered results will NOT be subjected to the automatic confirmation rules (i.e. they are immediately marked as Result.NeedsManualConfirm = YES).

Delta norm has precedence

If set, and a delta check is performed, then the delta norm severity overrules the norm severity for the auto-confirm/validate decisions (see below).  So, a result that is pathological (norm severity > 0), and that normally wouldn't be auto-validated, might still be auto-validated because it's similar to the previous result of the same patient.

The actual severity that is compared against the auto-confirm and auto-validate levels, will then be the final result severity (Result.Severity) without the norm severity component (Result.NormSeverity). So, it is calculated as the maximum of the absolute values of all severities, EXCEPT the norm severity.

QC auto-confirm severity

QC results with a severity up to this level will be auto-confirmed.

Warning severity

This field indicates the severity starting from which a warning is issued for a result.

QC alarm severity

This field indicates the minimum QC severity which will mark this method as unreliable. For more information, please check the chapter "Severities".

QC warning severity

This field indicates the QC severity starting from which a warning is issued for a QC result.

Note

The QC warning severity is also taken into account when manually confirming QC results in order to determine whether or not an external comment has to be specified by the user for failed QC results.

Phone severity

Results with a severity equal to or higher than this threshold will automatically be marked as to be phoned.

The Phone severity defined in the Assessment method is compared with the maximum of the result's phone list severity, delta norm severity and manual severity to decide whether the result should be marked as To be phoned.

Maximum delay / Max. Delay severity

It is possible to specify the maximum delay time between a result's object time and its availability time.  If this delay is exceeded, a configurable severity will be set.  This severity can then be used for instance to prevent automatic validation.

Maximum delay

A decimal indicating how many hours are allowed between the result's object time and availability time.

Max. Delay severity

An integer indicating what severity must be set for the result when the delay is exceeded. This is optional.

Note

  1. When the specified delay is exceeded, this will also be logged in the result log.

  2. The result's manual severity is used to store the max. delay severity.

More page

Method

A description that can be printed on patient reports and QC reports.  It can also be used for sending a station code during export of QC results to the external system Biorad.

The Result.Method field stores the value of AssessmentMethod.Method at the time of result entry.  The report program which can put the method field on the report (by specifying a row label for Method in the report template) uses Result.Method as soon as the result is available and Result.Method is known.  If not, AssessmentMethod.Method is used.

This allows storing the history of the method field in the result table itself, thus making this field reproducible on the report, even when the assessment method has been changed.

The function AssessmentMethod.SetResultMethod allows to explicitly fill this Result.Method field with either the value from the assessment method record, or a specific value. This may be useful for storing the method name in old result records.
