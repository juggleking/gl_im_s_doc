Diagnosis code system

Diagnosis code systems allow defining different code systems (e.g. ICD10).  They are available from the main menu via Start -> Configuration -> Diagnoses -> Diagnosis code systems.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

Full name of the diagnosis code system.

Is default

Codes received from an external system that are not accompanied by a diagnose code system will be searched in this code system.

Specific for

Specifies that the diagnosis code system is specific for and only used by a certain type of laboratory, e.g. genetics.  It is currently NOT used yet by the software, but will serve in future releases to restrict access to diagnosis code systems with the same Specialty as the user's login department.

Diagnosis code

The Diagnosis code table contains standardized diagnosis codes.

Defining diagnosis codes

For each diagnosis code system, you can define a number of diagnosis codes:

  • in the embedded browser

    or

  • by using the contextual menu/ ribbon item Diagnosis codes. 

Fields

System

Diagnosis code system to which this code belongs.

Code

The actual code.

Text

Text to be included in the actual diagnoses.

Data sheet

Background information about the diagnosis.

Consulting diagnosis codes

You can obtain a list of all diagnosis codes from the main menu via Start -> Configuration -> Diagnoses -> Diagnosis codes.

Diagnosis

The Diagnosis table contains the actual diagnoses related to orders or encounters.  

  • The diagnoses sent via HL7 communication are added to this table, while retaining the hierarchical structure derived from Type + Level + Time.
  • They can be viewed via the Order.Diagnoses, Object.Diagnoses and Encounter.Diagnoses functions that use a tree viewer to display the diagnoses, grouped by order and sorted in chronological order. 
  • Diagnoses can also manually be entered in this tree view.

Fields

Code

Optional reference to a diagnosis code defined in GLIMS. Either Code or Text should be specified.

Original code

Code as sent via HL7, i.e. the code that was used to create the diagnosis. May not be present in the Diagnosis code table.

Text

When different from the Text specified in the Diagnosis code record.

Type

Hospitalization, Admission, Follow-up, Treatment, Discharge,...

Original type

As sent via HL7 communication.

Clinician

Name of the clinician who formulated this diagnosis (string!)

Level

Similar to Priority from HL7.  Allows to distinguish between primary diagnoses and subdiagnoses.

Class

The classification of the diagnosis (Consultation, Diagnosis, Medication,...) (cfr. HL7)

Order, Encounter

Order and encounter to which the diagnosis belongs.

MISPL access

The MISPL functions Order.GetDiagnosis, Object.GetDiagnosis and Encounter.GetDiagnosis allow MISPL access to the diagnosis table.

Synopsis

DiagnosisId GetDiagnosis(DiagnosisId Previous, String CodePattern);

Previous

Previous diagnosis that was fetched. When unknown, the most recent one will be returned. Use this parameter to loop through all log entries:

Diagnosis Current;
Current := .GetDiagnosis(?, "A*");
WHILE Current <> ?
DO
/* Processing code comes here */
...
/* End of processing code */
Current := .GetDiagnosis(Current, "A*");
DONE;

CodePattern

When specified, then only diagnoses where Diagnosis.OriginalCode or DiagnosisCode.Code matches the specified pattern (wildcards allowed) are considered

Notes

  1. The most recent diagnosis is returned first.
  2. When using * as code pattern, diagnoses with unknown code will not be returned.
  3. Diagnoses with Diagnosis.Level = 0 are never returned.
