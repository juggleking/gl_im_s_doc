Materials

Contents: Introduction - Material fields

Introduction

Materials reflect the different specimen types that can be sampled for a patient and on which tests will be performed. Materials will mainly be used as procedure inputs.

Materials are accessible from the main menu via Start -> Configuration -> Materials.

Examples

  • Serum, plasma, urine, etc.
  • Arterial, capillary blood etc.
  • EDTA, Frozen EDTA, citrate, etc.
  • Tube 1L, etc.

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Description

Multilingual text that can be displayed on labels, reports etc.

Short name

Used to display the material in work lists, overviews etc.

Is root material

A root material can be received by the lab. During order entry, GLIMS allows to explicitly request a material on which the tests should be executed. This will only be possible for root materials.

Label, Label seq no

Refers to a text module that defines the layout and content of a specimen label. For more information, please check the chapter Root and work specimen labels.

Size unit

Unit in which the material should be expressed.

Minimal size

The minimal filling for a container type, expressed in the size unit. For more information, please check the chapter Specimen size calculation.

Maximal size

The maximal filling for a container type, expressed in the size unit. During order entry, GLIMS will determine the required volume of the materials in order to be able to execute all the tests. If this exceeds the maximal size of a material, additional labels will be printed for all the required tubes.

For more information, please check the chapter Specimen size calculation.

Dead distribution size

The required volume that should remain in a root specimen when a distributor system has to sort it into sub specimens. GLIMS will take this value into account when the distributor system is sending the measured volume of the root specimen in the specimen sort query.

The rule is: Total volume required for the root specimen >= total volume required for the sub specimens + remaining dead volume in the root specimen.

Quality choice list

Allows for the selection of a list containing material quality levels (such as absorbance ratios for DNA samples).

Property classification

Indicates the property classification which should be shown in the Property classification tab of the Specimen info screen.

The property classification outline is mapped as follows in the Specimen info screen:

  • node groups (i.e. nodes that are not linked to a property) are displayed as main tabs.
  • node subgroups (i.e. nodes that are not linked to a property and are placed within a group) are displayed as accordion tabs.

  • nodes that are linked to an informational property are displayed as fields. If such nodes are not placed within a group, they are displayed in a default tab called Other result properties.

    Note

    • The order in which the nodes are displayed in the routine screens (e.g. the Genetic exam screen, Specimen info screen) reflects the order of the nodes in the property classification.
    • Property classification nodes are usually referred to by their label. However, if a property classification node has no label, the property's description is used (if Show property description is activated). If the property's description is not indicated (or if Show property description is not activated), the property's short name is used. If the property's short name is not indicated, the property's mnemonic is used.

      If the data type of the property is

      • Numeric: only digits can be entered in the field; by default, the number of decimals allowed is 2 but can be modified in the Reported decimals field of the Report tab in the Property editor.
      • String: simple text field
      • String and the option Rich text is activated in the Property editor: a Rich text button is displayed and allows you to open the Rich text editor.

      • Enumerated: a configured choice list is displayed with radio buttons if Strict result checking is activated. However, a dropdown list is displayed instead if

        • there are more than 8 choices,
        • the choice list is empty,
        • a diagnosis code system is linked to the property,
        • Strict result checking is not activated.

        If a code system is indicated in the field Diagnosis code system, the codes of this system also appear in the choice list.

      • Image: a button is displayed and allows you to open the image viewer.
      • Other data types: nothing is displayed.

Specimens can have a quantity

Indicates whether the concept "received quantity" is relevant to specimens of this material. If activated, a Received quantity field is displayed in the Specimen info screen.

Specimens can have a container count

Indicates whether the concept "container count" is relevant is to specimens of this material. If activated, a Container count field is displayed in the Specimen info screen.

Sampling code

A sampling-related short code, mainly used for billing purposes. The MISPL function Order.Attribute("SamplingCodeList") for instance yields a list of all different sampling codes present in an order, which can be helpful to determine the specimen sampling charges.

Reusability

Period, expressed in days, during which a specimen can be reused. In the order entry options, one can indicate that GLIMS is allowed to reuse the specimens of a patient.

Default request definition

Used for billing purposes in cases where materials have been requested without the use of request definitions.

Creation trigger

Site function executed when a specimen of this material is created.

Comment

Can contain general handling guidelines for internal use.
