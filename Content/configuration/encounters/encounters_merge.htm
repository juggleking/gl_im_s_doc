Encounter merge

GLIMS allows merging double encounters. The encounter merge program is accessible: 

  • via the Double encounter browser
  • by manually selecting Encounter records and choosing the contextual menu / ribbon item Merge

Correct encounter

The correct encounter is displayed in green. 

When opened from the Double encounter browser, the encounter merge program will take the Ranking function into account in order to determine which encounter is the correct one.  When no Ranking is specified, or when two records yield the same ranking, the most recently created record has preference.

Incorrect encounter(s)

The incorrect encounters are displayed in red. 

In case of multiple incorrect encounters, they will be listed in the bottom left window.

Test mode

Allows testing the actual merge. All the database changes are undone at the end.
