Double encounter detection

Introduction - Query options - Double encounter browser - Other checks and utilities

Introduction

The double encounter program is accessible from the main menu via Start -> Routine - > Encounters -> Doubles.

  • It allows verifying all encounter records created within a specified period.
  • For each selected encounter, the database is searched for duplicates.
  • If any are found, the triggering encounter along with all its duplicates are shown in the browser.

Query options

Common patient

When enabled, double encounter external id's will only be included when assigned to the same patient (i.e. the same Person record).

Common institution

Only applies to multi-institution sites. When enabled, the external id's should be assigned by the same institution before being considered as identical.

Created since

For all encounter records with a creation date within this period, the database will be searched for duplicates. It is advised to start with a small time period to reduce the initial result list. You can choose an absolute or relative date.

Ranking

An Encounter-based, integer-valued function that returns the 'level of reliability' of the encounter record (e.g. records created by user "hl7batch" are more reliable than manually created records).

Identical records will be displayed in ranking order (descending, i.e. highest ranking first), and the record with the highest ranking will be passed as the 'good' record to the Encounter.Merge function.

When no ranking function is specified, or when two records yield the same ranking, the most recently created record has preference.

Double encounter browser

Grouping and pre-selection

Identical encounter records are automatically grouped and pre-selected. The pre-selection starts at the first block of duplicates and continues one by one until all blocks are processed. This pre-selection is intended to work together with the buttons Merge and Skip.

Merge

Passes the group of selected records to the Encounter.Merge function.

Skip

Selects the next block of identical encounters without any further processing

Alternating background color

Groups of identical records are displayed with an alternating background color to allow easy distinction. The color is fixed.

Extra info

In the lower left corner of the browser window, stay information related to the focused encounter is displayed. When the information doesn't fit in the area, the full content is displayed in the tool tip.

Other checks and utilities

In order to further prevent creation of and eliminate existing double encounters, some checks and utilities are built in:

  • When the external id or the institution of an encounter is updated manually via the encounter editor, it is verified if this causes a duplicate. If so, the user is invited to correct that situation. After confirmation, the double encounter browser appears from which the duplicates can be merged or corrected.
  • If the encounters to be merged belong to different patients, the program proposes to merge the persons instead. On successful completion, the original encounters will be automatically merged by the person merge program.
  • Assigning another person to an existing encounter is prohibited if orders are created within the scope of the encounter. The orders are to be corrected first.
