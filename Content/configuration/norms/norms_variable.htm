Introduction

This case study describes the GLIMS configuration for a Glucose Tolerance Test. This test is used to assist in the diagnosis of diabetes mellitus.

In the GT test, the patient’s ability to tolerate a standard oral glucose load is evaluated by obtaining serum specimens for glucose level determinations before glucose administration and then 30 minutes, 1 hour, 2 hours, 3 hours and sometimes 4 hours after administration. Normally there is a rapid insulin response to the ingestion of a large glucose load. This response peaks in 30 to 60 minutes and returns normal in about 3 hours.

Property

First you will need to create a property. The most important option in this case is Dynamic. If enabled, results obtained in one day will be grouped on reports. These results will be accompanied by a time indication.

Request definition

A request definition needs to be defined that requests a "Glucose Tolerance Test". As a "Glucose Tolerance Test" consists out of 5 tests (in fact it are the same tests but with a certain time interval), we will configure a request definition of type panel.

As members of this panel we define the request definition for the single tests with their relative time span.  Use the contextual menu/ribbon item Members and insert the GT request definition (automatically created for the GT property) with different Discriminators.

Variable norms (to be used in case of a relative time span)

Norm definition

In GLIMS one can define norms for each test. You can even configure specific norms for the different object attributes (such as pregnancy, diabetic, …), object templates (e.g. animals, humans, …), ages, sex and stations, but if you want to define specific norms depending on the time span you have to use the Eligibility option.

In this case the norm will differ according to the result's reference time. This field contains the time that the panel was requested. Using this time we can determinate the time span of the individual request.

E.g. MISPL of type Logical that returns TRUE if the time span is "01:00": Dynamic_60_min

/* Mispl that will calculate the time span (time between request of panel and */
/* request of panel member.   Start table = result                           */

Integer DynTime, DynHour, DynMin;
String DynDiff;
/* ObjectTime is request time of panel member                                 */
/* ReferenceTime is request time of panel                                     */
/* If you subtract two times, the result is an integer                         */
DynTime:=DateTimeToTime(.ObjectTime) - DateTimeToTime(.ReferenceTime);

/* convert integer in hours and minutes                                      */
DynHour:=DynTime / 3600;
DynMin:=(Dyntime - DynHour * 3600) / 60;
/* format integers in a string 99:99                                         */
DynDiff:=IntegerToString(DynHour,"%02d") + ":" + IntegerToString(DynMin,"%02d");
/* check time span, copy this MISPL for each time span                       */
RETURN DynDiff = "01:00";

Using this technique, you can define norms for each time span.

Other MISPL examples

Examples

Example for dynamic tests with relative times:

Integer DynTime, DynHour, DynMin;
String DynDiff,RelTime;

RelTime:="00:00";

IF EnumeratedToString("DynamicType",.DynamicType) = "Relative" THEN
  DynTime:=DateTimeToTime(.ObjectTime) - DateTimeToTime(.ReferenceTime);
  DynHour:=DynTime / 3600;
  DynMin:=Dyntime - (DynHour * 3600);
  DynDiff:=IntegerToString(DynHour,"%02d") + ":" + IntegerToString(DynMin,"%02d");
  RETURN DynDiff = RelTime;
ELSE
  RETURN false;
ENDIF;

Example for dynamic tests with absolute times:

String DynAbs,AbsTime;

AbsTime:="08:00";

IF EnumeratedToString("DynamicType",.DynamicType) = "Absolute" THEN
  DynAbs:=DateTimeToString(.ObjectTime,"%H:%M");
  RETURN DynAbs = AbsTime;
ELSE
  RETURN false;
ENDIF;
