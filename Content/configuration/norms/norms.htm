Norms

Contents: Introduction - Access - Query - Fields - Norm format - URL and variable text reporting

Introduction

For each property, you can specify one or more norms. Norms (or reference values) for a property may differ according to the patient's age, sex etc.

Access

Norms of one property

To show all norms of one particular property:

  1. select the property
  2. select the contextual menu/ribbon item Browse norms
  3. The Options button in the norm browser opens the norm query which allows to narrow down the displayed norms.

Tip

The Property editor also contains an embedded norm browser that shows all enabled norms for the selected property.

Norms of all properties

To shown all norms of all properties:

  1. Select the main menu function Start -> Configuration -> Properties -> Norms
  2. The norm query will be displayed allowing to define which norms you want to display.

Norm query

The norm query is available when

  1. Opening the norm browser from the main menu.
  2. Clicking the Options button in the norm browser.

Note

Please carefully read the description of each option as the behavior of this query screen is slightly different from other queries in GLIMS.

Property

If specified, only norms for this specific property will be shown.  If not specified, norms for all properties will be shown.

For results assessed on station

Allows to select station-specific norms:

  1. If enabled and a station is specified, all norms that are valid for results assessed on this station will be shown.

    Note

    This can either be norms that explicitly refer to a station (see Fields) or norms that are station independent. 

    However, as soon as a property has at least 1 station-specific norm, all other norms will no longer be valid for that station !!  Naturally the norm's property must have an assessment method that refers to the specified station.  

  2. If enabled and you select ? as station, all norms will be shown that do not explicitly refer to a station.

Example

Property "NA" has 5 different norms (that e.g. vary according to age, sex, etc).  

If none of these norms are station-specific (i.e. the Station field is not specified for that norm), ALL norms are valid for a specific station (on condition that NA has an assessment method for that station).  So when you query for a specific station, all these norms will be shown. 

However, as soon as 1 of the norms for NA explicitly refers to that station (i.e. the Station field is specified for that norm), the other norms are no longer valid for that station and will not be shown when you have specified this specific station as query option.

For results of study

Allows to select study-specific norms:

  1. If enabled and a study is specified, all norms that are valid for results belonging to this study will be shown. 

    Note

    This can either be norms that explicitly refer to a study (see Fields) or norms that are study independent. 

    However, as soon as a property has at least 1 study-specific norm, all other norms will no longer be valid for that study !!   

  2. If enabled and you select "?" as study, all norms will be shown that do not explicitly refer to a study.  

For results with discriminator

Allows to select discriminator-specific norms:

  1. If enabled and a discriminator is specified, all norms that are valid for results with this discriminator will be shown.

    Note

    This can either be norms that explicitly refer to a discriminator (see Fields) or norms that are discriminator independent.  

    However, as soon as a property has at least 1 discriminator-specific norm, all other norms will no longer be valid for that discriminator !!   

  2. If enabled and you select "?" as discriminator, all norms will be shown that do not explicitly refer to a discriminator.  

For objects with template

Allows to select object template-specific norms:

  1. If enabled and a template is specified, all norms that are valid for objects with this template will be shown.  

    Note

    This can either be norms that explicitly refer to an object template (see Fields) or norms that are object template independent.  

    However, as soon as a property has at least 1 object template-specific norm, all other norms will no longer be valid for that object template !! 

  2. If enabled and you select "?" as template, all norms will be shown that do not explicitly refer to a template.  

Sex

Only show norms that are valid for objects with this sex.  

Example

If you specify Male, all norms will be shown that are valid for objects that are male AND for objects of which the sex is unknown !!

Age

Only show norms that are valid for objects with this age.

Hide disabled norms

If checked, the norm browser will hide norms that are not enabled.

Fields

Discriminator

If specified, this norm will only be applied when the property was requested with the specified discriminator.  

When GLIMS evaluates which norms are applicable, the following logic is applied: as soon as 1 norm for the result's discriminator is found, only norms matching the result discriminator will be considered.  Other (non-discriminator specific) norms will no longer be taken into account !

Enabled

Allows disabling the norm without deleting it. Not deleting a norm allows it to be included with values assessed earlier, when the norm was still enabled.

Station

If specified, the norm will only be applied when the property is assessed on this station.   

When GLIMS evaluates which norms are applicable, the following logic is applied: as soon as 1 norm is found for the station on which the result is assessed, only norms specific for this station will be considered.  Other (not station-specific) norms will no longer be taken into account !

Reference type

Allows you to specify the reference type for the norms defined for the reports (because different norms may apply to a result depending on the objective, e. g. diagnostic reference values, therapeutic reference values, target values, etc.) This option can only be specified for norms with the check-box For reporting enabled.

The Description of a reference type is a multilingual field.

If several reporting norms are defined for a property with different reference types, the reference values of each reference type are displayed for the result of this property:

  • In the result editor (in the fields Norm / Alt. norm, the presentation format is Low limit\High limit\Mnemonic of the reference type).

  • In result browsers and on reports, for the selected norm format.

If several reporting norms are defined with the same Reference type, the best matching norm (age, gender, ...) for each reference type is displayed.

Object template

If specified, the norm is valid only for objects of the given template (e.g. norm only applicable for objects of object template 'horse').  

When GLIMS evaluates which norms are applicable, the following logic is applied: as soon as 1 norm for the object template of the result's patient is found, only norms specific for this object template will be considered.  Other (non object template-specific) norms will no longer be taken into account !

Object attribute

If specified, this norm will only apply to objects for which this object attribute is defined.

Attribute time limit

A number representing the attribute's time limit and expressed in the specified unit.

Note

Make sure that the Day unit is specified in the general option. If the object attribute is for example Pregnancy, the Attribute limit could be set to 2 and Unit to Month. The norm would then only apply to women pregnant for two months.

Study

If specified, this norm will only apply to results belonging to this study.

When GLIMS evaluates which norms are applicable, the following logic is applied: as soon as 1 norm for the study of the result's patient is found, only norms specific for this study will be considered.  Other (non study-specific) norms will no longer be taken into account !

Tip

To enhance the performance of study-specific norms:

  • For each study where you need study-specific norms, create an object template with the same name as the study mnemonic.
  • When selecting a study in the norm editor, the norm will automatically also refer to the corresponding object template.

Episode

Allows defining different norms depending on the study episode of the order.

Eligibility

References a Result-based, Logical-valued MISPL function evaluated on the result to be checked, to see whether or not the norm is applicable.  

Note

When executing the MISPL function, the updated result record is not yet written to the database.  This implies that start table methods like .NumericValue() will NOT work!

The only Result fields that can be relied upon in these MISPL functions are RawValue and .AvailabilityUser

Maximum age

If specified, the norm will only apply to patients not older than the specified age.

Sex

If specified, the norm will only apply to patients with the specified sex.

Severity

If the option For severity is enabled, this value will be used to set the result's norm severity (if the result falls outside the norm limits). 

You can enter a number ranging form 0 to 999.

Example

If you specify 1 as severity, then all results within the norm limits, will have severity 0. So, you can specify several norm ranges which each correspond to a severity scale.

Limits

The low and high limit of this norm. The severity will be applied below the low limit and above the high limit.

Notes

  • If the option For reporting is enabled, these are also the values which will be displayed on the report (unless you specify a Report text, see below).
  • Norms that are used for reporting and which do not have their limits specified are not taken into account when determining whether or not a result value is normal / abnormal.

Internal comment

Result-based multilingual message that is shown when the norm is violated.

External comment

Multilingual comment that is printed on the report when the norm is violated.

Report text

Text that will be shown on reports instead of the Limits.

For severity

If enabled, the Severity defined for the norm will be used to set the result's norm severity and can hence influence the final severity of the result.

Note

The Auto-confirm severity and Auto-validate severity defined in the Assessment method are compared with the result's final severity to decide whether the result can be automatically confirmed / validated.

For phone list

If enabled, the Severity defined for the norm will be used to set the result's phone list severity.

Note

The Phone severity defined in the Assessment method is compared with the maximum of the result's phone list severity, delta norm severity and manual severity to decide whether the result should be marked as To be phoned.

For reporting

If enabled, the norm's low and high limits can be printed as reference values on reports.

Norm format

The general option Norm format allows defining the format in which reference value ranges are shown in result browsers and reports.

  1. Click Start - System management -> Customize -> GLIMS general
  2. In the Results tab page, you will find the option Norm format

  3. Double-click the option to open a separate window that provides a visual example for each value.

Examples

Example 

Packed                :|LLLL-LLLL    |
(width: dynamic)       |<LLLL        |
                       |>LLLL        |

Justified             :|LLLL  -  RRRR|
(width=13)             |<        RRRR|
                       |>        RRRR|
Note that in some languages '>','<' is replaced by the words 'above', 'below'
Left-aligned          :|LLLL   - LLLL  |
(width=15)             |       - LLLL  |
                       |LLLL   -       |

Right-aligned         :|  RRRR -   RRRR|
(width=15)             |       -   RRRR|
                       |  RRRR -       |

Centered              :|  RRRR - LLLL  |
(width=15)             |       - LLLL  |
                       |  RRRR -       |

Justified brackets    :|( LLLL  -  RRRR )|
(width=17)             |( <        RRRR )|
                       |( >        RRRR )|

Note that in some languages '>','<' is replaced by the words 'above', 'below'

Left-aligned brackets :|(LLLL   - LLLL  )|
(width=17)             |(       -  RRRR )|
                       |(  RRRR -       )|

Right-aligned brackets:|(  RRRR -  RRRR )|
(width=17)             |(       -  RRRR )|
                       |(  RRRR -       )|

Centered brackets     :|(  RRRR - LLLL  )|
                       |(       - LLLL  )|
                       |(  RRRR -       )|

where RRRR stands for a right-aligned limit and LLLL stands for a left-aligned limit.

Notes

  • The Centered hyphen formats distinguish themselves from the other formats primarily because of the fact that the range delimiter (-, > or <) is always printed in the middle of the column and that a format ' < 200' or ' > 10' is used as opposed to ' - 200' or '10 - ' in the existing formats. There is also no use of words like 'above' or 'below' instead of the '>' and '<' symbols.
  • When there is not enough room, the brackets may be dropped. When the formatted norm is still too wide, it will be replaces by ?????' in order to avoid mistakes.
  • When a norm text is specified via the Report text field, it is still used exactly as defined: there is no formatting or aligning in this case.
  • Changing the report format will only have an immediate effect for the user who makes the change. Other users have to leave GLIMS and login again.

  • For reference values with a Reference type, the reference type Description is displayed in brackets. If the Description is not indicated, the Code is displayed. If the Code is also not indicated, the Mnemonic is displayed.

Result norm translation in Online and Variable text reporting

The following norm values entered via the Report text field will be interpreted as numeric norms in URL and variable text reporting:

'- y'

Low = ?

High = y		  'x -'

Low = x

High = ?		 'x - y'

Low = x

High = y		 '- -y'

Low = ?

High = -y		 '-x -'

Low = -x

High = ?
'-x - y'

Low = -x

High = y		  '-x - -y'

Low = -x

High = -y		 '> a'

Low = a

High = ? 		 '< b'

Low = ?

High = b

 

Notes

  • Limit format is '[+-] 9..9[.9..9] [e9..9]' or ' < 9..9[.9..9] [e9..9]' or ' > 9..9[.9..9] [e9..9]'.
  • Spaces are ignored in the norm limits.
  • If the preceding conditions are not met for the text norm limits format, the Low and High value will be ? and the norm limits will remain unchanged.

Warning

VariableText reporting: when the norm low or high limit is unknown, it will be put into the w_Result temp-table as ?. In the ReportTemplate.Body MISPL, each occurrence of .Low and .High should be preceded by 'IfKnownString(...)'.
