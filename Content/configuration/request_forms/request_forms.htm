Request form

Contents: Introduction - Configuration - Usage in order entry

Introduction

During order entry, it is possible to use request forms to add requests to an order. The chapters below explain how you can configure these request forms and what functionality they offer in the order entry screen.

Configuration

Request forms can be created by clicking Start > Configuration > Request definitions > Request forms. A browser will be displayed in which you can create new forms.

The content of the created request form can be defined by selecting Edit in the contextual menu / ribbon.

The tree editor allows to create tab folders (parent nodes without request definition) and underlying request definitions. When adding or changing a request form, the lookup browser for the request definition (called via Ctrl-F or Double-Click) has an options button which allows searching via the description of the request definition (e.g. the property name for property request definitions). In this search you can use wildcard (e.g. *blood* to see every request definition where the word "Blood" appears in the description)

You can copy request forms by selecting the contextual menu function Copy on the request form browser.  The copy function uses the standard Windows naming for copied records:

"Copy of ..."
"Copy (1) of ..."
"Copy (2) of ..."
...

Support for issuer-specific request definitions in request forms

The request form user interface for order entry supports the issuer-specific request definitions (where Requestable.Issuer has been filled in). To cope with this, the communication between the order entry program and the request form takes place entirely via the request definition mnemonic.

This implies that, even when a non-specific request definition is present in the request form definition, the issuer-specific request definition with the same mnemonic will be requested when this request definition is selected in the request form.

Warning

To avoid confusing situations, we strongly recommend NOT to put issuer-specific request definitions in the request form definition. Use the non-specific request definition instead.

Usage in order entry

Selecting the request form

On the Request page of the order entry screen, you can click the request form button.  A new window will be displayed in which you can select the request form that you would like to use.  By default, the request form specified in the Request form field of the order entry options will be shown.  If no request form is defined in the order entry options, the request form specified in the department of the current user will be used.  Moreover, if the option "Req. form is fixed" is enabled in the department, the user will only be able to use the request form specified in the department and no other.  Other defined request forms will not be selectable.  If this option is not enabled, the last used request form will be shown when order entry is called again.

Selecting a specimen in the request form

The Specimen spinner allows the user to:

  • always know which specimen requests apply to without having to look back at the order entry screen.
  • cycle through the requested specimens using the up and down arrow keys.
  • filter the request form contents, to show the requests referencing the selected specimen.

This field mirrors the Specimen field in the order entry screen: selecting a specimen in the Request form will select the same specimen in the order entry screen as well.

Selecting tests in the request form

The tab pages of the request form contain toggle boxes for every test. To select or deselect a request, simply use the selection mouse button. Clicking a request definition in the request form automatically updates the request definition in the embedded browser of the request page. This window synchronization does not only allow creation of request definitions, but assigns the object date and time, urgency request attribute (cito, phone, ...) and billing mark too, if any are specified.

Ignoring the current specimen selection

The Ignore option forces the Specimen field to be set to "?" so that the currently selected specimen is ignored. Activating Ignore has the effect that no duplicate materials can be selected and requests are not requested onto a material, meaning the scheduler will choose the specimen.

Examples

Scenario A

  1. Request properties by ticking property or panel checkboxes.

  2. Request a material by ticking a material checkbox.

    • Ignore current specimen selection enabled: all previously selected items remain selected.
    • Ignore current specimen selection disabled: all previously selected items become deselected.

Scenario B

  1. Request a material by ticking a material checkbox.

  2. Request another material by ticking another material checkbox.

    • Ignore current specimen selection enabled: the first selected material remains selected.
    • Ignore current specimen selection disabled: the first selected material becomes deselected.

Warning

Enable the option before adding any requests. Adding a request, enabling the option and then adding or removing a request is not supported.

Selecting a confirmable panel request definition

If the selected request definition is a request definition of type Panel whose Confirm option is activated,

  • It is displayed in italics and cannot be deselected in the request form (1).
  • The only way to undo the selection of this request definition is to delete all its members in the request list of the order entry screen (2).

Selecting tests with a specific urgency

If you have defined urgencies, you can request a test with a given urgency.  There are several ways to specify the urgency of a request:

  1. One can first change the urgency field at the top of the screen and then click the request definition.  

  2. You can use the right-button mouse and select the urgency from the main menu.  This can be used to directly add a request with a specific urgency or to change the urgency of a request.

  3. When you select an urgency at the top of the screen and then click the Apply button, the selected urgency will be applied to all currently selected request definitions.

The urgency short name will be displayed in front of the request definition name and the request definition name will be displayed in red. 

Note

For each request definition, you can define a minimal and maximal urgency.  If a minimal urgency is specified for a request definition and you add this request definition using the request form, the minimal urgency level will automatically be applied.
