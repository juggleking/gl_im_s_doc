Choice lists

Contents: Introduction - Configuration - Usage - Copy - Merge

Introduction

GLIMS features a general concept of choice lists and corresponding choices. A choice list simply defines the existence of a list of possible values, while the choices define the possible values themselves.  

Example

A typical example would be a choice list for properties with Data type set to Enumerated.  As result value for such a property, one of the choices defined in the choice list linked to this property can be chosen.

Configuration

Choice lists

The choice list browser is available from the main menu via Start -> Configuration -> Choice lists.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

Full name of the choice list.

Allow free text

Whenever a choice list is used in GLIMS, the user will have to choose one of the choices defined for that choice list.  However, if this option is enabled, the user will also be allowed to specify free text instead of one of the pre-defined choices.  A confirmation message will still be displayed though.

Note

In case of a regular result, the Property option Strict result checking still decides if the user gets the chance to store the result as is.

Alternative choice configuration table

Possible values: 

  • Microorganisms
  • Blood antigens
  • HLA antigens
  • HLA antibodies
  • Antibiotics
  • Reagents
  • Disorders

When e.g. a property is linked to a choice list for which the alternative table is specified, the possible result values for this property will be: 

  • One of the records from the specified alternative table
  • A result code

Note

If an alternative table is specified, the choices defined for the choice list will no longer be considered valid result values. They will therefore not appear in routine dropdown lists.

However, if the Property option Strict result checking is not activated, entering theses choices manually will still be possible.

Severity

Severity applied as norm severity to results whose value is not part of the choice list. If this field is empty, the norm severity applied to the results is 0.

Note

This severity is only applied for properties with Data type set to Enumerated.

Choices

Once you have defined a choice list, you can now define the actual choices in this choice list.  These will be the choices that a user can choose from wherever the choice list is used in GLIMS.

Select a Choice list and use the contextual menu / ribbon item Choices.

Mnemonic

Used for selecting a choice by directly typing its mnemonic (or an unambiguous part of it).

Name

Internal name primarily used to be shown to the technician (e.g. when he calls a list to the screen in order to pick a choice).

Description

External multilingual description used on outbound reports.

Severity

If a choice list is used to define possible values (choices) for a property (with Data type set to Enumerated), the severity of the chosen choice is applied to the result.

Notes

  • This severity is only applied for properties with Data type set to Enumerated!
  • If the value entered does not belong to the choice list, the severity of the choice list is applied to the result.

Enabled

Indicates whether this choice will be selectable.

Usage

Choice lists can be used whenever you want the user to make a selection between a number of pre-defined choices.  The same choice list can be used for multiple situations, e.g. several properties can share a choice list, which reduces maintenance.

Choice list can currently be used for:

  • Properties with Data type set to Enumerated:  the choices can be selected as result value.
  • Micro-organism tests: the choices can be selected as isolation test value.
  • Medium tests: the choices can be selected as carrier test value.
  • Media: the choices can be selected as carrier comment.
  • Study conclusion and study object role: for more information, please check the chapter Studies.

When entering a value in the cases listed above, you can double-click or use CTRL-F to obtain a list of available choices (and possibly result codes).

Choice or result code: which has priority ?

Default: result code has precedence

By default, any result entered is first checked against the result codes valid for this property. This means that result codes have precedence over property choices of enumerated results.

Option "Choices have precedence"

If the Specific site field Choices have precedence is enabled, any entered result is first checked against the property choices for an enumerated property. When the entered result matches with a property choice, it is selected and the result code table is not checked any more.  

Note

This option only works for properties with Data type set to Enumerated.  The result code selector (accessible by hitting CTRL-F in a result entry field) takes the precedence setting of choices and result codes into account.

Warning

Please carefully consider the possible side effects before activating this option! Remember that for instance a choice match is detected also when the entered result unambiguously matches the start of one of the choices, while for a result code an exact match is required.

Alternative table

When using the Alternative table field of a choice list (see above), the defined choices will no longer be used. 

Copy

From a choice list, select the contextual menu / ribbon item Copy to copy a choice list and all its choices.  You can use this function on several choice lists simultaneously.

Mnemonic

Specify a name for the new choice list.  If not specified, GLIMS will take the mnemonic of the original choice list and add #1, #2, ...

Merge into another choice list

This function allows to merge (replace) a selected choice list into another choice list.  Note that this is a single-record function, i.e. you can only merge 1 choice list into another one.

Target list

All choices of the first selected choice list will be copied to the choice list specified in the field Target list.

Replace references

If set, references (e.g. from properties, micro-organism tests, etc) to the old choice list will be re-directed to the new choice list.
