Request codes

Contents: Introduction - Fields - Options - Link request definitions - Order entry

Introduction

Request codes are codes that can be used to request something during order entry.

What is actually requested is determined by the request definitions that are linked to the request code. These request definitions can depend on issuer and time. So a request code can have several request definitions and a request definition always refers to a request code.

Request codes are available from the main menu via Start -> Configuration -> Request definitions -> Request codes.

Tip

When creating a property, material, etc GLIMS will automatically create a default request definition and corresponding request code.

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Used to enter the request code during order entry.

Name

Full name of the request code. Can refer to the name of the property, material, ... which is requested with this request code.

Enabled

A request code must be enabled before you can request it during order entry.

Explicitly requestable

If enabled, the request code can be requested explicitly, i.e. added manually, during manual order entry.

By disabling this option, you can prevent a specific request code from being requested manually (unless it is part of a manually requested panel). Such a code can still be requested electronically or via MISPL.

Note

When users tamper with a requested panel (i.e. delete one or more members, for instance), the individual panel members are requested but the panel itself is not. In that case, non-explicitly requestable request codes that are members of the tampered panel are nevertheless added to the order.

Electronically explicitly requestable

If enabled, the request code can be requested explicitly during electronic order entry.

By disabling this option, you can prevent a specific request code from being requested electronically (unless it is part of an electronically requested panel).

Note

The value of this field is currently only used during the LCSD export so that external systems communicating with GLIMS are aware of the request codes that can be electronically requested. Order entry does currently not check the Electronically explicitly requestable setting of an electronically received request code.

Universal

Automatically set by GLIMS, for internal program use only.

Definitions

Shows the request definitions that are linked to this request code.

Query options

The Options button in the Request code browser allows to narrow down the number of request codes by specifying criteria for the request definitions linked to the request codes.

Types

Only show request codes which have at least one request definition of this type.

Name pattern

Only show request codes which have at least one request definition with this name pattern.

Issuer, Check

GLIMS allows to define issuer-specific request definitions. The Issuer and Check fields allow to take this characteristic into account when looking up a request code.

You can specify the issuer and the relation between the issuer and the request definition:

None

The issuer is not taken into account. All request codes are shown.

Available

Only the request codes with at least one request definition which can be requested for the specified issuer are shown.

Specific

Only the request codes with at least one request definition that explicitly refers to this issuer are shown.

Enabled

Yes

Only show request codes which are enabled.

No

Only show request codes which are disabled.

Note

Disabled request codes are shown in gray.

?

Show both enabled and disabled request codes.

Note

Disabled request code are shown in gray.

Explicit

Yes

Only show request codes for which the option Explicitly requestable is enabled.

No

Only show request codes for which the option Explicitly requestable is disabled.

?

Show all request codes, regardless of the Explicitly requestable setting.

Link request definitions to the request code

To determine what is actually requested when entering this request code in order entry, you must link request definitions to the request code.

There are several ways to link a request definition to a request code:

  • When creating a property, material, etc. GLIMS will automatically create a default request definition and corresponding request code.
  • The Request code editor has an embedded request definition browser. Use the Insert key to create a new request definition.
  • Select a Request code and choose the contextual menu / ribbon item Request definitions.
  • Specify the Request code in the Request definition editor.

Order entry

When entering a request code in order entry, GLIMS will

  • Check the request definitions linked to the request code.
  • Check whether a request definition exists with an issuer that matches the order's issuer.
  • If this issuer-specific request definition is valid (by checking its start and end date), it will be requested.
  • If no issuer-specific request definition is found or it is not valid, the currently valid general request definition will be requested.
