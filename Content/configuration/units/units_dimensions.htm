Units and dimensions

Contents: Introduction - Dimensions - Units - Unit conversions - Unit on reports

Introduction

All Properties with Data type set to Numeric, must refer to a Unit.

Units are grouped in a dimension.  

Example

The dimension Volume can contain the units Liter, Milliliter and Micro liter. 

Dimensions

Tip

As a Dimension requires a Base unit, but a Unit also requires a Dimension, follow the instructions below to get started. Once you have completed the initial configuration of a Dimension, you can add Units afterward.

To define a dimension:

  1. Click Start -> Configuration -> Units -> Dimensions.

  2. Press the Insert key and specify a Name for the dimension.

  3. Choose the contextual menu / ribbon item Units.

  4. GLIMS will ask to insert the dimension. Click Yes.

  5. The Unit browser will be shown. Use the Insert key to define units for this dimension.
  6. Return to the Dimension window and specify one of the units as Base unit. The Base unit is used to convert results between to units belonging to the same dimension.

Units

Units can be accessed:

  1. via the main menu: Start -> Configuration -> Units -> Units.
  2. via the contextual menu / ribbon item Units starting from a Dimension

Name

Can be used to e.g. specify the UCUM code (Unified Code for Units of Measure).

Dimension

Dimension to which the unit belongs.

Coefficient

Multiplier that will be used to convert a result into the Base unit of the dimension.  

Tip

The Coefficient of the Base unit must be set to 1.

Term

Term that must be added to convert a result into the Base unit of the dimension.

Tip

The Term of the Base unit must be set to 0.

R unit x Coefficient unit + Term unit = R base unit

Description

Multilingual description of the unit. If not specified, the Name will be used.

Biorad code

Please check the chapter Quality control: Biorad.

Unit conversions

Introduction

GLIMS always stores results in the unit defined for the property.  GLIMS can convert results from one unit to another unit in the following cases:

Analyzer sends results in another unit than the property unit

Analyzers can send results in another unit than the one specified in the Property configuration. If so, GLIMS can automatically convert the result value to the unit defined in the Property configuration.

In the Assessment method configuration, you can specify an alternate unit that is used by the analyzer.  When the analyzer sends results to GLIMS in that alternate unit, the values will automatically be converted into the unit defined for the property.

Tip

Property unit and the Station unit belong to the same Dimension (see below)

GLIMS will convert the results.

Property unit and the Station unit belong to a different Dimension (see below)

GLIMS will not convert the results. You must use the Conversion field in the Assessment method configuration.

Manually entering a result in a work list

GLIMS allows to generate a work list for an analyzer in which the results are typed in manually.

In that case, you must indicate if the typed in results are expressed in the analyzer unit or the property unit. To do so, use the field Convert unit if manual in the Assessment method.

Enabled

GLIMS assumes that the user enters the values in the unit of the analyzer and the values will be converted.

Tip

Property unit and the Station unit belong to the same Dimension (see below)

GLIMS will convert the results.

Property unit and the Station unit belong to a different Dimension (see below)

GLIMS will not convert the results. You must use the Conversion field in the Assessment method configuration.

Disabled

GLIMS assumes that the user enters the values in the unit of the property and no conversion will be applied.

HC provider wants results to be reported in an alternate unit

Alternate unit in Property configuration

An Alternate unit can be specified in the Property configuration.

HC provider preference

For each individual HC provider, The Unit preference field allows to indicate if results should be reported in the Base unit or Alternate unit.

None

GLIMS will use the unit as defined by the general option Default unit preference (Start -> System management -> Customize -> GLIMS general).

Base

GLIMS will use the unit defined in the Unit field of the Property.

Alternate

GLIMS will use the unit defined in the Alternate unit field of the Property.

Base and Alternate, Alternate and Base

Allows to report a result in both units.

Note

We recommend not to use this option as electronic reports and GLIMS Report builder do not support this. Instead, you can you define a second (calculated) property with the alternate unit.

Tip

Property unit and the Station unit belong to the same Dimension (see below)

GLIMS will convert the results.

Property unit and the Station unit belong to a different Dimension (see below)

GLIMS will convert the results.

Conversion between units of the same dimension

Formula

GLIMS can convert between units of the same dimension by using the unit Coefficient and Term.

Coefficients and Terms should be set up so that results expressed in two different units of the same dimension, multiplied with their respective coefficients and added to their respective terms, are equal.  

R unit x Coefficient unit + Term unit = R base unit

Base unit

For each Dimension, a Base unit must be specified. A Base unit must have Coefficient "1" and Term "0".  This base unit will be used as reference for the unit conversions.

Conversion between units of different dimensions

GLIMS can also convert between units that belong to different dimensions. GLIMS uses a concept of starting dimension and target dimension.

Conversions using

  1. Select a Dimension (starting dimension).
  2. Choose the contextual menu / ribbon item Conversions using.
  3. A browser will be shown of inter-dimension conversions having the current dimension as input (starting dimension).

  4. Use the Insert key to define a new inter-dimension conversion:

    Output dimension

    The target dimension for which the conversion will be applied (in example above it is Weight).

    Coefficient, Term

    Ensure that the following logic applies:

    Result.Value(DimensionOut.BaseUnit) = Result.Value(DimensionIn.BaseUnit) * DimensionIn.Coefficient + DimensionIn.Term

    In other words, the Coefficient and Term make sure that the results expressed in their alternate units, multiplied by their coefficients and possibly added / subtracted with their respective terms match.

    Input material

    If specified, the conversion rule will only be applied to results for properties which are executed on this material. If not specified, the conversion applies irrespective of any material.

    Output property

    If specified, the conversion rule will only be applied to results of this property. If not specified, the conversion applies to any property expressed in either of the two dimensions.

Conversions yielding

  1. Select a Dimension (target dimension).
  2. Choose the contextual menu / ribbon item Conversions yielding.
  3. A browser will be shown of inter-dimension conversions having the current dimension as output (target dimension).
  4. Use the Insert key to define a new inter-dimension conversion. See above for details.

Unit on reports

The unit displayed on reports is determined as follows:

  1. If the Property option Report raw unit is enabled, results for this property will be reported in the raw unit (Result.RawUnit), i.e. the unit which was specified when the result was entered.
  2. If the report target is a HC provider, GLIMS will check the HC provider option Unit preference (see above).
  3. If the report target is not a HC provider or the HC provider option Unit preference is set to None, GLIMS will use the unit as defined by the general option Default unit preference (see above).
  4. If the general option Default unit preference is set to None, GLIMS will use the unit defined in the Unit field of the Property.
