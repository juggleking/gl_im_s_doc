Severities

Contents: Introduction - Patient results - QC results - Isolations - Configuration tips

Introduction

In GLIMS, patient results, QC results and isolations can receive a severity. The chapters below describe how these severities can be assigned and explain the possible consequences of such a severity.

Patient results

Severity assignment

A patient result can have different types of severities. The final severity - Result.Severity - of a patient result is calculated as the maximum of Result.NormSeverity, Result.TechnicalSeverity, Result.DeltanormSeverity, Result.StationSeverity, Result.QCSeverity and Result.ManualSeverity.

Note

The Result.PhoneSeverity is NOT considered when calculating the final Result.Severity ! Please check Severity-based actions for more information.

Technical severity / Norm severity

When the value of a patient result is outside the norm range, its Technical severity will inherit the Severity defined for the norm if the Norm option For severity is enabled.

When the value of a patient result is outside the norm range, its norm severity will inherit the Severity defined for the norm if the Norm option For reporting is enabled.

The Norm severity of a Result is

  • set to 0 when the result value lies within the reference range of the reporting norm,
  • equal to the Technical severity when the result value is outside the reference range of the reporting norm.

Example

Configured norms

  • Norm 1: For reporting, Low limit = 10, High limit = 20, Severity 1
  • Norm 2: For severity, Low limit = 12, High limit = 30, Severity 5

Result values and their severities

  • Result value 11: Technical severity = 5, Norm severity = 0 (norm for reporting is not violated).
  • Result value 25: Technical severity = 1, Norm severity = 1.
  • Result value 35: Technical severity = 5, Norm severity = 5.

Note

  • If the patient result is entered through a choice, the severity defined in the choice configuration is applied as the result's norm severity.
  • If the patient result is entered through a result code, the severity defined in the result code configuration is applied as the result's norm severity.

Delta norm severity

When a delta check fails, the patient result will inherit the Severity defined for the delta norm.

Note

In the assessment method, there is an option Delta norm has precedence. 

If set, and a delta check is performed, then the delta norm severity overrules the norm severity for the auto-confirm/validate decisions (see below).  So, a result that is pathological (norm severity > 0), and that normally wouldn't be auto-validated, might still be auto-validated because it's similar to the previous result of the same patient.

The actual severity that is compared against the auto-confirm and auto-validate levels, will then be the final result severity (Result.Severity) without the norm severity component (Result.NormSeverity). So, it is calculated as the maximum of the absolute values of all severities, EXCEPT the norm severity.

Station severity

A result sent by an analyzer can be accompanied by a flag. The station flag configuration in GLIMS allows to define a Severity for each flag. 

The result will inherit the Severity of the station flag.

Manual severity

The manual severity of a result can be set:

  1. By zooming into the result detail screen and entering a value in the Manual severity field.
  2. By entering a Manual severity in the Result Modify screen.
  3. By using the Result-based MISPL function SetManualSeverity

Note

This MISPL function allows to set the manual severity for a result to any value.  When specifying a manual severity in the Result.Modify screen, it will have precedence over the value that could possibly be set by the Property trigger MISPL. When specifying an unknown manual severity, the MISPL value will be preserved.

Phone list severity

A result can also have a phone list severity which is used to decide whether a result should be marked as To be phoned.

When the value of a patient result is outside the norm range, its phone list severity will inherit the Severity defined for the norm if the Norm option For phone list is enabled.

Note

  • If the patient result is entered through a choice, the severity defined in the choice configuration is applied as the result's phone list severity.
  • If the patient result is entered through a result code, the severity defined in the result code configuration is applied as the result's phone list severity.

Consulting the severities of a patient result

  1. All patient result severities can be consulted in the detail screen of a result.

  1. In many result browsers (order review, confirmation browser, phone list browser), a result flag column shows the highest severity.  
  2. In work lists, results with a severity greater than zero are automatically highlighted.

Severity-based actions

Automatic confirmation and validation

In the assessment method configuration, you can define the highest severity allowed for auto-confirmation and auto-validation.

If the Result.Severity (calculated as the maximum of Result.NormSeverity, Result.TechnicalSeverity, Result.DeltaNormSeverity, Result.StationSeverity, Result.QCSeverity and Result.ManualSeverity) exceeds these limits, the result will not be auto-confirmed/validated.

Note

If the Auto-confirm if manual option is NOT checked, manually entered results will NOT be subjected to the automatic confirmation rules (i.e. they are immediately marked as Result.NeedsManualConfirm = YES).

Warnings

In the assessment method configuration, you can define a warning severity.

If the Result.Severity (calculated as the maximum of Result.NormSeverity, Result.TechnicalSeverity, Result.DeltaNormSeverity, Result.StationSeverity, Result.QCSeverity and Result.ManualSeverity) exceeds this limit, a warning will be shown on screen when entering a result.

Marking a result as "to be phoned"

In the assessment method configuration, you can define a phone severity.

The Phone severity defined in the Assessment method is compared with the maximum of the result's phone list severity, delta norm severity and manual severity to decide whether the result should be marked as To be phoned.

Other actions

In addition to the fixed severity-based actions described above, you can also access the severity of a patient result be means of MISPL and take appropriate measures. 

Examples

  • You can add a mark next to a result value on printed reports if the result has a given severity (by using the result mark / mask fields in the report template).
  • The color manager allows to display patient results in different colors depending on their severity/status

Isolations

The concept of severities also exists in the microbiology module of GLIMS.

Severity formula

Severities can be assigned to isolations using a site function in the field Formula to calculate the severity of the micro-organism configuration. This field allows to specify an Isolation-based, integer-valued MISPL function. The severity of a micro-organism can depend on the isolation appraisal, material, material variable, etc. If an isolation is given a positive appraisal, the severity is calculated with this formula. The formula site function is executed each time an isolation appraisal is entered, an antibiotic result's RIS report value changes (e.g. when confirming the antibiogram) or an isolation test is added or confirmed for the isolation.

The formula is an inherited field, i.e. if the formula is not specified, the system searches for the next higher parent's severity formula.

Example

IF .Specimen.Material.Mnemonic = "M_Urine"
THEN
IF .Appraisal.Mnemonic = "+++" THEN RETURN 3; ENDIF;
IF .Appraisal.Mnemonic = "++" THEN RETURN 2; ENDIF;
RETURN 1;
ENDIF;
RETURN 0; /* normal */

Note

The severity of a microbiology action, infection and hospital infection is the highest severity of all its isolations.

Manual isolation severity

GLIMS allows to assign a manual severity to an isolation.  Select an Isolation and use the contextual menu/ribbon item Set manual severity.

When changing the manual severity, the severity formula mentioned above will be executed and can take the manual severity into account in order to calculate the final isolation severity.

Configuration tips

In order to allow fine-tuning your severities later on, we recommend to leave some room in between the severity values.  

If you would define 3 different norms for a property and assign the severities 1, 2 and 3, you may have difficulties later on when you want to add a new norm in between the existing norms.  

A better choice for the severities would be 10, 20 and 30 (so you can add a new norm with severity 15 or 25 later on, without having to change the severity of an existing norm).
