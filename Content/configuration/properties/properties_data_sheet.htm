Properties - Datasheet

Contents: Introduction - Output options - Content

Introduction

The property Datasheet function provides you with detailed information about one or more selected properties or request definitions. 

Although the function is available as contextual function on the Property table and on the Requestable table, these functions are identical.  When calling the data sheet for a request definition, the provided data sheet will be the one from the corresponding property.  The data sheet can also be shown for panel request definitions.  In that case, all the data of the panel, the panel members and the different (panel) tariffs are displayed.

Options

Select a Property / Request definition and choose the contextual menu / ribbon item Datasheet.

Format

The data sheet can be printed (with or without preview) as plain Text, as HTML as a MS Word document or as a CSV file. 

Note

  • When selecting HTML, you will not be able to specify a printer. The file will be printed to your default Windows printer. 
  • The layout of the HTML file is defined in a style sheet (datasheet.css) which you can find in the Dat directory of your GLIMS installation.

Output to file

All data sheets can be put in one file or there can be one file per property.  To make one file per property you have to add the symbol {Mnemonic} to the file name.

Example

If you specify DataSheet_{Mnemonic}.html as file name and select the properties Na and K, two files will be created:

  • DataSheet_Na.html
  • DataSheet_K.html

Content

Content when output is Text, HTML or Microsoft Word

General

General information about the property as e.g. Mnemonic, Short name, Code, Multilingual description, Unit, Alternate unit, Data type etc.

Material

Material information as e.g. Mnemonic, Multilingual description, etc.  The data sheet shows:

  • The material specified in the Property record.

  • The preferred material.

    Tip

    The preferred material is determined as follows:  Property -> Assessment method -> Property output -> Procedure output -> Procedure -> Procedure input -> Material input -> Material (with lowest PropertyOutput.SchedulePreference). 

    When determining the preferred material, GLIMS also checks whether the material input record is active. 

    If there are several preferred materials, the one with the lowest ProcedureInput.SeqNo will be used.

Request definition

Request definition information as e.g. Issuer, Template, Panel membership, Tariffs, Price, Billing code group membership etc.

Norms

A Norm table displaying the station, object template, severity, object attribute, period, sex and age for each norm definition.

Delta norms

A Delta norm table displaying all relevant delta norm characteristics (Severity, maximum lapse, pivot, etc.).

Procedures of active assessment methods

This chapter gives an overview of the procedures linked to the active assessment methods of the selected property / request definition.  For each procedure, the most relevant information is included: name, code, station, department, work list template, SOP, etc.

GLIMS also shows the preferred assessment method, which is determined as follows:  Property -> Assessment method -> Property output -> Procedure output -> Procedure -> Procedure input -> Material input -> Material (with lowest PropertyOutput.SchedulePreference).

Classifications containing property

This chapter gives a list of property classifications that contain the selected property.

External property codes 

Property codes as defined in coding systems.

Content when output is CSV

When the output option "CSV" is selected, a Comma Separated Value file will be created which can be edited with e.g. MS Excel.  It will have the following content:

  1. Property mnemonic
  2. Property description
  3. Minimal size (or 'Ansatze' attribute): Property site attribuut with label 'Menge' or if no such site attribute is defined: MaterialInput.MinimalSize + PropertyOutput.Consumption
  4. Material (preferred): determined as follows:  Property -> Assessment method -> Property output -> Procedure output -> Procedure -> Procedure input -> Material input -> Material (with lowest PropertyOutput.SchedulePreference).
  5. Destination (or 'Bestimmung' attribute)
  6. Turnover (or 'Menge' attribute)
  7. Datatype
  8. Unit
  9. Age
  10. Attribute
  11. Period
  12. Norm lowlimit (Male)
  13. Norm highlimit (Male)
  14. Norm lowlimit (Female)
  15. Norm highlimit (Female)
  16. Method: method of the preferred assessment method: the latter is determined as follows:  Property -> Assessment method -> Property output -> Procedure output -> Procedure -> Procedure input -> Material input -> Material (with lowest PropertyOutput.SchedulePreference).
  17. Station name
  18. Analysegroup=department: Procedure -> Department.Name
  19. Reported decimals
  20. Workplace name (or 'Stockwerk' attribute):  'Stockwerk' attribute or if not defined use the Procedure.Station.Workplace.Name
  21. Procedure responsible: Procedure -> Responsible: user last name + first name   
  22. Price
  23. Billing code: billing codes assigned to the property's default request definition. Only billing code assignments that are valid at the time of generation will be taken into account.
  24. Reimbursement class: reimbursement classes with mnemonic MI-II,MIII,MI,MII,M1-2,M3 or M2 have priority.  If not found the first existing will be used.
  25. Discount class

Note

With respect to norms, one line is outputted per age, attribute and period combination.  Only enabled norms with severity=1, object template=?, assessment method=? and eligible=? are exported.
