Properties - Replace

Introduction

The goal of this function is to replace all request definitions referring to a selected property with request definitions referring to the New property. The end effect is that the users can keep using the old request codes, while the system actually requests the new property instead.

GLIMS will close the existing request definitions using the specified Date and create a new request definition referring to the new property with a start date equal to the specified Date.

This ensures that from that date on, all usage of the original request definition will result in the new property being requested rather than the old property. The newly created request definition inherits all tariff and accounting class records of the original request definition, as well as its request definition variable settings.

Options

New property

Request definitions that refer to the selected property (old property) will be closed and new request definitions will be created that refer to the New property.

Date

Date that will be used to close the existing request definitions and start the new request definitions.
