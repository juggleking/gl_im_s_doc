Properties

Contents: Introduction - Fields - Update of data type & unit - Visualization

Introduction

In GLIMS, a Property corresponds with a test that can be executed in the laboratory. The list of properties can be accessed from the main menu via Start -> Configuration -> Properties.

Tip

To complete the configuration of a Property, you first need to define related data such as Units, Norms, etc.

Fields

General

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Description

Multilingual description. Can be used on outbound reports.

Short name

Appears on work lists, validation screens, etc.

Domain

Used to group a number of properties. E.g. group all properties by a specific domain (Hematology, ...).

Code

Used in places where space is limited.

Biorad code

Please check chapter Quality control: Biorad.

Value entry

Data type

Numeric

Numeric properties require a Unit in which the result is expressed.

String

For textual results, including Rich text results if the option Rich text is activated.

Enumerated

The result will be selected from a list of possible values. For more information, please check the chapter Choice lists.

Titer

A result with value 800 will be displayed as 1:800.   

Tip

An alternate separator character (e.g. "/", like in "1/8") can be specified in the general option Titer separator (Start ->System management -> Customize -> GLIMS general).

Note

Properties of data type Titer are always treated as standard Numeric results in GLIMS.  The prefix is only added for visualization in result browsers or when reporting. This means that:

  • A titer property can be used as input for a formula (in PropertyOutput.Value).
  • When updating a titer result in the confirmation / validation screens, the result cell will change from e.g. 1:800 to 800 on entry.

Culture

For results that will report a microbiology examination. Such properties are to be specified as Report property in the microbiology procedure configuration.

Antibiogram

For results that will report an antibiogram. Such properties are to be specified as Antibiogram property in the microorganism configuration.

Image

For results of which the value is an image. See Image results.

Graph

For results of which the value is a graph. See Graph results.

Foreign

For results that do not belong to one of the above-mentioned data types.

Genetics

For results that will report a genetic procedure. Such properties should be set as Report property in the Genetic procedure editor.

Genetic List

For results that will make it possible to insert locus and variant results in the genetics Results screen.

Choice list

Please consult the chapter Choice lists.

Diagnosis code system

Used in the anatomic pathology module.

List type

Indicates the type of list when the Data type is set to Genetic list.

Allow multiple result codes

If enabled, the results of this property can contain multiple result codes. This means that every word of a result is checked to see if it is a result code.

Note

When a result contains multiple result codes, the highest result code severity is used to set the result severity.

The other fields of the result codes (Action, Assessed, To charge) are not taken into consideration when there is more than one result code.

Allow general result codes

If disabled, the 'general' (or non property-specific) result codes cannot be used for results of this property. The selection list (e.g. opened via CTRL-F) will only contain result codes specific for this property (next to e.g. the applicable choices from Property.ChoiceList of course).

Note

The Specimen.Discontinue program allows to specify a result code for the cascaded processing of the results depending on this specimen.

This program only allows to select of a general result code. However, during the processing of the results, a property-specific result code with the same mnemonic has precedence over the general result code (just as is the case during manual result entry).

Special care should be taken for properties who do not allow general result codes: when the result code specified (as general result code!) in the specimen discontinue screen does not have a property-specific counterpart for this property, the mnemonic of the result code will silently be stored as result value, but the actions attached to the result code (e.g. Validate, Discontinue, ...) will NOT be executed.

Strict result checking

If enabled, results that do not correspond to the data type (or which are not in the Result code table) will be rejected.

Unit

Unit in which Numeric results are expressed. For more information, please check the chapter Units and dimensions.

Si/Conventional

Allows to indicate whether the unit of the property is considered to be an SI unit ("International System of Units") or a conventional unit. The default value is the unknown value: "?". The database field is called "BaseUnitSystem".

The SI/Conventional information is

  • included in electronic reports. Please check the applicable driver documentation to see if the driver (J_LDTRXV3 for instance) can process this information.

  • exported in the XML data for GLIMS Report Builder.

Online period (in days)

Period in days during which results of this property should be kept on-line. Results will not be purged from the active database as long as this period has not expired.

Dynamic

Should be enabled if this is a dynamic property. It means that results obtained in one day should be grouped on reports. For more information, please check the chapter Requesting tests with a discriminator.

Informational (entered during order entry)

Indicates that the result of such a property is considered to be information passed on during order entry, without being assessed in the laboratory.

Note

Such results are considered to be 'secondary' results:

  • They are never considered as 'new' for a report.
  • They are not a trigger for report scheduling.
  • They do not influence the status of an order in the same way as a regular result.
  • The result will be automatically validated if no assessment method can be found (so result origin is unknown).

Auto prompt (asked for during order entry)

Indicates that the value of the result for such a property should be asked for during order entry (e.g. weight, length,... of patient.

Note

The value will not be prompted for if entered immediately along with the request, as in e.g. WEIGHT=75. 

Number of editor lines

Defines the size of (rich) text fields in the Specimen info screen and routine screens of the anatomic pathology, consult registration, and extended Genetics modules:

  • "0": a short text field with one line of height appears.
  • "1": a long text field with one line of height appears.
  • "2", "3",...: a long text field with two, three lines of height appears.

On phone list

Allows to exclude a property from the phone list.

Rich text

This option is relevant for the Consult registration and extended Genetics modules. If it is activated, properties with data type String are displayed as rich text boxes:

Opening the rich text editor

The rich text editor can be opened by using:

  • F6 in the rich text box or
  • the icon on the right side of the rich text box.

The Preview button in the texts browser opens the rich text preview in the large rich text editor. It is accessible via the genetics results screen, if a genetic exam contains a genetic procedure with configured text results, or via the Preview button in the texts browser ribbon (via Start > System Management > Texts).

Usage

The rich text editor offers the user a broad range of text formatting options. Moreover, entering dynamic text and MISPL expressions is supported as well.

Dynamic text

Dynamic text can be entered

  • directly via the syntax {<}. For instance: {<pos} where "pos" is the mnemonic of a Result-based Text.

  • by using the Dynamic text / Dynamic text reference options of the Insert tab.

To show an expanded preview of the dynamic text, click the Review tab -> Preview.

MISPL expressions

Result-based MISPL expressions can be entered between brackets: {=}. For instance: {=.Order.InternalId} to insert the internal ID of the order.

Customizing the default font

The default font of the rich text editor is Verdana 12. However, the default font and its size can be customized as follows:

  1. Define two gp_Site-based site attributes:
    1. Choose Start > System management > Scheme > Tables
    2. Select the table gp_Site and choose the contextual menu / ribbon item Site attributes

    3. Use the Insert key to add the following site attributes

       Site attribute 1Site attribute 2
      NameRichTextDefaultFontNameRichTextDefaultFontSize
      Data typeStringInteger
      WidgetFill-inFill-In

  2. Choose Start > System management > Customize > MATE > Site attribute to fill in a value for the newly created site attributes.

    Note

    Make sure to specify a valid font name. The font names that are listed in the drop-down list of the rich text editor are valid font names.

Reporting

The content of Rich text fields can be displayed on reports. The required configuration is described here.

Tip

  • MISPL functions that can be used to report locus and variant results for the extended Genetics module are described on the page Site functions for reporting.
  • Additionally, several preconfigured text modules are available on the general Rich text results page and can be defined for general reporting.

Rich text: Known issues

  • In the large rich text editor, the feature to create custom styles is not supported. Any newly created styles will not be saved.
  • Inserting {<txtmodule> in the HTML source text leads to invalid HTML and is known to cause issues.
  • Users should not edit the HTML source text directly. Instead, they should use the provided rich text editor.
  • For now, including rich text inside rich text is not supported. Only plain text can be used within rich text, see the point below.
  • It is expressly not recommended to use unnecessarily complex expressions in the rich text editor.

This means: the use of spaces, quotation marks, brackets, etc. within MISPL expressions like {=IfKnownString(.AllelicState.Name)}.

Example: {=IfKnownString("(" + .AllelicState.Name +")")} would not work.

Scheduling

Use available specimen

Allows to indicate that this property should only be scheduled on an explicitly confirmed specimen. This will typically be used when the test can be derived from several materials, and the scheduler should wait until a suitable material is confirmed for this order.

In this context, 'explicit confirmation' can be obtained in the following ways:

  • explicitly requesting the specimen in order entry
  • adding a specimen automatically via the regular scheduling while the order entry option Automatic sample login is active
  • scanning and either creating or confirming a specimen via the specimen receipt program
  • scanning a specimen in the specimen scan or work list generation program
  • reacting to a query or result upload message from an analyzer or sample distributor connection

Report format

Determines how the results are displayed on reports.

Columnar

The results will be displayed in columns.  The number of columns to be shown is defined in the report template.  Is required, when the Property option Cumulative is enabled.

Full width

The full width of the page will be used.  The property description will be shown on the line above.

Text

Means "Starting from the Value column, until the page border". There is no space left for any subsequent columns.

Two column text

The result value will be spread over two columns with an identical width.

Hidden

The result will be completely hidden from the report (no line for this property is generated).

The main purpose is the possibility to steer the automatic printing and completion status of the report via a property, which itself is never printed on the report.

Note

The report format is ignored when reporting results based on Rich text properties.

Cumulative

If enabled, previous results will shown on the report. The option Report format should be set to Columnar.

Report raw unit

Triggers

On result entry, On confirmation, On validation, On discontinuation

A trigger allows to automatically execute an action (MISPL expression) immediately after the corresponding result status has been reached (Available, Confirmed, Validated, Discontinued).

Tips

  • The Confirmation and Validation triggers are supported throughout GLIMS by all programs, with two exceptions:

    1. Result import programs

    2. Blocked results (i.e. requested again within the blocking period)

  • The Discontinuation trigger is primarily intended for use in the Blood Transfusion module to allow clearing the medical record in case results are being discontinued while e.g. the blood group / rhesus has already been copied to the medical record.

MISPL expression

  1. The MISPL expression defined as trigger must be based on the ResultOutput table.

    Note

    This has the advantage that the Action, Procedure, Department, etc... can easily be accessed from the trigger, but the disadvantage that no trigger can be executed if no ResultOutput record is available (see below Run property triggers when no output of action). 

  2.  The return value is of data type Logical. The return value is ignored.

Run property triggers when no output of action

Case description

  1. A validated result is modified using the result menu function "Modify".
  2. GLIMS will discontinue and repeat the result (new result will get the modified value).
  3. Problem: the "Discontinuation trigger" defined on the related property is not executed.
  4. Reason: the discontinued result is disconnected from the action and does no longer have a "Result output" record.  As the trigger is based on the "ResultOutput" table, it was not executed.

Solution

To cope with this situation, an option Run property triggers when no output of action is available in the general options (Start -> System management -> Customize -> GLIMS general).

If enabled, all property triggers will be executed even if the result has no Result output. The triggers will be executed on a temporarily created dummy ResultOutput record, referring to the result.

So the MISPL can use all Result fields and methods. Of course, when the MISPL tries to navigate to the Action or to the PropertyOutput, it will encounter Unknown values.

Update of data type & unit

During configuration it is sometimes necessary to change the Data type or Unit of a property. This is allowed, but with some restrictions:

  • The minimal User type should be System manager for properties with at least one result.

  • The update is to be confirmed twice.

    Notes

    • Confirmation messages will only appear for existing properties with related results. For new properties, or existing properties without results, the update will proceed silently.
    • In order to have logging of these actions, activate the (automatically created) Audit Property log type.

Warning

Note that this may yield invalid results! Changing the data type or unit of a property may make its results useless for statistics, delta checks, graphs, etc...

Visualization

GLIMS allows to customize how a property should be visualized in any GLIMS screen.  

Default

By default, the property's Short name, or Mnemonic if no short name is indicated, is shown.

Property description

In the User preferences (Start -> Tools -> User preferences) the option Show property descriptions allows to indicate that a property should be visualized by its multilingual description (instead of short name / mnemonic) in any GLIMS screen.
