Procedure inputs

Introduction

Procedure inputs can be accessed:

  • By selecting a procedure and choosing the contextual menu / ribbon item Inputs.
  • In the Station - Edit procedures screen.

General

Procedure

Procedure to which this input applies.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Automatically generated by GLIMS.

Seq no

The relative order of this input within the procedure.  Among other, this is used to sort the inputs on work lists.

Enabled

Indicates if the input is required for this procedure.

A procedure input can be a Material, Property or Microorganism test.

Property

Property

Used for calculations where results are used as input.

Tip

The actual calculation is defined in the Property output field Value.

Automatically request input result

Can be disabled for calculations where it is not mandatory that all input results are available.

Concept

When defining the configuration of a calculation procedure, it is possible that it is not certain that a specific property will be available to.

If this property has been explicitly requested during order entry, it should be an input of the procedure. However, if it was NOT requested explicitly, this property should not be requested automatically. To support this case, Glims offers "optional" property inputs. A "property input" can be marked as "optional" by disabling the Procedure input option Automatically request input result.

Usage

When actions are scheduled for the explicitly requested tests, specimens, etc. the property inputs that are marked with "Automatically request input result=No" will simply be ignored.

The input for the action is not created, the result is not created nor requested. After all actions have been scheduled, GLIMS will look at all property inputs with "Automatically request input result=No".

If the result for that input has already been requested, GLIMS will add this result as input for the action. This also means that the action status will take this additional input result into account just like any other input result (e.g. to calculate the action status).

Summary

In short, an "optional"property input will only be used if the result already exists (during scheduling) because of other reasons (e.g. explicitly requested, or non-optional for another procedure,...).

Example

Consider a procedure named MakeSum with

  • 1 calculated output Sum
  • 3 inputs (type Property) named Input1, Input2 and Input3

Default behavior

When you request Sum during order entry, GLIMS will automatically request Input1, Input2 and Input3 as these are required to calculate Sum.

Optional input

GLIMS allows to define that a particular input is 'optional' and should not be requested automatically when the calculated output is requested.

Assume that the option Automatically request input result is disabled for Input3. When you request Sum during order entry, GLIMS will automatically request Input1 and Input2 but not Input3.

This implies that the calculation should take into account that the result for Input3 might not be available.

Example:

return fractionaltostring( Input1 + Input2 + (if Input3 = ? then 0 else Input3 ), "%g" );

will calculate the sum of all input results, either Input1+Input2 or, if Input3 exists (explicitly requested), Input1+Input2+Input3.
