What is MISPL?

Contents: Introduction - Via the main menu - Via a specific table - MISPL editor - Difference check - Usage - History

Introduction

The name MISPL is an acronym for MIPS Site Programming Language. MISPL is a simple programming language, available to users on the site. Programs written in MISPL are maintained by site personnel. 

In most sites people wish to customize their own working environment. MISPL plays a very important part here. For example, a specimen internal id in GLIMS starts its numbering with 1 and continues endlessly. Though, most users wish to keep their proper specimen numbering, similar to YYMMDD-NNNN. This can be done with MISPL. 

MISPL can be used at many places in GLIMS. So, MISPL customizes GLIMS for the different users of different countries in most various environments (hospital laboratory, industrial laboratory, ...). MISPL can be divided in two parts: texts and expressions.

  • Texts are mainly used for report headers, labels and so on.
  • Expressions are used to make decisions, with the possibility to use program structures. In other words, MISPL enables to display data , change, calculate, format and filter data based upon the relations between database fields.

MISPL programs are stored in the database, in the table of Site functions. There are two ways to access these site functions:

Via the main menu

Click Start -> System management -> Site functions:

Search box

Only show functions containing these words in their definition.

Query options

The icon at the top left allows to narrow down the number of site functions displayed:

Table

Only functions based on the specified table.

Data type

Only functions of the specified data type.

Via a specific table

The generic method for accessing site functions is by first establishing a table context via System management -> Scheme -> Tables, then right clicking on any table record and choosing the contextual menu/ribbon item Site functions.

MISPL editor

When zooming into the definition of a site function (F6), the MISPL editor is opened.  This editor offers line numbering and color coding.  It visually indicates any syntax errors. A warning icon is shown before the applicable line number and the affected lines are underlined in red.

Use the Insert key or ribbon item to open the MISPL expression builder.

Difference check

When you change the Definition of a site function, a screen will be displayed, showing the differences between the old and new definition of the site function.

  • Lines that have been removed are shown in red with a '-' sign at the front
  • Lines that have been added are shown in blue with a '+' sign at the front
  • Lines that remain unchanged are shown in black with any sign at the front.

You must confirm the changes before the new definition is saved in the database.  This feature is aimed at preventing errors when changing vital MISPL functions.

Usage

The contextual function Usage on SiteFunction displays a browser with all existing references to this site function; i.e. all records/contexts where this site function is being referred.  

A View button is supplied to get immediate detailed access to the record referring the site function (where possible).

History

To view all previous versions of a site function:

  1. Select a site function and use F6
  2. Choose the Field history function in the contextual Tools ribbon tab.  

Old values can be reactivated by pressing the Restore button or using the contextual menu / ribbon item Restore field history.
