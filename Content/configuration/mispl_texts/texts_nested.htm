Using variables in nested text modules

In case of multi-level dynamic text modules, program-supplied variables are inherited by any lower level module, unless overridden by a parameter with the same name.

Example

Let's say a variable {Issuer} is defined by the program as "Dr.Who".

A text module 1 can look like:

"This order was issued by: {Issuer}"

Or it can use another module:

"{<Module2}"

where Module2 is defined as:

"This order was issued by: {Issuer}"

After expansion this will result in:

"This order was issued by: Dr.Who"

However, if module 1 would contain:

"{<Module2 Issuer=Mr.Spock}"

the program-supplied variable 'Issuer' will be hidden by the parameter 'Issuer' and the resulting text will be:

"This order was issued by: Mr.Spock"
