EnumeratedToString

This function converts enumerator Value, belonging to domain EnumeratorClassName, to a string.  

In the reference guide, the exact 'EnumeratorClassName' is indicated by a link.  If you click this link, you will again see the correct EnumeratorClassName on top of the screen.

StringEnumeratedToString (StringEnumeratorClassName, EnumeratedValue)

Example

RETURN EnumeratedToString("SamplingLocation",.SamplingLocation);

FractionalToString

This function converts Fractional to a character string, according to the control string Format.

StringFractionalToString (FractionalFractional, StringFormat)

An overview of some number formatting directives is given below:

Directive    Description
%E The argument is converted to decimal notation of the form [-]m.nnnnnnE[Â±]xx where the length of the string of n's is specified by the precision.   The default precision is 6.
%f The argument is converted to decimal notation of the form [-]mmm.nnnnn where the length of the string of n's is specified by the precision. The default precision is 6.
%G Equal to %E or %f, whichever results in a shorter output string; non-significant zeros are not printed.

Examples:

FractionalToString(40.3399. "%9G"); returns: <space><space>40.3399
FractionalToString(40.3399. "%7G"); returns: 40.3399
FractionalToString(40.3399. "%7.1G"); returns: <space>4E+001
FractionalToString(40.3399. "%7.3G"); returns: <space><space><space>40.3
FractionalToString(40.3399. "%9f"); returns: 40.339900
FractionalToString(40.3399. "%5.2f"); returns: 40.34
FractionalToString(40.3399. "%3.3f"); returns: 40.340
FractionalToString(40.3399. "%2.4f"); returns: 40.3399

IntegerToString

This function converts Integer to a character string according to the specified Format.

StringIntegerToString (IntegerInteger, StringFormat)
Format 
Follows the syntax %[-][+][<FieldWidth>]d. The minus sign specifies left adjustment of the converted value in the indicated field. The plus sign specifies that the result of a conversion should always begin with a sign (+ or -). If the integer field width is specified, and the converted value has fewer characters, the field is padded with blanks or, if the field width begins with a zero, with zeros.

An overview of some number formatting directives is given below:

Directive Description
%d The argument is converted to decimal notation.
%o The argument is converted to unsigned octal notation (without a leading zero).
%x The argument is converted to hexadecimal notation (without a leading Ox).
