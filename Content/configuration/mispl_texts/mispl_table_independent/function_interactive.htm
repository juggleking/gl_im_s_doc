Message

This function displays an informational message on the screen.

VoidMessage (StringMessageString)

The following three MISPL built-ins (AskChoice, AskString, AskYesNo) allow user inquiry from within a MISPL program, and allow taking decisions depending on the answer.

Use these functions with care. When called from within a critical transaction (e.g. the order internal id expression), you may lock up other users until you provide an answer.

All three functions are batch-enabled. When running in batch mode (e.g. in a trigger of an analyzer batch process), the default answer will automatically be returned. Make sure to provide a default answer for these cases, else a MISPL run-time error will result!

AskChoice

This function displays a question in a dialog box, waits for the user to make a selection from the list of possible answers, and returns an integer containing the index in the list.

IntegerAskChoice (String Prompt, StringTitle, StringChoiceList, LogicalMandatory, IntegerDefaultAnswer)
string Prompt
String expression displayed as the question in the dialog box.
string Title
Optional. String expression displayed in the title bar of the dialog box.
string ChoiceList
Comma-separated list of possible answers.
logical Mandatory
Indicates whether or not an answer is required. If mandatory, the user will not be allowed to quit the     dialog without answering (by means ESC on Windows, PF4 on TTY). In the other case, a 'Cancel' button will be added automatically.
integer DefaultAnswer
Default answer, as an index in the value list.

AskString

This function displays a question in a dialog box, waits for the user to input text, and returns a string containing the answer.

StringAskString (StringPrompt, StringTitle, LogicalMandatory, StringDefaultAnswer)
string Prompt
String expression displayed as the question in the dialog box.
string Title
Optional. String expression displayed in the title bar of the dialog box.
logical Mandatory
Indicates whether or not an answer is required. If mandatory, the user will not be allowed to quit the dialog without answering (by means ESC on Windows, PF4 on TTY). In the other case, a 'Cancel' button will be added automatically. 
string DefaultAnswer
String expression displayed in the text box as the default response.

AskYesNo

This function displays a question in a dialog box, waits for the user to choose a button, and returns the logical value associated to the selected button.

LogicalAskYesNo (StringPrompt, StringTitle, LogicalMandatory, LogicalDefaultAnswer)
string Prompt
String expression displayed as the question in the dialog box.
string Title
Optional. String expression displayed in the title bar of the dialog box.
logical Mandatory
Indicates whether or not an answer is required. If mandatory, the user will not be allowed to quit the dialog without answering (by means ESC on Windows, PF4 on TTY). In the other case, a 'Cancel' button will be added automatically.
logical DefaultAnswer
Default answer. Will cause the corresponding button to get the initial focus.
