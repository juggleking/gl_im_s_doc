Introduction

ERD stands for Entity Relation Diagram. An ERD gives an overview of the database structure of GLIMS. It shows the relationships between the different tables within GLIMS. This structure will be very important in order to be able to build MISPL functions.

An ERD consists of:

  • Entities (the so-called Tables, e.g. the Order table)
  • Relations (one-to-one, one-to-many,...)
  • Routes

An ERD guides you through the internal structure of GLIMS and offers a map which allows to obtain the required data. In order to build a correct MISPL function this route has to be followed, taking into account the relationships between the different tables (entities).

Symbols used to represent the relationships:

|| 1 relation
0| 0 or 1 relation
0< 0 or several relations
|< 1 or several relations

E.g.

Every child has 1 mother
Every mother has 0 or more children
