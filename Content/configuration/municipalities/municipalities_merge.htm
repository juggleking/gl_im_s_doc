Municipality merge program

General municipality merge

Introduction

Due to the manual creation of doubles, or import of 'polluted' data, the same municipality may occur several times in the database, possibly with different spellings. To solve such problems, a municipality merge program is available.

Warning

Use this program with care!  Although the changes are logged to the logging tables (log type Municipality merge), this operation can NOT be undone!

Usage

In the Municipality browser, select the records to be merged and choose the contextual menu/ribbon item Merge.

Correct municipality

By default, the selected municipality is chosen as the correct municipality.

When more than 1 municipality is selected before choosing the Merge function, the first municipality will be considered as the correct municipality and the other will be added to the incorrect municipality list.

Incorrect municipality

Add

Add incorrect municipalities by double-clicking in one of the Incorrect municipalities fields. The selected municipality will be shown in the list below.

Remove

Remove incorrect municipalities by selecting them in the list and pressing the Delete key.

Municipality information

For both the correct municipality and the incorrect municipality, detailed information is shown.

Default

The default information is "<PostalCode>, <Name>", possibly followed by "(<BoxMinimum> - <BoxMaximum>)" when the country's Address type is set to Dutch.

Custom info

This information can be customized via a Municipality-based text module called MunicipalityMergeExternalization.

Test mode

If enabled all the database changes are undone at the end. This allows to perform a simulation.

Merging

Merge fails

When an error occurs during the merge operation, everything is undone and the database is restored to its original status.

Merge succeeds

All records referring an incorrect municipality are linked to the correct one (currently only Correspondent).

Tool to merge Dutch municipalities

For countries which have an address type Dutch, a tool has been created to merge old and recent Dutch municipalities based on postal code, street and house number information.

This tool will go through all Dutch municipalities where street and house number range information has not been entered and will try to merge these municipalities with other municipalities which do contain this information.  All correspondents will be linked to the proper municipality, containing the street and house number range information.

The tool will also automatically split the street and house number information as part of its process of linking correspondents to the proper municipality (see below).

If all correspondents can be updated, the old municipalities will be removed from the database.  If not, a log file will be created that identifies the correspondents that could not be updated because of lacking address information.

This one-time program has no user interface and must be executed via Start -> Development -> 4GL -> Run procedure.  File name must be mncp_mnl.r and leave the option Pass handle disabled.

Tool to split street and house number information for Dutch addresses

This program will check all Dutch correspondents of type Person and attempt to extract the street and house number information from the address line if the house number field contains '?' (a.k.a. unknown value). If the extraction is successful, the record fields AddressLine1, HouseNumber and HouseNumberAppendix are updated with this information, effectively splitting the street and house number information.

If the address line of the correspondent does not yield a valid house number, an entry will be added to a log file, containing the InternalId and Name of the correspondent. This overview makes it easier to find these correspondents that must be updated manually.

This one-time program has no user interface and must be executed via Start -> Development -> 4GL -> Run procedure. File name must be crsp_adnl.r and leave the option Pass handle disabled.
