Institution

Contents: Introduction - Fields

Introduction

An institution in GLIMS generally represents the patient information source or administrative center, for instance the hospital. The institution browser can be accessed by the main menu by selecting Start -> Correspondents -> Institutions.

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

The actual (long) name of the institution.

Head

The financial responsible for the institution, typically the director (for billing purposes).

Associated lab

E.g. the hospital's internal lab. This allows to make a distinction between tests executed in the internal lab or in an external lab.

Allow encounter overlap

If disabled, any new encounter will close the previous one.

Internal

Allows distinction of the internal institution from external ones.

Unique external id

Indicates if two encounters can have the same external id.

Encounter external id

Encounter-based, string-valued function that generates an External id for automatically created encounters.

Bill encounter check

If disabled, tariffication of (hospitalized) orders with an object time outside the encounter period will be allowed.

Bill stay check

If disabled, tariffication of (hospitalized) orders without an existing stay, related to the encounter, will be allowed.

Correspondent

This page contains correspondent information about this institution. Please check the chapter Correspondents for more detailed information.
