Person search via eID

Introduction

GLIMS allows using the Belgian electronic identity card (eID) for patient selection during order entry.

Additional software

Click here to download and install the eID middleware software provided by the Belgian government. This software is required to be able to read the eID card via a card reader.

Note

If this software is already installed, make sure your version of the software is up to date.

Configuration

Make sure that the following general options are set in Start > System Management > Customize > GLIMS General > Identification tab:

National PIN provider

The National PIN provider is the correspondent distributing the national patient identification numbers.

When using the person search via eID functionality, GLIMS searches for a Person record with an Identification record assigned by the National PIN provider that is equal to the national number retrieved from the eID card.

Person data reader

Should be set to eID.

Availability and activation

Person search via eID is available for

  • order entry
  • order consultation

Person search via eID can be activated via

  • F8 on a Person field: Last name, First name, Birth date, Spouse name, Internal id

    Note

    The PIN field cannot be used to activate person search via eID.

How does it work?

Confirming access to eID card

When GLIMS attempts to read the eID card, the user is asked for confirmation for this action. The following window appears, providing the option to always allow access to the inserted card, only once or to disallow access.

Note

When choosing to always allow access, the access granted is only valid for the current GLIMS session and the currently inserted card.

Search criteria

GLIMS reads the eID card and retrieves the following data in order to find an existing Person record.

  • National number (main criterion)
  • First name
  • Last name
  • Sex
  • Date of birth

Search results

Exact match found

When an exact match is found in the GLIMS database, that person will be selected automatically and entered in the order entry screen. Moreover, the order will be marked as Edited with eID (a hidden logical field that can be accessed via MISPL).

No exact match found

When no exact match is found in the GLIMS database, the user will need to select the correct person.

Note

It is not possible to have an existing Person record updated with the data from the eID card.

National number match but person data mismatch

There are Person records in GLIMS of which the national number equals the number from the eID card. However, some of the other person identification data does not match.

Example

This might occur when the person in GLIMS is only known by his initials while the eID card contains his/her full name.

The user will then be offered the choice of searching:

  1. with the national number retrieved from the card

    A person search screen will appear indicating the national number from the eID card and listing the matching Person records.

    When selecting a person (by double clicking or pressing Enter), the user will be informed about the differences between the selected person and the eID data, after which he/she has the option to proceed or cancel.

  2. with the first name, last name, sex and date of birth retrieved from the card (and ignoring the national number)

    A person search screen will appear indicating the person identification data from the eID card and listing the matching Person records.

    The user can select a person by double clicking or pressing Enter.

Tip

You can switch between both search modes by clicking the Reset button in the person search screen provided you haven't modified or added any search criteria.

Person data match but national number mismatch

When no Person records with a matching national number can be found, GLIMS will perform a second search based on the first name, last name, sex and date of birth retrieved from the eID card. Matching Person records will be listed in a person search screen for selection.

No match found

When no match is found in the GLIMS database, the person search screen will be opened, prefilled with the data (last name, first name, birth data, sex) from the eID card.

Tip

It is possible to create a new Person record based on the data from the eID card (first name, last name, sex, date of birth, address) by using the Insert key. An Identification record will also be created for this Person record with the national number from the eID card.

This will only work if a Country is defined in GLIMS with code B and all Belgian municipalities are linked to this country.

No card reader connected

GLIMS allows patient selection via eID even when no card reader is connected. When activating the search via eID functionailty, the person search screen will be launched containing an additional field having the specified National PIN provider as label and allowing to manually search by national number.

Logging

Errors that occur when using the eID functionality are logged using a log type named EidCardReader.
