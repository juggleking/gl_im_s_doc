Double patient detection - By PIN

Description

A specific tool is available that allows merging patients from 2 hospitals.  The features are based on the existing person merge program.

Configuration

This feature will only be available if the general option Support Alternate Double Person Query is enabled.

Click Start -> System management -> Customize -> GLIMS general.

Usage

Selecting double persons on basis of PIN providers

A special query selection screen has been designed to select doubles on basis of PIN providers.

When choosing the main menu function Start -> Correspondents -> Persons ->Lookalikes, the user gets the possibility to choose between the standard query screen and this alternative screen.

Click OK

The alternative query selection screen has parameters Pin provider, Alternate pin provider 1 and Alternate pin provider 2. The query will select all persons with the selected PIN provider, but which do not have both a code assigned by the first or second PIN provider.

The results are shown in the standard double patients browser.

Exporting double persons to a file

The double persons browser offers an extra Export button to export the data to a file with format: "Block no" "PIN" "Internal id" "Last name" "First name".

Merging persons from a file

A function Merge persons from file is available in the main menu.

It allows merging persons based on the data in the exported file.  The user interface is based on the existing person merge program, with options Test and Log to database.
