Introduction

GLIMS knows about different types of correspondents: Person, Organization, HC Provider, Ward, Institution, Lab, Department, Firm, Fund, Health office, Company, Bank, Study. Although each type has specific characteristics, most types share some common attributes.

Fields

Internal id

Unique id, automatically assigned by GLIMS.

Name

Automatically assigned by GLIMS. The contents depends on the specified correspondent type and the information specified for that correspondent type.

Example

In case of a correspondent of type Person, this field will show the first and last name of that person.

Correspondent lookup

In several GLIMS screens (order entry, order consultation, etc), you can look up a correspondent. GLIMS offers different ways of looking up a correspondent. This depends on your user preferences. 

From the main menu click Start -> Tools -> Set user preferences. The option Correspondent search method indicates how GLIMS should look up correspondents.

Via search screen

In analogy to the person search screen, a correspondent search screen will be opened when looking up a correspondent.  It allows searching for a correspondent by typing in a part of the Internal id, Name, Municipality or Address (Correspondent.AddressLine1).

It is also possible to look up correspondents using their LANR or BSNR (Germany specific functionality). Please check the chapter Order entry : main fields - Order info for more information.

Cust. Care

Provides access to the Customer care module.

Automatic

Determines whether the matching correspondent list is updated after each keystroke.

Note

Use this option only if the performance of your system allows it. In automatic mode, some of the browser columns are only updated when actually leaving the current search field (for performance reasons).

Use soundex

Allows to search correspondents of which the name sounds like....

Via browser

If the user preference Correspondent search method is set to Browser, and a correspondent is searched in the order entry screen, the following applies:

  1. The browser only shows correspondents where Correspondent.ActiveSince and Correspondent.ActiveUntil matches with the object date of the order.
  2. An Options button allows specifying the correspondent type, a wildcard pattern for the correspondent name and the reference date.
  3. Regardless of the type of correspondent selected in the order entry screen, the same correspondent browser is used instead of a type-specific browser.
