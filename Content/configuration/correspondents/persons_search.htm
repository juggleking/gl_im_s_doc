Person search screen

Introduction

In several places in Glims (order entry, order consultation, etc), you can use a special person search screen. Whether or not this special search screen will be used, depends on your user preferences.

From the main menu click Start -> Tools -> Set user preferences. The option Person search method indicates how GLIMS should search for persons.

Via browser

If the user preference Person search method is set to Standard, and a Person is searched, a standard Person browser will be shown.

Via search screen

If the user preference Person search method is set to Via search screen, a special person search screen will be opened when looking up a person, e.g. when typing in a character or pressing CTRL-F in the Internal id, Last name, First name, Birth date or Spouse last name field of the original screen.

Identification

For every Identification source specified in the user preferences, the person search screen will show a field allowing to lookup a PIN for that source (PIN provider).

Tip

If the user preference Lookup PIN when partially entered is enabled, the person search screen will also search for a PIN that starts with the specified identification. No full match is required.

Person

Each time a character is typed in the search fields (when the Automatic option is enabled, see below) or the user navigates to the next field, the browser shows all matching patients.

If the Municipality field is specified, only persons of whom the municipality postal code or name begins with the field's contents will be shown.

Browser

The first person in the list is always pre-selected.

  • When pressing the F2 / Enter key or double-clicking a person from the list, the selected person is chosen and shown in the original screen.
  • When pressing the F6 key, the details of the selected person are shown.
  • When pressing the Insert key, a new person record can be created.

Notes

  • If more than 100 persons match the search criteria, the user will be warned that collecting them all may take a long time. The user is given the option to continue the search by clicking the Show all button or refine the criteria.
  • The person browser offers customizable columns via Column cycle buttons situated on top of the browser columns.

Options

Automatic

Determines whether the matching person list is updated after each keystroke.

Note

Use this option only if the performance of your system allows it. In automatic mode, some of the browser columns are only updated when actually leaving the current search field (for performance reasons).

Use soundex

Allows to search persons of which the name sounds like.... GLIMS will search all patients of whom the last name or spouse last name 'sounds' like the entered name. The system used is the original soundex algorithm as described by Knuth e.a.

Buttons

Query (or F8)

In addition to looking up a person in the local GLIMS database, the person search screen can also use all the specified criteria to lookup a person in an external system (e.g. HIS).

When one or more patients are found in the external system, GLIMS will show a selection screen. For more information, please check the chapter External person query.

Query by PIN

Each of the Identification fields can be used to perform an external query. The focused field will determine which PIN is used as query criterion.

The translator that handles the communication with the external system is the Translator specified in the identification source (Correspondent configuration).

Query by Last name, First name, Birth date

When the focus is in one of these fields, all 3 fields will be used as query criteria.

The translator that handles the communication with the external system is the Translator specified in the Default identification source (Correspondent configuration).

Tip

When a field can be used to query an external system, its label is shown in bold.

Reset

Clear all search criteria.

Cancel

Close the search screen.

About query timing

When only one character of the person's last name is entered, tabbing to the first name and entering a first name could lead to very long queries when a lot of patients have a last name beginning with the entered letter.

To avoid such a situation, the program will only automatically start a query when the length of the last name or spouse last name criterion is at least 3 character, or when the length of the internal id criterion is at least 6 characters.

Forced execution of the query can be triggered by hitting the F5 key (CTRL-W on TTY). When the query could not be reopened automatically because of these length restrictions, a message in the person detail area at the bottom of the search screen informs the user about this.

Additionally, the query will also not be opened automatically when the user moves from the Last name to the First name field.The same timing rules apply as for normal typing: the query will be reopened when the user has not typed anything for 300 milliseconds.
