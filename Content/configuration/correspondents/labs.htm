Lab

Contents: Introduction - Fields

Introduction

Usually one internal laboratory and one external laboratory is created. The internal laboratory refers to your own laboratory while the external laboratory is used for all other external laboratories.  This will be of vital importance during billing.  As a procedure is linked to a department, and a department is part of a lab, you will be able to see if a test was executed by your own lab or by an external lab.  In the latter case, invoices will have to be created for this external lab.

The lab browser can be accessed from the main menu by selecting Start -> Correspondents -> Labs.

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

The actual (long) name of the lab.

Internal

Specifies whether the lab is considered as internal or external. This option is important for billing purposes.

Default firm

Tariffication by this lab will result in invoices of the specified firm.

Parent

Used in hierarchically structured labs. Provides access to the lab's parent lab. Procedures of a parent lab (and grandparents) are always considered to be executed locally.

Head

The financial responsible for the lab, typically the director (for billing purposes).

Billing item urgency

Provides access to the (executing) lab's billing item urgency site function (Request based, string valued). The function should return a string value as defined for the site enumerated BillingUrgency.

Each time a request is tariffed, the billing item urgency site function of the request's executing lab, is evaluated. The return value specifies its urgency level and is saved in the corresponding billing item(s). Returning '?' is accepted and considered as not urgent. Returning a value which is not a billing urgency level causes a tariffication error.

Executors are associates

Specifies whether the lab's executors must be restricted to its firm's associates. This is used in Germany only.

Validator is executor

Activate this option for each (executing) lab if you want to use the result responsible as executor of the billing items (instead of using the provisions).

In routine, if this option is activated but no result responsible is found, the provision executor is used.

Note

If you do not want GLIMS to look for an executor in the Provision table when there is no result responsible, leave the executor empty in this table for the executing lab in question.

Confirm executor

Not supported.
