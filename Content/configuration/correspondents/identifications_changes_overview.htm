Overview of patient identification changes

Introduction

GLIMS allows to generate an overview of all patient identification changes within a specified period.  This feature is available from the main menu via Start -> Correspondents -> Persons -> Patient Identity Vigilance.

The overview is based on patient identity changes stored in the GLIMS Log tables.

Options

From

Time period in which the patient identity change took place.

Severity

Severity of the log entries in which to look for patient identity changes.

Search for

Allows to lookup specific changes.  Example: by specifying "FirstName", you can lookup all changes to patient first names.

Log types

Allows to specify the log types in which to look for patient identity changes (Audit person, Person merge, Person fusion, Audit identification, Audit object).

Overview

The overview lists all changes, grouped by patient and log type.  For each change, it displays:

  1. The date / time of the change
  2. The user who performed the change
  3. The previous and new value
