Person identity changes log

GLIMS keeps track of all important changes to a patient's identity, i.e. Last name, First name, Middle name, Birth date, Sex, Spouse last name, Has twin, Display last name and Legal name.

Existence of a log is signaled by means of a View log button in the order entry screen and the patient editor. The button is always visible, but only becomes sensitive when the Person.Log field actually contains update logging information.

The Person.ViewLog function also provides access to this logging field, but will show an empty screen when no logging is available.

Notes

  • Case changes (uppercase <> lowercase) are not logged.
  • Updates occurring during the first 30 seconds after record creation are ignored.
