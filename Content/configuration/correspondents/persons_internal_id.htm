Person internal ID

Each Person is identified in GLIMS with an automatically generated Internal id that is composed of <Base><Discriminator><CheckDigit>

Where

<Base> is obtained by:

  1. Pre-processing the person's last name and first name:  all last name prefixes, as specified in SpecificSite.LastNamePrefixList are moved to the end of the last name. E.g. "Van de Velde" becomes "VeldeVande"

    All characters are translated to their uppercase equivalent and illegal characters are removed or replaced by valid characters:
    E.g. "Weiß" becomes "WEISS", "Hélène" becomes "HELENE"
  2. On this Person record with modified last and first name, either the user specified MISPL function SpecificSite.PersonInternalId or the built-in default function is executed.  The built-in default function returns YYYYMMDDLLLF
    Where:

    YYYY = Year of birth, 0000 if unknown
    MM = Month of birth, 00 if unknown
    DD = Day of birth, 00 if unknown
    LLL = the first 3 characters of the (modified) last name
    F= the first character of the (modified) first name

<Discriminator>

Starts at "0" and increments until the obtained internal id is unique.
0 > 9, A > Z, A0 > AZ, ..., Z0 > ZZ

<CheckDigit>

Check digit that is calculated from the Base + Discriminator. Allows to verify that a manually entered internal id is valid (analogous to a bank account number).
