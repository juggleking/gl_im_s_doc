Object attributes

Contents: Introduction - Fields - Link attribute to object - Usage - MISPL functions - Object attribute history

Introduction

By means of object attributes, extra diagnostic information can be stored and taken into account during result validation, norm interpretation, tariffication.

An example would be Pregnancy.

Fields

Object attributes are available from the main menu via Start -> Configuration -> Objects -> Object attributes.

Tip

GLIMS also offers an object attribute import program.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Seq no

Indicates both the order in which object attributes are shown and their relative importance.

Active

Only active object attributes can be assigned to an object.

Description

Optional, multilingual description.

Flag, Flag severity

The primary intention of these fields is to inform the user (e.g. on a work list, in the validation program) that a patient has a specific object attribute.

Note

GLIMS will not show the object attribute information automatically. You should use a MISPL expression in e.g.

  • the row label of a work list.
  • the externalization window of the validation screen.

You can use the following MISPL functions to retrieve object attributes with a specific flag or flag severity:

  • Object.AttributeList
  • Order.Attribute("ObjectAttributeFlags")

Default period

Indicates how long the object attribute is normally valid.  This field will automatically determine the object attribute's end date when specifying the start date in the order entry screen (see below).

Maximum period

The maximum number of days that this attribute can be active.  When manually specifying an object attribute's end date, GLIMS will use this field to check if the maximum period is not exceeded.

Department

Can be used in department shielding. Users will not be able to see or update object attributes which are shielded.

Linking an object attribute to an object

You can assign an object attribute to an object during order entry. The Attributes page of the order entry screen shows a browser with all object attributes.

When entering the Start date, GLIMS will automatically determine the End date using the object attribute's Default period.

 

Note

The order entry program takes the privileges on the ObjectAttributeHistory table into account by making the browser read-only when the user lacks one of the Create/Update/Delete privileges.

Usage

Object attributes can be used for:

  • Norms
  • Ranking rules (Transfusion)
  • Valab validation
  • Billing
  • ...

MISPL functions

Setting an object attribute

The Object.SetAttribute MISPL function allows to assign an attribute to an object.

Example

You could define a MISPL function that automatically sets an object attribute depending on the result value of a test.

Retrieving an object attribute

The following MISPL functions allow to retrieve the object attributes of an object:

  • Object.AttributeList
  • Order.Attribute("ObjectAttributeFlags")

They can be used to define object attribute dependent rules.

Clearing an object attribute

The Object.ClearAttribute MISPL function allows to specify an End date for a given attribute.

Note

An option is available to completely remove an object attribute history for a patient when the specified End date lies before the start date of the object attribute history record.  This is only allowed for object attribute history records that do not have an end date yet.

This option has been implemented as a Logical site attribute named _DelWhenClearedLT1Day on table ObjectAttribute.

Deleting an object attribute

The Object.DeleteAttribute MISPL function deletes any attribute history for a given object attribute that was active on the specified EndDate.  It treats the EndDate as an inclusive boundary (meaning any attributes that started or ended exactly on the specified date are also deleted).

Warning

Use this MISPL function with extreme care as it irreversibly deletes the object attribute ! Use the MISPL function Object.ClearAttribute if you only want to specify an end date without deleting the attribute.

Object attribute history

The contextual menu / ribbon item Attribute history on an Object allows to consult all object attributes that have been assigned to an object.
