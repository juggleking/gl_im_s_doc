Label content

Material vs Material input

GLIMS allows to define specimen labels (via a multilingual text) on two different levels:

Material

Example

.Object.Person().LastName
{< .InternalId} {< .Material.ShortName}
Dep.: {DepartmentMnemonic}, Size: {Size}
Work: {< .DepartmentPropertyCodeList:{DepartmentMnemonic}}

In the definition of the label, the {Size} parameter passed to this text also takes the minimal and maximal size of the material definition into account, as opposed to the Specimen.Size reference.

Material input

Note

The label on Material input has the advantage that more information is available at this level.  It allows for instance to include the procedure or station name.

Language

Default

By default, the content of the specimen label is expanded in the language of the current user.

Use issuer language

If the general option Labels in issuer language is enabled, the specimen labels will be expanded in the language of the issuer (as defined in the issuer's correspondent configuration).

Click Start -> System management -> Customize -> GLIMS general.

Terminology

Root specimen A root specimen is a specimen where Specimen.ParentSpecimen = ?.
Work specimen A work specimen is a specimen where Specimen.ParentSpecimen <> ?.
Own labels The own labels of a specimen are either the Material label or the MaterialInput labels with MaterialInput.IsSubLabel=NO. 
Sub labels The sub labels of a specimen are the MaterialInput labels with MaterialInput.IsSubLabel = YES.

Material input - Is sub label

The Material input option Is sub label allows to make a clear distinction between material input labels which are to be considered as 'daughter' labels of the material and material input labels which are to be considered as being part of the own labels of the material.

Disabled

The material input labels are considered to be part of the 'own' labels of the specimen.

This label will be printed together with the material label (if defined), and the specimen on which the label is put is not to be considered as a daughter specimen of the actual specimen.

Enabled

Indicates that this material input label is actually to be considered as a label intended for a subspecimen of the actual specimen, and should not always be printed together with the material label of the specimen.

Tip

This offers the possibility to treat material input labels as work labels and allows the definition of so-called 'root' labels and 'work' labels without having to define the work material or the procedure needed to convert the root material into the work material.

Which labels will be printed

General

Root specimen or work specimen

When deciding which labels are to be printed, GLIMS first evaluates for each specimen if it is a root specimen (no parent specimen) or a work specimen (has a parent specimen) as defined by your procedure configuration.

Labels for each department

When a specimen is used as input for actions executed in different departments, then the Order-based function Print labels function (also used during order entry) will by default print the Specimen.Material.Label once for each department, to indicate that the original specimen should be split into several equivalent specimens.  

You can deactivate this built-in functionality by disabling the general option "Label per department". The Specimen.Material label will then only be printed once.

Click Start -> System management -> Customize -> GLIMS general.

Avoid printing unnecessary specimen labels

Specimen labels will only be printed when output is expected. Overview of the implemented checks:

  1. Labels on the material of a specimen are only printed if the specimen is not discontinued.

  2. Labels on the material of a specimen are only printed if there is at least 1 specimen input with an action that contains at least 1 (solicited) result or specimen that is not discontinued.

    If the general option Label per department is set, then labels will only be printed for those departments where the procedure's action contains at least 1 result or specimen that is not discontinued.

  3. Labels for specimen inputs (defined on material input) are printed if the actions contain at least 1 (solicited) result or specimen that is not discontinued.

  4. Carrier labels are printed only if the associated microbiology action is not discontinued.

The checks mentioned above are limited to the next level in the action hierarchy, i.e. if the specimen (level 1) has an input pointing to another non-discontinued specimen (level2) that has only discontinued results, then the label will be printed for the first specimen (level1) anyway.

Avoid reprinting already printed specimen labels

Both the order entry options and the Order-based function Print labels have an option Check if printed.

If enabled, GLIMS will first check whether the specimen labels have already been printed before.  More precisely, GLIMS will check if the Specimen option Label printed is enabled.

During order entry

In the order entry options, you can indicate which labels should be printed.

Root specimen labels

If enabled, GLIMS will print the own labels of the root specimens.

Work specimen labels

If enabled, GLIMS will print the own labels of the work specimens AND the material input labels of the root specimens with MaterialInput.IsSubLabel=YES.

Order.PrintLabels

This function behaves identically to the label printing mechanism during order entry, but always prints both the root and work specimen labels.

Specimen scan

The specimen scan program has two fields that indicate which labels should be printed.

Own labels

If enabled, GLIMS will print the own labels of the scanned specimen.

Sublabels

If enabled, GLIMS will print the own labels of the actual, physical subspecimens (if any) AND the sublabels of the scanned specimen (material input labels with MaterialInput.IsSubLabel=YES).

Worklist.PrintLabels

This function will print the own labels of the output specimens.

Specimen.PrintLabel

This function will print the own labels of the selected specimen.

From within MISPL

When printing specimen labels from within MISPL by using the Order.PrintLabels() function or the Order.PostProcess() function, the root and work specimen labels are printed, taking the Specimen.LabelPrinted mark into account.

This implies that a label which was marked as printed will not be re-printed via these function calls. 

Only print specimen labels for next step in workflow

Case description

GLIMS offers a two-level concept for specimen handling: root specimens and work specimens.  In case of a third level (sub specimens of sub specimens), GLIMS does not make a distinction between the 2nd and the 3rd level; they are both considered as "work labels".

A solution is required to print labels for sub specimens by level, e.g. first the labels for the 2nd level sub specimens and later the labels for the 3rd level sub specimens.  

Solution

All programs that allow to print Work labels or Sub labels offer an option to only print the labels for sub specimens that have at least status Expected.  This allows to limit printing of work specimen labels to the next step in the work flow.

This option is available in the following programs:

Order entry options

Specimen scan

Specimens of order browser

Order function "Print labels"

Create order series

Note

There are two categories of specimen label printing programs, which have a slightly different functionality:

Printing labels for an order (order entry, Order.PrintLabels or Create order series program)

The check is applied for any work specimen label, whether they have sub specimens or not (cfr. order entry option Work specimen labels).  When an order is created with the option to activate the actions automatically, then the 1st level work specimens will get status Expected, while the 2nd level specimens are still in status Initial, so the labels will be printed for the 1st level specimens only. For root specimens the labels are always printed.

Printing labels for a specimen (Specimen.PrintLabels or Specimen scan program)

The check is applied for sub specimens (cfr. specimen print option Sublabels), so even for work specimens in status Initial, own labels can always be printed on demand. It is the printing of labels of their sub specimens that is suppressed.

Sequence in which the labels are printed

Sequence in which the labels are printed when both root and work specimen labels are requested:

Root specimen labels

To produce the root specimen labels, the 'own' labels of the root specimens are printed.  

Work specimen labels

Next, to produce the work specimen labels, the 'sub' labels of the root specimens and both the 'own' and 'sub' labels of the work specimens are printed.

Within each of these groups, the ordering is determined by Material.LabelSeqNo + Material.Mnemonic + Specimen.SamplingTime + Specimen.InternalId + Procedure.SeqNo (for the material input labels).

Note

The above ordering is used when the Department option Print order specimen label is set to Default. However, if Print order specimen label is set to By internal ID, the labels are printed based on the Specimen.InternalId (e.g. for pathology).

Note

The labels generated by the WorkList.PrintLabels() function are sorted in the following order:

  • For output specimen labels (i.e. sampling lists): By ProcedureOutput.SeqNo, By Material.LabelSeqNo, By Material.Mnemonic, By Specimen.InternalId
  • For input specimen labels (i.e. for regular worklists): By ProcedureInput.SeqNo, By Material.LabelSeqNo, By Material.Mnemonic

The ProcedureOutput/Input.SeqNo sorting should ensure that labels are printed in the same order as they are shown on a grid or outline printout of the sampling list. The Material.LabelSeqNo sorting ensures that when all ProcedureOutput.SeqNo values are the same (e.g. unknown), the default label sorting is followed like in the Order.PrintLabels function.
