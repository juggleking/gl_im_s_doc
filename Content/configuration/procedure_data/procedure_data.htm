Procedure data

Introduction

For each procedure, you can define procedure data.  Procedure data allows to show extra information in other GLIMS programs such as work lists, the cell counter, or allows to send extra information in analyzer connections.

Select a procedure and choose the contextual menu / ribbon item Procedure data.

General

Procedure

Procedure to which this procedure data applies.

Seq no

Indicates in which sequence the procedure data must be displayed in other GLIMS screens.

Property

Property of which the result should be shown as procedure data.

History index

Used to indicate that a historical value is to be shown.

Example

If set to "1"

The previous result, compared to the object time of the current result, will be shown.

If set to "0"

GLIMS will search for a result with the current object time.

When there are 2 orders for the same object time, each containing this property, preference will be given to the result of the order to which the original action belongs.

Show date

If enabled, the object date will be shown next to the result value.

Show

Allows to define whether a procedure data item (e.g. previous result of NA) is to be shown on the work lists if the corresponding property is not an output of this action (i.e. is not requested today).

The allowed values are:

Always

The previous result is shown even when no request was made today.

When requested today

The previous result will not be shown if the current action has no output for the same property.

Station

Outbound id

When procedure data results are to be downloaded to a station, you can specify an alternate code for the property mnemonic to be transmitted with the result.  

This alternate code will be sent in the Result record of the ASTM order message (R.3.7).

Scope

Specifies which procedure data is to be sent in a work order message to a station.

Example

Suppose the action outputs of a specimen are sent in a work order message and procedure data is defined on the procedure of that action.

The allowed values are:

Scope = Specimen

Only procedure data results obtained on the same specimen are sent.

Scope = Object

The procedure data results are not restricted to the same specimen, but to the same object.

Expand value

Allows coded result values (e.g. '{<MyText}') to be expanded in a work order message to a station.

Note

On transmission time, the result value is left unchanged in the database; it is not expanded.
