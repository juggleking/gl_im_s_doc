Objects

Introduction

The term Object is used in GLIMS as a common denominator for everything the laboratory assesses properties of. Typical objects for clinical laboratories are patients.  As GLIMS does not only support tests on 'human' patients, the term Object is used.

Object types are:

  • Person
  • Animal
  • Lot
  • QC lot
  • Unknown

Tip

An object is created during order entry.  You will e.g. first have to define or import persons and then create an order for these persons.  When saving the order, an object will automatically be created by GLIMS.

Unknown objects

Creation of unknown objects

When entering object information during order entry, GLIMS requires

  • the Person last name in order to select a Person record
  • the Animal name to select an Animal record.  

If this information is not available, an Unknown object record will be created at order entry.

Person and Animal information stored in Unknown object

The specified Object type (person / animal), Birth date and Sex are stored in the Unknown object record. This information can later be used as search criteria to associate the exact person / animal record with the order.

As it can take a while before the actual person / animal is associated to the order, GLIMS allows to use information stored in the unknown object.

Norms

The Birth date and Sex stored in the Unknown object will be used to evaluate norms.

MISPL access

The following Object-based MISPL functions will return or use the  Sex / Birth date values stored in the Unknown object record when applicable:

Only for Objects of type Person and for Unknown objects with type Person:

  1. Object.PersonData("BirthDate")
  2. Object.PersonData("Sex")

For Objects of types Person or Animal and for Unknown objects with type Person or Animal:

  1. Object.Age(?)
  2. Object.BirthDate()
  3. Object.Sex()
  4. Object.GetData("BirthDate,Sex")

Object type of Unknown object

The Unknown object field that contains the (preliminary) Type is named PreliminaryType.

This field can be accessed via the Object-based MISPL function Object.DeclaredType. 

  1. For Objects of Type UnknownObject, this function will return the value as stored in UnknownObject.PreliminaryType.
  2. For regular Object records, this function returns the value stored in Object.Type.
  3. A value of {&ObjectTypeUnknown} is never returned.  For unknown object records with an unspecified value for preliminary type, ? is returned.

Example 

String ObjectType;
ObjectType := EnumeratedToString("ObjectType", .Object.DeclaredType());  
RETURN ObjectType ;

Object template of Unknown object

For unknown objects with declared type Person and Animal, the Object template is saved in the Unknown object record.

The Object template stored in the Unknown object will be used when associating the order with the actual animal: only animals with this template can be selected.
