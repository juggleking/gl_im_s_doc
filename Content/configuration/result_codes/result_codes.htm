Result codes

Contents: Introduction - Fields - Entering result codes - Concatenation of result codes

Introduction

Result codes can be used to enter pseudo results such as Tube broken, Immeasurable, Out of limits, etc.

Result codes can be accessed from the main menu via Start -> Configuration -> Result codes.

Fields

Code

Code that is used to enter this result code as result value. E.g. Code is TB and Expansion is Tube broken.

Note

Spaces are not allowed in the code as they cannot be distinguished from a space used to separate two codes for properties of which the option Allow multiple result codes is enabled.

Property

If a property is specified, the result code can only be used as result value for that specific property.

Expansion

Multilingual, Result-based text to which the code should be expanded.

Severity

Severity that will be applied to the result when this code is entered as value.

To charge

Allows to change the billing status of the request.

?

The unknown value (?) means: do not change the current billing status.

Yes

Set the request's billing status to Yes.

No

Set the request's billing status to No.

Note

  • When a result code changes the request's billing status, this is registered in the result log.
  • When a result is repeated or the result value is changed, and the request field To charge is set to Yes or No, the program can determine from the logging that this flag has been set automatically. It will reset the Request.ToCharge flag to the unknown value, thus allowing billing of the request again.

Action

Action that will be performed on the result when this result code is entered as value.

Note

When Discontinue is selected as Action, the text specified in Expansion will be registered as Rejection reason.

Discontinue specimen

If enabled, GLIMS will behave as if the function Specimen.Discontinue is executed on the root specimen of the result, with the following settings:

  • Cascade: enabled
  • Result code: code of this result code

This will cause the root specimen (Result.Specimen) and any sub-specimen to be discontinued, and the entered result code to be applied to all results derived from any of the discontinued specimens.

A question will appear if an Available result is found during the discontinuation. The discontinuation reason of the specimen(s) is hard-coded to:

Discontinued via '<PropertyMnemonic>' result (result code <ResultCode>)".

Tip

This functionality was primarily introduced to facilitate "No material received" situations. By entering one single code for one single result, the complete situation can be 'handled'. 

Assessed

Indicates whether the assessment actually took place. This is used for statistical purposes.

QC severity

When entering a result code of which the QC severity exceeds the assessment method's QC alarm severity, the assessment method will be marked as Unreliable.

QC comment

Text that will be added to the result's internal comment and shown in the assessment method's quality comment.

Entering result codes

Result code/choice/diagnosis browser

The result code/choice/diagnosis browser which is opened when typing ctrl-f in a result value, an isolation test value or a carrier test value has some special features to enhance its clarity.  This browser can show different types of records simultaneously: result codes, choices of a choice list, and diagnosis codes.

  1. When multiple types are present in the browser, the browser header label of the 'description' column will be adapted according to the type of records actually present (e.g. "Name - Description" when both choices and result codes are shown).
     
  2. Moreover, when multiple types are present, each type will be preceded by a grey background 'header' line, delimiting the groups from each other and which at the same time allow collapsing or expanding a group by double-clicking on this header line.

Result code or choice: who has priority ?

By default, any result entered is first checked against the result codes valid for this property. This has the consequence that result codes have precedence over property choices of enumerated results.

Although we believe this is the best approach, the SpecificSite.ChoicesHavePrecendence option allows that the result is first checked against the property choices for an enumerated property. When the entered result matches with a property choice, it is selected and the result code table is not checked any more.  Note that this option only works for properties of datatype enumerated.  The result code selector (accessible by hitting CTRL-F in a result entry field) takes the precedence setting of choices and result codes into account.

Please consider carefully the possible side effect before activating this option! Remember that for instance a choice match is detected also when the entered result unambiguously matches the start of one of the choices, while for a result code an exact match is required.

Search for: only those result codes will be shown for which the description contains all the words entered in the search field. The result codes will be filtered when hitting the 'Return' or the filter button. To show all the result codes again after a filter has been applied, clear the search field and press enter (or the filter button).

Concatenation of result codes

It is possible to compose a string result by means of concatenation of result codes. To enable this feature you must set the property field 'Allow multiple result codes'. When that field is set, every word of a result string is checked to see if it is a valid result code.   The user can type multiple result codes, separated by spaces, or select them from  the result code lookup browser.

If a result of a numeric property contains multiple result codes, it will NOT be normalized (i.e. decomposed in mantissa and exponent), but saved as a string result instead.

When combining multiple result codes, the highest result code severity is used to set the result severity. The other fields of the result codes (Action, Assessed, To charge) are not taken into consideration when there is more than one result code.
