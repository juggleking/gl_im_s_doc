Site attributes

Creation

Site attributes allow adding user-defined fields of different data types to a specific table. Choose Start -> System Management -> Scheme -> Tables. Select the table for which you want to create a site attribute and choose the contextual menu function Site attributes:

The site attribute editor looks as follows:

Name

It is not allowed to use characters like comma, semicolon, colon, slash, backslash or a space in the name of a site attribute.

Sequence

Indicate a digit to sort the site attribute among the other site attributes.
Example

Example

If you indicate "10" for site attribute A and "20" for site attribute B, site attribute A will be displayed above (or to the left) of site attribute B.

Data type

A site attribute can have Reference as data type (next to integer, fractional, string, ...). This implies that the instances of such a site attribute will contain a reference to another record in the table indicated by the field Target table.

Note

Site attributes of data type Reference can refer to theoretically any record in any table, depending on the SiteAttribute.TargetTable. The configuration makes it possible to choose any table for SiteAttribute.TargetTable. This may lead to a situation where the record cannot be properly externalized in the Code editor. In such cases, the record will either be externalized via its internal identifier (e.g. StationFlag.Id) or via a special format (e.g. <Station Mnemonic>/<Property mnemonic> for assessment methods).

Although the externalization may be limited, this allows creating site attribute values referring to any record of any table via dedicated programs.

To navigate to this record via MISPL, the <AttributeName> parameter of the GetSiteAtttribute(<TableName>, <RecordId>, <AttributeName>) MISPL function can contain the path to a field of the referred record, using the "AttributeName.FieldName" syntax.

For instance, if "Boss" is the name of a User-based site attribute referring to another user, the following call would return the initials of a user's boss:

RETURN GetSiteAttribute("sc_User", .Id, "Boss.Initials")

Note that <FieldName> can contain a valid path to any field to which the program can navigate WITHOUT USING FUNCTIONS. Example:

RETURN GetSiteAttribute("sc_User", .Id, "Boss.Language.Name") would return the name of the language of the user's boss.

For a description of each field, please check the reference guide.

Related MISPL functions

GetSiteAttribute

This function returns the value of a site attribute.

string GetSiteAttribute(string Table, integer RecordId, string AttributeName)

Example : PersonReligion := GetSiteAttribute("Person", .Person.Id, "Religion") ;

SetSiteAttibute

This function sets the value of a site attribute.

logical SetSiteAttribute(string Table, integer RecordId, string AttributeName, string Value)

Note

This MISPL function cannot be used to update site attributes starting with "_". Site attributes whose names start with "_" are considered as "reserved". They are often used to control and implement a function for which otherwise a major upgrade would be necessary. Their values must not be changed from within MISPL.

Site domains

The Site domains browser can be opened via Start -> System management -> Scheme.

Site domains allow the creation of values for existing GLIMS fields. Currently site domains can only be used for enumerated fields in order to overrule the default value list.

Examples:

  • Urgencies
  • Billing marks
  • Reminder levels
  • Order identifier types
