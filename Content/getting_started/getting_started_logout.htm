Getting started - Logout

To leave the GLIMS application you can either:

  • Logout: closes your current GLIMS session and shows the login screen.
  • Exit: completely closes the GLIMS session without showing the login screen.

Additionally, if GLIMS is left idle for a certain (customizable) amount of time, the user is automatically logged out.

Logout

To logout, open the Start menu and click the Logout icon.

Exit

To exit, you can either:

  1. Click the Cross icon in upper right corner.
       

     
  2. Open the Start menu and click the Exit icon.

Auto-logout

Timeouts

If the logged in user does not use GLIMS for a significant period of time, security setup may automatically log them out. Auto-logout is typically a three-stage process: 

  1. Authorization timeout

    Maximum time in seconds during which GLIMS can be left idle. Once this period has elapsed, all application subwindows are hidden and the password dialog appears. By default this field contains a ?, which corresponds to 1800 seconds. If the correct password is supplied, the current context is resumed.

  2. Context timeout

    Maximum time in seconds during which the password dialog can remain idle. Once this period has elapsed, all contexts close and the login dialog is displayed. By default this field contains a ?, which corresponds to 1800 seconds.

    The Context timeout is only applied once the Authorization timeout has elapsed.

    Example

    Example

    • Authorization timeout = 60 seconds
    • Context timeout = 30 seconds

    The Context timeout takes place after 90 seconds.

  3. Connection timeout

    Maximum time in seconds during which the login dialog can remain idle. Once this period has elapsed, the connection with the database is closed, the login dialog disappears and focus is returned to the Windows shell. By default this field contains a ?, which corresponds to 1800 seconds.

    The Connection timeout is only applied once the Context timeout has elapsed.

    Example

    Example

    • Authorization timeout = 60 seconds
    • Context timeout = 30 seconds
    • Connection timeout= 20 seconds

    The Connection timeout takes place after 110 seconds.

Setting the authorization and context timeout

The authorization and context timeouts can be set

  1. For each individual User
  2. Generally for all application users via Start -> System management -> Customize -> MATE -> Security tab.

Note

The user-specific setting has priority.

Setting the connection timeout

The connection timeout can only be set generally and via Start -> System management -> Customize > MATE -> Security tab. It will be applied to all users.
