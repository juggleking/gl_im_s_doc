Getting started - Login

Login

When starting GLIMS, a login screen appears.

  1. Users must first provide a Name and a Password to gain access



  2. When the logged in user is assigned to several Departments, he/she has to specify for which department he/she will currently be working.  The departments are presented according to the sequence number of the user's employments.

     
     

  3. If the logged in user has several Roles within the selected department, he/she has to specify the role to be applied. The roles are presented according to the sequence number of the user's employments.

Notes

  • If a user is only employed in 1 department, the department selection screen will be skipped.
  • If a user only has a single role within a department, the role selection screen will be skipped.
  • If a user selects a department, and no role at all has been defined for that employee assignment, the user will be able to select any of the roles assigned to him/her.  We therefore strongly recommend to always specify a role for each employee assignment.
  • If a user has no roles at all (no user role assignment), the user will be logged in as a guest and will receive a warning message asking to contact the system administrator.

Auto-login

It is possible to indicate that a user can login without entering a name and password. GLIMS will use the Windows authentication to login the user.   

Example

  • If your Windows login name is John, a GLIMS user must exist with Login name set to John.
  • In GLIMS, the User option Autologin must be enabled.

Tip

On Windows, the GLIMS shortcut should contain " -a " (between quotes).

Example: -basekey ini -p gp_prun -T c:\temp -param " -a "

To create a new shortcut:

  1. Copy an existing GLIMS desktop shortcut and rename it to e.g. "Auto login GLIMS"

  2. The startup option of the original shortcut could be:

    D:\glims9\bin\prowin32.exe -basekey ini -ininame bin\hdprogress.ini -p gp_prun -param "-1 be/mips/ablframework/gui/start" -assemblies assemblies -preloadCLR

  3. The startup option of the new shortcut should then be:

    D:\glims9\bin\prowin32.exe -basekey ini -ininame bin\hdprogress.ini -p gp_prun -param "-a -1 be/mips/ablframework/gui/start" -assemblies assemblies -preloadCLR

Login failures

GLIMS allows to define the maximum number of login failures after which a user will be disabled.

  1. Click Start -> System management -> Customize -> Mate.
  2. Enter a value in the Max login failures field.

Note

All login failures are logged in GLIMS.

Customized login screen

GLIMS allows to customize the login screen by adding an image in its upper right corner. This can be used to e.g. distinguish between a production and test environment.

  1. Create an image of 200x40 pixels named GlimsLogin.png
  2. Put it in the GLIMS Dat folder
  3. It will be shown in the upper right corner of the login screen.

Server identifier

Using the F1, F10 or CTRL-P key combination in the login screen, GLIMS will show the active server identifier. This is mainly intended for MIPS support engineers.
