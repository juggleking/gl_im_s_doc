Business Activity Monitor - General settings

Introduction

By default, the General monitor looks as follows:

However, the layout, refresh rates and gauge maximal values can be customized. This chapter describes how to do so.

Note

The layout and refresh rates of the TAT monitor are not customizable.

Getting started

The customizable features of the General monitor are set in an XML code field, Business activity monitor settings, situated under System Management -> Customize -> GLIMS General -> Main page tab -> More tab page. Customizing the General monitor requires to modify the XML code in this field.

Example

Example

In the following example, the background color of the Order completeness pane is changed to white:

The sections below present an example of XML code and how to modify this code in order to change

  • the layout of the whole General monitor and individual panes,
  • the refresh rates of the data displayed on the General monitor,
  • the maximal values of the performance gauges.
Tip

Tip

Press F6 to open a larger version of the field and edit the code more easily.

Note

To undo the customization and come back to the default settings, delete the entire XML code in the Business activity monitor settings field.

XML code example

Here is a sample of code which can be inserted in the Business activity monitor settings field. The resulting General monitor is displayed underneath.

Example

Example 

<DashboardSettings>
	<Style>
 		<Color1><R>230</R><G>52</G><B>52</B></Color1>
		<Color2><R>240</R><G>196</G><B>79</B></Color2>
		<Color3><R>255</R> <G>153</G><B>51</B></Color3>
		<Color4><R>74</R><G>217</G><B>100</B></Color4>
		<Color5><R>150</R><G>195</G><B>235</B></Color5>
		<FontName>Arial</FontName>
		<FontSizeDetail>10</FontSizeDetail>
		<FontSizeLabel>16</FontSizeLabel>
	</Style>
	<ClockPanel>
                <Top>700</Top>
                <Style>
                       <ForeColor><Name>Red</Name></ForeColor>
                </Style>
        </ClockPanel>
        <ByStatusPanel>
		<Style>
			<FontName>Elephant</FontName>
			<FontSizeDetail>12</FontSizeDetail>
			<ForeColor><Name>Yellow</Name></ForeColor>
			<Color1><Name>Pink</Name></Color1>
			<BackColor><Name>Darkblue</Name></BackColor>
		</Style>
	</ByStatusPanel>
	<TitlePanel>
		<Style>
			<FontSizeTitle>25</FontSizeTitle>
		</Style>
	</TitlePanel>
	<RefreshRates>
		<Dashboard>120</Dashboard>
		<LastHourData>0.05</LastHourData>

Layout

As can be seen in the example of the previous section, the layout is defined inside a <Style> element by style subelements, such as <Color1> and <FontName>.

To define the layout for the whole monitor, add the <Style> element directly under the <DashboardSettings> element.

To define the layout for an individual pane, add the <Style> element directly under the element referring to this pane. The list of pane elements is provided below.

Subelements for styles

All customizable layout styles can be defined inside the <Style> element, using the following subelements:

Subelements Descriptions
<FontName> name of the font (e.g. Arial, Calibri)
<FontSizeDetail> font size for the data in the progress bars and legends
<FontSizeLabel> font size for the labels to the left of the progress bars
<FontSizeTitle> font size for the panel headers
<BackColor> background color of the pane
<ForeColor> color of the text
<Color1>

color used for

  • Unavailable in the Instrument status pane
  • Overdue in the Overdue orders pane
  • To do in the By status - Phone bar
<Color2>

color used for

  • To be sent in the Instrument status pane
  • Partial in By status - Orders bar
  • Available and confirmed in the By status - Results bar
<Color3> color used for the status Awaiting results in the Instrument status pane
<Color4>

color used for

  • Realized in the Instrument status pane
  • Complete in the Order completeness pane
  • In time in the Overdue orders pane
  • Complete in the By status - Orders bar
  • Validated in the By status - Results bar
  • Done in the By status - Phone bar
  • Performance gauges
<Color5>

color used for

  • Awaiting results in the Order completeness bar
  • Unspecified in the Overdue orders bar
  • Initial in the By status - Orders bar
  • Initial and expected in the By status - Results bar
<NeedleColor>

color of the gauge needles

Note

Note

The color of the gauge needle can only be defined at the level of the Performance pane. In other words, the color will only be customized if the <NeedleColor> element is placed within the <PerformancePanel> element.

The color values can be specified by name or RGB values (such as in the example below). The color names that can be used are listed here:

https://msdn.microsoft.com/en-us/library/system.windows.media.colors(v=vs.110).aspx

System colors, like ControlLight, can also be used:

https://msdn.microsoft.com/en-us/library/system.drawing.systemcolors(v=vs.110).aspx

Note

If a style is defined both on monitor and pane level but with different values, the value indicated at pane level overrides the value indicated at monitor level.

For instance, if <Color2> is defined as "blue" at monitor level but as "yellow" at the level of the Instrument status pane, the To be sent actions in the Instrument status pane are displayed in yellow, not in blue.

Pane elements

In the BAM settings XML code, the panes are referred to as:

  • <OrderCompletenessPanel>
  • <OverdueOrdersPanel>
  • <ByStatusPanel>
  • <TitlePanel> : header pane
  • <InstrumentStatusPanel>
  • <PerformancePanel>
  • <MonitorSelectionPanel>
  • <ClockPanel>

Pane position

The position of a pane can be specified using the elements <Left> and <Top> . The value is expressed in pixels. In the example above, the date and time frame (<ClockPanel> element) is placed 700 pixels away from the top of the screen.

Refresh rates

The data displayed by the General monitor can be refreshed at specific intervals. Refresh intervals can be set using the following elements:

Elements Descriptions Default refresh rate
<Dashboard> rate (in seconds) at which the General monitor will be updated. This does not mean that the data will be updated. Rather, this is the interval at which the monitor will request data from the statistics retriever. The latter can decide to return the same data if the data is still fresh enough, according to their refresh rates. 120 seconds
<LastHourData> maximum age (in decimal fraction of an hour) of the statistics data for the Orders and Results gauges. If the cached data is older than the value specified in the LastHourData refresh rate, then a new statistic will be generated and returned, otherwise the cached data will be returned. 0.05 hour
<TodayData> maximum age (in hours) of the statistics data for the Order completeness - Today, Overdue orders - Today, By status - Orders, By status - Results and By status - Phone progress bars. 0.1 hour
<YesterdayData> maximum age (in hours) of the statistics data for the Order completeness - Yesterday and Overdue orders - Yesterday progress bars. 0.5 hour
<TwoDaysAgoData> maximum age (in hours) of the statistics data for the Order completeness - Two days ago and the Overdue orders - Two days ago progress bars. 1 hour
<ActionData> maximum age (in hours) of the TAT data and of the statistics data for the Instrument status progress bar. 0.1 hour

Note

Both fullstops and commas can be used as decimal separators.

Gauges

As illustrated in the example above, the maximal values of the gauges in the Performance pane can be modified, using the elements <OrderGauge>, <ResultGauge> and the subelement <MaximalValue>.
