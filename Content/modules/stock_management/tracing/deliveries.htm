Stock management: Deliveries

Contents: Introduction - Delivery query - Delivery browser

Introduction

The stock management module allows tracing all product deliveries.  Deliveries are registered during product check-in. They can be accessed from the main menu via Start -> Stock management -> Deliveries.

Delivery query

Courier

Only deliveries from this courier.

Receipt from - Until

Only deliveries received within this date/time range.

Receipt user

Only deliveries received by this user.

Containing purchase order

When specified, only deliveries will be selected with a purchase order invoice record for the specified purchase order.

Delivery browser

When double-clicking a record in the delivery browser, the product check-in will be opened in read-only mode.

Generating a delivery report for individual products

To generate a delivery report containing the delivered products, a MISPL function ProductDelivery.ProductByNumber(Index) can be implemented. It returns specific (indexed) products of a product delivery.

The index indicates which product needs to be retrieved, based on the internal id in ascending order. Specify 1 for the first product, 2 for the second, etc. This function can be executed by running this MISPL on a product specification of a delivered article.

MISPL function to get the first product of a product delivery

STRING FirstProductOfDelivery;

FirstProductOfDelivery := .ProductByNumber(1).InternalId;

RETURN FirstProductOfDelivery;

MISPL function to get the complete product list of a product delivery

/* Build product list */

String ProductList, ProductInternalId;

Integer ProductCount;

Logical GoOn;

ProductList := "";

ProductCount := 1;

GoOn := TRUE;

WHILE GoOn

DO

ProductInternalId := .ProductByNumber(ProductCount).InternalId;

IF ProductInternalId = ?

THEN

GoOn := FALSE;

ELSE

ProductList := ProductList + ProductInternalId + ",";

ProductCount := ProductCount + 1;

ENDIF;

DONE;

RETURN ProductList;
