Mark products as expired

During product check-in, Glims allows setting an expiration date for a product.  However, when the expiration date is exceeded, currently Glims does not automatically set the product's status to expired.

A Product-based function "SetExpired" has now been added that can be used in a command/task to automate this process.  When creating a command, specify:

Table

Product

Query

Standard

Query parameter set

Tip

You can use a relative date indication in the "Expiration date" field to e.g. select all products of which the expiration day is "Yesterday".

Function

RunProcedure

Function parameter set

Select SetExpired as function.

Note

You can also manually set a product's status to "Expired" by using the menu function "Set expired" on a product record.
