Stock management: Logging

Contents: Introduction - Logged events - Overview - Other log actions

Introduction

Both the Product table and the Purchase order table have a log field that contains info about the main events in the history of a product or purchase order. This log can be visualized by a dedicated log viewer, which can be accessed from the contextual menu option View log on products and purchase orders. For purchase orders, the log is also available by clicking the "Log icon" (see product ordering).

Logged events

Products

Creation or update of the product log is triggered by the following events:

Product creation

Product discontinuation

Product status changes

Change of product Stock and/or Work place

Change of product External id, Lot, Production date, Expiration date, Comment

Usage of the Return to stock function

Purchase orders

Creation or update of the purchase order log is triggered by the following events:

Purchase order creation

Purchase order status changes

Purchase order discontinuation

Log info

The following info will be logged: the action, the user, the date and time of logging and an optional comment depending on the action.

Other log actions

Some events are also logged in other product fields. This info does not need to be viewed with the log viewer, but is visible in the product editor.

Check-in time and user

When the product passes to status Delivered.

Check-out time and user

When the product passes to status Check-out.

Discontinuation reason, time and user

When the product passes to status Discontinued.

Some events are also logged in other purchase order fields in non-decoded format. Most of this info is not shown in the purchase order editor, so it needs to be viewed with the log viewer anyway.

Creation time and user

When the purchase order is created. Shown in the purchase order browser, not in editor.

Approval time and user

When the purchase order passes to status "approved". Not shown in browser, nor in editor.

Sending time and user

When the purchase order passes to status "sent". Shown in the purchase order browser, not in editor.

Checking time

When the purchase order passes to status "checked". Not shown in browser, nor in editor.
