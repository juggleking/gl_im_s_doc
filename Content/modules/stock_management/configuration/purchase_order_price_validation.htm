Stock management: purchase order - price validation

Setting supplier minimal, maximal purchase order price and validation MISPL

For each supplier (organization), it is possible to specify a minimal and maximal purchase order price as well as a PurchaseOrder-based validation MISPL, returning a logical.

Optional display of minimal and maximal price in purchase order entry screen

In the purchase order entry screen, there is a button next to the total price, which can be used to set price display options (which are remembered in a user preference).

Display minimal price

If set, the minimal price is displayed next to the total price.

Display maximal price

If set, the maximal price is displayed next to the total price (can be combined with minimal price).

Mark too low prices

If set, the total price will be colored in red color.

Mark too high prices

If set, the total price will be colored in yellow color.

Total price validation during approval

When the purchase order is approved, the purchase order total price is validated against the minimal and maximal price defined for the supplier.

  • If a validation MISPL is set, then it will be executed.  If the condition is not satisfied, purchase order approval will be disallowed.
  • If no validation MISPL is set, a default validation will be applied: if the purchase order total price (*) is lower than the minimal price or higher than the maximal price, then a warning will be issued, but you will be allowed to proceed with the approval.  

    (*) with discount, but without VAT (cfr. MISPL function .TotalPrice(YES, NO))
