Stock management: Product specifications

Contents: Introduction - Fields - Browser - Query - Price currency - Data sheet

Introduction

This configuration table allows defining all products that will be processed by the stock and order management module. Product specifications can be defined via the main menu through Start -> Stock management -> Configuration -> Product specifications.

Fields

Image

The product specification editor contains an embedded product image, which can be updated by clicking on the image. When making use of images, make sure they are located on a shared network directory, accessible from every client in your network.

General fields

Name

Full name of the product.

Active

Allows indicating if this product is (still) in use.

Code

Short code for use where space is limited (e.g. labels)

Type/category

Refers to a product type defined in GLIMS. When selecting a product type with category Reagent, QC material, Blood product, Microbiology reagent or Microbiology QC material, a second field will be displayed which allows selecting an existing Reagent, QC material, Blood product, Microbiology reagent or Microbiology QC material record respectively. When selecting a Reagent or QC material record, its supplier will also be copied to the Supplier field of the product specification.

You can also use the Set type menu function to configure the type and category of multiple product specifications simultaneously.

Supplier

Refers to a correspondent of type Organization.

External reference no

Allows specifying a reference used in another stock program. (ex. SAP).

Manufacturer

Refers to a correspondent of type Organization.

Unit/packing unit/consumption unit

Please see the chapter The different units used in stock management.

Hazardous

Purchase page

Article no

Supplier's reference.

Price

Purchase price per packing unit in the specified currency.

Discount

Amount in euro will be subtracted from the price.

VAT

Can be specified, but is currently not yet used by the program.

Delivery period

Standard delay between ordering and delivery (in days). This is purely informational.

Minimum order quantity

Minimum purchase quantity in packings (packing = "packing unit " x "unit "). This will be checked by the purchase order program.

Storage page

Validity date

The function Set validity allows comparing this validity date with today and will set the product specification to active or inactive. This process can be automated by using the function in a command.

Storage period

Used to set the default expiration date of products during check in. It can be overridden manually or by choosing a lot with expiration date. Note that you still need to define a command to set product to status Expired when the expiration date has elapsed.

Perishable

If enabled, the product expiration date is mandatory during product check-in.

Requires product lot

If enabled,
  • The product lot must be specified (mandatory) when a product of this product specification is scanned (or confirmed) in the product check in screen. If no product lot is specified, but the purchase item has a product lot (product lot was set in the ordering phase), then this one will be proposed.

  • The Needs verification option of all the product lots linked to this specification will automatically be activated.

Storage conditions

Description of how the product should be stored.

Other page

Label

Product-based dynamic text that will be used as label content. Labels can be printed during product check-in.

Note

The Product.GetStorageLocation MISPL function (no parameters) gets the product's specification and work place, finds the corresponding ProductUsage record and returns the content of the site attribute ProductUsage.StorageLocation. This function can be used in the label text to return a product usage-specific storage location.

Print labels

Labels will only be printed if this option is enabled.

Comment

Free comment text.

The different units used in stock management

Product specifications have three different units:

Unit

This is the smallest unit in which the product specification is available.

Number of units per packing

When entering a purchase order, this will be the unit used for ordering product specifications. You can only order product specifications per packing and a packing = "number of units per packing" x "unit ".

Number of units per consumption

Number of units per check out. The number of products per stock check-out will always be a multiple of the number of units per consumption. When printing labels, a label will be printed per number of units per consumption.

Packing unit

A textual name for the packing (see example below).

Consumption unit

A textual name for the consumption (see example below).

Example

Consider a crate of Belgian beers:

Unit

The smallest unit for a beer is a bottle. You will need to define a new unit in the Unit table with name "1 bottle". The other Unit fields are of no importance.

Number of units per packing

The beers are shipped per crate (packing). A crate of beer contains 24 bottles. When your beer stock is low, you will purchase the beers by number of units per packing (i.e. one or more crate(s) of 24 bottles).

Number of units per consumption

The crate contains 24 bottles, but the bottles are strapped per 4 with a piece of cardboard. The minimum amount of beers that can be check-out of your stock is "4".

You can also define a name for packing unit and a consumption unit. In the example above the packing unit would be "Crate" and the consumption unit "Cardboard".

Browser

The main product specification browser can be accessed from the main menu through Start -> Stock management -> Configuration -> Product specifications.

Columns

Most columns are obvious, except for the following, which require some explanation:

Type

Name of product type (product type category).

Packing

Packing unit name (number of units per packing x unit).

Consumption

Consumption unit name (number of units per consumption x unit).

Other ways of accessing the product specification browser

The same product specification browser can be accessed in different ways. Columns are dynamically adapted according to the context:

  1. Main product specification browser (Stock management -> Configuration -> Product specifications).

  2. Product specifications of product type (Product type -> Product specifications).

  3. Product specifications of supplier (Organization -> Products -> Supplied by).

  4. Product specifications of manufacturer (Organization -> Products -> Manufactured by).

Query

The Options button in the browser allows to open the product specification query.

Price currency

  1. The list of supported currencies is to be defined by a site domain with name "Currency" and datatype Enumerated. Syntax: <currency 1 full name>\<currency 1 short name>,<currency 2 full name>\<currency 2 short name>, ...
  2. In the purchase order entry screen, the currency short name is displayed in a separate column. The total price also mentions the currency. The program allows adding items with a different currency, but in that case no total price and currency are displayed. This is not likely however, as the items are from the same supplier.
  3. In the product check-in screen, the currency is displayed in a separate column.
  4. The product specification data sheet and the purchase order form also mention the currency.
  5. The currency is also stored (as string) in the purchase item (during purchase order creation) and product (during check in).

Data sheet

The menu function Data sheet on product specification allows generating a detailed info sheet of a product specification in text, HTML or MS Word format.
