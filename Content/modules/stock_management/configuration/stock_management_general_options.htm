Stock management - General settings

Introduction

The stock management module can be tailored to the needs of a site using general settings available under Start -> System management -> Customize -> GLIMS Stock management. These general settings are presented below.

Fields

Options

Product internal id

Function that calculates the internal id for products.

Purchase order doc no

Function that calculates the document number for purchase orders.

Label printer

Default printer for labels printed during product check-in.

Checkout scan method

Determines if the product check-out screen should lookup the scanned product by internal id, external id or both.

Default work place

If set, the default workplace is specified as default in the purchase order editor, user interface to order products for a stock, check-in screen, check-out screen, and product usage editor, but it can be overridden.

On the WorkPlace table there is a field Default stock. If the default stock is specified, it will appear immediately when selecting the workplace. On selection of a workplace, the list of stocks are restricted to those where at least one product usage record is available for this stock and workplace.

Use default work place

If set, the default workplace will be shown in all screens as a read-only field. It cannot be changed manually.

Support unused products

If set you will be able to order products and check in products that are not used in the specified workplace and stock.

Support attachments

GLIMS allows to attach order forms to the purchase order.

Product bar code parser

Please check the chapter Advanced barcode scanning during product check-in and check-out.

Form printer

Please check the chapter Printer labels.

Default courier

Default courier for product check-in.

Check in trigger

This field is used to specify a configurable w_ProductCollection-based site function that must be executed when registering products. The w_ProductCollection table is a temp table which allows using product data that is not yet stored in the database, such as production date, expiration date, internal id, external id etc.
This trigger can be used, for instance, for product lot verification.

Check out trigger

This field is used to specify a configurable w_Product-based site function that must be executed when issuing products. The w_ProductCollection table is a temp table which allows using product data that is not yet stored in the database, such as production date, expiration date, internal id, external id etc.

This trigger can be used, for instance, for product lot verification or for stock level verification (see example below via the check-out trigger MISPL).

Example 

Message("Product: " + .Id.InternalId);
If .FirstOfSpecification = YES
THEN message("First of specification " + .Specification.Name);
ENDIF;
If .FirstOfPurchaseItem = YES
THEN message ("First of item " + .PurchaseItem.ProductSpecification.Name);
ENDIF;
If .FirstOfLot = YES
THEN message("First of lot " + .Lot.LotNo);
ENDIF;
If .FirstOfUsage = YES
THEN message("Available: " +  IntegerToString(.AvailableCount,"%d")  
+ "  Checked out: " +  IntegerToString(.CheckedOutCount,"%d")
+ " Warning: " + IntegerToString(.WarningCount,"%d") );
If .AvailableCount - .CheckedOutCount < .WarningCount
THEN message("Stock below warning level");
ENDIF;
ENDIF;
RETURN YES;
