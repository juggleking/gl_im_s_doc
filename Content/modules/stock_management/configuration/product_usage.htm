Stock management: Product usage

Contents: Introduction - Fields - Set usage - Remark

Introduction

A product usage record links a product specification to a work place, and determines the minimum and maximum stock amounts required for the specific work place. If specified and returning a valid number, the fixed amounts may be overruled by a site function, allowing implementation of a dynamic stock.

For general purpose products, a dummy work place may be created. This is however not required. The data model does not impose a ProductUsage. In that case no quantity requirements nor stock location is available however. Hence, automatic order form generation and ordering will not be possible for these products.

Fields

The product usage editor can be opened from the contextual menu of a product specification:

Work place

Work place that uses this product.

Stock

Stock in which this product is stored.

Minimum quantity

Minimum stock amount required for the specific work place.  This amount is expressed in the unit of the product specification and can be overruled by the amount returned by the minimum quantity function.

Minimum quantity function

ProductUsage-based, Integer-valued site function. Overrules the fixed minimum quantity amount.

Calculated minimum

Calculated field that shows the actual minimum quantity that will be applied.

Maximum quantity

Maximum stock amount required for the specific work place. This amount is expressed in the unit of the product specification and can be overruled by the amount returned by the maximum quantity function.

Maximum quantity function

ProductUsage-based, Integer-valued site function. Overrules the fixed maximum quantity amount.

Calculated maximum

Calculated field that shows the actual maximum quantity that will be applied.

Standard order quantity

When creating a new purchase order, articles are added with the standard order quantity by default.

Warning threshold

Amount below which a warning will be issued, i.e. the last columns of the product specification browser will be shown with a yellow background.  This threshold must be specified as a percentage within the minimum/maximum quantity range.  Example:

  • Minimum quantity = 5

  • Maximum quantity = 20

  • Warning threshold = 10

Range => 20 - 5 = 15.
Threshold => 10% of 15 = 1.5
Threshold value = 5 + 1.5 = 6.5 (rounded to 7)   

Warning threshold function

ProductUsage-based, Integer-valued site function. Overrules the fixed warning threshold.  If specified, the evaluated value of this function is used to calculate the warning threshold.  If not specified, the fixed formula applied to the specified "Waning threshold" is used (see above).

Calculated threshold

Calculated field that shows the actual amount for the warning threshold.

Set usage

In order to be able to configure the product usage of multiple product specifications simultaneously, a function Set usage is available in the product specification contextual menu.

  1. Both the work place and stock are mandatory.
  2. Calculated fields are not shown in this screen.
  3. Existing product usage records (for the specified work place) will only be updated if the option Update existing is enabled.

Remark

The product specification contextual menu Set usage also allows creating a new product usage or update an existing product usage (enable the option Update existing).
