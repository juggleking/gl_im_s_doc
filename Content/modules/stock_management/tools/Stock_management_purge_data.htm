Stock management: purge data

A tool is available to purge stock management data in GLIMS.  This can for instance be used to clear all stock management data after a testing period.

Starting the program

The program can be started via Start -> Development -> 4GL -> Run procedure.  Specify "ap_clean_stck" as "File name".  The program

  1. Will ask for a password confirmation.

  2. Allows to test the deletion.

  3. Logs all actions.

Which tables are purged

  1. Product
  2. ProductLot
  3. PurchaseItem
  4. ProductDelivery
  5. Delivery
  6. PurchaseOrder
  7. Stock
  8. ProductSpecification
  9. ProductType
  10. ProductUsage

The references from a work place to a stock will be removed.
