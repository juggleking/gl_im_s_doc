Stock management: Product ordering

Contents: Introduction - Purchase order entry - Product order list - Order unused products - Automated product ordering

Introduction

GLIMS offers different methods for ordering products:

The "Purchase order entry" screen

This screen allows to order products from a single supplier, for a specific work place and stock.

The "Product order list" screen

Many purchase orders, for different work places, stocks and suppliers, can be created in one operation.

Automatic product ordering

Allows automatic product ordering when the available stock for a given product is low.

Purchase order entry

The Purchase order screen can be accessed from the main menu via Start -> Stock management -> Purchase order entry. It is used to create, edit and process purchase orders for a specific supplier, work place and stock.

Description

Supplier selection

By default, GLIMS will first show a list of suppliers. Select a supplier and the purchase order entry screen will be displayed. This main menu function can be pre-configured which allows to add menu items with a predefined supplier. This allows to open the purchase order entry screen immediately without having to select a supplier first.

Tip

If you want to open the purchase order entry screen without supplier: create a dummy supplier (organization with customer no), add a pre-configured menu item with this supplier and then delete the supplier.

Main fields

Supplier

Allows to select the supplier of the products to be ordered. This field becomes read-only as soon as entering the embedded purchase item browser.

Work place

Work place where products must be checked in. This field becomes read-only as soon as entering the embedded purchase item browser.

Cost location

The cost location of the default work place / selected work place. The user can specify a different cost location than the one proposed.

Stock

Stock where products must be checked in. This field becomes read-only as soon as entering the embedded purchase item browser.

External comment

Comment that may be printed on official documents (e.g. order form).

Product specification image

Image of the selected product specification, if available.

Urgency

This information is stored with the purchase order, but is currently not used by the program. The possible values must be defined as a site domain with name "PurchaseOrderUrgency".

Product specification browser

This browser is used to specify which products you want to order. It allows adding, updating and deleting product specifications (which upon confirmation are registered in the database as purchase items). This editable browser contains the following columns:

Article no

Read-only column which is synchronized with the product specification name.

Name

Select the exact product specification name by pressing the Insert key and opening the product specifications in use browser. Only product specifications with usage in the specified work place and supplied by the specified supplier can be chosen.

Min exp date

Allows specifying a minimal expiration date which can be printed on the order form. It is not yet used in any other way by the stock management module.

Lot no

Allows selecting a product lot. Only lots of the specified product specification can be selected.

Comment

Allows supplying a purchase item-specific comment text.

Ord

Allows specifying the number of ordered items in number of units per packing. The minimal purchase order quantity is specified by default and a warning is given if the order quantity is lower or higher than the minimal/maximal order quantity.

Price

Read-only column displaying the product specification's price in number of units per packing, in which the discount is taken into account.

Total

Read-only column displaying price * quantity.

Currency

As specified in the product specification.

Note

A double-click on any read-only column of the browser will open the product specification editor in view mode.

Total price

Total price of all browser items, the sum of the values in the Total column.

Buttons

Insert new purchase items in the browser. You can also use the "Insert" key.
Remove purchase items from the browser. You can also use the "Delete" key.

Embedded "Billing" browser

The embedded Billing browser will only have records for purchase orders with status Partial or Complete. It displays the invoice and payment information of the purchase order invoice records that exist for the purchase order.

Notes

  • The invoice and payment information (invoice no, invoice date, payment no, payment date, cost location) cannot be updated in the purchase order screen. This information should be supplied during check in and can be modified via the Purchase order check function.
  • There is no link with GLIMS' existing Invoice and Payment tables.

Usage

This chapter describes the workflow for ordering products:

  1. The supplier, work place and stock must be specified first. If you want to order without worrying about the supplier, use the Order products for work place screen.
  2. Select the product specifications in product specifications browser by pressing Insert.
  3. Specify the quantity for each purchase item in the Ord column by clicking into the cell and confirming by pressing Enter.
  4. Optionally specify lot no, minimal expiration date and comment by using the same mechanism as described above.
  5. Optionally specify any further information in the purchase order creation screen.
  6. Push the Create button to create a purchase order or Update to update an existing one.
  7. Use the processing buttons to process the purchase order to the next status.

Processing

When creating a new purchase order, its status is set to Scheduled and the associated purchase items will be created. For each purchase item new product records in status Scheduled will be created. The number of created products equals:

  • Purchase item quantity (in packing units) * packing unit / consumption unit.

The stock and work place of the created products will be set to the purchase order's stock and work place.

On update, new purchase items may be added to the purchase orders, deleted items may be discontinued. If the ordered quantities have changed, the number of products will be adjusted accordingly.

It is not possible to modify the purchase item amount of a purchase order in status Sent and higher. It is also not possible to discontinue a purchase order from within the purchase order editor, since it cannot be saved without any purchase items.

When discontinuing all the purchase items of a purchase order via the product order list, the purchase order itself will be discontinued.

When a purchase order is discontinued in the Approved status, the status is demoted to Scheduled. A discontinuation reason must be entered, which will be added to the purchase order log.

Once your purchase order is final, you can approve it, generate an order form, check and close it with the following processing buttons:

Approve

This button is only active if the purchase order has status Scheduled. It allows promoting the purchase order's status to Approved. It also allows changing the urgency.

Note that only persons that have the privilege to approve purchase orders (privilege for function "PurchaseOrder.Approve") are allowed to update approved purchase orders. For users that don't have this privilege:

  • All purchase order fields, the product specification browser and the add/delete buttons are disabled in the purchase order browser (viewing the log, generating order forms and discontinuing are still allowed, unless restricted by another privilege).

  • Purchase items belonging to approved purchase orders are read-only in the "Order products for stock/workplace" screen.

Generate form

This button is only active if the purchase order has status Approved. It allows generating an order form for the purchase order and changes the status to Sent if the order form generation option Mark as sent is set.

For more info, please check the chapter Order forms.

Check

This button is only active if the purchase order has status Partial or Complete (after product check-in). It promotes the purchase order invoice status to Checked while allowing to update the purchase order invoice fields Invoice no, Invoice date, Payment no, Payment date and Cost location for the selected Delivery. When all purchase order invoices are Checked, then the purchase order status will also be set to Checked.

When promoting partial purchase orders to status Checked:

  • A reason must be specified when checking the last purchase order invoice. This reason is added to the purchase order's external comment.
  • All associated products in status Ordered are discontinued. The discontinuation reason for these products is set to the reason specified during checking.

Close

This button is only active if the purchase order has status Checked. It sets the purchase order's status to Closed.

Discontinue

Discontinues the entire purchase order:

  1. You can only discontinue purchase orders in status Scheduled, Approved or Sent.
  2. A discontinuation reason must always be specified. It will be added to the purchase order log. The discontinuation reason of all associated products will be set accordingly.
  3. In case of discontinuing a purchase order in status Sent, a warning will be issued stating that the supplier should be warned.

History

Shows the purchase order's history (status changes).

The functionality of the buttons described above is also available through the contextual menu on Purchase order.

Remarks

For purchase orders with status > Sent, the entire screen will be shown in read-only mode (except for the Log button).

Usage

This chapter describes the workflow for ordering products:

  1. Either you specify a work place and/or stock or you enable the option All stocks and work places.
  2. Use the Insert icon to add a product specification.
  3. Specify the quantity of each item by clicking into the Quantity column cell.
  4. Optionally specify lot no and comment.
  5. Confirm by pressing OK to create new purchase orders or update existing ones.

Processing

When the screen is saved, the browser may contain items that belong to existing purchase orders and others that have been added.

  1. The items that belong to existing purchase orders are updated, i.e. the quantities of purchase items are updated and the number of associated products is adjusted. Removed items will be discontinued for the existing orders; the associated products will be discontinued as well. If all items of a purchase order are discontinued, the purchase order is also discontinued (only possible for purchase orders in status Scheduled or Approved). A discontinuation reason must be specified, which is logged under Product.DiscontinuationReason.

  2. For newly added items the program will search for existing purchase orders to place the items on. It will search for the purchase order with the highest doc no (should be the most recent) with status Scheduled of the specified stock and work place. If a purchase order is found, it is updated.

    If no purchase order can be found, a new one with status Scheduled is created for the specified stock and work place. For each purchase item new product records in status Scheduled will be created. The number of created products equals:

    purchase item quantity (in number of units per packing) * number of units per packing / number of units per consumption

The stock and work place of the created products will be set to the purchase order's stock and work place.

Order unused products

GLIMS allows ordering products that are not used in a specified workplace/stock:

  1. The product specification query has an option Unused only to restrict the product specifications to those that have no product usage.
  2. If the general option Support unused products is set, the purchase order screen and product order list screen will offer an extra button for ordering products that are not in use.
  3. This extra button is enabled if the insert button is enabled, except in the product order list user interface: there stock and work place should be specified. When clicking this button, the product specification browser with option Unused only will be opened. Further processing is identical to processing of product specifications with usage.

Automated product ordering

Introduction

The stock management module also supports automatic product ordering when the available stock for a given product is low. To set this up, you need to schedule a command that selects all product usages for which the stock is low and then perform the function ProductUsage.OrderProducts.

Configuration

Define a new command as follows:

Table

ProductUsage

Description

Free text

Query

Standard

Query parameter set

Allows selecting the product specification for which you want to order products automatically.

One of the most important selection criteria will undoubtedly be the Maximal ordered state. This option allows selecting only the product specifications for which the scheduled + ordered quantity is lower than the alarm or threshold status.

Note

The option Maximal state only takes the available quantity into account.

Function

OrderProducts

Function parameter set

Order quantity

Base quantity

Determines the number of products that will be ordered. This number can be diminished by using the options below.

Subtract scheduled / ordered / delivered products

If the set, the number of ordered products (as determined by the "Base quantity") will be subtracted by the number of already scheduled / ordered / delivered products.

Purchase order

Append to existing purchase order

If set, GLIMS will look for the most recent purchase order in status Scheduled of the same supplier, stock and workplace and will update the items of this purchase order. If disabled, a new purchase order will be created.

Automatic approval

If set, automatically created purchase orders will automatically promoted to status Approved. If set, you can also enable the options below to generate an order form and hence mark the purchase order as sent.

Generate fax form

If set, and the supplier's form type is set to "fax" as well, then the fax form is created and the purchase order status is increased to Sent.

Generate and send email form

If set, and the supplier's form type is set to "email" as well, then the email form is created and sent to the supplier and the purchase order status is increased to Sent.

Generate and print paper form

If set, and the supplier's form type is set to "paper" as well, the form is generated and printed on the specified printer and the purchase order status is increased to "sent".

Remarks

  • Automatically created/approved/sent purchase orders get an internal comment "Automatically created/approved/sent".
  • The program creates a log file in which relevant info is logged. In batch mode this info will appear in the cron log file.
