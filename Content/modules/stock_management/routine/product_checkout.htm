Stock management: Product check-out

Contents: Introduction - Check-out purpose -Check-out screen -Processing

Introduction

This product check-out screen is used to check out delivered products for a specified work place and stock. In fact it is more general: it also allows to transfer delivered products to another stock and/or work place.

Check-out purpose

The product check-out screen can be accessed from the main menu via Start -> Stock management -> Check out.  A screen will be displayed which allows indicating why you want to check-out a product. Depending on the selected check-out purpose, the actual check-out screen will be customized.  This allows to configure different menu options for different check-out purposes.

Check out

Choose this one to start using the selected products in the specified work place. The status of the product will change to "Checked out".

Register loss

Choose this one to mark the selected products as lost (broken, etc...).  The status of the product will change to "discontinued".

Transfer

Choose this one to transfer the selected products to another stock and/or work place. The status of the product will not change. Afterwards these products should be checked in into the new stock/work place.

Exchange

Choose this one to mark the selected products to be sent back to the supplier. The status of the product will change to "Exchange".  No mechanism to send back the products has been implemented yet.

Check-out screen

There are two ways to indicate which products you want to check out:

By scanning the products

Scan by

Package

A "package" is defined as a number of products with the same specification, external id and delivery. This is in fact determined during product check-in.

If the scan option by package is chosen, then all products with the scanned external id will be added to the browser. If more than one package (= products belonging to different deliveries) is available, you will be offered the possibility to choose one. The product browser will show only one product of each package. All products of the package are supposed to have the same expiration date.  If an older package can be found, a similar browser to allow selection will pop up.

Product: scan

Individual products (which can be more than one physical product, based on the number of units per consumption).

Scan field

Allows entering/scanning the product by internal or external id.  The general option Scan method determines if the scanned product should be looked up by internal id, external id or both. When scanned (or filled in, followed by RETURN), the product appears in the product browser. 

The following checks will be performed when scanning a product:

  • If more than one product with the same external id is available, a list will be presented to select from.
  • If the scanned product is not optimal (products with lower expiration date are available), then a list with all products of the same specification, work place and stock will be presented to select from. This check is only performed in case of a real "check out" action (not "loss", etc...).

By manually selecting the products

Browse products

Enable this field to manually lookup products to be checked out.

Work place

Work place where products must be checked out.

Stock

Stock where products must be checked out. 

Specification

Product specification of the product to be checked out.

Product

Allows selecting products that match the specified work place, stock and specification.

Product browser

The product browser contains all products to be effectively processed (checked out, transferred, ...). This browser may contain a selection of products from different work places, stocks, specifications...  You can double-click any row in the browser to open the product editor. You can sort the browser by clicking the column header.

The browser has the following columns:

Name

The product specification's name.

Internal id

The product's unique identifier, which should be found on the label.

External id

Lot

The product's lot number.

Expires

The product's expiration date, mandatory for perishable products.

Work place

Stock

The last two columns are displayed with a yellow or red background if the number of products is below the warning or alarm level respectively. 

Other fields

Target stock

Stock where the products must be transferred to. Only enabled if option transfer is chosen. (see above).

Target work place

Work place where the products must be transferred to. Only enabled if option transfer is chosen. (see above).

Process reason

Comment that will be logged with the product. 

  • In case of check-out, this comment will appear in the products' checkout reason field.

  • In case of loss, this comment will appear in the products' discontinuation reason field.

Processing

  1. Processed products will get another status, depending on the process purpose (see above). Except for transferred products, these products will then no longer be "available" in the stock. The quantities in the product specification browser will decrease. After check out, products are no longer followed up, i.e. the consumption of the product in the work place is not logged.  

  2. If

    • a product is low in stock (warning or alarm), and
    • no Check-out trigger is defined,

    a warning message is displayed when the product is checked out.

  3. During check-out, an overview screen is shown with information about the checked-out products , including the stock warning / alarm level information. 

    The "Review" button in this screen allows you to print this overview. 

    Example

  4. Depending on the type of lot (QC, reagent, etc), the check-out of a product can lead to the activation or the creation of a lot.
