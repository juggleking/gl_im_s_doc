Product specifications in use

Content: Introduction - Browser - Query - Functions - Adding a pre-defined menu option / ribbon item

Introduction

The browser "Product specifications in use" can be accessed from the main menu via Start -> Stock management -> Product specifications in use.  When looking up a product specification during the purchase order creation, product check-in or product check-out, the same browser is shown.

Browser

Most columns are obvious, except for the following, which require some explanation:

Packing

Packing unit name (number of units per packing x unit)

Min, Max, Low columns

Represent the minimum quantity, maximum quantity and calculated threshold as defined in the product usage.  

"Scheduled", "Ordered" and "Available" quantity columns

These are aggregate fields, maintaining the situation of the related products. They are automatically updated along with product status changes or creation of new products. Based on the information found in the ProductUsage table, these columns may get an alternate background color, indicating a warning or an alarm.

Stock quantities below the minimal stock level are marked in red; quantities below the warning threshold (as defined in the product usage) in yellow. Note that, if several work places share the same product specification (i.e. they are using the same products), their minimal and maximal amounts are accumulated.

Query

Product specification

Selection on a specific product specification.

Product type

Product type to which the list should be restricted.

Supplier

Supplier (Organization) to which the list should be restricted.

Manufacturer

Manufacturer (Organization) to which the list should be restricted.

Name pattern

A character expression that you want to match with the product specification name. The pattern can contain wildcard characters: a period (.) in a particular position indicates that any single character is acceptable in that position; an asterisk (*) indicates that any group of characters is acceptable, including a null group of characters.

Code

Stock

Work place

When specified, only these products will be shown that are being used by this work place (in other words, for which corresponding ProductUsage records can be found).

Department

When specified, only these products will be shown that are being used by this department (via the WorkPlace.Department link).

Maximal state

One of "Alarm", "Low", "Normal". Allows selecting only these product specifications that require attention.

Maximal ordered state

Selecting only the product specifications for which the scheduled + ordered quantity is lower than the alarm or threshold status.

Filter

Allows to select a ProductUsage-based MISPL function to filter the "Product specifications in use". Example of such a MISPL function:

RETURN (.Stock.Mnemonic = "MIPSN" or .Stock.Mnemonic = "IntStock1");

Functions

The contextual menu function Show products shows scheduled, ordered and delivered products of the specified work place and stock.
