Logistics manager - Search functionality

In the Logistics manager, it is possible to look for a specific box or specimen or for packages on the basis of various criteria.

The search screens can be opened via the Start menu, as illustrated here:

Find a box

To look for a box, enter its (case-sensitive) barcode in the Enter a box number field. The screen then displays all the information concerning this box and the last package for which it has been used: box number, origin and destination department, preparation date, etc.

Find a specimen

To look for a specimen, enter its barcode, internal id or external id in the Enter a specimen identification field. The screen then displays:

  • all the information related to this specimen: the material it contains, its last location, it storage and home location, etc.
  • if the specimen is part of a package, the information concerning this package
  • the audit of the specimen: when it was packed / collected / delivered / unpacked and by whom.
Notes

Note

  1. Barcodes, internal and external ids are case-sensitive.
  2. When an identification is entered in the Enter a specimen identification field, GLIMS tries to identify it by looking for a matching identification in the SpecimenIdentification table. If it does not find any, it looks for a matching internal id in the Specimen table. And if there is no matching internal id, GLIMS looks for a matching external id in the Specimen table. If GLIMS cannot identify the barcode at all, the corresponding specimen is not added to the list.

    The following situation is thus theoretically possible: say there is a specimen with barcode 456 and a specimen with internal id 456. If a user looks for the specimen with internal id 456, the system will return the data of the specimen with barcode 456, not that of the specimen with internal id 456.

Overview of boxes

Clicking the Overview of boxes option opens the Package query screen. This screen is composed of two tab pages:

  • Normal: typically used to look for active packages.
  • Advanced: typically used to look for historical data and inactive packages.
