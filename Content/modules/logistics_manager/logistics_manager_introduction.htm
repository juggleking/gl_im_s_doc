Logistics manager - Introduction

Utility of the Logistics manager

Specimens often need to be transported, from the department where they are packed to the department where they will be analyzed. The Logistics manager module makes organising and monitoring this transport easier:

  • It guides users through the logistics workflow of specimen packages, from their preparation to their unpacking.
  • It makes it possible to pack and unpack several specimens at once instead of scanning them individually.
  • It saves and enables the query of all the specimen and transport data, ensuring traceability.

The use of the Logistics manager is subject to a license: GLIMS_LM.

Getting started

The Logistics manager is available via Start -> Logistics. Several options are available in the menu:

To open the initial page of the Logistics manager, choose Home screen.

To directly start with one of the four steps of the logistics workflow, click the corresponding label: Prepare box, Collect boxes, Deliver boxes or Unpack box.

To look for a specimen, a box or a package , click Find specimen, Find box or Overview of boxes respectively.

To configure a route for a package, click Configuration -> Routes.

Terminology

The terms box and package have a slightly different meaning:

  • box: container used to pack and transport specimens. A box can be re-used: when a package has been unpacked, the empty box can be used again to pack and transport new specimens. A box has a fixed, case-sensitive barcode which remains the same even if the box is used for several packages.
  • package: whole composed of a box and the specimens in it. A package is unique.

Among packages, it is also necessary to distinguish active from inactive packages:

  • active: a package is active if it is currently being prepared, collected, transported, delivered or unpacked.

  • inactive: a package can be set to inactive by GLIMS in two cases:

    • it has reached the end of the logistics workflow, i.e. it has been registered as unpacked.
    • it has been aborted: when a package has not been registered as unpacked, GLIMS cancels it before allowing the re-use of its box.

Saving data and closing screens

Every time a user performs an action in the Logistics manager, the data is saved into the database. The screens of this module can thus be closed at any time, no data will be lost. This also means that no specific action is required to save the data at the end of a logistics step: there is for instance no "Save" button to click once the preparation of a package is finished.
