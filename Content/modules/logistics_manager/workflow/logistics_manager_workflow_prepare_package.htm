Logistics manager - Prepare package

The first step in the logistics workflow is the preparation of a package. The Prepare package screen guides you through this step. To open it, go to Start -> Logistics manager -> Prepare box.

The following illustration of the Prepare package screen shows how to proceed:

  1. Scan a box number.

  2. Register the specimens that will go into the box.

  3. The information related to the registered specimens is then shown in the specimen table.
  4. Afterwards, you can also
    • print an overview of the displayed data.
    • empty the package.
    • prepare the next package.
    • re-open a prepared package.

Scan a box number

The box number is the (case-sensitive) barcode that identifies the box that will be used for the package. Scanning it is mandatory: if the window is closed before scanning a box number, the entire Prepare package screen closes. This ensures that a box is always identified before being filled with specimens.

When a box number is scanned, a record is automatically created in the GLIMS database. No configuration is thus required beforehand.

Register the specimens

Scanning or adding from a route

Once the box number is scanned, the specimens that will go into this box need to be registered. There are two ways to do so:

  • Scan the (case-sensitive) barcode of each specimen individually:

    When a specimen is scanned, GLIMS tries to identify it by looking for a matching identification in the SpecimenIdentification table. If it does not find any, it looks for a matching internal id in the Specimen table. And if there is no matching internal id, GLIMS looks for a matching external id in the Specimen table. If GLIMS cannot identify the barcode at all, the corresponding specimen is not added to the list.

  • Add specimens from a route:

    The button Add specimens from route on the right-hand side enables you to register in one go all the specimens linked with a specific route and whose status is inferior or equal to Available.

    Warning

    Warning

    It is possible to add specimens from several routes. However we advise you not to do so: different routes may have different destinations and maximum transport durations. Adding specimen from several routes might thus lead to specimens being delivered to a wrong department and/or too late.

When a specimen is added to a package, the location in which the package preparation is taking place is displayed together with the box number in the Last location field of the specimen editor. If a specimen is deleted after having been added to the package, the location and box number remain displayed in the Last location field.

Determining the package destination

While the specimens are being registered, GLIMS determines the destination of the package. GLIMS does this in the following way:

When a specimen is registered, i.e. scanned or added from a route, GLIMS looks of which procedures this specimen is an input. GLIMS then draws a list of the departments of these procedures. As soon as a list only contains a single department, GLIMS takes this department as the destination of the package being prepared. In other words, the destination of the package is the department of the first registered specimen that serves as procedure input in a single department.

Examples

Examples

Situation 1:

A package is being prepared. First, Specimen A is registered and placed into the box, followed by Specimen B and specimen C.

Specimen A is the input of two procedures: procedure P, which is to be executed in department P', and procedure Q, which is to be executed in department Q'.

Specimen B is the input of two procedures: procedure R and procedure T, which are both to be executed in department R'.

Specimen C is the input of a single procedure: procedure U, with department U'.

--> Specimen B is the first registered specimen which will serve as procedure input in a single department. Hence the destination of the package will be department R'.

Situation 2:

Same as use case 1 but Specimen C is registered before Specimen B.

--> Specimen C is the first registered specimen which will serve as procedure input in a single department. Hence the destination of the package will be department U'.

Situation 3:

Every registered specimen is input of several procedures which are to be executed in different departments.

--> None of the specimens is a procedure input for a single department, so the package will not get any destination.

Warning

Warning

Once the destination of a package is determined, it is not possible to change it anymore.

Overview of package information

While the specimens are being registered, the Scan a specimen area automatically displays the following elements:

  • the box number
  • the number of specimens that have been registered until now (and thus placed inside the box)

  • the mnemonic of the destination of the package (once the destination has been determined).

Specimen table

Once a specimen is registered, the following information is displayed in the overview table:

  • Specimen: barcode of the scanned specimen OR internal id in case of a specimen added via a route.
  • Material: mnemonic of the material of the registered specimen.

  • Info: if a registered specimen is not a procedure input in the destination department of the package being prepared, this column will indicate that the destination(s) of the specimen do(es) not match the destination of the package. In that case, the specimen in question will probably need to go to a different department, and does not need to be placed into this box. Note that this piece of information is not logged.

    Example

    Example

    Let's go back to Situation 1 described above. In this case, the following message will be displayed in the Info cell of Specimen C:

    The destination of the specimen, U', does not match the destination of the package, R'.

To delete a specimen in the table, and thus take it out of the package, select it and press the Delete key. To delete several specimens at once, press the Ctrl key while selecting the specimens.

Further actions

Once the specimens are registered, you can use the buttons on the right-hand side of the screen to execute the actions described below.

Print an overview

Click the button Print document to launch the print functionality of the Logistics manager and print an overview of the data displayed on the screen. The printer used to print the overview is that of the department where the Prepare package step is being executed.

Empty the package

Click the button Empty this package to remove all the specimens from the specimen table. The destination of the package will also be reset.

Prepare next package

Click the button Prepare next package to end the preparation of the package and open a new Prepare package screen.

Re-open a prepared package

Prepared packages can be re-opened, as long as they have not been collected by a courier. To re-open a package, re-scan the number of its box. All the specimens contained in the box then appear in the specimen table and the Prepare package screen can be used normally.
