Logistics manager - Unpack package

The fourth step in the logistics workflow is the unpacking of the package. The Unpack a package screen guides you through this step. To open it, go to Start -> Logistics manager -> Unpack box.

The following illustration of the Unpack a package screen shows how to proceed:

  1. Scan the box number of a package.
  2. The specimens contained in the package are then displayed in the Remaining specimen table.
  3. Unpack the specimens.
  4. The unpacked specimens appear in the Unpacked specimen table.

  5. Afterwards, you can

    • select specimens and enter unpacking comments.
    • unpack the next package.
    • finish unpacking.
Note

Note

Closing the screen is not possible if the specimens of the package have not all been unpacked, i.e. if the Remaining specimen table still contains specimens.

Scan a box number

The box number is the (case-sensitive) barcode of the box used for the package. Scanning it is mandatory: if the window is closed before scanning a box number, the entire Unpack package screen closes.

Remaining specimen table

Once a box number has been scanned, all the specimens contained in the box appear in the Remaining specimen table. This table has two columns:

  • Remaining specimen: barcodes or internal ids of the specimens registered during the preparation of the package and that are (still) in the box.
  • Material: mnemonics of the materials of the specimens that are (still) in the box.

Unpack the specimens

Once a box number has been scanned, the specimens contained in the box can be unpacked. There are two ways to do so:

  • Scan the (case-sensitive) barcode of each specimen individually.
  • Unpack all the specimens in one go without scanning them. To do so, click the Unpack everything button on the right-hand side. All the specimens move from the Remaining specimen table to the Unpacked specimen table.

The Scan a specimen area then shows the following information:

  • the box number
  • the number of specimens that are still in the box
  • the number of specimens that have been unpacked.

Unpacking a specimen makes it available (specimen status = Available) in the department where the unpacking takes place. This department must be the destination department of the package. If this is not the case, a mistake occurred during a previous step of the logistics workflow.

Note

Note

Normally, when a specimen is scanned, it is automatically confirmed if this was not yet the case. However, the general setting Confirm specimen on scan allows limiting the automatic confirmation to specific situations.

When a specimen is unpacked, the location in which the unpacking takes place is registered and displayed together with the box number in the Last location field of the specimen editor.

Unpacked specimen table

When a specimen is unpacked, it switches from the Remaining specimen table to the Unpacked specimen table. This table shows the following information:

  • Unpacked specimen: barcode or internal id of the specimen that has been taken out of the box
  • Material: mnemonic of the material of the specimen that has been taken out of the box
  • Info: warnings about potential errors. These pieces of information are not logged.
  • Comment: comments entered during the unpacking process (see below).

Further actions

Once the specimens of the package have been unpacked, you can use the buttons on the right-hand side to execute the actions described below.

Enter specimen comments

It is possible to write comments about the specimens displayed in the Unpacked specimen table. To do so, select at least one unpacked specimen and click the Specimen comment button. In the window that opens, enter text manually or press the Insert key to insert texts based on the 'Package' table. The comments are then saved in the SpecimenAudit table.

The Specimen comment button can be used more than once. If several comments are entered successively for a specimen, they are appended to each other.

Unpack the next package

Click the button Unpack new box to end the unpacking of the package and open a new Unpack a package screen.

Finish unpacking

Once all the specimens are unpacked, click the Finish button to end the unpacking process and close the screen.

If you click Finish although the specimens are not all unpacked, i.e. there are still specimens in the Remaining specimen table, you will be asked the following question:

There are still specimens that are not scanned, do you want to mark them as lost?

If you answer YES, all the non unpacked specimens will be logged as lost. If you answer NO or Cancel, the Unpack package screen reappears. You can then either scan the yet unscanned specimens or use the Unpack everything button.

Note

If a specimen is marked as lost, the Last location field of its editor remains filled with the value set during package preparation.
