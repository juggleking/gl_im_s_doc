Logistics manager - Collect package

The second step in the logistics workflow is the collection of the package. The Collect packages screen guides you through this step. To open it, go to Start -> Logistics manager -> Collect boxes.

The following illustration of the Collect packages screen shows how to proceed:

  1. Scan a courier identification.
  2. Scan the box numbers of the packages that the courier is collecting.
  3. The information related to scanned packages is then shown in the package table.

  4. Afterwards, you can

    • print an overview of the displayed data.
    • clear the package table.
    • collect the next package.

Scan courier identification

The courier identification is the (case-sensitive) barcode that identifies the courier. Scanning this identification allows a better monitoring of the transport but is not compulsory. You can thus leave the field blank and press the Enter key or close the window.

When a courier identification is scanned, a record is automatically created in the GLIMS database. No configuration is thus required beforehand.

Scan the box numbers

Once the courier identification is scanned (or left blank), scan the box barcodes of the packages that are being collected. The Scan a box number area then automatically displays the following elements:

  • courier identification
  • number of packages that have been scanned until now.

Package table

When a box number is scanned, GLIMS looks for this barcode and checks the status of the package. The following information is then displayed in the overview table:

  • Box number: barcode of the box of the scanned package
  • Destination: mnemonic of the department where the package should be delivered
  • #Specimens: number of specimens in the package
  • Info: can display warnings such as the ones listed below. Note that these pieces of information are not logged.
    WarningDescription
    Old package had wrong status, created new one.

    The status of the package is higher than the expected status. In the system, the scanned package is aborted, set to Inactive and a new package is created.

    This can happen at this stage in the following (unlikely) situation: specimens were delivered and removed from a box but the unpacking was not registered in GLIMS. In GLIMS, the package is thus delivered but not unpacked. Later on, the empty box is used to pack new specimens but this is not registered in GLIMS either. When the courier wants to register the collection of this new package by scanning the box number, GLIMS displays the warning because the box is still linked with a non unpacked package.

    Box number not found, created new package.

    This message appears when no box with this barcode is currently linked to an active package.

    This can happen at this stage in the following situation: a courier collects a package whose preparation has not been registered in GLIMS. As the preparation of a package requires scanning the number of the box of the package, if this step is skipped, GLIMS does not know that this box is being used for this package.

To remove a package from the table, select the corresponding box number and press the Delete key. Press the Ctrl key while clicking on the box numbers to select several packages.

Further actions

Once the packages are scanned, you can use the buttons on the right-hand side of the screen to execute the actions described below.

Print an overview

Click the button Print document to launch the print functionality of the Logistics manager and print an overview of the data displayed on the screen. The printer used to print the overview is that of the department where the Collect packages step is being executed.

Clear the package table

Click the button Clear to empty the entire package table. It will be as if the courier had not collected any package.

Collect the next package

Click the button Next collection to end the collection of the package and open a new Collect packages screen.
