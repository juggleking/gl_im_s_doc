Logistics manager - Deliver package

The third step in the logistics workflow is the delivery of the package. The Deliver packages screen guides you through this step. To open it, go to Start -> Logistics manager -> Deliver boxes.

The following illustration of the Deliver packages screen shows how to proceed:

  1. Scan the courier's identification.
  2. Scan the box numbers of the packages that the courier is delivering.
  3. The information related to the scanned packages is then shown in the package table.
  4. Afterwards, you can enter delivery remarks and deliver other packages.

Scan courier identification

The courier identification is the (case-sensitive) barcode that identifies the courier. Scanning this identification allows a better monitoring of the transport but is not compulsory. You can thus leave the field blank and press the Enter key or close the window.

When a courier identification is scanned, a record is automatically created in the GLIMS database. No configuration is thus required beforehand.

Scan a box number

Once the courier identification is scanned (or left blank), scan the box barcode of the packages that are being delivered. The Scan a box number area then automatically displays the following elements:

  • courier identification
  • number of packages that have been scanned until now.

Package table

When a box number is scanned, its barcode and the status of the package are checked. The following information is then displayed in the overview table:

  • Box number: barcode of the box of the scanned package
  • #S: number of specimens in the package
  • Origin: mnemonic of the department where the package was prepared
  • Destination: mnemonic of the department where the package should be delivered

  • Duration: time elapsed between the collection of the package and its delivery (in hours and minutes). If this time exceeds the maximum duration specified in the route configuration, it is shown in red.

  • Info: can display warnings such as the ones listed below. Note that these pieces of information are not logged.
    • Warning Description
    Delivered in department X, while Y is the destination. The place where the package is being delivered does not correspond to the destination of the package.
    The transport took longer than expected. The transport lasted longer than the maximum duration specified in the route configuration.
    Old package had wrong status, created new one.

    The status of the package is higher than the expected status. In the system, the scanned package is aborted, set to Inactive and a new package is created.

    This can happen at this stage in the following (unlikely) situation: specimens were delivered and removed from a box but the unpacking was not registered in GLIMS. In GLIMS, the package is thus delivered but not unpacked. Later on, the empty box is used to pack new specimens and then collected but these steps are not registered in GLIMS either. When the courier wants to register the delivery of this new package by scanning the box number, GLIMS displays the warning because the box is still linked to a non unpacked package.
    Box number not found, created new package.

    This message appears when no box with this barcode is currently linked to an active package.

    This can happen at this stage in the following situation: a courier delivers a package whose preparation and collection have not been registered in GLIMS. As the preparation of a package requires scanning the number of the box of the package, if this step is skipped, GLIMS does not know that this box is being used for this package.

Delivery remarks and new delivery

Once the package has been delivered, click the button New delivery. A window appears and allows you to enter remarks about the delivery. Enter text manually or press the Insert key to insert texts based on the 'Package' table. The delivery screen is subsequently reset so that a new delivery can start.

Warnings

Warning

If the screen is closed before the end of the delivery process, it will not be possible to enter delivery remarks.
