Logistics manager - Route configuration

Introduction

A route is the journey of a group of specimens, from the department where they are packed to the department where they should be unpacked.

Configuring a route makes it possible to prepare a package without scanning each specimen individually. Clicking the Add specimens from route button in the Prepare package screen indeed allows the addition of several specimens to a package in one go. But where do these specimens come from? How can you indicate which specimens should be added to the package using a route? This is explained in the following sections.

Route configuration

Concept

Specimens must be linked to routes via procedures. The reason is the following one: when specimens need to be analysed, they are defined as inputs of a procedure. By linking procedures to a route, the specimens that are the inputs of these procedures are linked to the route. Subsequently, when this route is chosen in the Prepare package screen, the linked specimens are added to the package.

In practice, the configuration of a route requires two steps: the selection or creation of a route and the linking of at least one procedure with the route.

Route selection or creation

To select or create a route,

  1. go to Start -> Logistics -> Configuration -> Routes.
  2. The Route browser opens. Double-click on an existing route or press the Insert key to create a new one.

  3. The Route editor opens. Fill in the following fields and click OK:

    • Max duration: maximum time allowed between the collection of the specimens and their delivery.
    • Origin: department where the journey of the specimens should start, i.e. where the specimens should be packed
    • Destination: department where the journey of the specimens should end, i.e. where the specimens should be unpacked.

Now the route must be linked with at least one procedure.

Linking a route with a procedure

To link a route with a procedure,

  1. In the Route browser, right-click on the route you have selected/created.
  2. Click Show all -> Route procedures. The Route procedure browser opens.
  3. Press Insert. The Route procedure editor opens, the route origin and destination are already specified and cannot be changed.
  4. Double-click in the Procedure field. The Procedure browser opens.
  5. Select a procedure whose department corresponds to the destination department of the route AND whose output is some property you would like to have executed in the destination department. You can also press Insert to create a new procedure.
Warnings

Warning

The Procedure browser displays procedures of various departments. Make sure that the department of the procedure you choose is the same as the destination department of the route you are configuring.
