Shielding (genetics)

Genetic exam, Approach, Locus/variant result, Family and Attachment records can be shielded. This chapter presents how to use shielding in some typical use case scenarios.

Requesting a genetic procedure

Genetic procedures can be requested by all users and genetic exams are automatically created when the order is saved: the shielding configuration plays no role at this step. However, the created genetic exams are only visible to users who have at least the Read access right for the department defined as Responsible department of the genetic procedure (see the following section).

Shielding genetic exams and approaches

Configuration

To shield genetic exams, configure the different access rights (Read, View, Create, etc) on the shielding class Results.

Note

The configured shielding also applies to approach records.

Department

The department taken in consideration for the shielding of genetic exams and related records is the Responsible department of the genetic procedure that is to be executed in the context of the genetic exam.

Use case 1: allow users to modify genetic exams

Say users with Access profile "Geneticists" should be able to modify genetic exams for the genetic procedures whose responsible department is DEPN. In the editor of the Access profile "Geneticists",

  • type "DEPN" in the Department column,
  • type "Results" in the Class column,
  • activate the Modify check box.

Notes

  • Given that the shielding configuration for genetic exams also applies to approaches, users will also be able to create and modify approaches linked to department DEPN.
  • If users have the View but not the Modify access right, the Genetic exam screen opens in read-only mode.

Warning

As for generated browsers and editors, the access right View is not taken in consideration for the Genetic exam screen. This means that it is not necessary to have access right View to be able to open the Genetic exam screen, the Read right suffices. The screen however only opens in read-only mode, unless the user also has the Modify access right.

Use case 2: prevent users from seeing genetic exams

Say users with Access profile "Internal_DEPN" should only be able to see the genetic exams created for genetic procedures with department DEPN. In the editor of the access profile "Internal_DEPN",

  • type "DEPN" in the Department column,
  • type "Results" in the Class column,
  • activate the Read check box,
  • do not activate access rights with shielding class Results for any other department (by default, as long as no access right is activated, nothing is allowed)

Note

Given that the shielding configuration for genetic exams also applies to approaches and results, users will only be able to see approaches and results linked to genetic procedures of department DEPN.

Locus and variant results

Configuration

To shield locus and variant results, configure the different access rights (Read, View, Create, etc) on the shielding class Results.

Department

The department taken in consideration for the shielding of locus and variant results is the Department of these records.

Families

Configuration

To shield families,

  • activate the option Person shielding,
  • configure the different access rights (Read, View, Create, etc) on the shielding class Person.

Note

Given that main suspected disorders, family members and family conditions can only be accessed from families, the access rights that apply to families also apply to these records.

Department

The department taken in consideration for the shielding of families is the department in which the user who created the family was logged in.

Use case 1: allow users to see families

Say users with access profile "Geneticists" should be allowed to see families created in all the departments of your site. In the editor of the access profile "Geneticists",

  • add all the departments of your site to the Department column,
  • type Person in the Class column of each row,
  • activate the Read check box on each row.

Warning

As for generated browsers and editors, the access right View is not taken in consideration for the Family screen. This means that it is not necessary to have access right View to be able to open the Family screen, the Read right suffices. The screen however only opens in read-only mode, unless the user also has the Modify access right.

Use case 2: allow users to create and modify families

Say users with access profile "Geneticists_DEPN" should be allowed to create families in department DEPN and to modify families created in this department. In the editor of the Access profile "Geneticists",

  • type "DEPN" in the Department column,
  • type "Person" in the Class column,
  • activate the Create and Modify check boxes.

Notes

  • Users who should be able to create families in a given department should also have the Modify access right for genetic exams linked to this department (see above). This is because families can only be added through the Genetic exam screen.
  • Users who have access right Read but not Modify are able to open the Family screen in read-only mode.

Use case 3: prevent users from selecting families

Say users with access profile "Geneticists_DEPN" should not be allowed to select families of department DEPZ for their genetic exams. In the editor of the access profile "Geneticists_DEPN", do not add "DEPZ" in the department column for shielding class Person. That way, families created in department DEPZ will not be displayed in the Family choice list of the Genetic exam screen.

Warning

An exception is made for families that are already linked to a Genetic exam. Even if a user does not have access right for families of department DEPZ, if such a family is linked to a genetic exam that the user is allowed to modify, then the family is displayed in the choice list and can be selected. However, the Family pane is not be displayed, which prevents the user from seeing information on this family.

Attachments

Attachments are shielded if the records (families, genetic exams, orders) they are attached to are shielded:

  Shielding class
Access right Result Order Person
Read The user can see files attached to genetic exams in the Attachments pane of the Genetic exam screen. The user can see files attached to orders in the Attachments pane of the Genetic exam screen. The user can see files attached to families in the Attachments pane of the Family screen.
View

The user can open files attached to genetic exams, copy them to another genetic exam for which s/he has the Modify access right.

The user can open the files attached to orders, move them to a genetic exam within the order

 The user can open files attached to families.
Modify

The user can

attach files to genetic exams,

modify or delete files attached to genetic exams.

The user can

attach files to orders,

modify or delete files attached to orders,

take over files attached to orders to attach them to genetic exams for which s/he has the Modify access right.

The user can

attach files to families,

modify or delete files attached to families.
Examples

Examples

Example 1

Should users be able to consult, but not to update documents attached to Family records, activate the access right View, but not the access right Modify in the shielding class Person.

Example 2

To copy an attachment from a genetic exam A to a genetic exam B, users need

the Modify access right (class Result) for genetic exam B, and

the View access right (class Result) for genetic exam A.

Example 3

To take over an attachment from an order to a genetic exam, users need the Modify access right for both

  • the order (class Order), and
  • the genetic exam (class Result).
