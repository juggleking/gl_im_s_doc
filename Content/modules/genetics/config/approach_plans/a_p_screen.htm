Approach plan screen

Contents: Introduction - Visual representation - Configuration panes

Introduction

Once an approach plan has been created, it must be set up in the Approach plan screen.

To open the Approach plan screen, use the contextual menu/ribbon item Visualize on the created approach plan.

The approach plan screen looks as follows:

The left-hand side frame provides a visual representation of the configured approach plan.

The right-hand side frame allows you to configure the approach plan. More precisely, it allows you to add, edit and delete the building blocks of the approach plan, i.e. activities and transitions, as well as loci to be tested.

Visual representation of the approach plan

The left-hand side of the screen displays the approach plan configured in the panes on the right-hand side of the screen. You can switch between a vertical or horizontal view using the Left-Right and Top-Bottom buttons above the approach plan.

The visual representation displays the activities and the transitions that link these activities with one another.

Note

This visual representation is only displayed if a Webhost URL is indicated.

Activities

Activities are represented by various shapes.

Activities of type

  • Procedure are displayed in coloured rectangles. The background colour depends on the Type chosen in the procedure editor:
    • pre-analytical: blue
    • analytical: orange
    • post-analytical: purple
    • validation: green
    • ?: no background colour
  • Start are displayed with a green outline.
  • End are displayed with a red outline.
  • Void are displayed without layout.
  • Router are displayed with a blue diamond outline.

When the option Duplicate by specimen is activated in the editor of an activity, an asterisk appears on the right of that activity's display name.

Configuration panes

The right-hand side of the Approach plan screen contains several panes in which plan loci, activities, transitions and procedures can be configured.

Loci to be tested

The pane Loci to be tested contains a list indicating the names and types of the loci to be requested automatically when the approach plan is selected to create an approach in the Genetic exam screen.

To add a locus to the list, press Insert and search for the locus using the Add loci screen. When a locus is configured on an approach plan, an Approach plan locus record is created.

Activity

The Activity pane contains the following buttons:

  • Insert next activity: add an outgoing transition followed by a new activity to an activity selected on the left-hand side.

    Note

    Activities can also be inserted by clicking on the left-hand side of the screen and pressing the Insert key.

  • Edit: to edit the activity selected on the left-hand side.

  • Delete: to delete the selected activity together with its incoming and outgoing transitions.

    Note

    A selected activity and its transitions can also be deleted by pressing the Delete key.

  • Insert incoming/outgoing transition: to add an incoming or outgoing transition to the selected activity.
  • Insert next procedure: to add a new activity of type Procedure after the selected activity.

To open the Activity editor, select an activity on the left-hand side and click Edit in the Activity pane:

Type

Activities can be of different types. This mandatory field allows you to choose a type for the activity:

  • Procedure: procedure to be executed.

  • Start: indicates the start of the approach plan.
  • End: indicates the end of the approach plan, or the end of an approach plan branch. If an approach plan has more than one activity of type End, and therefore more than one branch, the end of the approach plan is reached when all the branches have reached their activity of type End.

  • Void: indicates that the work flow can only continue once the preceding activities are completed.

    Example

    Assume that before testing the DNA extracted from a blood sample, a special prescription is required. The completion of the approach plan therefore depends on whether the special prescription has been received.

    To support this case, the following approach plan is configured:

    The activity G_Prescription is linked to a procedure whose output is a property with an (enumerated) result that can be : Yes ("the prescription has been received"), No ("the prescription has not been received").

    The Void activity "Waiting that all activities are completed" makes it possible to halt the continuation of the approach until

    • the DNA is extracted, AND
    • the result (Yes, or No) of G_Prescription has been entered in GLIMS.

    When both procedures are completed, the approach continues following one of the two transitions: if no prescription has been received, the transition "No test required" is used, leading to the end of the approach. If the prescription has been received, the transition "Test required" is used and the approach continues.

  • Router: dispatches available specimens to following activities. The routed specimens are only those whose material correspond to the material indicated in the Routed material field. Therefore, there must be one activity of type Router per material to be routed.

    Notes

    • The specimens dispatched by Router activities may result from previous activities of the approach plan (e.g. aliquoting) or have been created outside the approach plan.
    • Each available specimen is dispatched to exactly one activity via one outgoing transition.
    • Differently from activities of type Procedure, activities of type Router do not trigger the creation of action records.
    • The Seq. no. field is mandatory for outgoing transitions of a Router activity, see more details below.

Duplicate by specimen

This option can only be activated for activities of type Procedure with at least one input of type Material.
If this option is activated, the approach activity and its transitions will be added in the approach. They will be added for each specimen found in the approach and whose material corresponds to one of the configured material inputs of the activity's procedure.

Display name

Name of the activity. If this field is left empty, GLIMS creates a display name on the basis of
  • the activity type if the activity is not of type Procedure,
  • the mnemonic of the procedure if the activity type is Procedure.

Procedure

Procedure that should be executed at this point in the approach plan. This field can only be filled in when the field Type is set to Procedure, and is in this case mandatory.

Routed material

Material that specimens to be routed should contain. Specimens containing another material will not be routed. This field is mandatory, and only available, when the activity type is Router.

Await action status

This field is only editable when the field Type is set to Procedure. It indicates the minimal status that the activity's action should have for the outgoing transitions to be evaluated and so, for the next activity to be scheduled.

Tip

Set this field to a status lower than Ready if you want the action of the next activity to be created and appear on a work list before the current activity is finished.

Notes

If an activity called A is preceded by more than one activity, all the Await action statuses of these previous activities must be reached before activity A can be scheduled.

The value of this field is

  • also visible in the field Approach trigger status on the bottom of the Action editor.
  • not taken into account when finished activity is re-opened.

Incoming / Outgoing transitions

Allows you to insert and look up the transitions to and from the activity.

Transition

In the Transition pane, you can add and edit transitions that link activities to each other. It contains the following buttons:

  • Edit: to edit the transition selected on the left-hand side.

  • Delete: to delete the selected transition.

    Note

    A selected transition can also be deleted by pressing the Delete key.

When clicking on Edit, the editor of the selected transition opens:

Seq. no.

This field is mandatory for transitions starting from an activity of type Router. If the Display name of such a transition is empty, the sequence number is displayed on the approach plan view.
The lower the sequence number, the higher the priority of the transition. For instance:
  • a specimen routed via a transition with sequence number "1" is routed before a specimen routed via a transition with sequence number "2".

  • if there are three transitions but only two available specimens to be routed, the transition with the highest sequence number is deleted in the approach.

    Example

    Assume there are three transitions with, respectively, "1", "2" and "3" as sequence number.

    If there are only two available specimens to be routed, the transition with sequence number "3" is deleted, while the transitions with sequence number "1" and "2" are kept.

Display name

Optional string to be displayed on the transition in the visual representation.
It can for instance be used display the purpose of the transition (e.g. "DNA received, no extraction required").

Only when

ApproachActivity-based site function of data type Logical defining conditions under which the transition to the next activity can take place. In routine, this function will only be executed once the From activity has reached either status Finished or the status indicated in Await action status.

Example

Assume that

  • a given genetic procedure is to be executed on DNA aliquots.
  • the specimens provided are sometimes DNA samples, but sometimes blood or another material. In these cases the DNA first needs to be extracted.

To handle this case, do the following in the approach plan linked to the procedure:

  1. define a transition "if DNA sample" with a site function in its Only when field. This site function should dictate that the transition be only used when the sample received is a DNA sample:

    	IF .Approach.Specimen.Material.Mnemonic = "Mat_DNA" THEN RETURN TRUE;
    	ELSE RETURN FALSE;
    	ENDIF;
  2. to the transition "if DNA sample", add a To activity "Aliquoting".
  3. define a fallback transition "Otherwise". It will be used when the Only when condition of the transition "if DNA sample" returns FALSE, i.e. when the sample received is not a DNA sample.
  4. for the fallback transition, define a To activity "DNA extraction".
  5. link the DNA extraction to the "Aliquoting" activity.

Example

Assume the following configuration:

  • an activity A of type Procedure with an input material M and an output property O

Assume now that, in routine, the approach

  • should continue if the result of output property O is "1",
  • should stop if the result of output property O is different from "1".

Configuration

  • We create two outgoing transitions for activity A: T1 and T2.

  • On T1, we add an Only when function. In this function, we use the MISPL function Specimen.Result to retrieve the result of the output property O with material M as scoped specimen:

    RETURN .ScopedSpecimen.Result("OutputProperty_ActivityA","Confirmed", ?,.Approach.GeneticExam.Order).RawValue="1";
  • On T2, we do not add any Only when function but we activate the Is fallback option.

Is fallback

If this option is activated,
  • the transition is only used if the Only when function of all the other transitions with the same "from" activity return FALSE.
  • the transition cannot have a condition function.

Note

A transition on which no Only when function is defined and that is not set as fallback will always be used, independently of whether its sister transitions are used or not.

From / to activity

Indicate which two activities should be linked by this transition.

Procedure

The Procedure pane

  • allows you to configure and edit activities of type Procedure.

  • contains a read-only visual representation of the configured procedure linked to the activity.

It contains the following buttons:

  • Insert input: to open the Procedure input editor and create or select a material or property input.

    Inserting a property input P is only allowed the following conditions are respected:

    • the procedure being edited also has a material input M, and

    • the input property P is also the output property P' of a preceding procedure Q in the approach plan with the same material as the material input M.

    Example

    Notes

    • In routine, result R (instantiation of the input property P) must be the result R' (instantiation of the output property P') of a preceding action (created for procedure Q) in the SAME approach. Result R is scheduled with the same input specimen as the input specimen used for R'.
    • The configuration validation tool checks if the first condition is respected but not if the second one is.
  • Edit: to open the Procedure editor.

  • Insert output: to open the Procedure output editor and create or select a material or property output.

    Tip

    In routine, material and property outputs can be added to the order automatically. Two options are available to this effect:

    • Number of specimens to be auto requested in the Material output editor.

      If this field is set to a value higher than "1", the value is displayed in the visual representation of the configured procedure. If this field is set to "?", the output material is not displayed:

    • Auto request when starting activity in the Property output editor.

      If this option is activated, the property output is displayed in the visual representation of the configured procedure. Otherwise, it is not displayed:

Note

Procedure inputs and outputs are only taken into account in the approach flow and displayed in the visual representation if they are enabled.
