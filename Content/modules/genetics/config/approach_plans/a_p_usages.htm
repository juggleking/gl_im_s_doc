Approach plan usages

Approach plan usages define the conditions under which approach plans can be selected to create approaches in the Genetic exam screen. The Approach plan usage editor looks as follows:

Enabled

Indicates whether the approach plan can be selected in routine.

Responsible department

Responsible department that the genetic procedure of the genetic exam should have for the approach plan to be selectable. This field is mandatory.

Only for genetic procedure

Genetic procedure on which the genetic exam should be based for the approach plan to be selectable.

Only for material

Limits the use of the approach plan to a specific material.

Extra condition function

Approach-based site function defining additional conditions under which the approach plan can be selected in routine.

Tip

During the MISPL evaluation, there is no actual Approach context available. The MISPL function can only be used to navigate to other linked records (ApproachPlan, GeneticExam, Specimen).

Other functions like AddLocusRequest(),GetActivity(), GetVariantResult, StartTime, EndTime, CreationUser, â€¦ are not available.

Approach plan that can be used

Id of the approach plan whose usage is being defined

Score

Mandatory field. Number (1-99) on which approach plans can be sorted in the Approach plan selection browser. The higher the score, the more the approach plan is considered suitable in the context described in the approach plan usage (i.e. for a specific genetic procedure, for a specific material, etc). The default value is 1.
