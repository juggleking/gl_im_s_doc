Approach plan browser and editor

The approach plan browser contains the list of existing approach plans and allows you to create new ones. It can be opened via Start -> Genetics -> Configuration -> Approach plans.

Main page

To create a new approach plan,

  • Press Insert in the Approach plans browser to open a new Approach plan editor.

  • Fill in the editor fields:

    Expected TAT (in days)

    Expected period of time between the scheduling of an approach (based on the present approach plan) and the end of its execution.

    Internal comment

    Optional, free text. It can for instance be used to add internal, technical information to be displayed in
    • the Comment column of the Approaches grid at the bottom of the Genetic exam screen, and
    • the Internal comment column of the Approach plan selection browser.

    Period in which the plan can be selected

    By default, the mandatory From field is filled in with the date on which the approach plan is created.

    Cases where this plan will be used

    Embedded browser of Approach plan usages to which the approach plan is linked. Press Insert to add an approach plan usage.

    Loci of approach plan

    Embedded browser of the loci to be tested. Press Insert to add a locus.

Site attributes page

Creation trigger

The site attribute Creation trigger can be defined to be executed after the manual or automatic creation of a new approach.

It can be used to add locus requests with the MISPL function Approach.AddLocusRequest. This site attribute is defined automatically when upgrading to GLIMS 9.9.4 (or higher).

Once an approach plan is created, it must be set up in the Approach plan screen.

Tip

An approach plan can also be created by copying an existing approach plan. It suffices to select this existing approach plan in the Approach plans browser and to click on the contextual ribbon/menu item Make a copy. All the activities, loci, transitions and usages of the existing approach plan are copied.
