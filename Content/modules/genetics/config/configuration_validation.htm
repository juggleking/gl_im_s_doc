Procedure and configuration validation

The Genetics module offers two validation functions:

  • Validate genetic procedure
  • Validate configuration

Procedure validation

The procedure validation function can be executed using the menu/ribbon item Validate genetic procedure on a genetic procedure record.

This function checks if the selected genetic procedure has at least one

  • Material
  • Request definition

Configuration validation

Once the configuration for the Genetics module is ready, it can be validated. This can be done via Start -> Genetics -> Configuration -> Validate configuration.

The validation consists in checking if all currently active

  • Genetic procedures have at least one
    • Material
    • Request definition

  • Approach plans and their building blocks respect the following rules:

    Approach plan
    has at least one usage.
    contains exactly one activity of type Start.
    contains no cycle: no path leads back to an incoming transition of an already encountered activity.
    Approach plans on which loci to be tested are configured contain at least one activity of type Procedure. The procedure corresponding to this activity is of type Analytical or Post-analytical.
    Activity
    Activities of type Start do not have any incoming transition.
    Activities of type End do not have any outgoing transition.
    Activities with a type different from Start have at least one incoming transition.
    Activities with a type different from End have at least one outgoing transition.
    Activities having a fallback outgoing transition also have at least one outgoing transition with a condition.
    Activities for which Duplicate by specimen is activated are linked to a procedure having at least one enabled input of type Material.
    Procedure linked to an activity
    is Available.
    has a defined type for the Genetics module.
    if it is linked to a station, the option Scheduling activity on this station's editor is activated.

    has at least

    • one enabled output of type Material with a Number of specimens to be auto requested superior or equal to "1", or
    • one enabled output of type Property for which the option Auto request when starting activity is activated.
    also has an enabled input of type Material if it has an enabled input of type Property.
    Transition

    Among the outgoing transitions of an activity, at least one has no condition.

    As a consequence, if an activity only has one outgoing transition, this transition has no condition.
    Among the outgoing transitions of an activity, at most one is a fallback transition.

    Outgoing transitions of activities of type Router:

    • have each a different sequence number.
    • are not fallback and do not have a condition.

Notes

  • Genetic procedures with a disabled request code (or all request codes disabled) are not "active", and are therefore not checked.
  • An approach plan is "active" if the day on which the validation is executed lies within the Period in which the plan can be selected. Inactive approach plans are not checked.
