Genetic procedures

Introduction

Genetic procedures

  • are models of actions that can be performed in genetic laboratories,
  • have an input and deliver outputs (materials, properties), like non-genetic procedures
  • are requested via request codes of type Genetics in order entry.
  • are instantiated in routine by Genetic exams.

Browser and editor

Genetics procedures are to be defined via Start -> Genetics -> Configuration -> Genetic procedures. Click insert in the Genetic procedures browser to add a new procedure. The Genetic procedure editor looks as follows:

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Description

Multilingual text that can be displayed on labels, reports etc. Double-click in the field to insert translations of the text.

Responsible department

The only department in which the procedure can be requested. This field is mandatory.

Department function

Allows you to insert a site function of data type String that returns the mnemonic(s) of one or more departments in which the procedure can be requested. This function is based on the GeneticExam-table and can therefore access the Order table.
The departments returned by this function override the department indicated in the Responsible department field. Conversely, if this function returns no department, the department indicated in the Responsible department field is used.

Report property

Property used to report the genetic procedure. This field is mandatory. The property must have data type Genetics.
When the procedure is requested, a report result is automatically added to the order.
If Validator as executor is activated, the responsible person of this report result is used as billing item executor.

Property classification

The nodes of the indicated property classification are displayed on the screen of the genetic exams that are based on the genetic procedure.
These nodes should be linked to an informational property, otherwise they are not displayed. Additionally, properties should be configured on the first level (not in a folder) to be visible.

Note

  • The order in which the nodes are displayed in the routine screens (e.g. the Genetic exam screen, Specimen info screen) reflects the order of the nodes in the property classification.
  • Property classification nodes are usually referred to by their label. However, if a property classification node has no label, the property's description is used (if Show property description is activated). If the property's description is not indicated (or if Show property description is not activated), the property's short name is used. If the property's short name is not indicated, the property's mnemonic is used.

    If the data type of the property is

    • Numeric: only digits can be entered in the field; by default, the number of decimals allowed is 2 but can be modified in the Reported decimals field of the Report tab in the Property editor.
    • String: simple text field
    • String and the option Rich text is activated in the Property editor: a Rich text button is displayed and allows you to open the Rich text editor.

    • Enumerated: a configured choice list is displayed with radio buttons if Strict result checking is activated. However, a dropdown list is displayed instead if

      • there are more than 8 choices,
      • the choice list is empty,
      • a diagnosis code system is linked to the property,
      • Strict result checking is not activated.

      If a code system is indicated in the field Diagnosis code system, the codes of this system also appear in the choice list.

    • Image: a button is displayed and allows you to open the image viewer.
    • Other data types: nothing is displayed.

Suspected disorder

The disorder that is suspected when the procedure is requested. Double-click in the field to choose a disorder from the Disorder browser.
In routine, when the procedure is requested, the suspected disorder is automatically indicated in the created genetic exam.

Support family

Indicates whether genetics exams instantiating the procedure can refer to a family. Set by default to Yes.
Should be set to Yes for the creation/selection of families to be possible in the Genetic exam screen and for the Pedigree to be displayed in the family screen.

Request definitions

When a genetic procedure is created, a request definition and a request code of type Genetics are automatically created. Note that genetic procedures do not have default request definitions.
Press insert to add another Genetic request definition.

Materials that can be used

Materials on which the test can be executed. To add a material, use the contextual menu/ribbon item Genetic procedure material. These materials are stored in the GeneticProcedureMaterial table.

Genetic procedures are linked to approach plans.
