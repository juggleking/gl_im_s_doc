Import of HGNC file

Lists of genes can be imported, manually or in batch, using .json files downloaded from the HGNC website (external link > Complete HGNC dataset .json file). This file is updated on a regular basis with officially approved gene information and approved gene transcripts.

Example

Example

File: hgnc_complete_set.json

The importation leads to the creation of

  • gene records,
  • gene transcript records,
  • chromosome region records,
  • locus records,
  • HGNC and, if configured, other gene codes

in the GLIMS database.

Note

Other records, such as gene regions and chromosome records, must be created manually or by importing different files, in .csv format for instance.

To import the downloaded .json file, go to Start > Genetics > Configuration > Genome > Import genes from HGNC file.

The data contained in the import file is not entirely imported. The following chapters describe which data is imported and which is not: genes, chromosome regions, loci and codes. Imported data can also be modified manually.

Tip

To update the Gene table (the gene symbols table), repeat the .json file import by following the steps mentioned above, but with the newer version of the file.

The HGNC status of the gene symbol can change from Approved to Withdrawn, but the HGNC Id and the gene symbols always remain the same.

Logging

When the data is imported, a log file is created in your temp-folder. If you launch the import manually, a log window appears on your screen.

Logged

The following information is logged:

  • Gene that is not imported because
    • no location or chromosome is provided in the import file;
    • it has more than one location on the same chromosome.
  • Code(s) that are not imported because the configuration of the code set they belong to is incorrect.

Not logged

Genes with location "reserved" and "unplaced" are not imported. This is not logged.
