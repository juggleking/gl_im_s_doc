Site functions for locus results

This chapter presents MISPL functions based on the LocusResult table.

GetGeneticResultDetail

The MISPL function LocusResult.GetGeneticResultDetail() allows users to retrieve the locus results of the configured list detail types. In practice, the MISPL is commonly used as an approach transition evaluation MISPL.

Parameters:

  1. Mnemonic

    Mnemonic of the genetic result detail as configured in the list detail type. If the mnemonic parameter is provided, the index parameter can be '?'.

  2. Index

    This is a loop counter variable: when 1, the first GeneticResultdetail is returned (sorted by creation time, most recent one comes first). When 2, the second result is returned and so forth, until the return value becomes unknown. If the index parameter is provided, the mnemonic can be '?'.

Example MISPL (not an actual scenario)

integer i;

GeneticResultDetail rslt;

string text;

rslt := .GetGeneticResultDetail(?, 1);

i := 1;

WHILE rslt NE ? DO

rslt := .GetGeneticResultDetail(?, i);

IF rslt NE ?

THEN

text := text + IntegerToString(i, "%d");

text := text + " Mnemonic: " + IfKnownString(rslt.ListDetailType.Mnemonic);

text := text + " | Value: " + IfKnownString(rslt.Value);

text := text + CHR(13);

ENDIF;

i := i + 1;

DONE;

RETURN text;
