Site functions for variants and variant results

This chapter presents MISPL functions based on the Variant and VariantResult tables.

GetSequenceVariant - GetCopyNumberVariant

Two functions described in the following subsections allow you to retrieve information of your choice concerning a given variant.

GetSequenceVariant()

This function allows you to retrieve information on a specific sequence variant.

This function has no parameter and its data type is String.

Example

The site function

RETURN .GetSequenceVariant().ProteinChangeType.Mnemonic;

returns the mnemonic of the protein change type for the chosen sequence variant.

GetCopyNumberVariant()

This function allows you to retrieve information on a specific copy number variant.

This function has no parameter and its data type is String.

Example

The site function

RETURN .GetCopyNumberVariant().Ratio;

returns the ratio of the chosen copy number variant.

VariantResult.GetDetailValue

VariantResult.GetDetailValue(Mnemonic)

This MISPL function allows users to retrieve the values of the configured list detail types on variant results.

The input parameter Mnemonic is the mnemonic of the genetic result detail as configured in the list detail type (Property > List type > List detail types).

Example

RETURN .GetDetailValue("SV_Extra_Column");

Warning

Do not use the VariantResult.SetAnnotation / VariantResult.GetAnnotation MISPL functions (see below) to retrieve the values of the configured list detail types.

Such a configuration will no longer be compatible when upgrading to a newer version of GLIMS.

Variant.SetAnnotation / VariantResult.SetAnnotation

.SetAnnotation(StringKey,StringValue)

The parameter Key must be an annotation key. The parameter Value is a string that should be associated to this key.

This function allows you to associate the specified value to the specified key in the json-formatted field. If the parameter Value is empty, the "key-value" pair is deleted from the .json-formatted field.

The data type of this function is Logical.

Example

Variant.GetAnnotation / VariantResult.GetAnnotation

.GetAnnotation(StringKey)

The parameter Key must be an annotation key.

This function allows you to retrieve the value associated to the key that is specified as parameter.

The data type of this function is String.

Example

VariantResult.CopyAnnotationsToVariant

.CopyAnnotationsToVariant(StringKeys)

The parameter Keys can be a single annotation key, a list of keys or a key pattern.

This function allows you to retrieve, for a variant result, all annotation "key-value" pairs whose key matches a key or the key pattern specified as parameter. The retrieved "key-value" pairs are then copied in the Annotations field of the variant linked to the variant result. If the variant already had a value for a certain key, this value is replaced with the new one.

The data type of this function is Logical.

Example

Variant.SetClassification / VariantResult.SetClassification

Variant.SetClassification(PositiveIntegerClassification)

The parameter Classificationmust be a value between 1 and 5.

This function allows you to change the value of the variant (result) Clinical classification field. This can for instance be used if you want the variant classification to be synchronised with the variant result classification when the variant result is confirmed.

The data type of this function is Logical.
