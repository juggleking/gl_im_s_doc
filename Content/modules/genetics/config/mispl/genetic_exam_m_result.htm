GeneticExam.Result

The GeneticExam.Result MISPL function allows you to retrieve a result record for a given property linked to the genetic exam on which this function is executed.

GeneticExam.Result (Mnemonic PropertyMnemonic, String MinimalStatus, String MaximumStatus)

Example

The following site function returns the raw value of the result that is based on property Gen_Ratio_260_230 and whose status is Validated:

RETURN .Result("Gen_Ratio_260_230", ResultStatus["Validated"], ResultStatus["Validated"]).RawValue ;

Notes

  • Specimens are directly linked to orders, but not to genetic exams. Therefore, when this function is executed,
    • GLIMS first looks for results (in the same order) linked to one of the specimens of the genetic exam. The material of these specimens must correspond to a material defined in the genetic procedure.
    • If no such result is found, GLIMS looks for results that are linked to the order and do not have any specimen. If GLIMS finds more than one result, only one is returned (the choice is random).
    • If more than one genetic exam is performed on the same specimen in the same order, the returned result might belong to another genetic exam of this order.
  • The result returned by this function is a "normal" result. Other MISPL functions are available on the GeneticExam table to retrieve locus results and variant results.
