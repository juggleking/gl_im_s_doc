MISPL functions for billing

This chapter presents MISPL functions that can be used within site functions for the billing of genetic exams. These MISPL functions are all based on the GeneticExam table and their parameters are all optional.

GeneticExam.GetApproachCount

The .GetApproachCount MISPL function on GeneticExam allows users to retrieve the number of non-discontinued approaches linked to the genetic exam on which the function is executed. Its data type is Integer.

If approach plan mnemonics are given as parameters, only the approaches based on these approach plans are counted.

GeneticExam.GetApproachCount (ValueList ApproachPlanMnemonics)

Example

If the following function is executed on a genetic exam G that has five non-discontinued approaches, it returns "5":

RETURN .GetApproachCount (?);

If the following function is executed on genetic exam H that has five non-discontinued approaches, including two based on approach plan AP1 and one based on approach plan AP2, it returns "3":

RETURN .GetApproachCount ("AP1","AP2");

If the following site function is based on the Request-table and executed during tariffication, it will return the number of approaches to be tariffed:

RETURN .GeneticExam.GetApproachCount (?);

GeneticExam.GetApproachList

The .GetApproachList MISPL function on GeneticExam allows users to retrieve the mnemonics of the approach plans on which the non-discontinued approaches of the genetic exam are based. This function has no parameter and its data type is String.

If an approach plan is used for more than one approaches, its mnemonic appears in the returned list more than once.

GeneticExam.GetApproachList ()

GeneticExam.GetLocusResultCount

The .GetLocusResultCount MISPL function on GeneticExam allows users to retrieve the number of locus results for the non-discontinued approaches of the genetic exam. Its data type is Integer.

If the name of a locus is added as first parameter, only the locus results linked to this locus are counted. It is not possible to indicate more than one locus name.

If a value is provided as second parameter, only the locus results having this value as raw value are counted. It is not possible to indicate more than one value.

GeneticExam.GetLocusResultCount (LocusName LocusName,String LocusValue)

Action.ApproachActivity

The .ApproachActivity MISPL function on Action allows users to retrieve the first linked approach activity of a certain action (in theory this is only one).

This is a workaround that is required to retrieve the genetic exam and get the father.

Example

RETURN .ApproachActivity().DisplayName;
