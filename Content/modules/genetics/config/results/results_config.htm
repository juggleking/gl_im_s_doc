Results (configuration)

The Genetics module distinguishes different types of results. The distinction is based on the configuration of procedures and their outputs. This configuration is needed for two reasons:

  1. To display procedure outputs of different types in distinct panes in the routine Results screen: Specimens, Results,Locus results, CNV, Sequence variants and Interpretations panes.
  2. Concerning locus and variant results, to be able to add locus and variant tests on approaches manually.

This chapter describes how to configure the procedures and their outputs.

Specimens

If a procedure output is of type Material, the corresponding result is displayed in the Specimens pane of the routine Results screen.

Results

If

  • a procedure is of type Pre-analytical or Analytical, and
  • its procedure output is of type Property, and
  • the property's data type is Enumerated, Numeric, String or Image,

the result corresponding to the procedure output is displayed in the Results pane of the routine Results screen:

The way results are displayed in this pane depends on the properties' data types:

Datatype

String

Properties with datatype String will be shown as text boxes. When Number of editor lines > 1, a multiline text box will be shown.

Tips

  • Use the F6 key in the text box to open a larger text box that allows you to run spell checking on the entered text.
  • Enter result codes by using the Expansion sign for result codes specified in the GLIMS general settings, followed by the result code. This is only supported in non rich text boxes.
  • Configure text boxes as rich text boxes, if required. Click on the following links for more information on how results can be displayed in an image, graph or in the rich text format.

Numeric

Properties with datatype Numeric will be shown as a numeric field in which it is possible to use the up/down arrows.

See Reported decimals and Decimal threshold for more information on the number of decimal places shown.

The Unit is mandatory for numeric properties and will be shown between brackets next to the label. The unit's Description (or Name, if the description is not specified) will be used for visualization.

Enumerated

Properties with datatype Enumerated for which a Choice list is specified :

  • will be shown as radio buttons
    • if the number of choices is not higher than 8, and
    • if the Strict result checking option is activated.
  • will be shown as a drop down list
    • if the number of choices is higher than 8, or
    • if the Strict result checking option is not activated, or
    • if a diagnosis code system is linked to the property.

Locus results

Basics

If

  • a procedure (called "P") has an output of type Property, and
  • the property's Data type is Genetic List, and
  • the property's List type has Locus results as list content type,

then,

  • it is possible to add locus requests on approaches containing an activity linked to procedure P, and

  • the results of the locus requests linked to the approach in which the procedure is executed are displayed in the Locus results pane of the routine Results screen:

Note

Properties with data type Genetic List are "dummy" properties, similarly to report properties. They should not be part of the default property classification, so that they do not appear in classic result browsers. They are automatically confirmed and validated when the Confirm all and Validate all buttons are used in the genetic Results screen.

If

  • a locus result is discontinued or updated, and
  • the corresponding activity is finished,

the "dummy" property is discontinued-repeated.

Additional columns in the Locus results pane

Customised columns can be added to the Locus results grid. To this effect,

  1. configure a property of data type Genetic List and select a List type,

  2. right-click on the List type editor > List detail types, create a list detail type for each column to be added:

    In the List detail type editor, the following fields are available:

    Mnemonic

    Appears in the column header to refer to the list detail type if no short name is indicated.

    Short name

    Appears in the column header to refer to the list detail type.

    Choice list

    Add a choice list if users should only be allowed to enter a limited set of values in the column. If no choice list is defined, the cells will allow free text input.

    Seq. no

    Indicates the position of the list detail type column in the grid.

    Enabled

    Must be activated for the list detail type column to be visible in the grid.

Tip

See related MISPL function LocusResult.GetGeneticResultDetail() to retrieve the locus results of the configured list detail types.

Variant results

CNV

To

  • be able to add CNV requests on approaches containing an activity linked to a procedure called "P", and
  • see the results of CNV requests (linked to the approach in which the procedure is executed) in the CNV pane of the Results screen,

proceed as described for locus results above but select CNV in the List content type field.

Sequence variants

To

  • be able to add sequence variant requests on approaches containing an activity linked to a procedure called "P", and
  • see the results of Sequence variant requests (linked to the approach in which the procedure is executed) in the Sequence variant pane,

proceed as described for locus results above but select Sequence variants in the List content type field.

Additional columns in the Variant results pane

To add customised columns for the CNV and Sequence variant grids, configure list detail types as follows:

  1. Configure a property of data type Genetic List and select a List type (i.e. Sequence variant or CNV),
  2. right-click on the List type editor > List detail types and create a list detail type for each column to be added.

In the List detail type editor, the following fields are available:

Mnemonic

Appears in the column header to refer to the list detail type if no short name is indicated.

Short name

Appears in the column header to refer to the list detail type.

Choice list

Add a choice list if users should only be allowed to enter a limited set of values in the column. If no choice list is defined, the cells will allow free text input.

Seq. no

Indicates the position of the list detail type column in the grid.

Enabled

Must be activated for the list detail type column to be visible in the grid.

Tip

See related MISPL function VariantResult.GetDetailValue() to retrieve the values of the configured list detail types on variant results.

Interpretations

If

  • a procedure is of type Post-analytical or Validation, and
  • its procedure output is of type Property, and
  • the property's data type is Enumerated, Numeric, String or Image,

the result corresponding to the procedure output is displayed in the Interpretations pane of the routine Results screen:

Results are displayed in this pane in the same way as in the Results pane.

Confirm / Validate all summary screen

When clicking on Confirm all and Validate all in the Results screen, a summary screen appears. By default, check boxes are displayed next to each result. These check boxes should be replaced by small arrows that allow users to expand and collapse certain results. To do so,

  • start regedit
  • go to HKEY_CURRENT_USER (or MACHINE) -> Software -> Microsoft -> Internet explorer -> Main -> FeatureControl -> FEATURE_BROWSER_EMULATION
  • create a new DWORD key with name "prowin32.exe" and set its data (decimal value) to 11000 (which means Internet Explorer 11).
