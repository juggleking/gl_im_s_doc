Variants

This chapter presents

  • the Copy number variant and Sequence variant editors.

    These editors can be opened via Start -> Genetics -> Configuration -> Variants -> Copy number variants/Sequence variants.

  • how to approve/undo the approval of a variant
  • how to get a list of families in which a given variant has been found.
  • how to import lists of variants into the GLIMS database.

Note

The functions that can be used to retrieve specific information on a chosen variant are described here.

Copy number variants

Genomic build

Indicates which genomic standard is used. For CNVs, None cannot be chosen.

Locus

Chromosome or chromosome region.
Click the [...] button to open a list of chromosomes and chromosome regions, or use the Add locus button.

Start position

25 digits maximum

Stop position

25 digits maximum

Size - kbp - Mbp

Length of the variant: distance between the start and stop positions. Automatically calculated.

Type

Choose a DNA change type in the list. This list contains the active DNA change type records.

Used in particular for reporting and statistics.

Copies

Number of copies

Ratio

Free text

Gene or gene region

Loci of type Genes and Gene regions on which the CNV is located. Must refer to the same chromosome as the CNV.

Default transcript

Default transcript of the gene.

Sequence variants

Genomic build

Indicates which genomic standard is used. For sequence variants, None may be selected.

Locus

Gene or gene region.
Click the [...] button to open a list of genes and gene regions, or use the Add loci button.

Transcript

When a locus (gene or gene region) is indicated, this field is automatically filled in with the default transcript of the locus.
You can also
  • click the [...] button to open the list of all the transcripts of the selected locus.
  • type the name of a transcript.

LRG

Locus reference genomic id (40 characters maximum).

Start position

Start position of the sequence variant, is also valid without a subsequent Stop position.

Stop position

Stop position of the sequence variant, not allowed without a preceding Start position and cannot be smaller than the start position.

gDNA

Description of the genomic DNA change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "g."
  • position numbering in relation to the first nucleotide of the genomic reference sequence
  • change, e.g. "G>A", "delA", "dupC"

Example: g.12G>A

cDNA

Description of the coding DNA change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "c."
  • position numbering in relation to the first nucleotide of the translation start codon of the coding DNA reference sequence
  • change, e.g. "G>A", "delA", "dupC"

Example: c.76delA

Type

Choose a DNA change type in the list. This list contains the active DNA change type records.

Used in particular for reporting and statistics.

Amino acid

Description of the protein change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "p."
  • position numbering in relation to the first amino acid of the protein sequence
  • change, e.g. "G>A", "delA", "dupC"

Example: p.Met1ext-5

Type

Choose a protein change type in the list. This list contains the active Protein change type records.

nDNA

Description of the non-coding DNA change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "n."
  • position numbering in relation to the first nucleotide of the non-coding DNA reference sequence
  • change, e.g. "G>A", "delA", "dupC"

mDNA

Description of the mitochondrial DNA change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "m."
  • position numbering in relation to the first nucleotide of the mitochondrial DNA reference sequence
  • change, e.g. "G>A", "delA", "dupC"

RNA

Description of the RNA change (40 characters maximum).
Example

Example

It should contain the following elements in the following order:

  • "r."
  • position numbering in relation to the first nucleotide of the
    • translation start codon of the RNA reference sequence
    • non-coding RNA reference sequence
  • change, e.g. "G>A", "delA", "dupC"

Example: r.45C>U

Complementary information

The following fields appear in the editors of both copy number variants and sequence variants:

Method

Choose a variant method in the list. This list can be updated using the field Variant method list.

Method info

Free text field to add information about the method.

Clinical classification

Choose a variant classification in the list. This list can be updated as follows:
  1. open the site domain VariantClassification

  2. modify the list of values

The clinical classification can also be indicated in routine. If the classification indicated in routine differs from the classification configured here, the configured classification can be updated under conditions described here.

Approved by

Name of the user who approved the variant

Approved on

Date and time of the approval. If this field is empty, the variant has not been approved yet.

Comment

Rich text field for internal comments (read-only when the variant is approved).

Found internally

Select this option manually if the variant was found in a specimen managed in your lab with a LIS different from GLIMS.
If a variant has been detected in your lab using GLIMS, and is therefore stored as a confirmed or validated variant result of a genetic exam, the option Found internally is automatically activated and cannot be deactivated manually.

Found outside community

Select this option if the variant is known in other databases or networks in the world.

Found in community

Select this option if the variant is known on the network your laboratory is part of (such as the GeneInsight network VariantWire), independently of whether it was found in your lab or another lab of the network.

Active

Indicates whether the variant can be used in routine (selected in the variant search screen, for instance).

Codes

The Codes list is displayed in the editors of variants that have already been saved. It displays the codes used to refer to the variant in different coding systems. These coding systems can for instance represent the networks in which the variant has been found.

Double-click on a code to open a web page containing information on the variant.

The chapter Variant codes describes how to add codes to this list and link them to web pages.

Annotations

Read-only textual information on the variant. The content of this information field can be set or retrieved using MISPL functions.

Example

A list of comma-separated "key-value" pairs (nesting, i.e. lists within lists, is not supported):

{

"af": 0.1078, "afr_af": 0.087, "amr_af": 0.1297, "eas_af": 0.0813, "eur_af": 0.1481, "sas_af": 0.1063, "aa_af": 0.09623, "ea_af": 0.1543

}

Variant approval

Contextual menu/ribbon items, Approve and Undo approval, are displayed if

  • the user's role is System manager or Developer, or
  • the user has the privilege VariantApproval.

Once a variant is approved, the fields of the editor can no longer be edited. To edit these fields, you first need to withdraw the approval.

Families linked to a variant

Clicking Families in the ribbon of the CNV or Sequence variant editor opens a list of families in which the selected variant was found on at least one member. That is, at least one family member is linked to a confirmed or validated variant result for the selected variant.

Importing variants

Lists of variants can be imported using the denormalised export-import function of GLIMS.
