Genetic exam query

In the Genetic exam query screen, you can search for genetic exams meeting criteria of your choice. These criteria are defined below.

This screen can be opened via Start -> Genetics -> Genetic exam query.

The buttons Store and Load at the bottom of the screen allow you to save frequently used queries and reuse them without having to fill in the query criteria again.

At least one of the following criteria must be specified: Genetic exam status till (lower than Validated), Order reception date from, Issuer, Patient, Family.

The genetic exam query can also be executed in batch.

General

Genetic procedure

Genetic procedure that lead to the creation of the genetic exams searched for

Suspected disorder

Suspected disorder of the genetic procedure on which the genetic exams searched for are based

Genetic exam status from - till

Status range within which the Status of the genetic exams searched for should be situated

Order reception date from - till

Date range within which the Receipt time of the orders containing the genetic exams searched for should be situated

Desired date from - till

Date range within which the Desired date of the orders containing the genetic exams searched for should be situated

Genetic exam end date from - till

Date range within which the End date of the genetic exams searched for should be situated

Only overdue exams

Indicates whether the End date of the genetic exams searched for should be posterior to the order's Desired date.

Responsible department

Responsible department of the genetic exam on which the genetic exams searched for are based

Minimal urgency

Lowest urgency that the genetic exams searched for should have

Minimal severity

Lowest severity that the genetic exams searched for should have

Approach plan

Approach plan used for at least one approach of the genetic exams searched for

Note

The query does not return genetic exams that only contain discontinued approaches, unless the range indicated in Approach status from - till includes status Discontinued.

Approach status from - till

Status range within which the Status of at least one of the approaches of the genetic exams searched for should be situated.
If the Approach plan field is filled in, the approaches considered are those based on the selected approach plan.

Requested locus

Locus linked via an approach request to at least one of the non-discontinued approaches of the genetic exams searched for

Filter

Allows you to define additional query criteria using a GeneticExam-based site function of data type Logical.

Notes

  • Using such a filter may have a negative impact on the performance of the query.
  • To define more complex criteria on locus results than those made available in the Result tab, use the method .GetLocusResult(). The same applies to defining more complex criteria on variant results by using .GetVariantResult().

Specific

Issuer

Issuer who created the orders containing genetic exams searched for

Patient

Patient for whom the orders containing the genetic exams searched for were created

Family

Family linked to the genetic exams searched for

Result

Property

Property for which there should be at least one result in the orders containing the genetic exams searched for

Property value

Value that at least one result based on the selected property should have.

Locus name pattern

Name (pattern) of the locus for which the genetic exams searched for should have a result. Use an asterisk for missing characters.
Example

Example

If the locus name pattern is "T*", the genetic exams returned by the query are those that have results for loci whose names start with "T".

Locus value

The genetic exams searched for should have at least one locus result with the indicated value.

Variant name pattern

Name (pattern) of the variant for which the genetic exams searched for should have a result.
Example

Example

If the variant name pattern is "T*", the genetic exams returned by the query are those which have results for variants whose names start with "T".

Note

The results, locus results and variant results are only considered in the query if their status is Confirmed or Validated. This means, for instance, that a genetic exam containing the variant searched for will not be returned by the query if the variant result has not been confirmed yet.

Batch usage

It is possible to define a command with a genetic exam query

You can use genetic exam query in batch, for instance,

  • to execute a given action on certain genetic exams at regular intervals, such as delete all the discontinued genetic exams of the past month or print a list of urgent genetic exams every day.
  • in the Generic statistics module: select the GeneticExam table in the Base table field, select GeneticExamQuery in the Query field of the window Selection criteria.
