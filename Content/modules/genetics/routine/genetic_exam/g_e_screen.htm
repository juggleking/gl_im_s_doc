Genetic exam screen

Content: Introduction - Header - Left-hand side - Right-hand side - Approaches and requests

Introduction

The Genetic exam screen allows you to consult and add data to genetic exams created in the Order entry screen. It is composed of several modular frames.

It can be opened:

  • by clicking on the genetic exam button in the Order entry screen.
  • by right-clicking on an order (containing a genetic request) in an order browser -> Show all -> Genetic exams.
  • by right-clicking on a family (containing a genetic request) in the family browser -> Show all -> Genetic exams.
  • from the Genetic exam browser by double-clicking on a genetic exam or using the contextual menu/ribbon item Genetic exam screen.

This chapter describes the content of this screen.

Header

This frame is used to display information regarding the genetic exam such as status, urgency, comment, linked genetic procedure, materials and family. Which information is displayed can be customised.

Left-hand side

Desired date

This field contains the same date as the Desired date field in the Order tab of the Order entry screen. This date may be relevant for the choice of approach plans.

Note

If this date is modified in the Order entry screen, it will be updated in the Genetic exam screen, and vice versa.

Urgency

The urgency indicated in the Genetic exam screen corresponds to the highest urgency of all the requests in this exam. When the urgency is indicated or modified in the genetic exam, it is updated in the order and vice versa (in the Order entry screen, press F5 to refresh).

Family

The Family field allows you to select the patient's family in the context of which the genetic exam takes place.

Note

The Family field

  • is only displayed if the Support family option of the procedure instantiated by the genetic exam is set to Yes.
  • is read-only if the genetic exam has status Confirmed, Validated or Discontinued.

The choice list contains the number and name of the families of which the patient is a member.

When you select a family in the list, if a Suspected disorder is configured on the procedure of the genetic exam, GLIMS asks if this disorder should be added to the family (unless this disorder is already linked to the family). If you create a new family, the suspected disorder is automatically added to it.

If the list is empty or does not contain the right family, you can click the icon next to the Family field to open the Person search screen. This screen allows you to look up and create family records.

Family search

Person

When typing in one of the fields of the Person section, the Person search screen opens. Double-click on a person in the embedded browser of the Person search screen, then click Search in the Family search screen. The families returned are those of which the person selected in the Person search screen is a member. Select a family. The person of the genetic exam is automatically added as a member of the selected family (unless s/he was already a member of this family).

Family

Number - Name

Insert a complete family number or name (case-insensitive). Incomplete numbers or names are also supported if the missing characters are replaced by a single asterisk "*".

Example

Say the family names in the database are " Smith", "Stevens", "Sutter" and "Tames".

  • "S*" returns "Smith", "Stevens" and "Sutter".

Created after

The families returned are those that were created on or after the specified date.

  • those whose main suspected disorder is the selected disorder, or
  • those having the selected disorder as a family condition.

Proband

If this option is activated, the families returned are those which have at least one disorder with a proband.

List of returned families

Number of disorders

Family creation

After executing the search function, the New family button allows you to create a family for the patient.

This screen also allows the user to add the patient to an existing family by looking for a disorder, proband, ...

If a family is created, GLIMS gives it a unique internal id and a name.

If the mother and/or father of the patient is known in GLIMS, GLIMS asks you if they should be added to the new family.

Once the New family button has been clicked, the Family screen opens, allowing you to draw the pedigree of the family:

  • The patient, and his parents if they are known in GLIMS, are automatically added to the pedigree.
  • If a Suspected disorder is configured on the procedure of the genetic exam, it is added to the disorder list.

Note

GLIMS allows users to create a new family only if they have used the search function at least once. Before that, the New family button is not activated.

Indication

Note

If the genetic exam was requested within a panel, the request code name of the panel is displayed, not the names of the panel members.

  • Rich text or image fields: click the icon on the right-hand side or press F6 to open the rich text editor or image viewer.

Right-hand side

This frame is composed of accordion panes that can be expanded and collapsed using the double arrow icon on their right.

Specimens

The Specimens pane contains a list of the root specimens whose materials are related to the requested genetic procedure. The check-box Derived specimens allows to see the child specimens as well. The materials of the displayed derived specimens are not necessarily materials configured on the genetic procedure.

Note

When they are reused, previous derived specimens appear as root specimens in the Specimens pane. They are considered root specimens because they do not have any parent specimen in the current genetic exam.

Specimens

In the specimen list, you can

  • add a specimen: click in the list and press Insert. A browser of materials that can be used for the procedure requested in the genetic exam opens. Choose a root material, i.e. a material for which the option Is root material is activated. If mandatory or auto-prompt material variables are defined on the specimen's material, GLIMS then asks you to indicate a value for these variables. Once the variables are indicated, the Specimen info screen opens.

    The internal id of the new specimen is set via the common site function and its sampling time is set to the order lowest object time.

  • set the status of a single specimen as Available. There are two ways to do so:

    • Select the specimen and use the contextual menu/ribbon item Confirm availability.
    • Double-click the specimen to open the Specimen info screen. Click Confirm availability in the ribbon.
  • set the status of several specimens to Available or Discontinued at once: activate the checkbox next to each specimen and use the contextual menu/ribbon item Confirm availability or Discontinue.
  • delete or discontinue a non-reused specimen: select it and press Delete or click Discontinuation in the ribbon. In both cases, you need to justify the specimen discontinuation. The status of the specimen and specimen request code turns to Discontinued.

  • delete a reused specimen: select it and press Delete. Justify the request discontinuation. The specimen disappears from the list and moves to the Other specimens pane. Its status remains Available, since this specimen is used in other orders. However, the specimen request code is removed from the order.

  • discontinue a reused specimen: select it and click Discontinuation in the ribbon. Justify the specimen discontinuation. If the specimen is not linked to approaches, it disappears from the list and does not move to the Other specimens pane. The specimen request code remains in the order but its status turns to Discontinued.
  • consult detailed information on a specimen: double-click on this specimen, or select it and press F6, or click Specimen info in the ribbon. The Specimen info screen opens. If the specimen is used in more than one order and you click Specimen info in the ribbon, a window opens to let you choose the order before the Specimen info screen opens.

Approach

The Approach column contains a green icon that allows you to select a suitable approach plan for a specimen. When clicking on this icon, the approach plan selection either occurs automatically or the choice is left to the user.

Note

An approach plan is considered suitable for a specific specimen and genetic exam if it is currently valid and if on at least one of its Approach plan usages:

  • the Department is identical to the responsible department of the genetic exam,
  • the Genetic procedure is empty or identical to the genetic procedure of the genetic exam,
  • the Material is empty or is identical to the material of the specimen that will be analysed,
  • the Condition function is empty or returns TRUE,
  • the option Enabled is activated.

Automatically

The choice occurs automatically if, among the approach plans suitable for the analysis of the specimen,

  • only one has an approach plan usage
    • with a defined specific material (Only for material) identical to the specimen's material, and
    • with a defined specific genetic procedure (Only for genetic procedure) identical to that of the genetic exam, and
    • without an Extra condition functionor with one that returns TRUE or YES,
    • with the highest available Score and there is no other entry with an identical score,

    • and this approach plan has not been selected yet for the specimen.

    In this case, an approach is immediately created and displayed in the Approaches & requests pane.

    Manually

    If several approach plans are suitable for the specimen, the list of eligible approach plans opens for the user to choose:

    The approach plans are sorted by descending score, therefore with the most suitable approach plan at the top of the list. If an approach plan has already been selected for the specimen in the present genetic exam, it is displayed at the bottom of the list, regardless of its score.

    Note

    If an approach plan is eligible by more than one approach plan usage, the approach plan is sorted using the highest score of all its approach plan usages.

    For instance, if

    • two approach plan usages are configured on a given approach plan, and
    • both usages make the approach plan eligible for the current specimen and genetic exam, and
    • the score of one of the usages is 10 and the score of the other usage is 20,

    the approach plan is sorted in the Selectable approach plans list using score 20.

    Once the choice is made, an approach is created immediately and displayed in the Approaches & requests pane.

    It is possible to create more than one approach for a single specimen. To do so, just click the green icon again and select another approach plan.

    Family

    The Family pane is displayed if a family has been selected in the left-hand side frame. It contains the following elements:

    • Name: Name of the family
    • # Patients: the number of members in the family for whom there exists at least one genetic exam (no matter the procedure).

      Note

      Persons with only Discontinued genetic exams are not counted as a patient in the Family pane of the genetic exam screen and the result screen.

    • Main suspected disorder of the family
    • A Pedigree icon to open the Family screen.

    • A paperclip icon to attach files to the family or consult already attached files in the Attachments screen. When the family has attachments, an asterisk appears next to the paperclip icon:

      In the attachment screen, you can

      • zoom in and out (CTRL+ mouse scroll),
      • insert a description for the attached file.

    • Disorder grid

      • Disorders linked to the family via family conditions in the pedigree and added via the Disorder pane of the family screen.
      • Proband(s): a check mark is displayed if at least one family member (assigned or not) is marked as proband for the disorder.
      • # Patients: for each disorder, the number of assigned family members for whom a genetic exam exists and who are linked to at least one family condition of that disorder.
      Example

      Example

      Consider family Willis:

      • Three family members have a genetic exam: Yagmur, Joey and Brett.

        --> In the Family pane, the upper #Patients field is followed by the number 3.

      • Two family members, Joey and Brett, have a genetic exam and a family condition (Affected, Healthy) for the disorder X-fragile.

        --> In the Family pane, the # Patients cell for the X-fragile disorder contains the number 2.

      • Selina Ruiz and Maura Willis have a family condition, linked to the disorders X-fragile and Down5 respectively, but no genetic exam.

        --> In the Family pane, Selina and Maura are not counted as patients.

      • Joey has been set a Proband for the X-fragile disorder.

        --> In the Family pane, the Proband(s) cell of the X-fragile disorder contains a check-mark.

    • Variants grid

      Variants of a family for which at least one family member has non-discontinued results:

      • all the external variants with any classification,
      • the internal variants with the classification of 4 or 5.

      If a variant result is present twice in the family, the corresponding variant is only shown once with the highest variant result classification.

      You can use the filter icons to only display a subset of variants.

      The green icon To test allows you to add a variant request on an approach selected in the Approaches list at the bottom of the screen. This approach must have status Initial or Running and have an activity with status Pending or Running that allows you to add variant requests. When a variant request is added, a variant result is automatically created.

    Other specimens

    The Other specimens pane allows you to reuse specimens for the present genetic exam. It is only displayed if there are other specimens

    • of materials related to the requested genetic procedure,
    • available for the same person in another order,
    • that have not been reused yet for the present order.

    The Root specimens and Derived specimens options allow you to choose which specimens should be displayed. The derived specimens shown in this pane are only those whose material is one of the materials configured on the genetic procedure (otherwise, these specimens could not be reused).

    Additionally, this pane contains the specimens of the objects from when they were a foetus.

    Some of the columns are identical to those of the Specimens pane. The following columns are specific to the Other specimens pane:

    Material

    Material of the specimen. In the case of derived specimens, the mnemonic of material of the root specimen is also displayed between brackets.

    Used

    Number of approaches in which the specimen is used. Approaches in other genetic exams and for different genetic procedures are included as well.

    Available since

    Specimen availability time

    Reuse

    Click the icon to use the previous specimen in the current genetic exam. The specimen is now displayed in the Specimens pane. This triggers the creation of
    • a new requested code, with as request definition the default request definition of the specimen's material,
    • a new request linked to the reused specimen.

    Attachments

    The Attachments pane allows you to

    • see which files are already attached to the order and its genetic exams.
    • attach files to the currently open genetic exam. To do so,
      • press Insert: a dialog opens, select a file. The file appears in the attachment grid.
      • drag and drop a file from your Windows explorer into the grid.
      • indicate the type(s) of document(s) contained in the file.

    Notes

    • PDF and image files can be attached. Other file formats are not supported.
    • The configuration needed to make use of the attachment feature in GLIMS is described here.
    • Attachments may be shielded.
    • The User account control security mechanism may prevent you from using drag and drop. One way to avoid this issue is to run all your applications with the same type of rights: for instance, if you are running Windows with administrator rights, you should use GLIMS with administrator rights as well.

    Approaches & requests

    On the bottom of the genetic exam screen, you find

    • the list of approaches created for the genetic exam
    • the list of locus and variants requested for the approaches.
    • the visual representation of the approaches, as in the Approach screen.

    Approaches

    List of the specimens of the genetic exam and the approach plans that have been chosen for them. The approaches are sorted on the basis of their creation time, starting by the most recent one at the top.

    Note

    An order can contain more than one genetic procedure, so several genetic exams can be created for a single order. And given that the same approach plan can be selected for more than one genetic exam, there can be different approaches (instances) of the same approach plan for the same genetic exam in the same order.

    Specimen for which an approach plan has been chosen to create the approach

    Plan

    Approach plan on which the approach is based

    Status

    Status of the approach

    Approach status Short form  
    Initial Ini The approach has been created but its execution has not started yet because the specimen does not yet have the status Available.
    Running Run The approach has started: the specimen status is Available and at least the first activity of the approach has status Running.
    Finished Avl

    Note

    Start and end times of the approach

    Comment

    Internal comment of the approach plan

    Delete/discontinue an approach

    To discontinue an approach, select it and press Delete.

    If the discontinued approach had status Error or Running,

    • its first activity with status Pending is discontinued. The other activities with status Pending are deleted, together with their transitions.
    • its activities with status Finished are neither deleted nor discontinued.
    • its activities with status Running are discontinued.

    Requests

    Requests shown

    Once an approach has been created, the loci and variants configured on the approach plan are added to the Requests list in the bottom right corner of the screen. This list displays the following information:

    Locus

    Indicates either
    • the name of the locus that should be examined, or
    • the name of the locus where the variant should be searched for if the Variant cell is also filled in.

    Type

    Locus type

    Variant

    Variant searched for

    Origin

    Possible values:
    • Plan: the approach request was added automatically because it was configured in the Approach plan on which the approach is based.
    • Manual: the approach request was added manually by the user in the genetic exam screen.

    • Unrequested: an unrequested approach request is created when:

      • an approach request is deleted.
      • a result was added in the Results screen on a working activity belonging to approach A although the approach request corresponding to this result is linked to approach B.

      Note

      There can only be one approach request with Origin Unrequested for a given approach. All deleted approach requests and results added to the wrong activity refer to this unrequested approach request.

    You can open the read-only editor of an approach request by double-clicking on this approach request in the list.

    By default, the list shows the loci and variants requested for all the approaches of the genetic exam. You can also make the list only display the loci and variants of a single approach. To do so, select an approach on the left-hand side and tick the box as shown here:

    Adding approach requests

    You can also add requests, i.e. loci, locus groups or variants to be tested, to approaches in status Initial or Running. To do so,

    1. select an approach that contains at least one activity
      • with status Pending or Running,
      • for which locus or variant results can be added.

    2. add a locus (group) or a variant:

      • to add a locus (group), click in the Requests list and press Insert. The Add loci screen opens.
      • to add a variant, click in the Requests list and press Ctrl+ Insert. The Add variants screen opens.

    Before an approach request is added to a genetic exam, GLIMS checks if there is already an identical approach request

    • for the same person,
    • in the same or another non-discontinued approach,
    • in the same or another non-discontinued order.

    If GLIMS finds an identical approach request in

    • the same approach, the approach request cannot be added.

    Deleting approach requests

    Approach requests can only be deleted if the status of the approach is Initial or Running.

    When a locus approach request is deleted,

    • if all corresponding locus results have status Expected, they are deleted as well.

    • if at least one corresponding locus result has status Available or higher, none of these locus results is deleted. However, the approach request to which they refer is now the approach request with Origin Unrequested.

    When a variant request is deleted, the corresponding variant result is never deleted (because the status of variant results is always at least Available). However, the approach request to which this variant result refers is now the approach request with Origin Unrequested.
