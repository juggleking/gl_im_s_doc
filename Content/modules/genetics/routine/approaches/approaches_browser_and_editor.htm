Approaches

The Approach browser can be opened from an approach plan, a genetic exam and a specimen of a genetic exam using the contextual menu/ribbon item Approaches. Double-click or use the F6 key on an approach record to open the approach visualization.

Genetic exam order

Order that contains the genetic exam to which the approach belongs

Genetic exam genetic procedure

Requested genetic procedure for which the genetic exam has been created

Approach plan

Approach plan used to create the approach

Specimen

Specimen analyzed in the context of the approach

Status

Status of the approach

Approach status Short form  
Initial Ini The approach has been created but its execution has not started yet because the specimen does not yet have the status Available.
Running Run The approach has started: the specimen status is Available and at least the first activity of the approach has status Running.
Finished Avl

All the activities of type End have been reached.

Note

If an activity of a finished approach is re-opened (because a new action has been created for this activity), the approach status turns to Running.
Discontinued Dis

The approach is discontinued because the order has been discontinued (manually or by the Order.Discontinue function), for instance.

Note

  • When a discontinued approach contains a confirmed normal result with a higher severity than the severity of the genetic exam, the latter will be updated.
  • The discontinuation of an approach will not discontinue Confirmed or Validated locus / variant results.
Error Err One of the approach activities has status Error.

Approach activities

The approach activities browser can be opened from the Order outline. Select an activity and press F6 or double-click to open the activity's editor.

Approach activity editors can be opened using the contextual menu/ribbon item Approach activities on Approach records:

Approach plan - Display name - Procedure - Type - Must be duplicated for each specimen - Routed material

These fields contain the same information as the corresponding fields in the Plan activity editor.

Await action status

This field contains the same information as the corresponding field in the Plan activity editor. However, this field is emptied when

  • the action has reached this status

    Example

    Example

    In the Plan activity editor of Plan activity P, Await action status is set to Outputting.

    In the editor of the corresponding Approach activity P, Await action status initially also contains Outputting. However, when the action created in the context of this activity has reached status Outputting, the Await action status field on the Approach activity editor is cleared.

  • the activity has status Pending, Running or Error and is discontinued.

Start time - End time - Status - Transition error

These fields contain the same information as the corresponding fields in the Approach screen.

Approach specimen

Scoped specimen

If the activity is of type Start, the scoped specimen is the specimen of the approach.
If the activity is of type Procedure,
  • the scoped specimen is the input specimen of the action created for the procedure.
  • if the activity is duplicated, each duplicate activity has its own scoped specimen.
  • if the activity is a pooling activity, it has no scoped specimen.
  • if an action has no input specimen, the corresponding activity has obviously no scoped specimen.
If the activity directly follows an activity of type Router, the scoped specimen is the specimen that was routed to the activity.

Error message
