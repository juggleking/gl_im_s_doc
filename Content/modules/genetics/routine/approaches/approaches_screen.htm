Approach screen

Introduction

Once an approach has been created, it can be visualised in the Approach screen.

To open the Approach screen, use the contextual menu/ribbon item Visualize approach in the Approach browser or editor.

The Approach screen looks as follows:

The left-hand side frame provides a visual representation of the configured approach.

The right-hand side frame allows you to consult information regarding the activities, transitions and actions of the approach.

Note

The approach screen can be refreshed by pressing F5. Changes performed outside the screen (such as Specimen replace and Result repeat) become visible. For instance, when a result is repeated, the corresponding action is discontinued and a new action is created: the new action is visible in the Approach screen.

Visual representation of the approach

The visual representation shows the approach with its activities and their transitions. Users can switch between

  • a vertical and a horizontal view using the Left-Right and Top-Bottom buttons
  • a detailed or compact view using the Compact option. In the compact view,
    • only the approach activities based on approach plan activities of type Procedure and Router are shown.
    • the transitions are all represented by a plain arrow, independently of whether they are configured as fallback or with an Only when function.

The layout is similar to that of approach plans. In the following sections, only the approach-specific layout elements are described.

Note

This visual representation is only displayed if a Webhost URL is indicated.

Activities of type Void

When Duplicate by specimen is activated on a approach plan activity, an approach activity of type Void will be added automatically in the visual representation of the approaches based on the approach plan. This is to improve the readability.

Transitions

If a transition error occurs, the transition is displayed with a red dotted line and the word Error.

When the Condition function of a transition returns False, it is deleted together with the branch following it.

Transition error

If an error occurred during the evaluation of the activity's outgoing transition(s), this field indicates the error type. This can be for instance:
  • MISPL failure: the condition function of (one of) the activity's outgoing transition(s) raised an error.
  • Stuck: all the activity's outgoing transitions have an Only when function and none of these functions returned True.

If the transition is solved, the approach is resumed.

Discontinue

Click this button to discontinue the selected approach activity.
  • if the activity had status Pending, its status turns to Discontinued and the part of the branch following the activity is deleted,
  • if the activity had status Running or Error, its status turns to Discontinued and
    • all the non-available procedure in- and outputs are discontinued (i.e. results with status lower than Confirmed, specimens with status lower than Available). As a consequence the approach activity action gets status Ready (because now, all outputs are either confirmed, validated or discontinued and all output specimens are available or discontinued).
    • the outgoing transitions are deleted.
    • if the
      • next activities have status Pending, they are discontinued. All following pending activities that are part of the same branch are deleted.
      • next activities have status Running, they start or continue (unless their scoped specimen was the output specimen of the discontinued activity). The approach keeps the status Running.

Transition

Condition function - Display name - Is fall back - Seq. no

The information displayed in this editor is identical to that of the corresponding Plan transition editor.

Error message

If an error occurs during the execution of the transition's Condition function, this field displays the error.

Warning

If an error occurs during the execution of a transition's Condition function, the other transitions going out of the same 'From' activity are blocked.

After correcting the error, you can launch a re-evaluation of the outgoing transitions of an activity manually using the contextual menu/ribbon item Evaluate transition error in the activity context.

 

Note

If there are less specimens to be routed than transitions configured in the approach plan, the transitions with the highest Seq. no., i.e. with the lowest priority, are deleted.

Actions

The Actions pane is only open when an activity of type Procedure is selected in the visual representation.

The pane provides a visual representation of the action created to execute the procedure linked to the plan activity:

  • Users can choose between a compact, vertical or horizontal representation.
  • If a value has been entered for the property outputs, this value is shown (unless the compact representation is selected). Otherwise a "?" is displayed. If the data type of a property output is String with the option Rich text activated, the value displayed is "Rich text".
  • The status of each action is displayed (unless the compact representation is selected).
  • Material inputs and outputs are displayed with their internal ids.
  • If the value of the field Number of specimens to be auto requested is higher than 1, each material output is displayed individually.
