Results screen

Content: Introduction - General content - Left-hand side - Center - Right-hand side

Introduction

The Results screen is dedicated to the results of genetic exams. It can be opened using the contextual menu/ribbon item Result screen on Genetic exam or Approach records.

Warning

You should only insert, confirm and validate genetic results in the Results screen, not in the classic result browsers.

This chapter describes the content of this screen.

General content

The Result screen is composed of several modular frames.

All the data inserted and updated in this screen is saved immediately.

Header

This frame displays customizable information, for instance about the genetic exam, the patient, the issuer, etc.

Left hand-side

Introduction

The left-hand side of the screen provides an overview of the genetic exam approaches and their activities.

Header

Indication

Labels of the request codes used to request the genetic exam during order entry.

Note

If the genetic exam was requested within a panel, the request code name of the panel is displayed, not the names of the panel members.

Approaches

The Approaches pane displays the approaches of the genetic exam that are in status Running, Finished or Discontinued, if the status of the genetic exam is lower than Confirmed. Only approaches in status Finished or Discontinued are displayed if the status of the genetic exam is greater than or equal to Confirmed. The approaches are sorted on the basis of their creation time, starting by the most recent one at the top.

Tick the boxes on the left to select approaches of which you want to see the activities in the Activities pane underneath . To (de)select all the approaches at once, use the tick box on the left-hand side of the grid header.

Activities

The Activities pane displays the activities that are

  • part of the approaches selected in the Approaches pane,
  • of type Procedure,
  • in status Running or Finished.

The activities are sorted per approach (same order as in the Approaches pane), start time, internal id of the scoped specimen, and id.

The Specimen internal id column contains the word pool if the activity has several specimen inputs.

Tick the boxes on the left to select activities: the output specimens and (locus/variant) results of the corresponding actions appear in the middle of the screen. To (de)select all the activities at once, use the tick box on the left-hand side of the grid header. You can also define which activities should be shown in the grid, using the filter function.

Note

The working activity is set in bold: it is the activity whose specimens and results you want to update in the Results screen and to which documents can be attached in the Attachments pane underneath. Double-click on an activity to make it become the working activity.

Attachments (selected activities)

The Attachments pane displays the documents attached to the activities selected in the Activities pane.

To attach a document to the working activity:

  • select this activity using the tick box, and
  • press Insert, or
  • drag and drop the document

in the Attachments grid.

You can also open the List of attachments screen, by double-click in the Attachments grid. In this screen, you can

  • zoom in and out (CTRL+ mouse scroll),
  • select a file and insert a description for it.

Notes

  • PDF and image files can be attached. Other file formats are not supported.
  • The configuration needed to make use of the attachment feature in GLIMS is described here.
  • Attachments may be shielded.
  • The User account control security mechanism may prevent you from using drag and drop. One way to avoid this issue is to run all your applications with the same type of rights: for instance, if you are running Windows with administrator rights, you should use GLIMS with administrator rights as well.

Center

Introduction

The center of the Results screen contains all the specimens and results (normal results, locus results and variant results) configured as outputs for the activities selected in the Activities pane. From there, the effective results (numeric values, images, rich texts, etc) can be inserted, confirmed, validated or discontinued. These result updates make the related activities and approaches progress automatically.

The results are presented in different panes: Specimens, Results, Locus results, CNV (for retesting), Sequence variants (for retesting), Interpretations. These panes are only displayed if there are corresponding specimens/results to be shown. For instance, if there is no specimen output for the activity selected on the left-hand side, the Specimens pane is not displayed. Some of the panes contain grids in which the records can be filtered.

Note

The content of this part of the screen is similar to that of the advanced work list editor. There is however one crucial difference: in the present Results screen, the specimens and results displayed are the outputs for the genetic exam's activities. In the advanced work list editor, the specimens and results displayed are the outputs of the actions selected on the basis of a configured work list template.

Header

Above the specimen and result panes, you can see the status and mnemonic of the genetic exam, as well as two buttons:

Confirm all

Click to confirm all the specimens and (locus/variant) results currently displayed on the screen and with status Available.

Validate all

Click to validate all the specimens and results currently displayed on the screen and with status Available or Confirmed.

 

When clicking on these buttons, a summary screen provides you with a list of all the specimens and results that are about to be confirmed/validated. They appear in the same order as in the editor:

Notes

  • CNVs and sequence variants with status Available and retest status To do are skipped.
  • Confirming or validating all results is not allowed if
    • this action leads to the end of the approach, and
    • a locus result is lacking a value.
  • In the summary screen, small arrows allow you to expand and collapse certain results. This is solely to improve the readability but has no effect on the validation/confirmation: collapsed results will also be confirmed/validated.
  • In the summary screen, results of activities with the same name but for different specimens are followed by the specimen ID to avoid confusion.

To confirm or validate one or more specimens or results but not all, select them and click on the contextual ribbon items Confirm or Validate.

If all specimens and results are validated, the status of the genetic exam turns to Validated.

Warning

Validation by delegation is not yet supported in the genetics results screen!

Specimens

The Specimens pane displays the specimen outputs of the activities selected on the right-hand side, sorted per activity.

To confirm the availability of one or more specimens, select these specimens and click Confirm availability in the ribbon.

Results

The Results pane displays the results of (pre-)analytical procedures, except locus and variant results.

Note

The results are sorted per activity. Per activity, their order is defined by the sequence numbers of their corresponding procedure outputs.

Results of activities with the same name but for different specimens are followed by the specimen ID to avoid confusion.

Differently from the advanced work list editor, the present screen contains an additional pane, Interpretations, which contains the results of post-analytical and validation procedures

To edit

  • rich text or image fields: click the icon on the right-hand side or press F6 to open the rich text editor or image viewer.

  • other field types: press F6 to open the Result modify screen, as illustrated below. In dropdown lists, double-clicking opens a result code list.

Locus results

The Locus results pane displays the locus and variant results

  • of the locus requests added in the Genetic exam screen to the selected approaches.

  • added manually in the grid by pressing Insert or clicking the +Variant and +Locus buttons.

    Note

    It is only possible to add a locus or variant result if the working activity allows it: the procedure to which the working activity is linked must have an output property with List type Locus result or Variant result.

    Warning

    Manually added results will always be added to the approach to which the working activity belongs. Therefore, when adding a result manually, make sure the working activity belongs to the same approach as that of the approach request for which you are adding a result.

    Insert

    Pressing Insert allows you to add locus results directly to the working activity.

    +Variant

    Allows you to add a variant result to a selected locus result when a variant has been found on the tested locus.

    Notes

    • In the Add variants screen that opens when you click on this button, the Locus field contains the locus linked to the selected locus result. No other locus can be selected in this field.

    • If the selected locus result is linked to a locus of type

      • Gene (region), both CNV and sequence variant results can be added.
      • Chromosome (region), only CNV results can be added.
      • Locus group, neither CNV nor sequence variant results can be added.

    +Locus

    Allows you to add a child locus result to a locus result selected in the list. The latter locus result becomes the parent locus result.

    This can be used, for instance, if a more specific locus (e.g. chr 12, region q24.31) was tested than the locus initially requested (e.g. chr 12).

    Notes

    • If a child locus result C is added to a parent locus result P that is a member of a locus group result G,
      • the Requested column displays the locus group result G,
      • the Locus column displays the child locus result C,
      • the parent locus result P is visible by using the contextual menu/ribbon function Parent locus result.
    • If a parent locus result is discontinued, its children locus results are discontinued as well.
    • The child locus result refers to the same approach request as its parent.

The grid contains the following columns:

Requested

If an individual locus was requested, the name of this locus is displayed.
If a locus group was requested, the name of this locus group is displayed, for both the locus group result and the results of the locus group members.
If a variant was indicated in the approach request, the name of this variant is displayed.
If a locus result was added to the result screen manually, this field will remain empty.

Locus

If an individual locus was requested, the name of this locus is displayed.
If a locus group was requested, the name of this locus group and the names of the locus group members are displayed.
If a variant was indicated in the approach request, the name of the locus to which the variant is linked is displayed.

Rp.

Tick the box if the locus result should be displayed on reports.

Value

Select a value out of the customised list. When a value is selected, the severity associated to it appears automatically in the Sev column.

Sts

Status of the locus result: Expected, Initial, Confirmed, Validated, or Discontinued.

Sev

Severity of the locus result. Automatically set when a value is selected in the Value column. If no value is indicated, the severity is 0.

Comment

Free text

 

Note

These columns are only displayed if they have some content to show. For instance, if no comment has been entered for the locus results displayed, the Comment column is hidden.

Additional columns

Customised columns (e.g. Mitose, Nucleii)

CNVs

The CNVs pane displays the CNV results for CNV requests added in the Genetic exam screen to the selected approaches. If all the selected activities are linked to a retest procedure, the name of this pane is CNVs for retesting.

Press Insert to add a CNV result.

Note

It is only possible to add a CNV result if the working activity allows it: the procedure to which the working activity is linked must have an output property with List type CNV.

The columns correspond to the fields of the CNV editor except the following ones:

Rp.

Indicates whether the variant result should be displayed on reports. If the variant result was
  • inserted manually, and the clinical classification is superior or equal to 4, this box is automatically activated.
  • added through a .vcf file, this option is automatically activated or not, depending on the information provided in this file.

Clin. classif.

If the variant classification indicated in the Results screen is different from the classification configured for the tested variant, the configured classification can be updated under certain conditions:

If

  • the variant has not been approved yet, the user is asked whether the configured classification should be updated.
  • the variant has already been approved, and
    • the user has role user type Developer or System manager, they are asked whether the configured classification should be updated. If the user answers "yes", the variant's configured classification is updated and the initial approval date and user are replaced by the current date and user.
    • the user has a role user type different from Developer or System Manager, there are two possibilities:

      • if the user has a privilege for the Variant.ApprovalDateTime field, they are asked whether the configured classification should be updated. If the user answers "yes", the variant's configured classification, approval date and approval user are updated.

      • if the user has no privilege for the Variant.ApprovalDateTime field, no question is asked and the configured classification cannot be updated.

Sts

Status of the variant result: Available, Confirmed, Validated or Discontinued.
When variant results are created, their status is set to Available automatically.

Retest

Fixed choice list:

  • Not req.: no new test was requested. The variant result can be confirmed or validated.
  • To do: the variant must be tested again. The new test has been requested, it will be performed in an approach activity that is linked to a retest procedure. The variant result cannot be confirmed or validated.
  • Confirmed: the new test has been performed and the result matches that of the previous test. The variant result can be confirmed or validated.
  • Not confirmed: the new test has been performed and the result is different from that of the previous test. The variant result can be confirmed or validated, its status turns to Discontinued.

Tips

  • Add the retest status parameter in a MISPL to create work lists containing only variants to be retested.
  • Add the retest status parameter in a MISPL on an approach plan transition to skip a retest activity if there is no variant result with status To do.
  • Combine the status and retest status parameters in a MISPL to report retested, not confirmed variant results.

Relev.

Indicates whether the variant result is relevant or not in the context of the genetic exam. By default, if the variant result was

  • inserted manually and the clinical classification is superior or equal to 4, this box is automatically activated.
  • added through a .vcf file, this option is automatically activated or not, depending on the information provided in this file.

Interpretations

The Interpretations pane displays

  • the same Family field as in the genetic exam screen,
  • the results linked to post-analytical and validation procedures.

Note

The results are sorted per activity. Per activity, their order is defined by the sequence numbers of their corresponding procedure outputs.

Effect of result updates on activities and approaches

Inserting, modifying, confirming, validating or discontinuing results makes the related activities and approaches go to a following stage or go back to a previous stage. For instance,

  • when all the results linked to a given activity are validated, this activity gets the status Finished and the next step in the approach starts.
  • when one of the results linked to a finished activity is inserted or modified, the status of the finished activity goes back to Running (because the corresponding action is re-opened). If the finished activity is part of a finished approach, the status of this approach goes back to Running as well.

Right-hand side

The right-hand side of the screen provides information regarding the patient and his/her family.

Other genetic exams

The Other genetic exams pane displays all the other non-discontinued genetic exams existing for the patient in other orders. If there is no such exam, the pane is not displayed.

Order

Internal id of the order to which the genetic exam belongs

Date

Reference time of the requested code(s)

Indication

Labels of the request codes used to request the genetic exam during order entry.

Status

Status of the genetic exam

Mother's genetic exams

The Mother's genetic exams pane displays all the non-discontinued genetic exams of the person indicated in the Mother field of the patient's Person editor. If there is no such exam, the pane is not displayed. The columns of the grid are identical to those in the Other genetic exams pane.

Note

The Mother's genetic exams pane is displayed in the result screens of all the genetic exams created for the patient.

Father's genetic exams

The Father's genetic exams pane displays all the non-discontinued genetic exams of the patient's father.

If

  • the patient's father has no non-discontinued genetic exam, or
  • the genetic exam for which the result screen is open is not linked to a family, or
  • the patient's father cannot be determined on the basis of the pedigree of the family,

the pane is not displayed.

The columns of the grid are identical to those in Other genetic exams pane.

Family

The Family pane is identical to that of the Genetic exam screen.

Attachments (order/exam)

The Attachments pane is identical to that of the Genetic exam screen.
