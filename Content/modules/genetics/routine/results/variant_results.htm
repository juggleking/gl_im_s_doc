Variant results

Detailed information on a variant result is available in the Variant results browser and editor: right-click on a Variant record -> Show all -> Variant results.

Most of the fields of the Variant result editor correspond to the columns of the CNVs and Sequence variants panes of the Results screen. Additional fields are

Department

Department in which the result was entered

Object

Person on which the presence/absence of the variant has been tested.

New

The box is checked if the variant has been found in a genetic exam of the lab for the first time (at least one internal variant result for this variant was added while the Found internally check-box was not yet active).

Genetic exam order - Genetic exam genetic procedure

Order and genetic procedure in the context of which the variant result has been obtained.
These fields are empty if the variant result was added using Add variant result.

Entered on

Date on which the result was entered (when the variant was created), i.e. on which its status was set to Available.
This date is updated when a confirmed or validated result is modified.

Confirmed on

Date on which the result was confirmed.

Notes

  • When a result with status Available is directly validated, the confirmation date is set to the validation date.

  • When a confirmed or validated result is

    • discontinued, the confirmation date is not cleared.
    • updated, the confirmation time is cleared.
  • This field can be used to run a (daily) command based on confirmation time, instead of using a confirmation trigger.

Annotations

Read-only textual information on the variant. The content of this information field can be set or retrieved using MISPL functions.

Example

A list of comma-separated "key-value" pairs (nesting, i.e. lists within lists, is not supported):

{

"af": 0.1078, "afr_af": 0.087, "amr_af": 0.1297, "eas_af": 0.0813, "eur_af": 0.1481, "sas_af": 0.1063, "aa_af": 0.09623, "ea_af": 0.1543

}
