Family screen

The family screen allows you to edit families. It can be opened by:

  • opening a Family record in the Family browser,
  • selecting the contextual menu/ribbon item Pedigree in the Family browser,

  • clicking on the Pedigree icon in the Family pane on the right-hand side of the Genetic exam screen.

Note

The creation of a non yet existing family is only possible from the Genetic exam screen.

The family screen is composed of four modular frames. They are described in the following sections.

Header

The header frame is used to display information regarding the family. Which information is displayed can be customised.

Left-hand side

The frame on the left-hand side is composed of several accordion panes:

Family

The Family pane contains the following fields:

Number

Family internal id. Can be
  • configured,
  • modified manually,
  • composed of 15 characters maximum.

Must be unique.

Name

Name of the family. Can be
  • configured,
  • modified manually,
  • composed of 150 characters maximum.

Main suspected disorder

Main disorder under investigation for the family.
Click the ... button and select the main suspected disorder of the family from the list.

Note

The main suspected disorder can be chosen freely, even if a suspected disorder is configured on the genetic procedure on which the genetic exam is based.

History

Rich text field to add family (historical) information.

Attachments

The Attachments pane contains a grid that allows you to

  • attach files to the family. To do so
    • press Insert: a dialog opens, select a file. The file appears in the attachment grid.
    • drag and drop a file from your Windows explorer into the grid.

    Notes

    • PDF and image files can be attached. Other file formats are not supported.
    • The configuration needed to make use of the attachment feature in GLIMS is described here.
    • Attachments may be shielded.
    • The User account control security mechanism may prevent you from using drag and drop. One way to avoid this issue is to run all your applications with the same type of rights: for instance, if you are running Windows with administrator rights, you should use GLIMS with administrator rights as well.

  • see which files are attached to the family. Double-click a file to open it in the attachment screen.

    In the attachment screen you can:

    • press Insert to add a file,
    • drag and drop a file from your Windows explorer,
    • zoom in and out (CTRL + mouse scroll),
    • insert descriptions for the files.

Disorders

The Disorders pane contains the mnemonic and name of the disorders that are or can be linked to the family members via the legend.

To add a disorder, press Insert and select a disorder from the Disorder selection browser. The added disorder is visible in the Disorders pane and the Legend of the pedigree.

If you select a main suspected disorder in the Family pane, it is added automatically to the Disorders pane and to the legend of the pedigree. If you then select a different main suspected disorder in the Family pane, it is added to the Disorders pane. The previously selected one is deleted from this pane, unless some conditions linked to this disorder were assigned to some family members in the pedigree.

Variants

Variants of a family for which at least one family member has non-discontinued results:

  • all the external variants with any classification,
  • the internal variants with the classification of 4 or 5.

If a variant result is present twice in the family, the corresponding variant is only shown once with the highest variant result classification.

You can use the filter icons to only display a subset of variants.

This pane and the variants pane on the right-hand side of the screen are immediately automatically updated if i.e. a new person is added to the Pedigree grid (see further below). If the corresponding person record needs to be updated and/or used for any other purposes, the changes to the Family screen need to be saved first.

Pedigree

The pedigree frame allows you to create and edit the pedigree of a family using the standard NSGC symbols and naming.

Pedigree display

The pedigree is displayed if

  • the configuration is correct.
  • a family has been selected in the Genetic exam screen.

The following buttons can be used to modify the way the pedigree drawing is displayed:

Drawing a pedigree

This chapter describes how to draw a pedigree in the family screen.

Create a new family and add family members

Creating a new family is possible starting from a patient in the genetics exam screen.

Note

The option Create new family (accessible with a right click in the pedigree) allows you to add a family tree in the pedigree. The members of this new family tree will however be added to the same Family record in GLIMS as the member of the family tree that is already present in the Pedigree. No new family record is created in GLIMS.

To add a member to a family, right-click in the pedigree -> Create new entry.

To add a relatives to an existing family member, select this member and click on one of the coloured symbols:

Additional information on family members

When persons are assigned to family members, their first name, last name and birthdate appear under the family member symbol. The first name may be preceded by asterisks.

Asterisks

No asteriskThe person assigned to the family member is not an object in GLIMS, i.e. there is no order for this person in GLIMS.
[*]There is at least one order in GLIMS, but no genetic exam, for the person assigned to the family member. Discontinued orders are taken into account.
[**]There is one or more genetic exams in GLIMS, but no non-genetic orders, for the person assigned to the family member. Discontinued genetic exams are taken into account.

[***]

There is one or more genetic exams and more than one orders in GLIMS for the person assigned to the family member. Discontinued genetic exams are taken into account.

Note

If a non-discontinued foetal specimen is linked to a person (a foetus or a foetus on creation) assigned to a family member, this person is considered as having a genetic exam. So the name is preceded by at least two asterisks.

Birthdate

The birthdate displayed is identical to that indicated in the Person editor.
The birthdate format, as all date formats in GLIMS, can be customised in the session.pf file (cf chapter System management -> Parameter tuning -> session.pf).

Set family member as Proband

One or more probands can be set for each condition linked to the family. To set a family member as proband for a condition:

  1. make sure that the condition's disorder is the only one displayed in the pedigree: the tick box on the left of the disorder in the legend is activated. The tick box on the left of the other disorders should be deactivated.
  2. select the condition in the legend and drag and drop it to the symbol of the family member.
  3. right-click on the family member symbol -> Set proband.
  4. A Proband icon appears next to the lower left corner of the symbol.

A family member can be set as proband for more than one condition. In that case, however, only one Proband icon is displayed next to the symbol.

To unmark a member as proband:

  1. make sure that the disorder assigned to the member is the only one activated in the legend,
  2. right-click on the family member symbol -> Unset proband.

Merging symbols

Two symbols can be merged in the pedigree using drag and drop -> Merge entries. However, this is only possible if one the following conditions holds:

  • The two symbols have persons assigned that are linked to the same person in GLIMS.
  • At least one of the two symbols is not yet linked to a person.

Swapping persons

Two persons can be swapped in the pedigree using drag and drop -> Swap assignments. In that case, person A moves to the symbol and family conditions initially assigned to person B and vice versa.

Swapping entries

Two entries (person assignment together with a symbol and family conditions) can be swapped in the pedigree using drag and drop -> Swap entries. In that case, entry A moves to the place of entry B and vice versa.

Saving the changes

For the changes to be saved, you need to click Apply or OK on the bottom of the screen. Clicking OK also makes the screen close.

Legend

List of the disorders added in the Disorders pane. For each disorder, Family conditions, such as 'healthy' or 'carrier', can be added freely in the legend. The tick boxes allow you to indicate which disorders and family conditions should be displayed in the pedigree.

Pedigree data in GLIMS

Saved data

Once the Apply or OK button has been clicked, the pedigree data is registered in GLIMS and visible in the corresponding read-only browsers and editors:

  • When a family is added, it is listed in the Families browser.

    Note

    If

    • the pedigree already contains a family tree, and
    • another family is added in the pedigree (using the right mouse-click option Add family),

    the members of both family trees are linked to the same Family record in GLIMS and therefore appear in the same Family members browser.

  • When a member is added, it is listed in the Family members browser.
  • When a condition is added to a family member, it is listed in the Family member conditions browser.

Note

Genetic gender and administrative sex are kept separate in GLIMS:

When a gender symbol is replaced by a symbol with another shape in the pedigree, the administrative sex of the person assigned to that symbol is not modified in GLIMS. For instance, if a square symbol linked to Mr Peter Loyds is replaced by a circle symbol, Peter's administrative sex is not set to F in GLIMS.

Mother and father

Mother

If GLIMS can identify the mother on the basis of the pedigree, the mother's name is displayed in the Mother field of the patient's Person editor.
The identification of the mother works as follows:
  • If there is only one person assigned to a parent circle symbol, GLIMS takes this person to be the patient's mother. In the other cases (e.g. two parent circles linked to persons, no parent circle symbol), the mother remains undefined.
  • If a new person is assigned to the single circle symbol, the Mother field is automatically updated: it displays the name of the new mother.
  • Differently, if the assignment of the person to the circle is canceled (Unlink button), the Mother field is not cleared: it keeps displaying the name of the initially assigned person. Similarly, if the pedigree is updated in such a way that the mother can not longer be defined, the name of the mother previously defined remains in the Mother field.

Father

GLIMS determines the father as follows:
  1. Parent symbols with no Person record assigned are excluded.
  2. The parent symbol linked to the person indicated in the Mother field of the Person editor is excluded.
  3. If after 1 and 2 there remains
    • one assigned parent, this parent is the father:

    • two assigned parents, the father is the person assigned to the square symbol:

 

Note

When trying to identify the mother/father, GLIMS does not take the parent's administrative sex into account.

Right-hand side

The frame on the right-hand of the Family screen contains a series of panes.

Selected person

The Selected person pane is displayed when a pedigree member (square, circle or diamond) is selected. It contains two buttons:

  • Assign: opens a Person browser or search screen in which you can select or create a person and link him/her to the selected member in the pedigree.

    Note

    The gender of the selected person in GLIMS does not affect the gender in the pedigree. This means that

    • it is for instance possible to link a female family member (circle in the Pedigree) with a Person record of sex Male.
    • If the sex of the Person is modified in GLIMS, this change is not reflected in the Pedigree.

  • Unlink: deletes the link between a person and the selected member in the pedigree.

    Note

    This button is deactivated when the selected person is the object of

    • the current order
    • another order that contains a genetic exam linked to the same family as the one being edited.

Note

When one of these buttons is clicked, the current pedigree is first saved.

Foetus

The Foetus pane is displayed if the selected person is a foetus. The fields are read-only.

Genetic exams

The Genetic exams pane is shown if a person selected in the pedigree has non-discontinued genetic exams. It is similar to the Other genetic exams pane in the Genetic exam screen but also contains the current genetic exam.

Variants

The Variants pane displays all the non-discontinued variant results linked to the person selected in the pedigree, which are linked to a genetic exam (added internally) with a clinical classification of 4 or 5 OR which are not linked to a genetic exam (added externally) with any clinical classification. This pane allows filtering and refreshes immediately when a different person is selected, a person is unlinked, etc.

Columns

External

The External cell contains a check mark if the variant was added outside the context of a genetic exam, i.e manually using Add variant result (see next subsection). The cell is empty if the variant is internal, i.e. if it was found via a genetic exam in your lab. Only external variant results can be updated on the family screen, internal variant results appear in the list as read-only.

Locus

For more information, see the corresponding chapter.

Clinical classification

By default, this field contains the clinical classification configured on the variant. The clinical classification of an external variant can be changed directly in this pane by right-clicking on an entry in the list.
When the classification is updated in this variant result pane of the selected family member, the variant pane of the family on the left-hand side of the Family screen is updated accordingly.

Allelic state - Parental origin

These fields are described here.

Note

The colors inside the Variants pane are updated if

  • the clinical classification of an External variant is updated
  • Internal variant results are confirmed/validated via the contextual ribbon.

Add a variant on a person

You can add variants on a selected person via the ribbon item Add variant result on the selected Family member. The Variant search screen opens. To be able to add a variant result, an object is required. If none exists for the person in question, an object is created automatically. If there already is a validated variant result for the chosen variant, a confirmation is required if this (external/internal) variant result should be added again. The selected variants are validated automatically.

Warning

Variant results added to a person using Add variant result remain linked to the person, even if the assignment of that person to a family member in the pedigree is undone.

Main suspected disorder

Family member conditions

Opens the Family member conditions browser.

Person

Result overview

To consult the results of the selected family member.

Consults

To access the consults registered for the selected family member.

Data sheet

Opens the family member's data sheet.

Orders

To access the orders created for the selected family member.

Add variant result

See above.

Disorder tab

The ribbon tab Disorder is displayed when a disorder is selected in the Disorder pane on the left-hand side of the screen.

Disorder genes

Opens the list of genes related to the selected disorder.

Genetic procedures

Opens the list of genetic procedures whose Suspected disorder is the selected disorder.

Variant result
