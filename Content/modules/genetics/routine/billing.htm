Genetic exam tariffication

This chapter describes how genetic exams are tariffed. The required configuration is described here.

Workings

During the tariffication of an order set, the billing codes assigned to the request definitions of type Genetics are used to create billing items (via the tariffs the billing codes are linked to).

When a billing item is created,

  • it is linked to the request of type Genetics.
  • the responsible department of the genetic exam becomes the executing department of this billing item.

When the genetic exam, and therefore its report result, are validated, the responsible person of the report result becomes the executor of the billing item if

  • this report result responsible person is known, and
  • if the option Validator is executor is activated.

Otherwise, the billing item executor is taken from the Provision table.

Note

If the same (panel) request is added more than once in an order, it is only tariffed once.

Conditions

For the tariffication to work,

  • the status of the genetic exam should not be Discontinued,
  • the To charge option on the requests and request definitions must be activated.
