Export

Configured approach plans can be exported and then imported in other databases together with the records linked to them (Genetic procedure, approach plan loci, usages, activities, etc).

This chapter describes

  • how to export and import approach plans
  • which data is not exported-imported

How to export and import approach plans

To export an approach plan:

  1. Click the contextual menu/ribbon item Advanced -> Export approach plan.
  2. The Export approach plan window opens. Select the folder where the approach plan should be exported and activate the option Generate export script.
  3. Once the export is completed, open the export folder: it contains csv files and an import.txt file.

Now the import.txt file must be imported into a different environment:

  1. Go to Start -> System management -> Database -> Import.
  2. The Denormalised import window opens.
  3. In the Advanced tab,
    1. activate the Use script option
    2. in the Script file field, select your exported .txt file.
  4. Set the options according to your needs and click OK.
  5. Once the import is completed, open the approach plan browser and check if the exported-imported approach plan appears in it.

Note

If

  • an element E is deleted from an approach plan A that has already been exported-imported, and
  • approach plan A is re-exported/imported in the same database,

element E remains in the database after re-export, it is not deleted.

Limitations

The following data is not exported-imported

  • Default property classifications: to be visible in the result browser, each property has to be placed manually in the default property classification.
  • The property classification defined on the genetic procedure for use in the Genetic exam screen.
  • Property classifications defined on the materials for use in the Specimen info screen.

The export-import of (references to) Person records is not supported.
