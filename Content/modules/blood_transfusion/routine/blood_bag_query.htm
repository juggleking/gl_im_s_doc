Blood bag query

The blood bag query is available from the main menu via Start -> Transfusion management -> Query blood bags.

General page

Received / Transfused / Checked out / Returned / Status expired /Discontinued - from / to

To select blood bag(s) on the basis of a specific status set within a specific time range.
Status Expired allows to search on the basis of the time at which the blood bag was set to status Expired. To search on the basis of the actual expiration date, use the option Expiration date from/to (see below).

Status from / to

To look for blood bags whose status is within a certain status range.
If the minimal and the maximal status of the blood bag are both set to status Checked out, an additional field is displayed for you to enter a number of days. If you specify '3', for instance, only blood bags that have been checked out at least 3 days ago will be shown. The check-out time is not taken into account, so it doesn't make any difference whether the bag was checked out in the morning or in the afternoon, for instance.

Expiration date from / to

To specify a minimal and maximal expiration date. When the expiration date is exceeded, GLIMS does not automatically set the blood bag status to Expired. So to query on status Expired, use the option Status expired from/to mentioned above.

Product - Specification

To specify the blood product (specification) of the blood bags being looked for.

Blood group - Rhesus-D

To specify the blood group and rhesus-D of the blood bags being looked for.
It is also possible to specify a blood group pattern like in the recipient types and donor rules: A* (=A/A1/A2), A*B (=AB/A1B/A2B).
Selecting blood group N.A. will select blood bags with blood group 'N.A.' or '?' (unspecified blood group). Selecting rhesus * will select blood bags with rhesus 'N.A.' or '?' (unspecified rhesus).

Blood bag browser

The blood bags that meet your query criteria will be displayed in a browser.

It is possible to delete blood bags from the GLIMS database (e.g. in case they had been entered incorrectly). As long as there is no blood selection yet that is related to the blood bag, the user can delete a blood bag from the blood bag browser. This functionality can be restricted to certain users by applying a DELETE-privilege to the Blood bag table.

The column labeled Antigens shows all the antigens that are known to be positive or negative in the blood bag. They are ordered according to the antigen's sequence number, then the antigen's meaning field, and then alphabetically on the antigen's mnemonic.

The expiration date is shown in red when the blood bag has expired or in purple when the blood bag will expire later today.

Note

Note

For blood bags with status Reserved or Checked the browser will show the recipient, or all patients for whom the blood bag is currently reserved (i.e. for whom a blood selection exists). In case of multiple reservations, the number will be indicated. Example:

(2x) Cat Stevens, Bruce Springsteen

Donation type

This field is used to indicate the donation type of an existing blood bag. There are 3 choices:
  • Normal: the blood was not taken for a specific patient (i.e. neither directed to, nor autologous).
  • Directed: the blood was taken for a specific patient.
  • Autologous: the blood is that of the patient.

Directed to

This field is used to indicate the specific person for whom the blood has been taken or who owns the autologous blood bag. This can only be done for blood bags in status Verification or Initial. A warning message will be displayed and you will have to enter a reason. For tracing purposes, the modification of the donation type and/or the person is added to the blood bag's history, together with the reason that was entered by the user.
