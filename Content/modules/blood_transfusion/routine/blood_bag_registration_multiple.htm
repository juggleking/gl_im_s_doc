BTM - Multiple blood bag registration

Introduction

The multiple blood bag registration screen is available from the main menu via Start -> Transfusion management -> Process multiple bags.

This screen allow you to scan multiple bags and to then perform several actions on them. They can be moved from one depot to another, checked out, administered or returned.

General browser functionality

Blood bags can be scanned or manually entered. Please also check the chapter Unique identification of an ISBT blood bag.

The entered blood bags will automatically be pre-selected. The Clear screen button allows to remove all bags from the browser. It is also possible to use the DELETE key to remove the currently selected (and not the highlighted!) bags from the browser.

If none of the bags are selected, the highlighted record will be processed when hitting one of the available buttons. Bags that were successfully processed are automatically deselected and the bags which could not be processed remain selected (if there was a selection).

Specific functions

The buttons at the bottom will be hidden for users that do not have the privilege for the function associated with the button. The privileges that apply to each button are:

  • Move depot: BloodBag.ChangeDepartment
  • Check out: BloodBag.Checkout
  • Stop reservation: BloodBag.StopSelections
  • Return: BloodBag.BloodBagReturned
  • Administer: BloodBag.Administer

Move depot

The Move depot button will only be available if at least one of the blood bags in the browser has a status lower than Checked out. The screen allows selecting a new department to which the bag should be moved. Note that you can only select departments of which the option Specialty is set to Blood depot.

Check out

The Check out button will only be available if at least one of the blood bags in the browser has status Checked. Moreover, these blood bags must have exactly 1 blood selection with status lower than Discontinued. For information on the check-out options (Check out comment / Ward / In the past / To person), see Check-out options.

Stop reservation

It is possible that a blood bag has been reserved for a patient but -before it is checked out- it is decided that the blood bag is no longer needed by the patient. In that situation you can use this function to cancel the reservation of one or more blood bags (having status Reserved or Checked). If you select this function, a reason will have to be specified. The blood bag status will be reset to Initial. At the same time, each reservation (blood selection) for the blood bag will be discontinued and the blood bag will be available again for another patient.

Return

The Return button will only be available if at least one of the blood bags in the browser has status Checked out. The status of a processed blood bag will be reset to Initial and the blood selection will be discontinued.

Administer

The Administer button will only be available if at least one of the blood bags in the browser has status Checked out. As this blood bag registration screen allows the processing of multiple blood bags simultaneously, some options that are available in the general administration functions cannot be specified here (the responsible health care provider, transfusion reaction, advice and comment).
