BTM - Blood bag administration

Contents: Introduction - Administration

Introduction

After the transfusion, when the issuer returned the transfusion form, you can register the (checked-out) blood bag as being administered.

The blood product trigger When administered is executed when the blood bag is set to status Administered. When it returns the NO value, the administration will be cancelled.

Administration

Via object

In order to administer a blood bag, you can use the Object contextual menu function Transfusion -> Administer blood bags.

Select the blood bag and click OK.

In the screen that is displayed, you can enter the responsible HC provider and the administer date. When an administration date is specified, GLIMS checks the following:

  • It is not allowed to specify a date/time BEFORE the date/time of the creation of the blood bag.
  • When a date/time is specified that is earlier than the time the blood bag was selected or checked, a warning will be shown asking the user whether they want to continue or not.
  • When a date/time is specified that is earlier than the date/time that was specified when the blood bag was checked out, the user is asked if this date/time is correct and if GLIMS should automatically change the checkout date/time to this new date/time. This avoids that blood bags seem to have been transfused before they were checked out.

If the transfusion has been aborted, you can log this by enabling the Interrupted toggle box. The transfusion reaction and transfusion advice will be stored in the person's medical record.

Via the main menu

If the identification of the blood to be administered is known, you can use the blood bag administration program available from the main menu via Start -> Transfusion management -> Blood bags -> Administer. The following screen will be displayed:

Blood bag identification

Allows you to look for a blood bag on the basis of its internal id (CTRL+F), external id, or barcode (possibly by scanning it), its external id or its internal id. After entering this id and hitting RETURN, the cursor will automatically find the blood bag and jump to the patient identification field. Please also check the chapter Unique identification of an ISBT blood bag.

External id

Allows performing a lookup (CTRL-F) by external id.

Patient identification

3 fields are available:
  1. the PIN code
  2. the last name
  3. the first name

If the PIN code is entered and followed by RETURN, the patient is automatically selected. This way, it is sufficient to scan 2 the patient and the blood bag barcodes (each ended by a RETURN code) to have the next screen popup automatically (for entering the transfusion date etc.).

Note

Note

Only blood bags with status Checked out can be selected in this screen.

In batch

The Administer function on the BloodBag table can be used as a function in batch commands. It has no parameters. When doing so, the selected blood bags will be promoted to status Administered and the transfusion date/time will be set to the date/time at which this command is executed. This allows customers to perform automatic promotion of for example all blood bags without reactions.
