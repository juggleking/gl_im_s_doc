BTM - Blood bag check-out

Contents: Introduction - Preliminary remarks - Check-out

Introduction

When the blood bag has been called for (actually left the laboratory), it can be checked out in GLIMS. A blood bag can usually only be checked out if the associated cross match was negative and has been validated.

Preliminary remarks

Non-validated cross-match

When someone tries to check out a blood unit for which the crossmatch result either has no value yet or is not validated yet, the checkout is inhibited with a message saying "The crossmatch result is not validated!". This is for security reasons.

However some labs want to be able to postpone the crossmatch in some circumstances, e.g. when the check-out must be done very urgently. Therefore it is possible to overrule this message. To do so, enable the option Allow checkout without validated crossmatch in the BTM general settings.

From then on, GLIMS will still give a strong warning about the fact that the user is checking out the blood bag without validated crossmatch, and ask if the user is sure. The latter can answer YES to this question to overrule the check.

The warning which is given on the screen will also be appended to the checkout comment that was entered by the user (which can be found in the tab folder Check out of the blood bag).

Re-evaluation of compatibility

Whenever a blood bag is checked out, GLIMS will verify if the blood bag is still compatible. If the blood bag is no longer suitable, the user will be warned with a message and asked whether the bag has already physically left the lab. If not, then he is strongly advised to take another bag. If the user ignores this advice, this will be logged as checkout comment with the bag. If he accepts the advice, the blood selection is automatically discontinue-repeated and another bag will have to be selected.

Which actions are or can be taken when checking out a blood bag

The following triggers defined in the blood product will be executed:

Before select, When selected, Utmost transfusion time

Check-out

Via object

In order to check out a blood bag, you can use the contextual menu function Transfusion -> Checkout blood bags on an Object. The list of blood bags that have been reserved for this patient will be displayed.

Check out options

Check out comment

Optionally, you can enter extra check-out comments.

Ward

Optionally, you can enter the ward to which the blood bag is checked out. The default ward displayed is the ward where the patient currently resides. When checking out multiple blood bags, and if the default ward is the same for all those blood bags, it will also be shown as default ward in the dialog screen. If 2 (or more) different wards are found as default, the dialog will show ?.

In the past

It is also possible to fill in a date and/or time in the past in order to specify the correct date/time after the blood bag was checked out. 

To person

Allows to fill in the hospital employee who picked up the blood bag. Double clicking allows you to select a Hospital employee from the list of hospital employees configured in GLIMS. The specified hospital employee will be saved in the To person field of the Blood bag editor.

Note

Note

Filling in the To person field during check-out can be made mandatory by enabling the To person required during checkout field in the BTM general settings.

In that case, a warning will be displayed when the To person field is not filled in during check-out.

Via the main menu

If the identification of the blood bag to be checked out is known, you can use the blood bag check-out program available from the main menu via Start -> Transfusion management -> Check out.

Note

An option Blood bag and person id required during checkout is available in the BTM general settings.  If enabled, both the blood bag identification and patient identification MUST be entered or scanned in this blood bag checkout screen. It is disabled by default.

Blood bag identification

You can scan or type the blood bag identification. Please also check the chapter "Unique identification of an ISBT blood bag".

Person

Whether the PIN or the encounter number of the patient is used, depends on the setting Patient id type in the BTM general settings. The default value is PIN.

  1. If you enter or scan the patient identification, GLIMS will check if the blood bag and patient match. In case of a mismatch, you will not be able to checkout the blood bag.

  2. When the BTM general setting Blood bag and person id required during checkout is disabled and the blood bag id is the only field entered in the checkout screen, a browser is shown displaying all eligible patients for this blood bag. The browser allows to select the patient.

    For information on the options Check out comment, Ward, In the past and To person, see Check out options above.

  3. When a patient is selected and no blood bag is specified, a browser appears with Checked blood bags for this patient.

    For information on the options Check out comment, Ward, In the past and To person, see Check out options above.

Transform barcode

Allows to transform the scanned patient PIN / encounter ID.

The option Patient id transformer in the BTM general settings allows manipulating the scanned barcode before using it in GLIMS to look up the matching PIN or encounter. The MISPL expression is based on the SpecificSite table and should return a string which will be used to lookup the PIN or encounter.

You can use the MISPL functions GetEncounterBarcode() or GetPINBarcode().

Example

Example

MISPL function (removing the first two characters):

string Barcode;
BarCode := GetEncounterBarcode();
RETURN Substr(BarCode, 3,-1);
