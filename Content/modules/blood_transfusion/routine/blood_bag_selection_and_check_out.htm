BTM - Blood bag selection and check-out

Introduction

GLIMS features separate functions to select compatible blood bags for a patient and to check-out these blood bags. However GLIMS also offers a way of combining these two functions in one single step: i.e. simultaneous selection and check-out. Check the documentation below to see how you can accomplish this.

Usage

The contextual menu of Blood selections offers 2 functions which allow to select and check-out blood bags simultaneously:

  • Select blood bag manually + Check out
  • Select compatible bags + Check out

These functions do exactly the same as the existing counterparts Select blood bag manually and Select compatible bags respectively. But in the same action they will also look if they can check out the blood bags that have been selected.

So first for each blood selection a bag is selected, and then the screen for checking out automatically appears.

For information on the options Comment, Ward, In the past and To person, see Check out options. This screen appears only once for all selected blood bags together. So when these functions are used, the selection of the blood bag will happen in the same way as with the regular selection functions, but at the end, once the blood bag selection is finished, GLIMS will also try to check out the blood bags.

Checking out is of course only possible when the blood selection (and blood bag) have reached status Checked, so this is only possible when either of following is the case:

  • the blood product is marked as Crossmatch not applicable (so no crossmatch result will be created for it)
  • the blood product is not marked as auto-confirm, but the crossmatch result automatically receives a value indicating that the blood bag is compatible, so the status is automatically and immediately promoted to Checked. This is the usual scenario when 'Type & Screen' is applicable.

If immediate checkout is requested and none of the blood selections has reached status Checked, the checkout screen is not shown. This allows using these new functions as a complete replacement of the existing functions: for products that need cross matching, they will behave in the same way as the single-step functions.
