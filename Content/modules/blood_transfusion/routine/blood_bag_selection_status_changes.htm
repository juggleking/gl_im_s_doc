BTM - Other status transitions

GLIMS offers a number of functions to modify the status of a blood bag and/or blood selection.

Functions on Blood bag

From Blood bag browsers, you can select the contextual menu/ribbon item Status transition and then one of several statuses and status changes:

Verification passed

This function enables you to promote the status of a blood bag to Initial once this blood bag has been verified.

Stop selection(s)

It is possible that a blood bag has been reserved for a patient but -before it is checked out- it is decided that the blood bag is no longer needed by the patient. In that situation you can use this function to cancel the reservation of one or ore blood bags (having status Reserved or Checked). If you select this function, a reason will have to be specified. The blood bag status will be reset to Initial. At the same time, each 'reservation' (blood selection) for the blood bag will be discontinued and the blood bag will be available again for another patient.

Check out

Please check the chapter Blood bag check-out.

Cancel checkout

This function will demote the status of the selected blood bag(s) from Checked out to Checked. The related blood selection(s) will automatically be demoted from status Ready to Checked.

Return to 'Initial'

This function will demote the status of the selected blood bag(s) from Checked out to Initial. The related blood selection(s) will automatically be discontinued. The blood product trigger When returned will be executed.

Return to 'Verification'

This function will demote the blood bag's status from Checked out to Verification. The related blood selection(s) will automatically be discontinued. The blood product trigger When returned will be executed.

Administer

Please check the chapter Blood bag administration.

Discontinue

This function will set the status of the selected blood bags to Discontinued. When clicking on this function, a confirmation window opens:

  • The default value for Context type is Other which requires a discontinuation reason. For all other types, the reason is optional. If not specified, the selected type is stored as reason in the Discontinued tab of the blood bag editor.

  • It is not possible to discontinue a blood bag with status Checked out.

  • When a blood bag with status lower than Checked out is discontinued, the corresponding blood selection is discontinued and repeated (a new blood selection with status Initial will be created).
  • The blood product trigger When discontinued will be executed.

Expired

This function will set the blood bag's status to Expired. The related blood selection(s) will be discontinued and repeated (a new blood selection with status Initial will be created).

The blood product trigger When expired will be executed.

Wasted

Allows indicating that a blood bag can no longer be used (e.g. the bag was torn). When clicking on this function, the same confirmation window opens as when choosing the status transition Discontinue described above.
If the blood bag was already assigned to a blood selection, the latter will be discontinued and repeated so that a new blood bag can be selected.

Wasted to 'Initial' - Discontinued to 'Initial'

Allow setting one or more selected blood bags in status Wasted or Discontinued back to status Initial.
The fields of the Discontinued tab in the blood bag editor are cleared. If the selected blood bags had been checked out or returned, the fields of the Checked out or Returned tabs are also cleared.

Note

It is advised to reserve this functionality to specific users, using privileges.

Undo administer/expiry/discontinue

GLIMS also offers a function to undo the administration, expiry or discontinuation of a blood bag. It can be accessed via the main menu under Transfusion management -> Undo transfusion. In the window that opens, choose the action you want to execute: Undo administer, Undo expiry or Undo discontinue. The internal ID of the blood bag and a reason must be specified.

Warning

  • This function is not meant for normal routine but for exceptional error cases. We therefore advise to protect this function with a privilege.
  • It is not possible to reset a Wasted, Expired or Discontinued blood bag to Initial by using the Undo transfusion functionality.

This function can be used, for instance:

  • when a blood bag was ERRONEOUSLY put in the status Administered (e.g. a user has registered the administration of 4 blood bags for the same patient, while actually only 3 were administered but 1 was returned to the lab). When the blood bag was administered more than 2 months ago, a confirmation will be asked.

  • when a blood bag has accidentally been marked as Expired (e.g. a blood bag was checked out just before its expiry, but it had not been immediately checked out in GLIMS. The day after it was marked as expired...). The function can be used to put it back into status Initial.

  • when a blood bag was ERRONEOUSLY discontinued. After an Undo discontinue, the blood bag status will be reset to Initial. Note that, just like for Undo expiry, this is not a complete 'undo': blood selections that have been discontinued as a consequence of the blood bag discontinuation will remain discontinued.

When using the Undo discontinue function for a check-out blood bag, all data related to the previous check-out of the blood bag is cleared: recipient, check-out time, check-out ward, check-out employee, check-out user, transfusion times, transfusion responsible.

Log types for unusual blood bag work flow

Each time the status of a blood bag is demoted, a log entry is added to a specific log type. These log entries can be accessed via:

  • the main menu: Start > System management > Logging,
  • the contextual ribbon item Audit > Log entries on a blood bag.

Functions on Blood selection

Discontinue

This function will set the blood selection status to Discontinued. The related blood bag(s) will be demoted to status Initial except if a blood bag has also been reserved for another patient.

Repeat

This function will set the blood selection status to Discontinued and a new blood selection with status Initial will be created. The related blood bag(s) will be demoted to status Initial except if a blood bag has also been reserved for another patient.

Discontinue in batch

This function allows the discontinuation (and possibly repetition) of a blood selection using a batch command. This function has 2 parameters that can be configured in the function parameter set:
  • Reason: the textual reason why the blood selection should be discontinued or repeated.
  • Also repeat: if this toggle is enabled, the blood selections will also be "repeated".

This allows you to automatically stop reservations of blood bags that are waiting too long before being checked out. For instance, some labs want to automatically discontinue blood selections for which the utmost transfusion date/time has passed.
