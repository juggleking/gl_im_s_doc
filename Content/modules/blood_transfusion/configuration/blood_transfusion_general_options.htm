BTM - General settings

The blood transfusion module can be tailored to the needs of a site using general settings available under Start -> System management -> Customize -> GLIMS transfusion module. These general settings are presented below.

Transfusion tab pages

General fields

Bloodbag internal / external id

These fields reference a BloodBag-based character site function used to produce the blood bag internal id and external id. When using the ISBT 128 standard, which is fully supported by GLIMS, you do not need to specify an external id MISPL. GLIMS will automatically recognize the identifications. The external id MISPL is used when the incoming number is e.g. 'D1234D'. The MISPL can then be built as such that the identification is stripped to '1234'. The external id expression is evaluated when scanning blood bags or when importing blood bags through the diskette import program.

Checks

Always scan the product code

When this option is enabled, the following applies: Wherever in GLIMS you enter the barcode of a blood bag, a window will show up, asking you to scan the product barcode of the blood bag as well. Here you should scan the product barcode (or enter it by hand). This applies for the following programs:

  • Multiple blood bag registration
  • Blood bag check-out via the main menu
  • Blood bag administration via the main menu
  • Blood bag disk import programs

Use extended antigen list

Used when registering blood bags in GLIMS.

Blood bag antigen check

Allows to specify a site function that will be executed whenever an antigen is added to a blood bag. Can e.g. be used to avoid that users enter impossible combinations of blood bag antigens and person antigens/antibodies.

Person antigen check

Allows to specify a site function that will be executed whenever an antigen is added to a person. This can for instance be used to avoid that users enter impossible combinations of blood bag antigens and person antigens/antibodies.

Selection

Double cross allowed

If enabled, a blood bag can be reserved for two different selections. Once this blood bag is actually checked-out, the other selection will be discontinued and repeated.

Distinct departments

If enabled, only blood bags from the department of the order will be available during blood bag selection. If you set this field to ?, GLIMS will still allow you to select blood bags from other departments (similar as setting it to No), but the list of compatible blood bags will first show all compatible blood bags of the own department, and then beneath that all compatible blood bags of other departments.

Reporting

Report property

See the chapter Blood product report.

Report expression

The report expression is a BloodSelection- based MISPL expression which determines the text to be reported by the report property. This MISPL expression is re-evaluated on each blood selection status change.

Validate report result at checkout

If this option is activated, the report result for the blood selection will be confirmed and validated at the time when the blood bag is checked out.
If this option is not activated, the report result will remain 'available' after the checkout and it will only be confirmed and validated after the transfusion has been registered (blood bag administered).
The reported string is defined by the report MISPL function.
Note

Note

When using electronic reports to send transfusion data from GLIMS to an external system, it may be important that each status change of the transfusion unit is reported. In that case, do not enable this option.

See also Blood product report.

Administer

Copy order issuer (or agent) to responsible

If this option is set, then the following applies: when a blood bag is promoted to Administered and no transfusion responsible has been specified yet, the order's issuer (if it is a HC provider) is automatically considered as transfusion responsible HC provider for the blood bag. If the order's issuer is no HC provider, the order's agent (if it is a HC provider) is taken as responsible HC provider.

Always need transf. responsible

If this option is set, then no blood bag can be promoted to Administered without having a responsible HC provider.
Note

Note

CyberTrack always requires a transfusion responsible. Whether this option is set or not plays thus no role in CyberTrack.

Persons

Strip leading zeroes when PIN entered

When a PIN code contains leading zeros ("000...."), these will be removed before the PIN is looked up in the database. This allows specifying the PIN with or without leading zeroes. This is used in all screens where a patient PIN code can be entered (e.g. blood bag check-out, administration, etc).

Order entry

Action after order entry

Allows specifying a MISPL expression that is to be executed after an order has been entered. Currently this field only provides support for a MISPL expression that returns the string "BrowseBloodSelections" or "BrowseSpecimens".

If so, the blood selection browser or specimen browser will automatically be displayed after creating or updating an order. If you want to check if any blood selections are available, you should do so in the MISPL expression itself.

Example

Example 

IF LOOKUP("Ery", .Attribute("RequestList"), ",") > 0
THEN
  RETURN "BrowseBloodSelections";
ENDIF;
