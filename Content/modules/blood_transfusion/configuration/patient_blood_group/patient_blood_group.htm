BTM - Patient blood characteristics

Contents: Introduction - Blood characteristics - Person medical record - Patient data sheet

Introduction

The first step in blood transfusion management is the determination of a patient's blood group, rhesus and antigen/antibody presence. In GLIMS this is implemented through 'normal' procedures which do not relate to the actual Blood Transfusion Module. This document merely gives some examples of how this could be configured in GLIMS. You can implement these procedures according to your own needs.

Blood characteristics

A patient's blood characteristics are measured by determining:

  • Blood group
  • Rhesus-D
  • Rhesus phenotype C,c,E,e
  • Other antigens such as Lewis, Kell, Duffy, Luther, etc..

These characteristics can be measured by checking antigen presence (e.g. on EDTA) and antibody presence (e.g. on SERUM).

GLIMS allows you to

  • Calculate or derive results from sub-tests
  • Execute consistency checks between different tests (EDTA/Serum), the same test done by different people (e.g. ABO), current and previous results, etc.
  • Reuse previous test results (e.g. final ABO)
  • Validate test results with property choices
  • Schedule and execute only necessary steps on a conditional basis (e.g. patient age)
  • ....

Screening test

The image below outlines a possible procedure structure for a screening test.

The request definition which triggers this procedure is IrrAntiScr. The last procedure is a calculation which has three properties as input and the final screening result as output. This calculation can be configured using a MISPL expression in the Value field of the procedure output (e.g. IrrAntiScr):

Example

Example

MISPL: 

/* Name : panelstoscreen.prco */
/* What : As soon as at least one of the screeningpanels */
/* SI, SII, SIII is positive, the screening is positive. */
/* A negative result can only occur when each panel is negative */
/* Basetable: action */
RETURN( if SI = "+" or SI = "+" or SII = "+" or SIII = "+"
then "pos"
else "neg");

Storing the blood characteristics in the person medical record

The final blood characteristics can be automatically stored in the patient's medical record. The automatic storage is configured by means of a property trigger. In order to do so, you can use the MISPL function "Person.SetMedicalRecord()". For an example, see the BGRhesus property trigger as defined in the example above.

The field determination status in the person medical record can also be changed from within MISPL functions by using the MISPL function 'SetMedicalRecord' on 'Person'. The specified value must be either "1", "2" or "3" (including quotes) or the unknown value.

Example

Example

.Person.SetMedicalRecord( "DeterminationStatus", "2" );

Note

Note

The meaning of the values 1/2/3 is defined in the site-domain "ABODeterminationStatus".

The person medical record allows to store the

  • Blood group, rhesus, pheno information
  • Presence or absence of antigens and antibodies
  • Screening info
  • Transfusion reactions
  • Transfusion advices

The person medical record can be consulted via a contextual menu function on Object (Transfusion -> Person medical record).

Tip

Tip

The Person.SetMedicalRecord() MISPL function also allows to specify unknown (?) as value for the blood group or the rhesus. This way the blood group or rhesus can be cleared again by means of a MISPL function.
