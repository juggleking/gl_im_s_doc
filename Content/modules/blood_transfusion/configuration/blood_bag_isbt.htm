BTM - Unique identification of ISBT blood bags

Contents: Introduction - GLIMS solution

Introduction

ISBT blood bags are not uniquely defined by their donation identification number (the 'external id'), since more than 1 bag can be derived form the same donation. However these similar blood bags will have different product codes. So ISBT blood bags are always uniquely identified by their identification number + their product code.

GLIMS solution

For blood banks that prefer to do so, GLIMS offers the following way of working: when scanning an ISBT blood bag number, always immediately scan the blood bag's product barcode as well. This is a consistent and safe way of working, and implies not so much extra work when done with a barcode scanner.

To activate this feature, you must enable the option Always scan the product code in the BTM general settings. When this option is enabled, the following applies:

Wherever in GLIMS you enter the ISBT barcode of a blood bag, a window will show up, asking you to scan the product barcode of the blood bag as well. Here you should scan the product barcode (or enter it by hand). This applies for the following programs:

  • Multiple blood bag registration
  • Blood bag check-out via the main menu
  • Blood bag administration via the main menu
  • Blood bag disk import programs

For this purpose the product code of each blood bag that enters GLIMS is also stored. GLIMS will allow to select a blood bag with an unknown product code, to allow for a migration period. This is because existing blood bags that have been entered before this version don't have a known product code.
