Automatic detection

When GLIMS does the automatic detection of isolation doubles, GLIMS will group identical isolations (of the same patient) together in a so-called 'Isolation series'. At the same time GLIMS will also look if the isolation series can be grouped with one or more other isolation series (of other patients). If this is the case, a 'Hospital infection' will be created, grouping all these related isolation series together.

All information regarding possibly related infections is stored in 2 fields:

  1. 'From' : contains the list of other infections that could have transmitted the germ + in which ward; 
    it also contains the list of other identical infections but for which no common ward was found (patient was not in same ward during transmission period).
      
  2. 'Toward' : contains same information as 'From', but reversed: which other infections have been caused by this infection ?

The conditions that must be fulfilled to link 2 isolation series together, are (all parameters can be set in 'System management -> Customize -> GLIMS microbiology'):

  1. The microorganism must be the same or 'similar'. The detection of hospital infections - which by default searches for identical organisms for a certain patient, can be defined to consider all organisms within a certain taxonomic group. The level of this group can be defined with the option "Slightly different microorganisms may be identical" in the general 'microbiology settings'.
     
  2. Cross infections are only created if the first isolation of the source infection is validated.
     
  3. GLIMS also compares the discriminator of the isolations and if these are different, the isolations will be considered as different and no cross infection will be created between these isolation series.
     
  4. The resistance patterns of the detected isolations of both series must be 'similar'. The 'Maximum difference ' is used.   The antibiograms of 2 consecutive isolations match with a maximal difference of 'x'.  The antibiograms of the first isolation and last isolation of the isolation series match with a maximal difference of 'y'

    where 'x' and 'y' are decimal numbers that can be configured in 'System management -> Customize -> GLIMS microbiology'.

    5. For more information about antibiogram differences, please check the chapter "Isolation doubles".

  5. Time/place matching:

    For the detection of cross infections, isolation series are considered to continue till the end of the patient's encounter.  When 2 patients have stayed in the same ward, and for one of these patients (patient A) an isolation was detected *before* both patients were in the same ward, it can still be detected as a cross infection (from patient A to B), even *after* the last isolation was detected, as long as that isolation was detected during the same encounter of that patient (A). The reason is that we assume that after an isolation has been detected, it does not just disappear immediately after. The micro-organism is assumed to reside in the patient until the end of the encounter.

    6. The cross infection detection mechanism does not use the option "Maximum number of days between 2 consecutive isolations of a series". Instead, the mechanism goes back as far as the beginning of the patients' admission to the hospital, to find possible causing cross infections (contamination sources). 

    If several hospitalized encounters are valid at the same time for the same patient, only the most recent one is considered as valid (the most recently started, or -if the start time is the same- the most recently created one). The others are ignored.

    When several stays within an encounter have same the start time, only the most recently created one is considered as valid.

  6. For isolation series (infections), the field 'Hospital acquired', is automatically set to Yes or No according to the parameter 'Maximum number of days between admission and hospital acquired' in the microbiology specific settings and/or in the microorganism 'Epidemiology' settings.  The field is set by the function 'Determine infection'.

    Example: if the "Maximum number of days between admission and hospital acquired" is set to "3" and a patient enters on 01/01/2008 then an isolation detected on 02/01/2008 will NOT be considered as "hospital acquired".

    Only isolation series of which the option "hospital acquired" is enabled are considered during the detection of hospital infections.

The following detection parameters can also be specified per micro-organism or micro-organism family:

  • Maximum antibiogram difference between 2 consecutive isolations 
  • Maximum total antibiogram difference between first and last isolation of a series
  • Maximum number of days between admission and hospital acquired

This means that each parameter that is specified for a certain microorganism is automatically inherited by all descendant microorganisms. 

Detection of cross infections caused by ambulatory patients

If an isolation was detected for patient A while he was not hospitalized (so during an ambulatory encounter), and afterwards he is hospitalized and stays in the same ward as patient B (for which the same micro-organism is also detected), then Glims is able to detect this as a cross infection.

This is not the default behaviour however.  To obtain this, the following configuration is required:

  1. In the general microbiology options (Start -> Microbiology -> Configuration -> Settings):  a field named "Infection lag before encounter" is available. With this option, you can specify how many days Glims should look back in time (before the start of the hospitalized encounter), while searching for isolations of patient A *before* patient A was hospitalized. (default period = 0 days)
     
  2. In the micro-organism configuration screen: the same option is available, so that it is possible to override the general value and vary the value of this period according to the micro-organism or family. This option is also inherited by all micro-organisms below it (in the tree).

Automatic warning

GLIMS provides the possibility of being informed automatically of the fact that a hospital infection has been detected, or that a known hospital infection expands. For this you can set a 'Infection trigger' in 'System management -> Settings -> GLIMS microbiology'.

This trigger is executed:

  1. When a hospital infection is detected the first time.
  2. Whenever a new isolation series (for some other patient) is added to an existing hospital infection.

Cross infections

The function 'Determine infections' also creates a 'Cross-infection' record as a link between different infections (Isolation series) of different patients that were in the same ward in the same period. They can be viewed from an 'Isolation series' with the contextual menu functions 'Possibly caused by' and 'Possibly transmitted toward'.

Glims also offers a trigger that runs each time a cross infection is created.  This cross infection trigger can be specified in the general microbiology options (Start -> System management -> Customize -> Glims microbiology).  The defined MISPL expression must have "CrossInfection" as base table.  The main advantage that the cross infection trigger has over the "Infection trigger" is that more information is available: from patient, to patient, via ward, via room etc. 

Cross infection requires antibiogram

It is possible to define that GLIMS should only create cross infections when both the source and target infection have an antibiogram.  This can be specified on two levels:

General setting

In the general settings (Start -> System management -> Customize -> GLIMS microbiology), the option "Antibiogram required for cross infections" allows to indicate that GLIMS should not create cross infections between 2 infections if both do not have reportable antibiotic results.

Per micro-organism

If for certain micro-organism (families), an exception should be made to the general setting, the micro-organism option "Antibiogram required for cross infections" can be used.  The setting of this option will be inherited by all underlying micro-organisms.

Possible values:

YES

For this microorganism (and all underlying organisms) GLIMS will only create cross infections when antibiograms are available for source and target infection.

NO

For this microorganism (and all underlying organisms) GLIMS will create cross infections even when no antibiograms are available for source and target.

?

When left unknown, the value is inherited from the parent microorganism family to which it belongs.  When no parent is available,  the general setting is applied.

Viewing

  1. From an isolation, you can see its isolation series and its hospital infection with the 'Infection status' menu option. You can also navigate to the isolation series and from there on to the hospital infection.
  1. From the main menu of microbiology, there is an option to browse all hospital infections. From a hospital infection, you can

    1. ask the list of contained isolation series; for each series you can ask the list of isolations. 

    2. ask an outline overview, which shows the isolation series, their isolations, etc....

Note: it is possible to show only infections, cross infections and hospital infections in browsers if they have a minimum infection severity.  This minimum severity can be specified in the general epidemiology options (Start -> System management -> Customize -> Glims microbiology -> tab page "Epidemiology"), in the field "Only severity >=".

Chronological overview

From a hospital infection, the contextual menu function 'Chronological overview' is available. It shows in a horizontal time line:

  • in RED bars: all contained isolation series, each with a sequence number, the detection date, the patient and the original ward;
     
  • in YELLOW bars: for each isolation series: the cross-infections that might have transmitted the organism onto another patient; showing: the common ward where transmitted, the probable date/period of transmission, and the sequence number of the other isolation series (of the other patient)

This overview can be printed as a HTML page by right-clicking in the window.
