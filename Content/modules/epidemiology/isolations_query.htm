Isolation query

The isolation query is available from the main menu via Start -> Microbiology -> Isolation query. 

General fields

Sampling date from

Use a fixed date or a relative date indication (e.g. "Yesterday")

Doubles

Tip

  • Patient double = an isolation that has a predecessor (detected in the same patient).
  • Material double = an isolation that has a predecessor for the same material (an isolation that has an identical predecessor that was detected in the same material).
  • Selection double = an isolation that has a predecessor for the same selection criteria (an isolation that has an identical predecessor that also fulfills the applied selection criteria).

Don't exclude doubles

No filtering is performed.

Exclude doubles ...

within patient

Only new isolations (unprecedented - without any duplicates). This removes all isolations that are 'double of' another isolation.

within material

Only new isolations for the material in which they were found.

within chosen selection criteria

One single result per isolation series (no identical isolations within selection). Of all isolations of the same isolation series, only the first isolation that fulfills the selection criteria is selected.

Only doubles ...

within patient

Opposite of Exclude doubles within patient.

within material

Opposite of Exclude doubles within material.

within chosen selection criteria

Opposite of Exclude doubles within chosen selection criteria.

Hospital acquired

Select only or exclude the isolations of which the series is (assumed to be) 'hospital acquired'.

Status

All (excl. discontinue)

Select all isolations irrespective of their status, with the exception of discontinued isolations.

Confirmed and validated

Select only confirmed and validated isolations.

Only validated

Select only validated isolations.

Modified with x days

If specified, only isolations that have had changes in the (x) passed days will be selected. These changes can be changes on "isolation", "isolation test" or "Antibiotic result".

Example

When specifying 1.5, only isolations which have had changes in the last day and a half will be selected.  A value of 1.25 means a day and a quarter (i.e. 30 hours).

Study type, Study

Select only isolations linked to a specific study type or study.

Resistance pattern

The resistance pattern is also known as antibiotype (e.g. "AMP = R ; CEFA = S").

It offers a selection on multiple antibiotics, each one with a certain R/I/S selection. This way it is e.g. possible to select all isolations resistant to antibiotic A, sensitive to antibiotic B and resistant or intermediate to antibiotic C.

  1. Select an antibiotic from the list.
  2. Press Enter or use the arrow (<<) button to add the selected antibiotic.
  3. To modify the R/I/S selection of an antibiotic, select the line in the resistance pattern and change the check boxes for R/I/S/U/D/V/E/-. The options below allow indicating if this is the measured or reported value.
  4. To delete an antibiotic, select the line in the resistance pattern and hit the DELETE key.

Tip

When a specific RIS value is not enabled, this is visualized with a "-" sign in the resistance pattern.

Test pattern

Adding isolation tests

You can select one or more isolation tests. 

Tip

When selecting multiple isolation tests, GLIMS will only select the isolations that have ALL of these isolation tests.  So when adding the isolations tests ALPAgar and Katalyse, GLIMS will only select isolations on which both tests have been performed.

Adding isolation tests with a specific value

In addition to simply searching for isolations with a specific isolation test, GLIMS also allows to search for isolations with an isolation test with a specific value.  

You can even specify multiple values and in that case GLIMS will search for isolations with an isolation test with one of these values.  So when adding an isolation test with value "P" and "N", GLIMS will lookup all isolations with the specified isolation test that has either "P" OR "N" as value.

To specify multiple isolation test values, use the "|" character as separator (e.g. P|N ).  GLIMS also supports the use of wildcards (*).

Notes

  • Do not enter a space before and after the separator (|), because the exact value is used. If you specify N |P, then GLIMS will use "N " (N + space) and not "N" as search criterion.
  • You can specify ? as test value which will return all isolations with that specific isolation test for which no value is available yet.

Patient

Residence type

Select only (or exclude) hospitalized / ambulatory patients (at the date of sampling).

Tip

  • When selecting H, you must select a ward, else the query will be empty.  
  • To include all wards, specify "*" in the Ward field.

    • In ward(s)

    Select only patients that resided in specific ward(s) (at the date of sampling).  You have the choice between:

    • Entering the mnemonic of 1 single ward

    • Entering a wildcard specification for the ward mnemonic: this is a name containing one or more '*' characters.

      Example

      • IC* = all wards where mnemonic starts with 'IC'
      • *01 = all wards where mnemonic ends with '01'

    • A comma-separated list of multiple entries

      Example

      • 001,IC*,Urg* = ward '001' + all wards where name starts with 'IC' or 'URG'

    It is possible to lookup a ward in the list by double-clicking in the field or hitting CTRL-F. The ward(s) selected in the lookup browser will simply be added to the existing list.  

    Storage criteria

    This section contains all options of the archive storage query that are applicable to isolations.

    Isolation browser

    The selected isolation will be displayed with their antibiogram in the isolation browser.

    It is impossible to delete isolations in the isolation query browser. Also inserting, copying or updating isolations is not possible in this browser. The microbiology screen is the only place where such things can and should be done.

    The following information columns are shown in the browser:

    • Specimen internal id
    • Sampling date 
    • Patient name
    • Admission date
    • Ward
    • Material + primary material variable
    • Microorganism
    • Appraisal
    • All antibiogram results (1 column per antibiotic): the toolbox button at the bottom of the screen offers an option to show either the antibiotic names or mnemonics as column headers.
    • Column 'D' shows whether the isolation is a 'double' of another one;
    • Column 'H' shows whether the isolation (or rather: its isolation series) is (assumed to be) 'hospital acquired'.

    Similar to the isolation browser embedded in the microbiology screen, the isolation status is reflected by a background colour.  The background colours are defined through the color scheme (Start -> System management -> Customize -> Colors).

    Other features

    Antibiotic column

    You can choose which antibiotic columns have to be displayed/hidden.  A button labeled Antibiotics... is available in the isolation browser.

    Note

    The settings are stored as user preferences and per view (separate settings for 'isolations of object' browser).  By default, the user will see all the antibiotics.  After changing your settings, the browser will automatically be refreshed.

    Isolation test column

    You can choose which isolation test columns have to be displayed/hidden.  A button labeled Tests... is available in the isolation browser.

    Column sorting

    Clicking on a column header sorts the records according to the column. Clicking a second time on the same column sorts the records in the other direction. Initial sorting is set to specimen internal id.  

    It is also possible to use positioning on the sort column (except for fields with a data type different from data type string). 

    Automation

    The standard isolation query which can be used in a command uses the same selection screen, so all these selection options are available there too, e.g. to make daily reports about detected germs. 

    Isolation statistics

    The isolation statistics use the same selection screen, so all these selection options are available there too.

    Isolation history

    The same browser (with all its selection options) is used for the isolation history of an object / patient (via Object -> Microbiology -> Isolations).

    Open microbiology work screen

    Double click an isolation to open the microbiology work screen.

    Note

    Use F6 in the isolation browser to open the isolation detail screen.

    Export to MS Excel/MS Word

    I do not have the 'Office interface' license 

    If you do not have the 'Office interface' license, you can still use a basic version of the Excel export tool.  If you hit CTRL-W or use the Excel button at the bottom of the screen immediately after the isolation browser has been displayed, its content will be copied into an Excel sheet.

    I have the 'Office interface' license

    If you have the 'Office interface' license, you can use the MS Excel and MS Word buttons at the bottom of the screen.  These export tools offer more advanced features than the non-licensed Excel export tool.
