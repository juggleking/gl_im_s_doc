Epidemiology - Officially reporting isolations

Introduction

In some countries, labs are required to send a report to a health office when specific microorganisms are found.

GLIMS allows to mark isolations as "To be reported officially" and generate a document for these isolations.

Marking an isolation as "to be reported officially"

When marking an isolation as "To be reported officially", the Isolation field Report officially will be set to Yes.

There are different ways to accomplish this:

Using the Microorganism field "Must be reported officially"

When the Microorganism option Must be reported officially is enabled, Isolations registered for this microorganism will automatically be marked as "To be reported officially".

Using the interactive function "ReportOfficially"

An Isolation-based function ReportOfficially is available that can be added to the Isolation-based context menu or ribbon.

It allows to mark one or several isolations as "To be reported officially".

Note

By default, the function is not visible in the application.

You must manually define a Tool (Base table: Isolation, Function mnemonic: ReportOfficially) and add this tool to the context menu or ribbon.

Using the MISPL function "Isolation.SetReportOfficially"

An Isolation-based MISPL function SetReportOfficially is available.

This MISPL function can e.g. be used in

  • The Microorganism field When isolation confirmed.
  • The Microorganism field When antibiogram confirmed.

This MISPL function can overrule the Microorganism option Must be reported officially.

Generating a document for isolation that should be officially reported

Query

The isolation query allows to search all isolations that should be reported officially by enabling the option Only officially reportable.

Generate document

The Isolation-based function Generate document allows to generate a document for isolations that should be reported officially.

Notes

This function can be used in a command.

Generate isolation document screen

Target type

Indicates who should receive the generated document. This option is also used to group the isolations that should be reported in one document.

Issuer

Issuer of the order.

Health office

Health office specified in the Municipality of the issuer of the order.

Mark as reported

If set, the option Report officially of the related isolations will be set to Done.

Header, Body, Footer

Allow to specify WorkIsolation-based text modules that define the content and layout of the generated document.

If a Header text module is specified that starts with '<HTML', the generated document will be a '.HTM' file and will be treated as such (previewed and printed using Internet Explorer).

File specification screen

Append

When one document is generated (one isolation is selected),

and Append is set,

  • and there is a pre-existing document of the same name, the new document will append itself to the existing document.
  • and there is no document of the same name, a new file is created.

When Append is not set,

  • and there is a pre-existing document of the same name, the new document will overwrite the existing document.
  • and there is no document of the same name, a new file is created.

When multiple documents are generated (multiple isolations are selected),

and the Append option is set,

  • and there is a pre-existing document of the same name, the new documents are appended to the existing document.
  • and there is no document of the same name, the documents are appended together in one single file.

and the Append option is not set,

  • and there is a pre-existing document of the same name, the new documents will overwrite the existing document.
  • and there is no document of the same name, individual files are created for each document.

Logging

It might be useful to see if and when a document was printed and by whom.  When a document has been sent to a printer using this function:

  1. the audit trail of the isolation will mention the fact that a document was sent to a printer, and for which health office or which issuer the document was generated.  

  2. a generic log entry is attached to the Isolation record. Log entries of a record can be viewed by the contextual tool View -> Log entries.

    The log entry mentions the printer name and the health office or issuer to whom the document was addressed.

    Note

    On this log entry you can also do a View -> Audit to see the user who did the printing.
