Lab archive - Configuration

Introduction

Before specimens, isolations and carriers can be stored in a lab archive, the physical structure of the lab archive must be defined in GLIMS.  GLIMS allows to define multiple lab archives, each with an unlimited number of sublevels.

Archives

First the main lab archive(s) must be defined.  A lab archive could e.g. correspond with a physical storage room.

Click Start -> Archive -> Configuration -> Lab archives.

Main fields and characteristics

Mnemonic

Unique identification of the lab archive.

Name

Name that is used to visualize the archive in GLIMS.

Department

Department to which the lab archive belongs.

Icon path for main part

Path to the icon that will be used to visualize the main parts of this archive in the lab archive overview.

Storage period

The Storage period is inherited from an Archive to its Archive parts and then to its Racks. As archive parts can be structured hierarchically, the storage period of an archive part is also inherited from its parent archive part.

Example

  1. Assign a Storage period to Archive A: B, C and D will inherit the same value.
  2. Assign a different Storage period to C: B will still inherit from A while D will now inherit from C.

Default rack type

Indicates the capacity of the racks (number of specimens) and their layout (rectangular or linear).

Label text

Dynamic text to define labels that can be printed for items stored in this archive.

Auto-confirm store

Defines the default value of the Auto confirm option when selecting this archive in the Define location function.

Main parts

The main parts represent the highest level of your lab archive. A main part could e.g. correspond with a refrigerator. Use the Insert key to add main parts.

Seq no

Sequence of the main part within the archive.

Mnemonic

Unique identification of the main part.

Description

Multilingual description of the main part.

Unique code

Will be used for barcode scanning in future versions. Not yet supported.

Storage period

The Storage period is inherited from an Archive to its Archive parts and then to its Racks. As archive parts can be structured hierarchically, the storage period of an archive part is also inherited from its parent archive part.

Example

  1. Assign a Storage period to Archive A: B, C and D will inherit the same value.
  2. Assign a different Storage period to C: B will still inherit from A while D will now inherit from C.

Subdivisions

Subdivisions allow to automatically create hierarchical structures for each main part. As soon as a subdivision is created, corresponding archive subparts (shelves) will be created for all main parts. Use the Insert key to add subdivisions.

Sequence number

Represents the hierarchical level of the archive sub part.  Start with "1" and increase it for each sublevel.

Name

Name that is used to identify the archive part (e.g. "shelf").

Default number of parts

Number of archive parts that will be created for each parent part.  E.g. the number of shelves (subdivision) per refrigerator (main part).

Icon path

Path to the icon that will be used to visualize the subparts of this archive in the lab archive overview.

Racks

Last used rack

Is used to allow the specimen scan program to continue with the last rack scanned for this archive.

Racks, rack types and rack usages

Introduction

Once the structure of the lab archives has been defined (as described above), the final configuration is the definition of the actual racks that will hold the specimens, carriers, isolations.

Rack types

First the rack types have to be defined. Click Start -> Archive -> Configuration -> Rack types.

General

Mnemonic

Unique identification of the rack type.

Name

Name that is used to visualize the rack type in GLIMS.

Numbering

Linear - 1D

Numbering that continues through all rows.  The layout section (see below) allows to define the numbering format ("Linear numbering").

Example:

Rectangular - 2D

Dual numbering per row and column. The layout section (see below) allows to define the format for row and position numbering ("Row/Position numbering").

Example:

Capacity

Number of items that can be stored in a rack of this type.  Automatically calculated based on Layout options Rows and Positions per row.

Use first free position

Indicates that when searching a free position in a rack, the first free position should be used, as opposed to the position following the highest taken position in the rack (see specimen archive scan program).

Layout

Rows / Positions per row

Used to specify the capacity of the rack.

Linear numbering (Numbering = "Linear - 1D")

Only available when the option Numbering is set to Linear - 1D.  Indicates how the positions in the rack are numbered.

Row numbering (Numbering = "Rectangular - 2D")

Only available when the option Numbering is set to Rectangular - 2D.  Indicates how the rows are numbered.

Position numbering (Numbering = "Rectangular - 2D")

Only available when the option Numbering is set to Rectangular - 2D.  Indicates how the positions (columns) are numbered.

Rack usages

GLIMS allows to reserve the use of a number of racks for a special purpose (e.g. a study). First you need to define the possible rack usages. Once defined, a rack can refer to a rack usage (see below).

Click Start -> Archive -> Configuration -> Rack usages.

Mnemonic

Unique identification of the rack usage.

Description

Text description for the rack usage.

Racks

Now the individual racks can be defined. Click Start -> Archive -> Racks.

Archive (part) where stored

In combination with the rack code, the archive (part) will be used to determine the correct rack in the specimen archive scan program.  This reference allows a direct connection between a rack and its archive, without the need to assign the rack to an archive. Whenever the rack is put in another archive part, this field will be synchronized. It can however also be filled manually, stating that the rack is intended for a specific archive.

Type

Refers to the rack type (see above).

Seq no

Allows to determine a sequence between racks of the same Type.

Code

This is a very important field.  It will, among other, be used to identify a rack in the specimen archive scan program.

Usage

Refers to a rack usage (see above).

This field allows to reserve the use of a number of racks for a special purpose (e.g. a study). When the user is filling a rack with a specific usage, the next rack proposed will be a rack with the same Usage field.  Examples:

  1. specimen archive scan program
  2. when storing isolations (define location)
  3. MISPL function Specimen.SetStorage

Usage sequence no

Allows to determine a sequence between racks of the same Usage.
