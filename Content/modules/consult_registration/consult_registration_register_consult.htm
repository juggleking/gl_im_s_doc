Consult registration - Registering a consult

Introduction

This topic describes how to access the consult registration screen, the general functionality of the screen and the creation of consult orders.

Accessibility of the registration screen

The work screen for registering a consult can be accessed via

Main menu

Start > Routine > Register consult

This dialog lets the user select the consult's object : Person, Animal or Lot.

Person tab

Allows to select a person based on PIN, Name or Encounter.

In these fields, the arrow down / up keys can be used to select the person. The "..." button gives access to the list of identifications or encounters or in case of the Name field access to the person search screen.

Animal tab

Allows to select an animal based on the animal's Name or Owner.

Owner

The arrow down / up keys can be used to select the owner. The "..." button gives access to a person search screen. In both cases, only persons that are animal owners will be accepted.

Name

The arrow down / up keys can be used to select the animal. The "..." button gives access to the list of all animals in GLIMS.

Note

Specifying an owner will restrict the list of selectable animals to the animals of that owner. The first animal of the specified owner will be pre-selected.

Lot tab

Allows to select a lot based on Supplier or the lot's External id or Internal id.

In these fields, the arrow down / up keys can be used. The "..." button gives access to the list of lots or in case of the Supplier field to a correspondent search screen.

Note

Specifying a supplier will restrict the list of selectable lots to the lots of that supplier.

Context menu

The Consult registration screen can be accessed from the following contexts:

  • Animal
  • Isolation
  • Lot
  • Microbiology action
  • Object
  • Order
  • Person
  • Result
  • Specimen

Results to be phoned

When phoning through results, the left-most button (with the facing faces) allows to register a consult. When clicking this button, the selected results will be phoned through and a phone log will be added. Then the consult registration screen will be opened.

General functionality of the registration screen

Fields

Consultant

Read-only field, automatically filled with the first name and last name of the consultant. The consultant is the person in the lab who has made a call or who has received a call.

Contact

The correspondent with whom the lab has conferred. Depending on the context from which the registration screen was launched, the drop-down list will be prefilled with one or more of the following values:

  • Issuer of the subject order
  • Owner of the animal
  • Patient
  • Supplier of the lot
  • Ward of the current stay of the current encounter of the subject order
  • Agent of the subject order

The "..." button gives access to a correspondent search screen.

Time

The consult's registration time. By default, the current date and time.

Type

Allows to select the consult type which defines the make-up of the result section (configurable section below the fields and info windows).

Info windows

Depending on the context from which the registration screen was launched, one or more info windows are shown at the top of the screen:

Subject info window

The leftmost info window shows information on the consult's subject, i.e. it describes what triggered the consult.

Object info window

The rightmost info window shows object information e.g. on the patient of the order, the lot's supplier, ...

Tip

The contents of the window can be customized :

  1. by means of a text module that can be specified in the general settings : Start > System Management > Customize > GLIMS general > tab Consult registration > field Info to show on object
  2. When this field is not set, the text module with mnemonic ObjectExternalization is taken into account
  3. When this text module does not exist, the Object.Externalization field will be taken into account

Preceding / subsequent consults

The registration screen shows an overview of the preceding / subsequent consults (if there are any) for the consult's object (patient, animal or lot) in a dockable window which can be accessed through the tab page on the right side of the screen. The overview includes the current consult as well of which the details are displayed in italic.

Result section

The purpose of this section is to register the results of the consult. Its make-up is configurable and depends on the chosen Consult type. See Configuration for more information.

When a value is entered for a property, a green check mark will appear in front of the field label.

Logs tab page

The consult registration module offers support for a work flow where the consults registered by HC providers in training need to be reviewed by their supervisors. The consult registration screen and the consult overview thus contain a Logs tab page that gives access to an Add log button and a browser with the added logs.

Adding a log

Use the Add log button in the Logs tab page to open the Consult log editor and add a comment or instructions.

For user

Specify the user for whom the log is intended.

Send mail

If enabled, an internal or external e-mail - depending on the Mail method of the User record - will be sent to the specified user.

Consulting a previously added log

Double-click / use F6 on a log in the Logs tab page of the consult overview to open the Consult log editor in read-only mode.

Creation of Consult order

When clicking the OK button in the registration screen, a Consult record and an Order record will be created. The Consult order can be accessed from a Consult by choosing Consult order in the context menu. It can be charged just like any other order.

The following data will be available in the Consult order :
