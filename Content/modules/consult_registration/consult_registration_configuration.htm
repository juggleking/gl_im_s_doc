Consult registration - Configuration

Introduction

Before a consult can be registered, the content of the consult registration screen needs to be configured. GLIMS allows the user to define:

  1. Property classifications
  2. Consult types
  3. Consult statuses

Property classifications

A property classification is used to define the tab pages, sections and fields that will be visualized in the result section of the registration screen.

Click here to see how the example above is visualized in the Consult registration screen.

How to define tab pages?

A property classification node with a Label, but without a reference to a Property is a title node. A 1st level title node will be shown as a tab page which can contain sections and fields.

How to define sections?

A 2nd level title node will be shown as a collapsible section which can contain fields.

How to define fields?

A property classification node that refers to a property will be shown as a labeled field for which the user can enter a result value. How this value can be specified depends on the property's configuration. Some property classification node fields are relevant as well for the definition of fields.

Property classification node

Default value

If an Order-based MISPL function of data type String is specified, the return value of the MISPL function will be used as default value for the corresponding field in the consult registration screen.

Notes

  • The MISPL function should return the mnemonic of a choice if the property classification node is linked to a property for which a choice list is specified.
  • The MISPL function is only evaluated for new consults and not for registered consults that are being edited.

Mandatory

If the Property classification node field Mandatory is set, the corresponding field in the consult registration screen will be a mandatory field with an asterisk next to its label. A warning message will be shown when trying to save the consult registration screen and the mandatory field does not have a value.

Property

Description

The field label that is shown in the registration screen will be determined as follows:

  1. Label of the property classification node (multilingual)
  2. Description of the property (multilingual)
  3. Short name of the property
  4. Mnemonic of the property

Datatype

String

Properties with datatype String will be shown as text boxes. When Number of editor lines > 1, a multiline text box will be shown.

Tips

  • Use the F6 key in the text box to open a larger text box that allows you to run spell checking on the entered text.
  • Enter result codes by using the Expansion sign for result codes specified in the GLIMS general settings, followed by the result code. This is only supported in non rich text boxes.
  • Configure text boxes as rich text boxes, if required. Click on the following links for more information on how results can be displayed in an image, graph or in the rich text format.

Numeric

Properties with datatype Numeric will be shown as a numeric field in which it is possible to use the up/down arrows.

See Reported decimals and Decimal threshold for more information on the number of decimal places shown.

The Unit is mandatory for numeric properties and will be shown between brackets next to the label. The unit's Description (or Name, if the description is not specified) will be used for visualization.

Enumerated

Properties with datatype Enumerated for which a Choice list is specified :

  • will be shown as radio buttons
    • if the number of choices is not higher than 8, and
    • if the Strict result checking option is activated.
  • will be shown as a drop down list
    • if the number of choices is higher than 8, or
    • if the Strict result checking option is not activated, or
    • if a diagnosis code system is linked to the property.

Diagnosis code system

When specified, the codes of the diagnosis code system will be selectable as a result value. This does not apply for properties with datatype Numeric.

Allow general result codes

When enabled, it will be possible to use or select a result code. The expanded version will be shown in the Consult overview screen.

Strict result checking

When enabled, it will not be possible to enter text in the corresponding field in the consult registration screen.

Rich text

This field is described in chapter Properties.

Reported decimals

Determines the number of decimal places (digits after the decimal point) for properties with datatype Numeric.

Decimal threshold

When specified, the number of decimal places (digits after the decimal point) for properties with datatype Numeric will be equal to Reported decimals + 2.

Example

When

Reported decimals = 2

Decimal threshold = 8

this will result in 4 decimal places.

Consult types

A Consult type represents a question form. You can define multiple consult types and choose one of them during consult registration.

Consult types can be accessed / created via Start > Configuration > Consult types.

Property classification

Defines the make-up (see above) of the registration screen's result section. The property classification of the consult type is also taken into account when displaying the results of the preceding / subsequent consults.

Valid from / Valid until

Users can only select Consult types that are valid at the time of consult registration. This allows e.g. to start the configuration of a new consult type without already presenting it to the user.

Comment

Internal comment.

Consult statuses

Configuration

Consult statuses can be defined via Start > Configuration > Consult statuses.

Seq

The sequence number of the status determines the sorting order of the status in the life cycle of the consult. A newly created consult will have the status with the lowest sequence number.

Is editable

Determines if a consult with this status can be edited.

Role

If set, only users with this role can change the consult's status to this status.

Note

By default, GLIMS contains one Consult status record with:

  • Seq =1
  • Name = Created
  • Is editable = enabled

Visualisation

The Status of the consult is visible in the consult registration screen, the consult overview and the Consults browser.

Status update

The status of a consult can be changed via the contextual menu / ribbon items Upgrade status / Downgrade status.
