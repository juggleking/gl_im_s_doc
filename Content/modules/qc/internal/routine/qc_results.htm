Quality control - QC results

Contents: Browser - Editor - Reference range

QC result browser

From each QC population, you can use the contextual function Results to open the browser with all QC results for the selected QC population.  You can also use this browser to manually insert new QC results for a population.

The Options button allows to narrow down the results by specifying a status and/or severity range.

QC result editor

Double click a result or use the F6 key to open the QC result editor. The following buttons are available:

Detail

Click this button to consult more detailed information (such as logging) about this QC result.

Expand

Expands text modules used in the comment.

Reference range for QC results

Whenever a browser which normally shows the reference values for results shows a QC result, the settings of the QC population will be consulted to determine the 'normal range' for results of this population.

Accuracy control populations

The Target value and Allowed deviation will be used.

Precision control populations

  1. First the QC norm deviation count of the Assessment method will be checked.
  2. If the QC norm deviation count of the Assessment method is not specified, GLIMS will check the QC norm deviation count in the general settings (Start > System management > Customize > GLIMS QC).
  3. If the QC norm deviation count in the general settings is not specified, 3 is taken as the default.

The QCPopulation.LeveyJenningsGraph and QCPopuation.Print graphs take this into account: the 'red' zone of the graphs corresponds to the range outside this 'QC reference range'. In the cases where this reference range equals or is larger than 2 sigma, the zone starting 1 sigma below the 'bad' zone is indicated as the 'warning' zone (typically: the zone between 2 and 3 sigma when the QC norm deviation count is set to 3).
