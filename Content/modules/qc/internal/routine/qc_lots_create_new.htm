Quality control - "Create new lot" function on QC lot

Create new lot

Introduction

The first creation of a QC lot for a QC material is done by a manual insertion of a QC lot in the QC lot browser. You then have to manually add QC populations for that QC lot.

However, once the QC routine has started, you can use the contextual function Create new lot on a QC lot. This function can typically be used when a certain lot is finished and you want to continue with a new lot.

Tip

GLIMS offers a more user-friendly "Create new QC lot" function with more features. It allows to create a new QC lot from an existing one, activate the new QC lot and copy / activate / deactivate QC populations in one operation.

Functionality

The Create new lot function makes a copy of this lot and all its related records (QC populations and their checks), allowing easy creation of a new QC lot. For each existing QC object (an object for which QC populations can be found), the upgrade procedure will create a dedicated QCLot record. Of course, this lot will not contain the results of its predecessor.

The program will create one single QC population per assessment method: an open evaluation period (Evaluation status = Evaluation) when the original lot had an evaluation period for this method OR a control period (Evaluation status = None) when the original lot only had a control period. Click here for more information about evaluation periods.

Note

The inactive QC populations of the original QC lot are also copied to the new QC lot. The QC populations of the new QC lot will be active.

Activate new lot

By default, the new QC lot will not be active (Active option will be disabled). In order to activate the new lot, you should use the contextual function Activate on the new QC lot:

Activate populations

If set, all QC populations of the new QC lot will be activated.

Deactivate old lots

If set, the old QC lot will be deactivated, but its populations will remain active. This option can be used if you want to reuse this QC lot later on.

Deactivate old lots and populations

If set, the old QC lot and all its corresponding QC populations will be deactivated.
