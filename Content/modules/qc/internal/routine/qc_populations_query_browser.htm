Quality control - QC population query

Contents : Introduction - Query options  - Browser

Introduction

The QC population browser can be accessed:

  1. from the main menu via Start > Quality control > QC populations
  2. from a QC lot via the QC populations context function

QC population query options

Station / Property / QC material

Only QC populations for the specified Station, Property or QC material.

Combined / Accuracy / Precision / RMSD / Qualitative / Titer / Other

Include QC populations of type Combined, Accuracy, Precision, RMSD, Qualitative, Titer or other.

Evaluation status

Possible values:

None

Only select QC populations which do not serve as evaluation population.

Evaluation

Only select evaluation populations which are still in evaluation.

Closed

Only select evaluation populations which have already been closed.

Complete

Only select evaluation populations which could be closed (i.e. there are sufficient QC results to close the evaluation period), but have not yet been closed.

?

Do not take the evaluation status into account.

For more information, please check the chapter on evaluation populations.

Special

This field is particularly useful in case of automation via commands and tasks:

Auditable

Only select precision control populations with at least 15 unaudited results from different days and accuracy control populations with at least 1 unaudited result (both since the date of the last QC audit).

Unreliable channel

A QC population is selected when its channel status is Unreliable.

Error in last result

The severity of the most recent QC result of the population is > 0.

Error on channel

One of today's QC results of the population's assessment method has a severity > 0.

Error today

One of today's QC results of this population has a severity > 0.

Today

There is at least one QC result of today for this population.

Channel overdue

The QC validity of the Assessment method has passed since the last QC result for this assessment method was entered.

These Special query options make it possible to generate a printout containing only the results in which the QC manager is interested. The QCPopulation.Print function has additional options (regarding the formatting and suppression of the result detail table) to generate such an overview.

Active lot

Allows to select only populations from active or inactive lots.

Active population

Allows to select only active or inactive populations.

QC population browser

The query will produce a browser containing the requested populations.

Columns

Mark

The Percentile and Median values are only calculated when using the QC population function Recalculate statistics. To inform the user about outdated (*) percentiles or median values, a mark is added in the first column of the browser (a non-labeled column at the left shows a '*') and the editor (a '*' mark appears next to the percentiles and median values).

(*) Outdated means that a QC result was added or discontinued after a calculation took place (or when no calculation ever took place for a QC population).

Type

Possible entries are (language dependent):

  • 'A' = Accuracy
  • 'P' = Precision
  • 'C' = Combined (accuracy + precision)
  • 'S' = RMSD
  • 'e' = Population status Evaluation or Closed
  • 'DMed' = Population sub type is Day median
  • 'MAvg'  = Population sub type is Moving average
  • 'Qt' = Qualitative choice list
  • 'Tr' = Titer

Qual.

Channel quality of the related assessment method. This column is shown in red when the quality is unreliable.

Last sev.

The severity of the last QC result of today (shown in red when the corresponding severity is different from 0).

High sev.

The highest QC severity of today (shown in red when the corresponding severity is different from 0).

Choice list

Please check the chapter QC for qualitative results.

Titer

Please check the chapter  QC for titer results.

Bias, CPK, Sigma score, Bias (Tukey), CPK (Tukey), Sigma score (Tukey)

Please check the chapter Configuration: QC populations.

Detail box

The QC population browser, when opened from e.g. a QC lot, contains a detail box at the bottom with information on the selected QC population.

Examples

[RMSD]: Target value = 100.00, Reference value = ?, Low limit = 90.00, High limit = 110.00

[Accuracy]: Target value = 100.00, Accuracy limit (%) = 5.00, Accuracy high limit = 105.00, Accuracy low limit = 95.00

[Precision]: Target mean = 100.00, Target deviation = 13.895

[Qualitative]: Choice list = QCChoiceListPositive (100), Choices: Positive (10), StrongPositive (20)

[Titer]: Titer low limit = 10, Titer high limit = 15, Titer severity = 45

Note

The Target mean and Target deviation are displayed with the same number of decimals (i.e. digits after the decimal separator) as the QC population's calculated Mean value.
