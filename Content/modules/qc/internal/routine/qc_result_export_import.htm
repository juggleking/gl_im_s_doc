Quality control -> QC result export / import

Contents: Introduction - Export - Import - Restrictions

Introduction

GLIMS allows export QC results from one Glims database and importing them into another Glims database.  

Export QC results

Exporting the results can be done via the QCPopulation.Export() function and choosing the 'Internal' format.

  • Format
    • QBase: see the QBase chapter
    • Biorad: see the Biorad chapter
    • Internal: used for QC result export
    • Extra: for future use
  • Correspondent: only used for QBase and Biorad export
  • From - until: date range related to the QC result availability time
  • File: file name (use the 'Select button' for easy file specification)
  • Append: if the specified file already exists, add the export to that file or overwrite the file.

Import QC results

Importing the results can be done via the 'System Management -> Database -> Import -> Table-specific -> QC results' main menu option.  GLIMS 6.x users can access the import program via "Development -> 4GL -> Run program" menu option, specifying 'qcp_iui' as program and setting 'Pass handle' to TRUE.

Restrictions

  • Only results of existing QC populations are imported! The main purpose of this function is to be used during a Glims version upgrade during which users continue working on a copy of the original database. Make sure all the QC populations required during the upgrade period are created before taking the database copy !
      
  • Only the latest version of each result is exported. Results which have a successor (created via Result.Discontinue/Repeat, or Result.Modify()) are not exported.
