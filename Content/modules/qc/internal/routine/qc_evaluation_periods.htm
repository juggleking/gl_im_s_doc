Working with an evaluation period

Introduction

An evaluation QC population is used when a new lot is going to be used. In many countries, the lab has to use this lot for at least 20 days next to the current lot. After 20 days, the calculated Mean and Standard deviation obtained during the evaluation period are stored in a new 'control period' QC population, the old evaluation QC population is closed, and the new lot is now used instead of the old lot. So, during this period, at least 2 active lots exist for the same QC material. The evaluation status helps to distinguish between them, as well as offering a way to check if certain new lots have completed their evaluation period.

Note

GLIMS will verify the Evaluation status of a QC population when a QC result would set the assessment method's Quality to Unreliable. If this Evaluation status differs from None (i.e. Evaluation or Closed), GLIMS will not set the assessment method's Quality to Unreliable.

For more information, see Deviation actions.

When is an evaluation period complete

If you have specified an Evaluation period (in days) for the QC population, a message will be displayed when you manually enter a QC result for a completed evaluation period (i.e. there are valid results from at least QCPopulation.EvaluationPeriod different shifts). Additionally, a mail message will be sent to the user optionally specified in the QC manager field on Station, stating the fact that the evaluation period for this lot on this channel is full. Note that the option Mail method in the User configuration screen can specify whether he prefers to have internal mail or external mail (to be read via his normal mail program). The latter of course requires a valid E-mail address in the User configuration screen.

Closing an evaluation period

Querying QC populations which can be closed

You can query the QC populations which could be closed (i.e. there are sufficient QC results to close the evaluation period) by setting the query option Evaluation status to Complete.

Contextual function "Close evaluation period" on QC population

To actually close an evaluation population, you can use the contextual function Close evaluation period on the evaluation QC population. When closing the evaluation period, you have the following options:

Create control period

If set, an actual control population will be created (i.e. a population with Evaluation status set to None). The results of the evaluation population will be used to calculate the Target mean and Target deviation of the actual control population.

Create QC audit

Indicates that a QC audit record should be created prior to the closing of the current period. The audit record will contain the results used in the calculation of the targets for the new period.

2 rules are effective:

  • A QC audit can only be created for QC populations with at least 15 valid results.
  • A QC population in evaluation should have at least 20 results in order to be closed.

If these conditions are not met, the QC audit will not be created.

Recalculate mean / Recalculate SD

Indicates if the Target Mean and Target Deviation of the control period should be replaced by the calculated Mean and Standard deviation of the evaluation period.

One result per shift

Indicates that only one result per shift (the first) should be used for the new Mean.

Shift count

Indicates how many results of different shifts have to be taken into account (when different from the default in QCPopulation.EvaluationPeriod). It allows closing an evaluation period but taking more than QCPopulation.EvaluationPeriod shifts into account.

Notes

  • If you had already pre-configured a control population (and Create control period is set when closing the evaluation period), GLIMS will try and reuse this population. If you had already assigned QC checks to the control population, these will not be overwritten with the checks of the evaluation population. However, the Target mean and Target deviation will be overwritten!
  • The Close evaluation period function also takes the Ignored deviation count of the QC population into account.
