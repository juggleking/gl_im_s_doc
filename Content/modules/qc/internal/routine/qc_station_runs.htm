Quality control -> Station runs

Contents: Introduction - Creating a run - Starting a run - Stopping a run - Processing a run - Run confirmation

Introduction

A station run is considered to be a collection of QC and patient results which have been assessed in a certain time lapse.

Typically, a run starts with one or more QC results (start QCs), followed by a series of patient results, and ends with one ore more QC results (end QCs). A QC run is limited to one single assessment method, for instance a QC run on Hitachi 747 for Chloride. By means of a run, actions (these are not actions from the Action table!) can be taken on the results of that particular run.

A run is defined by its assessment method, the time at which it was started (this can possibly coincide with the first QC result), the time at which it was ended (can be unknown if not all end-QC results are available) and the possible action that was executed on the run.

Please also read the chapter Automated channel quality monitoring which discusses the implementation of "runs" in GLIMS.

Starting a run

This part describes how you can manually start a station run. For automated station run starts, please check the chapter Automated channel quality monitoring.

There are several ways to start a run:

  1. By selecting the Start run contextual menu function on an assessment method:

    This method will look for the most recent run, starting from the specified Start time. If the most recent run is already closed (end time has been entered), a new run will be started. If the most recent run has not yet been closed but already has at least one result, this run will be closed and again a new run will be started.

  2. By selecting the Start runs contextual menu function on a station:

    The Start run function on assessment method will be executed for all assessment methods of the selected station.

    When clicking Start runs, a window opens allowing you to set the Start time and (de)activate the following parameters:

    • Immediately show ended runs: to open up the Run confirmation browser immediately.
    • Only end runs with enough results: to close runs that have a sufficient number of results, as determined by the corresponding assessment method's Median result count limit.

  3. By selecting the Process run contextual menu function on a result:

    This method will look for the run which contains the selected result. If there should not yet be a station run, it will be created automatically. The run will start with QC samples that were not included in the previous run. The new station run will be displayed in the station run browser.

  4. The functions Station.StartRuns and Assessmentmethod.StartRun are batch-enabled, which means that they can be used as function in commands and be executed via the task scheduler.

Stopping a run

When the required end QCs (as defined in the assessment method) have been done, the run will automatically be closed.

A run can also be stopped as follows:

  1. By selecting the Stop run contextual menu function on an assessment method.

    This method will look for the most recent run. The run will be closed by entering an end time. One can prevent an empty run from being closed by enabling the Exclude empty run toggle box. If the Exclude unprocessed run toggle box is activated, runs on which no action has yet been executed will not be closed.

  2. By selecting the Stop runs contextual menu function on station.

    The Stop run function on assessment method will be executed for all assessment methods of the selected station.

    When clicking Stop runs, a window opens allowing you to set the End time and (de)activate the following parameters:

    • Only end runs with enough results: to close runs that have a sufficient number of results, as determined by the corresponding assessment method's Median result count limit.
    • Immediately show ended runs: to open the Run confirmation browser immediately.
  3. The functions Station.StopRuns and AssessmentMethod.StopRun are batch-enabled, which means that they can be used as function in commands and be executed via the task scheduler.

Figure 16

The station run browser, which can be opened from a station, assessment method or result, allows processing a run, i.e. executing a certain action on a run. The 'Forward' and 'Back' arrows allow navigating to the run immediately following or preceding the current run, independently of the context from which the review browser was started.

The table below indicates the implications of the available actions:

Action QC results Patient results Remarks
Accept QC - Accept regular Confirm Auto-confirm + auto-validate (1)  
Accept QC - Confirm regular Confirm confirm (2)  
Accept QC - Repeat regular Confirm Discontinue - repeat  
Reject QC - Accept regular Discontinue Auto-confirm + auto-validate (1)  
Reject QC - Confirm regular Discontinue Confirm (2)  
Reject QC - Repeat regular Discontinue Discontinue - repeat  
Wait No action No action The motivation is stored along with the run.

(1) auto-confirm + auto-validate:

General:

  • result must be available
  • the channel status of the result will be set to reliable

Confirmation if:

  • the result does not require manual confirmation
  • the auto-confirm severity of the channel is specified (not unknown)
  • the severity of the result does not exceed the auto-confirm severity of the channel

Validation if:

  • the result does not require manual validation
  • the auto-validate severity of the channel is specified (not unknown)
  • the severity of the result does not exceed the auto-validate severity of the channel

(2) confirm:

Confirmation if:

  • the result is available

The result will also be validated if:

  • the result does not require manual validation
  • the severity of the result does not exceed the auto-validate severity of the channel

Run confirmation

Introduction

A contextual menu function on Station records, Runs -> Confirm runs allows processing multiple unprocessed, ended station runs of a station simultaneously.

Browser

Color usage

The run rows, QC cells and median cells can have different background colors:

Green Red Blue
QC cell All the QC results of the station run lie within the limits defined under Patient day median. In other words, no limit severity has been applied to the QC results of the station run period. At least one QC result of the station run lies outside the limits defined under Patient day median. In other words, there is at least one QC result with a limit severity. At least one QC result of the station run could not be obtained (because of a missing patient result, for instance).
Median cell The median of all the QC results lies within the limits defined under Patient day median. In other words, there is no alarm severity on the global QC result corresponding to the global median. The median of all the QC results lies outside the limits defined under Patient day median. In other words, there is an alarm severity on the global QC result corresponding to the global median. The median of all the QC results could not be calculated (because a QC result is missing for instance).
Station run row

The QC and Median cells are not red (i.e. either both are green, or both are blue, or one is green and the other is blue).

If the option Preselect runs is activated, the row is preselected.

The QC cell and/or the Median cell is red. n.a.

Columns

  • Mnemonic: the station (or client station in case of concentrators)
  • Method: the assessment method of the station run
  • Start/End time: Start/end time of the station run
  • QC: The number of abnormal QC results / Number of QC results in the station run
  • #: The number of normal results in the station run
  • Median: the Median of the normal results in the station run
  • Range: the range this median should be in (as determined by the above mentioned settings on assessment method).
