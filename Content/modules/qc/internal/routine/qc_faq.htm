Mandatory external comment for failed QC results

Confirmation of QC results with severity > 0 and without specifying an external comment is not allowed, except via the automatic confirmation (Severity <=AssessmentMethod.QCConfirmSeverity).

When manually confirming such QC results, a popup screen will allow the user to enter this comment during the confirmation step, if the QC severity is equal to or higher than the assessment method's QC warning severity.

Retrieve a list of QC-lots for which there were abnormal results (severity > 100)

This list can be obtained using the Result Statistics module.

  • In the first screen, use a MISPL function for the row classification: RETURN .QCPopulation.Lot.Externalization;
  • In the selection screen: QC results = Select, Min. severity = 100

Norms for QC results

Whenever a browser which normally shows the reference values for results shows a QC result, the settings of the QC population will be consulted to determine the 'normal range' for results of this population.

  • For accuracy control populations, the TargetValue and AllowedDeviation will be used.
  • For precision control populations, this can be defined in the assessment method field QC norm deviation count or in the QC norm deviation count field of SpecificSite (accessible via Start -> System management -> Customize -> GLIMS - used if the assessment method field is not set).

Simultaneous status change of multiple QC populations

Using the QC population query, you can select all the required QC populations.  Using the contextual menu function 'Close evaluation period', you can change the status of the selected populations to 'Closed'.  See also 'Evaluation periods'.
