Patient moving averages

Configuration

The GLIMS implementation of "patient moving averages" is based on two Assessment method fields named Moving average interval and Batch size.

For every Moving average interval patient results, GLIMS will calculate an average of the last Batch size results.

Excluding values

It might be required to exclude result values from the calculation. The Assessment method option Result evaluation allows to specify a Result-based logical site function. If the site function returns FALSE, the value will be excluded.

QC population

The patient moving average results are stored in a QC population of type Moving average.  

If such a population does not exist, it will be created automatically, using the first QC lot of a QC material with mnemonic "MovingAverage". If necessary, such a QC material and QC lot are also created automatically.

Manual calculation

Although moving averages are automatically calculated as soon as results become available, it is possible to do the calculation manually - for instance when the moving average is configured for the first time on an assessment method.

The Assessment method context function Calculate moving average will do the calculation on patient results on that method for results which were available between the specified start date end date.

Reset moving average

From GLIMS 9.9.7 onwards, it is possible to reset the moving average for patient results via the contextual menu function Reset moving average of the assessment method of a Moving average QC population.

The moving average is then recalculated from the given Start time (this might not trigger the creation of new QC results if the number of patient results is insufficient). All QC results of the moving average QC population that are more recent than the given start time will be deleted.

Note

For GLIMS 9.9.7, the customer needs to create a tool (with Function mnemonic = RunProcedure) and add it to the menu of the table AssessmentMethod. In the function parameter setup, the function ResetMovingAverage needs to be selected.

Note

This function is useful when using the moving average as a continuous analytical quality control tool. It allows the user to handle situations where certain alarms were raised due to an outlier result and the user, after handling the alarms, wants to avoid that undesired alarms are still generated because it takes several measurements before an average result value returns to its 'normal' area.

Warning

When the Moving average QC population, and thus the assessment method, are Unreliable, it is very likely that new patient results are registered. All of these patient results will be marked with the Q flag.

Resetting the Moving average QC population will NOT influence this flag. It is up to the user to take the necessary actions to process the flagged patient results.
