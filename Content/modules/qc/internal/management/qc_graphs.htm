Quality control - Levey-Jennings graph

Introduction

GLIMS can also display the results of a QC population in a graph.

There are several ways to open a QC graph:

  1. Choose the context function Levey-Jennings graph on a QC population.
  2. Choose the context function Show QC graph on an assessment method or result.

General functionality

  • All QC results are represented by dots, the tooltip shows more detailed information.

  • For precision graphs, the Y-axis shows the reference mean and the 2/3 sigma lines, together with their absolute values. When the graph contains more than one population, the specific absolute values are only shown when selecting one of the populations by clicking on its label in the graph legend.

    When the QC population fields Limit 1 and Limit 2 are specified, these custom limits will be used to set the zones and labels (instead of the fixed 2/3 SD limits).

  • When multiple datasets are shown in the same graph, they also have common minimal and maximal limits for their Y-axis, so the scale of the Y-Axis does not change when selecting the individual datasets by clicking on their legend entry. QC populations which are explicitly selected in the parent browser (using the space bar, visible by the ">" indicator in the browser) are always shown in the graphs. When clicking on another QC population while the graph is being shown will show this QC population in combination with the populations which are explicitly selected.
  • The graph will also show important StationLog events (where StationLog.ShowOnQCGraph=YES) and the ReagentUsage records on the qc graph. These events are indicated by a vertical line in the graph. Holding the mouse pointer over such a line shows a tooltip describing the event.
  • Results having a comment will be marked by drawing a circle around the data point in the qc graphs.

  • Discontinued results are also shown in the QC graphs, but have a cross as graphical mark and are not connected to the other points of the graph. The tooltip of a discontinued result will show its comment and its rejection reason.

    When selecting multiple QC populations to be shown in the graph and these populations have different limits, GLIMS will not be able to show these areas separately for each QC population.

Buttons

Populations

Allows calling the browser of active QC populations for the current assessment method.

Back / Forward

Allows moving the graph back or forward in time. See also Date range (below).

Date range

Range

Offers a number of predefined ranges. The default is 2 weeks. The end date will automatically be set to Today.

  • When selecting the option Explicit, you can specify a specific start and end date.
  • The option All will disable the Start date and End date fields and show all QC results.

Start and end date

Allows to enter a specific start and end date when the Range option is set to Explicit.

Other features

  1. When changing the Range or Start/end date, the graph will automatically be refreshed.

  2. The Back and Forward buttons will use a date shift that corresponds to the specified range (or the number of days between start and end date).

  3. The Range and Start/end date will be remembered when navigating to e.g. another QC population. These settings are also remembered per user.

  4. The graph screen can be pre-configured as a menu or ribbon item. However, when setting up the function from a Result context, all parameters will have a fixed value.

  5. When starting the graph from a QC population, Assessment method, Microbiology QC test or Microbiology QC result, the graph end date is by default set to Today. The graph start date is determined by the Range. In case Range is set to Explicit, then the Start and End date values are used.

  6. When starting the graph from a QC result, the graph end is set to the availability time of the last result (+1 day). The Range value is set to Explicit (cannot be changed in this case) and the start date is set to 2 weeks earlier than the end date.

Show moving average

By default, when displaying a QC graph for more than one QC population (including a QC population of type Moving average), the QC population of type Moving average will be skipped.

In order to display the QC population of type Moving average on the QC graph, use the option Show moving average in the Accuracy tab page.

Notes

  1. The QC population of type Moving average will be displayed immediately on the QC graph (Precision and Accuracy tab) if it is the only QC population selected for display.
  2. The Show moving average setting is not stored as a user preference.
