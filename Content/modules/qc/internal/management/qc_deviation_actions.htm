Introduction

This page explains how a QC result severity is assigned and what the possible influences of a QC result severity are.

Severity of a QC result

There are several places in GLIMS which allow to assign a severity to a QC result. The final severity of a QC result is calculated as the maximum of all absolute severity values. The possible QC result severities are:

QC severity

The QC severity of a QC result is set when the QC result fails one or more configured QC checks. For each type of QC check you can define an individual severity.

Note

If a non-numeric QC result is entered for a QC population of type Quantitative or Titer, the QC severity for non-numeric results will be applied as QC severity for that QC result.

Station severity

A QC result sent by an analyzer can be accompanied by a flag. The station flag configuration in GLIMS allows to set a Severity for QC for each flag. The Severity for QC specified for the station flag accompanying the QC result will be stored as the QC result's QC severity if it is higher than the already assigned QC severity.

Manual severity

The manual severity of a QC result can be set as follows:

  1. By zooming into the QC result detail screen and entering a value in the Manual severity field.
  2. By entering a manual severity when using the contextual function Modify on a QC result.

Note

Consulting the severities of a QC result:

  1. All QC result severities can be consulted in the detail screen of a result.
  2. In many QC result browsers, a dedicated column indicates the severity.

Reliability of a QC population

Set to unreliable

Automatically

If a QC result is entered of which the QC severity exceeds the assessment method's QC alarm severity, the QC population is automatically set to Unreliable.

Manually

It is possible to manually mark a QC population as Unreliable. In order to do so, use the contextual function Set QC population to unreliable.

Note

If a QC population is marked as Unreliable, the Quality field of the associated assessment method will also be set to Unreliable.

Reset to reliable

Automatically

A QC population is automatically reset to reliable when a valid QC result is entered.

Manually

It is possible to manually mark a QC population as reliable. In order to do so, use the contextual function Set QC population to reliable.

Note

The Quality field of the associated assessment method may also be reset to Reliable depending on the assessment method's Quality reset setting.

Reliability of an assessment method

Set to unreliable

Automatically

"QC severity" exceeds "QC alarm severity"

In the assessment method configuration, you can define a QC alarm severity. It indicates the minimum QC severity which will mark this method as Unreliable. 

This has the following effects:

  • From that moment onwards, all patient results will also be marked as unreliable in their Channel quality field.
  • If an assessment method is marked as Unreliable, the automatic confirmation and validation rules will be blocked until the quality status has been reset. However, when manually confirming a result, the auto-validation rules will be applied anyhow.
  • The Quality comment field of an assessment method indicates the reason of the method (un)reliability.

Note

GLIMS will verify the Evaluation status of a QC population when a QC result would set the assessment method's Quality to Unreliable. If this Evaluation status differs from None (i.e. Evaluation or Closed), GLIMS will not set the assessment method's Quality to Unreliable.

"Last QC time" versus "QC Validity"

Each time a QC result enters the system, the time is recorded in the assessment method's Last QC time field. Each time a patient result enters the system, the Last QC time and QC Validity are compared. When the Last QC time is longer than <QCValidity> hours ago, the Quality field of the assessment method (and the Channel quality of the patient result) is set to Unreliable.

Note

When a moving average result or a median result is entered, the Last QC time of the Assessment method will only be updated if the result causes the channel quality to be set to Unreliable.

Manually

Use the Change channel quality contextual function on Assessment method. A justification is to be specified which will be added to the assessment method's Quality comment field.

Reset to reliable

When the Quality of an assessment method has been set to Unreliable, there are several ways to reset the Quality to Reliable. The QC tab folder of the assessment method contains a field Quality reset. Its possible values are:

Manual

The Quality of the assessment method must be reset manually. To do so, use the Change channel quality contextual function on Assessment method. A justification is to be specified which will be added to the assessment method's Quality comment field.

GLIMS will also add a station log entry with the following data:

  • Event type = Log
  • Code = ChQual
  • Short description = Channel quality changed from 'old-quality' to 'new quality'
  • Long description = Channel quality changed from 'old-quality' to 'new quality' + the justification text  

Automatic

The first QC result having a severity of 0 will reset the Quality to Reliable.

Automatic - same material

The first QC result of the same QC material having a severity of 0 will reset the Quality to Reliable.

Automatic - all populations

The first QC result of the same QC population having a severity of 0 will reset the Quality to Reliable provided that all active QC populations linked to the same assessment method are reliable.

Tip

This is particularly useful when monitoring different concentration levels for the same property, which implies that several active QC populations are linked to the same assessment method. This way, upon entering a valid QC result, the Quality of the assessment method will not be reset to Reliable if the other QC populations (levels) are still Unreliable.

End of run

See Automated channel quality monitoring.

Prevent automatic confirmation

In the assessment method configuration, you can define a QC auto-confirm severity. QC results with a Severity up to this level will be auto-confirmed.

Show warning for QC result with QC severity

In the assessment method configuration, you can define a QC warning severity. This field indicates the QC severity starting from which a warning is issued on screen when entering a QC result.

Change quality comment of assessment method

In order to change an assessment method's Quality comment, use the Add quality comment context function on Assessment method.

This function has an option Clear old comment which is initially disabled. When set, the current value of the assessment method's quality comment is cleared. When clearing is set, the user is not obliged to enter a comment value. When clearing is not set, the user must enter a new comment (or cancel the operation).
