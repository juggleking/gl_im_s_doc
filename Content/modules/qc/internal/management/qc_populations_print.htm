Quality control -> The QC Population Print function

Contents: Introduction - Selection options - Specific options - Output options - Automation - Output examples

Introduction

From each QC population, you can select the contextual menu function Print.  The contents of the QC population printout is determined by the type of QC population (Precision, Accuracy or Combined) for which the audit has been performed.

  • QC population printouts for precision checks will only mention the Maximal CV check results.
  • QC population printouts for accuracy checks will only mention the Maximal R check results (absolute or relative, depending on the pivot and/or target deviation configuration).
  • QC population printouts for combined checks will mention the Maximal CV check results and the Maximal R check results (absolute or relative, depending on the pivot and/or target deviation configuration).

Selection options

Period

This month / Previous month  

Allows selecting all QC populations of this month or the previous month.

Until Yesterday / Until Today / Until Previous Month

The main purpose of these three options is to allow results which were not audited in the 2 previous months to be automatically included in this month's audit.

When using one of these three options and no start date is specified, then the latter is calculated as the highest end date of the stored audits + 1, with a maximum of 2 months ago. The actual start date stored in the QCAudit record will be the date of the first actual result in the audit, with a minimum of the start of the previous month for 'Until Previous Month' and the start of the current month for the others.

In all cases, when the end date is not specified, it will be derived from the period setting. When using the 'Until Today' or 'This Month' option, the actual end date stored in the QCAudit record will be the date of the last result in the audit, with a minimum of 'Yesterday'.

From - to

Allows selecting all QC populations within a date range

Accuracy, precision, combined: 

Include accuracy, precision and/or combined populations.

Shift count

Only include results of the first x shifts.

Official results

Determines which results are used to calculate the statistics for the created QCAudit record. It also marks the used results with 'Official=YES' in the created QCResult records.  See also: QC shifts

Store as official audit 

During the printing of a QC population, QC audit records will be created.  For more detailed information, please check the chapter on audit reports. 

Note

GLIMS will only create a QC audit if the population contains

  • at least 15 confirmed QC results for a RiliBÄK population (within the date range specified in the QC population print settings).

    If the date range specified in the QC population print settings covers a period longer than 3 months, the minimum of 15 confirmed QC results is not required. A message will still inform the user that less than 15 confirmed results are available.

  • at least 20 confirmed QC results for a non-RiliBÄK population (within the date range specified in the QC population print settings).

Specific options

Create document

If set, an actual document will be printed.  If not audit reports will be created but no actual document will be generated.

Only print official results

If set only official results (as selected with the 'Official results' option) will be included in the printout.  If not, all results will be printed.

Header, Body and Footer

For the header, the footer as well as the result body, it is possible to specify a QCAudit or QCAuditResult-based dynamic text to be used instead of the default.  If such a text is specified, the corresponding default is suppressed.

When the returned text contains the string "<Html>" and the chosen format is html, then the text is added to the output as is. This makes it possible to specify your own html formatting via standard html tags. In the other case, the text is presumed to be passed to html as a preformatted text.  

For the body text, the passed text is passed to HTML between <TR> and </TR> tags, so it is treated by HTML as being the contents of a row.  

Note that the QCPopulation.Material field is included in the property description of the default header. This optional field indicates the basic patient material for which this QC control is intended.  The property description on the printout can then be "Na (Serum)" when QCPopulation.AssessmentMethod.Property.Mnemonic = NA and QCPopulation.Material.Mnemonic = Serum.

Example of a header text - Example of a body text

For PDF and XML reports, the header, body and footer texts can be used for exporting additional data in the XML file. The texts can be flat text or a (well-formed !) XML string.  In the first case, the text is inserted as such as one node in the XML. In the second case, the XML text is inserted as a sub-tree in the XML file.

Page width, page height

Allow to fine-tune the layout of the document to be generated.

Page skip

Allows suppressing the automatic page break between the printouts. This allows for a more concise overview.

Graphs

Allows selecting which graphs should be included

Graph orientation

For PDF and XML reports, this option allows printing the Levey-Jennings graph horizontally (above the results) or vertically (in the results table).

Remarks

  1. In case of HTML precision print-outs, an actual graph will be included (see example below).  Note that you need the GRAPHICS license for this.

  2. The chosen "Format" is passed to the text module as a variable called "Format", and can thus be accessed via the "{Format}" syntax.

Automation

It is possible to automate this document generation using commands and tasks.   The base table is QCPopulation, the query to be used is ExtendedQuery and the function should be QCPPrint. 

The QC Population ExtendedQuery has a 'Special' parameter. When this is set to 'Auditable', the query only selects precision control populations with at least 15 unaudited results from different days and accuracy control populations with at least 1 unaudited result (both since the date of the last QC audit).

Remark: this program also takes the 'Ignored deviation count' of the QC population into account.
