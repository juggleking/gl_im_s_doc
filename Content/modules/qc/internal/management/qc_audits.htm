QC audits

Contents: QC audit creation - QC audit query/browser - QC audit print - Reusing & updating existing QC audits

QC audit creation

QC audits are created during the QCPopulation.Print() program. This program proceeds in 3 steps:

  • QC audit records are created for each processed QC population. These QC audit records store the date range, the user's id, the audit date and whether a deviation was found. The deviations currently detected is the fact that in the selected result range, the calculated coefficient of variation (CV) exceeds the QC population's CV Max (%) for precision control populations and that the calculated inaccuracy (R) exceeds the QC population's Maximal inaccuracy for accuracy control populations.  Note that this program also takes the 'Ignored deviation count' of the QC population into account.
      
  • If the CreateDocument parameter of QCpopulation.Print() was set, the documents are generated starting from the QCAudit record. Among other information, the generated documents also contain the following data:
    • the department of the QC population (with address) is added to the header
    • the QC audit date and result date range is added to the header
    • any deviation and review notes (+ reviewer identify) are added at the bottom of the document
        
  • When the Store as official audit parameter of QCPopulation.Print() was not set, the QCaudit record is silently removed again.  Note that when the 'Store as official audit' option is set, the result date range is automatically adapted to make sure no QC audit records are created with overlapping date ranges.  Moreover, 'Previous month' is interpreted as 'Since last audit up till last day of previous month, while 'This month' is interpreted as 'Since last audit till yesterday'.

QC audit query/browser

Query

The QC audit browser is available from the main menu (Start -> Quality control -> QC audits).  A typical query would result in a list of QC audits for which a deviation was detected but which were not yet reviewed.

Selection criteria include Department, Station, Type (Precision, accuracy, combined, qualitative or Titer) and whether a deviation (Yes/No) was detected and/or a Review Note (Yes/No) was added. 

Executing department (group)

A QC audit will be selected if:

Any of these departments:

    • QC Audit -> QC audit result -> Result -> Department

Matches any of the following executing departments:

    • the specified "Executing Department"

    • OR any of the executing departments in the specified "Executing department group"

Browser

The QC audit browser shows the following data:

  • Station

  • Property

  • Type: a combination of the basic QC type (Precision, Accuracy, Combined or Qualitative or Titer) and the evaluation status of the QC population at the time of creation of the audit.
    Example: the 'Precision' audit of a population in evaluation would be 'PE'.  Note that since Precision and Accuracy are split over separate audit records, the combination 'AP' or 'APE' can not appear here.

  • Start and end date of the audit

  • Deviation detected

  • Reviewed

  • First and last result date

  • Official result count

  • Mean, CV and R value

Note

The number of significant digits of the mean and standard deviation values of the QC audit is the maximum number of significant digits of the QC results + 1.

For instance:

For the following result values 12, 12.1, 12.10, 13, 11.8, 11.90, the maximum number of significant digits = 4 (values 12.10 and 11.90).

--> QC audit: the number of significant digits will be 5.

Marking a QC audit as "Reviewed"

Two menu functions are available:

Review

Marks the QC audit as reviewed.  The user must enter a review note.

Undo review

Allows to mark a reviewed QC audit as not yet reviewed.  This user has the option to clear the review note.

Remarks:

  1. The QC audit's "Review" field can also be changed manually in the QC audit editor.

  2. The Review note field in the QC audit editor can always be changed, even if the audit was already reviewed.
     

  3. The Review note will be printed at the end of the re-generated document.  You can reprint a document using the contextual menu function Print on the QC audit record.

QC audit print

The QC audit menu function Print allow to print a QC audit report.  Note that the contents of the QC audit report is determined by the type of QC population (Precision, Accuracy or Combined) for which the audit has been performed.

  • QC audit reports for precision checks will only mention the Maximal CV check results.
  • QC audit reports for accuracy checks will only mention the Maximal R check results.
  • QC audit reports for combined checks will mention the Maximal CV check results and the Maximal R check results.

The QC audit print function offers the same functionality as the QCPopulation.Print function.

Reusing & updating existing QC audits

The QCPopulation.Print() function allows specifying a date range and whether or not an official qc audit has to be created.  If the actual start date of the selected results corresponds with the start date of an already existing QC audit record, this record is reused (with an appropriate message).

  • When no period or start date is specified in the user interface, the end date of the last official QC audit + 1 is used as start date.
      
  • When the resulting start date corresponds with an existing QC audit and StoreAudit=NO, then the existing audit is reused without any modification. When StoreAudit=YES, the new data will overwrite the old data.
