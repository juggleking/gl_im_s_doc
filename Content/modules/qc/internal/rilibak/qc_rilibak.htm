Introduction

In Germany, the quality control of lab results has to be performed under strict regulations dictated by RiliBÄK. Since 15-2-2008, new regulations are defined and had to be fully operational in the German laboratories at the latest on 1-4-2010.

The RiliBÄK feature requires a separate license "Quality Control RiliBÄK 2008". The license activates parts of the QC module user interface.

Database changes

Table QC population

Prefix

All new fields related to the ‘QMMA‘ (Quadratischer Mittelwert der Messabweichung) regulation are prefixed with ‘RMSD’ (Root mean square deviation).

See also: http://en.wikipedia.org/wiki/Root_mean_square_deviation

RMSD target value

  • Name: RMSDTargetValue

  • Data type: Fractional

A target value (“Zielwert”) can be delivered by the supplier or might be calculated as the mean of RMSD supplier low limit and RMSD supplier high limit.

RMSD reference value

  • Name: RMSDReferenceValue

  • Data type: Positive fractional (percentage)

  • Size: 5 (inclusive decimal point)

Allowed relative value for deviation described in the RiliBÄK regulations annex table B1 3rd column (Spalte 3).

RMSD low limit and RMSD high limit

  • Names: RMSDLowLimit, RMSDHighLimit

  • Data type: Fractional.

  • Access: Display

Read-only fields, calculated from the RMSD target value and the RMSD reference value. If the RMSD reference value is unknown, the values are taken over from RMSD supplier low limit and RMSD supplier high limit.

RMSD supplier low limit and RMSD supplier high limit

  • Names: RMSDSupplLowLimit, RMSDSupplHighLimit

  • Data type: Fractional.

Reference values as documented by the QC-material supplier in low and high value.

RMSD severity

  • Name RMSDSeverity

  • Data type: Positive integer.

  • Size: 3

An alarm severity when results are not within the reference values (RMSD low limit and RMSD high limit). The concept remains the same as with “AccuracySeverity” , “Severity2S”, and “Severity3S”.

RiliBÄK origin

  • Name: RilibakOrigin

  • Data type: Char 10.

  • Initial value: “B1”

This is an informational text to reference the used annex or table from RiliBÄK documentation.

RMSD factor K

  • Name: RMSDFactorK

  • Data type: Fractional.

  • Initial value: 3.00

QC Audits are calculated using a formula with a “K” factor. Currently only the value 3.00 is used in formulas. It is suggested to initialize this field with “3.00” as the default value during new configuration since this is currently the only value that is known by the current RiliBÄK tables. Future RiliBÄK tables might suggest using other K-factors.

Needs evaluation period

This field is used to decide if an evaluation period is to be created when creating a new lot. Default value = NO.  This option will only be shown when the RiliBÄK license is active.  

Auditable

  • Name: Auditable

  • Data type: LogicalBox

  • Initial value: Checked (YES)

Under some conditions, RiliBÄK accepts no audits. Example: When less than 15 results are measured within a time frame of 3 months, no audits are required.

 

Note

QC population data provided by suppliers can be imported in the fields of the QC population table via Excel or XML files: Sysmex, Sapphire, Advia.

Table QC audit

Extent of QCAudit.Type

The enumerated field QCAudit.Type consists of: ’ Accuracy’, ‘Precision’, ‘Combined’. Item RMSD (in German: ‘QMMA’) has been added

RMSD value

Name: RMSDValue: this is the calculated Root mean square deviation (in German: ‘QMMA’) value, shown as a fractional.

RMSD low limit, RMSD high limit, RMSD reference value

These values are specified:

  • When closing an RMSD evaluation period while requesting the creation of a QC audit and a control period. The limits and reference value calculated for the control period are stored in the QC Audit. One of the purposes of storing these values in the QC audit, is not to lose them when control period creation failed because of a deviation was detected in this closing process.

  • When printing an RMSD QC population in evaluation status 'Evaluation'. GLIMS will calculate the low and high limits as if a control period is created while printing. If audit storage is requested, the values are stored in the audit record. The calculated RMSD limits are available on the printout (even if no audit storage is requested).

Changes in the QC population browser

Type column

The browser has a column showing information that types the population.  The values the column can have are:

  • ‘C’ = Combined (TargetValue and MaximalInaccuracy and MaximalVC), in German: ‘K’

  • ‘A’ = Accuracy (TargetValue or AccuracyLimit or AccuractyHighLimit or AccuractyLowLimit), in German: ‘R’

  • ‘P’ = Precision (TargetMean or TargetDeviation), in German: ‘P’

  • ‘AP’ = Accuracy & Precision, in German: ‘RP’

  • ‘CE’ = Combined & Evaluation, in German: ‘KE’

  • ‘AE’ = Accuracy & Evaluation , in German: ‘RE’

  • ‘PE’ = Precision & Evaluation, in German: ‘PE’

  • ‘DMed’ (Sub type = day median)

Evaluation means 'in evaluation' or 'evaluation period closed' 

New values for type column are: ‘S’ (RMSD) and ‘SE’ (RMSD + evaluation),  in German: ‘Q’ (QMMA) and ‘QE’ (QMMA + Evaluation)

New column ‘verification’

This column is shared with other columns (by means of a column cycle). The column cycle items are ‘cluster name’, ‘quality’ and ‘verification’

The verification column shows the defects of RMSD QC population for which the configuration parameters are not correct (not complete).

Parameter ‘auditable’

The QC population query has a parameter ‘Special’, which is an enumerated. One of the existing items is ‘auditable’.  The meaning was: Auditable only selects precision control populations with at least 15 unaudited results from different days and accuracy control populations with at least 1 unaudited result (both since the date of the last QC audit).

For RMSD populations, the minimal required results is different depending on the evaluation status of the population: When not in evaluation, the minimum is 15 results, for populations in evaluation, the limit is 15 results, but only a single result per day.

This ‘auditable’ query criteria has been extended by doing an additional check on QC population field ‘auditable’. If latter is not set, the QC population is not selected. It is not a RMSD-specific feature , it is also valid for other types of QC populations

QC Audit query

Parameter ‘type’

The query parameter type has the values (combo box): ‘?’,’ Accuracy’, ‘Precision’, and ‘Combined’. The choice ‘RMSD’ has been added, in German: ‘QMMA’ . This choice is only shown with an active RiliBÄK 8 license.

QC Population – Close QC Evaluation Period

This existing method on QC population has been adapted for RMSD QC populations (see also work scenario below).  The restriction for RMSD QC is that a minimal of 15 results (only one per day) is required to close an evaluation period (as for other population types).

When closing the population, the fields RMSD reference value, RMSD low limit and RMSD high limit are calculated (see life cycle below for details) for the control period population.

When audit report creation was requested, it will also mark the audit record when a deviation is detected, being: the calculated limits have a conflict with the supplier limits (more specifically, when the RMSD Low limit is lower than the RMSD Supplier low limit or the RMSD High limit is higher than the RMSD Supplier high limit).

QC population – Evaluation upon new result

As for any other QC result, QC results of a RMSD population are also evaluated against the limits. The result gets a severity when a deviation is detected.

Evaluation QC populations

For evaluation QC populations, the check is against the supplier limits.

If a result of an evaluation QC population lies outside the supplier low and high limits, the Channel quality of the assessment method is set to Unreliable.

Control QC populations

For control QC populations, the check is done against the calculated limits.
