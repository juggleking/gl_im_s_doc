Configuration: Assign checks to populations

Once the populations have been defined, it is possible to assign checks. Assigning a check to a population ensures its execution, each time a new result is added to the population. The results of these checks can influence the reliability status of the associated assessment method. This will be further explored in the chapter on QC management.

Use the contextual function Checks on QC population, insert a new check and select the QC check type (Westgard, Accuracy, Trend or Trigg) and the corresponding Template.

Note

It is also possible to directly define one accuracy and precision check in the QC population, without having to define separate QC checks.
