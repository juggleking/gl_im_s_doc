Quality control - QC checks

Introduction

A good way to start the QC configuration is to define the QC checks that you want to apply for your internal quality control.

GLIMS offers the following check types: Accuracy check, Trend check, Trigg check and Westgard check. For each of these, a dedicated template class has to be configured. These templates are available from the main menu via Start > Quality control > Configuration > Checks.

Note

It is also possible to directly define one accuracy and precision check in the QC population, without having to define separate QC checks.

Accuracy check template

Fields

Name

Accuracy checks are identified by a name.

Accuracy

This check is violated if the distance between the result and the target value of the population (in %) exceeds the Accuracy.

Severity

The Severity indicates the result severity when the check is violated.

Internal comment

The internal comment contains the user message which is shown when the check is violated.

Variables

The following variables can be used in the internal comment text:

  • {Name}

  • {Limit}

  • {Deviation}

Example

QC accuracy alarm: {Name}

Limit = {Limit}%, Deviation= {Deviation}%

Trend check template

This check evaluates the trend of consecutive results within a QC population.

Fields

Name

Trend checks are identified by a name.

Count limit

The Count limit is the maximal allowable number of results with an equally-signed (+ / -) difference from their predecessor, i.e. the number of consecutive results needed for violation.

Severity

If the Count limit is exceeded, the last result's QC severity is increased to Severity.

One result per shift

The option One result per shift indicates that only one result per shift should be taken into account for this check. This also implies that all other results are not checked.

Internal comment

The user-definable internal comment is shown when the check is violated.

Variables

The following variables can be used in the internal comment text:

  • {Name}

  • {CountLimit}

  • {CurrentTrend}

  • {TrendLength}

Example

QC trend alarm: {Name}

Current trend: {CurrentTrend} Length: {TrendLength}

Westgard check template

The standard Westgard rules can be defined with the following values for Population count, Value count, Deviation count and Placing (supposing only 2 materials are involved in the checks "across" materials):

Rules Population count Value count Deviation count Placing
1S2 1 1 2 N/A
1S3 1 1 3 N/A
2S2A 2 1 2 Gathered
2S2W 1 2 2 Gathered
RS4A 2 1 2 Mirrored
4S1A 2 2 1 Gathered
4S1W 1 4 1 Gathered
10XA 2 5 0 Gathered
10XW 1 10 0 Gathered
Su3    (*) 3 1 4.13 Summed

* The rules where 3 results of 3 levels are cross-compared (one result of each level) and where the sum of the absolute eccentricities should be smaller than 4.13. This is NOT a 'regular' Westgard rule. It means: take 1 result of each of the 3 populations and calculate the SUM of the absolute value of (Value-Mean)/Mean. If this sum is > 4.13, then an error is raised. So instead of checking against specific limits, this Summed rule tries to find a measure for the TOTAL or COMBINED deviation of the control samples. 

Note

  • The placing Mirrored only makes sense if PopulationCount = 2.
  • When using a Westgard check 'across' populations, e.g. <PopulationCount> = 2, <ValueCount> = 1, the <ValueCount> last results of each population on the same assessment method are checked. The rule fails if at least <PopulationCount> of these populations (including the current population) violate the rule.
  • The placing N/A also has a meaning for multi-population or multi-value rules. Example: <PopulationCount> = 2, <ValueCount> = 1, <Placing> = N/A, <DeviationCount> = 2 means: check if of at least 2 populations (current included) the last result is more than 1 sigma away from its target mean.
  • The Summed placing also works correctly when more populations than <PopulationCount> can be found: the sum of the eccentricities of the last <ValueCount> results of each population is calculated. When the total of the <PopulationCount> largest sums (current population included) is more than <DeviationCount>, the rule is violated.

Fields

For more information on each individual field, please check the reference guide. For more information on Westgard rules in general, check our dedicated Westgard manual or visit the Westgard website.

Variables

The following variables can be used in the internal comment text:

  • {Name}

  • {ValueCount}

  • {Deviation}

  • {Limit}

Example

Result date {=DateTimeToString(.AvailabilityTime,"%d/%m/%Y %H:%M")}

QC Westgard alarm: {Name}

ValueCount {ValueCount}, Excentricity {Deviation}, Limit = {Limit}

Station : {= .QCPopulation.AssessmentMethod.Station.Mnemonic}

Material: {=.Specimen.Material.Description()}
