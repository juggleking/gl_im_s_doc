Quality control - QC lots

Contents: Introduction - QC lot browser - QC lot query - Fields

Introduction

The first time you enter a QC lot (delivered by the supplier) of a certain QC material, you will have to do this manually by inserting a new QC lot in the QC lot browser.

Later, when a new QC lot for this material is to be used, you can create a new lot using one of the following contextual functions:

  • Create new QC lot on a QC lot (offers more features than the contextual function Create new lot)
  • Create new lot on a QC lot

QC lots can also be created automatically when the first product of a product lot is checked-out in the Stock management module.

QC lot browser

How to access the browser

The QC lot browser is available via:

  1. the main menu: Start > Quality control > QC lots

  2. the contextual function Show all > QC lots on a QC material

Columns

The following columns are shown:

  1. Externalization

  2. QC material mnemonic

  3. QC material code

  4. QC lot number

  5. Active

  6. QC lot code

  7. QC lot production date

  8. QC lot start and end date

  9. QC lot expiration date

Other features

  1. QC lots which are not active are displayed in "read-only color"

  2. QC lot start date cells are in red when QC lot start date is in the future

  3. QC lot end date cells are in red when QC lot end date is in the past

  4. QC lot expiration date cells are in red when QC lot is currently expired 

QC lot query

The QC lot browser has an Options button to define query criteria:

QC material

Only QC lots of this QC material.

Status

Only active, Only inactive, Both (default).

Expired status

Not expired, Expired, Both (default).

Lot number

Only QC lots with this lot number.

Lot code

Only QC lots with this code.

Externalization prefix

Prefix the QC lot externalization should start with.

In use since

QC lots with start date since this date (relative date supported).

In use until

QC lots with end date until this date (relative date supported).

Produced since

QC lots with production date since this date (relative date supported).

Produced until

QC lots with production date until this date (relative date supported).

Filter

Logical site function based on table "QCLot".

Fields

Externalization

Is an identification which is automatically generated by means of the QC lot externalization specified in the QC setup screen.

Material

A QC lot always refers to one QC material.

Lot number

Allows to store the actual lot number.

Code

A (bar)code that may be used to identify this QC lot. This code can be very important to store barcode identifications originating from an analyzer. Whenever a barcode is received from a station (e.g. during result upload), GLIMS will check whether this barcode represents a QC lot where this code equals the barcode. If specified, the QC lot code will take priority over the QC material code.

Tip

It is recommended to use the Code field on QC lot level instead of the Code field on QC material level. This allows you to have a different (bar)code for each QC lot so that several QC lots can be used simultaneously.

Expiration date

Indicates when the QC lot expires and should no longer be used.

Tip

  • If the QC lot is expired when entering a QC result, a severity will be applied to that QC result.

    For more information, please check the option QC lot expiration severity in the general settings of the QC module.

  • In the work list entry browser (select a work list and choose the contextual function Entries), expired lots are shown in red (in the column Action Object).

Object

Reference to an Object record and an object template.
