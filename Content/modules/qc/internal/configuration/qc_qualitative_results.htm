QC for qualitative results

Introduction

Quality control for qualitative results means comparing values against a list.

A typical list could be: "positive,strong positive,...".  If the result value does not match any entry, a severity is raised. Note that the list choice itself can also have a severity.

Possible violation messages (stored in the result's internal comment) are:

  1. When a matching choice has a severity: Qualitative control violation for '&1': QC choice '&2', severity &3
  2. When no choice matches and the choice list has a severity: Qualitative control violation for '&1': No QC choice '&2' found, severity &3

The violations follow the common behavior for all QC population violations (warnings, channel blocking, etc...).

Features

QCPopulation.QCChoiceList

The QC population field "QC choice list" is a reference to a the table "QCChoiceList", which is very similar to the "ChoiceList" table. A "QC choice list" refers to QC choices (table "QCChoice").

A "QC choice list" has a mandatory field "Severity": this is the severity that will be applied when the QC result value does not match any of the QC choice mnemonics (case sensitive).

QCPopulation.Configuration

This field allows making a clear distinction between quantitative QC populations and others like qualitative QC populations. Possible values are: Quantitative (default), Qualitative and Titer.

Once the configuration is chosen and a representative field related to this configuration is filled in, the configuration field becomes read-only.

QCAudit.Type

An entry "Qualitative" is available for the domain QCType for QC audits:

  • "Precision,Accuracy,Combined,RMSD,Qualitative"

Short values are: P,A,C,S,Qt

QC population journal - Report builder

The XML output for the QC population journal includes the data of the QC choice list. The XML is available as input for Jasper reports. QC results have an XML node "InRange" which will be "false" or "true" (true if the result value is member of the choice list).

QC population print (audit report)

  • An option "Qualitative" is available for the QC population print function: if enabled, qualitative QC results will be included.
  • The XML output includes the data of the QC choice list. The XML is available as input for Jasper reports.
  • Deviations are not applicable to qualitative audit reports (QCAudit)
  • No support for graphs on qualitative audit reports.

QC screen graph

The QC screen graph is not implemented for qualitative populations.

QC population query

An option "Qualitative" allows to include/exclude qualitative populations. It is enabled by default.

QC population browser

Choice list

A column "Choice list" shows the mnemonic of the QC choice list in the QC population browser

Type

The column "Type" of the QC population browser consists of a comma-separated list. Possible entries are (language dependent):

  • ‘A’ = Accuracy

  • ‘P’ = Precision

  • ‘C’ = Combined (accuracy + precision)

  • ‘S’ = RMSD

  • ‘e’ = Population status Evaluation or Closed

  • ‘DMed’ = Population sub type is day median

  • ‘MAvg’  = Population sub type is moving average

  • ‘Qt’ = Qualitative choice list

Detail box

The detail box shows the QC choice list, the severity, the choices and there severity. Example:

[Qualitative]: Choice list = QCChoiceListPositive (100), Choices: Positive (10), StrongPositive (20)

QC audit query

The QC audit query option "Type" allows to select only qualitative audit reports.
