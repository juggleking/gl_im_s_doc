Quality control -> Configuration: QC shifts

GLIMS allows to define for 'shifts' for quality control.  In several places in GLIMS, it is possible to use the 'one result per shift' concept.  Examples: 

  • Only take one result per shift into account when performing certain QC checks
  • Only use one result per shift in QC printout (both for calculating the statistics and selecting the printed results)
  • Only use one result per shift when calculating statistics during close of evaluation period controls.

The default shift is 1 day, starting at 00:00 and ending at 24:00:00.  The shift is shown in de QCPopulation.Results( ) browser in the form of a mmddss number (mm=month, dd=day, ss=shift within day).

The shift of a result is calculated via the use of an optional result-based site function referred to by SpecificSite.ShiftScheme, which has to return the start times of the shifts of the day on which the result is measured, and return these start times in a format like "02:00,10:00,20:00", meaning that we have 3 shifts on this day, starting at 02:00, 10:00 and 20:00.

The only guaranteed fields when executing this site function for a new QC result are Result.AvailabilityTime and Result.Department.  Once calculated, the shift of a result is stored in Result.Shift (only when a Shift Scheme was specified!)  When a result's time lies before any of the shifts returned from the site function, we assume it belongs to the last shift of the day before. 

Note that GLIMS has to be restarted when changing the ShiftScheme site function.
