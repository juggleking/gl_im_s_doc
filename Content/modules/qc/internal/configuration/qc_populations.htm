Quality control - QC populations

Contents: Introduction - Access to QC populations - Fields - Precision check - Accuracy check - Combined check - Copy - Activate/Deactivate

Introduction

As each QC lot might be used for reference assessments of different properties, the respective results and statistics are gathered in different QC populations. A population consists of an entire set of results. Each population is associated with a QC lot and with an assessment method.

During the initial QC configuration, you have to create the QC populations manually using the contextual function QC populations on QC lot. Later, when a new QC lot is to be used, you can create a new QC lot using one of the contextual functions:

  • Create new QC lot on a QC lot (allows to indicate whether to copy all related QC populations or only the active or inactive ones).
  • Create new lot on a QC lot (automatically copies all related QC populations).

The fields in the Precision and Accuracy sections of the Configuration tab folder allow to directly define one accuracy and precision check in the QC population, without having to define separate QC checks. For multiple checks, you will have to define separate QC checks.

Access to QC populations

QC populations can be accessed by

  1. the QC population query
  2. using the contextual functions on QC lot or Assessment method.
  3. from a result context. The contextual function QC Populations on Result shows a browser of all active QC populations for the assessment method the result is referring to. When calling Result.QCpopulations() from one of the confirmation or validation browsers, the focus has to be positioned inside a specific result value (and not on a property name!).

Note

GLIMS uses the precision settings of the QC population's assessment method to limit the number of significant digits that are displayed:

  1. The assessment method's QC precision + 1 determines the number of significant digits that are displayed.
  2. If the QC precision is not specified, the assessment method's Precision + 1 is used to determine the number of significant digits.
  3. If the Precision is not specified, the Reported decimals / Decimal threshold of the Property of the assessment method linked to the QC population is used to determine the number of decimals (i.e. digits after the decimal separator) that are displayed.

To avoid rounding errors, the precision applied is one more (+1) than when it is applied to the result values of the QC population.

Median (Percentile 50) / Percentile 25 / Percentile 75

In statistics, a percentile is the value of a variable below which a certain percent of observations fall. In GLIMS, the observations are the results. As in the GLIMS statistics module, percentile values are calculated as follows:

The pth percentile is obtained by first calculating the rank (formula: l = p(n+1)/100), rounded to the nearest integer and then taking the value that corresponds to that rank.

Example

Take 10 values = 120, 125, 125, 145, 145, 150, 150, 160, 170, 175

Then P25 => 0.25 * (10 + 1) => 2.75 => 3, corresponds with 125

Then P75 => 0.75 * (10 + 1) => 8.25 => 8, corresponds with 160

Note: Round of 1.5 yields 2

The Percentile and Median values are only calculated when using the QC population function Recalculate statistics.

To inform the user about outdated (*) percentiles or median values, a mark is added to the browser (a non-labeled column at the left shows a '*') and the editor (a '*' mark appears next to the percentiles and median values).

In the browser that is embedded into the QC population editor, the row with the Tukey values can also display a '*' mark as these Tukey values are based on the percentile and median values. Note that the outdated values displayed are the ones stored in the QC population at the time the last calculation took place.

(*) Outdated means that a QC result was added or discontinued after a calculation took place (or when no calculation ever took place for a QC population).

Note

GLIMS uses the number of decimals (i.e. digits after the decimal separator) of the QC population's calculated Mean to display the percentile 25, median and percentile 75 values. Click here for more information on how the calculated mean value is formatted.

Technical notes

  1. The statistics recalculation stores these 3 values (Percentile 25, Percentile 75, Median) in the QC population.
  2. Moreover, the recalculation also stores the time the calculation took place in the QC population field Statistics calculation time.
  3. When a QC result is added or removed (discontinued) from a QC population, GLIMS will also set the QC population field Last QC result time.

Standard / Tukey table

This table shows calculated values using the Standard and Tukey method.

Standard SD

Calculated standard deviation (SD) of all numeric results:

Formula: SD = SQRT(SUM(Value - Mean)**2) / (Count - 1))

Note

GLIMS uses the number of decimals (i.e. digits after the decimal separator) of the QC population's calculated Mean to display the Standard SD value. Click here for more information on how the calculated mean value is formatted.

Tukey SD

The Tukey method is an alternate method for calculating a standard deviation (and derived fields), which is less vulnerable for outlier values.

The standard deviation according to Tukey is calculated as follows:

Formula: SDTukey = | Percentile75 - Percentile25| / 1.349

Note

GLIMS uses the number of decimals (i.e. digits after the decimal separator) of the QC population's calculated Mean to display the Tukey SD value. Click here for more information on how the calculated mean value is formatted.

Standard CV (%)

Formula: Standard Deviation / Mean * 100

Tukey CV (%)

The coefficient of variation according to Tukey, calculated as follows:

Formula: CVTukey = 100 * (SDTukey / Median)

Standard Bias (%)

The Bias  or systematical error, calculated as follows:

Formula: Bias = (|QCPopulation.TargetValue – Mean| / QCPopulation.TargetValue) * 100

Tukey Bias (%)

Same as Standard Bias, but using the Median:

Formula: BiasTukey = (|QCPopulation.TargetValue – Median| / QCPopulation.TargetValue) * 100

Standard Cpk

Represents the "Process capability" (ability of a process to produce output within specification limits).

Formula: (TEa - Bias) / (3 * CV)

Tukey Cpk

Same as Standard Cpk, but using the Tukey values:

Formula: CpkTukey = (TEa - BiasTukey) / (3 * CVTukey)

Standard sigma score

The six sigma score is calculated as follows:

Formula: Sigma score  = (TEa - Bias) / CV

Click here for more information on six sigma.

Tukey sigma score

Same as the standard Sigma score, but using the Tukey values:

Formula: Sigma scoreTukey  = (TEa - BiasTukey) / CVTukey

Click here for more information on six sigma.

Precision

Target mean / Target deviation

The Target mean and the Target deviation are used to quantify the method's precision.

When these precision fields are specified, a precision check will be imposed on each individual QC result. This precision check verifies if the result ranges between the 2S and 3S limits (see Severity 2S and Severity 3S).

Limit 1 / severity and Limit 2 / severity

These limits allow defining alternative SD factors with related severities (in addition to the fixed 2S and 3S limits, see below).

Example

A Maximal SD Factor 1 of 2.3, sets a limit to 2.3 times the QCPopulation.TargetDeviation.

Note

  1. Limit 2 should be higher than Limit 1.
  2. When Limit 1 and/or Limit 2 are defined with a severity, these limits are checked upon QC result creation or update. They will generate a QC result violation message analogue to the Severity 2S and Severity 3S. Obviously, the highest severity of the violated precision limits 2S, 3S, Limit 1 and Limit 2 is considered.
  3. When Limit 1 and/or Limit 2 is specified, these custom limits will be used to set the zones and labels in the Levey-Jennings graph.
  4. GLIMS first verifies if a Limit 2 severity or Limit 1 severity is specified. If so, the result value is verified against Limit 2 and Limit 1. If not, the result value is verified against 3SD and 2SD (in that order, and if related severity is specified).
  5. The QC population editor will make the fields Severity 2S and Severity 3S insensitive whenever one of the following fields are filled in: Limit 1, Limit 2, Limit 1 severity, Limit 2 severity. It is an indication to the user that those fields will no longer have effect.

CV max. (%) / severity

When executing the QCPopulation.Print function (which also creates the QC audits) GLIMS verifies whether the coefficient of variation (CV) does not exceed the maximal coefficient of variation (CV max). If exceeded, the CV max severity is applied.

Formulas:

CV = Standard Deviation / Mean * 100

Standard Deviation: calculated standard deviation (SD) of all numeric results:  = SQRT(SUM(Value - Mean)**2) / (Count - 1))

Mean: calculated mean value of all numeric results: Mean = SUM(Value) / Count

Delta SD limit / severity

These fields allow to define the delta deviation between QC results.

Delta SD limit

Delta SD factor limit between two subsequent QC results.

Delta SD limit severity

Severity when delta SD limit is violated.

When a new result value is added to a QC population and these fields are specified, GLIMS will check if there is a (non discontinued) result available prior to the new result.

If the absolute difference between the 2 values divided by the QCPopulation.TargetDeviation exceeds the Delta SD limit, GLIMS will apply the Delta SD limit severity and add a deviation text to the internal comment.

Severity 2S / Severity 3S

2S limit: determined by Target mean and twice the Target deviation.

3S limit: determined by Target mean and three times the Target deviation.

The 2S and 3S severity allow assigning a severity to the QC results which do not comply with the check described above. See also Deviation actions.

The 2S and 3S limits are not applied when one of the Limit 1 / severity and Limit 2 / severity are specified (see above).

Note

Certain instruments can send additional QC population-related data to GLIMS. GLIMS can update the QC population based on the target and deviation values received from the instrument. For more information, see Stations.

Maximal R

When executing the QCPopulation.Print function (which also creates the QC audits), GLIMS verifies whether the maximal inaccuracy (Maximal R) is not exceeded. The inaccuracy (R) is the difference between the calculated mean and the target value divided by the target value.

Pivot

This has been introduced for the German QC norms. Its main purpose it to make sure that the allowed deviation of a result for an accuracy or a combined control does not fall below a minimum value, even when the target value of the control is very small (normally, the allowed deviation is expressed as a percentage of the target value).

When the target value of a population is lower than the pivot of the population, then the allowed single result deviations from the target value, the allowed imprecision (CV max check) and the allowed inaccuracy (R max check) of a selected period are interpreted with absolute limits instead of percentages.  Fundamentally, for CV-max, R-max and MaxDev, instead of interpreting these as a percentage, they are interpreted as absolute values, derived from the percentage value by calculating the value of the limits on the pivot level instead of the target value level.

Example

Unit = nmol/l
TargetValue = 40
AccuracyLimit = 20%
CVMax = 5%
RMax = 10%
Pivot = 50

Then

the allowed limits for the single results are 30 - 50 nmol/l (and not 32 - 48!),
the maximum allowed SD for a given period is 2.5 nmol/l (and not 2.0 !),
the maximum allowed inaccuray Mean - TargetValue for a given period is 5 nm (and not 4 !)

In this case, the QCPopulation.Print() and QCAudit.Print() functions will show the absolute numbers on the printout and not the relative numbers.

Precision check

The precision check can be used instead of defining separate Westgard checks. 

Precision check for each individual QC result

The Target mean and the Target deviation are used to quantify the method's precision. They are expressed in the unit of the property. The values of the target mean and target deviation can typically be determined by means of an evaluation period.

If the standard deviation of the QC results matches the target deviation, the method is precise: it delivers reproducible values. 

When these precision fields are specified, a precision check will be imposed on each individual QC result. This precision check verifies if the result ranges between the 2S and 3S limits.

  • 2S limit: determined by Target mean and twice the Target deviation
  • 3S limit: determined by Target mean and three times the Target deviation

The Severity 2S and Severity 3S allow assigning a severity to the QC results which do not comply with the check described above. See also Deviation actions.

Precision check for a set of QC results

In addition to the individual precision QC result checks, GLIMS also allows to perform a general precision check on a set of QC results. This check will be executed when using the QCPopulation.Print function (which also creates the QC audits) and verifies whether the coefficient of variation (CV) does not exceed the maximal coefficient of variation (CV Max). The coefficient of variation is calculated as follows:

CV = Standard Deviation / Mean * 100

Standard Deviation

The calculated standard deviation (SD) of all the numeric results:  = SQRT(SUM(Value - Mean)**2) / (Count - 1))

Mean

The calculated mean value of all the numeric results: Mean = SUM(Value) / Count

QC audit reports for precision checks will only mention the CV Max check results.

Accuracy check

Accuracy check for each individual QC result

The accuracy fields allow to define one accuracy check. When specifying a Target value and an Accuracy limit, the Accuracy low limit and Accuracy high limit will be calculated automatically. When entering a fixed low and high limit, the target value will be calculated as the mean value of these two limits. The target value and target mean can  be obtained via reference method, averaging, ...

The Accuracy severity field allows assigning a severity to the QC results which do not comply with the check described above. See also Deviation actions.

Accuracy check for a set of QC results

In addition to the individual accuracy QC result checks, GLIMS also allows to perform a general accuracy check on a set of QC results. This check will be executed when using the QCPopulation.Print function (which also creates the QC audits) and verifies whether the maximal inaccuracy (Maximal R) is not exceeded. The inaccuracy (R) is the difference between the calculated mean and the target value divided by the target value.

QC audit reports for accuracy checks will only mention the Maximal R check results.

Combined check

In addition to the precision and accuracy QC types, GLIMS also supports 'Combined' QC populations. A combined population is a population intended to monitor accuracy and precision simultaneously by introducing a 'Maximal inaccuracy' next to the already existing 'Maximal CV' which can be seen as a 'Maximal imprecision'.

A combined control is recognized by the fact that it has a target value, a maximal CV AND a maximal inaccuracy. The maximal inaccuracy (Max R) could be interpreted as the link between the precision and accuracy checks, thereby explaining the term 'Combined'.

A classical accuracy control is detected by the fact that it has a target value or an accuracy limit or an accuracy low or high limit. A classical precision control is detected by the fact that it has a target mean or a target deviation. A control can be simultaneously of type precision and accuracy, without being of type combined (i.e. maximal inaccuracy is not checked).

Combined controls are typically used as follows:

  • Each separate result should fall within the accuracy limits.
  • At the end of the month (or any other period including at least 15 results), a QC audit is made for the results of that period. The 'imprecision' (CV), defined as the coefficient of variation of the selected results should not exceed the maximal imprecision (CV Max), and the 'inaccuracy' (R), defined as the difference between the calculated mean and the target value divided by the target value should not exceed the maximal inaccuracy (Rmax).

QC audit reports for combined checks will mention the Maximal CV check results and the Maximal R check results.

Copy

The contextual function Copy on QC population allows copying one or more QC populations, including their QC checks, while optionally modifying the QC lot, the station, the property and the evaluation status. The new populations are created inactive, unless the Activate new populations option is set. The old populations' status is left unchanged, unless the Deactivate old populations option is set.

Activate/Deactivate

To activate or deactivate a QC population, two contextual functions are available on QC population: Activate and Deactivate. These functions can also be used in a command.

Note

GLIMS only allows one active QC population per assessment method and QC lot. Activating one QC population automatically deactivates all other related QC populations. 

This is why the user will be warned when activating multiple related inactive populations at once. If the user wants to continue, GLIMS will activate only the most recent inactive population in the set of selected related populations. This is also is the default behavior when using the function in a command.
