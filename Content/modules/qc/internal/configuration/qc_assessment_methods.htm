Quality control - Assessment methods

Introduction

You can open the Assessment method browser using the context menu/ribbon of properties or stations. Click then on the QC tab.

General fields

QC norm deviation count

Indicates how the norm for QC results of precision controls in result browsers should be calculated: typically 2 or 3 to show the 2 or 3 sigma limits of the QC population. Or 3.5 for Westgard rules.

Please check the chapter Reference range for QC results for more information.

Required start qc / Required end qc / QC cycle length / QC cycle phase / QC phase position

These fields can be used to implement an automated channel quality monitoring system.

Last QC time - QC validity

Each time a QC result enters the system, the time is recorded in the assessment method's Last QC time field. Each time a patient result enters the system, the Last QC time and QC Validity are compared. When the Last QC time is longer than <QCValidity> hours ago, the Quality field of the assessment method (and the Channel quality of the patient result) is set to Unreliable.

Note

When a moving average result or a median result is entered, the Last QC time of the Assessment method will only be updated if the result causes the channel quality to be set to Unreliable.

Both fields are also shown in the Procedure.ChannelStatus() browser and editor.

Patient day median

Median result count

Number of patient results after which the calculation of a median should take place again, and thus after which another QC result should be added to the QC population.

Low limit / high limit

If the value of a median lies below / above the values indicated in these fields, a severity is applied to the QC result which was added to the QC population upon calculation of that median.

Limit severity

Severity applied to a QC result if the median (calculated on the number of patient results indicated in Median result count) lies outside the limits defined in the low / high limit fields.
Limit severities are used to define the colors of the QC cells in the browser of unprocessed ended stations runs.

Alarm severity

The alarm severity is relevant when the recalculate statistics function is used. This function allows the calculation of a median on all the QC results (i.e on all the medians calculated on a number of patient results) added to the QC population during a defined period. The calculation of this "global" median leads to the addition of a "global" QC result to the QC population.
If the global median lies outside the limits defined in the low / high limit fields, an alarm severity is applied to the global QC result.
Alarm severities are used to define the colors of the Median cells in the browser of unprocessed ended stations runs.

 

Example

Example

Median result count: 3

High limit: 65

Low limit: 30

Limit severity: 20

Alarm severity: 30

Measured patient results: 52 - 60 - 71 - 74 - 80 - 78

 

Every time 3 patient results have been measured, a median is calculated:

  • Median of the first three patient results : 60
  • Median the last three patient results : 78

The median of the first three patient results lies between the defined low and high limits: no limit severity is applied to the QC result added to the QC population.

The median of the last three patient results exceeds the defined high limit: a limit severity of 20 is applied to the QC result added to the QC population.

Now, the Recalculate statistics function is used and the median of all the QC results (60 and 78) is calculated: 69.

This global median exceeds the defined high limit: an alarm severity of 30 is applied to the global QC result added to the QC population.

Please also check the chapter Quality monitoring via patient median calculation for more information.
