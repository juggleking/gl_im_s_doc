Genetics (former features)

Introduction

This page describes the obsolete features in GLIMS that can be used to support genetics. This features are independent of the new Genetics module.

Using anatomic pathology configuration and routine screens

When implementing genetics in Glims, you can use the configuration and routine screens that were originally designed to support anatomic pathology in Glims.  For more information, please check the chapter "Anatomic pathology".

The pathology screen can be used in a more adapted way to genetics labs.  To activate this, the user must be logged on in a department for which the field "Specialty" is set to "Genetics".

When this is the case, the pathology screen (now 'genetic work screen') will behave different in the following ways:

In the tree view section, it will show all specimens contained in the order, even if they are not root specimens (in pathology mode, only root specimens are shown).

  1. When clicking on a sub-specimen (non-root-specimen) in the tree view section, the result section will show all results that were directly derived form this sub-specimen (also known as 'work specimen'). When clicking on the root specimen, these results will not be shown there.
     
  2. The buttons 'Add block(s)' and 'Add slide(s)' are not visible, since they are typically used for anatomic pathology.
     
  3. A button 'Add request(s)' is visible in the screen's toolbar. This button performs the method 'Specimen.AddRequest()' on the specimen that is currently selected in the tree view section (root specimen or sub specimen).
     
  4. Some buttons that are available in the contextual ribbon (at the top of the GLIMS window) are 'inherited' in this screen's toolbar. The concerned buttons are:
     
    • buttons that have as base table one of the following: Specimen, Procedure, Result, PathologyExam, Property, Object, HealthCareProvider, Order.
    • buttons that have a valid 'image up' icon.
    • buttons for which the user is privileged.
       
  5. Results will be grouped by the action of which they are output.

By default results in the pathology screen are displayed in the following sequence:

  • first by the sequence number of the procedure (which they are output of)
  • then by the sequence number of their procedure output
  • then by creation time

In some cases, e.g. if the same procedure and the same tests are present multiple times for the same specimen (e.g. in genetics), this may look confusing.

Example (with 2 actions for procedure 'Proc1'):

Proc1  - Test1      (from Action1)
       - Test1      (from Action2)
       - Test2      (from Action1)
       - Test2      (from Action2)
Proc2  - ...

Therefore, when the pathology screen is used as a genetics work screen, results will be grouped by their 'Action' before grouping by their procedure output. The example output above would then become:

Proc1  - Test1      (from Action1)
       - Test2      (from Action1)
       - Test1      (from Action2)
       - Test2      (from Action2)
Proc2  - ...

Foetus

GLIMS allows to mark a person as a 'foetus' (an unborn person). This is can be useful when tests are being performed on a pregnant mother or on her foetus.  When this person has been born (the birth has taken place) successfully, it is possible to create a new person for this, which of course will not be marked as 'foetus'.  In this case it can be interesting to link this born person to the original 'foetus' person. 

Click here for more information on how to configure this in Glims.
