HLA-antibody

The HLA-antibody table is available from the main menu via Start -> HLA -> HLA-antibodies.

  • Seq no: defines the sequence of the HLA-antibodies within the HLA-antibody browser

  • Name: name of the HLA-antibody

  • Description: free description field

HLA-antigen

The HLA-antigen table is available from the main menu via Start -> HLA -> HLA-antigens -> Browse.

  • Name: named of the HLA-antigen

  • Sequence number: defines the sequence of the HLA-antigens within the HLA-antigen browser

  • Description: free description field

  • Parent: displays the parent antigen in the hierarchical antigen tree (see below)

  • DNA: allows to indicate if the antigen was assessed with DNA methods (Yes/No)

Person relation

Introduction

The PersonRelation table allows to define multiple relations between any 2 persons in the Person table. A Person Relation record connects a Person with a related person, while specifying the relation type, the notion if the related person is a donor for the original person, and an optional comment.

Creating a person relation

From the Person contextual menu, click Relations -> Browse to display all relations of a person.

The person relation browser shows:

1. The type of relation
2. An indication to mark those relations that are no genetic (family) relations; they contain "N" in this column.
3. An indication to mark those relations that are indirectly calculated (X)
4. The externalization of the related person.

Other browser characteristics:

- If the person has a mother in the 'Mother' field, it is also shown
- If other persons (the children) have this person in the 'Mother' field, they are also shown
- If the person has a person in the 'Before birth' field (to indicate the same person as foetus), it is also shown
- If a person exists with this person in the 'Before birth' field, this person (the non-foetus) is shown also
- For each relative (family member), the persons related as family to those relatives are also shown.  The 'Type' field in the first column is then preceded by the name of this relative (see screen above)
- F6 on the whole record zooms in on the relation record.  F6 on the person's name zooms in on the person itself.

Hit Insert to add a new relation.

  • Is: select the related person

  • As: select the relation type

  • Since-until: Some relation types are not permanent.  For this purpose, a Start and End date are provided. 

  • Genetic relation: automatically set depending on the relation type

  • Comment: free text comment

  • Automatically created: when creating a new relation, a record for the reverse relation is automatically created (e.g. A is child of B implies that B is parent of A). These automatically created records are marked as such.

The person relation type is only supported for manual interactive consultation and maintenance and can not be used for reporting or order entry purposes.

Get a list of related persons via MISPL

It often happens -especially e.g. in HLA and genetics labs- that from within MISPL a list of related persons has to be built, including also some attributes or properties
of these persons.  To do so, you can use the MISPL function 'RelationsOverview' on the 'Person' table.  This function delivers in 1 call a text that contains all wanted related persons, with all there wanted attributes.

It expects 3 parameters:

1. 'TextMnemonic' (mandatory):

This is the mnemonic of another text module that should be based on the Person table.  This other text module will be evaluated for each related Person, and the final result text will be this text, repeated for each related person. So you have to configure once in that text module all the attributes that you want to see for one related person.

Additionally this text module is allowed to contain the variable {RelationType}, which will be expanded automatically into the textual description (in the appropriate language) of the type of relation that this person has to the 'main' person; this can be a string like 'Partner of daughter of nephew'.  A very simple example of such a text module could be (showing the relation type and the person's default externalization):

{RelationType} : {=.Externalization}

2. 'Type' (optional)

This parameter puts a restriction on the Type of related persons. If specified, only relations that satisfy the restriction will be returned. This restriction can be:

  • The name of a particular PersonRelationType: e.g. "Parent" will return only the parents of the person;

  • "Family" will only return the relations which are family, not e.g. blood donors;

  • "No-family" will only return the relations which are not family-bound: e.g. blood donor relations will only appear with "No-family".

3. 'Recursive' (YES or NO)

If NO, only the direct relations of the main person will be listed.  If YES, the persons that are not directly related to the main person, but that are related to 'related persons', will also be listed. 

So if A is related to B and B is related to C and C is related to D, then:

  • if 'recursive' is NO, only B will be listed as relation of A;

  • if 'recursive' is YES, not only B, but also C and D will be listed as relations of A.

Note that the list of related persons will also contain:

  • the mother: the person that is specified as 'mother' in the 'mother' field of the person.

  • the children: all persons that have this person as 'mother' (in their 'mother' field)

  • the person before birth: the 'fetal' person that is specified as the person 'before birth' of the person.

  • the person after birth (only for a foetus): the (born) person that has this fetal person in their 'before birth' field.

The persons appearing in the list will be sorted in the following sequence:

  • by 'level': first direct relations of the main person, then relations of those people, etc.

  • within each level: by relation type (first parents, then children, then siblings,...)
