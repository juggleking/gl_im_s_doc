Genetics -> foetus

Contents: Introduction - Configuration - Usage

Introduction

In GLIMS a person can be marked as a 'foetus' (an unborn person). This can be useful when tests are being performed on a pregnant mother or her foetus. When this person has been born (the birth has taken place) successfully, it is possible to create a new person, which of course will not be marked as 'foetus'. In this case it can be interesting to link this born person to the original 'foetus' person.

Configuration

In the Person editor, the following fields are important to mark a person as a foetus:

Foetus on creation

A person can be marked as a 'foetus' (an unborn person). This is can be useful when tests are being performed on a pregnant mother or her foetus. If this option is enabled, the field Before birth is hidden.

Before birth

Reference to the person record which represents the same person but as a foetus. You can only select persons of which the field Foetus on creation is enabled.

Mother

For obvious reasons, a foetus must always have a reference to its mother.

Usage

Navigate between the foetus and the born person

Via the Person contextual menu

Navigate from the one to the other via the Person contextual menu on the Person record: Relations -> Person before birth and Person after birth.

Via MISPL

Within a MISPL it is possible to navigate from the one to the other using the functions Person.BeforeBirth and Person.AfterBirth(). This way it is for instance possible to include in a patient's report or data sheet the test results or attributes of the foetus, e.g. for comparison.

Consultation in the relation browser and relation tree view

Via the Person contextual menu, choose Relations -> Browse or Outline.
