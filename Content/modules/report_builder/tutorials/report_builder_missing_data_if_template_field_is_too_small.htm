Report Builder - Missing data if template field is too small

Problem description

When designing Report builder templates using Jaspersoft Studio, you can easily drag fields onto the layout. It is important to note that the "Stretch with overflow" option is turned off by default for all fields that you add to the layout.

This has as side-effect that if the field is not large enough to hold the entire content, the part of the text that does not fit is excluded from the report output. As a result, essential parts of the data may be missing - such as the last word of a long description.

We strongly advise that customers review all their .jrxml templates and turn on the "Stretch with overflow" option on all fields whose content may exceed the provided space.

Solutions

There are different solutions in Jaspersoft Studio to ensure that content is fully displayed.

1. Enlarge the field

The simplest solution is to enlarge the field, provided you have enough space on the layout. The field should be able to display the maximum number of characters.

2. Use the option "Stretch with overflow"

If no space is available to enlarge the field or the maximum number of characters is uncertain, use the option "Stretch with overflow" to vertically expand the field.

As you can see in the example above, the elements below the "stretched" element now shift down, but the field label (BIRTH DATE) does not (as it is not positioned below the "stretched" element). Depending on the actual template, different solutions can be applied:

1. Add an invisible line

By adding an invisible, floating line (line element with pen width "0"), ALL elements positioned below that line will shift down. To ensure that the float mechanism works correctly, the different elements MUST NOT coincide. Coinciding elements are identified by green markers.

2. Span the field label element

This solution is typically used in case of rows each having a field label and corresponding text element. Drag the end point of the field label element towards the end of the text element. This has the same effect as adding an invisible line.

3. Group elements in rectangle

For each row containing elements that can expand vertically, create a frame element holding all elements in that row.

Additional note regarding differences between Windows and UNIX

The Report builder in GLIMS asks the underlying operating system to calculate the space needed for a block of text, given the font name, style and size. Unfortunately, there may be a difference between the dimensions as calculated by Windows, and the dimensions as calculated by UNIX, for the exact same block of text with the exact same font definition. For instance, Windows may claim the block of text "This is a test" needs 80x20 pixels, while UNIX may claim this block of text needs 82x20 pixels.

As a result, a block of text that fits into a field when the report is generated on Windows, may not fit into that same field when the report is generated on UNIX. If you are planning to generate reports on both platforms, you need to thoroughly test your templates on both platforms, and be prepared to modify your templates in order to compensate for these differences.
