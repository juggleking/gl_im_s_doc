Report Builder - Blood transfusion forms

Introduction

Blood transfusion forms can be exported in an XML format which can be converted to a PDF document using Report Builder.

Output options

The transfusion form printing screen allows to export the form to XML or PDF. The form can be sent directly to a printer or can be saved as a file on a specified directory with a specified file name.

Configuration

In order to be able to use PDF or XML, the text module in the "Form" field must follow a specific structure, as illustrated here:

The following tag must be present on the first line: <?GRB?>

If so, the "Format" option will contain "XML" and "PDF" as possible output formats:

If no form is specified, then all type of export formats can be chosen. Reason: when no form is specified, the form defined on the corresponding blood product is taken. As GLIMS does not know which form will be selected (specific site form or default form from blood product), all output formats are possible when the form is "?".  

This has as a consequence that the following situation is possible: the form does not indicate <?GRB?> but PDF output is chosen or vice versa.  So the program has to make a decision and the form configuration will have precedence over the selected output format.  All the possible decisions are documented below:

Output options Form content Final output
Text <?GRB PDF output (form overrides user’s choice)
  No <?GRB Text
     
XML <?GRB XML
  No <?GRB Text (form overrides user’s choice)
     
PDF <?GRB PDF
  No <?GRB Text (form overrides user’s choice)

When multiple records are selected to be printed at once then the same rules are applied.  For every blood selection, the output format is determined as described above and the corresponding file is generated.

Layout template file

As shown in the example above, the <?GRB?> tag can have an extra attribute "path".  This is the location of the Jasper layout template file. If no such path is specified, GLIMS will look for the template as follows:

  1. Template file specified in the user's department

  2. Template file specified in general options (specific site)

  3. In the application directory "templates/examples/BTF/BloodTransfusionForm"

If the layout file specified in the text module is used, then it is indicated in the status bar:

Remarks

The configuration tags <?GRB?> or <?GRB  path=”C:\temp”?> must be defined correctly.  Any leading or additional spaces between "<?GRB" and "path" will be ignored and "path" can be in upper or lowercase but any other divergence will lead to a text output !
