Report Builder - Financial reports

Introduction

GLIMS allows to print invoices, invoice summaries and financial shipments via Report Builder.  This functionality is implemented through the "Print" function on Invoice, Invoice summary and Financial shipment.

Configuration

XML output

The XML data that serves as input for the Jasper template must be built using dynamic text modules in the billing document template configuration.

The exported XML (e.g. text specified in the invoice's first page header) must start with :

<?GRB Path=”JasperLayoutFileName=xxx” BaseXPath=”yyy”?> 

If so, the document will printed with Report Builder.

  1. Path: path to the Jasper template file to be used
  2. BaseXPath: corresponds with the XPath expression of the Jasper report

Other options

Other important billing document template fields:

  1. To avoid invalid XML output, enable the option "Suppress form feeds" and disable the option "Add form feed"
  2. Set the field "Page length" to "?"

Print screen

The print screens allow to

  1. print the documents immediately on a specified printer
  2. generate a PDF file (with or without preview)

Jasper template

A Jasper template must be defined with Jaspersoft Studio to process the XML output.
