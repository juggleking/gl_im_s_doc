Report Builder - Result reports: validate Jasper template

For result reports generated with Report builder, the Report template option Generator is set to Glims report builder and a Jasper template file is specified.

The Report template based function Validate report template allows to check for potential problems in the Jasper template.

Warning

The Validate report template function performs some basic checks, but this does not guarantee a 100% flawless Jasper template.

Always thoroughly test the Jasper templates before using them in a production environment !

Validate a report template

  1. Select a Report template with Generator set to Glims report builder.

  2. Choose the contextual menu/ribbon item Validate report template.

  3. A summary will be shown with potential problems in the Jasper template.

Notes

  1. The validation summary will be empty when:
    • A Report builder service is defined but it is not running.
    • No Jasper template file is specified in the Report template configuration.
    • The Jasper template file specified in the Report template configuration cannot be found.
  2. The same validation checks are performed during report generation, but this time with the actual report data (e.g. GLIMS will not log a warning when the Stretch with overflow is not set if the actual report data does not exceed the text field limit). Warnings will be shown on screen or added to the log file.
  3. The Validate report template function checks both main and sub reports.
