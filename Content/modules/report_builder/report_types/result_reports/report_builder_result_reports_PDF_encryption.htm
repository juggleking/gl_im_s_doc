Report Builder - Result reports: PDF encryption

Description

Report Builder supports encrypted (password-protected) PDF files as email attachments.

Report medium

PDF reports will only be encrypted when they are sent as email attachments.  This implies that the "Report medium" option of the "Default report" is set to "Email".

Passwords

Introduction

An "owner" and "user" password must be defined in GLIMS. Only if both passwords are defined, can the PDF file be encrypted.

Owner password

The "Owner" password is only meant for internal use, as it governs the digital rights management of the PDF file.

It can be defined in the General options (Start -> System management -> Customize -> GLIMS general).

User password

The "User" password is the password that the report receiver has to enter in order to open the PDF file.

It can be defined in the Correspondent (i.e. the report target).
