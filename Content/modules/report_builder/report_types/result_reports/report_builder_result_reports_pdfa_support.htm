Report Builder - Result reports: PDF/A support

Introduction

PDF/A-1 is an ISO-standardized variant of the Adobe Portable Document Format file specification, specifically designed for long-term archival of electronic documents. It initially comes in two variants: PDF/A-1a and PDF/A-1b.

GLIMS Report Builder now offers the ability to export result reports in PDF/A-1 format. This development was driven by several market requirements, e.g. DMP in France or ELGA in Austria.

Configuration

Note that the options documented below have no effect on report templates with a generator other than "GlimsReportBuilder" and are ignored if PdfAConformance is set to "None".

Report template options

Author, Keywords, Subject, Title, PdfAConformance: for detailed information, please check the chapter "Report templates".

General options

The following options can be set via System management -> Customize -> MATE:

ICCProfileWindows: file path

This allows you to set the Windows path to the designated ICC color profile used for PDF/A generation.

ICCProfileUNIX: file path

This allows you to set the UNIX path to the designated ICC color profile used for PDF/A generation.

Known restrictions and limitations

PDF/A-1 is effectively a subset of the PDF specification, and contains an extensive list of mandatory and prohibited features. While not exhaustive, here's a short list of items to bear in mind:

PDF/A-1a and PDF/A-1b both require that:

  • An ICC color profile is provided
  • All fonts are embedded

PDF/A-1a and PDF/A-1b both prohibit that:

  • The documents contain "transparent" images or graphics
  • The document is password-protected or encrypted

PDF/A-1a requires that:

  • The document structure be "Section 508" compliant. For this, you need to apply the proper "PDF 508 tags" to the layout items using Jaspersoft Studio.

For a complete list of the PDF/A-1 requirements, please consult the ISO 19005-1:2005 specification:

http://www.iso.org/iso/iso_catalogue/catalogue_tc/catalogue_detail.htm?csnumber=38920

Known issues

Due to limitations in JasperReports, GLIMS Report Builder cannot automatically remove transparency from images and graphics. While MIPS will continue to investigate these limitations, users are advised to restrict PDF/A export to report templates and layout files configured without graphs or transparent images.
