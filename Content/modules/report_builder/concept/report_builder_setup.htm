JasperReports

To use the GLIMS Report Builder, no additional software installation is required. JasperReports is included in the GLIMS installation.  GLIMS currently uses JasperReports version 6.4.1, no other version is supported !

Java

Report Builder uses the Java Runtime Environment shipped with GLIMS 9.9.0 (JRE 1.8.0_181).

Jaspersoft Studio

To customize the example Jasper templates (.jasper files) provided by MIPS, you need to download and install the free Jaspersoft Studio application.

  1. Go to https://sourceforge.net/projects/jasperstudio/files/JaspersoftStudio-6.4.0/
  2. After downloading the appropriate installation file, a Wizard will guide you through the installation.
  3. To edit or compile the layout files in Jaspersoft Studio, you need to copy the "be" and "org" directories from "<GLIMS home>/jar/cls" to the directory with the .jrxml files.

Jasper templates

When new features are added for GLIMS Report Builder, an upgrade to a higher version of JasperReports can be required.  JasperReports is shipped automatically with GLIMS, but all custom Jasper templates (.jrxml files) need to be re-compiled for the new JasperReports version.

Automatically

As of GLIMS version 9.1.1, GLIMS will automatically check whether the custom templates have been compiled for the appropriate JasperReports version. If not, the templates will be recompiled automatically.

Using Jaspersoft Studio

To manually recompile your templates, open the main reports with the appropriate Jaspersoft Studio version and click the "recompile" button. All sub-reports will automatically be recompiled along with the main report.

PDF viewer

When using Report builder for screen reports, you need a PDF viewer.
