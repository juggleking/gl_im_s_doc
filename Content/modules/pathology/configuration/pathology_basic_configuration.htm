Anatomic pathology - Basic configuration

This document lists the different steps to be taken in order to complete the basic configuration of the pathology module.

  • Materials
  • Blocks and slides
  • Properties
  • Procedures
  • Pathology procedures
  • Request definitions (panel)

Materials

This is a reference to the input material, i.e. the material of the specimen used for the pathology procedure. Materials used for pathology examinations are stored in the material table. 

Blocks and slides

GLIMS also offers support for the "Block" and "Slides" concepts.  For more information, please check the chapter "Blocks and slides".

Properties

For all tests done or information that should be described in the pathology examination a property should be defined.  Some examples of properties that could be defined are: clinical info, material description, photo, tumor size, diagnosis code, etc.  These can also be mere informational properties.

Some of the fields displayed in the property editor are specifically designed for the pathology module:

Number of editor lines

Contains the number of lines (to enter text) that will be shown for that property in the pathology screen. Standard this field is set to 1.

Code system

For pathology examinations it is possible to define a coding system for a each individual property. That way different diagnosis code systems can be used at the same time. The 'AP-ICD10Code' property above will contain an ICD10 code.

How to create diagnosis code systems

From the main menu, click Start -> Diagnoses -> Diagnosis code systems an hit Insert to create a diagnosis code system:

  • Is default: not yet supported

  • Mnemonic: choose a mnemonic

  • Name: choose a name

Use the contextual menu function Diagnosis codes and enter the codes for the code system:

For each code, specify a mnemonic and a free text.

Procedures

The procedures should be defined analogous as in the regular chemistry. The procedure should get as input the material of the pathology procedure (see below) and as outputs the properties you've defined above. You can choose whether you want to add all your properties to one procedure or split your examination up in different parts and create a procedure for each part. The latter allows defining an enhanced structure for your examinations.  Click here for an example of a procedure structure for a pathology examination.  It is imperative that the material input of the these procedures is identical to the material specified in your pathology procedure.

Pathology procedures

For each type of pathology examination a pathology procedure should be defined.

General fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Description

Material

Material which is to be examined.

Domain

Allows classifying the pathology procedures. The domain is a 1-or-2-character indication (abbreviation) that can be used to group or classify the pathology procedures according to their type, usage, medical discipline, ...

Example

Cytology -> C
Cervical cytology -> CC
Histology -> H
Autopsy -> A

The list of possible domains must be configured in a site-domain with the name 'Domains' (this must be created manually). Specify the list of possible domain abbreviations as a comma-separated list in the field 'Value list'; e.g.: 'C,CC,H,A'.  If you want, you can also add longer names to this; e.g.: 'Cytology\C,Cervical cytology\CC,Histology\H,Autopsy\A'.  In the pathology query browser it is possible to select on pathology exams of a certain domain.

Staff page

Automatic determination

It is possible to specify a MISPL to automatically determine the responsible for a pathology examination.  This MISPL function should return a HC provider that is assigned as responsible in any of the employment records for the department (in which the order is entered).  Moreover, the returned HC provider should be accepted by the MISPL 'Allowed if' that is defined for the pathology procedure.

Allowed if

Extra filter to check the responsible for the pathology exam.

Determine employee

Please check the chapter User assignment.

Conclusion page

Conclusion property

This conclusion property will automatically be added to the order when a pathology examination is requested and will be displayed in the pathology work screen.

Triggers page

Not yet supported, for future use.

Other page

Specimen scope

Allows specifying that such a procedure requires a specimen with a specific scope. This allows making sure that a procedure or group of procedures gets its own specimen.

Example

If you request two pathology procedures (or a combination of regular procedures and pathology procedures) which would normally be able to use the same specimen, you can indicate that each procedure requires its own specimen by defining a different specimen scope for each procedure.

Default request definition

Request definition to be used when requesting this procedure in order entry.

Instruction text (SOP)

Allows to define a text that can be shown in the pathology work screen (using the question mark button).

Only slides

When set, the pathology module will be adjusted to only use slides and no blocks (e.g. for cytology).

For pathology procedures with Only slides enabled:

  1. The initial work status of a newly created pathology examination is Staining.
  2. The pathology examination's work status can only be set to Reading or Staining via the dedicated context functions.
  3. The Blocks > Generate context function cannot be executed for the pathology examination.
  4. In the pathology work screen,
    1. Only work status Reading and Staining can be selected.
    2. The Add blocks icon is hidden and the ALT-F1 key combination cannot be used.

Request definitions

In GLIMS, request definitions are automatically created for all properties and pathology examinations but to ease order entry one could define a panel request definition which contains all request definitions that are to be included in the pathology examination.

The first member of the panel is the default request definition of the pathology procedure. The request definitions refer to the properties of the 'regular' procedures.  When requesting a new examination you can now simply request the panel 'cervix-examination' to order the pathology examination and all the other properties defined in the panel.
