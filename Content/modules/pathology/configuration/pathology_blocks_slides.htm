Anatomic pathology - Blocks and slides

Introduction

In the anatomic pathology module it is possible to use the concepts 'Blocks' (or 'cassettes') and 'Slides'.  These concepts are realized using the GLIMS concept of 'Carriers', which is a combination of a certain 'Medium' and a certain 'Specimen'.

Configuration

Introduction

In GLIMS, you can link a material to a medium through via medium usage.

Medium

Because carriers are also used in microbiology, the field Organism type allows to distinguish between media of type Block, media of type Slide and media of type "Microbiology (germ, virus, parasite)".

In addition, you can group media of type slide in subcategories: The Type of slide field contains by default 4 possible values. These values can be adapted by the application manager. They should also be configured in a site-domain called "PathologySlideType".

Medium usage

In order to link Media to a certain Material, the 'Medium usage' table of that material should be filled with the appropriate entries. You can specify the number of carriers that should be generated automatically by default in the Count field. 

The Count field indicates how many carriers (block or slides) have to be made per medium.  

Note

When multiple MediumUsages are configured for the same material, the most specific (the one with a procedure and or eligible specified) is applied.

In order to link a Medium as 'child' (being derived) of another Medium, the Medium usage record has to specify the Parent medium for which it is applicable.

Carrier

Carriers of type Slide can also have a Parent carrier of type Block. This hierarchical structure will be visualized in the tree view of the pathology work screen. 

Specifically for anatomic pathology a field Internal id has been provided  in the Carrier table. This field will be automatically calculated for each carrier used as a block or slide.

This will be done according to the following scheme:

  1. Carriers at the highest level (without parent carrier) are numbered '1', '2', ... (numbering restarts for each specimen).
  2. Carriers that are derived form another parent carrier get their parent's internal id + "-" + a sequence number (numbering restarts for each parent carrier).

Example

When there are 2 blocks each having 3 slides, the blocks will get internal ids '1' and '2' and the slides will get internal ids '1-1', '1-2' and '1-3' (for the first block) and '2-1', '2-2' and '2-3' (for the second block).  The internal id of each block or slide will be visible in the hierarchical tree view in the pathology screen.

Generating blocks and slides

This chapter describes how the necessary carriers (block and slides) can be generated:

Automatically

When requesting a pathology examination, GLIMS will check which materials are needed.  If the material is linked to a medium (through a medium usage), GLIMS will automatically generate carriers for each medium (block or slide) linked to the material.  The number of carriers generated depends on the Count field of the medium usage record.

Through status transitions

  • When the pathology exam status goes up to Embedding, the function Generate blocks is automatically executed. This will add the default number of blocks for each medium, according to the configuration. If at least 1 block already exists, no more blocks will be added.
  • When the pathology exam status goes up to Microtomy, the function Generate slides is automatically executed. This will -for each block- add the default number of slides for each medium, according to the configuration.  If at least 1 slide already exists for a block, no more slides will be added for that block. 

Note

Whenever a block or slide is added, the pathology examination status goes down to "running" if necessary.  In case of blocks the substatus also goes down to "Embedding" and in case of slides it goes down to "Staining" (in case the status was higher of course).

Manually

contextual menu functions "Generate blocks/slides" on pathology exam

Using the contextual menu functions Generate blocks/slides on pathology exam, you can manually generate the configured blocks slides.  The  blocks/slides will be generated in accordance with the medium usage configuration.

Note

When Only slides is enabled for the pathology procedure, The Blocks > Generate context function cannot be executed for the pathology examination.

In the pathology work screen

In the pathology screen 2 buttons are available on the left side, one with a cassette icon and one with a slide icon.  These buttons can be used to manually add 1 or more blocks or slides (i.e. carriers for which the medium has type block or slide).  

You can also use key combinations to add blocks or slides (instead of using the dedicated buttons). The key combinations are:

ALT-F1 = add blocks
ALT-F2 = add slides

Note

When Only slides is enabled for the pathology procedure, the ALT-F1 key combination cannot be used.

Whenever slides or blocks are added, they are immediately visible in the tree view section of the pathology screen.

Adding blocks

On the left, the specimen is shown.

The list of media of type Block will be shown.

The user has 3 ways of adding blocks:

  1. Simply double-click on the medium type you want to add. This will add 1 block at a time.  
  2. Select the medium type in the list, specify the number of blocks to be added (if more than 1) and hit the Add block(s) button.  This will add as many blocks of the selected type as you requested.  
  3. Instead of hitting the Add block(s) button, you can hit the Add block(s) and close button.  This will do the same but the window will also disappear (same as hitting the Add block(s) button and then the Close button).

Also print labels

This option allows to print the block labels.

Notes

  • When the selected specimen is a pathology specimen, the other pathology specimens of the order are also displayed.
  • When the selected specimen is a non pathology specimen, no other specimen will be displayed.

Adding slides

When clicking the Add slides button, you should first select one of the available slide types (as defined in the site domain "PathologySlideType", see above).

After having selected the slide type, another window shows the possible slide media that have this selected slide type.  A selection field at the top allows changing to another slide type if desired.

On the left, you can select the block from which the slides should be derived. If you already selected a block in the tree view, the slides will automatically be added to that block.

The user has 3 ways of adding slides:

  1. Simply double-click on the medium type you want to add. This will add 1 slide at a time.  
  2. Select the medium type in the list, specify the number of slides to be added (if more than 1) and hit the 'Add slide(s)' button.  This will add as many slides of the selected type as you requested.  
  3. Instead of hitting the 'Add slide(s)' button, you can hit the 'Add slide(s) and close' button.  This will do the same but the window will also disappear (same as hitting the 'Add slide(s)' button and then the 'Close' button).

Also print labels

This option allows to print the slide labels.

Show blocks of all specimens

This option allows to not only show the blocks of the currently selected specimen but also those of the other specimens of the same order. This way, slides can be added to blocks of multiple specimens. When set, the displayed block name will be prefixed with a reference to the specimen. This setting is not saved in the user preferences.

Deleting blocks and slides

It is also possible to delete slides and blocks. This can be done with the DELETE key. 

Warning

  1. You cannot e.g. delete a block for which there are slides; you must first delete those slides.
  2. You can only delete the last slide on a block, or the last block on the specimen, not intermediate slides or blocks.

Via MISPL

A new function is available to add blocks to a specimen:

Specimen.AddBlocks(MediumMnemonic, NumberToAdd, NumberToReach, PrintBlockLabel)

MediumMnemonic

Specifies which type of block is to be added.

NumberToAdd

The number of blocks to be added to the specimen.  Cannot be combined with the parameter NumberToReach.

NumberToReach

Total number of blocks that should be available for the specimen.  If a number of blocks had already been defined for the specimen in the past, this allows to add a number of additional blocks in order to reach the desired total. Example specimen X already has 5 blocks.  If you set this parameter to "20", Glims will add 15 new blocks.

Cannot be combined with the parameter NumberToAdd.

PrintBlockLabel

Logical.  If enabled, the block labels will be printed automatically.

Confirm grafting of blocks and slides

Manually

Using the contextual menu functions Confirm blocks/slides grafted on pathology exam, you can set the processing date and time of each block that was not 'processed' yet.

Through status transitions

When the pathology exam status goes up to Microtomy, the function Confirm blocks grafted is automatically executed.

Printing block and slide labels

From each pathology exam, there are two contextual menu functions Print block labels and Print slide labels which will print the label defined in the medium. For more information on label printers, please check the chapter Block/slides labels in the Set devices page.

Query and browser for blocks and slides

Introduction

A query and browser specifically for blocks and slides is available.  It allows to make a selection on carriers used for anatomic pathology (blocks and slides).

Query

General and Specific page

All options that exist for querying pathology exams.

Blocks and slides page

Status

Not processed / processed / all.

Type

Only blocks, only slides or both.

Requested from/to

Allows selecting on the carrier's creation date/time.

Selection of media

A list of medium mnemonics; you can also use wildcards (*) here or you can also use CTRL-F or double-click to select the medium records from a list.

Filter

Carrier-based MISPL filter.

Browser

The blocks and slides browser contains following the columns:

Specimen internal id

No.

Number of the block or slide (within the specimen).

Sampled

Sampling date of the specimen or creation date of the carrier (switch with the cycle button).

Medium

Mnemonic of the medium.

Done

A mark ('V') indicating that the block or slide has been processed.

Processed

The date it was processed ('grafting date').

Comment

This column is editable. This means that you can enter or change the value directly in the browser.

Responsible

Notes

  • If a choice list is configured for the medium,  you can choose a value from this list by using CTRL-F or double-click. 
  • If the choice list doesn't allow free text, you can only enter a value from the list. 
  • If the choice list allows free text, you will be warned if you enter something else than a value from the list. This behavior is the same as in the carrier browser in the microbiology screen.
