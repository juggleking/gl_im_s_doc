Pathology work screen

Contents: Introduction - General information - Results section - View mode - Procedure selector - Image preview - Revision - Conclusion

Introduction

After having requested a pathology procedure in the order entry screen, a corresponding pathology examination is created. From this examination, the pathology screen can be called by double-clicking it.  When the pathology screen is displayed, you can use the Page Up and Page Down key to browse through the selected examinations.

You can also open the pathology work screen from a specimen or order using the contextual menu function Pathology > Open work screen.  Attention, when using the menu function from an order, and there is more than 1 pathology exam in the order, the screen will open on the first exam that is found in the order (in arbitrary sequence).

General information

The history button allows the user to consult historical results of this patient. Clicking this button allows to previous examinations, results or generate an overview by property.
Patient and specimen information

In the left upper corner, you will find object and specimen information. This information can be customized via the pathology general settings.

When hovering over these fields, the content will be displayed in the tool tip and when these fields get the focus, the editor will be enlarged so that the full content can be shown. When leaving the field it will be resized to its normal size. 
Responsible In the middle the responsible for the examination is shown and can be changed. Only HC providers that have been defined as responsible (employee assignment) can be selected.

You can also change the responsible for multiple examination simultaneously using the contextual menu function Change responsible on pathology examination.

Notes

  1. A responsible can only be changed when the status of the pathology exam is below "Validated-Complete".  If the selected exam has already reached the status "Validated-complete", an error message will be displayed.
  2. When the responsible is changed for the selected exam, the responsible for all the other exams within the same order will be also be changed (if the status is below "Validated-complete").  If one of these exams has already reached the status "Validated-complete", the responsible will not be changed for any of the exams.  No error message is shown in this case.
Status Indicates the current status of the examination.  If the status of a pathology exam is 'Running', its substatus will be displayed instead of its status.  Increasing and decreasing the substatus can be done in the pathology screen with the two curved arrow buttons left/right of the status indicator.

It can contain:

  • Plain text. (non formatted)
  • HTML (or XML)
  • You can also have e.g. a Word document displayed; to do this, put in the contents of the text module the followin HTML code:

<html>
<head>
<script type="text/javascript">
function newLocation()
~{
window.location="file://///MyServer/SOPdoc/InstructionsOnCutUp.doc"
}
</script>
</head>
<body onload="newLocation()">
<H1> You will be redirected ... </H1>
</body>
</html>

When hitting the instruction button on the pathology work screen, a window will open to show the contents of the text module, after it has been dynamically expanded.

Note

Each '{' sign that is meant for HTML coding has to be preceded by a '~' sign because else it will be interpreted by MISPL.

Results section

The results section

This is an overview of the results defined for the pathology examination. If there are more results than can be displayed in the overview a vertical scroll-bar will appear. All results can be filled in here. The results can also be confirmed here by clicking ; the confirmation image will be replaced by the validation image . Clicking on the validation image will validate the result. If you want to change a validated result, the status of the result should first be set back to 'Available' by clicking the button.

Note

When you click on this button, the result status "Validated" will be set to "Available".
  1. The result is thus not being Discontinued/Repeated!
  2. When this result is being used as Input for an Output(calculation)-result, the reset of the input result status by this button to "Available" does not repeat the output result. After value-change and confirmation of the Input value(s) it will lead to a re-evaluation of the output result, but the Output result still remains in its status. When the status of the output result was already Confirmed or Validated, only the availability-trigger will be executed on this Output result.

Note

If the user has no privilege to confirm/validate results, the white and green buttons will not be visible on the right side of the results.

For text results, its possible to enlarge the editor to type your findings by pressing 'F6'. When pressed once again a full-screen editor will be displayed. Its also possible to give results a color according to their status. For more information, please check the chapter on color schemes. A '@' after the label of the results indicates that this are auto-prompt results, i.e. not explicitly requested, but also outputs of the procedure (of which the Autoprompt field is set). A '*' after the label of the result indicates that this result has an internal, external comment or a request comment.

One can enter his findings by directly typing text, but it is also possible to add result or diagnosis codes. To do this press 'CRL-F'. Depending on how the property of the result is defined, result codes or diagnosis codes will be displayed. If 'allow multiple result codes' is checked for the property, one can select multiple codes at once. This way it is possible to create standard texts which will be used frequently. By means of MISPL, it is even possible to add some parameters to these codes.  Another (and far more faster) way to expand those codes in the editor is by typing the 'code' for the result or diagnosis code preceded by the 'expansion sign' which can be defined at the specific site preferences set.

If a language is specified for an issuer (in the Correspondent editor), and it is different from the user's language, the pathology work screen takes this into account:

  • All texts and result codes are expanded in the issuer's language instead of the user's language.
  • A clear indication (in red) appears at the top of the pathology work screen asking the user to write any free text in the issuer's language.

View mode

The tab pages at the top of the page allow switching the view mode of the pathology work screen. This view mode is stored as the user's preference and will be used by default the next time the screen is opened again.

Displays a tree-like overview of the specimen(s) and related carriers (blocks and slides).  The tree view not only shows the specimen to which the current pathology exam points, but also any other specimens that are contained in the same Order, except those specimens having their own pathology exam (another exam than the current exam).

After clicking on a certain specimen node or carrier node in the tree view window, the results window will display only the results on that specific specimen (initially this is the specimen of the current pathology exam).

The tree view also contains check boxes.  

  • For specimens, the checkbox is ON if the specimen has been confirmed (status 'Available').
  • For carriers (blocks and slides) the checkbox is ON if the carrier's grafting time has been set, i.e. the carrier (block or slide) has been 'processed'.

By clicking on a non-checked checkbox you can:

  • Confirm the specimen availability (on a specimen node).
  • Confirm that a carrier (block or slide) has been processed (on a carrier node).  If carrier comment is available, it is also shown (in brackets after the medium name).

Clicking on a checked checkbox will not perform changes (although the checkbox seems to clear) and makes no sense.

Procedure selector

Procedure

If several procedures are defined, the procedure selector allows you to select one of them and only the results belonging to that procedure will be shown in the results section. If results are requested which do not belong to a procedure (e.g. informational properties), a dummy procedure "(...)" will be displayed in the procedure list. When selecting this dummy procedure, only those results without a procedure will be displayed.

Image preview

The Image preview shows a preview of the image selected in the examination. When opening the pathology work screen, all results of type image will be checked and if an image is found, that image will be automatically previewed. If a result of type image gets the focus, that image will be shown in the preview. The name of the result will always be shown as label above the image itself. If you double-click on the image, the image will be shown enlarged in a pop-up screen. If you click on the paintbrush in the lower right corner the external program that is defined in windows for that type of image will be opened to view or edit the image.

Note

You can drag-and-drop an image file into a result field of type 'Image'.

Revision

GLIMS allows to perform a 'revision' of a pathology exam, after it was completed.  If the status of the exam is 'Validated-complete', a combo-box labeled 'Rev.' is visible in the top right corner of the pathology work screen.

If you want to perform a revision of the exam, change this field from No to To do. You will be asked to enter a reason for the revision. After the revision has been done, you can change this field from To do to Revised. You will be asked to enter the conclusion of the revision.

In the query browser on pathology exams (and in the pathology exam statistics), it is possible to make a selection based on this 'revision status'.

The conclusion

The conclusion is the most important result of the examination. It is requested automatically when a pathology procedure is requested. The conclusion is actually a result like the results from the result overview. The conclusion result is listed among the results in the result section. The conclusion can also be shown at the bottom of the pathology screen (e.g. for better visibility) by means of a parameter in the options screen.
