Anatomic pathology - MediSpeech connection

Contents: Introduction - Batch or interactive mode - Start dictation in GLIMS

Introduction

GLIMS offers a direct connection with the dictation application MediSpeech of the company G2Speech.  The MediSpeech application must be configured in GLIMS as a regular analyzer by means of a station.  

When creating an order and tests are to be executed on the MediSpeech station (scheduled through your procedure configuration), GLIMS will generate an order file (XML) with patient and sample information and send it to the MediSpeech application using MIPS' g2ms translator. The latter will also take care of the result communication from MediSpeech to GLIMS. For detailed information about the communication, please check the g2ms translator documentation.

Specimen identification

GLIMS will first look for a station linked to the driver "g2ms" and use the specimen identification settings of this station to determine which specimen id should be passed along.  If such a station is not found, the specimen internal id is used.

Batch or interactive mode

Glims supports both the batch and the interactive mode of the dictation application MediSpeech. 

To choose the desired mode:

  1. Choose Start -> Tools -> Set user preferences.
  2. In the Pathology tab page, set the option Dictation method to Batch dictation or Interactive dictation.

Start dictation in GLIMS

Laboratories that have purchased the MediSpeech speech recognition application for anatomic pathology and the MIPS's g2ms translator, can start the MediSpeech dictation module from within GLIMS.  This can be done in the following ways:

  1. Use the Specimen contextual menu function Start dictation (in submenu Pathology).  
  2. Using the Process by number program which can be accessed from the main menu via Start -> Routine -> Specimens -> Process by number and choosing the option Start dictation.

When choosing one of the options above, the MediSpeech application will be opened and you can start the dictation. MediSpeech will convert the dictation into text and these textual values will be reported back to GLIMS through the g2ms translator.  

Tip

In order to avoid getting the MediSpeech login screen when trying to start a dictation, the user id (actually the user's 'login name') and passwords in GLIMS and in MediSpeech must both be identical for those users who will dictate (the pathologists).

For more information about the MediSpeech application itself, please see the product manual.
