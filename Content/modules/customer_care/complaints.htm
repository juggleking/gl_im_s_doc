Complaints

Contents: Introduction - Creation - Query - Browser - Fields - Categories

Introduction

As part of the customer care module, GLIMS allows to register and manage complaints received by the laboratory.   Note that this complaint module is subject to a separate license.

Creation

New complaints can be created

  1. via the main GLIMS menu (Start -> Correspondents -> Customer care -> Complaints -> Create complaint).  
  2. By opening the list of current complaints (see below) and then using the Insert key.

Query

Complaints are best accessed through Start -> Correspondents -> Customer care -> Complaints.

Main

Internal id prefix

Restricts the result set to only complaints the internal id of which is prefixed with the specified string.

Correspondent

Is the correspondent for which the complaint is created (you can enable the extended correspondent search method in the order entry options).

Coordinator

Is the coordinator as specified for the correspondent for which the complaint is created.

Minimal receipt date and Maximal receipt date

Restrict the result set to only complaints with receipt date in the specified range.

Department

Only select complaints linked to this department.

Category

Only select complaints that belong to this category.

Without initial response date

If enabled, only complaints with initial response date unspecified are returned.

Response overdue

Allows selection of all complaints where the initial response date is overdue with respect to the Complaint.ResponseDueDate field.

Correction

Correction assignment

Gives you the opportunity to select the complaints assigned to a specific user for corrective action. 

Minimal correction status and Maximal correction status

Restrict the output to complaint satisfying the selected correction status range. 

From last correction time

To retrieve only complaints from a specified last correction time.

Prevention assignment

Gives you the opportunity to select the complaints assigned to a specific user for preventive action.

Prevention

Minimal prevention status and Maximal prevention status

Restricts the output to complaints satisfying the selected prevention status range.

From last prevention time

To retrieve only complaints from a specified last prevention time.

Browser

Executing the query will yield a complaint browser.  When a new complaint is created for a correspondent, a mail is sent to its coordinator, containing the description of the complaint and a hyperlink (which calls the complaint to the screen when using the "Hyperlink" item of the contextual ribbon tab "Tools").

Fields

Background and Correction Page

Internal id

Will be set automatically upon committing the new record, as determined by a site function defined in Specific Site (Start -> System management -> Customize -> GLIMS).

Correspondent

Is mandatory as a complaint is always linked to a specific correspondent.

Receipt date

Is mandatory too. There is however no limitation as to what date is specified. It can be the current date, a date in the past or a date in the future.

Response due date

When should a response be given to this complaint.

Order

Allows to connect a complaint to an order.

Department

The complaint selection screen allows filtering on Department.

Category

Allows classifying complaints in categories.

Initial response date

Can be set at creation time or at any other given time in the complaint's life cycle. Note that once set it can no longer be updated.

Short description

Is a short description of the complaint which also appears in complaint browsers.

Long description

Is the place to supply a more extensive description of the complaint.

Correction start and end date

Can once set no longer be updated.

Correction status

Is set as initial at creation time and will alter once processed to a different status.

Last correction time

Is blank at creation time but will automatically adopt the creation date and time. From then on each time the complaint is processed and a correction status override is performed, the last correction time will be updated.

Correction assignment

Is set to "?" at creation time. Appointment of a correction assignment can only be done while processing a complaint. It will remain assigned to that particular user until a new correction assignment is performed. 

Prevention Page

Prevention start and end date

Can once set no longer be updated.

Prevention status

Is set as initial at creation time and will alter once processed to a different status.

Last prevention time

Is blank at creation time but will automatically adopt the creation date and time. From then on each time the complaint is processed and a correction status override is performed, the last correction time will be updated.

Prevention assignment

Is set to "?" at creation time. Appointment of a correction assignment can only be done while processing a complaint. It will remain assigned to that particular user until a new correction assignment is performed.

Categories

This hierarchical table allows the definition of complaint categories.  Click Start -> Correspondents -> Customer care -> Complaints -> Complaint categories.

The complaint selection screen has a selection criterion Category. Filling in a category during complaint creation is not mandatory.

The complaint categories are structured in a tree view. Please check the chapter Tree editors, for more information on how to work with a tree editor.
