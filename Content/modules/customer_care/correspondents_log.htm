Correspondent log

Contents: Introduction - Access - Adding a correspondent log - Log report

Introduction

The correspondent log module in GLIMS allows to store any information (special needs, points of attention, actions to be taken, ...) about a correspondent.

Access

The CorrespondentLog table allows to maintain a record of log entries for each correspondent. It can be accessed from the Correspondent contextual menu through Customer care -> Log - > Browse. The status of the log entry indicates whether it requires an action or a review.

Creation time and user

Indicates who created the log and at what time.

Correspondent

Correspondent to which this log belongs.

Status

Action, review or closed.

Priority

The importance or priority of this log

Notes

The actual log notes.

Action

The action to be undertaken in response to this log

Adding a correspondent log

There are several ways to a log for a correspondent:

  1. Hit Insert in the correspondent log browser.
  2. Using the Correspondent.AddLog() contextual menu function.  This contextual menu function is available through Customer care -> Log -> Add log.

Priority

The importance or priority of this log.

Notes

Specify your notes.

Mail coordinator

Determines if a mail is to be sent to the coordinator of the correspondent.

Action required

Determines if the status of the log entry should be marked for further action.

  1. While creating a task for this correspondent.  

Log report

The correspondent contextual menu function Log report, accessible through Customer care -> Log -> Log report, shows a viewer with the selected log entries, ready for printing.

Query options

Start date

Only log entries since this date.

Minimal and maximal status

Only log entries between this status range.

Keyword

Only log entries containing these keywords.

Format

Automatic (Windows = HTML, TTY = Text), Text, HTML or MS Word (deprecated).

Printer

Printer to be used for the log report.

Page width/height

Determines the page layout of the log report.

Preview

Show a preview before actually printing the report.

File name

File in which the log report should be saved.
