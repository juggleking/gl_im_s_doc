Introduction

A tour determines a visit sequence for a set of correspondents.  Its primary use is for sample collection and report delivery.  Note that this module is subject to a separate license.

Tours

The tour browser is available from the main menu via Start -> Correspondents -> Customer care -> Tours.

Fields

Mnemonic

Example

Morning01, Evening02, Special12

Description

A descriptive text regarding this tour's purpose.

Responsible

The designated responsible for this tour (e.g. the driver).

Frequency

I ndicates when the tour is planned.

Start

The normal start time of this tour.

Final lapse

The time needed to return from the last correspondent. It is used in combination with the TourMember.TimeLapse fields to determine the total duration of the tour.

Duration

The normal duration of this tour.

Final distance

The distance to return from the last correspondent of the tour. It is used in combination with the TourMember.Distance field to calculate the total distance of the tour.

Distance

The approximate distance of the entire tour.

Recalculate

Indicates that the tour data (e.g. Distance, Duration, ...) should be recalculated.

Municipalities

This field is automatically filled with the municipalities from this tour's members.

Tour members

After having created a tour, you can now define the members of this tour, i.e. the correspondents that should be visited during this tour.  Use the contextual menu function Members to do so.

Tour

Tour to which this member belongs.

Correspondent

Reference to a correspondent.

Seq no

Determines the relative position of the correspondent in the visit sequence of the tour.

Distance

Distance between this tour member and the preceding one.

Time lapse

Time in hours and minutes between arrival and arrival at previous tour member.

Arrival

The expected arrival of the driver at the correspondent's location.  Both the sequence no and time lapse have to be specified before the arrival is calculated.  The calculation is triggered when changing one of both fields.  The calculation takes the entered sequence no into account.  

Note

The TourMember.Arrival is calculated as Tour.StartTime + TourMember.TimeLapse when the tour member has no valid predecessor.

Frequency

Frequency with which the correspondent is actually visited via this tour. Currently, only a list of weekdays is supported.

Frequency info

May contain a comma-separated list of weekdays on which the correspondent can be visited.

Comment

A comment specific for the membership of the correspondent to this specific tour.

You can also add a correspondent to a tour using the correspondent contextual menu function 'Add to tour' (Customer care -> Tours -> Add to tour).

Tour report

The contextual menu function Report generates a complete overview of the tour members.  The function has the following options:

Type

Currently only 'Overview' is supported.

Format

Automatic (Windows = HTML, TTY = Text), Text, HTML or MS Word (deprecated).

Printer

Printer to be used for the task report.

Page width/height

Determines the page layout of the tour report.

Preview

Show a preview before actually printing the tour report.

File name

File in which the task report should be saved.

Output example

Tasks

When creating a new task, you can assign it to a specific tour.  For more information, please see the chapter Task management.  Using the Tour contextual menu function Tasks you can open a browser with all tasks related to the selected tour (and print a task report).

Correspondent tour data

Correspondent.TourData()

The function Correspondent.TourData() shows an overview of the tours to which the correspondent belongs, including arrival time, preceding and following tour members. When selecting the "Tour data" contextual menu function (via Customer care -> Tours -> Tour data), you can opt to also show alternative tours, i.e. tours passing through the same municipality.

Output example

Correspondent.TourMnemonicList() MISPL function

This MISPL function returns a comma-separated list of the mnemonics of the tours where this correspondent is a member, sorted by tour mnemonic. When the optional <Pattern> is specified, only the tours with a mnemonic matching the pattern are retrieved.

Example

.TourMnemonics("OT*").

String TourMnemonicList (String Pattern)

String Pattern
An optional search pattern for the tour mnemonics

Tour-specific reports

Report generation can be restricted to tours with the same prefix, while generating the reports in the order of the correspondents within the tour.  For more information, please check the chapter on reports.
