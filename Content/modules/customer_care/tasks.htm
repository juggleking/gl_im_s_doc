Task management

Contents: Introduction - Creating a new task - Task query/browser - Task processing - Task report

Introduction

As part of the customer care module, GLIMS offers a task management module.  

Note

This module is subject to a separate license.

A task registers that an action has to be undertaken by a specific user (optional), for a specific station (optional), a specific correspondent (optional), a specific tour (optional),...  Tasks are also a convenient way of registering incoming calls from customers, assigning work to the customer's tour responsible or a lab biologist.

Creating a new task

A central task creation screen allows creation of new tasks.  It is available from the main menu via Start -> Routine -> Tasks -> Add task.

Internal id

A generated internal id for the task.

Date

The scheduled date for the task.

Correspondent

The correspondent for whom the task is to be executed.

Customer care

Provides access to the tours to which the selected correspondent belongs and to the complaints and tasks registered for that correspondent.

Tour

The tour for which the task is scheduled.  When entering a correspondent and the prefix of a tour mnemonic, the program tries to determine the tour automatically by finding a unique tour with this prefix to which this correspondent is assigned. This allows entering "ET" (evening tour) and leaving it up to the program to figure out to which ET tour this correspondent belongs.

User

User to who this task will be assigned.

Station

The station for which the task is to be executed.  You could e.g. create a dummy station representing the administrative department of your lab and register the tasks for this department.

Description

A short description of the task.

Comment

The long description of the task.

Mail user

Send a mail to notify the specified user.

Mail coordinator

Send a mail to the correspondent's coordinator. 

Add to correspondent log

Add a note the correspondent's log.

Priority

How important/urgent is this task (Low, Normal, High, Alarm).

Status

The status of the task (Open, Closed,...).  This status can be promoted/demoted when processing the task.

Log

A comment regarding the execution of the task

Task query/browser

A central task browser allows easy selection of tasks assigned to a user, a tour, etc.  This browser is accessible via Start -> Routine -> Tasks -> Tasks.

Internal id prefix

Only tasks with this internal id prefix.

User

Only tasks for this user.

Correspondent

Only tasks for this correspondent.

Tour prefix

If specified, only tasks for tours with this prefix will be selected. Note that the 'Tour' parameter has precedence.

Tour

Only tasks of this tour will.

Station

Only tasks of this station

Today

If set, only tasks scheduled for today will be selected. Note that minimal date and maximal date have precedence.

Minimal-maximal date

Only tasks within this date range.

Minimal-maximal status

Only tasks within this status range.

Minimal priority

Only tasks with this priority or higher.

Description pattern

Only tasks where the description matches this pattern.

Correspondents  

Customer care -> Tasks -> Browse

Tours

Tasks

Stations

Tasks -> Browse

Task processing

Tasks can be processed by entering comments and changing their status via the Process record contextual menu function.

New status

Status of the task after having processed it.

Send mail

Send a mail to notify the user assigned to this task.

Reassign

Allows assigning the task to another user.

Notes

Notes that will be appended to the task.

Task report

You can also print a task report using the contextual menu function Report.

Format

Automatic (Windows = HTML, TTY = Text), Text, HTML or MS Word (deprecated).

Include log

Also print a log of the task processing.

Printer

Printer to be used for the task report.

Page width/height

Determines the page layout of the task report.

Preview

Show a preview before actually printing the task report.

File name

File in which the task report should be saved.
