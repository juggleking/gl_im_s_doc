Creating the KV-DT tape

Creating a Financial Shipment

'Create KV-DT tape' is a method on the 'FinancialShipment' table. Therefore in order to create a tape, a FinancialShipment must be made that includes all InvoiceSummaries of the 'Kassen' that must be put on tape. Use the 'Kassen' CorrespondentGroup for easy selection of InvoiceSummaries. Make also sure only to select the 'KV-DT' series InvoiceSummaries.

The destination of the FinancialShipment must be the KV (as an organisation) that will receive the tape. Make sure that The identification of this KV with the 'HC code provider' is the UKV kennung of that KV.

Note that the fshp_ExternalDate is important: it determines the quarter!

Creating the tape.

Select the method "Create KVDT tape". The following options are available:

Parameter Description
Directory Directory where files will be written
Test If set, the status of the FinancialShipment is not increased to fixed, and therefore, the shipmentdate is not set neither. Also, for a test tape, even erranous reecords are written to the file.
Check file If set, each XDT record is checked against the rules set defined by the KBV. All errors are logged. If an error is detected, that record is not written to file, except for a test tape. Also the name of the file will have an extension 'bad' iso 'CON'.
Replace Character Depending on the character set used, not all possible characters are allowed. If Replace Character is not set, then the occurrence of a forbidden character is considered to be an error. If Replace Character is set, then any illegal character is silently translated to this value.
CharacterSet Defines the character set to use on file output.\

Mit Wirkung zum 01.04.2010 ist ausschließlich der Zeichensatz nach ISO 8859-15 erlaubt

Lkz Sub Level Normally, when an Lkz is required, its created per executor. If the Lkz Sub Level is set a further subdivision per executing class is made. Note that only tar_ExecutingClass is evaluated, not bca_ExecutingClass!!!! Also, only executing classes for which the Executor has provisions in the current quarter are taken into account. If no provision id s found for a tariff record, then the first available one is taken.
Force period In some cases, customers want to use the 'Abrechnungsquartal' on the KV tapes instead of the actual 'BehandlungsQuartal' (in order to avoid that the KV refuses a tape due to 'non-allowed encounter splitting').

If this option is set, and orders of a previous quarter are included, the Quarter field (FK 4101) will be forced to the current quarter (=quarter of shipment external date).  Note that in this case the Leistungstag field (FK 5000) will be filled with the invoice creation date (to comply with REGEL 324..).

The Financial Shipment should only contain InvoiceSummaries. This program creates 1 file in the KV-DT format. Only ADT records are written.

The file name is according to the KBV requirements: Zhhnnnnnnnnn_TT.MM.JJJJ_hh.mm.eee
in glims ADT file Z01nnnnnnnnn_TT.MM.JJJJ_hh.mm.con

  • Z = ISO 8859-15 Code
  • hh = Enthaltene Datenpakete in hexadezimaler Darstellung (01 for ADT)
  • nnnnnnnnn = die 9-stellige (Neben)Betriebsstättennummer des jeweiligen Or-tes, an dem die Abrechnungsdatei erstellt wird.
    This is in glims the lab's recognition number (identification of the Firm with the HC Code Provider).
  • TT.MM.JJJJ_hh.mm = Zeitstempel
  • eee = Dateierweiterung CON (=Defaultwert), WTK (siehe Kapitel 1.9.3)

Note that creation of the tape can take a considerable amount of time.

The file is created in the specified directory. Also a logging is made. All blocking issues are logged as "errors": they must be solved before shipping the tape to the KV.

Entries logged as "warnings" are no blocking problems.

Leistungsketten

The tape program does not accept 'Leistungsketten'. See the description for modification GLIMS_BILL-00373. When a 'Leistungskette' is detected an error is generated. Use the option 'Billing item confirmation' to first acknowledge these items. Alternativally redo the tariffication of the erranous order.
