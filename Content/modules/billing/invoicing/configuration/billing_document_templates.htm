Invoicing: Billing document templates

Contents: Introduction - Fields - Examples

Introduction

The most important parameter when printing a billing document is the ‘billing document template’. This template determines the layout of the printed document.  It can be specified in the following functions: Invoice.Print, InvoiceSummary.Print and FinancialShipment.Print.

By default, all billing documents will be fixed after successful printing. The printed billing documents can be marked (recommended) by creating a ‘billing document out prints’ which stores when, how many times and with which layout a billing document has been printed.

Billing document templates also play an important role in the configuration of the Emailing function for billing documents.

Fields

For a full description of all fields, please check the reference manual.

  • Document type can be:
    • Invoice: templates of this type can be used to print invoices
    • Financial transaction: templates of this type can be used to print invoice summaries and payments (indirectly via the transaction)
    • Financial shipment: templates of this type can be used to print financial shipments.  It can also be used to print multiple invoice summaries as one document.
    • Financial shipment list: templates of this type can be used to print multiple shipments as one document.

Symbols that can be used in the header/footer texts:

&PageNo: Will display the current page number

&TotalPages: Will display the total page number.  E.g. {Page &PageNo of &TotalPages} could be translated into 'Page 2 of 4'.

&BatchPageNo: can be used in the header or footer to print the running page number in the current batch.  This batch page number is reset upon each call of the billing document generator and keeps increasing until the end of the current batch.   This batch page number can assist in checking whether all pages that were generated by the program have physically been printed.

For large laboratories that use the billing document generation programs to generate large documents, an option has been added to improve performance.  To obtain maximum speed, one can set the option 'LoadItemData' in the BillingDocumentTemplate to NO. In this case, the field bdln_TotalAmountOthers will not be taken into account when used in text expressions.

The billing document print program allows to specify if a trailing form feed should be added to the output file via the field BillingDocumentTemplate.AddFormFeed.

Examples

  • Print an invoice summary which merely contains the number of occurring billing items in the invoice summary.
  • Print an invoice summary containing all billing items which occur in the invoice summary.
  • Print an invoice summary containing an introductory page which displays the invoice summary number and the total amount.  The next page contains all invoices occurring in the invoice summary and the corresponding patient data, issuer, order internal id, invoice document number, billing items and fee.
  • Print an invoice summary containing all occurring invoices with their corresponding order internal id, invoice document number and billing items.
