Discounts

GLIMS offers support for four kind of discounts:

  • Discount fields
  • Turnover discounts
  • Standard discounts
  • Issuer-specific discounts
  • Panel discount for issuer invoices
  • Copying a discount formula

Discount fields

  • Discountable:  it is possible to mark a price list so that discounts do not apply to the tariffs of this list.  This would typically be used for a payer-specific price list.  This does not apply to tariffs retrieved from the 'parent' default pricelist.

Standard discounts

Standard discounts can be assigned to certain lab customers for the execution of certain tests (billing codes).  These discounts are independent of the turnover.

Example: a certain customer receives a 10% reduction on the tests that the lab executes itself and a a 5% reduction on the tests that have to be executed by another laboratory.

1. Define a discount code

Click Site -> Billing -> Configuration -> Discount -> Discount codes and insert a new discount code:

2. Define the discount classes

Click Site -> Billing -> Configuration -> Discount -> Discount codes and insert the necessary discount classes.

Example:

  • ClassIn: Class for tests executed within our lab
  • ClassEx: Class for tests that have to be executed by an external lab

3. Define the actual discounts per discount class

For both discount classes created above (ClassIn and ClassEx), you now have to define formula's.  Use the contextual menu function Discount formula's and insert a formula for each class.  Enter the discount code created earlier (Discount) and a discount rate (5% for ClassIn and 10% for ClassEx).

4.  Assign the billing codes to the appropriate discount class

In the billing code, you can specify the discount class to be applied.  Click Site -> Billing -> Configuration -> Billing codes -> By code to open the browser with your billing codes.

5. Assign the discount code to the correspondent

In the payment agreement for the correspondent (e.g. AZ MIPS), you can specify the discount code (e.g. Discount).

Issuer-specific discounts

The standard discounts described above are applied for all patients having the payment agreement where the discount code is specified.  However, it is also possible to configure issuer-specific discounts via the Correspondent.DiscountCode field.  This discount will be applied when the payment agreement does not have a specific discount code.  It will only be applied to the invoices that are sent to the issuer !

In order to activate this, you must enable the PolicyClause.ApplyDiscount option.  If set, this option indicates that for this clause the issuer's discount must be applied. However, even when this option is set, the payment agreement discount code has precedence over this correspondent discount code.

Panel discount for issuer invoices

Introduction

GLIMS also allows defining a discount rate on panel request definition level.  This discount rate will only be applied to the issuer invoice !  If no discount rate is specified, the standard discount mechanisms as described above will still be evaluated.  If a discount rate is specified on panel request definition level, the other discount mechanisms will be ignored for issuer invoices.

The discount rate is applied to all billing codes generated by the panel.

Configuration

Discount rate definition

  1. Select or create a request definition of type "Panel".  
  2. Select the new panel request definition and choose the contextual menu function 'As panel request definition -> Edit'.
  3. Specify the discount rate that must be applied to some or all request definitions that are member of this panel

Mark members for which the panel discount rate must be applied

  1. Select the panel request definition for which you specified a discount rate
  2. Select the contextual menu function 'As panel request definition -> members'.
  3. For each member for which the discount should be applied, select the contextual menu function 'Request definition' and enable the 'Allow discount' option

Note

When you enable the 'Allow discount' option for the panel request definition itself, the discount rate will be applied to the billing codes assigned to that panel request definition.

Summary

  1. The applied discount rate is specified in the requesting PanelRequestable.DiscountRate.
  2. This type of discount is only applied to members (and panel billing code) with Requestable.AllowDiscount enabled
  3. This type of discount is only applied to issuer invoice items
  4. PanelRequestable.DiscountRate=0 means discount 0% (and also skipping the standard discount class mechanism)

Copying a discount formula

The DiscountFormula.Copy contextual menu function allows selection of one or more discount formulas belonging to a certain discount code and copying them to a new, an existing, or even the same discount code.  Using this functionality, complete sets of discount codes with their related discount formulas can easily be copied.

Options in the user interface:

Target discount code

Selected formulas must be copied to this discount code (same, other or new one)

Start date

If indicated, start date used for new formulas

Use start date

If set, the specified start date will be used

End predeceasing discount formula
