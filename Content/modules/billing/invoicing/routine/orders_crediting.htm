Order crediting

Contents: Introduction - User interface

Introduction

The order crediting program can be started from :

  • An order: contextual menu -> Billing -> Order crediting
  • An invoice: contextual menu -> Order crediting
  • A person: contextual menu -> Patient order crediting

This program combines 3 functions: crediting invoices, force financing of order(sets) and tariffing order(set)s.   It saves a lot of time (less clicking/navigating from one screen to another) and reduces possible wrong manipulations.

The user interface gives a clear overview of a payment agreement’s usage (which orders, relation between orders, status) and the needed data for executing the desired function(s) can be inputted and updated centrally.

User interface

The user interface has 4 main sections:

1. Reimbursement party and actual financing of order set

  • Visualization of base order’s order set reimbursement party
     
  • Edit-button for editing the reimbursement party if needed (for modifying party’ address, name, updating payment agreement data, ... )
     
  • Browser displaying actual payment agreement(s) used for base order financing
     
  • "Select payment agreement" button : only active when started from Person-level in order to select the base payment agreement.

2. New financing of order set to apply

  • Buttons: same behavior as in order set editor: one can select a payer, third payer or general agreement or combine multiple
     
  • "Force finance order set" option: is enabled automatically when a new payment agreement (different from the original one above) is selected. the new payment agreement(s) will be set (stored as new financing of the selected orders) during refinancing the selected orders.

3. Select orders to be credited and refinanced

  • Browser displaying all orders related to the actual (original) financing if no order filter is active.  The list of selected orders can be manipulated by using one of the filter options (default options or order based options).  Orders can also be selected/deselected using the default Glims-behavior (pressing the spacebar or with CTRL-A or CTRL-N …)
     
  • "Credit invoices" option and optional credit parameter set
     
  • "Retariff orders" option and optional tariff parameter set
     
  • Rejection type: additional info about why crediting is performed:
     
    • Allows using the "InvoiceRejectionType" and "InvoiceRejectionTypeAction" tables
       
    • These tables can contain data about invoice grouping, use of series, invoice suppression, … for each possible type of newly created invoices/credit notes. Used during creation of new invoice summaries after crediting/re-tariffing.
       
  • Base (reference) order and related button
     
  • "Filter on base order" option : to (de)activate filtering on base order criteria
     
  • Filter selection criteria list : criteria depend of (de)activated base order filter.

4. Credit and force financing button

After having entered the needed data for the desired functions that have to be executed, press the ’r;Credit and Force Financing’ button to continue.

At least the ’r;Credit invoices’ option must be set:  if this option is not set, the program is aborted (with appropriate message).  his prevents problems like force financing on non-credited orders (this is not possible on completely tariffed orders) and also prevents inconsistent results.  The crediting function on already credited orders or non-tariffed ones can be done without any problem, so must always be activated !!

After pressing the ’r;Credit and Force Financing’ button a pop-up message appears mentioning the number of selected orders and the selected function(s).  Acknowledge to continue (or choose Cancel to abort).

If acknowledged and the needed parameters sets for crediting and/or tariffing are not specified in the interface, Glims will ask to do so.  When completed: for all selected orders, the desired actions/operations are executed simultaneously: this program allows multi-record processing of orders. 

In the Glims logging, the logging details of all executed actions can be found.
