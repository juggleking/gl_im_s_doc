Invoice.BillingItemConfirmation

This function sets the 'Confirmation status' field of the billing items on the selected invoices. This method actually has 2 phases:

  1. In the first phase, the function verifies which billing items need confirmation. Those that don't need confirmation receive a confirmation status 'NotRequired'. The others receive a status 'NotAcknowledged'.  The most important criteria for this function to decide that a billing item doesn't require a manual confirmation: the first billing item for requests that
    • are explicitly requested and have a result or are explicitly validated.
    • are explicitly marked as to be charged

    Other billing items must be confirmed manually.
     

  2. In a second phase all billing items that are left in the status 'NotAcknowledged' are presented to the user, grouped per order. The user can then manually confirm these billing items, giving them the status 'Acknowledged'.  This function should be used in Germany for all invoices send to the KV using a KVDT file; the KVDT creation method will refuse any billing item that is not yet confirmed.

  • Log process name: if specified, this is the process name of the log that is created by this run.
