Invoicing: Pro forma invoices

Contents: Introduction - Configuration - Generation

Introduction

Pro forma invoices can be considered as 'unofficial' invoices which are not permanently stored in GLIMS.  They can only be printed during report generation and will be deleted as soon as they have been printed.

Configuration

There are three important 'Policy' fields for the configuration of pro forma invoices:

Pro forma

If set, this policy will generate pro forma invoices

Add to report

As pro forma invoices can only be printed during report generation, this option must be enabled.

Tariff members

This option must be set to 'Yes'.  It indicates that the panel members must be tariffed, regardless of the PanelRequestable.TariffMembers setting).

Generation

To print a pro forma invoice during report generation, the following requirements must be met:

  • A 'pro forma' policy (and general payment agreement) must be defined.  Enable the "Pro forma", "Add to report" and "Tariff members" options.
  • Select the pro forma payment agreement for those orders for which you want to generate a pro forma invoice (manually or via the "Order set default" MISPL).
  • In the report generation screen, specify
    • A tariffication parameter set
    • The desired print settings (which invoice layout should be used)
