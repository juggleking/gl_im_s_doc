Invoice validation

Contents: Introduction - Configuration & Routine

Introduction

GLIMS allows to impose an invoice validation.  This implies that invoices must be verified before they can be grouped on invoice summaries and that the latter cannot be printed or put on a financial shipment without verification.

Configuration & Routine

How can invoices and invoice summaries be marked as 'To check':

Invoice

  • for one of the order sets billed in this invoice 'Check invoicing' was set or,
  • the 'Check invoicing' field of the payer (correspondent) is set.

Invoice summary

  • for one of the order sets billed in this invoice summary 'check invoicing' was set or,
  • the 'Check invoicing' field of the payer (correspondent) was set or,
  • the 'Check mark' option was set when using the invoice's function 'Invoice summary creation'.

How can unverified invoices be refrained from being processed:

Both the invoice and invoice summary query feature an option 'To check'.  This allows to exclude all invoices or invoice summaries that are marked as "To check" when printing invoices, grouping invoices on invoice summaries, printing invoice summaries, placing invoice summaries on financial shipments, etc.

How can we clear the 'To check' mark:

Invoice

  • The value of the 'To check' field can manually be changed to 'No'
  • Via the invoice validation program which is available from the main menu via Start -> Billing -> Invoices -> Validate.

Invoice summary

  • The value of the 'To check' field must manually be changed to 'No'
