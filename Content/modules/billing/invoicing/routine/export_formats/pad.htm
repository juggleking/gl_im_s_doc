Introduction

By means of the FinancialShipment.Send function, GLIMS supports the billing export 'PrivatAbrechnung mittels Datentraeger' (PAD).  The FinancialShipment.Send() function with protocol = PAD will create a file named [PV]nnnnnn.DAT and Begleitzettel' [PV]nnnnnn.TXT.

Use a file name prefix 'PV' if you want the invoice document numbers to be assigned by the PVS (Privataertzlichen Verrechnungsstellen).

Invoice document numbering (Rechnungsnummerung)

If a file name PVnnnnnn.dat (nnnnnn=arztnummer) is used then PVS will assign the Rechnungsnummern.  If you want Glims to send the document numbers, you should use a filename nnnnnn.dat and make sure that the 'Invoice DocNo' expression generates a document number with format 999999 with leading zero’s and exact 6 characters long.

Begleitzettel

A 'Begleitzettel' file is required in case of floppy disks (not for 'DFU mit Verschlusselung').  When an export file is created, a 'Begleitzettel' file will automatically be created with the same file name but with extension .txt.
