FinancialShipment.Send -> C-Page

Contents: Introduction - Identifications - Remarks - Routine - Message layout

Introduction

Using the FinancialShipment.Send function and selecting "C-Page" as protocol, GLIMS allows creating files conform "C-Page 'Facturation des actes' version 1996".  Note that the GLIMS 'C-Page billing' program only creates these files and that the customer is responsible for transferring the generated files to the appropriate destination.

Before transferring billing data, you must collect all data on a financial shipment.  Once a shipment is sent, the shipment and its underlying data will be fixed and protected against later modifications. As a result, corrections are collected on a new shipment (version).

Identifications

All transferred correspondents id's (lab, executors, ..) are identification codes assigned by the financial shipment’s destination (= source = id provider). If no identification code is available for the correspondent, the correspondent’s mnemonic is used.

Routine

Select one or more financial shipments and execute the "Send" function (with protocol C-Page).  If the shipment is successfully processed then:

  • One or more file(s) are produced and the financial shipment will be fixed.
  • A summary of transfer will be written in the shipment's comment.
  • Detected errors are logged in the GLIMS logging.

Remarks

The GLIMS 'C-Page billing' program does not support 'C-Page Frais divers' messages.
