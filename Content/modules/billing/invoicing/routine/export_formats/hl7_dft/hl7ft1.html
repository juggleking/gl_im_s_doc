FT1.4 Transaction date

The value of this field depends of what you select as 'Execution time' in the FinancialShipment.Send function:

  • Result availability time: the transaction date can be
    • Result.AvailabilityTime
    • Order.ObjectTime
    • MicrobiologyAction.ObjectTime
    • BloodSelection.ObjectTime
    • In case of supplements that have no request charges, we use the order sets tariff time.
  • Lowest object time: the transaction date will be the order's lowest object time
  • Tariff time: the transaction date will correspond with the actual tariffication time

FT1.13 Department code depends of the Result type

  • Request Type is Result : WorkPlace.wrkp_CostCode
  • Request Type is Specimen : WorkPlace.wkrp_CostCode
  • ELSE : BillingCode.bc_CostCode
  • IF the Billing code's cost location is also unknown then the specific sites cost location is used.
  • (since glims5.4D) An alternative set of codes can be defined by means of a cost location site attribute: "_Code-"<Shipment's destination correspondent internal id>.
    Note that an alternative set of codes must be exhaustive (for each cost location). The program uses either the site attribute values or the cost location codes. NOT a mixture of both. The alternate code will be stored in FT1 13 subfield 4.
