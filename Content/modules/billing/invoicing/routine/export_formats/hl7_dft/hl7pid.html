HL7: PID - patient identification segment

The PID segment is used by all applications as the primary means of communicating patient identification information. This segment contains permanent patient identifying and demographic information that, for the most part, is not likely to change frequently.

3. Person Id

Here, the GLIMS patient (HIS) Identification Code will be put, or, the patient Correspondent InternalId if no Code was found. For more information, please check the HL7 export startup option -x.

5. Person Name

  • Export:
    • 5.1  : Person LastName
    • 5.2. : Person FirstName
    • 5.3. : Person MiddleName
    • 5.5 : Person Prefix
    • 5.6. : Person Title
  • Import:
    • 5.1 + 5.5 : Person LastName
    • 5.2. : Person FirstName
    • 5.3. : Person MiddleName
    • 5.6. : Person Title

8. Person Sex

  • F : Female
  • M : Male
  • U : Unknown

11. Person (Correspondent)Address

  • Export v2.2:
    • 11.1 : Correspondent AddressLine1 (street address + house number)
    • 11.3 : Correspondent Municipality Name
    • 11.5 : Correspondent Municipality PostalCode
    • 11.6 : Correspondent Municipality Country
  • Export v2.3 and up:
    • 11.1 : Correspondent AddressLine1 (street address)
    • 11.2 : Correspondent AddressLine1 (house number)
    • 11.3 : Correspondent Municipality Name
    • 11.5 : Correspondent Municipality PostalCode
    • 11.6 : Correspondent Municipality Country
  • Import:
    • 11.1 + 11.2: Correspondent AddressLine1
    • 11.3 : Correspondent Municipality Name
    • 11.5 : Correspondent Municipality PostalCode
    • 11.6 : Correspondent Municipality Country

13. Person (Correspondent) Phone

  • 13. Repeat 1 : Correspondent Phone
  • 13. Repeat 2 : Correspondent Phone 2

23. Birth Place

Glims site attribute of table Person (Person BirthPlace). If the table BirthPlace is created as a site attribute table of the person table then the birth place will be reported/set.

24. Multiple Birth Indicator

  • Y : Multiple Birth
  • N : No Multiple Birth

Version 2.3 and up only

Remark: for v2.2, a dedicated ZPI segment will be used.
