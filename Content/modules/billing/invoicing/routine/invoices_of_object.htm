Invoices of object

Introduction

The function "Invoices of object" in the contextual menu of Object and Order displays all invoices of the patient's orders (sorted by receipt date).  Its main goal is the detection of unpaid invoices.

Screen features and options

Title

Title containing the object's total unpaid amount  (=sum of unbalanced amount of all fixed invoice summaries).

Invoice browser

Rows

One row per invoice: untariffed orders can also be displayed.

Columns

  • Order (internal id + receipt date),
  • invoice (payer type and amount)
  • Invoice summary (amount and unbalanced amount)
  • Payment information (amount, date, payment type)
  • The last column contains additional information (second payment, Credit note rejection reason, ... )

Colours

  • The order column is displayed using the standard order color scheme.
  • An invoice summary with unbalanced amounts is displayed in red.
  • Credited invoices and credit invoices is displayed in grey

Buttons

In the top right corner, you will find 4 buttons:

  • payment of the invoice summary (of the focused record)
  • order quick invoice function (of the focused record)
  • 2 additional functions can be chosen in the settings screen (see below).

The "Settings" button at the bottom of the screen offers the following options:

  • A number of query options to limit the invoices that should be shown
  • The definition of two extra functions that are available in the main screen (see above).  The selected function should have one of the following base tables: Order, Invoice, OrderSet or InvoiceSummary

Unbalanced payment browser

Shows a list of unbalanced payments of the selected patient.  Preferable, each payment should be balanced with an invoice summary.  From this browser you can open the unbalanced payment(s) and link them with the right invoice summary or register a refund (Payment.Refund function).
