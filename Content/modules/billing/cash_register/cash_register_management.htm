Cash register - Management

Introduction

The cash register management screen is available from the main menu via Start -> Billing -> Cash register management.

Opening a cash register session

A cash register session is still active for the logged in user

The cash register management screen will automatically show the active cash register session.

No cash register session is active for the logged in user

The cash register management screen will show the most recently closed cash register session for the logged in user.

Open a new session

  • Click the Open button in the lower right corner

  • Choose a Cash register

  • A new Cash register session is started for the selected Cash register

Tip

Case description

  • A cash register is linked to a user group.
  • One of the users forgot to close a cash register session and is not present in the lab.
  • Another user from the group wants to open a session for the same cash register.
  • GLIMS does not support simultaneous sessions for one cash register.

Solution

  • One of the users in the user group can be marked as Cash register responsible.

  • In the User group assignment screen, enable the Logical site attribute named _CashRegisterResponsible.

  • The cash register responsible can now open a session for the same cash register. By doing so, the user from the active (locked) session will be changed to the responsible user. The responsible user can now close the session.

Registering manual payments as cash register movements

When a cash register session has been opened by a user, all payments manually entered by this user will now be registered as cash register movements.

Payment registration

Payments can be manually registered:

  • Via the payment entry screen.
  • Via the quick invoice screen.

Cash register movements

The payments are listed in the section Overview session movements.

The following actions cause the creation of a movement :

  • Open session
  • Validate session
  • Close session
  • Payment
  • Deposit (see below)
  • Withdrawal (see below)
  • Balance calculation (see below)

Overview session amounts and totals

The lower left part of the screen shows an overview of the amounts and totals:

Allows to register a deposit. This can be used to e.g. add a start amount in the cash register in the morning.
Allows to register a withdrawal. This can be used to e.g. register that an excess of money has been put is a safer place.
Allows to calculate the actual balance of the cash register.
Will refresh the screen, e.g. to renew the data when the screen was not closed while payments were registered.

Note

The buttons for registering a deposit / withdrawal / calculation will be grayed out if the Cash register option (site attribute) Deactivation deposit is enabled.

If the buttons are not available although the option Deactivation deposit is disabled, do the following:

  • Enable the Cash register option Deactivation deposit.
  • Open the Cash register management screen.
  • Disable the Cash register option Deactivation deposit.
  • Refresh the Cash register management screen.
  • The buttons should now be enabled.

Tip

Each deposit, withdrawal, and balance calculation will be registered as a cash register movement.

Closing a cash register session

To close a cash register session:

  1. Click the Validate button to confirm that the GLIMS data matches the actual content of the cash register.

  2. Click the Close button.

Printing a cash register session report

GLIMS can print a cash register report using report builder (for one or several sessions).

For more information, please check the Report builder chapter.
