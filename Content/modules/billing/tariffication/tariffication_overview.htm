General overview of the tariffication process

This page merely gives an overview of how all the tariffication tables are linked to each other.

Determining the price...

A request is requested via a request definition
A request definition has billing codes
A  billing code has a tariff in a price list
a tariff refers to a price code
a price code has a price formula
a billing code belongs to a reimbursement class

Parties involved

Both at order, order set and payment agreement level, a number of correspondents are introduced in the tariffication process. All these correspondents can later be selected as reimburser type in the policy clause.

Reimbursement

At order set level a reimbursement party and payment agreement will be specified or proposed via the order set default MISPL.  By means of a policy name, the payment agreement refers to a policy.  A policy has one or more policy clauses.

Conclusion

So when tariffing an order set, the policy to be used will be known. Each policy clause specifies for which reimbursement class it is valid (+ other criteria).  In that way, GLIMS knows who will have to pay which part of a certain price under which conditions.
