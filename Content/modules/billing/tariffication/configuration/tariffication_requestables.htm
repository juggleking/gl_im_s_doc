Request definitions

Each property has a default request definition. When inserting a property in GLIMS, a default request definition is automatically created with the same name of the property mnemonic.

An order consists of different requests. Requests are only billed by means of the request definition that was used to enter the request. Request definitions can be defined for properties, materials, microbiology procedures and blood products. For each property, material, microbiology procedure and blood product, one can specify, optionally, a "default request definition" of the corresponding type.

The default request definition is used if the request definition used has no billing code or if there is no request definition:

  • To simplify the association of billing codes to request definitions:
    A request is tariffed by means of the billing codes of the (bottom-level) request definition that was used to create the request. If no billing codes are specified for that request definition, then the billing codes of the corresponding default request definition are used. This means that if a property, material, microbiology procedure or blood product needs to be tariffed independently from the request definition it was created with, it suffices to assign billing codes to the corresponding default request definition.
  • To tariff requests which were requested without the use of a request definition:
    Requests which are created by the scheduling mechanism are created without the use of a request definition. If these requests need to be tariffed, then the billing codes of the default request definition of the corresponding property, material, microbiology procedure or blood product, will be used.
