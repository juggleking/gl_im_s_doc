Partial order billing

Content: Introduction - Configuration - Routine

Introduction

Glims allows to bill partial orders.  This has been introduced in order to allow billing an order in several steps: e.g. first the chemistry tests, then the microbiology tests and then the transfusion part of the order.

To do so, you can group request definition in billing categories.  When tariffing an order (set), you can indicate that you first want to bill the request definitions of a particular billing category and then all other request definitions (e.g. when the order is complete).

This new functionality makes it possible to create multiple invoices for the same order, at different points in time.

Configuration

You can define billing categories in a dedicated table "BillingCategory" to group the request definition that should be tariffed together.  A request definition can refer to a billing category via its field "Billing category".

  • Minimal order status: minimum order status before this category will be tariffed.

  • One-time billing: if checked, this category will only be tariffed once, even if tests belonging to this category are later on added to the order.

Routine

In the tariffication screen (OrderSet.Tariff), you can check the option "Partial order billing" and select a billing category for which the request definitions will be tariffed.
