Provisions

From each executing class you can choose the contextual menu function 'Provisions' and assign a provision to determine the responsible for the tariffication:

Executor

The responsible health care provider.

Lab

The responsible lab (and thereby its associated default firm).

Accredited

Indicates whether the executor is accredited during the provision time.

Valid from-until

Validity date range.

Exclusive

If set, the executor of this provision will be set as executor of all internally executed tests.  Note that this is not applicable for anatomic pathology examinations.

Department

It is possible to define a provision on department-level: such a provision is only valid for a test executed in the specified specific lab AND specific department.  This provision will be used if the executing lab and the executing department (linked to procedure) match (otherwise the already existing logic is followed).

Replaceable

This field indicates whether a provision related executor can be replaced by an "exclusive" executor (in case an "exclusive" provision exists).  This allows to replace some (but not all) of the executors by an "exclusive" executor, depending (for example) from the executing class.  However, in case of anatomic pathology or externally executed tests, the executor will never be replaced (this is hard-coded).

MISPL

  • The MISPL function GetProvision allows to lookup a provision in a MISPL expression.  Check the reference manual for more info.

Example

RETURN (GetProvision("LAB1A", ?, "Cold", .Order.ReceiptTime).Executor.Correspondent.InternalId);

will return the name of the health care provider responsible for Lab 'Lab1A' and ExecutingClass 'Cold'.
