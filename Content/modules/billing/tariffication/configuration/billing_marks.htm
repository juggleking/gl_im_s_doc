Billing marks

Contents: Introduction - Billing mark definition - Billing mark assignment - Support in policy clauses

Introduction

GLIMS allows assigning a billing mark to each request entered in the order entry screen.  This billing mark may for instance indicate that the request is to be billed to the patient or the insurance.

Billing mark definition

The definition of billing marks in GLIMS is fully customizable and can be done through a site domain named 'BillingMarks'.  For more information, check the chapter 'Site attributes and site domains'.

Billing mark assignment

The billing mark assignment happens at order entry time.  The order entry option "Billing mark required" allows specifying that the billing mark must be specified before entering a request.

The order entry screen allows to assign a specific billing mark for each request or a default billing mark for the entire order. If the latter is specified, this billing mark is used for each request where no explicit billing mark is specified (including requests added later on, e.g. via MISPL).

When adding a request via ResultOutput.CascadeRequest(), when the original request has a billing mark, the new request will inherit this billing mark. Else, the order billing mark is used.

The order billing mark is also supported via the electronic order entry programs.

Correspondent.BillingMark

The field Correspondent.BillingMark allows specifying a default billing mark which is to be used for all new orders issued by this correspondent. Currently, only the manual order entry program and the OMR order entry program support this option.

Panels

In the "Panel request definition" you can define 'Tariff member billing marks' (comma-separated list of billing marks).  When specified and the option "Tariff members" is set to "Yes", only requests with a billing mark that appears in this list will be charged.  Note that all requests belonging to the same panel are supposed to have the same billing mark, otherwise the result is unpredictable.  The field PanelMember.BillingMarkList allows to specify that the panel member should only be requested when the request has a billing mark which appears in the list.

When Panel.TariffMembers = TRUE and TariffMemberBillingMarkList is specified, the panel billing codes will NOT be charged when the members are charged.

Support in policy clauses

In the policy clause, the "Billing marks" field allows specifying a list of billing marks which will be used as a condition.  The policy clause will only be applicable for requests with a billing mark appearing in that list.
