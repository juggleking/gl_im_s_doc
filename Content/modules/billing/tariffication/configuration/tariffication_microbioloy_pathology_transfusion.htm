Microbiology, pathology and blood transfusion tariffication

In previous versions

In microbiology, the billing codes typically depend on the medium or micro-organism test.  To support this in previous Glims versions, this required complex MISPL expressions in the field BillingCodeAssignment.EligibleCount.  Moreover, the billing codes could only be defined per request definition.

In Glims version 8

The configuration now allows charging billing codes for microbiology sub records (more specific than a request):

  • per carrier (billing code assignment of medium)
  • per carrier test (billing code assignment of medium test)
  • per isolation (billing code assignment of micro-organism)
  • per isolation test (billing code assignment of micro-organism test)
  • per antibiotic result (billing code assignment of antibiotic)

On request level, billing codes can now be defined:

  • per microbiology procedure (billing code assignment of microbiology procedure)
  • per pathology procedure (billing code assignment of pathology procedure)

Billing code assignments

  • Billing code assignments now have a "type" field indicating on which level which they will generate a billing code. Billing code assignments created in Glims version 7 will automatically have type "Request".
  • The billing code assignment editor has been improved: conflicting configuration is prohibited, e.g. if the type is not "Request", you cannot select a request definition.
  • From a medium, medium test, micro-organism, etc, you can navigate to the assigned billing codes.

Tariffication rules

  • The carriers, carrier tests, isolation and isolation tests will only be tariffed if the request definition (used to request the microbiology procedure) is requested with the option "To charge" enabled during order entry.
  • Discontinued microbiology actions and pathology examinations are never tariffed.
  • The executor (of the generated billing item) of carriers, isolations, etc. is determined exactly the same ways as for microbiology actions and pathology examinations (responsible)
  • You can assign a billing code to the request definition (used to request the microbiology procedure / pathology exam) AND assign billing codes to media, micro-organisms.  Glims will take both into account !
  • If you mark a request definition (used to request the microbiology procedure / pathology exam) as "To charge" during order entry, at lease one billing code assignment must be defined (on request definition level or on medium, micro-organism etc. level).
  • The "Eligibility count" MISPL expression defined in the billing code assignment will now have a different start table, depending on the type of billing code assignment.

Request charge

  • After the tariffication, the originating carrier, carrier test, etc. is saved in the request charge for reference information.

Notes

The billing code assignment has 3 other types not yet discussed:

  • Blood product specification (per blood bag): the billing code assignment mechanism used in previous Glims versions with an optional blood product and a mandatory request definition definition is still allowed in Glims version 8, but these billing code assignments now have type "request".
  • Property / Material: the billing code assignment mechanism used in previous Glims versions with Pathology procedure / Microbiology procedure / Property and the material's default request definition is still allowed in Glims version 8.  If both billing code assignments are defined, only the default request definition's billing code will be taken into account.
  • This modification does not yet include the introduction of the billing rule scope "Isolation".  This planned for Glims version 8.1.
