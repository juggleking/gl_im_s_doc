Price setting: configuration overview

This document provides an overview of the GLIMS tables that are to be configured for price setting.  Each table has been given a sequence number in order to indicate a logical configuration path.

  1. Requestables (request definitions): the Requestable table is not really included in the price setting scheme as these should normally already be configured.
  2. Price codes: a price code indicates how the actual price will be calculated.  All tariff records for which the price calculation is and will be based on the same algorithm should refer the same price code.
  3. Price formulas: the price formula calculates the effective price based on the base value specified with the tariff.
  4. Price lists: A price list determines which tariff records are used for a billing code.
  5. Reimbursement classes: reimbursement classes group a number of billing codes which should be reimbursed in a similar way.
  6. Discount classes: GLIMS offers support for turnover and standard discounts.
  7. Billing codes: the billing code refers to the tariff to be used
  8. Tariffs: a tariff indicates a base value for a billing code.
  9. Billing code assignments: time-related connection between the request definition and the billing code
  10. Executing class: the executing class and provision determine who is responsible for the tariffication
  11. Provision: the executing class and provision determine who is responsible for the tariffication
  12. Policy clause: the policy clause is the link towards the determination of who will pay which part of the costs.
