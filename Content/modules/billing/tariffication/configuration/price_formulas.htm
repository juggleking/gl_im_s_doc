Price formulas

Introduction

A price code is used to refer indirectly to a formula determining a price. The indirection is necessary since the exact implementation of such a formula may depend on time. The indirection offers the possibility to modify the price calculation of a group of tariffs by means of one manipulation instead of changing it for each related tariff record.

Fields

The price formula calculates the effective price based on the base value specified with the tariff. Every price formula record has a limited validity period. The actual price is calculated by means of a multiplier and a fixed amount.

Example:

Base value of the tariff  10
Multiplier

3

 In price formula
Fixed amount +4
Formula: (10x3) +4  
Actual price 34

The ERD section of the tutorial contains a price setting ERD which explains the relationships between request definitions, billing codes, tariffs, price lists, price codes, price formulas, reimbursement classes,...

Context multiplier

Allows to indicate that the base value of a Tariff should be multiplied by a context-specific factor.

Example

If set to Order's distance, this option can be used to calculate the price of the distance traveled for sampling (specified during order entry).

Combined price formula / price code browser

A browser is available from the main menu that shows a concise overview of all currently valid price formulas and associated price codes.  Glims version 8 users must manually add this function to the main menu using the "Run procedure" mechanism.  The function to be used is "ActivePriceFormulas".
