Price lists

A price list determines which tariff records are used for a billing code. Usually there is a default price list which is specified by an official institution (fund) as e.g. RIZIV (Belgium). Other price lists can be created for e.g. tests executed in a commercial context, etc.

Price lists can be defined via Start -> Billing -> Configuration -> Price lists:

The price list to be used can be indicated at three different levels:

  • The price list specified in the policy clause
  • The price list specified in the payment agreement overrides the one specified in the policy clause
  • If not specified in the payment agreement nor in the policy clause, the default price list will be used.

Extension

Refers to another price list: if no tariff is found for a billing code in a certain price list, then the extension price list is used to look up the tariff.

Extension multiplier

Can increase or decrease the price if the tariff of the extension price list has to be used.
