Billing codes - Validation

Description

GLIMS offers a feature to validate/modify the billing code and/or executor for tests executed by subcontractor lab.

Configuration

  1. Define a new site attribute on the table "Lab", type "Logical", named "_ConfirmBillingCodes"
     
  2. Enable this new option for the executing labs for which you want to validate/modify the billing code and/or executor.
     
  3. This is typically used for tests executed by external labs when the actual executors and billing codes are not known in advance.

Notes:

  1. The confirmation, that is mandatory for labs with site attribute "_ConfirmBillingCodes" =YES can be disabled per policy by setting the Policy site attribute "_SkipConfirmBillingCodes" value to "YES"
     
  2. When using the site attribute "_ConfirmBillingCodes", the validation is either mandatory or not possible at all.  To make this validation optional, a site attribute "_ConfirmBillingCodesAllow" is available on table "Lab".  If enabled, GLIMS allows to validate billing codes, but this is not mandatory for the tariffication process.
     
  3. Performance note: customers not requiring this functionality should NOT define this site attribute as it may degrade the performance during tariffication.

Usage

  1. A contextual menu function "Specify executors and billing codes" is available on "Order set".  
     
  2. This function opens a screen that allows validating or modifying the billing codes and executors before the tariffication.  This will typically be used for specific executing labs (for which "Confirm billing codes" is enabled), more specifically for subcontractor labs.
     
  3. A pre-tariffication is performed (before applying) any billing rules.  The initial billing codes and executors are proposed, but can be modified.  The executors can be selected via their identification code  (identification provider = HC code provider defined in specific site) or via mnemonic.
     
  4. Additional billing codes can be added via a button (only for charged requests)
     
  5. The executing lab cab be updated in this validation screen.
     

  6. The lines can be filtered by entering a lab mnemonic at the bottom of the screen.

  7. Modified or confirmed billing codes are still applicable as input for the billing rules.
     
  8. Modified executors (HC provider) are checked against the availability of a provision (for that executing lab, department, executing class and at tariff time)
     
  9. The user can optionally specify comment + info amount (later stored in BillingItem.Comment)
     
  10. It is also possible to specify a date for each billing item.  This date is only stored for consultation purposes and is only visible in this billing code validation screen.  The date is stored in the associated Request site attribute "_ConfirmedBilling" (in the "#" tag , sub tag "Date").
     
  11. This function is also applicable for consultation purposes after the tariffication process
     

  12. When closing the screen, the user can reopen the validation screen for another order by specifying its order internal id. 

  13. During tariffication, Glims will check if there are executing labs for which the billing codes/executors need to be validated.  If so:
     
    1. previously entered values are used
       
    2. if not yet (completely) validated, an error message is logged

Technical note

The entered billing codes, executors and comments are stored temporarily in a request site attribute "_ConfirmedBilling" with  format: \bc=BillingCode.Code\ex=Hcprovider.Mnemonic\Comment=txt\checked=yes/no [,....]).  A comma separated list is used in case of several billing codes for the same request.
