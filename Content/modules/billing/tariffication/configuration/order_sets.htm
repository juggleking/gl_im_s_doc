Order sets

Contents: Introduction - Order set defaults and order set determination MISPL - Order set query and tariffication

Introduction

It is essential to know that GLIMS tariffs order sets and not orders.

Whenever the tariffication of a request is influenced by the presence of other requests in other orders and vice versa, it is impossible to tariff an order on its own. In those cases the related orders should be tariffed simultaneously. This can be achieved by grouping those orders in one order set. Hence, an order set is a group of one or more orders, to be considered as a whole during the tariffication. The label of an order set is a string, containing a specification for the grouping of its related orders.

Examples

Examples

  • According to the Belgian clinical nomenclature, the calculation of administrative and urgency supplements must be based on all orders for a given patient on a single day. In this case each order of a patient on the same day should be part of the same order set. The order receipt date can be stored in the order set's label.
  • In commercial environments payment agreements may span several orders. If these agreements need to be detected automatically, all the related orders need to be tariffed simultaneously.

Order set defaults and order set determination MISPL

In GLIMS one can define an "order set defaults" and an "order set determination" MISPL. These MISPLs can propose an order set, payment agreement, etc during order entry. When opening the Billing page in the order entry screen, for a new order, all relevant fields are filled with values proposed by the "order set defaults" MISPL. The user can then accept or change these values. When the order is saved, the "order set determination" MISPL is run and any fields from the billing page which are still unknown (?) are replaced with values from that last MISPL.

The "order set defaults" and "order set determination" MISPL can be specified in the specific site preference set (Start -> System Management -> Customize -> GLIMS Billing).

Note

When creating a new order by using the 'Insert' key in the Orders of order set browser, the traditional MISPL calculation of the order set will not be fully reevaluated and the order will automatically be linked to the concerned order set.

The order set defaults are applied when creating an order. They will also be recalculated after each significant order change when

  • The order set defaults were determined automatically (by MISPL) and have not been changed manually. The fact that an order set has been changed manually will be marked by setting the field "Orderset.ManuallyUpdated" to "Yes".
  • The order set is not yet completely tariffed.

Order set query and tariffication

As said above, GLIMS tariffs order sets. In order to tariff order sets, you can use the order set query (Start -> Billing -> Order sets -> Query) to select a number of order sets and then apply the contextual menu function 'Tariffication'. Naturally, this process can be automated through commands and tasks.

Days completed

When specifying the "Days completed", discontinued orders can never be selected (as the latter do not have a completion time) resulting in an incomplete tariffication. Therefore, this option is only taken into account for non-discontinued orders.
