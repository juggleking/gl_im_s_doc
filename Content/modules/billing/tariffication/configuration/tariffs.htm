Tariffs

As explained in the previous chapter, request definitions are linked to billing codes. These billing codes are now linked to tariffs. Each tariff belongs to one price list, allowing for different nomenclature codes and prices per price list.

A tariff specifies the nomenclature code and price for a billing code in a price list. The tariff does not specify the actual price of a test. It does indicate a base value. The actual price of a test will be based on this base value but will be calculated by means of a price code and a price formula.

MISPL functions 

Two MISPL functions are available to retrieve a tariff record:

  1. Request.DefaultTariff(<PriceListMnemonic>, <Date>) 
  2. Requestable.Tariff(<PriceListMnemonic>, <Date>) 

These functions retrieve a tariff record which is applicable for this request(able) within the optional given price list and validity date.  If multiple tariffs matches, the one with the highest Tariff.StartDate is taken.

Request.DefaultTariff, searches via Request->Result->Property->DefaultRequestable...Tariff.
Requestable.Tariff() searches via Requestable->BillingCodeAssignment(s)->BillingCode->Tariff.

Examples: 

RETURN .Request.DefaultTariff("NABM", ?).NomenclatureCode ; Tariff from price list NABM
RETURN .Requestable.Tariff("NABM", TODAY()).NomenclatureCode ; Valid today
RETURN .Result.LastRequest(?).DefaultTariff("NABM", ?).NomenclatureCode ; Start from a result
RETURN .Requestable.Tariff(?, TODAY()).NomenclatureCode ; Search in all price lists
RETURN .Request.DefaultTariff(Glims().DefaultPriceList.Mnemonic, DateTimeToDate(.Order.Receipttime)).NomenclatureCode; Search in standard price list and valid on receipt time
RETURN .Requestable.Tariff(?, ?).PriceCode.BaseValueCode; Return price code (B,HN,...)
RETURN .Requestable.Tariff(?, ?).BaseValue; Return BaseValue
