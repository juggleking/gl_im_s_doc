Introduction

As explained in the "Order set" chapter, GLIMS tariffs order sets and not orders.  There are two ways to tariff an order set, via the OrderSet.Tariffication function or via the quick invoice program.

OrderSet.Tariffication

From each order set record you can activate the contextual menu function "Tariffication".  This function performs the actual tariffication of the order set.  You can create a command and task (with an order set query) to automate the tariffication process.

Partial order billing: please check the chapter "Partial order billing".

Logging level:   optionally, the tariffication process can log details about the applied billing codes and billing rules:

  • Standard (like before): logs only the mnemonics of the applied billing rules
  • Billing code assignments: logs the request - billing code links
  • Billing rule details: logs per applied billing rule the related billing codes and the billing rule description
  • Extended logging (for MIPS staff only): dumps the tariffication temp tables

Quick invoice

The quick invoice tool (which simulates the tariffication process) also allows to perform the tariffication for a single order.  For more information about this tool, please check the 'Quick invoice' chapter.
