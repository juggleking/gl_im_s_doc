Quick invoice

The quick invoice program can be started by selecting the contextual menu function Billing -> Quick invoice from any order. When doing so, the order will be temporarily tariffed and a quick invoice will be displayed. When performing a temporary tariffication, the completeness check defined in the policy will be ignored.

Status page

The Status page contains the following elements:

Order set status

Unpaid amount

This is the total unpaid amount for all invoices of the current payer (as defined in the Payment field of the Settings page).

Payment button

On the right hand side of the unpaid amount, this button allows introducing a payment for all unbalanced invoice summaries of the current payer (as defined in the Payment field of the Settings page). When doing so, a browser will be displayed with the concerned invoice summaries. Select one and click OK to make the payment. After having done so, the quick invoice screen will automatically be refreshed.

Refund button

"F" button

Allows executing a user-defined function on the selected transaction. First select the table (FinancialTransaction, InvoiceSummary or Payment), then select the function and function parameter set. This will be remembered in the user preferences and the label of the button will be changed to the first character of the chosen function.

Transactions

Transactions for this order (invoice summary, payments, etc.)

Settings page

The Settings page allows defining a number of settings for the displayed quick invoice (document printing, payment and tariffication parameters). These settings are stored per user.

Preview party

Allows indicating for which party the initial invoice preview should be displayed/printed. The next field allows specifying which billing document type should be used (invoice or financial transaction). You can then select a billing document template and define printing settings. The latter allow executing special actions during the print-out.

First and second print

The settings for the first and second print will be applied when using the first and second printing buttons.
The Preview party and First print/Second print options have two special values. These allow indicating which type of invoice should preferably be used for previewing or print invoices.
  • Prefer Person/Fund/Agent: preference will be given to invoices with payer type Object person. If not available, invoices with payer type Fund or Fund 2 will be used. Again, if not available, invoices with payer type Agent or any other invoice will be used.
  • Prefer Fund/Person: preference will be given to invoices with payer type Fund.If not available, invoices with payer type Object person or any other invoice will be used.

Note

For both options (and in contrast with the other options), credit invoices will be ignored in order to always use the most recent invoice version.

Payment

Allows defining for whom the payment should be introduced when using the payment buttons.

Print payments

If set, the payments will also be printed when printing the quick invoice.

Tariff parameter set

Allows defining special actions that should be carried out during tariffication (mark order set as complete, make invoice summaries for other parties than the ones that are printed, etc.).

TR

Allows to use the Progress transaction mechanism to undo temporary invoices.

PP (Pre-payments)

In some sites, the patient has to pay an advance, even before the patient invoice is final or even generated. It is possible to store this advance and when the patient invoice is made part of an invoice summary, the payment is connected to the invoice summary.

To support pre-payments, enable the Pre-payments (PP) option. If you now click the payment button, you can enter an invoice-based payment.

Note

The MISPL function Invoice.GetPayment(n) allows retrieving the n-th (pre)payment associated with an invoice.

Price consultation

The price consultation program can be started via Start -> Routine -> Orders -> Price consultation. The price consultation uses the same program as the quick invoice, but the tariffication will NOT be stored.

When starting the price consultation program, GLIMS firsts open the order entry screen. This allows creating a 'dummy' order for which a quick invoice will be generated. Enter an order and click OK. GLIMS will (temporarily) tariff the dummy order and generate a quick invoice. The same functionality is available as described above.
