How to re-tariff an order set

The way to re-tariff an order set depends on the current tariffication status of the order set, invoice, invoice summary and financial shipment.  An order set can only be re-tariffed if it has not yet reached the status 'Complete'. Several situations can occur:

1. Order set status = complete AND invoice, invoice summary or financial shipment is not yet fixed (i.e. not yet printed or sent)

  • Lower the order set status, using the contextual menu function 'Change tariffing status' and enabling the 'Downgrade option'

  • Re-tariff the order set, using the Order set contextual menu function 'Tariffication'

2. Order set status = complete AND invoice, invoice summary or financial shipment are fixed

a. The invoice summary or financial shipment has not yet been sent.

The status of an invoice, invoice summary or financial shipment is a protection mechanism which prohibits changing or deleting billing data that has already been charged to a patient or other party.  However, GLIMS still allows to undo this mechanism by lowering these states.  In order to do so, take the following steps in chronological order.

  • Lower the order set status, using the contextual menu function 'Change tariffing status' and enabling the 'Downgrade option'
  • If available, first lower the tariffing status of the financial shipment.  The financial shipment can be reached via:
    • Invoice (contextual menu of Order set)
    • Shipment entry (contextual menu of Invoice: Invoice summary -> Transaction -> Shipment entry)
    • Financial shipment (contextual menu on Shipment entry)

    Now use the contextual menu 'Change status' on Financial shipment and downgrade the status.

  • If available, now lower the tariffing status of the Invoice summary.  From the invoice record, select the contextual menu function Invoice summary -> Change status and select the option Downgrade.

  • And finally, lower the status of the Invoice, using the contextual menu function on Invoice 'Change status' and selecting the option Downgrade.

  • Now that all the necessary states have been lowered, you can delete the tariffication.  You should only do this if the invoice summary or financial shipment has not yet been sent.  Deleting the tariffication can be done by the contextual menu function on Order set 'Clear tariffication'.

  • Now re-tariff the order set, using the Order set contextual menu function 'Tariffication'

b. The invoice summary or financial shipment has already been sent.

If the invoice summary or financial shipment has already been sent, you must not delete nor lower the status of the invoices, invoice summaries or financial shipment.  Instead, credit notes and new invoices should be created:

  • First lower the order set status, using the contextual menu function 'Change tariffing status' and enabling the 'Downgrade option'
  • Now immediately re-tariff the order set , using the Order set contextual menu function 'Tariffication'.  GLIMS will automatically create credit notes for the previous, fixed invoices and create the necessary new invoices.
