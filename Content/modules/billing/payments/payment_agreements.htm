Payment agreements

Introduction

Each payment agreement represents an agreement (policy name) which can be used to tariff an order (set). The payment agreement editor looks as follows:

The fields are described in the database manual.

Correspondent specific agreement

If the agreement is only applicable for a specific correspondent, then that correspondent is specified in the Party field. If the Party field is left empty, then the agreement is generally applicable for all kinds of orders (sets).

Agreement start date

When a new order is created for a new patient (via electronic order entry), the start time of the related payment agreement is set to the creation time of the order.

However, if

  • Afterwards the order's lowest object time is updated via, for instance, an update of the sampling time of a specimen or an order import message (SynchronizeLowestObjectTime), and
  • The difference between the payment agreement's start time and the order's lowest object time not higher than 7 days, and
  • The payment agreement's Party is specified,

Then, the payment agreement's start time is set to the order's updated lowest object time.
