Introduction

The payment import program in GLIMS allows to import payment files according to the BVB I.5 format.  Click here for a PDF document describing the BVB protocol.

  • BVB I.5 file type

ome Belgian customers (having a KBC account) receive two files when downloading account-info from their Isabel package (BVB I.5 standard): one file (type '00') contains all operations/actions performed on the account, the second one (type '01') typically only contains incoming (credit) transfers (with a valid OGM) on the customer account. 

These files almost contain identical information and the format and naming of both files is also identical.  In order to prevent customers from importing double payment info in Glims (by importing both payment files), the payment import program allows selecting the type of file which has to be imported 

  • All actions on account = type 00
  • Transfers = type 01
  • Ignore = all files will be imported, regardless their type

During import of the payment file, the file type is identified by checking the code 'Afzonderlijke Toepassing' in the header record (record '0') : 

Examples: 

'00000' = family '00' (non-defined operations/actions) 
actiontype '00' (all operations/actions on account) 
'0' (no further sorting or splitting up)

'01005' = family '01' (transfers)
actiontype '00' (all transfers)
'5' (= ex. = only transfers with structured OGM)

Importing the wrong type of file causes an error (displayed on screen and logged).
