Payment import -> HPRIM Règlement

Contents: Introduction - Mapping

  Introduction

The GLIMS payment import program allows importing payment files in HPRIM Règlement format.  The payments in the imported files will be registered by Glims as payments.  The payments of one received file are also collected in a financial shipment of type ‘Received’.

The payments are balanced (with the corresponding invoice summary) via REG20.3 ‘Numero dossier administrative’ which contains the Glims.Encounter external Id (because we send the encounter’s external id in the HPRIM FAC message).
