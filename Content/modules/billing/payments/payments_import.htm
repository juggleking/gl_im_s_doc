Payment import

The payment import program is available from the main menu via Start -> Billing -> Payments -> Import payments file.

Avoid importing same payment file twice

The payment import program will produce a warning when importing an already imported payment file again.  When importing a file interactively, Glims will produce a warning message and will offer the possibility to either cancel (default) or discontinue the import.  When importing files in batch, already imported payment files will not be imported again.  When the import is cancelled, both interactively or in batch, an error will be logged in the Glims logging tables.

Double payment files are detected by the existence of financial shipments (type incoming) with the same firm, bank, payment amounts and payment dates.

Note

Even when the import is canceled, the import file will be moved to the "good" subdirectory (if supported for that protocol).
