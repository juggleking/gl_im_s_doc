Payment entry

Next to the payment import program, GLIMS offers two other ways to register payments: via the payment entry program or via the quick invoice program.

Via the payment entry program

The manual payment entry program is available from the main menu via Start -> Billing -> Payments -> By internal Id and hitting the Insert key.  The first thing you should do is enter the firm (for which the payment was made) and the date (on which the payment executed).  Then, there are two commonly used methods to actually enter the payment:

By reference number

Enter the reference number and hit the lookup button next to it.  The corresponding transaction will be displayed pre-selected in the embedded transactions browser.

Note: the "Payment subtype" field allows making a subdivision for the payment type.  The values of the payment subtype need to be defined in a site domain with name "PaymentSubType" and data type "Site enumerated".

By payer and amount

Enter the amount that you want to register, specify the payer and hit the lookup button next to it.  All current transactions (fixed but not yet paid) will be displayed in the embedded transactions browser.  You can select the transactions to be paid until the entered amount is reached. Note that you can (de)select all records in the financial transaction browser using CTRL-A  and CTRL-N.  The amount can also be calculated based on the selected transactions of that payment.

Note: the "Payment subtype" field allows making a subdivision for the payment type.  The values of the payment subtype need to be defined in a site domain with name "PaymentSubType" and data type "Site enumerated".

Via the quick invoice program
