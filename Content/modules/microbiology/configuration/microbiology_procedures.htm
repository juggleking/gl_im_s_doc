Microbiology - Microbiology procedures

Introduction

The microbiology procedure is the central part of the configuration. It represents the different steps in the microbiology examination.

When a microbiology procedure is requested, a microbiology action will automatically be created in GLIMS. From the microbiology action, you will then be able to open the microbiology work screen, which allows to enter all findings during this particular examination.

Microbiology procedures can be accessed from the main menu via Start -> Microbiology -> Configuration -> Microbiology procedures.

General

Sequence number

Used to sort microbiology actions that originate from this microbiology procedure.

Examples

  • In the microbiology work screen: procedures are sorted first by sequence number, then by creation date.
  • In the following MISPL functions: Specimen.Attribute("MicrobiologyProcedure...List") and Order.Attribute("MicrobiologyProcedure...List")
  • In microbiology statistics that classify by microbiology procedure (in rows/columns/tables).

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Description

Multilingual description.

Material

A microbiology procedure is valid for one particular Material.

Note

You must specify a root material, i.e. a material which is not derived from a parent material.

Type

Germ, virus or parasite.

Main page

Specimen scope

Allows specifying that the procedure requires a specimen with a specific scope. This allows making sure that a procedure or group of procedures gets its own specimen.

Example

If you request two microbiology procedures (or a combination of regular procedures and microbiology procedures) which would normally be able to use the same specimen, you can indicate that each procedure requires its own specimen by defining a different specimen scope for each procedure.

Initial micro-organism

This micro-organism will be selected by default when adding a new isolation in the microbiology work screen.

SOP

Standard operating procedure (free text)

Station page

Initial

This is the default station assigned to the microbiology action. If there is no Next MISPL (see below) or it returns unknown, the station remains what it was.

Next

Will be evaluated upon each status change of the microbiology action to determine the next station to which this action should be assigned.

Properties page

Report property

Should always be specified, else the result of the microbiology action will not be displayed on the report. By including this report property in a property classification, you can determine where the microbiology results should be placed on the report.

Tip

The report property is a dummy property which must have the data type Culture!

Conclusion property

Allows to include the conclusion text (specified in the microbiology work screen) on the report. By including this property in a property classification, you can determine where the conclusion should be placed on the report.

Grafting property, Microscopy property, Processing property

These dummy properties represent the different phases of the microbiology examination.

Notes

  • The Grafting property and Microscopy property automatically get a result when the status of the microbiology action is promoted.
  • The Processing property automatically gets a result when the microbiology action is validated.

These properties can specified to create a link between microbiology and 'normal' clinical chemistry procedures. This has two main advantages:

  • The microbiology procedure can be included on work lists. The functionality of the work list concept in GLIMS is then also available for microbiology procedures.
  • It allows to view other clinical tests in the microbiology screen (see examples below). These 3 properties can be outputs of certain non-microbiology procedures which can also have other outputs. These other outputs can also be displayed in the microbiology screen if their Autoprompt option is set or if they have already been requested.

Screen page

Show carrier browse

If set, the carrier browser will be displayed in the microbiology work screen for microbiology actions derived from this microbiology procedure.

Show review dates

If set, two review date fields will be displayed in the microbiology work screen. If you specify a date in these fields, you can later query on these review dates in the microbiology action query.

Show carrier test browse

If set, the carrier test browser will be displayed in the microbiology work screen for microbiology actions derived from this microbiology procedure.

Note

If the microbiology procedure option Show carrier browse is disabled, the carrier test browser will not be shown either, even if the option Show carrier test browse is enabled.

Negative page

Incubation period

Indicates the incubation period, i.e. the maximum number of days necessary for the procedure to be executed.

Negative answer - Preliminary, Final

Please check the chapter Negative answer when no growth is detected.

Request definitions page

Default request definition

The microbiology procedure can be requested in order entry by its default request definition.

The default request definition will be created automatically by GLIMS when saving the microbiology procedure. It will be a combination of the specified Material and Mnemonic.

Request definitions

Embedded browser showing the request definitions assigned to the microbiology procedure.

Blocking period

To support blocking periods for microbiology procedures, GLIMS checks if a Blocking period is defined for the Report property of the requested microbiology procedure. If so (and the user accepts the blocking period in order entry), the microbiology action will not be created and the Text used as result during blocking period will be stored as result value.
