Antibiotics

Antibiotic codes of the instrument are found in GLIMS as follows:

  1. The instrument antibiotic code matches with the external mnemonic of an antibiotic in GLIMS
  2. The instrument antibiotic code matches with an antibiotic identification code in GLIMS

If the laboratory has only one microbiology instrument or one type of microbiology instruments, it can map the antibiotic codes of the instrument with the external mnemonic of the antibiotics in GLIMS.

If the antibiotic definitions in GLIMS are to be instrument-independent, one can create antibiotic identification records for that instrument:

Microbiology instrument station -> Antibiotic codes: inbound id's = instrument antibiotic code

Microorganisms

Microorganism codes of the instrument are found in GLIMS as follows:

  1. The instrument microorganism code matches with the mnemonic of a microorganism in GLIMS
  2. The instrument microorganism code matches with a microorganism identification code in GLIMS

If the laboratory has only one microbiology instrument or one type of microbiology instruments, it can map the microorganism codes of the instrument with the external mnemonic of the microorganisms in GLIMS.

If the microorganism definitions in GLIMS are to be instrument-independent, one can create microorganism identification records for that instrument:

Microbiology instrument station -> Organism codes: inbound id's = outbound id's = instrument organism code

Microbiology procedures

The work done by a microbiology instrument can be represented in GLIMS by the reading phase of a microbiology examination. The reading phase on itself is represented as a non-microbiology procedure, say the "processing procedure". The processing procedure is configured on the microbiology instrument station. The processing property of the microbiology procedure should be output of the processing procedure.

During the reading phase, the current station of the microbiology action should be the microbiology instrument station. A "next station" expression in the microbiology procedure assures that the processing phase is done on the processing station. For instance:

if .status < StringToEnumerated("MicrobiologyActionStatus", "Reading")
then return "GraftingStation";
else return "ProcessingStation";
endif;

Where "GraftingStation" and "ProcessingStation" are the mnemonic of the grafting- and processing station respectively. In this example, phases before the reading phase (status less than "reading"), are done on the station with mnemonic "GraftingStation". From the reading phase on (status greater than or equal "Reading"), the microbiology action station is set to the station with mnemonic "ProcessingStation".

Downloading from GLIMS to the instrument

Download of specimens to the instrument by means of the processing property

When the reading phase of the microbiology examination is started, the non-microbiology action of the processing procedure becomes outputting. A microbiology instrument inquiry for specimens reflects in an inquiry for such actions.

To enable download of microbiology specimens to the instrument via the processing property, the assessment method of the processing property on the microbiology instrument station should have an outbound id different from unknown.

Download of isolations to the instrument by means of microorganism tests

Suppose that the instrument also requires isolated microorganisms to be downloaded rather than only specimen information. To download isolations to the instrument, microorganism tests should be configured on the microbiology instrument station. The microorganism tests are output of a non-microbiology procedure on the microbiology instrument station, say the "microorganism test procedure". The input of the procedure is the microbiology material.

The microorganism test methods should have an outbound id filled in and should have a non-disabled channel status.

Entering isolations and microorganism tests in the microbiology screen results in the creation of actions for the microorganism test procedure. A microbiology instrument inquiry for isolated organisms reflects in an inquiry for such actions.

Note:
In order not to have only specimen information being downloaded to the instrument, be sure to check the assessment methods of the processing property on the microbiology instrument station. If there are any, then the outbound id's should have value unknown. Otherwise, GLIMS assumes the instrument only needs specimen information to be downloaded.

Uploading results from the instrument to GLIMS

Uploading an isolation

The upload of an isolation to GLIMS consists of at least a specimen and an isolation number. The way isolation numbers are interpreted by GLIMS depends on the fact whether the station has an isolation sequence type or not.

GLIMS offers the possibility to automatically confirm isolations uploaded from the instrument. Set the toggle box "Auto confirm isolations" in the microbiology tab folder of the instrument's station. Unset the toggle box to disable auto-confirmation.

Uploading extra information with an isolation

In most of the cases, the microorganism code and antibiotic results are uploaded with the isolation. The microorganism code and the antibiotic codes should be defined in GLIMS as described above.

Additional information can be uploaded with the isolation:

  • Internal comment

    Some instruments provide extra information about the isolated organism. This information is stored in GLIMS in the internal comment of the isolation.
  • Microbiology examination code

    In a particular case it is required that the instrument uploads to GLIMS for which microbiology examination on a specimen the isolation was found. The uploaded examination code should match with a mnemonic of a microbiology procedure in GLIMS. See below for more details.
  • Trigger test code

    The trigger test code is an alternative for the microbiology examination code mentioned above. In this case, the trigger test code should match with the inbound id of a processing property assessment method. See below for more details.

Uploading isolations for specimens with multiple examinations

This paragraph concerns microbiology stations with an isolation sequence type. Remember that isolation sequences imply that isolations in GLIMS are not supposed to be created beforehand (see "Uploading an isolation").

Some laboratories have multiple microbiology procedures defined for the same material. This results in several microbiology actions for the same specimen. Suppose a new isolation is uploaded to GLIMS and the station is configured with an isolation sequence type. In that case, GLIMS must know for which microbiology procedure this new isolation is to be created.

There are several ways to solve this problem:

  1. (Re)configure the 2 microbiology procedures as examinations on a different type of material. This has as side-effect that GLIMS creates 2 specimens. Consequently, the instrument should use 2 different specimen id's, one for each examination.

  2. The instrument uploads a microbiology examination code along with the isolation. The examination code should match with the mnemonic of the microbiology procedure in GLIMS.

  3. As an alternative for 2), the instrument uploads a trigger test code. The trigger test code should match with the inbound id of a processing property assessment method. The microbiology procedure is found by means of that processing property.

Uploading antibiotic results

GLIMS supports several types of antibiotic results that can be uploaded with an isolation. Which antibiotic result types are uploaded depends on what the instrument is capable of and what its communication protocol supports.

  1. RIS value: The value is one of 'R', 'I' or 'S'
  2. RIS report value: The value is one of 'R', 'I' or 'S'
  3. MIC raw value: The MIC raw value is a string value consisting of an optional comparator sign (>, <, >=, <=) and a decimal value. For instance "<= 20.0"
  4. Agar diffusion value: A decimal value
  5. E-test value: Any string value

GLIMS offers the possibility to automatically confirm antibiotic results uploaded from the instrument. Set the toggle box Auto confirm antibiotic results in the microbiology tab folder of the instrument's station. If set, the Antibiogram Validation MISPL of the micro-organism will also be executed.

GLIMS also allows automatic antibiogram creation. This can be configured using the Antibiotic panel mode field in the Microbiology tab folder of the Station. Possible values:

  • None (no panel creation, allow any result)
  • Reject non members
  • Non members not reportable

If set to None, all antibiotic results transmitted by the instrument are stored.

With the second and third option, an antibiogram will be generated automatically at the moment the first result arrives. The results of non antibiotic panel members that are transmitted by the instrument will be rejected or will be set to not reportable (2nd and 3rd option).

Note:
When the 2nd or the 3rd option is active, beware that results will also be rejected or will also be set to not reportable when no antibiotic panel is generated for the isolation concerned.

When uploading microbiology results from an analyzer, the following rules are applied:

  • An antibiotic result which is confirmed will not be overwritten when a new value is received from the analyzer.
  • The microbiology action status is automatically upgraded to 'reading' when the first valid isolation is received from the analyzer.
  • When a microbiology action with status read-confirmed, read-validated or ready receives an uploaded isolation with a micro-organism different from the isolation already stored, the microbiology action status will be set back to 'rereading'.
  • When a microbiology action with status read-confirmed, read-validated or ready receives a new uploaded isolation (with a sequence number different from the already stored isolations), the microbiology action status will be set back to 'rereading'.
  • Isolations with invalid appraisals are no longer accepted.

Uploading isolation test results

Results of microorganism tests on an isolation can be uploaded to GLIMS as well. Configure microorganism test methods on the instrument's station with inbound id's matching the isolation test codes of the instrument. The microorganism test methods should have a non-disabled channel status.

Isolation numbering setting on microbiology instrument station

Please check the chapter "Microbiology analyzers - Isolation sequence types".

Configuration of multiple microbiology stations using the same microbiology procedure

Suppose the laboratory has a new microbiology instrument and would like to reuse the already configured microbiology procedures from an existing station and would like to have the option to choose on which station the specimen is to be processed.

This can be realized by filling in an identical remote id on both stations (the value of the remote id can be chosen freely). If the new instrument should also be able to process the microbiology actions of the station on which they were initially scheduled, one should create dummy processing procedures for the new station. Such a dummy procedure has no input but shares the same processing property.

Carrier information in communication between GLIMS and analyzer

Introduction

GLIMS is capable of sending microbiology carrier information in communication with microbiology instruments. GLIMS will also take into account the MIPS ASTM field R.11.1.1 (Instrument operator) in order to log the user that caused an update of the following GLIMS records: Isolation (isolation audit trail) and AntibioticResult (antibiotic result audit trail). This will be active if the field Station.StoreOperator is set. If the option is set, the user must exist in GLIMS (login name or initials should match with R.11.1.1). If not, upload is rejected!

Download

Download of carriers is supported for queries on single specimen only (not for batch download ALL-query). Upon download to a microbiology instrument, GLIMS can send carriers to the instrument (when the translator requests for it). If in the MIPS ASTM query message - a query specific tag Q.15.1.1 'Carriers' is present, GLIMS will try to send carrier information for:

  1. specimens being downloaded by means of a grafting/processing property - no isolations are sent. In this case, GLIMS will send all carriers related to that specimen.
  2. specimens being downloaded via isolation tests - isolations are sent. In this case, GLIMS will send all carriers related to the specimen and also the carriers related to the isolation (meaning a carrier might be sent twice for a specimen)

The carrier information is downloaded in a M-segment of the MIPS ASTM message:

M | segment.no | carr ^ carrier | carrier_seqno | medium

where

  • carrier_seqno = Carrier.SeqNo
  • medium = Carrier -> Medium.Mnemonic or Station.CodingSytem.CodeSet(Medium).Code.Value

Note: If a code set is defined, all media to be downloaded should have a code value.

Upload

GLIMS can process carrier information that is sent with a specimen or an isolation. The information is sent in M-segments of the MIPS ASTM message. When a carrier is sent with a specimen, GLIMS will not link isolations to that carrier. When a carrier is sent with an isolation, GLIMS will set the isolation's carrier to that one.

Two message types are used:

Carrier record:

M | segment.no | carr ^ Carrier | carrier_seqno | medium | actioncode | operatorid

where

  • carrier_seqno = the carrier sequence number in GLIMS
  • medium = Medium.Mnemonic or Station.CodingSytem.CodeSet(Medium).Code.Value
  • actioncode = 'N' or 'U' (new/update)

Carrier comment record:

M | segment.no | carc ^ CarrierComment | ^ text ^ actioncode

  • text = carrier comment field in GLIMS
  • actioncode = 'R' or 'A' (replace/add)

A carrier comment record will always follow a carrier record, meaning that the comment is related to that carrier.
