Microbiology - Antibiotics

Introduction

A lab can decide to execute a number of antibiotic tests on the isolation. The results of these antibiotic tests are displayed or can be entered in the microbiology screen by means of RIS (Resistant, Intermediary, Sensitive) or non-RIS (e.g. numeric) values.

Antibiotics can be accessed from the main menu via Start -> Microbiology -> Configuration -> Antibiotics -> Antibiotics.

Tip

Antibiotics can be grouped in Antibiotic panels.

General

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

Name to visualize the antibiotic in GLIMS.

Seq no

Determines the sequence in which antibiotics appear on reports, in the microbiology work screen, etc.

Warning

An antibiotic without sequence number will not appear on reports !

Disabled

If checked, this record can no longer be used.

Report

Description

Multilingual description used on reports.

Internal comment

Used for internal purposes.

External comment

Can be shown on reports.

External footnote

Can be shown as footnote on reports.

Auto reportable

If enabled, the antibiotic results will by default be included on reports.

Tip

The reportability of antibiotic can be changed in the microbiology screen.

Conversion

Formula type

Used when entering a numeric (non-RIS) value in the microbiology work screen. The antibiotic conversion rules (see below) allow to automatically convert numeric values to RIS values. GLIMS will only convert the value for this formula type to a RIS value.

Note that the Formula type specified in the Antibiotic panel takes precedence.

GLIMS offers two methods to convert numeric antibiotic values to RIS values: via MISPL or via conversion rules.

Warning

The two methods cannot be combined for the same antibiotics. This implies an OR-OR system: OR you use the conversion rules, OR you use the conversion MISPL, not both for the same antibiotic.

As soon as 1 conversion rule is defined, the site function defined in the Formula field will no longer be taken into account!

Via MISPL (less used)

Formula

Site function to convert the numeric value into an RIS value. These conversion formula's can sometimes be quite large, hard to oversee and difficult to maintain. Instead, you can also use the built-in antibiotic conversion rule mechanism.

MIC low pivot, MIC high pivot

These fields can be used in the Formula MISPL expression but are not used in any other place.

Via conversion rules (recommended)

Use the Insert key to define the conversion rules.

Germ or family

If specified, the rule only applies to that microorganism and to all microorganism below it (in the hierarchical tree). It overrides any rules that are defined for a microorganism that is higher in the hierarchical tree.

If not specified, the rule applies to all microorganisms.

Additional condition(s)

Allows to specify a site function with additional conditions that should be met before the rule is applied.

For agar diffusion

Enabled

The rule is applicable to Agar diffusion diameter values and not to MIC or ETest values.

Disabled

The rule is applicable to MIC and ETest values.

Note

This option has an effect on the interpretation of the Low pivot value and High pivot value (see below).

Low pivot value, High pivot value

Result values below the Low pivot value will be converted to S.

Result values above the High pivot value will be converted to R.

Values between the Low pivot value and High pivot value are converted to I.

Note

If the option For agar diffusion is enabled, the logic is reversed, i.e. Sensitive and Resistant are switched.

'Intermediary' includes low pivot, 'Intermediary' includes high pivot

Indicates if the Low pivot value and the High pivot value are part of the Intermediary range rather then the Sensitive respectively Resistant range (reversed for Agar diffusion values).

Example

Examples for MIC and E-Test values

Both options are disabled

  • Value <= low pivot => Sensitive
  • Value >= high pivot => Resistant
  • Low pivot < value < high pivot => Intermediary

Both options are enabled

  • Value < low pivot => Sensitive
  • Value > high pivot => Resistant
  • Low pivot <= value <= high pivot => Intermediary

For Agar diffusion values, Sensitive and Resistant are reversed.

Rule

Automatically generated description of the rule.

Tip

Alternate way to access the conversion rules:

  • from the contextual menu / ribbon on Antibiotic: Conversion rules

Default values

Embedded browser showing the default values defined for this antibiotic. Use the Insert key to add a default value.

Note

These default values are also available via the contextual menu / ribbon item Antibiotic result defaults on Microorganism.

Microorganism

The default value will only apply to this microorganism and the underlying microorganisms in the hierarchical tree.

Antibiotic

Antibiotic to which the default value applies.

Material

If specified, the default value will only be applied when this material is used.

Eligibility

Allows to specify a site function that defines extra criteria that should be met before the default value is applied.

Note

When several default values are configured for the same microorganism, the Eligibility should be different! In that case the default values without Eligibility will be evaluated last.

Initial value

The actual value that will be applied as default.

Tip

GLIMS allows to define a default value of '?' (unknown) for a specific antibiotic and microorganism. This may look useless at first sight, but consider the following concept:

Suppose for a certain antibiotic the same default value applies to a whole family of microorganisms, except for one where there should not appear any default value. You can define the default value once for the whole family together (for the 'parent' of the family), and you can specify a default value of '?' for the one germ that should not have a default value.

Auto confirm

If this option is activated, the standard antibiotic result is automatically confirmed when it is applied. This option in combination with the Station option Check antibiotic result status can cause GLIMS to prevent the default value of an antibiotic result from being updated with an incoming result message. To do this, the Check antibiotic result status option must be activated in the Station editor.

Note

  • By default, the Auto confirm option is disabled.
  • The audit trail of an antibiotic result will mention the use of an Antibiotic result default and the automatic confirmation.

Allowed values

These fields enable the user to select a list of non-RIS values for each type of antibiotic result (MIC, Agar diffusion or E-test). The purpose of using such a list is to limit the antibiotic result values that can be entered in the antibiogram browser of the microbiology work screen to specific ones.

Epid. relevance

Allows defining rules which determine the relevance of specific antibiotics in epidemiology. These rules are applied when GLIMS tries to detect

  • if isolations are doubles (identical)

  • if patients have infected each other (cross infections)
