Microbiology reporting - Isolations

How are isolations reported

The isolations and their appraisal are reported via the Report property of the microbiology procedure.

Which isolations are reported

For each microbiology action, the detected isolations with their corresponding isolation appraisal will be displayed on the report. These are the isolations specified in the microbiology work screen.

Note

In the microbiology work screen, it is possible to mark an isolation as not-reportable (by clicking in the 'Rp' column of the isolation browser).

Which isolation data is reported

The report will display:

  • The micro-organism linked to the isolation
  • The isolation appraisal
  • The isolation's external comment

About the isolation appraisal

The reported isolation appraisal text can be configured in different ways (which should not be combined):

1. Specify a text in the isolation appraisal itself.

If specified, the Text field of the isolation appraisal will be shown on the report. You can use the variable {Name} to include the name of the micro-organism.

Example

{Name}\t++ would result in the name of the micro-organism, followed by '++' in the next column.

Note

This means that this text will be used on all reports. If you want different texts for each individual report type, you should use the report template field Isolation comment text (see below).

2. Using the report template field Isolation comment text

This field can contain a dynamic text module. If specified, the reports generated with this template will automatically add this text module after the appraisal of each isolation in the report.

See also About the isolation's external comment

Note

If an EMPTY text module is entered, no comments will automatically be added (except when specified in the isolation appraisals).

If NO text module is given, the following default is automatically added:

{= IfKnownString("~n" + .ExternalComment) + IfKnownString("~n" + .Microorganism.ExternalComment) }

So the isolation's comment is added (if any) on a new line and then the organism's comment (if any) is added on a new line.

If no text module is specified and the appraisals also contain a comment layout, the comments will appear TWICE on the reports. So either remove them from the appraisals or specify an empty text module in the report templates.

About the isolation's external comment

The report will display the isolation's external comment, as specified in the microbiology work screen.

Tip

The Report template field Isolation comments can overrule these external comments.

If specified, reports generated with this template will automatically add this text module after the appraisal of each isolation in the report.

Example:

The text below will first display the isolation's comment (on the next line), then the microorganism's comment, then the medium on which it was detected (if specified). If the report layout is not 2-column, the "\t" (on the last line of the text module) should be replaced by "".

{= RTrim(FitText(
IfKnownString(chr(10) + .ExternalComment)
+ IfKnownString(chr(10) + .Microorganism.ExternalComment)
+ IfKnownString(chr(10) + "Medium: " + .Carrier.Medium.Mnemonic)
, 45, "", "\t", ?), ?) }

Sequence and numbering of the isolations

Sequence

For each culture, the isolations are sorted according to their internal sequence number in GLIMS.

Numbering

Displaying a sequence number in front of the reported isolations

GLIMS allows displaying a sequence number in front of the isolations on the report. This is the internal sequence number as shown in the microbiology work screen. The Report template option Show seq nos indicates when this should happen:

Never

Never number the isolations

When antibiogram

Only number the isolations when an antibiogram is available

Always

Always number the isolations.

Note

This option does not affect the sequence numbers used in the antibiogram column headers.

If you do not want a sequence number in front of each isolation, you will probably also want to change the default column headers in the antibiogram (which use the same sequence number). To do so, you can specify a dynamic text in the Report template field Germ headers which displays e.g. the name of the micro-organism.

Isolation renumbering

When displaying isolation sequence numbers, "gaps" may exist when one or more isolations are marked as "not reportable". To solve this, you can enable the Report template option Renumber sequence numbers on report.

  • If enabled, the isolations will be renumbered according to their sequence of appearance on the report.
  • For each separate specimen counting will restart from 1.
  • The isolation in the antibiogram will also be renumbered to maintain synchronization.

Example

Suppose there are 2 cultures:

  • An anaerobe culture in which 2 germs are found:

2. germ b
5. germ e

  • An aerobe culture in which 3 germs are found:

1. germ a 
3. germ c (which is not reportable)
4. germ d

Assume that all germs have an antibiogram and in the property classification the anaerobe culture comes first.

Report with Renumbering disabled

Anaerobe culture:

2. Germ b
5. Germ e

Aerobe culture:

1. germ a
4. germ d

Antibiogram:

           1. (a) 2. (b) 4. (d) 5. (e)
Amp.   ...
Oxa.    ...

Report with Renumbering enabled

Anaerobe culture:

1. Germ b
2. Germ e

Aerobe culture:

3. germ a
4. germ d

Antibiogram:

           1. (b) 2. (e) 3. (a) 4. (d)
Amp.   ...
Oxa.    ...

Notes

Note

If the report includes an antibiogram and

  1. the Report template option Show single antibiogram per is set to Report

    and

  2. the antibiogram is exported before all cultures

then the antibiogram will only contain the isolations exported so far.

Note

If the report includes an antibiogram and

  1. the Report template option Show single antibiogram per is set to Isolation or Specimen

    and

  2. the antibiogram is exported before its corresponding culture

then the Renumber isolations option is ignored.
