Microbiology reporting - Antibiogram customization

In the report template (in the tab folder Microbiology), some options are available to customize the antibiogram on reports.

Show single antibiogram per

The option Show single antibiogram per has the following possible values:

Isolation

Text reports will not show multiple columns, but the antibiograms of each isolation will be put one below the other: first the header for isolation 1, then the results for isolation 1, then the header for isolation 2, then the results for isolation 2, etc.

Example

If the option is disabled:

            1. XX   2. YY
  Amp       R
  Gent      S       I
  Cefa              R
  Norflo    S       R
  Oxa       I

If the option is enabled:

1. XX
  Amp       R
  Gent      S
  Norflo    S
  Oxa       I
2. YY
  Gent      I
  Cefa      R
  Norflo    R

Specimen

The antibiograms will be grouped per specimen. A single antibiogram will be printed per specimen showing columns for all isolations detected on that specimen.

Report

In addition to printing an antibiogram per isolation and per specimen, it is also possible to print a single antibiogram per report.

Whereas the antibiogram per specimen has headers per isolation of the form [Isolation sequence], the single antibiogram per report has headers of the form [[specimen sequence].[isolation sequence]] to ease data comparison.

Note

In order for this mechanism to work correctly, you must also check your property classification configuration. There should only be one antibiogram node and it should not be placed within a specimen loop, but come after that loop.

Include isolations not reported earlier

Purpose

This option decides whether an antibiogram should be included on a report if its isolation has not been reported earlier on in the report. Such cases may arise when:

  • in the applicable property classification, the antibiogram is positioned before the culture property that is used to report the isolation.
  • the culture property that should be used to report the isolation is not included in the property classification at all.

Possible values

Automatic (default)

  • If at least 1 isolation has been shown earlier on the report, then the antibiogram of isolations not yet shown will also NOT be shown.
  • If NO isolation has been shown at all before the antibiogram, then all antibiograms will be shown.

    • Never

    Antibiograms for isolations that have not been reported earlier in the report will be hidden.

    Always

    Antibiograms for isolations that have not been reported earlier in the report will always be printed.

    Note

    If this option is set on Always and the Report template option Renumber sequence numbers on report is enabled, renumbering will not be done during report generation.

    Character to use in empty spaces

    This character will be printed in all empty places (cells) in the antibiogram, i.e. where there is no combination of antibiotic and isolation. This is where the testing of the antibiotic on that isolation was not requested, usually because the antibiotic is not a member of the selected antibiotic panel. The default for this field is " " (blank).

    Example

    Suppose an antibiogram looks by default like this:

            1  2  3 
AMP     R  I  S 
CEFA    S     I 
PEN           S

    If the character for empty spaces was changed into e.g. '-', then the antibiogram would look like:

            1  2  3 
AMP     R  I  S 
CEFA    S  -  I 
PEN     -  -  S

    Hide cells with unknown value

    Disabled (default)

    By default, antibiotic results without a value will be reported in the antibiogram as a '.' sign.

    Example 

            1  2  3 
AMP     R  .  .
CEFA    .     .
PEN           S

    Enabled

    The report will look as if these antibiotic results (without a value) were never requested.

    Example 

            1  3
AMP     R
PEN        S

     

    Note

    If the option Character to use in empty spaces is used, the unknown values will not be shown as '.' but as '?' by default. This is because the dots would visually be lost between the characters for empty spaces.

    So the above example antibiogram would become (with e.g. '-' as empty space character):

            1  2  3
AMP     R  ?  ?
CEFA    ?  -  ?
PEN     -  -  S

    Tip

    If the unknown values should be shown in a different way than this default (e.g. as "Not Tested"), use the Report template field Show values as to achieve this.

    Example:

    STRING str;
str := ?;

IF .Antibiotic <> ? AND .RISReportValue = ? THEN 
   str := "Not Tested"; 
ENDIF;

RETURN str;

    Split in two columns when line count >

    Allows specifying that when the number of lines in an antibiogram on a text report exceeds this number, the antibiogram should be split in two parts which will be printed next to each other.

    Example

    If this option is set to 4, an antibiogram like

    Antibiogram     1   2
PEN             R
AMP                 S
AUG             R
NOR             S

    would be printed on a text report as

    Antibiogram     1 2   Antibiogram      1 2
PEN             R     AUG              R
AMP               S   NOR              S
    Notes

    Note

    In this mode, the distance between the antibiotic result columns is 1 space instead of 3 spaces.

    Note

    This is only supported for text reports (not HTML reports). The use of comments in the antibiogram may disturb this mechanism. Therefore, it should only be used for 'simple' antibiograms.

    Germ headers

    This field can be used to customize the header information for each germ in the antibiogram. It is a MISPL function based on Isolation, which returns a String. This string will be put on top of each antibiogram column as information on the tested germ.

    Example 

    RETURN ToString(.InternalSequencer, "%d") + ". " +
IF .Microorganism.ShortName <> ? THEN 
   .Microorganism.ShortName 
ELSE 
   .Microorganism.Mnemonic);

    Tip

    If the option Show single antibiogram per is set to Isolation, this customized header will be put above the antibiogram, using the whole width. In that case it is also allowed to contain multiple lines.
