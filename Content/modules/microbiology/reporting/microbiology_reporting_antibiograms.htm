Microbiology reporting - Antibiograms

How are antibiotic results reported

Antibiotic results are reported via the Antibiogram property of the micro-organism. This is a dummy property which must have Data type set to Antibiogram.

By including this Antibiogram property in a property classification, you can determine where the antibiotic results should be put on the report.

Tip

We strongly recommend to place the Antibiogram property AFTER the microbiology procedure's Report property !

Which antibiotics are reported

  • An antibiotic result can only be reported if the Sequence number is specified for the Antibiotic!
  • For each antibiotic, the field Auto reportable can be set in order to report the antibiotic result automatically. However, in the antibiogram browser of the microbiology screen, one can use the 'p' key to change the reportability of an antibiotic result.
  • If an isolation is marked as non-reportable, then the linked antibiotic results will not be reported either.

Which antibiotic data are reported

The report will display:

  • The antibiotic result's RIS report value
  • The antibiotic result's External comment / footnote

RIS report value

An antibiotic result contains a RIS raw value and a RIS report value.

The RIS report value is the one printed on the report. In most cases both RIS values are identical but, it is possible that a raw value is modified by the MISPL function Microorganism.AntibiogramConfirmation before it is printed on a report.

During report generation, the actually printed antibiotic result value can be modified using a MISPL expression defined in the Report template field Show values as.

External comment / footnote

Each antibiotic can contain External comments or External footnotes which are automatically displayed in antibiograms.

Tip

The Report template field Comments can overrule the fixed antibiotic comment.

The MISPL expression can return a fixed string (also the empty string), or can browse into .Antibiotic and .Specimen to compose a string depending on those tables and their fields.

If this MISPL is not specified or returns ? then the external comment of the antibiotic will be reported.

Sequence of the antibiotics

The antibiotic's Sequence number determines the sequence of the antibiotics on the report.

Tip

The Report template field Sort by can overrule the fixed antibiotic sequence number. If no MISPL expression is defined, or the MISPL function is available but returns "?", then the antibiotic's sequence number will be used.

Example

The following MISPL expression sorts by name if the order was issued by urgency departments, else it sorts by the antibiotic's sequence number.

IF SUBSTR(.Isolation.Action.Order.Issuer.Name,1,3) = "URG"
THEN
   RETURN .Antibiotic.Name;
ELSE
   RETURN "?";
ENDIF;

The following MISPL expression sorts by mnemonic except for Dr. Zivago: take special sort sequence defined in site attribute on antibiotic table.

IF .Isolation.Action.Order.Issuer.Name = "Zivago" THEN
   RETURN GetSiteAttribute("Antibiotic", .Antibiotic.Id, "SequenceZIVAGO");
ELSE
   RETURN(.Antibiotic.Mnemonic);
ENDIF;
