WASPLab asks GLIMS which plates it should use for grafting

 

WASPLab asks GLIMS which plates it should use to examine a specimen.
In its reply message, GLIMS lists the carriers to be used.

WASPLab

For a specific specimen, WASPLab sends a query message to retrieve the tests (microbiology actions) and carriers. The query message contains the specimen number.

GLIMS

GLIMS sends a reply message that contains

  • for each microbiology action on the specimen: grafting property
  • for each carrier on the microbiology action: medium mnemonic and carrier number
  • for each normal result assessed on the specimen: property mnemonic and result value

Communication protocol

For detail information about the communication messages, please check the chapter GLIMS internal ASTM protocol.

Actual grafting

The carriers are now automatically grafted by WASPLab.

WASPLab sends GLIMS grafting results

At regular intervals, WASPLab sends grafting results to GLIMS.

WASPLab

WASPLab regularly sends grafting results:

  • Reference time = '0h' (grafting time).

  • Completion time: time when grafting was finished.

  • Grafting status

    S success  
    R temporary failure (re-workable rejection)  
    X failed (permanent rejection) Not enough material, unknown processing, SID unknown to the LIS, etc.
    U unfinished This is not an error status and it can be sent along with a successful M-type report if not all the media ordered for that test (e.g. 'GNMDR') have been streaked yet.

  • Lot number of the plate that was used.

  • Expiration date of the plate that was used.

  • Operator name (only for manual streaking station)

  • Aggregate line contains grafting result of all carriers (for same microbiology action) as a summary.

GLIMS

The grafting results are stored in GLIMS:

  • The grafting status and completion time are stored in each carrier.
  • The lot number and expiration date are stored in the MBReagent lot to which the carrier refers.
  • If available, the operator name id stored.
Note

Note

If the carrier is not yet linked to an MBReagent lot, then

  • GLIMS first tries to find the lot number for whichever MBReagent linked to the Medium (of the carrier).
  • If such a lot is not found, GLIMS looks for or creates an MBReagent for a supplier corresponding to the Own organisation and creates a new lot for this MBReagent.
