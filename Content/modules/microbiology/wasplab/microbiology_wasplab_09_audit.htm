Detailed information about the communication between GLIMS and WASPLab is stored in an audit trail for Carriers and Microbiology actions.

Carrier audit trail

All result messages sent by an analyzer and containing carrier information are automatically logged for the corresponding Carrier record.

Activation

To be able to consult the created logs, you first need to activate the audit on the Carrier table. To do so, go to Start -> System management -> Logging -> Log types -> Audit Carrier and activate the Enabled option.

Note

As every change on a carrier is logged, the genrw-database will be populated quickly.

Warning

Users with the role of “System administrator” or “Developer” might be exempt during auditing, so ensure that this is not the case by enabling logging/auditing for those roles or that any service used for analyzer communication (i.e. translator service) is using a batch user that does not have any of those 2 roles. Otherwise any change applied to a carrier as a result of an incoming message will not get audited, only the receipt of the message will get audited, but, again, any subsequent change from such an incoming message to the carrier record will not get logged. This could falsely create the illusion that the incoming message had no effect on the carrier, while it actually did have an effect on it.

Consultation

To view all changes related to a carrier, select a carrier and choose the contextual function Audit trail.

The audit trail shows:

  • All log entries for the carrier.
  • All log entries for all isolations detected on the carrier.

Microbiology action audit trail

To view all changes related to a microbiology action, select a microbiology action and choose the contextual function Audit trail.

The audit trail shows:

  • The Internal log of the microbiology action
  • All log entries for the microbiology action.
  • All log entries for all isolations of the microbiology action.
  • All log entries for all carriers of the microbiology action.
