Microbiology work screen - Antibiogram

Antibiogram browser

Content

Default

For each detected isolation, the antibiogram will show the antibiotic panels linked to the microorganism of the isolation.

Add antibiotics

  • Add an antibiotic panel: use the Insert key
  • Add a single antibiotic: use the CTRL-Insert key combination

The newly added antibiotics will be shown at the bottom, in the order defined in the selected panel.

Display

Columns

The antibiogram browser displays the different antibiotic panels for each isolation (1,2,3,4 etc.).

The first column (C) contains an asterisk (*) if the antibiotic result has a comment.

Title

When you enter one of the columns of the antibiogram, its title will contain the short name of the microorganism to which that column relates.

Sorting of antibiotics

By default, the antibiotics are sorted according to their sequence number within the selected panel.

  1. Antibiotic results that were generated from a panel are sorted by their sequence number in that panel.
  2. Antibiotic results that were not generated from a panel are sorted by the sequence number in the antibiotic configuration.

This default behaviour has the following consequence:

When selecting another column (by clicking in a cell or using the TAB key), the antibiotic sequence can change. This is because a different antibiotic panel, which contains the same antibiotics but sorted in a different order, may have been requested.

If you do not want the antibiotic sequence to change when selecting another column, uncheck the option When inside the antibiogram, sort according to the antibiotic panel of that column in the Options of the microbiology work screen.

Notes

  • To see the order of antibiotics as they will appear on reports, single-click the header of the antibiotic name column.

  • Any antibiotics that do not have a sequence number within the panel are displayed directly after the other antibiotics of the panel. The same applies to manually added antibiotics.

Changing the order of the columns

You can change the order of the columns by dragging the column headers to the left or to the right. This can be useful e.g. for comparison or a better overview if there are many empty columns between two non-empty columns.

Background colors

Antibiotic name

Antibiotics without sequence number are never displayed on reports.

Therefore, the names of such antibiotics will have a gray background.

RIS value

The background color of the R/I/S results adapts to the value:

  • 'R' appears in red background,
  • 'I' appears in yellow background,
  • 'S' appears in green background.

This behavior can be disabled by:

  • right-clicking in the antibiogram and choosing Antibiogram -> Use R/I/S colors.
  • disabling the microbiology option Use R/I/S colors.

Note

When an antibiotic (or its related isolation) is marked as non-reportable, the value will be displayed in gray.

Status / Font

The text font reflects the status:

Status Font
Available normal
Confirmed bold italic
Validated bold
RIS report value changed after confirmation underlined (see below)

Entering values

Note

When an antibiotic result in status Validated is changed, a justification (reason) will be asked.

A log entry containing this reason will be created. The audit trail of the antibiotic result will display an extra entry with the content of this log:

Validated antibiotic result modified, reason: [reason]

Entering a RIS value

The antibiogram allows entering RIS values. The supported RIS values are:

Value Description
R Resistant
I Intermediary
S Susceptible / Sensitive
U Susceptible urine
D Susceptible dose-dependent
V Variable MIC limits
E Insufficient evidence
- Not tested

Entering a non-RIS value

When entering a non-RIS value (e.g. numeric value), a screen will be displayed which allows to specify the type of result (MIC, Agar diffusion or E-test).

GLIMS allows to limit the allowed values by specifying a choice list in the allowed values of the antibiotic.

Choice list is not specified

Any value is accepted.

Choice list is specified

Only choices linked to the specified choice list are accepted as values.

Note

If the choice list option Allow free text is enabled, the user will also be allowed to specify free text instead of one of the pre-defined choices. In this case, a confirmation message will be displayed.

Format

MIC values can be directly entered as numeric values or can be preceded by "<", ">", "<=" or ">=".

MIC values of antibiotic results are stored as strings, not as decimal values. The string value is accessible via the field .MICRawValue of the AntibioticResult table.

Conversion

Numeric values can automatically be converted to a RIS values if a conversion rule is defined for the antibiotic. If no conversion rules are defined, the RIS value will be left unchanged (or remain unknown).

The Formula type (defined in the antibiotic panel or antibiotic) determines which type of value (MIC, Agar diffusion or E-test) will actually be used to determine the final RIS raw value.

Tip

In the zoomed antibiogram window, you can enter different types of values and you can even select which value should be used for the conversion to an RIS value (overriding the Formula type)

Changing the reportability

If the Antibiotic option Auto reportable is enabled, antibiotic results for this antibiotic will by default be included on reports.

To change the reportability of an antibiotic result in the antibiogram window:

  1. Select a RIS value.
  2. Press the p key.
  3. The color of the value will become gray.
  4. Press the p key again, to make the value reportable again.

When the reportability of an antibiotic result is changed, the status of the microbiology action automatically demotes to ReReading, so that confirmation, validation, and report generation must be re-done.

Notes

  • Confirmed isolations that are reportable can not be reset to non-reportable.
  • When resetting non-confirmed isolations from reportable to non-reportable, a warning message will be displayed with the option to cancel the action.
  • When confirming a non-reportable isolation, a warning is issued that the isolation will not appear on reports.
  • Changing the reportability of the isolation is also logged in the isolation's audit trail.

Changing the micro-organism of an isolation

When changing the micro-organism of an isolation (change from one organism into another organism), GLIMS will by default automatically remove all antibiogram cells for that isolation that did not have a value yet. Depending on the new organism, another antibiogram can be added automatically as well.

However, some labs prefer that the empty antibiogram cells are NOT removed automatically when the organism is changed. This can be accomplished by enabling the option Keep whole antibiogram when organism is changed in the general settings.

Zoomed antibiogram

Opening the zoomed antibiogram screen

From the antibiogram window in the microbiology screen, you can open a zoomed antibiogram screen:

  • Click the spyglass button in the top right corner of the antibiogram window

    OR

  • Use the CTRL-Z key combination when positioned in the antibiogram browser.

Content

Isolations

The detected isolations

Antibiotics

For each isolation, the antibiotics to be tested.

MIC, AgDif, ETest

These columns allow to view or enter MIC, Agar diffusion and ETest values. Each column is followed by an extra column that can show the corresponding RIS value (if a conversion rule is defined).

The value displayed in red indicates that this value determines the final RIS raw value. The Formula type configured in the antibiotic panel (or in the antibiotic for manually added antibiotics) decides which value will be copied to the raw value by default. This can be changed with the Select buttons (see Features).

Raw

The raw value can be

  • manually entered.
  • calculated by the conversion of a non-RIS value.

Report

The report value is the value displayed on reports. Upon confirmation the Raw value is copied to the Report value.

Tip

By default, the Raw value and Report value will be identical.

However, GLIMS does allow to change the report value. For more information, please check the chapter Microbiology reporting - Antibiograms.

Sts

Status of the antibiotic result.

Comment

Allows specifying or updating the comment directly in the browser when the microbiology action status has not yet reached status Closed.

Features

Select all

Allows indicating which value (MIC, Agar diffusion or E-Test) must be used to determine the actual raw RIS value, overriding the default formula type defined in the antibiotic panel or in the antibiotic.

See also: entering values.

Note

When using the Select all button, this will be applied to ALL antibiotics of the isolation.

Select single

Has the same effect as the Select all button but for a single selected antibiotic result.

Confirm

Allows confirming the entire antibiogram.

External antibiogram screen

Advantages

The external antibiogram screen combines functionality of the antibiogram browser and of the zoomed antibiogram. It offers the following advantages:

  • The external antibiogram screen does not only display RIS values (contrary to the antibiogram browser).
  • The external antibiogram screen allows to continue working in the microbiology work screen when both screens are open (contrary to the zoomed antibiogram).

Opening the external antibiogram screen

The external antibiogram screen can be opened:

  1. manually from the microbiology work screen via the button (which can be found next to the Refresh button). The button disappears when the external antibiogram screen is open and reappears when the external antibiogram screen is closed.
  2. automatically when opening the microbiology work screen if the option Show the external antibiogram screen when opening the work screen is enabled.

Notes

When the external antibiogram screen is open, the antibiogram browser in the microbiology work screen disappears, leaving more space for the other browsers in the microbiology work screen.

Tip

The external antibiogram screen is ideally used in a two monitor setup, where the microbiology work screen is displayed on one monitor and the external antibiogram screen on another.

Content of the external antibiogram screen

Screen layout

The external antibiogram screen consists of a browser where the main columns are the detected isolations and the rows are the antibiotics to be tested. For each isolation, multiple columns are available which display the different types of antibiotic result values.

Sorting of the antibiotics

The antibiotic's Seq no determines the sequence of the antibiotics in the external antibiogram screen.

MIC, AgDif, ETest

These columns allow to view or enter MIC, Agar diffusion and ETest values. The value displayed in red indicates that this value determines the final RIS raw value. The Formula type configured in the antibiotic panel (or in the antibiotic for manually added antibiotics) decides which value will be copied to the raw value by default.

Raw / Report

For more information on these columns, see Zoomed antibiogram.

C / Comment

The first column (C) contains an asterisk (*) if the antibiotic result has a comment. The actual comment for the selected antibiotic result is displayed / can be modified at the bottom of each main column.

Note

Use the Enter key after entering or modifying a comment. Otherwise, the comment will not be saved.

Buttons

Confirm

Allows confirming the entire antibiogram.

Close

Allows to close the external antibiogram screen.

Zoomed antibiotic result

When zooming into an antibiotic result (F6) in the antibiogram browser or the zoomed antibiogram, you will see the originally entered result.

  • There is a RIS raw value and a RIS report value (see Microbiology reporting - Antibiograms).
  • If an antibiogram result has been changed after confirmation of the antibiogram, the RIS report value changed option will be enabled. The antibiotic result in the antibiogram will be underlined (see above).
  • The comment can be entered or updated when the microbiology action status is not yet Closed.

Copy antibiogram

When using the Isolation context function Copy antibiogram, a browser is shown with isolations of the same object and organism that have an antibiogram. You can choose one of these isolations.

Its antibiogram will be copied to the isolation you started from. In case the isolation already contains antibiotic results for the same antibiotic, you will be asked whether to overwrite the existing results. If yes, the existing results with the same antibiotic as the new results will be overwritten, otherwise they stay as they are. All other new antibiotic results will be copied to the isolation we started from.

The audit trail of the antibiotic result is updated with the information that this result was copied from a result of another isolation. The specimen, organism and sampling date of that isolation are mentioned.
