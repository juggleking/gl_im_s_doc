Microbiology work screen - General info & options

Screen title

The following information is shown in the title of the microbiology work screen:

  1. Default: specimen internal id

  2. Text module specified in the field Title bar of the general microbiology settings. Replaces the default.

  3. Specimen internal id - <SpecimenInfo>: where <SpecimenInfo> is a Text module specified in the field Specimen info of the general microbiology settings. Only applies when no Title bar is specified.

    Note

    If the Specimen info text is too long to fit in the screen title, an error may occur. To avoid this, you can add a "new line" in the specimen info text, as the screen title will only use the part before the first new line.

Information windows

In the left upper corner, you will find patient and specimen information.

Custom content

The content of these info windows can be customized via the fields Object info and Specimen info in the general microbiology settings.

Merged info windows

If you choose the same text module for both Object info and Specimen info, the work screen will show only one info window, taking a larger size.

Patient history

The Object history button allows the user to consult historical results of the patient. Clicking this button allows to

  • show all detected isolations with antibiogram
  • show all microbiology actions
  • show all results available for this patient, grouped by object time
  • show a result overview for this patient, grouped by property

Options

The Options button allow the setting of user-specific parameters. These settings will be saved for future sessions of the same user.

Show column with creation date of isolation tests

If set, an extra column with the creation date is shown in the isolation test browser.

Only show the tests of the focused isolation

If set, the isolation test browser will only contain the tests for the selected isolation.

Show full names/values for isolations, tests, results (read only)

When this option is enabled:

  • for isolations, the microorganism's short name (if available) will be shown instead of its mnemonic.
  • for carriers, carrier tests and isolations tests: if a value from a choice list has been chosen, the name of this choice will be shown instead of its mnemonic.
  • for results, the "expanded value" (like on reports) will be shown instead of the entered value.

Note

If this option is enabled, the idea is to view the results, not to change or add things. This option is also available through a button in the main window (see Show full names).

Use R/I/S colors

If set, the background color of the R/I/S results adapts to the value:

  • 'R' appears in red background
  • 'I' appears in yellow background
  • 'S' appears in green background

Show phone status of results

If set, an extra column labelled 'P' is added that shows the phone status of each result.

Show comment indicator

If set, an extra column labelled 'C' (in front of the Value column), which contains a '*' if the result has comments (internal or external or request comments).

Use function keys F7 and F8 for confirmation and navigating to next (F7=ReadConfirmed F8=Ready)

If set, you can use function keys F7 and F8 for confirmation of the screen, combined with navigating to the next microbiology action in the list (the one below the current one - like hitting PAGE-DOWN).

F7 = set status to 'ReadConfirmed' (confirmed preliminarily)

F8 = set status to 'Ready' (confirmed finally)

Note

When the microbiology work screen is opened from a single microbiology action (and not a browser), it is not possible to navigate to the next action. In such cases, GLIMS will close the microbiology work screen after confirmation.

Use function keys F11 and F12 for validation and navigating to next (F11=ReadValidated F12=Closed)

If set, you can use function keys F11 and F12 for validation of the screen, combined with navigating to the next microbiology action in the list (the one below the current one - like hitting PAGE-DOWN).

F11 = set status to 'ReadValidated' (validated preliminarily)

F12 = set status to 'Closed' (validated finally)

Note

When the microbiology work screen is opened from a single microbiology action (and not a browser), it is not possible to navigate to the next action. In such cases, GLIMS will close the microbiology work screen after validation.

Also show unconnected results on same specimen

When this option is disabled, the microbiology screen will no longer show the results and procedures that will not automatically be confirmed or validated together with the microbiology action.

This means that:

  • If the option Also conf/val loose results on same specimen in the general settings is disabled:

    It will hide all results and procedures that are not linked to the microbiology action via grafting/microscopy/processing properties.

  • If the option Also conf/val loose results on same specimen in the general settings is enabled:

    From all results and procedures that are not linked to the microbiology action via grafting/microscopy/processing properties, only those where the option on the procedure Don't confirm/validate with microbiology is enabled will be hidden.

    On the other hand those 'unlinked' results that are still shown (because the option on the procedure is disabled), will be shown in italic.

The option might thus also be called "Show also results that will not be confirmed/validated together with this microbiology action". See also Confirmation/validation of related results.

Number of lines in the info fields (on top)

Allows to customize the height of the object / specimen info boxes on top of the microbiology screen. The info boxes can have a minimum height of 1.5 lines and a maximum height of 3 lines. By default, this field will be set to 1.5.

Number of rows for the isolation and carrier browser

Allows to customize the height of the carrier / isolation browsers in the microbiology screen. The carrier / isolation browsers can have a minimum height of 4 rows and a maximum height of 20 rows. By default, this field will be set to 4.

When inside the antibiogram, sort according to the antibiotic panel of that column

If this option is activated, the antibiotics in the antibiogram browser will be sorted according to the sequence number they have been assigned in the currently selected panel. As a consequence, when clicking in another column, i.e. when selecting another panel, the order of the antibiotics might change. This is because the antibiotics might have a different sequence number in this other panel.

If you do not want the order of the antibiotics to change when selecting another panel, deactivate this option.

Show the extended version of the carrier browse

If enabled, the carrier browser in the microbiology work screen will show more detailed information such as grafting results and growth results.

Show results that are discontinued

If enabled, discontinued results will be shown in the Results of specimen browser of the microbiology work screen.

Note

By default, this option is disabled and discontinued results are not shown.

Show the external antibiogram screen when opening the work screen

If enabled, the external antibiogram screen will be opened when opening the microbiology work screen.

Microbiology procedures

If several microbiology procedures have been requested on the same specimen, a radio set shows these microbiology procedures.

The procedures are sorted in the same order as they were requested. If multiple microbiology procedures have been requested with one request panel, the sort order within this panel (defined by the sequence number) will also be evaluated.

Example

Suppose you have defined procedures 'Aero' and 'Anaero' on 'Urine'. When you create an order and first request 'Urine-Aero' and then 'Urine-Anaero' then the screen will show:

Procedure: o Urine-Aero o Urine-Anaero o All

Station

This field shows the station linked to the microbiology action. This can be:

  • The station defined in the Initial field of the microbiology procedure
  • The station determined by the site function specified in the Next field of the microbiology procedure

You can also manually change the station.

Status

Allows changing the status of the microbiology action. Please check the chapter Status overview of microbiology actions for more information.

Neg. answer

Phone status

Possible values

The phone status can be modified to one of the three values and the new value is saved immediately.

No

The results should not be phoned.

Yes

The results should be phoned.

Done

The results have been phoned.

If the phone status is set to Done, the phone-through screen appears.

The selected results are shown, the telephone numbers associated with the order can be selected and a comment can be added.

  • If the Phoned button is clicked, the phone status is set to Done and an order phone log Phoned is saved.
  • If the Not phoned button is clicked, a comment must be added and an order phone log Not phoned is saved. No new phone status will be saved.

Phone status is hidden

If there is no Report result for the current microbiology action, the phone status is hidden. If the microbiology screen is read-only, the phone status will also be read-only.

Several procedures

If several procedures are requested by the microbiology procedure, a radio set with label Procedure is displayed in the microbiology screen. This can be set to All. If this is the case, the phone status is read-only and the possible values are:

  • If the phone status of one of the procedures is Yes the phone status of "all" procedures will be Yes.
  • If none of the procedures has phone status Yes and one procedure has status Done then the phone status of "all" procedures will be Done.
  • If the phone status of all the procedures is No, the status of "all" the procedures will be No.

Icon with yellow phone

This icon only appears if the currently visible action is to be phoned. Clicking it has the same effect as choosing status Done in the phone status list.

Check the chapter about the phone list program for more detailed information.
