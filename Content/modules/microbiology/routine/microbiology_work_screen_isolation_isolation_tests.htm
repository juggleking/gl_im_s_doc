Microbiology work screen - Isolation & isolation tests

Isolations

Adding isolations

Insert key

When using the Insert key to add a new isolation, an isolation with unknown (?) microorganism is added.

Tip

If a microorganism is specified in the Initial microorganism field of the microbiology procedure, this microorganism will by default be chosen when adding a new isolation via the Insert key.

This avoids having to specify the microorganism for microbiology procedures where it is always the same organism (or organism family) to start with.

* button or CTRL-Insert key combination

When using the * button or the CTRL-Insert key combination, the Initial microorganism field of the microbiology procedure is ignored and the user will have to select a microorganism from a list.

MISPL

Isolations can also automatically be added using the following MISPL functions:

  • MicrobiologyAction.AddIsolation
  • Carrier.AddIsolation
  • Carrier.AddIsolationWithAppraisal

Status change

When an isolation is added in the microbiology screen, the status of the microbiology action is automatically promoted to Reading if it was lower.

Columns

First column

Contains a reference (1, 2, etc...) which is also used in the other browsers to refer to the specific organism.

Second column

Carrier to which the isolation applies. When entering a new isolation, the focused carrier will be taken as default.

Third column

Mnemonic of the microorganism

Note

When the microorganism of a validated isolation is changed to another one that is unrelated to the previous one (no child, grandchild, etc...), a log entry about this specific change will be created.

The audit trail of the isolation will display an extra entry with the content of this log:

Validated isolation modified, organisms [old morg] and [new morg] unrelated

Fourth column (Apprl)

The isolation appraisal of the microorganism. You can manually type in an appraisal or double- click to select one from a list.

Tip

If an Initial appraisal is specified in the microorganism configuration, this appraisal will be used as default. This allows confirming isolations without having typed in an appraisal manually.

Fifth column (Rp)

Indicates if the isolation should be printed on reports. Click in the cell to change the reportability.

See also: Mark as reportable.

Note

When an isolation is marked as non-reportable, the foreground color of the microorganism's mnemonic will become gray.

In addition, the related appraisal, isolation test values and antibiotic results will also be displayed in gray in the microbiology work screen, the zoomed antibiogram screen, the external antibiogram screen and the isolation browser.

The icons in the column header allow the user to:

  1. hide / show non-reportable isolations,
  2. sort the isolations by whether they are reportable.

Sixth column (Cf)

Click in the cell to set the status of the isolation to Confirmed.

You cannot undo this confirmation.

Seventh column (AB)

Click in the cell to add/remove the antibiogram for the selected isolation.

  • If an antibiotic panel is linked to the microorganism of the isolation, it will automatically be added to the antibiogram.
  • If no antibiotic panel is linked to the microorganism of the isolation, you can select an antibiotic panel manually.

Note

When the user clicks in the Ab cell to undo the antibiogram selection for an isolation (the [X] is removed), the linked antibiogram results for which no result value has been entered yet will be removed.

However, the already entered antibiotic results of the isolation in question remain available for reporting. Removing the [X] will then result in the appearance of [*].

I.comm

The internal comment field allows to enter comments which are for internal use only.

Note

When changing the microorganism of an isolation (e.g. GramNeg is changed to Ecoli), the previous microorganism will be logged in the Audit trail. This audit trail is accessible via the contextual function on Isolation.

E.comm

The external comment field can be displayed on reports.

External id

Some microbiology instruments use their proper isolation sequence numbers to distinguish isolations from each other. Those numbers can be stored upon result upload. The microbiology screen shows those external numbers in the isolation browser (whenever the numbers are filled in).

Note

The columns of the integrated isolation browser can be resized if ResizeInCell=NO is set in the MA Section of the progress.ini file.

Isolation statuses

The microbiology screen differentiates between isolations that have already been validated and isolations that have not yet been validated (available or confirmed).

Confirming an isolation

An isolation can be confirmed by clicking in the Cf cell of the isolation browser.

Validating an isolation

When a microbiology action is promoted to status ReadValidated, all contained isolations will be promoted to status Validated.

Note

When status of the microbiology action is demoted to ReReading, the isolation status remain Validated (unless e.g. the organism is changed).

Color usage

Different background and foreground colors can be defined to distinguish between validated and non-validated isolations. GLIMS uses the color scheme defined for the Result table.

Copy antibiogram

For a selected isolation, you can copy the antibiogram from another isolation.

  1. Select an isolation.
  2. Choose the contextual menu item Copy antibiogram.
  3. A list of isolations (having an antibiogram) will be shown for the same object and organism.
  4. Select one of these isolations and its antibiogram will be copied to isolation you started from.

Audit trail / Isolation history

Audit trail

Select an isolation and use the contextual function View audit trail.

Isolation history

Select an isolation and use the contextual function Show all > Organism history.

Isolation tests

Adding isolation tests

Manually

The isolation test browser allows to add additional tests which can contribute to the determination of the microorganism.

  1. Select the isolation in the isolation browser for which you would like to execute an isolation test.
  2. Choose the contextual function Generate -> tests.
  3. The isolation tests (linked to the isolation by means of a microorganism test usage) will be displayed in the isolation test browser.
  4. When a Default value is specified for a microorganism test, a newly created isolation test of that kind will automatically get this value.

MISPL

Isolation tests can also automatically be added using the MISPL function Isolation.AddTest.

Hierarchical structure

Isolation tests can be structured in a hierarchical way. This means that one isolation test can have other isolation tests as sub-tests, and these sub-tests can also have sub-tests etc. This hierarchical structure is also visible in the microbiology screen. The Â» sign is used to indicate the level depth.

Sub-tests of isolation tests can be added in 2 ways:

  • with the context function Add sub-test on an isolation test (or many) you can manually add a sub-test.

  • with the MISPL function '.AddSubTest( <testmnemonic> )' you can automatically add a sub-test from within MISPL.

    Tip

    When the result value of an isolation test is entered or modified, a trigger will be executed that is defined on the microorganism test. This can be used e.g. to add a sub-test.

Window options

Minimize / Maximize

The isolation test window can be minimized / maximized by clicking the arrow icon in the top right corner. The carrier test window will occupy the free space. The carrier test window can also be minimized to provide more space for the isolation test window .

Width

Double click the title bar of the isolation test window to enlarge the width. This can be useful to provide more space in case of hierarchically structured sub-tests.

Note

When the width of the isolation test browser is enlarged, it can overlap the antibiogram. As soon as you enter the antibiogram, the default width of the isolation test browser will be restored.

Only show tests for focused isolation

It is possible to only show the isolation tests executed on the focused isolation. There are two ways to achieve this:

  • Enable the option Only show the tests of the focused isolation via the Options button at the top left of the microbiology screen.
  • Click the Test column header in the isolation test window.

Printing isolation test labels

Click the Printer icon in the isolation test browser to print the isolation test labels.

Note

If the microbiology screen option Only show the tests of the focused isolation is enabled, GLIMS will only print the isolation test labels for the focused isolation.

If you want print all isolation test labels, right-click the title of the isolation test window and choose Print -> Print isolation test labels.

Isolation test statuses

Status Description
Expected Isolation test is added, but result is not available yet.
Available A result is available for the isolation test.
Confirmed The isolation test has been confirmed. An isolation test can be confirmed:
  1. By confirming the microbiology action.
  2. By selecting an isolation test (e.g. in the microbiology work screen, a microbiology work list) and using the context function Confirm.
Validated The isolation test has been validated. An isolation test can be validated:
  1. By validating the microbiology action.
Discontinued The isolation test has been discontinued. An isolation test can be discontinued:
  1. By selecting an isolation test (e.g. in the microbiology work screen, a microbiology work list) and using the context function Discontinue.

Note

In the microbiology screen, you can change the value of a discontinued isolation test. Its status will be reset to Available.

Audit trail

All changes to the isolation test values are registered in an audit trail. Select an isolation test and use the context function View audit trail to see a full history.

Reports

The isolation test status can be exported in reports through dynamic text modules.

Example:

IsoTestStatus := EnumeratedToString("IsolationTestStatus", .Test("<MicroorganismTestMnemonic>").Status);
