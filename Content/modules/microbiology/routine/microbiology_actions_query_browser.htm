Microbiology - Microbiology actions

Introduction

When a microbiology procedure is requested during order entry, GLIMS will create a microbiology action (which represents the microbiology examination).

From a microbiology action, you can open the microbiology work screen in which you can register isolations, antibiotic results, ...

The microbiology actions are available from the main menu via Start -> Microbiology -> Browse actions.

Query

Note

The settings of this query are always remembered from the previous run (except for the Object / Patient / Specimen / Order / Encounter specified in the second tab page).

General

Sampled from - to

Allows specifying a minimal and maximal object time. You can specify a relative date (e.g. last week) or a fixed date (select the option <date>) and a fixed time indication (e.g. 12:00).

Material

Allows selecting only microbiology actions executed on this material.

Note

If the specified material has material variables, you will be able to select one of the choices defined for the first material variable, i.e. the material variable with the lowest sequence number.

Procedure

Allows selecting the actions for a single microbiology procedure.

Procedure mnemonic

Allows selecting the actions for all different microbiology procedures with the same mnemonic.

Department

Allows selecting the actions of a specific department. Only departments for which microbiology actions exist can be selected.

Organism

Allows selecting the actions where this organism was isolated.

Type

Allows selecting the actions where this organism type was isolated.

Station

Allows selecting the actions executed on this station.

Pos/Neg

Allows selecting actions of which the result is positive or negative.

Note

The result of a microbiology action is considered to be positive if at least one isolation is registered with a positive appraisal.

Review date

Allows selecting actions with this review date.

Severity

Minimal severity level of the isolations.

Phone status

Allows selecting actions with results that are (not) to be phoned or that have been phoned.

No

Results are not to be phoned.

Yes

Results are to be phoned.

Done

Results have been phoned.

Urgency

Minimal urgency.

Incubation period

Outside

Means that the end of incubation time has passed or is on the present day.

Inside

Means that the end of incubation time lies in the future, after the present day.

For more information, please check the chapter Incubation period.

Status

Specific status

Allows selecting the actions with this status.

To be confirmed

Allows selecting the actions with status Reading and Rereading.

To be validated

Allows selecting the actions with status ReadConfirmed and Ready.

Range

Allows to specify a status range.

Residence type

Select only (or exclude) hospitalized patients (at the date of sampling or now, see Now/Sampled)

In ward(s)

Select only patients that resided in specific ward(s) (at the date of sampling or now, see Now/Sampled).

Tip

To specify one or more wards, you can:

  • Enter the mnemonic of 1 single ward.

  • Enter a wild card specification for the ward mnemonic. Examples:

    • IC* --> all wards where mnemonic starts with 'IC'
    • *01 --> all wards where mnemonic ends with '01'

  • Enter a comma-separated list of multiple entries. Example:

    • 001,IC*,Urg* --> ward '001' + all wards where mnemonic starts with 'IC' or 'URG'

It is possible to lookup a ward by double-clicking in the field or hitting CTRL-F. The ward(s) selected in the browser will be added to the existing list.

Now/Sampled

Applies to Residence type and In ward(s). See above.

Object attribute

Only patients having this object attribute.

Note

You can also indicate whether you would like the patients who had this attribute in the past (end date of object attribute is specified and in the past) or whether you want to see the patients that currently have the object attribute (end date not specified or in the future).

Executing dept, Executing department group

A microbiology action will be selected if:

ANY of these departments:

  • MicrobiologyAction.Department
  • GraftingResult -> Department
  • ProcessingResult -> Department
  • MicroscopyResult -> Department

Matches ANY of the following executing departments:

  • The specified Executing Department
  • OR any of the executing departments in the specified Executing department group

Filter

Site function to define additional criteria that should be met before the action is selected.

Specific

Object, PIN, Patient, Spouse name

An object can be selected by entering the patient's name, spouse name or PIN.

Issuer

Allows selecting actions for orders from this issuer.

Issuer group

Allows selecting actions for orders issued by members of this correspondent group.

Order

Allows selecting actions for the order with this internal id.

Short id

Allows selecting actions for the order with this short id.

Encounter

Allows selecting actions with specimens linked to this encounter.

Specimen id

Allows selecting actions with specimens within this specimen id range.

Browser

Info boxes in header

First info box

Shows Object and specimen related information.

The general setting Info header allows to customize the contents of this info box.

Second info box

Shows the isolated organisms and their appraisal. If no isolations have been detected, the negative answer specified in he microbiology procedure is displayed.

Note

Non-reportable isolations are also shown, but preceded by a '~' sign .

Third info box

Displays the normal test results applicable to the selected specimen.

Notes

  • The results are not expanded (like in other browsers and overviews), but the {< } signs are stripped to improve readability.
  • The results are sorted in the same order as in the default property classification;.
  • The results are not shown if
    • the property doesn't appear in the default property classification.
    • or if the Report format of the property is set to Hidden.
    • or if the Display type of the property classification node is set to Never.
    • or if the result is the conclusion of a microbiology action.

Customization

The tool box icon in the lower left corner allows the user to customize the microbiology action browser.

Headers

Show object information

If disabled, the first info box will be hidden.

Show culture information

If disabled, the second info box will be hidden.

Show other related results

If disabled, the third info box will be hidden.

Columns

Show object column

If enabled, an Object internal id column will be shown.

Hide non-reportable isolations

If enabled, the microbiology action browser's Value column will only show the reportable isolations.

Features

Open microbiology work screen

Double click a microbiology action (or use the Enter key) to open the microbiology work screen.

Show microbiology action details

To open a window the microbiology action details, use the F6 key.

Change query option

The Options button allows to refine the query options.

Screen sync select

Please check the chapter Synchronization between GLIMS microbiology screen and Kiestra.
