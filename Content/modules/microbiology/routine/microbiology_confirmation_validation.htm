Confirmation and validation of microbiology action

There are 2 ways to confirm and validate a microbiology action.

In the microbiology work screen

In the upper right corner of the microbiology work screen, two icons allow confirming / validating the microbiology action.

Confirm the microbiology action
Validate the microbiology action
If this flag is disabled, the action will only be Read-confirmed or Read-validated upon confirmation / validation.

Shortcut keys

You can use the following shortcut keys for confirmation / validating microbiology actions in the microbiology work screen.

Status change Key combination Action
Confirmation
To status ReadConfirmed		 PAGE DOWN (or UP) key, while holding down the CTRL key Confirm the microbiology action and go to the next (or previous) in the list.
Final confirmation PAGE DOWN (or UP) key, while holding down the CTRL+ALT keys Final-confirm the microbiology action and go to the next (or previous) in the list.
Validation
To status ReadValidated		 ARROW DOWN (or UP) key, while holding down the CTRL key Validate the microbiology action and go to the next (or previous) in the list.
Final validation
To status Closed		 ARROW DOWN (or UP) key, while holding down the CTRL+ALT keys Final-validate the microbiology action and go to the next (or previous) in the list.

By changing the status of a microbiology action

You can change the status of a microbiology action:

  • by selecting the contextual function Set status on a microbiology action.
  • by selecting a status in the upper right corner of the microbiology work screen.

Check the chapter Status overview of microbiology actions for more details on each individual status.

Confirmation and validation of the antibiogram

When confirming and / or validating a microbiology action, not only the detected isolations but also the antibiogram will also be confirmed and / or validated.

Tip

You can also individually confirm the antibiogram

  • by selecting the contextual function Antibiogram -> Confirm in the antibiogram window of the microbiology screen.
  • by clicking the Confirm button in the zoomed antibiogram screen.
  • by clicking the Confirm button in the external antibiogram screen.

Confirmation and validation of grafting, microscopy , processing results

In the microbiology procedure, a grafting / microscopy / processing property can be specified. These results for these properties are entered / confirmed / validated as follows:

Grafting property

When the microbiology action is promoted to status Microscopy, the Grafting property automatically gets a default value (OK) and is auto-validated.

Microscopy property

When the microbiology action is promoted to status Reading, the Microscopy property automatically gets a default value (OK) and is auto-validated.

Processing property

When the microbiology action is promoted to status (Read)Confirmed or (Read)Validated, the Processing property automatically gets a default value (OK) and is auto-validated.

Confirmation and validation of related results

In the bottom left area of the microbiology screen, related procedures are shown along with a result browser.

When confirming / validating a microbiology action, the following results are also confirmed / validated:

  • Results that are only linked to the current microbiology action.

    Note

    By 'linked', we mean that they are output of the same procedure as the Grafting, Microscopy or Processing property specified in the Microbiology procedure.

  • Results that are linked to the current microbiology action AND to one or more other microbiology actions.

  • Results obtained on the same specimen that are linked to NO microbiology action at all (loose results).

    This behavior only applies when the general setting Also conf/val loose results on same specimen is set to Yes.

    Tip

    Even if the general setting Also conf/val loose results on same specimen is set to Yes, it is still possible to prevent automatic confirmation / validation of the results obtained by a specific procedure.

    To achieve this, enable the option Don't confirm/validate with microbiology in the Procedure configuration.

    This option does not have any effect for procedures that are linked to the microbiology procedure via a Grafting, Microscopy or Processing property.

The following results are not automatically confirmed / validated:

  • Results that are only linked to ANOTHER microbiology action

Synchronizing the status of the report result

The status of the result for the report property (specified in the microbiology procedure) is synchronized with the status of the microbiology action.

Status of microbiology action Status of result for report property
ReadConfirmed Confirmed
ReadValidated Validated

This is important information when defining the Minimal result status in your default report.

Confirmation and validation triggers

GLIMS offers several confirmation and validation triggers which allow to execute a site function when an isolation / antibiogram is confirmed / validated.

Tip

The isolation confirmation and validation triggers can be specified at two levels:

  1. General settings (specific site)
  2. Microorganism

When a trigger is specified at both levels, both will be executed. First the trigger in the general settings will be executed and then the microorganism-specific trigger.

In the general settings

From the main menu, click Start -> System management -> Customize -> Microbiology.

When isolation is confirmed

This trigger is valid for all microorganisms and is executed whenever an isolation is confirmed, or when at least one of its antibiotic results is confirmed.

Note

This trigger is only run for isolations with a positive appraisal.

When isolation is validated

This trigger is valid for all microorganisms and is executed whenever an isolation is validated, or when at least one of its antibiotic results is validated.

In the micro-organism configuration

In the Trigger tab page of the Microorganism configuration screen, you will find the following fields:

When isolation entered

The specified site function will be executed whenever a new isolation is entered for this micro-organism, or when for an existing isolation the organism is changed into this micro-organism.

This applies for manual creation / update via the microbiology work screen as well as for automatic creation / update via analyzer communication.

Example

  1. Create isolation for Germ1: When isolation entered of Germ1 is executed.
  2. Change Germ1 into Germ2 for this isolation: When isolation entered of Germ2 is executed.

Note

If no site function is specified for a microorganism, GLIMS will NOT check if the parent microorganism has a When isolation entered site function. In other words, this field is not inherited from the parent microorganism.

When isolation confirmed

The specified site function will be executed whenever an isolation of this micro-organism is confirmed.

Notes

  • If no site function is specified for a microorganism, GLIMS will NOT check if the parent microorganism has a When isolation confirmed site function. In other words, this field is not inherited from the parent microorganism.
  • This trigger is only run for isolations with a positive appraisal.

When isolation is validated

The specified site function will be executed whenever an isolation of this micro-organism is validated.

Note

If no site function is specified for a microorganism, GLIMS will check if the parent microorganism has a When isolation validated site function. In other words, this field is inherited from the parent microorganism !

Example

Send an email as soon as a MRSA is validated: 

IF .MicroOrganism.Mnemonic = "StAur" AND 
   .AntibioticResult("OXA").RISReportValue = AntibioticResistance["Resistent"] 
THEN SendMail( ... );
ENDIF;
RETURN YES;

When antibiogram confirmed

The specified site function will be executed when:

  • an antibiogram is confirmed (either explicitly or because the microbiology action status is increased)

  • all the antibiogram results have already been confirmed (they have status Confirmed or Validated)

  • the isolation is changed after the confirmation of each of the results in the antibiogram of the isolation.

Note

If no site function is specified for a microorganism, GLIMS will check if the parent microorganism has a When antibiogram confirmed site function. In other words, this field is inherited from the parent microorganism !

The following Isolation-based MISPL functions can typically be used in the specified site function:

  • SetRIS
  • ReplaceRIS
  • EqualRIS
  • HideRIS

Examples

IF .EqualRIS("CTZ","S",YES) OR .EqualRIS("AZT","S",YES) THEN

.HideRIS("IMI","S");

.HideRIS("MER","S");

ENDIF;

RETURN Yes;

 

IF (.Specimen.Material.Mnemonic <> "B_URI") THEN

.HideRIS("NOR","R,I,S");

.HideRIS("FOS","R,I,S");

.HideRIS("FUR","R,I,S");

ENDIF;

RETURN Yes;

 

IF .EqualRIS("AB1","R",YES) THEN

.SetRIS("AB2","R",YES);

.SetRIS("AB3","R",YES);

ENDIF;

RETURN Yes;

Discontinuation of related results

At discontinuation of a microbiology action, the following results are also discontinued:

  • Results that are only linked to the current microbiology action

The following are not automatically discontinued:

  • Results that are linked to NO microbiology action at all (loose results)
  • Results that are linked to the current microbiology action AND to one or more other microbiology actions
  • Results that are only linked to ANOTHER microbiology action.
