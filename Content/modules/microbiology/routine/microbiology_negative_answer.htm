Microbiology - Negative answer when no growth is detected

Introduction

When no growth is detected (no reportable isolations are registered), GLIMS allows to store and report a so-called "negative answer".

Default negative answer

In the microbiology procedure, a Preliminary negative answer and Final negative answer can be defined. Both fields allow specifying a MicrobiologyAction-based text module.

Preliminary

Will be printed when the microbiology action reaches status Read-Confirmed or Read-Validated.

Final

Will be printed when the microbiology actions reached status Confirmed or Validated.

Note

The negative answer is re-calculated when

  • going from Reading/Re-Reading (or lower) to ReadConfirmed or ReadValidated (preliminary answer)
  • going from ReadConfirmed or ReadValidated to Ready or Closed (final answer)
  • going back from Ready to ReadConfirmed (or lower) (preliminary answer)
  • when a report is generated (paper report, electronic report or screen report). This ensures that negative answers defined as multi-lingual text modules are expanded in the language required for the generated report.

The negative answer is NOT re-calculated when

  • going from ReadConfirmed to ReadValidated
  • going from Ready to Closed
  • whenever the status is not changed.

Note

  • The negative answer is not displayed (in browser, on report,...) if the action status is not yet 'ReadConfirmed' or higher.
  • If the negative answer is displayed in a separate window of the action browser, all "\t" and "~n" are replaced by blanks and the answer is stripped from leading and trailing blanks.
  • The negative answer is also displayed in the microbiology screen (isolation window).

Example

How to give a different negative answer, depending on the fact that the incubation period has expired or not.

{=FitText(
IF DateTimeToDate(.EndOfIncubationTime) > Today()
THEN "up to now no Adeno-virus found"
ELSE "no Adeno-virus found",
40, "\t", "\t", ?)}

Explicit negative answer

If you do not want to use the default negative answer (as described above), you can also specify an "explicit negative answer".

Tip

An explicit negative answer has priority over the default negative answer.

Configuration

In the general microbiology settings, the option List of explicit negative answers allows to specify a choice list. The choices of the specified choice list represent the possible values of the explicit negative answer.

The Description of the choice will be used for reporting (see below).

Microbiology work screen

For microbiology actions that do not have any reportable isolations (no growth has been detected), the user can choose the explicit negative answer in the microbiology work screen.

The selection list contains the mnemonic of the choices linked to the choice list specified in the general microbiology setting List of explicit negative answers.

Notes

  • The Neg. answer option will only be visible if there are no reportable isolations for the microbiology action.
  • The chosen value will be stored in the microbiology action field Explicit negative answer.

WASPLab

The explicit negative answer can also be set during communication with WASPLab.

The growth messages sent by WASPLab to GLIMS can contain an explicit negative answer if the growth is negative.

Tip

The code (of the explicit negative answer) sent by WASPLab must match the mnemonic of a choice linked to the choice list specified in the general microbiology setting List of explicit negative answers.

In case of mismatch, the negative answer will be rejected.

If required, a code set (for the Choice table) can be used to provide a mapping. This code set must be defined for the station's coding system.

Reporting

A Report property can be defined in the microbiology procedure to report microbiology results (isolations). If there are no reportable isolations, this property will report the negative answer.

  1. If the microbiology action field Explicit negative answer contains a value, this will be reported.

    Note

    The value of the field Explicit negative answer matches the mnemonic of a choice linked to the choice list specified in the general microbiology setting List of explicit negative answers.

    It is the Description (multilingual text) of that choice that will be reported.

  2. If the microbiology action field Explicit negative answer is unknown, the default negative answer (see above) will be reported.
