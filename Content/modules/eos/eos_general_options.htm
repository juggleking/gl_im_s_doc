Order update

On pending electronic order upd.

This Order-based MISPL function (Logical) is evaluated upon electronic pending order update. If it returns TRUE, the evaluation of the On electronic order entry MISPL function is triggered. If the requirements defined in the On electronic order entry MISPL function are no longer met, the pending order is activated automatically.

This functionality is intended exclusively for the improvement of the communication between GLIMS and CyberLab and can therefore not be used with other third party Order Entry solutions.

Site attributes

Check panel member derivability from material

During manual order entry, when the user tries to explicitly request a panel on a specimen/material and one of the panel members cannot be derived from that specimen/material, a warning message appears. The user can then decide to cancel or to request the panel separately (meaning: not on the specimen/material).

The option Check panel member derivability from material allows the user to enable the additional check on panel member derivability for electronic order entry as well. By default, the option is disabled.

Enabling this option will have GLIMS execute an additional check on the derivability of all panel members from the material of the specimen when requesting a panel on a specific specimen. GLIMS will only schedule the member requests on the specimen when all panel members can be derived from the material of the specimen.

Run person comparator when found by PIN

When new Person records are imported via URL messages being sent to a GLIMS service, GLIMS first verifies if the patient already exists in the database, in order to avoid that more than one Person record is created for the same patient. The criteria used to determine whether or not Person records are identical are customer-specific. These criteria can therefore be defined in the Person comparator MISPL function (Start > System management > Customize > GLIMS General > Main page > Persons). The outcome of the MISPL function determines whether or not the existing Person record can be used.

Enabling the option Run person comparator when found by PIN will have GLIMS execute an additional check on the (best matching target) person record (found via the provided identifications) by running the Person comparator MISPL function to ensure that the correct person was found. By default, the option is disabled.
