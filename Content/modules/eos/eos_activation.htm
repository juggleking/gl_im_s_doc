Electronic order scheduler - Activation of pending orders

Manual activation

To manually activate a pending order, use the contextual menu / ribbon item Pending orders -> Activate.

Tip

This function can also be used in a Command to automate this process.

Activation using scan program

Depending on the use case, you can use

  • the Specimen reception scan program
  • the Specimen scan program
  • the Specimens by order program

Specimen reception scan (Start -> Routine -> Specimens -> Reception scan)

  1. This program can be used when the samples for the pending order have arrived in the lab.
  2. Typically, the order will have been created in GLIMS through an electronic order entry system.
  3. The electronic order system generates its own order id which is stored in GLIMS as "Order short id".
  4. The specimen barcodes generated by the electronic order entry system should be based on that order id.
  5. In the specimen reception scan program you can now scan the specimens. The barcode can be manipulated by GLIMS as such (by using the field SpecificSite.BarCodeParser) that it corresponds to the order id (short id in this case).
  6. If GLIMS finds the order and detects that this is a pending order, the order activation screen will be started.
  7. During activation, the specimen will be created/looked up by GLIMS and the scanned specimens will be mapped.
  8. Note that the specimen scan program allows using a button screen to indicate which material belongs to the scanned specimen.
  9. You can also use the "Buffered mode" option of the scan program. In that case, the orders will only be activated (scheduled) after all specimens have been added and the "Save all" button is clicked.

For more information about this scan program, please check the chapter Specimen reception scan.

Specimen scan (Start -> Routine -> Specimens -> Scan)

Offers the same support for activating pending orders as the Specimen reception scan program.

For more information about this scan program, please check the chapter Specimen scan.

Specimens of order (Start -> Routine -> Specimens -> By order)

  1. This program can be used when an order has to be activated while the actual specimens are not yet available. Only the order's id is known.
  2. Scan the barcode of an order that has status "Pending".
  3. GLIMS will now first show the "Activation screen".
  4. When the pending order has been activated, the specimens will be created / looked up.
  5. The created specimens are shown in the "Specimens of order" browser.
  6. Depending on the options of this program, the specimen labels will be printed automatically and the specimens can now be sampled.

For more information about this scan program, please check the chapter Specimens of order

The activation screen

Header zone

Left

Shows order information. Can be customized by specifying an Order-based dynamic text module in the field Order activation header in the general options (Start -> System management -> Customize -> GLIMS electronic order scheduler). If not specified, the default header will be displayed.

Right

Shows property information. For each property, the content of the property field Comment text displayed in alert-box (Triggers tab page) is shown.

Notes

  • Any comments on requested materials and properties are also shown.
  • If a result value was specified during original order entry, this value is also shown.
  • If the right zone is empty (no property comments are available), the order information will use the entire header zone.

Order data that can be updated

When activating a pending order, GLIMS allows to update the following order data:

Object date / time

The default object time during order activation is determined as follows:

  1. Object time specified in the parameters set of the order activation function (pre-configuration)
  2. General option Object time calculation (Start -> System management -> Customize -> GLIMS electronic order scheduler). Allows to define a site function (of type DateTime) that returns the date / time that will be used as default object time when activating a pending order.
  3. Depending on how the order is activated:
    1. Interactively: NOW
    2. In batch: Order.LowestObjectTime

Note

When a Rounding interval has been specified in the Order activation options (user preferences), the date/time will be rounded accordingly.

External id

Department

Encounter

Sampler / Sampling location

When activating an order, GLIMS will first propose the Default sampling location specified in the Order activation options (user preferences). If not available, the sampling location of the order will be proposed.

Order activation options (Preferences)

The tab page Order activation in the user preferences (Start -> Tools -> Set user preferences) allows to specify default values/options that are applied when activating a pending order.

The order activation window has a button Preferences which provides direct access to this User preferences window.

What happens when an order is activated

Scheduling

When activating a pending order, GLIMS will perform all actions as if a regular order was created:

  1. Actions are scheduled
  2. Specimens are created (if not explicitly requested during order entry)
  3. Reports are scheduled
  4. Result records are created
  5. ...

MISPL triggers

  1. When activating a pending order, GLIMS will execute the site function defined in the order entry option Evaluation after scheduling of the user who activated the order.
  2. When clicking OK in the order activation window, GLIMS will execute the site function defined in the order activation option Before activation of the user who activated the order.
  3. After the order has been activated, GLIMS will execute the site function defined in the order activation option After activation of the user who activated the order.

Recalculate order internal id

When activating an order, GLIMS allows to recalculate its internal id. This can be achieved by specifying a MISPL expression in the field Order internal id on activation of the general options (Start -> System management -> Customize -> GLIMS electronic order scheduler).

Note

Using this feature implies that the old internal id of the pending order will no longer exist. This may cause a gap in order numbering system. To avoid this, one could consider using a different number system for pending orders.

In the MISPL function that calculates the order's internal id, the status of the order can be checked. If it is Pending, a different numbering system could be defined that does not interfere with the one for "regular" orders. When the order is activated, it will be promoted to status Initial and the order internal id can be recalculated using the regular numbering system.

Currently not supported

  1. Currently GLIMS does not yet support automated activation of pending orders through communication with pre-analytical instruments.

  2. GLIMS does not allow merging several "future" orders for the same patient
