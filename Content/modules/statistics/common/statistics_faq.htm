I cannot use MISPL in the layout screen - no output to Excel or E-mail

For both the activity statistics (order and results statistics) and the financial statistics, you need the Expert Upgrade license to:

  • Use MISPL in the first (layout) screen (rows/columns/...).
  • Use direct output to Excel (in the output options screen).
  • Use direct output to e-mail (in the output options screen).

Dates are not displayed correctly in MS Excel

Due to a date formatting issue in MS Excel, dates exported from Glims are not always displayed correctly in MS Excel.  To avoid this, you can use a MISPL function to format (ISO) the dates before exporting them to MS Excel.  Example for Order.ObjectTime:

RETURN DateToString(.ObjectTime,"%Y-%m-%d");

Exporting statistics to Excel fails when run via a command

It has been reported that exporting GLIMS statistics to MS Excel may fail when this export is performed via a command (not interactively).  

Solution:

Create the following empty folders on the GLIMS server:

C:\Windows\SysWOW64\config\systemprofile\Desktop
C:\Windows\System32\config\systemprofile\Desktop
