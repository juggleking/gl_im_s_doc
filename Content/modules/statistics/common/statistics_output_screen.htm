Statistics - The output options screen

Contents: Introduction - Output channels - Excel - Headers - Markers - CSV - Additional output criteria

Introduction

The output options screen is identical for all statistic modules. It will only be discussed here.

Notes

  • If the Clear saved data from previous run option is set, data from previous statistics runs will not be re-used.
  • The Statistics module's week of year calculation is based on ISO 8601 / ISO week date.

Output channels

There are four possible output channels which can be activated simultaneously:

On screen

  • Plain text (in a GLIMS window)
  • Directly into MS Excel (Note this feature is part of the Office Application Link module which has to be order separately.)
  • HTML format, which is suitable for display with a web browser like Internet Explorer or Netscape Navigator.

To file

  • Plain text
  • CSV (comma separated values) format, which is very suitable to be imported with other third party tools like Excel, Access, ...
  • Excel or HTML

 When the statistics output is to be saved in a file (whatever format), you can either:

  1. Enter the whole output path (directory + file); this can also contain following symbols: {TmpDir}, {User}, {Date}, {##}. For more information on these variable, please check the chapter Using variables in the directory and file name specification of the report generation documentation.
  2. Enter only the directory; a file name of the form 'stat-{user}-{date}-{##}' will be added automatically with the right extension.
  3. Enter only the filename; the file will be placed in the session's temporary directory.
  4. Enter nothing; the output will be saved in the session's temporary directory, in a file name of the form 'stat-{user}-{date}-{##}.{extension}'.  

Note

When you specify a file which Glims cannot write to, an error message will appear and it will automatically switch to option 'd' above (as if no file was specified). 

To printer

  • Plain text
  • CSV (comma separated values) format, which is very suitable to be imported with other third party tools like Excel, Access, ...
  • Excel or HTML: Make sure the specified printer contains a 'Windows path'.  
    • If no printer is chosen (field empty), your standard Glims printer will be chosen.  
    • If this printer is no valid windows path, the printer configured as the default printer in Windows is used. 
    • The printer that is used will always be shown in a message (or in the log file when run in batch).

Note

You can also use a Windows command like _Notepad, or a Unix script, etc.

E-mail to

  • Beside screen, file or print output, it is also possible to mail the statistics results to a certain e-mail address or a list of e-mail addresses. The supported formats for this are plain text, Excel file (attachment), CSV file (attachment) and HTML. When specifying multiple e-mail addresses, separate them with semicolons (";") or commas (",").

    In both the To and CC field, you an use the F6 key to open a larger window.

Note

When the output is formatted in HTML, a reference is made to 2 style sheet files:
  • default.css (the default styles for all GLIMS HTML output)
  • statistics.css (the styles for all GLIMS HTML statistics output).

These style sheet files both reside in the 'dat'-subdirectory. 

Excel

Charts

The statistics programs are capable of adding graphic charts to the statistics tables which are displayed using Excel on the screen. One can choose between embedded charts (on the same worksheet as the data table) and separate charts (on a new worksheet), and between charts of type (vertical) "columns", (horizontal) "bars" or "pie".

Show identical row names in Excel only once

If set, and several lines start with the same label (in one or more of the first columns), this label is only shown once (only on the first line where it appears). 

Example

Disabled:

Department    Month      
       D1            Jan 04      12
       D1            Feb 04      23
       D1            Mar 04      31
       D2            Jan 04      44
       D2            Feb 04      35
       D2            Mar 04      26

 

Enabled:

Department    Month      
       D1            Jan 04      12
                     Feb 04      23
                     Mar 04      31
       D2            Jan 04      44
                     Feb 04      35
                     Mar 04      26

Column header orientation

Allows defining how column headers should be oriented in an Excel file.  The value should be a degree ranging between 0 to 90.  The default value is 30 degrees.

Headers

It is possible to specify a user-defined header for both the first page and for each separate table. They are text modules that are based on the table "w_StatisticsHeader". The MISPL expression builder shows all available fields to choose from.

Another way to compose the headers is by using the following variables in your text module:

{PeriodBegin} and {PeriodEnd}

Contain the begin and end date (+hours if specified)

{Selection}

Contains the selection criteria which were specified

{Contents}

Contains a description of which numbers the table contains and in which layout

{TableSubject}

This is only valid in the table header when "multi-table" is selected; it contains the name of the item the table contains data for; e.g. if multi-table = "issuers" then {TableSubject} will contain the name of the issuer.

If the header(s) contain "?", the standard predefined header(s) will be used. These are:

DocumentHeader

===== <Title> =====             

Time period: {PeriodBegin} - {PeriodEnd}
Selection: {= FitText("{Selection}",65,""," ",?)}
Table contents: {= FitText("{Contents}",65,""," ",?)}

TableHeader

<multi-table selection>: {TableSubject}

Markers

Marker characters can individually be configured to be shown in the statistics output:

  • No value marker: Default value '-'
  • No occurrence marker: Default value '-'
  • Presence marker: Default value '*'

CSV

Separator for CSV

When selecting 'CSV' as output format, this options allows to define the character that is used as separator between the different columns (by default a comma). 

Note

Fields in the output file will be quoted automatically if and only if they contain the separator character.  Suppose the separator is a comma, then

Chloride (ml) -> will appear in the output file as: ...,Chloride (ml),...

however

Chloride, ml -> will appear in the output file as: ...,"Chloride, ml",...

Additional output criteria

Only output when alerts occur

If enabled, Glims will not generate the files or emails when the statistics do not contain alerts.  Note that the alert condition can be defined in the first screen of the statistics wizard (layout screen).
  

Only output non-empty statistics

If enabled, Glims will not generate empty files or emails.
