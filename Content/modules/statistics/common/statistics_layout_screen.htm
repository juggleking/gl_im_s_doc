 Rows and columns

Which items (related to the records) should be displayed in the rows and which items in the columns? This means how should the records be distributed among the different rows and/or the different columns. The available options depend on the base table in use. The time options are identical in all statistics. Also available in all statistics is the option to specify a user-defined MISPL function. This function must be of type "String". The string values which the function returns will appear in the output table as row or column headers. E.g. to have statistics of the isolations per person who confirmed the isolation, one can make a MISPL function which returns the login name of the person who confirmed the isolation:

Note

The length of the returned string is limited. The maximum number of characters is defined on the basis of two parameters:

  • the number N of classifications to be displayed in the rows and columns
  • among these classifications, the number N' of user-defined ones (N'<N)

and is calculated as follows:

Max. length = ( ( 1970 - 12 * ([N] - [N']) -20 ) / [N'] ) - 1

For instance, if N=1 and N'=1, the maximum number of characters that can be displayed is 1949. If the returned string is longer, it will not be entirely displayed.

Often the user wants to use a standard option, but with a slightly different output layout.

Examples

  • For issuers: add their telephone number.
  • For properties: show the mnemonic instead of the name, or vice versa.

 

If a standard option is used in the first statistics screen for rows, columns or multi-table, a MISPL can additionally be specified to format the layout. This MISPL function is not based on the base table (like 'results', 'orders', 'isolations', ...) but on the table that is selected by the chosen option ('properties', 'issuers', ...).

Another benefit is that additional information can be added in this way.

Example

  • For properties: show the name + add the material of each property + add its statistical weight.
  • For antibiotics: add the commercial name when stored in the antibiotics external comment;
  • For microorganisms: add the name of the parent microorganism (species)
  • For wards: add the name of the 'institution' (hospital) it belongs to.

Using this MISPL feature has no negative influence on the statistics speed performance.

Multi-level row classifications

Introduction

In all statistics programs (activity, financial, microbiology, transfusion), it is possible to specify more than 1 criterion for splitting up the rows.  What does this mean?

Suppose you want to see the number of orders per month, per department and per issuer.  You could accomplish this by using the multi-table option (e.g. a table per month, a row per issuer and a column per department).  However, the multi-level row classification allows to get the same in 1 table.  

Example 

Month           Issuer     DEPT1 DEPT2
______________________________________
March 2004      Doctor 1     10     15
                Doctor 2     24     26
April 2004      Doctor 1      -      1
                Doctor 2     26     55
______________________________________

In this example there are 2 levels for the row classifications: the months and the issuers. You can go up to 4 levels.

How to configure this: in the first screen of the statistics (the layout screen) a radio set is available to select the row-level. For each of the 4 levels you can select a different classification. Each one can be a standard option, a user-defined option, or a combination of both (e.g. 'issuers' + a MISPL expression to change the appearance).  The INSERT-key can be used on the radio-set for the row levels.  It shifts the levels to the right or to the left if possible, so that a blank row level is created.

Except for CSV output, (so for output in plain text format, HTML format or Excel), the names in the rows will not be repeated when they are the same as the row above (see example).  For CSV output however, the output of the same example will be:

Month,Issuer,DEPT1,DEPT2
March 2004,Doctor 1,10,15
March 2004,Doctor 2,24,26
April 2004,Doctor 1,-,1
April 2004,Doctor 2,26,55

Column summaries for each row-classification

GLIMS allows to add column summaries for each individual row-classification.  To enable this functionality, enable the option "Total/Mean/..." below the row selection for which you would like a column summary. 

For the row levels where a column summary is requested, the letter that is shown above the radio-box will be underlined. The tool tip of the radio-box shows a compact overview of the chosen row-level classifications. Note that the label of this option corresponds to the value you selected in the column summary field.

 Characteristic values

Which characteristic values - specific to each record - are you interested in and want to use for calculations?

The simplest example - valid for all statistics, is the record occurrence, which can be used to count the number of records. Another example specific to 'result' records is its property weight (price):

The characteristic value can also be extracted by means of a user-defined MISPL function. This function must be of type "integer", "fractional", "logical", "time" or "string". 

  • When type 'time' is chosen, the output values are displayed in a typical time format like: mm:ss / hh:mm:ss / dd hh:mm:ss.   
  • Typical usage of a MISPL function of type 'logical' would be the calculation of percentages:
    • % positive examinations
    • % of results used for QC
    • % of orders for patients older than 60 years
    •  etc.

The gathering options (which are in front of the characteristic combo-box) are automatically adapted to the data type of the selected characteristic:

  • When occurrence is chosen as characteristic, the gathering options are: frequency (=number), rel. frequency (%)/column and rel. frequency (%)/row
  • For characteristics of type 'logical' (such as e.g. "isolation sensible" in the isolation statistics) the only gathering option is "percentage".
  • For data types 'integer', 'fractional' and 'time'. the options are: sum, mean, maximum and minimum.
  • For data type 'String', the possibilities are restricted to 2 possible options: 'First' and 'Last'. This means that each cell in your statistics output table will contain either the value of the first or the last encountered record (chronologically according to the date/time period specified in the selection screen, e.g. for orders: the receipt date/time).

  • For occurrences, you can choose between the absolute frequency and the relative frequency. The relative frequency can be related to the columns or the rows.

    Example of frequency per row: for each property the percentage per weekday is displayed. The total per column (per property) is always 100%. At the right the totals per row (per weekday) are also related to the other rows.

 

  • For characteristic values which are numeric or time values, possible options are the sum, the mean, the maximum, the minimum, the standard deviation, variance, median, percentiles and coefficient of variation. 

Standard deviation and variance

When statistics are performed on numerical values (either standard built-in functions or MISPL functions delivering a numerical value), it is possible to calculate the Standard deviation or the Variance of these values. For accuracy reasons, there is a difference whether the values are gathered from the entire population or only a sample of the population.

The following options are available:

Variance (population)

Calculates the population variance, which is the variance based on the entire population. Also known as 'sigma-square' or 'standard deviation square'. It is the average of the square of the distance of each data point from the mean.

Standard deviation (population)

This is the square root of the variance on the entire population. Also known as 'sigma'.

If your data represents a sample of the population, then use the options:

Variance (sample)

Calculates the sample variance. Also known as 'S-square'.

Standard deviation (sample)

Calculates the standard deviation based on a sample. Also known as 's'. It is the square root of the sample variance.

Median and percentiles

In all statistics modules you have the option to calculate

  • the median (which is the 50th percentile)  
  • any percentile: when choosing the option Percentile... the user is prompted to specify which percentile (a number between 1 and 99, usually something like 5, 25, 75 or 95).

Note

  • The pth percentile is obtained by first calculating the rank l = p(n+1)/100, rounded to the nearest integer and then taking the value that corresponds to that rank.  As an example, let us take the systolic blood pressures of 10 patients sorted in ascending order: 115, 120, 125, 145, 145, 150, 150, 160, 170, 175.  The 25th percentile will give 125 and the 75th percentile 160.
  • Excel calculates this slightly different. See the article "Statistical flaws in Excel" on www.mis.coventry.ac.uk/~nhunt/pottel.pdf
  • Multiple different percentiles can be calculated by use of combined statistics.

Coefficient of variation

The coefficient of variation (expressed in percentage) is calculated as follows: (Standard deviation / Mean) * 100.

In probability theory and statistics, the coefficient of variation (CV) is a measure of dispersion of a probability distribution. It is defined as the ratio of the standard deviation to the mean. The coefficient of variation is a dimensionless number. It is often reported as a percentage (%) by multiplying the above calculation by 100.

The following options are available:

Coeff. of variation (population)

Calculates the coefficient of variation based on a population and is calculated using the population-based standard deviation.

Coeff. of variation (sample)

Calculates the coefficient of variation based on a sample and is calculated using the sample-based standard deviation.

Splitting up the first column

The first column, containing the names of the rows (e.g. the names of the issuers if the rows are per issuer), can be split up automatically into multiple columns. This could be useful e.g. in the example of the issuers you could want a column with  the issuer's name, a column with the issuer's code and a column with the issuer's phone number.

How can this be done?

If the row names contain '|' characters, GLIMS automatically considers this as the separator between different columns. So for example if you have the issuers displayed (via a MISPL) as name|code|phone  then GLIMS will split the first column automatically into 3 columns.  

Warning

A maximum of 20 separators are supported!  When this maximum is exceeded, the following error will be produced "** Array subscript 21 out of range (26)".

This can be done with following MISPL: RETURN .Name + "|" + ifknownstring( .InternalId ) + "|" + ifknownstring(.Phone);

Note

If the name of this MISPL also contains the "|" character, the name will also be split into parts to get the names of those columns.

Mark alerting values in output

Glims allows configuring what kind of values are 'abnormal' or 'alerting'. These values will be marked in the output file:

  • In plain text: with an exclamation mark (!) behind the value
  • In HTML and Excel: value in red and bold

To decide whether values are alerting or not, a condition can be entered by enabling the toggle-box with the exclamation mark icon.  You will be asked to enter a logical expression (using MISPL syntax). In this expression you should use the variable called 'Value' to refer to the value that is calculated in each cell of the statistics output.  The variable 'Value' represents the cell value (numerical or textual). 

Example

  • For numeric statistics: Value < 0 OR Value > 100 OR Value = ?
  • For textual values: Value = "Pos" OR Value = "+"
  • For time values, the alert condition is based on the numeric value, expressed in seconds.

Note

For textual values the length is restricted to 250 before being evaluated; thus only the first 250 characters can be used in this alert evaluation.

General options

Options in alphabetical order

Allows to sort the options in the screen in an alphabetical order

The options button left of the Back button offers the following options:

Decimal places

Allows to specify the number of decimal places that should be shown in the statistics output of decimal numbers.  When not specified, this defaults to '0' for percentages and to '2' for other values.

If possible, show also rows and columns for which there is nothing

This option only has effect when you select months, quarters, years, issuers or agents for the rows or columns.

Months, quarters and years

When the new option is enabled, GLIMS will not only show the months for which it could find something, but also all other months within the selected time period (second screen).  The same applies to quarters and years.  This functionality is general for all statistics modules.

Issuers and agents

When the new option is enabled, GLIMS will not only show the issuers/agents for which it could find something, but also all other issuers and agents within the selected issuer group / agent group (second screen).  This applies to result statistics, order statistics and billing item statistics.

Note

This feature does not apply to "multi-table" statistics: empty tables will not be added.

Restricted MISPL usage

When using MISPL in statistics, the only purpose is reading information from the database (for purposes like: classification, formatting, ...). These MISPL functions should not perform changes to the database.  Since some customers could accidentally change their database 'Sequencers' by specifying the wrong site function, some checks are performed.

All statistics programs check if the MISPL functions that are specified for rows, columns, multi-table or for the content values are functions that do NOT contain one of the following calls: 'Identifier()', 'NextValue()' or 'SetSiteAttribute()'.

This check is performed: 

  1. When you try to go from the first to the second user interface screen.
  2. When the real execution of the statistics run is started (interactively or in batch).
