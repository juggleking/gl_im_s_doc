Statistics -> The selection screen: Which records are you (not) interested in ?

Example

Example screen (result statistics):

This screen allows to define a selection of records on which statistics should be performed.

One of the main components here is the time period, which is common to all statistics. The begin date is required. The end date is optional. If left empty, today is assumed. For both begin and end date the time is optional. If left empty, 00:00 is assumed for the beginning and 23:59 for the end.

It is also possible to specify a relative time period instead of an absolute time period. This means a time indication in relation to the moment the statistics will be executed:

  • today, yesterday, 2 days ago,
  • this week, last week, 2 weeks ago,
  • this month, last month, 2 months ago,
  • this quarter,last quarter, 2 quarters ago,
  • this year, last year, 2 years ago.

This is especially useful when configuring batch task commands to be executed periodically (each day, each week, ...). 

How to use this ? Next to the field where you can specify a fixed date, there is a little down arrow. When clicking this, a combo-box appears and you can choose a relative indication. On the other hand, if you want to go back to a fixed date, choose the indication "<date>" in the combo-box and the fixed-date field will re-appear.

The possible choices for begin date and end date are the same, but when chosen for the begin date, the begin of the relative period is meant, whereas taken as end date, the end of the period is meant.

Example

begindate = "last month" -> means: begin date = first day of last month
enddate = "last month" -> means: end date = last day of last month

So if you want statistics for last month, specify "last month" for both the begin and end date.

Another field common to all statistics is the "Filter" field at the bottom, where a MISPL function can be chosen for further user-defined selection criteria which do not appear amongst the standard criteria.
