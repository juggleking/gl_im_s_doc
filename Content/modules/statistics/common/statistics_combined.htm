Statistics -> Combined statistics

Contents: Introduction - How does it work - Configuration - Notes

Introduction

The combined statistics concept can be applied to any type of statistics in GLIMS.  It is possible to take several pre-defined statistics and merge (combine) them into 1 single output.

These several statistics to be combined can 

  • Show different calculations on the same characteristic values (e.g. the average + the minimum + the maximum).
  • Show different characteristic values (e.g. the number + the average value).
  • Be based on different selections (e.g. orders of issuer group A last month + orders of issuer group B this month).
  • Be based on different base tables (e.g. number of results and number of orders).
  • Be based on information from different statistical modules (e.g. production statistics and financial statistics).

The only requirement for these combinations is that criterion for the rows and the criterion for the 'multi-tables' are always identical. 

Note

You can only use sub-statistics from modules for which the license has been purchased.

How does it work

All sub-statistics will be executed sequentially one after the other, and their outputs will be combined as followed:

  • If they have rows that appear identically in other sub-statistics, these rows will be merged and they will appear only once.
  • If multi-table output is applicable and they have tables that appear with the same name in other sub-statistics, these tables will be merged and will thus appear only once.
  • The columns are not merged but they are put one aside from the other: beginning at the left, each sub-statistic's columns will be added right from the already existing columns, from left to right.

Configuration

In the first screen of all statistics modules, a button can be found at the top for combining statistics. When you hit this button, the screen is replaced with another screen where you can specify which sub-statistics you want to  combine, and in which sequence.

The sub-statistics are specified by their function parameter sets.

Add

Click the Add button or use the Insert-key to add a sub-statistic: first you choose from which statistical module, then you select a parameter set in which the sub-statistic is configured. If the parameter set does not exist yet, you can create it with the insert-key.  Give it a name and then use the contextual menu function 'Setup' to predefine the statistics.

The second column shows the subject (the base table) of each component.  After having used F6, double-click or 'Setup' on a component, the component list is automatically refreshed in order to show the correct contents.

Remove    

Click the Remove button or use the Delete-key to remove a sub-statistic from the list. (note: this does not delete the  function parameter set itself)
  

Arrows

With the arrow buttons you can change the sequence of the sub-statistics.    

Setup

To change the configuration of a certain sub-statistic: use the Setup button or double-click on the sub-statistic line. You can also use F6 to zoom into the parameter set.    

Next

Once you finished the list, hit Next to go to the output configuration screen.   

Normal

The Normal button can be used to switch back to normal (non-combined) statistics.

Combine by

When using combined statistics, the global output is built by adding the columns (one or more) of each separate statistic at the right side of the columns of the other parts.
In some cases this is not the most useful way.

Example

When combining:

a. the number of orders per month (stat 1) and

           (1) Number of orders:
                       | Jul 05 | Aug 05
               Issuer1 | ...    | ...
               Issuer2 | ...    | ...

b. the invoiced amount per month (stat 2),

           (2) Total amount on invoices:
                       | Jul 05 | Aug 05
               Issuer1 | ...    | ...
               Issuer2 | ...    | ...

then the combination of both would give something like:

                       | Number of orders | Number of orders | Invoiced amount  | Invoiced amount
                       | Jul 05           | Aug 05           | Jul 05           | Aug 05           
               Issuer1 | ...              | ...              | ...              | ...
               Issuer2 | ...              | ...              | ...              | ...

In this case it could be nice to have the columns 'merged' by month, which would look like this:

                       |              Jul 05                 |              Aug 05 
                       | Number of orders | Invoiced amount  | Number of orders | Invoiced amount
               Issuer1 | ...              | ...              | ...              | ...
               Issuer2 | ...              | ...              | ...              | ...

To obtain this effect, you can use the option Combine by....  Its default value is "adding columns at the right side". It adds each separate statistic as one or more columns to the global output.  For merging and grouping the columns by 'month' like the example above, choose the value "adding sub-columns if possible (grouped on column name)".

Additionally it is also possible to have the statistics added vertically, i.e. by adding rows. Similarly you can choose between:

 "adding rows at the bottom"; this will give something like:

                                   | Jul 05 | Aug 05
        Number of orders | Issuer1 | ...    | ...
                         | Issuer2 | ...    | ...
        Invoiced amount  | Issuer1 | ...    | ...
                         | Issuer2 | ...    | ...

"adding sub-rows if possible (grouped on row name)"; this will give something like:

                                   | Jul 05 | Aug 05
        Issuer1 | Number of orders | ...    | ...
                | Invoiced amount  | ...    | ...
        Issuer2 | Number of orders | ...    | ...
                | Invoiced amount  | ...    | ...

Notes

  • Each sub-statistics can also be run on its own as a normal statistic run. This can be done by configuring a Command that refers to that function parameter set. This way each sub-statistics can also be tested separately.   
  • When you want to configure a Command for combined statistics, you can choose any of the 'Statistics...' functions (e.g. StatisticsProduction) that are defined on table 'gp_Site'.  It does not matter which at this point.
  • In case you changed your mind and you want to go back to the regular non-combined statistics, make the list of sub-statistics empty and hit NEXT. You will then get back into the normal first screen (the layout configuration screen).
