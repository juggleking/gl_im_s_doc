Statistics -> Running statistics in batch as commands

All statistics programs can be run in batch. This has the advantages that:

  • The program runs on the server, not on the client, so that there are more system resources available for the program and it will run faster.
  • The program runs in background, so that the GLIMS client is free to do other jobs.
  • The parameters have to be set up only once.

Detailed information on the configuration of commands can be found in the Commands and Tasks section.

How can a batch command for statistics be configured ?

  • From the menu Start -> System management -> Commands -> Commands. Then click insert.
  • As table choose: gp_Site.
  • Enter a description for the command.
  • Choose for query: Default.
  • Leave query parameter set and sort function empty.
  • As function, choose: StatisticsProduction, StatisticsMedical or StatisticsFinancial.
  • Function parameter set:  Create a new function parameter set and then select the Setup function from the contextual menu. This will bring you in exactly the same three input screens as described before. After finishing the third input screen, press the OK buttons in both windows shown here.
  • The command is now configured. You may always adapt the parameter settings afterwards if required.

How can the command be executed ?

  • You can start the command interactively (e.g. to test it) by means of the Do function in its contextual menu.
  • To have the command start automatically at (a) certain time(s), you will have to create a "task" for the command. For more information on how to do this, see Commands and Tasks.
