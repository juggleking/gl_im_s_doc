Status overview in GLIMS

OrderStatus - SpecimenStatus - ResultStatus - ActionInputStatus - ActionStatus - Microbiology actions - Microorganism code - Blood bag and blood selection

OrderStatus

  1. An order starts in status Initial.
  2. When at least one primary request is confirmed, the order promotes to Partial.
  3. When there are NO primary requests, and at least one secondary request is confirmed, the order promotes to Partial.
  4. When all primary and all secondary requests are validated (or confirmed in the case of specimens), the order becomes Complete.

Note

  • Primary requests: 'normal' results, microbiology requests and pathology requests.
  • Secondary requests: calculated or informational results and specimens.
Status Code Description
Pending Pnd Please check the chapter Electronic order scheduler
Initial Ini Initial status after creation
Partial  Prt At least one primary request is confirmed or when there are NO primary requests and at least one secondary request is confirmed
Complete         Cmp All primary and all secondary requests are validated (or confirmed in the case of specimens)
Closed Cls No more modifications are allowed
Discontinued  Dsc Explicitly discontinued

SpecimenStatus

Status Code Description
Initial Ini Initial status after creation
Expected Exp Entered when the delivering action (if any) becomes Outputting
Available Avl Entered on specimen delivery
Discontinued Dsc Entered on manual specification (due to e.g. material loss), or on destructive procedure activation

ResultStatus

Status Code Description
Initial Ini Initial status after creation
Expected Exp Entered when the delivering action (if any) becomes Outputting
Available Avl Entered on property value assignment
Confirmed Cnf Entered on manual or automatic confirmation (technical validation). No more value modifications are allowed. Pre-confirmation overwriting cannot be tracked back later. It means that if a result has not been confirmed yet no tracking or logging is possible afterwards.
Validated Val Entered on manual or (template-defined) automatic validation (biological validation)
Discontinued Dsc Entered on manual specification (property rejection)

ActionInputStatus

Status Code Description
Connecting Cnc Initial status after creation
Expected Exp Entered when connected (either manually or automatically) to another action's output (either a specimen or a result). May return to Connecting when the delivering action link is broken (due to specimen/result discontinuation).
Available Avl Entered when the delivering specimen status becomes Available or the delivering result status becomes Confirmed.

ActionStatus

Status Code Description
Frozen Frz Internal state, for optimization purposes
Connecting Cnc The action has been created. In practice a property or a material was requested and scheduling is in progress.
Inputting Inp Result and specimen inputs are 'Expected' (i.e. the delivering action is Outputting) or there are no inputs. May return to Connecting if inputs are discontinued.
Pending Pnd Normally entered if all inputs are 'Available' or if there are no inputs. Serves in the first place to impede the work until the signal is given implicitly (by generation of a work list) or explicitly (by entering the Action start time) to start the job (becomes Outputting). If correctly configured, Pending actions do not appear on the work list, if not wanted.
Outputting Out Entered :
1) when Procedure.AutoStart is TRUE/YES. In such case, the Pending status is simply skipped;
2) manually, via the contextual function on Action.
Consequences:
1) result and specimen outputs automatically receive status Expected;
2) no additional outputs can be assigned or added to the Action (train has left). This implies that an ancillary examination will generate a new Action, and cause an extra line on the work list;
3) standard results are entered and formulas calculated.
Partial Prt At least one result or specimen output is available.
Realized Rlz All result and specimen outputs are available, but some results are not yet confirmed.
Ready Rdy All results are confirmed, validated or discontinued and all materials are available or discontinued; or there are no outputs.

Microbiology actions

Status Code Description
Initial Ini Nothing has happened yet; specimen not available yet.
Grafting Graft Specimen is available; user has to graft the specimen onto a carrier.
Microscopy Mic Grafting is done; user is performing a microscopy investigation. This step is only done if the microbiology procedure contains a microscopy property.
Reading Read The lab technician is investigating the specimen and entering isolations, isolation test results, carrier test results, antibiotics, antibiotic results, ...
ReReading RRead After statuses ReadConfirmed, ReadValidated or Closed: if the results have changed, they will have to be re-confirmed and re-validated; the status is then demoted to re-reading (which is functionally equivalent to Reading). Another case is when the biologist thinks further investigation is required (before validation).
ReadConfirmed ReadC The lab technician confirmed the results they found and entered in the microbiology screen (in the Netherlands: "Voorlopig geconfirmeerd") (confirmation by lab technician).
ReadValidated ReadV The biologist thinks the results in the microbiology work screen are OK and validates them (in the Netherlands: "Voorlopig gevalideerd") (confirmation by biologist).
Ready Rdy The lab technician thinks that no further investigation is required (in the Netherlands: "Definitief geconfirmeerd") (confirmation by lab technician).
Closed Cld The biologist confirms the previous decision (in the Netherlands: "Definitief gevalideerd") (confirmation by biologist).
Discontinued Dsc The investigation has been discontinued without final result.

Blood bag and blood selection

Blood bag

Status Code Description
Verification Vrf If the content of a blood bag still needs to be verified before storing it, it should have status Verification. This depends on the corresponding blood product configuration.
Initial Ini A blood bag has been created manually or by scanning the barcodes. Or the blood bag was demoted to this state due to the discontinuation of a related blood selection.
Reserved Rsv A compatible blood bag has been found and is selected as candidate for at least one selection.
Checked Chk To indicate whether a blood bag has passed the physical compatibility check, and thus to distinguish it from the previous status, the blood bag can attain the status Checked.
Checked out Cko The blood bag has been distributed to the proper ward where it was required. As this happens, the blood bag is no longer available in the laboratory. To indicate this, the status Checked out is used.
Administered Adm A blood bag attaining status Checked out is no longer available in the laboratory and will normally be transfused. To distinguish between the first and the latter status, the blood bag will attain the status Administered to indicate that the person has really received the blood.
Expired Exp A blood bag contains a material that has a limited lifetime indicated by its expiration time. If that time has elapsed, the blood bag cannot be used anymore. To mark this particular event, the blood bag will be assigned the status Expired.
Discontinued Dsc A blood bag can be manipulated wrongly during the transfusion procedure, e.g. dropped out of the refrigerator, heated too fast, …. Hence, it cannot be used for further transfusions anymore. To indicate this irregular manipulation, the blood bag can be discontinued, implying the attainment of that status.
Wasted Wst  
Undelivered Udlv  

Blood selection

Status Code Description
Initial Ini Means that a request for a blood product was made but no blood bag has been selected for it yet.
Reserved Rsv Means that a compatible blood bag has been selected but that the blood bag has not yet passed the physical compatibility check.
Checked Chk Means that the blood bag has passed the physical compatibility check but that the blood bag has not yet been distributed.
Ready Rdy Means that the specific laboratory manipulations of the blood bag, i.e. selection and compatibility check are carried out and that the blood bag is taken from the laboratory to the ward where it is required.
Discontinued Dsc Means that the request for a blood product has become superfluous.
