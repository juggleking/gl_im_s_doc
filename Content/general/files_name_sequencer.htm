Advanced configuration -> Creating file names using sequencers

Introduction

In many programs which export data to a file you can use symbolic file names. There are two ways to add a sequence number to the file name. Both symbols must be enclosed in brackets, e.g. {##} or {Seq}.

Using the # symbol

The '#:format' symbol is replaced by an increasing number. It indicates that a version number should be used 1 larger than the largest version number already found in the same directory. However, if a file gets deleted after being processed, its sequence number can be reused. If this is not desired, use the Seq symbol (see below).

The length and the cycle of the sequencer is determined by the format:

#:999 : a 3-digit sequencer, running from 000 to 999 and back to 1.
#:>>9 : the same, but the leading zeroes of the sequencer are suppressed
The default format is ">>>>>>9", providing a cycle between 1 and 9999999.

Remarks:

  • An alternative format specification is e.g. ### which is identical to #:999.
  • A * symbol indicates that more digits are allowed. A ? symbol indicates that the first file should not have a number.

    Examples:
    test### -> test001 test002 test003 ...
test#*  -> test1 test2 ... test10 test11 ...
test#*? -> test test1 test2 ... test10 ...

Using the Seq symbol

You can also use the 'Seq:format' symbol to indicate that a database sequencer should be used in order to ensure that the version number keeps increasing, even when the previous file was already deleted.  If this symbol is used, a database sequencer (see System Management > Scheme > Sequencers) with a mnemonic = "File-" + the part of the file name BEFORE the sequencer will automatically be created and incremented.

The length and the cycle of the sequencer is determined by the format:

Seq:999 : a 3-digit sequencer, running from 000 to 999 and back to 1.
Seq:>>9 : the same, but the leading zeroes of the sequencer are suppressed
The default format is ">>>>>>9", providing a cycle between 1 and 9999999.

Example:

symbolic file name = Out{Seq:999}.hpr will use a sequencer named 'File-Out' to generate file names from Out001.hpr to Out999.hpr before restarting with Out001.hpr.

Tool to replace {###} with {Seq:999}

GLIMS offers a tool to replace the {###} symbol with the {Seq:999} symbol in symbolic file names in function parameter sets.

To run the tool :

  1. choose Start > Development > 4GL > Run procedure,
  2. as File name, specify UseSeqInFileNames.
