Dynamic OS command processing

Introduction

This feature, similar to dynamic URL processing, allows to start a dynamic OS-command from any GLIMS table.

This feature is protected by a separate license named MATE_HTTP. The configuration is not protected, but when actually executing the function, the license is required.

Configuration

The feature described is available on all GLIMS tables. On each table, the existing RunProcedure function offers a sub function OSCommand which allows to build a dynamic operating system command.

How to activate this feature in GLIMS

  1. Click Start > System management > Scheme > Tables
  2. Select the table you want to use as base table
  3. Right click Show all... > Tools
  4. Insert a new Tool with:
      
    1. Function mnemonic: RunProcedure
    2. Create a new Function parameter set and choose Setup in the contextual menu / ribbon

    3. In the setup screen, choose the function OSCommand

    4. In the next screen, you can define an OS-command builder

  5. This Tool can be used as a menu or ribbon item for instance.

OS-command builder

This text should result in a valid OS command. The text can be built dynamically, based on any field or expression of the record which is accessible by MISPL.

Example

The OS-command builder text contains the following definition:

NOTEPAD C:\temp\{=.CreationUser.FirstName}.txt

Provided that the name of the user who created the current record is 'john', the command being launched is

NOTEPAD c:\temp\john.txt

Provided that c:\temp\john.txt exists, it will show its content in NOTEPAD.
