Dynamic URL processing

Content: Introduction - Configuration - Examples

Introduction

This document describes how you can implement dynamic web and file access in GLIMS.

This feature is protected by a separate license named MATE_HTTP. The configuration of the web and file is access is not protected, but when actually executing the function, the license is required.

GLIMS also offers a Dynamic OS command processing feature.

Configuration

The feature described is available on all GLIMS tables. On each table, the existing RunProcedure function offers a sub function URL which allows to build a dynamic link to a web address or a file.

How to activate this feature in GLIMS

  1. Click Start > System management > Scheme > Tables
  2. Select the table you want to use as base table
  3. Right click Show all... > Tools
  4. Insert a new Tool with:
     
    1. Function mnemonic: RunProcedure
    2. Create a new Function parameter set and choose Setup in the contextual menu / ribbon

    3. In the setup screen, choose the function URL

    4. In the next screen, you can define a Title text and a URL builder

  5. This Tool can be used as a menu or ribbon item for instance.

Title text

This text will be shown in the title of the window. The text is optional and can be left to "?" if desired. In that case the URL will be shown in the title of the window.

URL builder

This text should result in a valid URL (which can be processed by Internet Explorer). The text can be built dynamically, based on any field or expression of the record which is accessible by MISPL. The URL being launched is at the same time shown in the message area on the bottom of the window.

Examples

Web access

The URL could result in a static internet page.

http://www.sopres.be/cgi/build_fn_01.asp?pre={=ToUpper(.CreationUser.FirstName)}&lg=E

Here, the expression {=ToUpper(.CreationUser.FirstName)} will be replaced by the first name of the user who created the record, in uppercase, eg. "JOHN", so the actual URL could be

http://www.sopres.be/cgi/build_fn_01.asp?pre=JOHN&lg=E

www.sopres.be processes the URL and will show a distribution of usage of the name 'JOHN' in Belgium.

File access

The URL could refer to a file.

file://C:/temp/{=.CreationUser.FirstName}.txt

In a similar context, this URL would result in

file://C:/temp/john.txt

Provided that c:\temp\john.txt exists, it will show its content in the viewer.
