Introduction

Both in the object data sheet text and in text reports, it is possible to insert an external file.  This file should of course exist and be accessible from the executing computer (the client for the object data sheets and the client or the server for text reports).  Moreover, it is possible to have this file automatically deleted after it has been included.

Syntax

Including a file can be done by introducing in the text module itself an indication of the following format:

<FILE SRC=xxx > 

in which 'xxx' is the path of the file, either the full path name or a relative path name (in which case PROPATH will be searched for the file). If the file is to be deleted after it has been included you should use the syntax:

<FILE SRC=xxx DEL=YES>

Purpose

The original reason why this functionality was created is to allow some MISPL methods to create very large text output.  As MISPL itself has a 4kb output limit, this functionality allows saving its output text into a (temporary) file and specify the pathname of this file.  After this temporary file has been included into the text, it is simply deleted again.  Example: Object.MicrobiologicHistory()

In the object data sheet

There is no restriction with respect to the type and size of the file that is included, as long as the total text is:

  1. A plain text file 
  2. A HTML or XML file that can be viewed with a normal web browser (so in this case the included file might contain HTML or XML).

In text reports

A file can be included in a property classification node.  As long as the total size of the paragraph (including the file) does not exceed 30kB, the file is included in the current paragraph.  If not, new paragraphs with a maximum size of 30kB will be created.  These added paragraphs will have the same settings as the original paragraph, but with PageBreakBefore=NO.  In that case the original paragraph will have 'KeepWithNext=Yes' if it had 'KeepLinesToGether=Yes'.

File inclusion is also possible in the headers and footers of the report, but with the restriction that the file is always included in the current section and that no additional sections can be created if the file is too large.
