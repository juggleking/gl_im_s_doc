Release notes 9.9.5 - Actions were not scheduled on specimen with modified variables (GLIMS-12989)

An issue was reported where, if default material variables were defined on the request definition and the user modified the specimen variables during order entry, then no actions were scheduled on the specimen with the modified variables.

Example

Configuration

  • A material "blood" with at least one material variable.
  • A sampling procedure outputting "blood".
  • Two procedures: one outputting "property1", the other outputting "property2", both having "blood" as input material.

  • "property1"

    • has an On result entry trigger which adds "property2" to the order
    • its request definition has a material variable choice filled in
    • its procedure output has a return value specified in the Value field.
  • The order entry option Automatic sample login is enabled.

Routine

  • During order entry, request "blood" and "property1".
  • During order creation, the Specimen variables window appears.
  • When the user changed the default material variable choice, "property2" was not scheduled correctly.

This issue, which occurred since GLIMS 9.8, has been corrected.
