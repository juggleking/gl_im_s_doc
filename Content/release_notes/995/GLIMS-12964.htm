Release notes 9.9.5 - Specimen archive scan: next rack position was not always incremented correctly (GLIMS-12964)

An issue was detected when archiving specimens via the specimen scan program: for every second rack, only the first position was used to store the specimen. Then, the next rack position incorrectly moved on to the next rack.

Example

Five racks exist with a capacity of five positions:

  • Racks: A1, A2, A3, A4, A5
  • Positions: 1, 2, 3, 4, 5

Scanning specimens results in the following archive positions:

  • A1,1
  • A1,2
  • A1,3
  • A1,4
  • A1,5
  • A2,1
  • A3,1
  • A3,2
  • A3,3
  • A3,4
  • A3,5
  • A4,1
  • A5,1
  • A5,2

This issue, which occurred since GLIMS 9.9.3, has been corrected.
