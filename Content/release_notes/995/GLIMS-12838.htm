Release notes 9.9.5 - Correction of "cycle in procedure calls" error that occurred during specimen confirmation (GLIMS-12838)

An issue was reported where the error Cycle in procedure calls detected in SetSpecimenSize spmn_siz while executing WRITE trigger for Specimen. (2868) occurred in the following scenario:

Configuration

Two procedures having the same material as input and each a property as output. A Minimal size is defined for the material input of the first procedure. A Consumption is defined for both property outputs. The property output of the first procedure has a MISPL expression defined discontinuing the property outputted by the second procedure.

Routine

  • Request both properties in order entry.
  • Confirm the availability of the specimen.
  • The above-mentioned error occurs.

Because of this error, the specimen's availability could not be confirmed. This issue has been corrected.
