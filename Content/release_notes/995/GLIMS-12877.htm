Release notes 9.9.5 - Fixed issue with MISPL added request on material with time discriminator (GLIMS-12877)

Problem description

Configuration

  • A procedure exists with an input material and an output property.
  • The material has a Creation trigger that adds a request. For instance: ".AddRequest("PropertyMnenonic", ?, ?); RETURN TRUE;"

Routine

  • When requesting the material with a time discriminator, for instance "01:00", a message appeared saying that the request cannot be derived from the material.

Solution

This issue has been corrected.
