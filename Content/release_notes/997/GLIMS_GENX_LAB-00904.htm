Release notes 9.9.7 - Support for changing the object of a genetics order (GLIMS_GENX_LAB-00904)

Issue

Some results of an order for which the object had been changed were still linked to the old object. This could lead to errors like "No locus result found with id ..." when trying to access the results of that genetics order.

Scenario

  1. Create a Genetics order.
  2. Add a variant result to the order.

  3. Send an object change object through communication.

    Note

    • This has to be sent from a reliable channel.
    • The 'ForceUpdateObject' command needs to be added to the message.

    => The order is linked to the new Object but the variant result is still linked to the old object.

This issue, which occurred since GLIMS 9.9, has been corrected.
