Release notes 9.9.7 - Fixed reporting issue when referring to classification node with sequence number over 9999 (GLIMS-13811)

A problem was reported where an error occurred when the user wanted to generate an online report for an order containing a property linked to a property classification node with a sequence number higher than 9999 and that had Repeat per specimen ticked.

The following error message appeared: '** Value 10000 cannot be displayed using 9999. (74)'.

The problem, which occurred since GLIMS 8, has been corrected. Sequence numbers are now supported up to 6 digits.
