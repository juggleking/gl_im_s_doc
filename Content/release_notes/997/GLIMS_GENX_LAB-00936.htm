Release notes 9.9.7 - Genetics REST: aminoAcidChange field no longer required in Variant JSON + support for automatic gene region creation for sequence variants (GLIMS_GENX_LAB-00936)

Issues

  1. The J_GENX_RI translator gives an error when the aminoAcidChange field is not specified in the input field as it is marked as a mandatory field.
  2. When a sequence variant is created via the REST service, GLIMS passes the name of the locus. However, certain customers define their own gene regions on top of the ones that GLIMS imported via HGNC. As a result, it is possible that the locus does not exist and GLIMS returns an error.

Solution

These issues, which occurred since GLIMS 9.9.0, have been corrected:

  1. The aminoAcidChange field is now removed from the list of mandatory fields, and not specifying the field will no longer result in a JSON import error.

  2. GLIMS will now check to see if the locus name corresponds to a gene region. If this is the case, GLIMS will try to create that gene region on the fly.

    Note

    Loci of type 'Gene region' must adhere to the following naming format:

    "Gene:GeneRegionTypeNumber"

    E.g.: CFTR:Intron1, BRCA1:Exon3

    GeneRegionType is one of the following entries (case sensitive!):

    • Exon
    • Intron
    • 5'UTR
    • 3'UTR
    • Amplicon

    If the user specifies a wrong GeneRegionType, an error message is shown and the variant is not created/updated.
