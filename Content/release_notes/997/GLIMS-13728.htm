Release notes 9.9.7 - Adding a request for a pending order with a Before activation trigger gave the wrong internal id (GLIMS-13728)

Issue

When the user had a Before activation trigger MISPL configured in the User options (Start > Tools > Set user preferences > Triggers) to add a request (AddRequest()) to the order, explicitly requested specimens got the wrong internal id upon order activation. This issue only occurred when the specimen internal id MISPL used the order internal id to base its id on.

Solution

This issue, which occurred since GLIMS 9.9, has been corrected: the execution of the Before activation MISPL now happens before any updates to the order status.
