Release notes 9.9.7 - Prioritize already existing actions when scheduling a new request (GLIMS-13673)

Issue

An order was scheduled differently when multiple properties were requested at the same time or when these properties were requested at different times.

When multiple properties are requested at the same time, the scheduler will try to schedule them on the same action. However, when these properties were requested at different times, no priority was given to already existing actions, which resulted in an incorrect scheduling.

Configuration

Two properties PropA and PropB with two procedures:

  • Procedure A on station A with PropA and PropB as outputs.
  • Procedure B on station B with PropA and PropB as outputs.

This issue, which occurred since GLIMS 8.11, has been corrected.

Warning

The previous behaviour was incorrect. Customers having used the previous behaviour intentionally in their workflows must be aware that GLIMS now no longer behaves as before.
