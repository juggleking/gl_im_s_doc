Release notes 9.9.7 - "Cycle in procedure calls" error after confirming the availability of a specimen (GLIMS-13657)

Issue

A "Cycle in procedure calls" error occurred when confirming the availability of a specimen.

Configuration

  • Material M1
  • Properties P1 and P2
  • Availability trigger of P1:
    • RETURN .CascadeRequest("P2");
  • Procedure PC1:
    • Inputmaterial M1, Minimal size 50
    • Property output P1, Consumption 50, Value: Return "1";
  • Procedure PC2:
    • Inputmaterial M1
    • Property output P2, Consumption 20

Scenario

  1. Request property P1 in order entry and create the order.
  2. Open the Specimens of order browser for the new order.

  3. Confirm the availability of the specimen.

    => A "cycle in procedure calls" error occurred.

This issue, which occurred since GLIMS 9.9, has been corrected.
