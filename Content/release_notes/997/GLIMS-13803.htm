Release notes 9.9.7 - Generating an empty report should disable the 'Needs checking' flag (GLIMS-13803)

Issue

GLIMS_RX-00576 (9.9.0) fixed the issue where the Last output time of a report was set during report generation even when the generated report was empty. However, this modification had as side effect that the Needs checking flag was not disabled when the generated report was empty.

Solution

This issue, which occurred since GLIMS 9.9, has been corrected.

Warning - backward incompatibility

This modification restores the behaviour of GLIMS 8.11: generating an empty report disables the Needs checking flag again.
