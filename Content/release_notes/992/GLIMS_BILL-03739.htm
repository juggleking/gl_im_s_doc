Release notes 9.9.2 - Correction of error occurring during check digit calculation of Belgian national number (GLIMS_BILL-03739)

An issue was reported where the error Value too large to fit in INTEGER occurred when

  • entering a payment agreement with Matriculation type set to Belgian national number or
  • sending a MyCareNet insurability request

for a person born in 2015 or later. This issue, which occurred since GLIMS 9.6, has been corrected.
