Release notes 9.6.0 - Restore locale when changed by external component (MATE-04078)

An issue has been detected where an external component (e.g. Adobe Reader when displaying a PDF during a Quick report) could change the locale settings of GLIMS. As a result of a locale change, decimal values in GLIMS could be interpreted incorrectly.

GLIMS will now automatically detect if an embedded 3rd party component has modified the process locale settings and will restore them, if necessary. The MA_LC_NUMERIC variable can be removed in Progress.ini.
