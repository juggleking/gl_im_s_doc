Release notes 9.6.0 - Introduction of result responsible (GLIMS-07583)

A new concept of result responsibles has been introduced. For each individual result, GLIMS now stores a responsible. This can be used for:

  • Printing the result responsibles on result reports.
  • Determining the report responsible.
  • Determining the billing item's executor during tariffication (see GLIMS_BILL-03251)

For a full description, please check the chapter Result: Responsible.

Warning

The new concept of result responsibles has an effect on:

  • Printing responsibles (previously known as "validators") on result reports.
  • The determination of the report responsible, which is also used in the context of report approval.

Please thoroughly test this new functionality !
