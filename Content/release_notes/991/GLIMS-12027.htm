Release notes 9.9.1 - Automatically marking results with phone status set to No as To be phoned should be possible (GLIMS-12027)

The phone status of a result which had been manually set to No was never automatically set to Yes again, for instance, if the result value changed and its urgency or severity should have triggered the result to be marked as To be phoned.

This has been corrected: such a result can now automatically be marked as To be phoned.
