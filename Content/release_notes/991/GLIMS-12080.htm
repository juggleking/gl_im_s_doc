Release notes 9.9.1 - Fix missing images on GLIMS Report Builder reports because of heterogeneous environment (GLIMS-12080)

An issue was reported where images did not appear on reports generated by GLIMS Report Builder because of the image path verification which failed in the following circumstances:

Configuration

  • AppServer on Unix machine
  • Default image location on Windows machine (the ImageServerDir variable points to the default image location)

Routine

A report containing images that are located in the default image location is generated by GLIMS Report Builder using a GLIMS Report Builder service that is running on the AppServer.

Issue

The Unix AppServer cannot validate the Windows path of the images. As a result, the report is generated without images.

This issue has been corrected.
