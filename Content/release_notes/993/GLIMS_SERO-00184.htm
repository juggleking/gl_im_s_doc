Release notes 9.9.3 - Correction for scanning a specimen to an archive (GLIMS_SERO-00184)

The following issue regarding the storage of specimens in a lab archive has been solved:

  1. Three racks are available: r1, r2 and r3 (r2 and r3 both contain expired storage items).
  2. The user scans a specimen in the last free position of rack r1.
  3. GLIMS asks if the content of r2 should be cleared.
  4. The user confirms and scans another specimen.
  5. Instead of storing the scanned specimen in the position that has become available, GLIMS asks if the content of r3 should be cleared.
