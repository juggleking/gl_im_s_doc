Release notes 9.9.3 - MISPL expression builder: entering value for enumerated parameter made more user-friendly (MATE_MISPL-00115)

Entering a value for an (enumerated) parameter during the creation of a MISPL expression via the MISPL expression builder is now more user-friendly.

For instance:

  • The user has to provide a value for the MinimalStatus parameter and enters Confirmed (one of the values of the value list).

  • The entered value is now translated automatically into the syntax VariantResultStatus["Confirmed"] and added to the MISPL expression. In previous versions, only the value Confirmed was added to the MISPL expression, causing a syntax error.
