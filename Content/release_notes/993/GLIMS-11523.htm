Release notes 9.9.3 - Correction of visualisation issue in Result browser after using Esc (GLIMS-11523)

An issue was reported where, in the following scenario, an incorrect value was shown in a Result browser after using the Escape key.

  1. In an Order browser, select an order and click on the contextual ribbon item Results.
  2. In the Result browser, click in a result cell that already has a value (for instance: "A").
  3. Navigate to the previous / next order (via the page up / page down key or the Previous / Next button in the contextual Tools ribbon).
  4. Start typing without first putting the cursor in a cell.
  5. Press Esc.
  6. "A" is displayed in the result cell.

This issue has been corrected.
