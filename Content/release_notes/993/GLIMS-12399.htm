Release notes 9.9.3 - No more rescheduling when a specimen's availability is implicitly confirmed (GLIMS-12399)

Action rescheduling (which can be activated via the general option Reschedule when specimen becomes available) now no longer occurs if a specimen becomes available in an implicit way. That is, when

  • a result value is entered for the property that is assessed on the specimen,
  • executing the Specimen-based MISPL function .Confirm().

Rescheduling now only takes place if a specimen's availability is explicitly confirmed.
