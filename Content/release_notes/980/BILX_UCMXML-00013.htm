Release notes 9.8.0 - Export of financial data in "UCM XML" format (BILX_UCMXML-00013)

The following improvements have been implemented for the export of financial data in the UCM XML format:

  • When an invoice contains requests that are linked to the same billing code, these are exported in one <prestation> node. The quantity (<nombre>) is exported as the number of occurrences. The gross (<brut>) and net amounts (<net>) are now exported as: unit price x quantity.

    Example

    <demandePrestation>

    <nombre>2</nombre>

    <brut>17.36</brut> --> 8.68 (unit price) X 2

    <net>17.36</net> --> 8.68 (unit price) X 2

    <participation>0.00</participation>

    <devise>EUR</devise>

    </demandePrestation>

  • The <prestataire> fields in the export are numerical. During the export, non numerical characters in the identifications exported from GLIMS will now be filtered out. This will reduce the need for MISPL.
