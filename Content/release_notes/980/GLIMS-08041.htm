Release notes 9.8.0 - Avoid deadlock for reports (Oracle only) (GLIMS-08041)

Problem description

Assume the following situation:

  1. Two users are logged in GLIMS and perform the following actions simultaneously:
    1. User 1 validates result A. Result A will appear on Report X. Report X is locked as its NeedsChecking flag needs to be set.
    2. User 2 validates result B. Result B will appear on Report Y. Report Y is locked as its NeedsChecking flag needs to be set.
    3. User 1 validates result C. Result C will appear on Report Y. The NeedsChecking flag of Report Y cannot be set as it is locked by user 2.
    4. User 2 validates result D. Result D will appear on report X. The NeedsChecking flag of Report X cannot be set as it is locked by user 1.

This deadlock situation (which only occurred on Oracle systems) has now been solved.
