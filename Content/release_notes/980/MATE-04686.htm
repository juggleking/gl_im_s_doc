Release notes 9.8.0 - Update of the [WinChar Startup] section in the progress.ini file after OpenEdge upgrade (MATE-04686)

Since GLIMS 9.5.4, the GLIMS services use _progres.exe instead of prowin32.exe. While prowin32.exe uses the [Startup] section in the progress.ini file, _progres.exe uses the [WinChar Startup] section. We therefore made sure that, after an upgrade to a new OpenEdge version (e.g. from OE 11.3 to OE 11.6), the [WinChar Startup] section is updated. That way, the definition of DLC points to the installation directory of the new OE version, c:\dlc116 for instance.
