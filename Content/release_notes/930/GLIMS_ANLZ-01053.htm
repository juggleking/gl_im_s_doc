Release notes 9.3.0 - Issue when storing results received from analyzer (GLIMS_ANLZ-01053)

An issue was detected where a result sent by an analyzer was not stored because it was locked by another user or process. This has been improved.

Note: GLIMS will now try to store the result twice within the same second, but if the result is still locked the result will still be rejected.

However, GLIMS will from now create a log entry for each result that could not be stored. The auto-created log type in GLIMS is named "Station result".
