Release notes 9.3.0 - Incomplete phone list for "combined" orders (GLIMS-05966)

Problem description:

  • Two different issuers place the same order for the same patient on the exact same time.
  • The order entry option "Allows result reuse" is activated.
  • The result exceeds the phone list threshold.
  • In this case, only the first result will appear on the phone list. The re-used result does not.
  • Because of this issue, only one issuer was notified about the result while both issuers should have been.

This has been corrected.
