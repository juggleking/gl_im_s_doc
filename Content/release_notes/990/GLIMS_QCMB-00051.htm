Release notes 9.9.0 - Erroneous selection of inactive microbiology QC lot during automatic scheduling of a microbiology QC test (GLIMS_QCMB-00051)

Problem description

If a microbiology QC material has two microbiology QC lots,

  1. for current use: an existing one that does not yet have an end date and that is active,
  2. for future use: a new one that does not yet have an end date and that is not yet active,

then the most recent but inactive microbiology QC lot was selected upon scheduling of a microbiology QC test.

Solution

This has been corrected: the Active flag is now taken into account. If more than one microbiology QC lot is eligible, then the microbiology QC lot with the most recent start date is selected.
