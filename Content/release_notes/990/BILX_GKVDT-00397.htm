Release notes 9.9.0 - German KVDT: correct functionality of Referral.OfficialIssuerLANR restored (BILX_GKVDT-00397)

Issue

In GLIMS order entry, the Referral composite field Official issuer is split into separate fields that contain the LANR/BSNR/ASV numbers. Even if two different values for Referral.OfficialIssuerLANR for the same issuer happened to be available, only one LANR value per Order.Issuer was displayed in the BESA header of the KVDT file.

This issue only concerned LG shipments (LG site attribute on Firm), because LG shipments need to contain all BSNR and LANR numbers for all Order.Issuers in the BESA header. Non-LG shipments only contain the Firm associated LANR and BSNR numbers.

Solution

From now on, multiple different LANR numbers will be displayed for the same Order.Issuer (HC provider in case of LG shipments) in the KVDT BESA header.

If there is no value for Referral.OfficalIssuerLANR, the routine uses the Order.Issuer identification code as alternative.

Example

xxx0201 BSNR

xxx0203 HCProvider.Name (for LG shipments)

xxx0212 LANR 1

0219 HCProvider.Title

...

0218 HCProvider.Correspondent.EMail

xxx0212 LANR 2

...
