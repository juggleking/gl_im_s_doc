Release notes 9.9.0 - The function gp_Site.MergePersonsFromFile() can be run in batch (GLIMS-07200)

Issue

An issue was reported where the Person merge function Merge persons from file on gp_Site could not reliably be executed in batch. When executing the command on the server, the mandatory file name was not saved, so no setup was done and no merge action was performed. Therefore, large amounts of records could only be merged successively in limited amounts and over a long period of time.

Additionally, no errors about merging appeared in the task service log file.

Solution

The function gp_Site.MergePersonsFromFile() now fully supports execution in batch mode, i.e. via a task. All errors are correctly logged in the service log file.

Additionally, the function setup can now be skipped entirely.
