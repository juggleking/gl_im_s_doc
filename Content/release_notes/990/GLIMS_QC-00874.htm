Release notes 9.9.0 - Mandatory external comment for failed QC results: QC warning severity taken into account (GLIMS_QC-00874)

Context

When manually confirming QC results with a QC severity higher than 0 and without an external comment, the user is required to enter this comment during the confirmation step.

Current behaviour

The assessment method's QC warning severity is now taken into account: if the QC warning severity is defined and the QC severity is lower than this QC warning severity, no external comment is required.
