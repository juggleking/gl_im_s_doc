Release notes 9.9.0 - Correction of incorrect file generation with protocol HPRIM (BILX_HPRIMXML-00039)

When sending a financial shipment using the protocol HPRIM XML 1.07, two problems occurred:

  • a .ok file was generated, even when the OK option was not activated.
  • the file was written to the temp directory, not to the directory specified by the user.

These issues have been corrected.
