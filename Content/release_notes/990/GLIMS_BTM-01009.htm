Release notes 9.9.0 - Support of DGTI files with <CR> as end of line marker (GLIMS_BTM-01009)

Issue

GLIMS was only able to process DGTI blood bag files in which end of lines were marked with "<CR><LF>" (carriage return/line feed). However the DGTI documentation states that end of lines should only be marked with "<CR>".

Solution

The import system has been adapted: both import files using "<CR>" and import files using "<CR><LF>" as end of line markers can now be processed.
