Release notes 9.9.0 - Specimen size of new implicit specimen is recalculated after rescheduling (GLIMS-12038)

An issue was detected where the specimen size of a new implicit specimen was not set after rescheduling. The specimen size of all specimens should be recalculated.

Example

A user enabled Reschedule when specimen becomes available and created an order for which rescheduling is applicable. The availability of a specimen was confirmed, the specimens were correctly rescheduled but the intended specimen size was wrong.

Furthermore, in certain cases rescheduling was taking place while there was no need for it. Intermediate specimens should not be rescheduled when the resulting property, for which they were created, cannot be rescheduled.

These issues have been corrected: the specimen size is recalculated after rescheduling. Intermediate specimens are not rescheduled if the resulting property cannot be rescheduled.
