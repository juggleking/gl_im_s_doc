Release notes 9.9.0 - Correct execution of MISPL function .SetStorage on carriers (GLIMS_SERO-00165)

In the following scenario,

  1. Creation of a Carrier-based MISPL function running .SetStorage(),
  2. Execution of this function on a carrier,

a message error, "Rack and position must be empty in status To do" was displayed.

This error has been corrected and the carrier is stored.
