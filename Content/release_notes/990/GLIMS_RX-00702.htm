Release notes 9.9.0 - Support for nested LOINC coding system during result export (GLIMS_RX-00702)

Issue

If GLIMS is configured as described below, the LOINC code of the result of a property which belongs to a nested coding system of type LOINC is not exported when sending out result reports for which the LOINC code must be exported.

Note

This issue occurred since GLIMS 9.5.

Configuration (example)

Coding systems

A coding system KWS contains two code sets. No codes are defined for these code sets which refer to another coding system (via their Original coding system field).

 

  1. The LN coding system has its Type set to LOINC and refers to a code set containing codes for the Property table.
  2. The UCUM coding system has its Type set to Default and refers to a code set containing codes for the Unit table.

Report template

A report template exists which refers to the KWS coding system.

Solution

This has been corrected.
