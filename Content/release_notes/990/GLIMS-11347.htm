Release notes 9.9.0 - Remember last entered justification during successive request discontinuation (GLIMS-11347)

Scenario:

  1. Create an order containing several request codes.
  2. Reopen the order.
  3. Delete / Discontinue one of the request codes.
  4. Enter a Justification.
  5. Delete / Discontinue another request code.
  6. The user is again asked to enter a Justification. However, since GLIMS 9.3, the previously entered Justification was no longer displayed in the Discontinue requests screen.

This has been corrected: the Discontinue requests screen will now contain the previously entered Justification when deleting request codes, one after the other, without closing the Order screen.
