Release notes 9.9.0 - ADT event creation when merging Person records (GLIMS-11843)

Context

GLIMS offers an event handling system that can be used to send ADT messages to a result server or other system when relevant patient data is updated.

Issue

Until now, when merging multiple Person records, only one ADT event was created containing information of the ('incorrect') person that was merged last and the remaining ('correct') person. However, because of this, the result server was not updated correctly.

Solution

This has been corrected. GLIMS will now create ADT events for the other merged Person records as well.

Example

GLIMS and the result server contain four similar Person records. These four records are merged into one 'correct' Person record.

GLIMS will now publish three ADT events to ensure that the result server is synchronized correctly: one 'correct' record remains, the other three 'incorrect' records are deleted. If GLIMS had published only one event, the result server would still contain two 'incorrect' records.
