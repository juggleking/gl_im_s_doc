Release notes 9.9.0 - Incorrect lowest object time and reference time for order containing a result with absolute time discriminator (GLIMS-11206)

In the following scenario, the Lowest object time of the Order and the Reference time of the Requested code and of the Result linked to the Requested code will now also be updated.

  1. Create an order and request a property with an absolute time discriminator (e.g. 18:00)

    • Order - Lowest object time = 18:00
    • Requested code / Result - Reference time = 18:00
    • Specimen - Sampling time / Result - Object time = 18:00

  2. Select the related specimen and choose Change state > Change sampling time in the contextual menu / ribbon. Set New time to e.g. today 12:00.

    • Order - Lowest object time = 12:00
    • Requested code / Result - Reference time = 12:00
    • Specimen - Sampling time / Result - Object time = 12:00

In previous versions, the Lowest object time of the Order and the Reference time of the Requested code and of the Result linked to the Requested code did not change.
