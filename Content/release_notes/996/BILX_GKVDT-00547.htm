Release notes 9.9.6 - KVDT financial export for LG should ignore "Erstveranlasser" (FK 4217) (BILX_GKVDT-00547)

For LG KVDT shipments, the "Erstveranlasser" (FK 4217) is not used. However, it happens that an Order in GLIMS contains an LG and non-LG part (2 invoices with a different Firm). The "Erstveranlasser" (see order entry Referral page/Uschein from scan) value is in fact only relevant for the non-LG part. This mixed Order will produce an error during the KVDT export (of the LG firm).

This issue has been corrected: the "Erstveranlasser" is now ignored during the KVDT financial export for LG (firm).
