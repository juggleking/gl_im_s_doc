Release notes 9.4.0 - Report builder - Optional PCL stripping (GLIMS_ARep-00171)

For reasons of backward compatibility with existing property classification data, Report builder strips PCL print codes out of the content while building the XML files. Testing has revealed that this can have a negative impact on performance in those cases where it is not needed.

Therefore, we have added the ability for individual Report templates to opt out of PCL code stripping. To achieve this, disable the new Report template option Remove PCL printer codes.

By default, this setting is turned off when creating a new Report template. During the conversion to GLIMS 9.4, this option has been turned on for existing Report templates.
