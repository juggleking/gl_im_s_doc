Release notes 9.4.0 - Date format in order entry (GLIMS-06529)

Issue

In the order entry screen, dates were visualized in DD/MM/YY format, which could cause administration errors. When entering e.g. 09/01/2012, this could result in 09/01/20 instead of 09/01/12.

Solution

In the order entry screen, dates are now visualized using the full year format, i.e. DD/MM/YYYY.
