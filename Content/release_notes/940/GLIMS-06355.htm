Release notes 9.4.0 - Support new values for Patient.Sex (GLIMS-06355)

GLIMS now supports the following values for the Patient field Sex:

Short value Description
M Male
F Female
O Other
U Unknown
A Ambiguous
N Not applicable

Warning

These new values may not be supported by drivers and/or external software.

Warning

Customers may have site functions with IF ... THEN ... constructs that only check if the value is Male and that consider that the ELSE part is applied to the Female value.

Example:

IF Sex = "M" THEN <Male logic>;
ELSE <Female logic>;
ENDIF;

This should be replaced with:

IF Sex = "M" THEN <Male logic>;
ELSE IF Sex = "F" THEN <Female logic>;
ELSE <new enumeration logic>;
ENDIF;
