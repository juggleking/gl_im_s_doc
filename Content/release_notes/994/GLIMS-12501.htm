Release notes 9.9.4 - Fixed visualization error in order entry for order created via specimen reception scan (GLIMS-12501)

The following visualization issue, which occurred in order entry since GLIMS 9.8, has been corrected:

  1. Open the specimen reception scan.
  2. Enter a station and a material.
  3. Make sure that the specimen reception scan option Allow creation is enabled.
  4. Enter a barcode that does not yet exist and click the Scan button.
  5. Open the newly created order from an Order browser.
  6. The order entry screen showed duplicate specimens while the order outline only showed one and only one had been saved to the database.
