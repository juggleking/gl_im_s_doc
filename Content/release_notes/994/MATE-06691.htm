Release notes 9.9.4 - Lack of fixed extent for new areas added to a database with After Imaging (MATE-06691)

When migrating from GLIMS 9.8 to 9.9, two new areas are added to the database: an Instance Lob Area and a Template Lob Area.

For new areas, a fixed extent and an overflow extent should have been added. However, to avoid an issue in OpenEdge from occurring when After Imaging is enabled, a workaround was implemented in GLIMS 9.9.0. Due to this workaround, only an overflow extent was added but not the fixed extent that should have been added first.

This issue has been corrected.
