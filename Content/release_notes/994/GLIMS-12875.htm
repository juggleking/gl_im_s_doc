Release notes 9.9.4 - Ensure correct property output evaluation upon specimen confirmation (GLIMS-12875)

When the property outputs of a child specimen both have a trigger that updates the specimen (example: .SetStorage, .SetMeasuredSize, .SetReplacementLevel), an error occurred upon confirmation of the specimen.

Error message

Value xxx too large to fit in INTEGER.

Line xxxx in ProcessPropertyOutput spmn_cnf.

This issue has been corrected.
