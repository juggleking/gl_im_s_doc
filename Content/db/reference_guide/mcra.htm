Status

The microbiology action statuses have changed into:

Initial: initial status, specimen is not yet available.

Grafting: as soon as grafting starts until finished.

Microscopy: while microscopic analysis is being done. This status van overlap with grafting and reading.

Reading: while test results (isolations/antibiotic results) are being read.

Rereading: While more test results are read (After having been confirmed...)

ReadConfirmed: set in this status by the analist when he has finished reading. All isolations and results are confirmed.(auto-prompt results, grafting result, processing result, microscopy result, conclusion result, report result)

ReadValidated: set in this status by the biologist 'validating' all results and sending them back to the analist (in case of rejection, he will set the status to Rereading).

Ready: previous actions are done, but more results can still be entered, confirmed by the analist. If not ready, the status is set to Rereading.

Closed: action is closed by the biologist. Status changes are no longer allowed unless 'Reopened' by a person with the necessary priviliges.

Discontinued: action discontinued.

Isolations and all the results are confirmed when setting the status to ReadConfirmed and Ready. Analogue to the antibiogram results, they are validated when setting the status to ReadValidated & Closed. Except for the status discontinued, all statuses can be changed to another status if the user has the privileges to do so. The 4 first statuses cannot be changed to status ReReading.

When changing the status, the priviliges for the required methods have to be checked (see Microbiology Action).

Mandatory Automatic MicrobiologyActionStatus Status

One of: Initial, Grafting, Microscopy, Reading, ReReading, ReadConfirmed, ReadValidated, Ready, Closed, Discontinued

MISPL functions

AddCarriers 

This MISPL function allows adding one or more new carrier(s) to the current microbiology action.

It can be used, for instance, in the Carrier field "Trigger when comment changed" so that new carriers are automatically requested when a carrier comment changes to "P" ('positive'). Indeed, when a comment turns to "P", new tests need to be done, for which new carriers need to be requested.

Example:

if substr( .comment,1,1) = "P"
then RETURN .Action.AddCarriers("ut_Medium2",?,YES,NO,?);
else return YES;
endif;

Caution:

- Every parameter may have the value "?" except the first one: at least one medium mnemonic should be specified.
- Unfilled logical parameters are by default interpreted as NO.

Logical AddCarriers (String MnemonicList, Logical Print, Logical MarkAsGrafted, Logical DoubleAllowed, String Comment)

String MnemonicList
Comma-separated list of the mnemonics of the media to be automatically added. It should contain at least one mnemonic. It should not contain any white space.
Logical Print
If the value is YES, the label(s) of the added carrier(s) (if defined) will be printed.
Logical MarkAsGrafted
If the value is YES, the added carrier(s) will be grafted and the grafting time will be set to the current date and time.
Logical DoubleAllowed
Usually two carriers with the same medium cannot be added to one and the same microbiology action. However, if the value is YES, it will be possible to add two carriers with the same medium.
String Comment
The comment for the added carrier(s).

AntibioticCount 

This function returns the number of tested antibiotics. This count can be limited according to the parameter values of the function call.

PositiveInteger AntibioticCount (String OrganismMnemonicList, Logical Aerobe, String OrganismBillingMarkList, Logical Reportable, String AppraisalList, Logical Answered, String FormulaTypeList)

String OrganismMnemonicList
Comma separated list of microorganism mnemonics. If specified, the count will be limited to antibiotics tested on the listed organisms.
Logical Aerobe
If specified, the count will be limited to antibiotics tested on (an)aerobe organisms.
String OrganismBillingMarkList
Comma separated list of billing mark strings. If specified, the count will be limited to antibiotics tested on the organisms marked with one of the listed billing marks.
Logical Reportable
The 'Reportable' parameter is applied to both the antibiotic results themselves as well as to the isolations.  This implies that when the 'Reportable' parameter is set to TRUE, only reported antibiotics of reported isolations are counted. 
String AppraisalList
Comma separated list of isolation appraisal mnemonics. If specified, the count will be limited to antibiotics tested on isolations specifying one of the listed appraisals.
Logical Answered
If specified, the count will be limited to antibiotic results for which a value is (not) specified.
String FormulaTypeList
This parameter contains an optional list of applicable formula types.
If specified, only antibiotic results where one of the specified formula types has been applied (and a value exists for the formula type), will be counted.
The possible formula types are:
- "M" for MIC tests
- "E" for E-tests
- "D" for agar diffusion tests
Example: AntibioticCount( ... , "M,D" ) will count only antibiotic results where the RIS value was calculated from either the MIC or the agar diffusion value.

TextualSummary 

This field gives a summary of the microbiology action in textual format:
result (pos/neg) + isolations found + antibiotic results (optionally via parameter).

Example output:

isolaat,isolvol,antibiogram,

Corynebact ++,AMP:R SXT:S ....
Coagulase neg.Stafylokok +++

String TextualSummary (Logical PositiveOnly, PositiveInteger MinimalSeverity, String ViewFormat)

Logical PositiveOnly
If YES, only positive isolations (isolations with positive appraisal) will be reported. Any negative answer will be suppressed as well.
PositiveInteger MinimalSeverity
If specified, only isolations with this severity or higher will be reported. Any negative answer will be suppressed as well.
String ViewFormat
View format for the results. Possible values (separated by semicolons ';'):

Classification = <name>:

By default, the default property classification (or the default of your department) is used to sort the results and cultures. This option allows to specify a different classification. Example: Classification = Bacterio

ResultSeparator = ... :

By default, the regular tests that were executed during the microbiology actions are listed in the history one after the other on the same line and separated by ', '. Example output: Gram: neg, Cell count: >10000, ...

You can choose to separate them in a different way e.g. '; ' by specifying: ResultSeparator ='; '. You can also have them listed on separate lines. In order to do so, use '~n' to indicate the position of the newline character. Example: ResultSeparator =';~n' will put ';' after each result and each result will start at a new line.

The example output would then become:

Gram: neg;
Cell count: >10000;
...

Note: if the result separator contains at least one newline character, then this separator is not only used to separate the regular results from each other, but also between normal results and culture results or conclusion results, and also between the different isolations of a culture (in case there is more than one isolation).

Format = ... :

By default the values of results are shown in a format called "WorkFormat". You can choose to have them displayed in one of the following formats:

WorkFormat: when the value contains text modules (e.g. '{<TB}'), the text modules will not be expanded. Only the name of the text module will be shown (e.g. 'TB')

ReportFormat: when the value contains text modules, the text modules are expanded (e.g.: 'Tube broken').

Nonreportable = YES : By default, isolations and antibiotic results that are marked as 'non-reportable' (which means they will not appear on reports) are not listed by this function. However if you still want to report these 'non-reportable' items with this function, specify Nonreportable = YES. Note that isolation and antibiotic names will be prefixed by '~' to indicate their non-reportability.
