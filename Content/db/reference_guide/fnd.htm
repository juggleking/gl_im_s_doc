Fund fnd glims yes yes Normal

Funds are organizations which can get involved in the reimbursement of an order set by means of its Financing records. Note that a fund is not related with an order for other than financial reasons. Issuers, patients, object owners, etc. cannot be funds, since they are already connected to an order or its order set for other than financial reasons. Health care insurance organizations are the most common example of funds.

Extra > Default alternate id

In Germany the "Alternate HCCode" of a Fund is an IK number. A Fund can have multiple IK numbers and a Fund is usually identified with such an IK number. If the fund of a patient is known but not its IK number then this "default alternate id" must be used as the IK number for that fund.

In France this field contains the 'Type de destinataire' of the first fund. Example CT(CPAM), SM(Grande mutuelle), ... .

Field protection

This field is used to implement a record update blocking mechanism. If this field contains a value, then the fund record can not be updated using the fund editor. The FieldProtection field itself is not visible: it can only be set by dedicated programs. In Germany this field is used to prohibit manual changes to funds ('Kasse') that are imported from the official KBV file ('KostentragerStammdatei').
