Identification idnt glims yes yes Normal

Identifications can be used for communication with external applications or correspondents. For instance, a HCProvider can be known to a specific Institution under some code, or Persons can be known to external person data providers (HIS system) under some code. Identifications are also used to associate personnel to specific laboratory departments. This can be further used to implement some aspects of row-level security, such as access rights to procedures scoped to the department.

Queries

Double identifications

Query searching identification records with the same code

DoubleIdentifications (Logical ForCommonSource, String MinimalCreationDate, gp_SiteFunction RankingFunction)

Logical ForCommonSource
If set, double identification codes will only be selected when issued by the same provider.
String MinimalCreationDate
Only identifications created during this period will be checked for existence of doubles.
gp_SiteFunction RankingFunction
The identification with the highest ranking will be considered as the preferred one. Duplicate identifications will be displayed in descending ranking order and the identification with the highest ranking will be passed as the 'good' one to the merge function.
