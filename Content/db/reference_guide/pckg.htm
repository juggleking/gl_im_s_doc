This table holds the info of a package. One package is a journey a box (container of specimens) makes from department X to department Y. This is a part of Logistics Manager.

Fields

Courier info > Aborted on

This field is filled in when the boxtrip is aborted.

DateTime AbortionTime

Main > Active

This logical value is when a boxtrip is being processed. When a boxtrip is unpacked or aborted, this value is set to FALSE.

Mandatory Logical Active

Main > Box number

This is the Id of the physical box. This is a string since a physical box is not modelled and this insures us that we can save anything that is scanned in.

Mandatory String BoxNumber

Courier info > Collecting courier

This stores the Id of the collecting courier (simple string, no real DB connection to a person)

String CollectingCourierId

Courier info > Collected on

This field is filled in when the boxtrip is collected.

DateTime CollectTime

Automatic DateTime CreationTime

Creation user

Courier info > Delivering courier

This stores the Id of the delivering courier (simple string, no real DB connection to a person)

String DeliveringCourierId

Main > Delivery

This field holds the department id of the department where the box is delivered, it is thus possible that this is different from the destination department.

Department DeliveryDepartment

Main > Remarks

When the box is delivered, the courier can fill in some remarks, these are stored here

String DeliveryRemarks
