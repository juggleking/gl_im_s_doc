Invoice items represent the lines of an invoice. They contain the data (e.g. price, VAT, ...) of a billing item to be payed by a specific payer.

Credit-note creation 

This function allows the user to 'reject' a number of invoice items. Rejecting means crediting those lines by putting them on a new or existing credit note. The related interface is similar to the 'reject invoice' interface.

MultiRecord Void InvoiceItemReject (Logical AssignDocumentNo, String RejectionReason, Logical Reuse, Logical CreditOtherParties, Logical DowngradeOsStatus, Logical UpdateRequests)

Logical AssignDocumentNo
If TRUE, newly created invoice summaries are immediately numbered.
String RejectionReason
Free text, describing the rejection reason. Will be copied to the corresponding field of the credit invoice.
Logical Reuse
If TRUE, this method will try to reuse any existing credit invoice (for the same order, firm, payer, ...) that has not yet reached the status fixed.
Logical CreditOtherParties
Should other invoices items (for another payer) of the same billing items be credited as well?
Logical DowngradeOsStatus
Should the order set status be downgraded also? That way the order can be automatically be selected for a retariffication.
Logical UpdateRequests
Should requests status be downgraded as well ?

Standard invoice item query

The standard invoice item query.
performance note:
The query is normally opened starting from the Invoice table.
If an order issuer or a minimal receipt date is specified then it is opened starting from the Order table.
