This table holds the budget per correspondent, per order and per budget class. Creation of BudgetInvoices is defined by BudgetDefinitions

Fields

Amount

The BudgetInvoice amount is the sum of the amounts of the budget items

Mandatory Automatic Fractional Amount

Budget

The budget that is charged with this budget invoice

Mandatory Invariant Budget Budget

Budget class assignment

The budget class assignment that caused this budget invoice.

Mandatory Invariant BudgetClassAssignment BudgetClassAssignment

Creation time

Creation user

Dirty

If set, this budget invoice is a candidate for recalculation

Mandatory Logical Dirty

External date

This date determines to which budget (period) this budget invoice is assigned. This field is set by the budget calculation program based on the settings of the budget policy.

Mandatory Invariant Date ExternalDate

Id

Primary key. Automatically generated identifier whose value uniquely identifies each record.

Mandatory Unique Automatic BudgetInvoice Id

Last update time

Automatic DateTime LastUpdateTime

Last update user

Order

The order for which this budget invoice is calculated

Mandatory Invariant Order Order

Owner type

The reimbursertype of the owner of this budget invoice.

Mandatory Invariant ReimburserType OwnerType

One of: Object person, Issuer, Fund, Object owner, Object supplier, Order agent, Institution, Not Applicable, Dispatching Lab, Receiving Lab, Reimbursement party, Holder, Site billing partner, Specific instance, Fund 2, Study

Points

The budget invoice's points are calculated by multiplying the budget invoice amount with the budget class assignment's price code

Mandatory Automatic Fractional Points

Relevant

Is this budget invoice relevant for the budget? This is the result of the evaluation of the budget class assignment's 'Eligibility' site function (MISPL), but can be manually overwritten. A budget invoice that is not relevant is not added to the budget although it does maintain a reference to that budget.

Mandatory Logical Relevant
