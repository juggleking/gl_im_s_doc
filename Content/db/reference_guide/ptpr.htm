Staff > Responsible > Determine responsible

It is possible to specify a MISPL to automatically determine the responsible for a pathology examination. This MISPL function should return a HC provider that is assigned as responsible in any of the employment records for the department (in which the order is entered). Moreover, the returned HC provider should accepted by the MISPL 'Allowed as responsible' that is defined for the pathology procedure.

gp_SiteFunction DetermineResponsible

Staff > Technician > Determine employee

As soon as an exam is created -that is during order entry- the exam can be assigned automatically by means of a MISPL function. This function must be defined on the pathology procedure, in the field "Determine technician"

The MISPL function must be based on table 'PathologyExam' and must return the 'login name' of the GLIMS user whom the exam should be assigned to. This user should be (actively) employed in the department of the exam.

Example MISPL:

---------------------------------------------------------------------------
RETURN IF .Specimen.Material.Mnemonic = "BIOPSY" THEN "Fred" ELSE "Barney";
---------------------------------------------------------------------------

This mechanism can be used e.g. to select the cut-up technician for each exam.

gp_SiteFunction DetermineTechnician

Domain

Allows classifying the pathology procedures. The domain is a 1-or-2-character indication (abbreviation) that can be used to group or classify the pathology procedures according to their type, usage, medical discipline, ...

Examples:
Cytology -> C
Cervical cytology -> CC
Histology -> H
Autopsy -> A

The list of possible domains must be configured in a site-domain with the name 'Domains' (this must be created manually). Specify the list of possible domain abbreviations as a comma-separated list in the field 'Value list'; e.g.: 'C,CC,H,A'. If you want, you can also add longer names to this; e.g.: 'Cytology\C,Cervical cytology\CC,Histology\H,Autopsy\A'. In the pathology query browser it is possible to select on pathology exams of a certain domain.

Other > Specimen scope

Allows specifying that such a procedure requires a specimen with a specific scope. This allows making sure that a procedure or group of procedures gets its own specimen.

Example:

If you request two pathology procedures (or a combination of regular procedures and pathology procedures) which would normally be able to use the same specimen, you can indicate that each procedure requires its own specimen by defining a different specimen scope for each procedure.
