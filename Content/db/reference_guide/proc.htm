Procedure proc glims yes yes Normal

Procedures are models of basic laboratory activity, i.o.w. this table stores models for actions that can be performed in the laboratory. Actions are defined in the instance space, whereas procedures are defined in the template space. A procedure is defined by its inputs and its outputs, each either material or property. A procedure can be regarded as a declaration of what can maximally be obtained from a given set of inputs simultaneously, in a single pass. From another point of view, it defines what is required as input to obtain a specific output. A procedure also defines where the derived actions will take place. When an action instantiates a procedure, all the procedure inputs must be instantiated (realized). However, only a (requested) subset of the procedure's possible outputs must be instantiated. All inputs of an action must be available for it to produce any output.

A procedure requires zero or more inputs and can deliver zero or more outputs. Inputs and outputs are collectively called channels. Channels are either of type material or type property.

Main > Cell counter > Cell count cycle

Makes it possible to get an audible signal each time this number of cells have been counted. The user can continue counting until the cell count limit is reached. This can be helpful if the user decides that not enough cells have been counted to get a reliable result. When no cell count cycle is specified, the counter simply stops when the cell count limit is reached.

PositiveInteger CellCountCycle

Main > Cell counter > Cell count decimals

Allows specifying how many decimals have to be used when calculating the percent value of the fraction. The default value is 0, but can be changed to 1 to allow storing the fractions with a greater precision. This may be needed when the "extended" counting yields counts like "1 in 200".

Mandatory PositiveInteger CellCountDecimals

Main > Cell counter > Cell count limit

Allows to specify the maximum number of cells that you would like to count in order to obtain realistic statistics. In practice this usually is 50 or 100. The cell counter stops counting as soon as this limit has been reached. When this happens, an audible signal is given (i.e. the snd/CountFinished.wav file is played) to attract the user's attention.

PositiveInteger CellCountLimit

Main > Cell counter > Cell count property

Allows to specify for which property the fractions are being counted via this procedure (e.g. white blood cells for the case of the haematology formula counter). When this property is specified, the cell counter will show a new column in the counter browser, displaying the absolute values of the counted fractions (i.e. fraction (in %) * Procedure.CellCountProperty result value / 100). These values are only shown to check the results of the counting process. They are not stored in the database.

Property CellCountProperty

Main > Microbiology > Don't confirm/validate with microbiology

This option can be used to avoid that the procedure's output results are automatically confirmed or validated whenever a microbiology action on the same specimen is confirmed or validated. This option does not have any effects for procedures that are connected to the microbiology action via a Grafting/Microscopy/Processing property.

Main > Specimen scope

The scope of a specimen defines by which procedures this specimen can be used. If specified, the specimen can only be used by procedures having the same scope. If an order requires 2 procedures for the same material but with a different specimen scope, 2 specimens will be scheduled, each destined for its own procedure.
