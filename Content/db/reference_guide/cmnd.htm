For certain tables, a number of commands are available. A command executes a function on the results of a query. By means of a task, the defined command can be executed according to several time and date-related criteria. The combination of a table with a query and a function is stored in the Command table. Both the query and the function can be tuned by assigning values to parameters. Associated with the command are actual query parameters (AQP), each defining a value for their corresponding formal query parameter (FQP), and actual function parameters (AFP), each defining a value for their corresponding formal function parameter (FFP).

Menu functions

Do 

Choose Do from the command contextual menu to run the command interactively. Note that commands are usually run in batch mode through a task scheduler. This function is most often used to test command definition. It will bring up the query and function dialogs respectively, pre-configured according to any actual parameter sets you referenced from the command. If you click OK in both dialogs, the command will be executed.

MultiRecord Void Do ()

Do via task scheduler 

Choose Do Via Task Scheduler from the command contextual menu to setup any query and function parameters, then create an associated task of type "Once", which can be picked up for immediate background execution by a task scheduler. The task will be executed only once and its record will be deleted after execution.
