Family fml glims yes yes Normal

Collection of (usually related) persons (family members) where it is suspected that some of them have a certain genetic disorder.
A family can also be used to investigate more than one disorder.
A family is owned by a certain department and can be visualised graphically in a pedigree.

Id

Mandatory Unique Automatic Family Id

Number

Similar to e.g. Person Internal Id, this field will be filled automatically on creation, by use of an (optional) mispl function that can be defined in the GLIMS general settings.
If the mispl is not defined, GLIMS will build the internal id as if the mispl contained
Identifier( “Family”, DateToString(Today(), "%y") + "-", 4 )
(which means it will be a 4-digit sequential number within each year, prefixed by the year).
Afterwards the internalid of a family can be changed manually.

Mandatory Unique String InternalId

Disorder MainSuspectedDisorder

Name

This is a longer string that contains more info on the family, like name and purpose.
On creation of the Family, we will handle the same system as for the internal id:
• It will be filled automatically on creation, by use of an (optional) mispl function that can be defined in the GLIMS general settings.
• If the mispl is not defined, GLIMS will simply copy the patient’s LastName to the family Name field.
