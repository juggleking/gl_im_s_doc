Identification > Order > Alternate order data function

Allows specifying an order-based MISPL function returning a string value. When specified, GLIMS will download tagged values from this string value to the station. If no tags are present, GLIMS will download the default values. The table below shows the supported tags and the corresponding MIPS ASTM fields:

Tag, Field Default when no tag
WARD1 P.26 Order current encounter -> current Stay -> Ward
WARD2 O.28.1 If requested in download query:
Order.Agent->Correspondent.InternalId (if ward) or
Issuer.Order->Correspondent.InternalId (if ward)
WARD3 O.28.2 If requested in download query:
Order.Encounter -> current Stay -> Ward
ISSUER O.17 Order.Issuer->Correspondent.InternalId

Example of a site function:

STRING List;
STRING IssuerId;
STRING WardId;
IssuerId := .Issuer.Identification("ISRPROV");
WardId := "DEP100"; /* Default ward */
List := PutTag("Issuer", IssuerId, List, "\");
List := PutTag("Ward1", WardId, List, "\");
RETURN List;

Microbiology > Results > Antibiotic panel mode

GLIMS allows automatic antibiogram creation. Possible values:

- None (no panel creation, allow any result): all antibiotic results transmitted by the instrument are stored.
- Reject non members: an antibiogram will be generated automatically at the moment the first result arrives. The results of non antibiotic panel members that are transmitted by the instrument will be rejected.
- Non members not reportable: an antibiogram will be generated automatically at the moment the first result arrives. The results of non antibiotic panel members that are transmitted by the instrument will be set to not reportable (2nd and 3rd option).

Note: When the 2nd or the 3th option is active, beware that results will also be rejected or will also be set to not reportable when no antibiotic panel is generated for the isolation concerned.

Mandatory StationAntibioticPanelMode AntibioticPanelMode

One of: None, Reject non members, Non members not reportable

Work order > General > Test selection > Download finished tests

Glims is able to download confirmed/validated and/or discontinued tests in a download message. The finished/discontinued tests are downloaded together with new requested tests. This functionality is activated if following conditions are met:

1. To download confirmed/validated tests: the station receives a test selection query with an indication 'FinishedTests' (MIPS ASTM field Q.15.x)
To download discontinued tests: the station receives a test selection query with an indication 'CancelledTests' (MIPS ASTM field Q.15.x)

2. The station for which finished tests should be downloaded, should have the field "Download finished tests" enabled. Note that the field has only effect on the requests scheduled for that station.

For performance reasons, Glims will only search maximal 24 hours in the past for finished results. For each new ALL-query, results finished before the last query are not downloaded again. Even if a downloaded and confirmed result becomes validated, the result test is no longer downloaded.

Mandatory Logical DnlFinishedTests

Identification > Specimen variables > Specimen variables

This option determines which specimen variables should be downloaded to the analyzer.

1. Only source body site: Only the specimen variable known as the specimen source body site (E.g.: left arm, left hand, right lung) is downloaded. The source body site variable is identified by the material variable for which the name equals the one specified in the "Source body site" field. In case this field is not specified, Glims will assume that the body site variable is the one with the lowest sequence number.

2. All: all specimen variables are downloaded. If the specimen source body site is identified in the "Source body site" field, Glims will download this specimen variable as source body site. All other specimen variables are downloaded as non specimen source body site variables. If no specimen source is identified as being the source body site, Glims assumes there is no source body site and will download all specimen variables as non specimen source body site variables.
