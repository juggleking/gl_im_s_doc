Start time

Status Pending:
Date-time that the activity is expected to start (based on calculation)
It will hold the expected start date-time, based on calculations. This has to be recalculated each time one of the preceding activities gets another start or end date-time.

Other Status values (>Pending):
Date-time that the activity has started (status changed to Running)
As soon as the status becomes Running, the effective start date-time is entered and then it is no more changed.

DateTime StartTime

Status

An ApproachActivity has a status.
This is an enumerated with following values:
Pending: Activity is not started yet and waits for preceding activities to be finished first.
Note that is not always certain that it will ever be executed.
Running: Activity has started and is running (busy), but not finished yet.
Finished: Activity has completely finished.
If the type=Procedure, it means that all linked Actions have status Ready, which means that their outputs are all either Confirmed, Validated or Discontinued.
Discontinued: Execution of the activity was aborted, e.g. due to discontinue of the order.
Error: Error occurred when starting or running the activity
E.g. the Procedure to be used (or its Station) is disabled.

Mandatory ApproachActivityStatus Status

One of: Pending, Running, Error, Finished, Discontinued

Transition error

If some error occurred while evaluating the outgoing transitions, the execution of the approach has to be put on hold.
This field will indicate which type of error has occurred.

MISPL failure: one of the outgoing transitions has a restriction mispl that raised an error.
Stuck: All outgoing transitions have a restriction, but none of them evaluated to TRUE

TransitionError TransitionError
