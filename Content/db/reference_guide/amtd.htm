Limits > Delta norm has precedence

If set, and a delta check is done, then the delta norm severity overrules the norm severity for the auto-confirm/validate decisions. So, a result that is pathological (norm severity > 0), and that normally wouldn't be auto-validated, might still be auto-validated because it's similar to the previous result of the same patient.

The actual severity that is compared against the auto-confirm and auto-validate levels, is the global result severity (Result.Severity) without the norm severity component (Result.NormSeverity). So, it is calculated as the maximum of the absolute values of all severities, EXCEPT the norm severity.

Mandatory Logical DeltaNormPrecedence

QC > Quality reset

If you want to setup an automated channel quality monitoring system, this field should be set to "End of run". By doing so, the following mechanism will be applied:

- When a new run is started, the channel quality is set to 'Reliable'

- As soon as the first result enters, the channel switches to 'Dubious'.

- When a qc result with a qc severity exceeding the assessment method's QC Alarm Severity enters, the channel quality is set to 'Unreliable' until a new run is started

- As long as the required start qc and the required end qc results have not yet been entered, the Quality remains 'Dubious', thus prohibiting any automatic confirmation or validation of patient results. This means that you will have to process these patient results manually either using the validation programs in GLIMS or by processing the corresponding station run.

- When no end qc is required, entering the required number of start qc results without alarm values will result in the status of the channel being switched to 'Reliable'

- When no end qc is required, entering a start qc result too many will be interpreted as the beginning of a new start phase (thus requiring the entry of an additional number of required start QC results).

- When more patient results are entered than specified in the QC cycle length without QC results in between, the quality is switched to 'Unreliable'

Note that even when the "Quality reset" field is NOT set to "End of run", a patient result entering the system more than <AssessmentMethod.QCValidity> hours after the last qc result will switch the quality to "Unreliable".

Mandatory ChannelQualityReset QualityReset

One of: Manual, Automatic, Automatic - same material, End of run (non supported), Automatic - all populations

QC > Required end qc

Indicates the number of QC samples with which the run ends. If not specified, the first QC samples after the patient samples will be considered as the start of a new run, i.e. the first start QC of a new run.

Beware: when defining end QCs, the quality status of the patient samples in the run will never automatically be set to "reliable", which implies that you will have to process these patient results manually either using the validation programs in GLIMS or by processing the corresponding station run. If the start QC is OK, the status of patient samples will be "Dubious" even if the end QC was also OK !! If the start or end QC is not OK, the status of the patient samples will naturally be set to "Unreliable". In both cases, the patient results will have to confirmed/validated manually !

PositiveInteger RequiredEndQC

Status > Status

This field can only be set manually to one of the following values:

- Disabled: indicates that this channel should no longer be used.
- Enabled: if you want to make full use of an assessment method, its status should be set to "Enabled".
- Closing: indicates that this channel will no longer be used for scheduling, but result upload from the analyzer to Glims is still possible.
- Locked: indicates that this channel is still used for scheduling, but Glims will not download orders via this channel. This can for instance be used to buffer all tests requiring an expensive reagent. When all tests requiring this reagent are scheduled, you can reset the status to "Enabled" and all tests will be downloaded to the analyzer.

Mandatory LISStatus Status

One of: Disabled, Closing, Enabled, Locked
