Lab lab glims yes yes Normal

A lab is an organization that is authorized to perform procedures. An authorization by the national health care is represented by means of a unique health care code. Both internal and external (storage required for billing) labs are represented here.

Fields

Billing item urgency

This field provides access to the (executing) lab's billing item urgency site function (Request based, string valued). The function should return a string value as defined for the site enumerated 'BillingUrgency'.

Each time a request is tariffed, the billing item urgency site function of the request's executing lab, is evaluated. The return value specifies its urgency level and is saved in the corresponding billing item(s). Returning '?' is accepted and considered as not urgent. Returning a value which is not a billing urgency level causes a tariffication error.

Validator is executor

This field affects the way the billing item executor is determined. If set, the tariffication program will evaluate first whether the test's or microbiology action's validator can be considered as the executor. This can be checked by means of dedicated provision records (Purpose: Validator verification).
If the validator verification fails, the executor will be determined by means of other provision records (Purpose: Executor determination).

Mandatory Logical ValidatorIsExecutor
