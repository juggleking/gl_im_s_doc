This table stores models for the output of properties by a given procedure. It indicates that a given property output is carried out by applying an assessment method.

Fields

Method

The assessment method is used to assess this property.

Mandatory Invariant AssessmentMethod AssessmentMethod

Auto prompt

If the toggle box is set, no request is needed for the property to appear on the work list.

Mandatory Logical AutoPrompt

Behavior in genetics approach plan > Auto request when starting activity

Used for genetics.
When -in an approach plan- an activity (of type 'Procedure') has to be started, an action must be created.
If this flag is TRUE, then this new action must get -as output- a result for this property.

Mandatory Logical AutoRequest

Consumption

This field specifies the minimal volume or size required/consumed to assess the related property. It is expressed in the common unit of all the procedure's material inputs. If this field is not set, then it is assumed that no consumption takes place.

Schedule preference

This field provides support for alternative possible procedures for a given property. During action scheduling, the alternative property outputs for a given property will be tested for eligibility in the order of their schedule preference. Also, when using the immediate specimen scan option in order entry, this field helps to determine which of the available materials should be used for the requested property.

PositiveInteger SchedulePreference

Value

This field is currently used for standard results and formulas (calculations). It is a default value expression (Action-based). If known, it is automatically evaluated when the corresponding Action reaches the status 'Outputting'. The expression yields a fractional for numerical properties, else a string.
