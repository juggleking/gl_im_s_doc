Each tariff (= price list entry) record represents an entry of a billing code in a price list. It has a limited validity date range.

Fields

Other > Account

This field provides access to the tariff's account. It is used for reporting to an accounting module. The field is optional, but the accounting transfer program will produce an error if the field is unknown.

Validity > Valid until

This field indicates the validity period's end date. The tariff is valid up to and including the specified end date. If no end date is specified, then the validity period is implicitly ended (excluding) at the successing record's start date. The successor is the next tariff record for the same price list and billing code in order (ascending) of start date. If there is no successor, the validity period is considered as not ending.

Date EndDate

Other > Executing class

This field provides access to the tariff's executing class. The executor determination requires for each billing code an executing class. It can be specified at the tariff level or the billing code assignment level. The latter has the highest priority.

ExecutingClass ExecutingClass

Queries

Price report query

Used only when generating price reports

PriceReport (PriceList PriceList, Correspondent Issuer, Logical ConsiderIssuerPriceList, Logical IssuerSpecific, Correspondent Correspondent, PaymentAgreement PaymentAgreement, PriceList PayaPriceList, Logical UsePayaPriceList, Logical WithExtensions, Logical ShowCalculation, String ValidityDate, String SortMethod)

PriceList PriceList
If specified, and no pricelist relating to a specified payment agreement is used as report input, then a price report is generated with tariffs of this pricelist.
Correspondent Issuer
Used when an issuer specific price report has to be generated, during selection of an issuer default the issuer's price list (if known) is taken as pricelist + automatic indication 'consider as issuer price list'.
Logical ConsiderIssuerPriceList
Additional parameter to indicate that in case of use of an issuer-specific pricelist, only tariffs of issuer-specific requestables (or general ones if not known) may b shown (with issuer-specific requestable preferred above general requestable).
Logical IssuerSpecific
Additional indicator : if set using an issuer-specific pricelist, only tariffs of issuer-specific requestables will be shown (this indicator only has effect with the previous parameter ConsiderIssuerPriceList set TRUE)
Correspondent Correspondent
Used when a payer specific price report has to be generated, a payer specific payment agreement can be selected.
PaymentAgreement PaymentAgreement
General or payer specific payment agreement, can refer to a specific price list, discount, ...
PriceList PayaPriceList
If specified (related to a general or specific payment agreement), and to be used as report input, then a price report containing valid tariffs of this pricelist will be generated.
Logical UsePayaPriceList
Indicates whether the specified pricelist of a payment agreement is input for the price report. If not the first specified pricelist is input for the price report
Logical WithExtensions
Indicates whether also the price list extensions (tariffs) must be included in the price report
Logical ShowCalculation
Indicates whether some more detail has to be include in the price report
String ValidityDate
If specified, only tariffs valid on this date will be included in the price report
String SortMethod
Indicates how the price report tariffs must be sorted : by billing code (standard), by requestable mnemonic or by nomenclature code
