Work list template wlt glims yes yes Normal

Work lists are ordered sets of procedure realizations, or actions. Work list templates are models for work lists, determining the basic criteria for inclusion and sorting of actions in work lists, as well as some layout directives.

Work list templates are empty until they contain one or more work list template entries, identifying procedures to include realizations from.

Note that a work list template, grouping only procedures without inputs and with material type outputs, is a sampling list template.

Work lists are commonly used as a medium for listing the work to be done by a particular work unit (person or station) in the laboratory. There are two alternative ways of visualizing work lists: Grid view and Outline view. The template record provides attributes for both types of work list visualization.

In Grid view, the work list is visualized as a two-dimensional grid of cells, organized in rows and columns. Each row corresponds with a work list entry, each column corresponds with an action output (either specimen or result). Grid view is most often used to edit work lists containing assessment actions. It is implemented with a generic grid control: in each cell, the result value can be viewed or modified. The template's expression attribute ActionLabel delivers a string for labeling each entry's row in the leftmost grid column.

In Outline view, the work list is represented as a tree-like hierarchy with object, action and specimen/result output levels in increasing order of detail. Outline view is most often used to print sampling lists. Of particular importance in outline view are the template's text attributes ObjectSection, ActionSection, SpecimenOutputSection and ResultOutputSection. They describe the layout and contents of sections to be generated whenever the outline generator encounters a new record of the corresponding type.

Layout > Grid > Rows > Grid history count

Indicates how many history rows have to be added to each action during printout or editing. A history row is a virtual action consisting of all results of a previous order. The previous order is searched by tracing the history of the results of the property of the first column of the work list.
A history row can be recognized by MISPL (e.g. for row label purposes) via its Action.Status field which is set to a value < 0. The virtual action's object time is set to the object time of the triggering historical result.
