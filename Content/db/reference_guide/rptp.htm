Microbiology > Antibiogram > Character to use in empty spaces

This character will be printed in all empty places (cells) in the antibiogram, i.e. where the combination of antibiotic and germ was never applicable. This is where the test of the germ on that antibiotic was not requested, usually because the antibiotic is not a member of the selected antibiotic panel. The default for this field is " " (blank).

Example:

Suppose an antibiogram looks by default like this:
1 2 3
AMP R I S
CEFA S I
PEN S

If the fill character would be changed into e.g. '-', then the antibiogram would look like:

1 2 3
AMP R I S
CEFA S - I
PEN - - S

Microbiology > Antibiogram > Specific layout > Antibiotic sorting

This is a MISPL function to change the order in which antibiotics are sorted within the antibiogram. The function should return a string according to which the antibiotics will be sorted on the report. The default sorting is done on the antibiotic sequence number.

Example mispls:

/* like default: */
RETURN String(.Antibiotic.SeqNo,"%009d");

/* sort by mnemonic except for Dr.Zivago: take special sort sequence defined in site attribute on antibiotic table */
IF .Order.Issuer.Name = "Zivagho" THEN
RETURN GetSiteAttribute("Antibiotic", .Antibiotic.Id, "SequenceZIVAGO");
ELSE RETURN(.Antibiotic.Mnemonic);
ENDIF;

Layout > Footnote type

The footnote type determines how footnote references are used. Possible values:

* None: do not use footnotes at all and put all values and comments in the cell. In MS-Word reports, the cell formatting is left to MS Word, while in text reports, the cell formatting is done by the report program itself.

* Before: use the Glims footnote mechanism, taking the PropertyClassificationNode.FlushCommentsAfter into account and add the footnote reference to the left of the result value. This option applies to text reports and MS Word reports.

* After: same as above, but the footnote reference is added to the right of the value.

* Native: only applies to MS word reports: use the MS-Word native footnote mechanism, disregarding the FlushCommentsAfter settings for title nodes.

Mandatory ReportFootnoteType FootnoteType

One of: None, After, Before, Native

Layout > Group results in columns by

Determines how columns are set up on the report. Possible values are:

* ObjectTime: if an order contains different object times, a column will be created for each object time. However, a report column always refers to a single order. This implies that two orders with identical object times will be displayed in two columns !
* Order: one column per order.
* OrderDate: one column per order per object date. When selecting 'OrderDate', an order that is spread out over several days will be subdivided. For each day, a separate column will be created. However, there is one exception: dynamic tests spread out over more than one day will be grouped on the day of the first test.
* Specimen: one column per specimen.

Microbiology > Antibiogram > Hide untested cells in antibiogram

If 'off' (default), antibiotic results without a value will still be reported in the antibiogram as a '.' sign (by default).

Example:
1 2 3
AMP R . .
CEFA . .
PEN S
But if the "." places should not be shown at all, switch on this toggle.
The example would become:
1 3
AMP R
EN S

So it looks on the report just as if these antibiotic tests were never requested in the panel.

Important remark:

If a fill character has been specified (non blank) for the empty cells (see option 1), the untested values will not be shown as '.' but as '?' by default. This is because the dots are visually lost between the fill characters. So the above example antibiogram would become (with e.g. '-' as fill character):

1 2 3
AMP R ? ?
CEFA ? - ?
PEN - - S

If the untested cells should be shown in a different way than this standard (e.g. as "Not Tested"), use the "antibiogram values" MISPL in the same tab folder to achieve this. Example:

String str;
str := ?;
IF .Antibiotic <> ? AND .RISReportValue = ? THEN str := "Not Tested"; ENDIF;
RETURN str;

Rows > Text reports > Row labels > Method row label

Allows to add Result.Method (or AssessmentMethod.Method, see note below) on a text report as a new row following the result row. The method is added as soon as row label is specified in this field or the field ReportTemplate.MethodRowMask is filled and the current result's method has a name. The method name after a test will be suppressed when the next test on the report has the same method name.

Note:

The Result.Method field stores the value of AssessmentMethod.Method at the time of result entry. The report program which can put the method field on the report uses Result.Method as soon as the result is available and Result.Method is known. If not, AssessmentMethod.Method is used.

This allows storing the history of the method field in the result table itself, thus making this field reproducible on the report, even when the assessment method has been changed.

The function AssessmentMethod.SetResultMethod() allows to explicitly fill this Result.Method field with either the value from the assessment method record, or a specific value. This may be useful for storing the method name in old result records.

String MethodRowLabel

Columns > Text reports > Headers > Value

This field (displayed as a small text editor) allows to determine what will be printed on top of the result columns. It can contain a fixed multilingual text, optionally using a number of keywords to indicate that database information must be printed.

You can add as many header lines as you like by typing them in, each item per line.

Example:

Value header = &Date5
&Time

The possible keywords consist of:

&DateTime: format DDMMYY HH:MM

&Date5, &Date6, &Date8, &Date10: shows the date part of the object time associated with this report column. The difference lies in the format (without slashes, with century, only day and month,...)

&Time: shows the time part associated with this order column in format HH:MM

&OrderInternalId: the order internal id of the order associated with this column. Note that in case of a report on which results are grouped in columns by object time (as opposed to order), this may yield unexpected results.

&OrderExternalId: the order external id

&OrderShortid: the order short id

&Issuer: the internal id of the issuer

&IssuerName: the full name of the issuer

&IssuerCode: returns the Code field of the HCProvider issuer

&IssuerHCCode: returns the Code field of the identification record connecting the issuer correspondent with the HCCode provider correspondent as defined in the Specific Site (or Glims) Setup screen.

&Specimen (only useful in case of columns grouped by specimen)

&Material (only useful in case of columns grouped by specimen)

&MaterialShortName

&MaterialDescription

This field can also contain real dynamic text embedded with MISPL.

When such dynamic text is encountered, it is expanded based on a temporary Result record where Result.Object, Result.ObjectTime, Result.Order and Result.Specimen have been copied from the current column.

Example:

{=
"Date" + "~n" +
DateTimeToString(.Order.LowestObjectTime, "%d-%m-%y") + "~n" +
"Reference" + "~n" +
RPAD(.Order.InternalId, 10, " ") + "~n" +
"Issuer" + "~n" +
.Order.Issuer.Name;
}

Could result in a header like:

Date
12-01-00
Reference
01-0032344
Issuer
Josephson
