Discontinue 

The regular scheme a bloodproduct is subject to is the transfusionscheme
from INITIAL over RESERVED, CHECKED, CHECKEDOUT and ADMINISTERED.
However, during the transfusionscheme a bloodbag can become useless
because of damage, ...
At that moment, the bloodbag diverges from the regular transfusionscheme
and can no longer be used in a regular transfusion. To be able to
distinguish that bloodbag it will attain the state DISCONTINUED.
If no reservation was made for this bloodbag then it can be promoted safely. If however a reservation was made for this bloodbag, the bloodbag can still promote upto discontinued, but the selectionprocedure must be repeated for that person. The bloodselection that refers that bloodbag is DISCONTINUED and a new bloodselection is created.
Obviously, bloodbags which attain state ADMINISTERED, DISCONTINUED
or EXPIRED, cannot be promoted.

MultiRecord Void BloodBagUpToDiscontinued ()

AntigenByNumber 

The MISPL function BloodBag.AntigenByNumber() can be used to navigate to the blood bag's n-th antigen record, where 'n' is the parameter. Sort order is by the sequence number defined in the antigen table.

BloodBagAntigen AntigenByNumber (PositiveInteger Number)

PositiveInteger Number
n going from 1,... till the number of available records. If n is larger, the function returns '?'.
If n is unknown, 1 will be assumed.

Attribute 

Logical Attribute (Mnemonic AttributeMnemonic)

Mnemonic AttributeMnemonic
Here you should specify the mnemonic of the BloodAttribute you are interested in.
Alternatively you can specify a number in the form e.g. "#2" to get the second attribute. This can be used to loop through all attributes of a blood bag.

AttributeOverview 

This returns a summary of all blood bag attributes attached to the blood bag.
It is possible to restrict to only 'known properties', only transfusion reactions, etc.
You can specify the layout by means of a text module that is evaluated for each BloodBagAttribute. This text module should have base table 'BloodBagAttribute'.
The sequence in which the attributes are shown is:
1. blood attribute usage
2. blood attribute groupname
3. blood attribute seqno
4. blood attribute mnemonic.

String AttributeOverview (BloodAttributeUsage RestrictTo, Mnemonic TextMnemonic)

BloodAttributeUsage RestrictTo
One of: Known property, Transfusion reaction, Diagnosis, Parameter
Mnemonic TextMnemonic
Specify the mnemonic of the text module that is to be evaluated for each BloodBagAttribute.
This text module should have base table 'BloodBagAttribute'.
If unspecified, a standard text will be shown (Mnemonic+Presence+ExtraInfo+newline).
