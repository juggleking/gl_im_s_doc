This table displays the QC populations of the QC Lot. A population is a set of quality control results and accumulates its statistics. Each population is associated with a QC Lot and an assessment method.

Pivot

The Pivot field has been introduced for the German QC norms. Its main purpose it to make sure that the allowed deviation of a result for an accuracy or a combined control does not fall below a minimum value, even when the target value of the control is very small (normally, the allowed deviation is expressed as a percentage of the target value).

When the target value of a population is lower than the pivot of the population, then the allowed single result deviations from the target value, the allowed imprecision (CV max check) and the allowed inaccuracy (R max check) of a selected period are interpreted with absolute limits instead of percentages. Fundamentally, for CV-max, R-max and MaxDev, instead of interpreting these as a percentage, they are interpreted as absolute values, derived from the percentage value by calculating the value of the limits on the pivot level instead of the target value level.

Example:

Unit = nmol/l
TargetValue = 40
AccuracyLimit = 20%
CVMax = 5%
RMax = 10%
Pivot = 50

Then

the allowed limits for the single results are 30 - 50 nmol/l (and not 32 - 48!),
the maximum allowed SD for a given period is 2.5 nmol/l (and not 2.0 !),
the maximum allowed inaccuray Mean - TargetValue for a given period is 5 nm (and not 4 !)

In this case, the QCPopulation.Print() and QCAudit.Print() functions will shown the absolute numbers on the printout and not the relative numbers.

Target deviation

The Target mean and the Target deviation are used to quantify the method's precision. They are expressed in the unit of the property. The values of the target mean and target deviation can typically be determined by means of an evaluation period.

If the standard deviation of the QC results matches the target deviation, the method is precise: it delivers reproducible values.

PositiveInteger Period
This month / Previous month: allows selecting all QC populations of this month or the previous month.

Until Yesterday / Until Today / Until Previous Month: the main purpose of these three options is to allow results which were not audited in the 2 previous months to be automatically included in this month's audit (remember that a 'precision' audit requires at least 15 results before the audit is created).

When using one of these three options and no start date is specified, then the latter is calculated as the highest end date of the stored audits + 1, with a maximum of 2 months ago. The actual start date stored in the QCAudit record will be the date of the first actual result in the audit, with a minimum of the start of the previous month for 'Until Previous Month' and the start of the current month for the others.

In all cases, when the end date is not specified, it will be derived from the period setting. When using the 'Until Today' or 'This Month' option, the actual end date stored in the QCAudit record will be the date of the last result in the audit, with a minimum of 'Yesterday'.

MISPL functions

Attribute 

Returns qc population attributes that are impossible or very cumbersome to obtain by means of standard MISPL constructs.

String Attribute (String AttributeName)

String AttributeName
Supported values:

"Severity2S": returns the contents of the QCPopulation.2SSeverity field
"Severity3S": returns the contents of the QCPopulation.3SSeverity field
"Type": returns the QC population type, based on the configuration field values of the QC population
"Mean": returns the calculated mean value of all numeric results, Mean = SUM(Value) / Count
"StandardDeviation": returns the calculated standard deviation (SD) of all the numeric results, SD = SQRT(SUM(Value - Mean)**2) / (Count - 1))
"VC": returns the calculated coefficient of variation (CV) of all the numeric results, CV = 100 * SD / Mean
"ProcessCapability": returns the calculated process capability value, Cpk = TEa / 3 * CV (TEa = Total allowable error)
