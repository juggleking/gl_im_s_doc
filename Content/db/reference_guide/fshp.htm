Financial shipments are collections of financial transactions: invoice summaries, payments and/or gains. They are used for transfer of financial data between Glims and other applications, represented as the shipment's 'destination'. A financial shipment can contain any number of financial transactions. A financial transaction can be shipped to several correspondents. Examples of such shipments are:
+ transfers to accounting systems typically inlcude invoice summaries, payments as well as gains.
+ billing transfers to official gouvernment organisations (KV, RIZIV/INAMI) typically only contain invoice summaries.
+ a shipment with payments received from a bank.

Fields

Addendum

The shipment's addendum: this is always empty for new shipments. This field is only important if you create additions of existing shipments, using the 'MakeNewShipmentVersion' method with option 'collect changes'. In that case a new shipment with the same version number but the next higher addendum is created. An addendum is always a letter: 'A', 'B', ... The 'MakeNewShipmentVersion' method also creates addenda when 'full crediting' if a seperate shipment for credit notes is requested.

Make new shipment version or addendum 

This method creates a new version of an existing financial shipment by calling the 'Collect Changes' method for all invoice summaries on the shipment. All changes found are then collected on a new financial shipment.
If a new version is asked of all invoices, then a complete new shipment with the next higher version number is made. If only changed invoices are collected then an addendum to the shipment is made.
If a shipment already has more than one version, this method can only be run again on the highest version.
A typical use would be when an official organization refuses a shipment that was sent. In that case corrections to some of the orders/invoices must be performed, after which one wants to 'send' the corrected shipment. This corrected shipment is actually a new version.
See also the 'Collect Changes' method on the invoice summary tabel.

This parameter defines the output protocol. The following protocols are available:
- 'HL7' creates HL7 DFT P03 messages (HL7= Health level 7; event type P03= Post detail financial transactions).
- 'BelgianFiles': creates a file with billing information conform the Belgian RIZIV/INAMI instructions. It must be sent monthly (in CD format) to each Belgian main fund ('landsbonden').
- 'Infohos': format used by the 'INFOHOS' hospital information system (Belgium).
- 'KVDT': creates a KV-DT file containing the ADT billing information for all the invoices on this shipment. Used in Germany to send billing information to the KV.
- 'DTAUS0' and 'DTAUS1' are used in Germany and contain the payment instructions for the bank for customers that have a bank domicile.
- 'AustrianFiles': format used in Austria to send the billing to the official funds.
- 'HPRIM FAC': creates HPRIM 'Facturation' messages (France).
- 'SMS Fac': creates SMS 'Cotation des actes' messages (France).
- 'C Page': creates C-Page 'Facturation des actes' messages (Version V01.M) (France).
-' Ascii': format used by the 'ASCII' accounting program.
- 'Vds': format used by the 'VDS' accounting program.
- 'UDB': dedicated interface in MIPS propriatary format ('UmsatzDatenBank')
- 'Vektis 5': creates a file conform the Dutch Vektis 5 standard, supporting message ZH38.

The standard financial shipment query.

Standard (Firm Firm, ShipmentDirection Direction, Correspondent Destination, String FromInternalId, String ToInternalId, String ExternalDateSince, String ExternalDateUntil, BillingDocumentStatus MinimalStatus, BillingDocumentStatus MaximalStatus, SendStatus MinimalSendStatus, SendStatus MaximalSendStatus, PositiveInteger MinPrintOutCount, PositiveInteger MaxPrintOutCount, DateTime FromPrintTime, DateTime ToPrintTime, BillingDocumentTemplate PrintedWithTemplate, gp_SiteFunction FilterFunction)

Firm Firm
If present, selects only financial shipments of this firm.
ShipmentDirection Direction
One of: Inbound, Outbound, Any
Correspondent Destination
If present, only selects financial shipments with this destination.
String FromInternalId
If specified, selects only financial shipments with an internal id higher or equal to the specified one.
String ToInternalId
If specified, selects only financial shipments with an internal id lower or equal to the specified one.
String ExternalDateSince
If present only selects financial shipments with an external date later or equal to this date.
String ExternalDateUntil
If present only selects financial shipments with an external date before or equal to this date.
BillingDocumentStatus MinimalStatus
One of: Initial, Numbered, Fixed
Selects only financial shipments with a status higher or equal to this one.
BillingDocumentStatus MaximalStatus
One of: Initial, Numbered, Fixed
Selects only financial shipments with a status lower or equal to this one.
SendStatus MinimalSendStatus
One of: Sending, Transfer Failed, Sent, Rejected, Accepted, Processed
SendStatus MaximalSendStatus
One of: Sending, Transfer Failed, Sent, Rejected, Accepted, Processed
PositiveInteger MinPrintOutCount
If specified, selects only financial shipments with a printout count above or equal to the specified value.

PositiveInteger MaxPrintOutCount
If specified, selects only financial shipments with a printout count below or equal to the specified value.
DateTime FromPrintTime
If specified, selects only financial shipments that have been printed (with the 'mark' setting on) after this time. The BillingDocumentOutPrint table is consulted to fulfill this criterium.
DateTime ToPrintTime
If specified, selects only financial shipments that have been printed (with the 'mark' setting on) before this time. The BillingDocumentOutPrint table is consulted to fulfill this criterium.
BillingDocumentTemplate PrintedWithTemplate
If specified, selects only financial shipments that have been printed (with the 'mark' setting on) once with this billing document template. The BillingDocumentOutPrint table is consulted to fulfill this criterium.
gp_SiteFunction FilterFunction
When present, this financial shipment based, logical valued site function must return TRUE for a financial shipment to be selected.
