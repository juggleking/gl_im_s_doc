Delta norm dnrm glims yes yes Normal

This table stores norms for delta checks. Delta checking checks the evolution of assessments on a particular object. It subtracts the value of a reference assessment (typically: the previous assessment) from the new assessment, and compares the (absolute or relative) difference with specified norms. The time lapse between reference and new assessment may be significant.

Discriminator

If specified, this delta norm will only be applied when the property was requested with the specified discriminator. When Glims evaluates which delta norms are applicable, the following logic is applied: as soon as 1 delta norm for the result's discriminator is found, only delta norms matching the result discriminator will be considered. Other delta norms will no longer be taken into account.

Eligibility

The Eligible spinner references a Result-based, Logical-valued MISPL expression. It is evaluated as a final criterion for each delta norm fitting the result context. If it evaluates to false, the delta norm is not eligible. If it evaluates to true or if the reference is cleared, the delta norm is eligible. When executing the referenced MISPL function, the updated result record is not yet written to the database. This implies that start table methods like .NumericValue() will NOT work! (Since the result method dispatcher will re-read the outdated record from the database). The only Result fields that can be relied upon in these MISPL functions are .RawValue and .AvailabilityUser

Lapse denominator

The Lapse denominator is a fractional day count. If set, the delta norm's limits are multiplied with the number of times this denominator period has elapsed since the previous result.

Example in case of a linear delta norm: low limit= -10, high limit= 20 and lapse denominator= 14. After 14 days, the high limit=20%, after 28 days, the high limit=40%, etc. Note: after 7 days, the high limit=10%. The same goes for the low limit. The lapse denominator is valid for both relative (%) and absolute (see pivot) value shifts.

Pivot

It is possible that the allowed shift value is calculated by applying the specified limits to the pivot value. This implies that a fixed limit range will be used instead of a relative one. When will this fixed range be applied:

- For linear delta norms: if the previous result value is lower than the pivot value
- For exponential delta norms: if the time between two assessments is shorter than the pivot value (which in this case is a time indication in fractional days). This prevents the allowed difference from approaching zero when the time difference between the old and the new value becomes too small.

Theoretical example of a linear delta norm: Low limit=-20, high limit=10, pivot=15, previous value=prev

Previous result equal to or higher than 15 -> relative shift value -> acceptance range : prev -20% - prev +10%
Previous result lower than 15 -> fixed shift value -> acceptance range: prev -3 - prev +1.5

The Pivot is a threshold value, expressed in the unit of the property. Below this value, the absolute shift extremes at the pivot point apply.

Type

The delta norm can be linear of exponential.

1. Linear
In this case, we assume that the new result is allowed to deviate from the previous result by the same number PER <lapse denominator>. E.g., when the allowed deviation is 10% per 2 days (lapse denominator), then the allowed deviation after 4 days is 20% of the original value.

2. Exponential:
Here, we assume that the result can deviate by the same percentage per <lapse denominator> and that this is a cumulative process. E.g., when the allowed deviation is -10% per 2 days (lapse denominator) and the original value is 100, then the allowed value value after is 90 after 2 days, 81 after 4 days, 72.9 after 6 days,...
The formula used is: X = X0 * EXP(1 + <p>/100, <time> / <lapse denominator>), where <p> is expressed in percent and typically is negative.

Mandatory DeltaNormType Type

One of: Linear, Exponential
