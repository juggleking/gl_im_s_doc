Change object 

This function allows:

1. updating the identity of the order's object in the appropriate object editor.
2. replacing the order's object by another object. The order entry program will be invoked, allowing the user to specify the correct object.
3. replacing and merging the order's object. This option will only be available if the object is of type 'Person' and the user owns the Person.Merge() privilege. The function allows replacing a temporary patient by its final version.

The previous option (Replace by another object) has the disadvantage that the temporary patient is only disconnected, and remains in the database where it serves of no more purpose and only pollutes the database.

With this third option, the person search screen appears, from which you can select the right patient. On selection, the original (temporary) patient is merged with the selected one, and hence disappears from the database. So, you end up with the order linked to the right patient and the temporary patient is gone.
