Default reports are used at order entry time, to determine which report records must be created, based on the preferences of the order issuer. Creating a report record effectively schedules the generation of a document delivering results. Default reports contain default values for these actual report records.

Previous results > Previous results

The Previous results field determines which previous results are printed on the report. There are four options:

None: no previous results are printed.

All: all results of previous results within the current Property Classification are printed.

ByChapter: a previous result is only printed if the current order has a result for a property from the same property classification chapter (i.e.: only print previous hematology results if the current order also contains hematology)

Settings > Report discontinued results

This field allows indicating whether or not discontinued results are to be reported.

Remarks:

- For medicard, medidoc, EDI1.0, medigest, Imed, HealthOne and MEDVRI1.0 reports, the value of this field has been fixed to 'NO', i.e. discontinued results will never be put on the report (as these reports cannot transfer result status information).

- Discontinued results are considered as being new results, so that a report is triggered for generation even if the 'New results' field in the report generation screen is set to "New results required". However, if the latter is set to "Only new results" (and the "Report discontinued results" option is not set) and the only new result is a discontinued result, no report will be sent.

Main > Scope

The scope indicates WHAT should appear on the report. It imposes object time criteria on the results to be included in the report.

- Order reports: All results of a particular order will be displayed on the report. This is the standard report type.

- Exclusive reports are like special order reports, ruling out the creation of other reports. They offer the possibility to create special reports for specific orders suppressing the normal reports.

- Encounter reports: All results obtained during the encounter of a patient will be reported. These results will be divided into different segments (columns). Placing all the reports in a row, will provide a complete overview.

- Year reports: all results obtained for a patient during an entire year will be reported.

- Object report: extended year reports. All results for a patient will appear on the report, without any time-related criteria.

- Specimen report: all results for one particular specimen will be put on one report. E.g. An order has three different specimen => each specimen will be put on a separate report. Results which are not linked to a specimen (e.g. weight of the patient) will be included on each report.

- Study: used for study reports.

Mandatory ReportScope Scope

One of: Order, Day, Encounter, Period, Object, Exclusive, Specimen, Year, Study, Ward, Medical record, Obsolete

Main > Target-specific

The Target-specific field indicates if a separate report should be created for each target (e.g. issuer). A default report with TargetSpecific = NO is not allowed for default reports with scope Order, Specimen or Study.

For instance, for issuer-targeted object reports:

- When not set, there will be only one object report per patient and per report template.
- When set, there will be a separate report record per patient per template and per issuer
Also, when this field is set, an issuer-targeted report will only contain pages which contain at least one result requested by this issuer

Settings > Trigger status

If a result reaches this status, the report becomes eligible for further inspection by setting its NeedsChecking field.

'Status reports': reports with a trigger status below 'Available' are considered to be 'Status' reports. This implies that each status change makes the report eligible for generation.

'Value reports': reports with a trigger status >= 'Available' are considered to be 'Value' reports. This implies that only actual result value changes or external comment changes makes the report eligible for generation. Result status changes (except the moment that the trigger status is exceeded) will not be considered when checking if the report has to be generated. So a value report can only be triggered:

- The moment that the trigger status is exceeded (e.g. first report generation)

- The result value or external comment has changed (and naturally the status has to exceed the trigger status)

There is one exception: a 'value' report will also be triggered upon each result status increase when it is a 'communication' or 'electronic' report (i.e. Report.Medium = Communication).

This differentiation between 'Status' and 'Value' reports also has an influence on the interpretation of the 'New results' option in the report generation screen.

Mandatory ResultStatus TriggerStatus

One of: Initial, Expected, Available, Confirmed, Validated, Discontinued

Main > Usage

The Usage field determines for what type of purpose this default report is used, and when it will be eligible by the report scheduler.

'Standard': this is the 'normal' case. It indicates a report that is automatically scheduled during order entry, as an original. Also, when an order for a given issuer is entered, his private Default Report configuration will only be used when at least one active Default Report with Usage = 'Standard' is found for this issuer (see below, Linking default reports to an issuer).

'Standard copy': It indicates a report that is automatically scheduled during order entry, but should be considered as a copy rather than an original. Also, after scheduling of the site's organization default reports (thus skipping the issuer's default reports), any of the issuer's default reports where Usage = Standard Copy are added to the already scheduled reports.

When 'Screen', it is only eligible when a screen report is generated. The Microsoft Word report generator can also be used for screen report consultation. The viewer will be the same file viewer as used in the preview of such documents (i.e., a Microsoft Web Browser viewer, or the default application installed for .doc documents).

'Manual copy': this indicates a default report which is only taken into account when reports are being scheduled for Order.ReportCopyReceiver (or being added manually or via MISPL). Remember that you can specify such a report copy receiver in the order entry screen. In this case, after scheduling of the original reports, the order is rescheduled as if the copy receiver was the issuer. In this case, the default reports marked as 'Manual Copy' will also be eligible for scheduling. Moreover, all reports scheduled during such an operation will be marked with 'Usage = Manual Copy', regardless of the original default report's Usage setting.

When 'Manual', it will not be used by the report scheduler at all. It indicates a report that is typically added manually or via MISPL.

Mandatory ReportUsage Usage

One of: Screen, Standard, Standard copy, Manual copy, Manual
