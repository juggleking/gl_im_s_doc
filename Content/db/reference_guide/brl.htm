This table stores the contents or lines of a billing rule. Billing rule lines specify the conversion between its billing code inputs and billing code outputs.

Line expansion

This calculated field displays the content of the billing rule line in a coded way, according the following syntax:
<LineMode>[<InputList>]["="[<OutputList1>]"|"[<OutputList2>]]
<LineMode>: '*' | ' ' | '+'
<InputList>: Comma seperated list of billing codes (billing rule line inputs)
<OuputList1>: Comma seperated list of billing codes (billing rule line outputs)
<OuputList2>: Comma seperated list of billing codes (billing rule line outputs)
Output list 1 and 2 are the billing rule line outputs that are applicable respectively before and after the maximum translations are exceeded.
'*', ' ', and '+' represent respectively mandatory, singular and not singular billing rule lines.

Calculated String LineExpansion

Mandatory

If a line is set as mandatory, then the presence of the inputs of this line is a condition for the complete billing rule to be applicable. If the line is not set as mandatory, then the line substitution will be executed if its inputs are present.

Mandatory Logical Mandatory

Seq no

This field indicates the order in which the billing rule lines are being evaluated and substituted.

Mandatory PositiveInteger SeqNo

Singular

A billing rule line which is marked as singular represents a substitution which can only be executed once within the evaluation of the billing rule. If the rule is not singular, then each billing code instance that matches the input criteria will be substituted.

Mandatory Logical Singular

Substitute

The billing rule line inputs can be specified in order to be substituted or as a billing rule condition without the necessity to be converted (e.g. to grant a discount). Only in case a substitution is required the field 'substitute' should be set.

Mandatory Logical Substitute

Update request charges

This field specifies if the RequestCharges of the inputs must be moved to the outputs if this billing rule line is activated. Three cases:
- For lines without inputs, request charges created for all inputs already translated in this billing rule.
- For lines without outputs, request charges created for the outputs of the last billing rule line with outputs.
- For lines with inputs and outputs, the request charges moved from the inputs to the outputs

Mandatory Logical UpdateRequestCharges
