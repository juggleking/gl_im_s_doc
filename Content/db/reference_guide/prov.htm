This table stores the time dependent availability of executors (health care providers) in a lab.

Fields

Accredited

This field indicates whether the executor is accredited during the provision time.

Logical Accredited

Alternate accredited

This field indicates wether the alternate executor is accredited during the provision time.

Logical AlternateAccredited

Alternate executor

This field provides access to the provision's alternate executor : only set as billing item executor when mentionned and when the order issuer equals the default (primary mentionned) providing executor.

Invariant HCProvider AlternateExecutor

Creation time

Automatic DateTime CreationTime

Creation user

Automatic sc_User CreationUser

Department

This field indicates the department within the lab where executor is available.

Invariant Department Department

Valid until

This field indicates the end of the validity period. The provision is valid up to and including the specified end date and time. No end specification is considered as a not ending validity period.

DateTime EndTime

Exclusive

When exclusive, and the related provision is utilized (and has ExlusiveAllowed=YES), then the executor of such provision will be set as executor of all internally executed tests

Mandatory Logical Exclusive

Executing class

This field provides access to the provision's executing class. If specified, the provision is restricted to that executing class, otherwise the provision can be applied for all executing classes.

Invariant ExecutingClass ExecutingClass

Executor

This field provides access to the provision's executor.

Mandatory Invariant HCProvider Executor

Id

Primary key. Automatically generated identifier whose value uniquely identifies each record.

Mandatory Unique Automatic Provision Id

Lab

This field indicates the lab where executor is available.

Mandatory Invariant Lab Lab

Last update time

Automatic DateTime LastUpdateTime

Last update user

Automatic sc_User LastUpdateUser

Purpose

This field specifies for which purpose the provision record can be used:
Validator verification: The provision should be used to verify whether the validator can be considered as a valid executor.
Executor determination: The provision can be used to determine the executor automatically independent from the validator.

Mandatory Automatic ProvisionPurpose Purpose

One of: Validator verification, Executor determination, Executor default

Replaceable

When set and an exclusive executor exists, then the executor of this provision will be reset by the exclusive executor (of the exclusive provision).
However, in case of anatomo-pathology or externally executed tests, the executor will never be replaced (this is hard-coded).

Mandatory Logical Replaceable

Valid from

This field indicates the validity start date and time of the provision.

Mandatory DateTime StartTime

Version

Record version number. Automatically incremented each time the contents of a record changes.

Automatic PositiveInteger Version
