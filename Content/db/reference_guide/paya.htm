Payment agreement paya glims yes yes Normal

Each payment agreement represents an agreement (policy name) which can be used to tariff an order (set). If the agreement is only applicable for a specific correspondent, then that correspondent is specified in the party field. If the party field is left empty, then the agreement is generally applicable for all kinds of orders (sets).

Checked

As long as this field is not set, the agreement cannot be used by the tariffication program. Setting this field to 'true' triggers the evaluation of the associated policy's payment agreement check function. If the evaluation of that function returns the empty string, then the payment agreement can be set as checked. If it returns a non empty string, which is considered as an error message, the agreement can only be saved as not checked. Note that in this way, incomplete payment agreements can be introduced and associated with an order(set) without creating the possibility to tariff the order(set).

Party

This field provides access to payment agreement's party. It is the correspondent, the (payment) agreement was made with. If this field is provided, the payment agreement is specific, otherwise it's a general payment agreement. General payment agreements can be applied for all kinds of order sets. Specific payment agreements can only be applied, if the order set reimbursement party is the payment agreement party.
