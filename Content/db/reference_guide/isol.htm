Epidemiology > Infection update mode

Denotes how the automatic infection detection system should treat the isolation:

- Automatic: the system will try to figure out whether it belongs to an isolation series and whether it is a double or original
- explicit double: the user explicitly declared it as being identical to another isolation
- explicit original: the user explicitly declared it as the first occurrence of an isolation series
- no infection: the user explicitly declared it as not being an isolation series at all (not important for statistics)

Mandatory Automatic InfectionUpdateMode InfectionUpdateMode

One of: Automatic determination, Explicit double, Explicit original, No infection

String HandleDoubles
Doubles of isolations are excluded/included/selected. Possible values are:

- Don't exclude doubles: no filtering is performed.
- Exclude doubles:

1. within patient: only new isolations (unprecedented - without any duplicates); -> this removes all isolations that are 'double of' another isolation.
2. within material: only new isolations for the material in which they were found.
3. within chosen selection criteria: one single result per isolation series (no identicals within selection). Of all isolations of the same isolation series, only the first isolation that fulfills the selection criteria is selected.

- Only doubles:

1. within patient: opposite of 'Exclude doubles within patient'.
2. within material: opposite of 'Exclude doubles within material'.
3. within chosen selection criteria: opposite of 'Exclude doubles within chosen selection criteria'.
