Is fallback

If this is TRUE, then this transition should be taken if and only if all other transitions that have a restriction (starting from the same from-activity) evaluate to FALSE.
This makes it easier to construct an exclusive split, where a choice must be made e.g. between 2 mutually exclusive paths (transitions).
If transition1 has a restriction, then –by setting ‘IsFallBack’ to YES on transition2- it is no more necessary to make a reverse restriction (that is the opposite of the restriction of transition1).
Consider it like the ELSE clause in an IF statement or an OTHERWISE clause in a CASE statement.
Note:
- it can only be YES if the transition's restriction field is empty
- only 1 of the transitions leaving an activity can be marked as 'FallBack'
