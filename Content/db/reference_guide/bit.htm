Billing item bit glims yes yes Normal

Each billing item record represents the usage/instanciation of a tariff record. It contains information (e.g. executor, cost location) which is determined during the tariffication.

Fields

Accounting class

This field provides access to the billing item's accounting class; filled-in during tarification when accounting classes are supported (see SpecificSite level). If supported, the tariffication program determines the accounting class
according to the following steps and in the order listed:
- The accounting class assigned to the requestable (combined with order department or executing department and yes/no urgent if set in parameterlist).

- The accounting class assigned to the costlocation (combined with order department or executing department and yes/no urgent if set in parameterlist).

- The accounting class corresponding with the combination of order department and executing department.

AccountingClass AccountingClass

Accounting class code

The billing item's accounting class code, filled-in during tarification when accounting classes are supported.
How determined see field AccountingClass.
This was the original field used in previous Glims versions to store the accounting class code (as string) : not removed but still filled-in and kept (next to field AccountingClass) for archive and statistical reasons.

Billing category

This field provides access to the billing item's billing category. In case of partial order billing, the tariffication program determines the billing category by retreiving it in the requestable (record) which led to the tariffed request.

Comment

Confirmation status

This field is an optional enumerated and checked by the KVDT-program (Germany):
- acknowledged: The billing item is marked as explicitly acknowledged.
- not acknowledged: The billing item is marked as to be acknowledged.
- not required: The billing item should not be acknowledged.
This field can be set manually or by a specific interface.

BillingItemConfirmationStatus ConfirmationStatus

One of: Acknowledged, Not acknowledged, Not required

Cost location

This field provides access to the billing item's cost location. The tariffication program determines the costlocation according to the following steps and in the order listed:
- The cost location where a corresponding request is realized.
- The billing code's cost location.
- The general default cost location (specific site table)

Tariffing lab

This field provides access to billing item's tariffing lab. A billing item represents a billing code instance for the tariffication of a specific lab, the tariffing lab. Example:
Internal lab L1 receives an order from a physician. One of the requested tests (T1) is forwarded to another internal lab (L2). During the tariffication for L1 a billing item BI1 is created. BI1 is charged to the patient. During the tariffication for L2, a billing item (BI2) is created for the same test T1. BI2 is charged to the L1.
BI1 and BI2 both represent the tariffication of T1. The tariffing lab of BI1 and BI2 are respectively L1 and L2.

Mandatory Invariant Lab TariffingLab
