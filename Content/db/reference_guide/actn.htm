The actual realizations of procedures are called actions. For one procedure, there are usually many actions. Note that, out of several procedure possibilities, a single one will eventually be instantiated as an action. The choice may depend on station availability, or more complex criteria. Eventually actions are grouped into work lists.

Queried by station

Whenever a single specimen is queried explicitly by an analyzer station, the actions for this specimen which are sent to the analyzer are marked via their QueriedByStation member.

The actions which are marked as such are not longer sent in a batch-download to other stations in the same download group (see Station.DownloadGroup).
This prevents these other stations not to be overloaded with work orders which are ought to be done on a station that explicitly queried the specimen.

Remark that QueriedByStation will not be set by stations of type concentrator or distributor.

Mandatory Automatic Logical QueriedByStation

Station download status

Whenever an action is downloaded to an analyzer, this is stored in the action "Download status" field. The download status marks to which station(s) or download group(s) the action has been downloaded.

You can use the Action contextual menu function "View download status" to get a list of stations (or download groups) for which the action has been downloaded.

Automatic String StationDownloadStatus

Status

Frozen:
Internal state, for optimization purposes.

Connecting:
The action has been created; in practice a property/material was requested and scheduling is in progress.

Inputting:
Result and specimen inputs are 'Expected' or there are no inputs. May return to 'Connecting' if inputs are discontinued.

Pending:
Normally entered if all inputs are 'Available' or if there are no inputs.
Serves in the first place to impede the work until the signal is given implicitly (by generation of a work list) or explicitly (by entering the Action start time) to start the job (becomes Outputting). If correctly configured, 'Pending' Actions do not appear on the worklist, if not wanted.

Outputting:
Entered when:
1) .Procedure.AutoStart is TRUE/YES. In such case the 'Pending' status is simply skipped;
2) manually, via 'Start' method on Action.
Consequences:
1) result and specimen outputs automatically receive status 'Expected';
2) no additional outputs can be assigned or added to the Action (train has left). This implies that an ancillary examination will generate a new Action, and cause an extra line on the worklist;
3) standard results are entered and formulas calculated.

Partial:
At least one result or specimen output is available.

Realized:
All result and specimen outputs are available, but some results are not yet confirmed.

Ready:
All results are confirmed, validated or discontinued and all materials are available or discontinued; or there are no outputs.

Mandatory Automatic ActionStatus Status

One of: Frozen, Connecting, Inputting, Pending, Outputting, Partial, Realized, Ready

MISPL functions

Attribute 

String Attribute (String AttributeName)

String AttributeName
Supported values :

"InputSpecimen"
The internal id of the first specimen that is input to the action.

"InputSpecimenList"
Comma-separated list of specimen internal id's that are input to the action. The list is sorted by ascending internal id.

"SamplingTime"
The sampling time of the first specimen that is input to the action in "DD/MM HH:MM" format.

"Issuer"
Searches a unique order that refers the action and returns the internal id of its issuer. If ambiguous (i.e. more than one order is found), unknown is returned.

"Agent" (or "Contact")
Searches a unique order that refers the action and returns the internal id of its agent. If ambiguous (i.e. more than one order is found), unknown is returned.
Note that "Contact" is only there for backward compatability and may disappear in a future release.

"SamplingSequence"
Applies only to objects of type 'Person'.
If a valid stay can be found for the action's object at the moment of execution of the .Attribute() function, the sampling sequence is returned in the format "SSS-RRR" (where SSS = Stay.Ward.VisitSequence and RRR = Stay.Room).

"PropertyList"
Comma separated property mnemonic list (Property.Mnemonic) of all property outputs.
Format: [Mnemonic][,Mnemonic]...

"PropertyCodeList"
Comma separated property code list (Property.Code) of all properties derived from the specimen. Properties without a code are excluded.
Format: [Code][,Code]...

"CompletedPropertyCodeList"

Comma separated property code list (Property.Code) of all properties derived from the specimen.  Properties without a code are included and the mnemonic is used instead.
Format: [Code][,Code]...

"Position"
The sequence number of an action on a work list can now be attained via the Action.Attribute("Position") MISPL function. This will return the position of the current action within the selected group (i.e. its sequence no on the worklist).

"NormalPosition"
This function will yield the sequence number of the current action WITHOUT taking the QC samples into account (i.e. the first non-qc sample gets Normal Position 1, while all QC actions get Normal Position 0).

"ObjectAttributeList[:Delimiter]"
Yields a comma-separated list of the mnemonics of the object attributes which are active on <Action.ObjectTime>, together with an indication of the length of the active period.
If the optional <Delimiter> modifier is specified, it will be used in stead of the standard comma.

"ObjectAttributeNameList[:Delimiter]"
Same as "ObjectAttributelist", but the descriptions of the object attributes are shown in stead of the mnemonics.

"ObjectAttributeFlags"
Same as "ObjectAttributeList", but only a list of object attribute flags is returned (without time indication). No modifier is taken into account.
