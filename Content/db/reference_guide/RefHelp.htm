Fields

The fields of the selected table are displayed with their translated side label (as it appears in editors) and are sorted by un-prefixed field name.

First, the documentation text of the field is displayed. The frame below presents more detailed, technical characteristics of the field and contains:

PropertyList DataTypeName

PropertyList can be composed of

"Mandatory"
The field cannot contain an unknown value. Hence, it will be impossible to create a new record without specifying this field.
"Unique"
A particular value can occur only once in the the complete table. For example, the language table cannot contain two records with the same language name. However, since many users will speak the same language, the user's language will NOT be unique.
"Invariant"
Once the record is created or the field gets a value it can no longer be changed.
"Automatic"
The field is completely under program control and hence cannot be changed manually. It may be visual (read-only) or hidden.
"Multilingual"
The field is multi-lingual. Therefore, it doesn't physically exist. Instead, a child table contains a record for each translation.
"Calculated"
The field is calculated on the basis of other fields of the same or related records. It doesn't physically belong to the record, but is calculated at the moment the record is to be displayed.
DataType
The type of data the field accommodates.
Name
Unprefixed field name

And finally, if the field is of data type enumerated, the list of possible values will be specified below the frame in green.  The correct EnumeratorClassName is indicated by the 'link'.  Click the link to get more info on the EnumeratorClassName.

Menu functions

Menu functions are intended for interactive use only.  They are displayed by their translated menu option, as it appears in the GLIMS context menu.

First, the documentation text of the menu function is displayed. The frame below presents more detailed, technical characteristics of the menu function and contains:

[MultiRecord] DataTypeName ()
 
"MultiRecord "
Indicates if the function can process several (selected) records simultaneously. If not (i.e. in the case of a single record command), the affected record will always be the focused one, regardless of the current selection.
DataType
The type of data the function returns. As menu functions are intended for interactive use, most of them have no return value, and hence are of data type 'Void'.
Name
Name of the function.

MISPL functions

MISPL functions are intended to be used by MISPL programs only. They are listed by function name as they should literally be used in MISPL.

First, the documentation text of the MISPL function is displayed. The frame below presents more detailed, technical characteristics of the MISPL function and contains:

DataTypeName(ParameterList)

DataType
The type of data the function returns.
Name
Name of the function.
ParameterList
[DataTypeName][, DataTypeName]...

This is the list of function parameters, along with their data type. As MISPL doesn't support variable argument lists, all parameters should be supplied.

If the MISPL function is of data type enumerated, the possible values are listed below the frame.

Children

Children are all tables referring the current table. This means that the value of the referring field equals a primary key of this table (the .Id field).  The children are displayed by the browser title (e.g. "Users speaking language")

The frame below presents more detailed, technical characteristics of the child-parent relation and contains:

PropertyList SourceTable.SourceField

PropertyList can be composed of

"Shareable" or "Private"
Indicates whether or not the reference can be shared by multiple children. For example, different users may reference the same language record.
"Supportive"
If the last referring record disappears, the referred record no longer has a reason for existence and hence will also be deleted.
"Restrict" or "Cascade" or "Nullify"
Indicates the action be taken when the parent record is deleted:
"Restrict"
The parent cannot be deleted as long as it is referred by at least one child.
"Cascade"
If the parent record is deleted, all children will automatically be deleted as well.
"Nullify"
If the parent record is deleted, the children will become orphans. Their parent reference will be set to the unknown value.
SourceTable
Name of the child table. Is also a hyperlink to the reference guide section of that table.
SourceField
Name of the referring field in the child table. Is also a hyperlink to the referring field documentation.

First, the documentation text of the query is displayed. The frame below presents more detailed, technical characteristics of the query and contains:

Name (ParameterList)

Name
The name of the query.
ParameterList
[DataTypeName][, DataTypeName]...
The list of query parameters, along with their data type.

Indexes

The list of all indexes defined on the table. Each index is listed with its attributes and its segments.

Possible attribute values are:

"Unique"

A unique index is used to prevent and detect duplicates. For a unique index, each combination of its segment values may occur only once.

"Externalization"

The most comprehensible (preferably unique) index after the primary key. Because the primary key is almost always an automatically generated identifier, it carrier no information. Therefore, each record is presented to the user in a human readable form (= externalized) by means of this index. This is also the index that is used by the generic (denormalized) import/export functions to uniquely identify a record.

The segments are the fields used to build the index. Each field is preceded by a "+" or "-" sign, indicating whether the index segment is ascending or descending. The direction of a segment indicates its natural sorting order. When ascending, the lowest value will be the first; when descending it will be the last.

Validations

A list of relational restrictions imposed by the user interface.
