Optional description of the price list.

String Description

Allow discount

If disabled this means that discounts do NOT apply to the tariffs of this price list. This is typically a price list specific for the payer. This does not apply to tariffs retrieved from the 'parent' default price list.

Mandatory Logical Discountable

Extension

This optional field provides access to the price list's extension price list. If no tariff record can be found (for some billing code) in a price list, the search will be continued in the extension price list. The extension price list is usually a more general or base price list. The starting price list can be considered as list of exceptions of the extension price list.

PriceList Extension

Extension multiplier

Each time, the tariffication program retrieves a price via the extension price list, the returned price is multiplied with the extension multiplier. Note that price correction can be avoided by setting this multiplier to '1'.

Mandatory PositiveFractional ExtensionMultiplier

External id

The official name of the price list. This id is used when the price list must be reported to external systems.

Nomenclature code check

This field provides access to the nomenclature code check site function (tariff based, string valued). Each time a nomenclature code is adapted the check function, specified in the referred price list is executed. The return value is stored as the, possibly corrected, nomenclature code. If the return value is ?, an error is produced and the tariff record is not saved.

gp_SiteFunction NomenclatureCodeCheck

Synchronize nomenclature code

This field specifies whether the price list's nomenclature codes should be kept synchronized with their corresponding billing codes:
YES: If a nomenclature code equals its billing code, and the billing code (code) is updated, then the nomenclature code will be set to the new billing code.
NO: No synchronization will not take place.

Mandatory Logical SyncNomenclatureCodes

Version

Record version number. Automatically incremented each time the contents of a record changes.
