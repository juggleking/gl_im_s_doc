Correspondent crsp glims yes yes Normal

GLIMS knows about different types of correspondents. These types are: Departments, Labs, Firms, Funds, Health care providers, Institutions, Organizations, Persons and Wards. Although each type has specific attributes, all types share some common attributes. Each correspondent must belong to one single type.

Report > Maximum interim report versions

Indicates the maximum number of different interim reports the correspondent wishes to receive. As soon as this maximum is reached, the next report will only be sent when it is complete.

This option is not taken into account during a 'reprint' query, i.e. a report query where MinimalLastOutputTime or MaximalLastOutputTime has been specified. Moreover, this interim report limit is only taken into account for reports where Report.Medium is either 'Courier' or 'Fax' (the 'paper' reports). This implies that this limit is ignored for e.g. electronic reports.

Setting this option to "0" indicates that this correspondent does not wish to receive any interim reports.

Select correspondents with a last update date no more than <x> days ago. A value of 0 indicates you want to select correspondents updated today.

When actually updating a subtype of correspondent (person, HC provider, organization, ...), the program makes sure that the last update time of the matching correspondent record is automatically updated together with the last update time of the sub-type record.
