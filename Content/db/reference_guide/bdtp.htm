Each billing document template describes how a specific sort of billing documents needs to be printed.

Fields

Main > Add form feed

Indicates if a form feed character should be added at the end of the document. This may cause an additionial page skip on some printers.

Mandatory Logical AddFormFeed

Main > Calculated fields

This fields provides access to the template's calculated fields site function (w_BillingDocumentLine based, string valued). The return value should be a backslash seperated tagged list. The following tags are supported: 'UserFractional1', 'UserFractional2', 'UserFractional3', 'UserFractional4' and 'UserInteger1'. This site function is evaluated at the lowest printing level of the template. These 4 fractionals and 1 integer numbers are summed continuously during the printing of the document. These numbers can be printed as subtotals and running totals.

gp_SiteFunction CalculatedFields

This field provides access to the template's line data text. The referred text must be based on the w_BillingDocumentLine table. Main > Page length

This field contains the number of lines that can be printed on 1 page. This line count includes header and footer length and depends on the physical page length and font size. '?' is considered as an endless page. As a result no page breaks are produced.
