This table stores result codes and their meaning which can be used as pseudo results (e.g. tube broken, unmeasurable, out of limits,...)

Fields

Action

Specifies the action to be taken for the result. On discontinuation, the expansion becomes the rejection reason.

Mandatory ResultAction Action

One of: None, Confirm, Validate, Repeat, Dilute, Discontinue

Assessed

Specifies whether the assessment actually took place. This is used for statistical purposes.

Mandatory Logical Assessed

Code

Easy-to-remember result replacing code. It cannot be a mnemonic as this may contain all kind of characters (e.g. "+++", "#", ...). Spaces are not allowed in the code as they cannot be distinguished from a space used to separate two codes for properties with "Allow multiple result codes" = YES. Warning the code is case insensitive.

Mandatory String Code

Discontinue specimen

If this option is set, the program behaves as if the Specimen.Discontinue() function was called for the root specimen of the result with "Cascade = YES" and "Result code = <the entered code>". This will cause the root specimen (Result.Specimen) and any sub-specimen to be discontinued, and the entered result code to be applied to all results derived from any of the discontinued specimens.

A question will appear if an available result is found during processing. The discontinuation reason of the specimen(s) is hard-coded to: "Discontinued via '<PropertyMnemonic>' result (result code <ResultCode>)".

This functionality was primarily introduced to facilitate "No material received" situations. By entering one single code for one single result, the complete situation can be 'cleaned up'.

Mandatory Logical DiscontinueSpecimen
