Firm firm glims yes yes Normal

Firms are organizations authorized to create billing documents (invoices, invoice summaries, ...). Therefore a firm is that part of a lab that produces billing documents. Never use a firm for anything else: use 'Organization' or 'Company' if needed.

Quarter statistics 

This function produces month and quarters statistics over a given invoicing period. The invoice summaries, invoices, billing codes, etc that need to be taken into account can be specified by means of the function parameters. The output layout is user configurable.

MultiRecord Void QuarterStatistics (Date InvoicingStartDate, Date InvoicingEndDate, CorrespondentGroup CorrespondentGroup, ResidenceType ResidenceType, PriceList DeclarationPriceList, String MainContrRmbClasses, String SubContrRmbClasses, String ReimburserTypeListA, String PayerTypeListA, String ReimburserTypeListB, String PayerTypeListB, gp_Text MonthHeader, gp_Text MonthFooter, gp_Text MonthDetail, gp_Text MonthDetailPerYear, gp_Text QuarterHeader, gp_Text QuarterFooter, gp_Text QuarterDetail, rp_Printer Printer, Integer CopyCount, Integer PageLength, Logical Preview)

Date InvoicingStartDate
Start date of the invoicing period for which statistics are required.
Date InvoicingEndDate
End date of the invoicing period for which statistics are required.
CorrespondentGroup CorrespondentGroup
If specified, restrict the statistics to the invoices for which
- the payment agreement's fund or
- the invoice summary's payer or
- the invoice payer or reimburser
is a member of the specified group.
ResidenceType ResidenceType
One of: Hospitalized, Ambulatory
If specified, restrict the statistics to invoices of the specified residence type.
PriceList DeclarationPriceList
If specified, recalculate the invoiced amounts according to the specified price list.
Note: Amounts obtained during the tariffication with the specified price list are not recalculated. For billing codes that are payed by several payers, the new price is split according to the same ratio.
String MainContrRmbClasses
String SubContrRmbClasses
String ReimburserTypeListA
Restrict the data mentioned in column 'A' to data obtained from invoices of which the reimburser type belongs to the specified list.
String PayerTypeListA
Restrict the data mentioned in column 'A' to data obtained from invoices of which the payer type belongs to the specified list.
String ReimburserTypeListB
Restrict the data mentioned in column 'B' to data obtained from invoices of which the reimburser type belongs to the specified list.
String PayerTypeListB
Restrict the data mentioned in column 'B' to data obtained from invoices of which the payer type belongs to the specified list.
gp_Text MonthHeader
Dynamic text to be printed at the head of each month table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text MonthFooter
Dynamic text to be printed at the end of each month table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text MonthDetail
Dynamic text to be printed for each execution month in the month table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text MonthDetailPerYear
Dynamic text to be printed for each execution year in the month table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text QuarterHeader
Dynamic text to be printed at the head of the quarter table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text QuarterFooter
Dynamic text to be printed at the end of the quarter table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
gp_Text QuarterDetail
Dynamic text to be printed per detail line in the quarter table. Base table: w_QuarterlyBillingStat. Leaving this parameter to '?' causes a fixed layout to be used.
