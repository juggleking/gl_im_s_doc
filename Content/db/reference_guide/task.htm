Tasks further specify a command because they add the notion of timing. Tasks are fully defined commands to be carried out at predetermined moments in time. Commands are task templates, not defining any timing information. Tasks specify several criteria on how a command should be executed.

Schedule interval

The Schedule interval field determines the time lapse between consecutive runs of the command. The unit of this interval is implied by the task type. It is minute for instantly tasks, day for daily tasks, weeks for weekly tasks, month for monthly tasks, year for yearly tasks. For example, if the task type is weekly and the schedule interval is 2, then the task will be executed every two weeks.

On holiday

Indicates what should be done when the command is to run on a holiday:

* Ignore: ignore the holidays and execute the task as scheduled.
* Skip: the task will not be executed on holidays. The schedule will be recalculated.
* Shift backward: task is to be executed the day before the holiday.
* Shift forward: task is to be executed the day after the holiday.

Type

The Type combo-box allows you to determine the periodicity of the task execution. Possible values:

Instantly: task to be executed at a certain scheduled minute interval, on certain days, between the schedule start and end date. Select the appropriate day by marking its toggle box. For example schedule a task each Monday, every forty-five minutes between 06h00 and 18h00.

Daily: task to be executed once every day at a pre-defined moment, starting from the specified Schedule start. For example every day at 09h00

Weekly: task to be executed at a scheduled week interval, on certain days of the week at a certain time, starting from the specified Schedule start. Select the appropriate day by marking its toggle box. For example every two weeks, each Monday, Wednesday and Friday at 12h00.

Monthly: task to be executed at a certain scheduled month interval. The day within the month is determined by the Day type and Day type in month. For example every two months, every second Friday of the month. The Schedule start field defines the start time.

Yearly: task to be executed at a certain scheduled year interval on a specified month. The day within the month is determined by the Day type and Day type in month. For example, every two years, in January, every second Friday of the month.

Mandatory TaskType Type

One of: Instantly, Daily, Weekly, Monthly, Yearly, Once

Interrupt

With this function, you can schedule an interrupt for a running task. The task will continue to be executed by the Task Scheduler until it comes to a 'safe' place to stop processing more records. You can optionally also disable the task as soon as it has been interrupted, to prevent that the task is picked up again for execution by a Task Scheduler.
