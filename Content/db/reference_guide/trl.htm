Logging > Log file keep

"File keep and File keep unit are used together to indicate after how long a period log files should be automatically deleted. If not specified, you will have to delete the old log files manually. You can't delete a log file of a running translator.

PositiveInteger LogFileKeep

Logging > Log file keep unit

"File keep" and "File keep unit" are used together to indicate after how long a period log files should be automatically deleted. If not specified, you will have to delete the old log files manually. You can't delete a log file of a running translator.

Mandatory LoggingTimeUnit LogFileKeepUnit

One of: Hour, Day

Logging > Log file max size

Sets the maximal size (in MB) of the log file.

PositiveInteger LogFileMaxSize

Logging > Log file refresh

"File refresh" and "File refresh unit" are used together to indicate after how long a period open log files should be closed and a new log file should be opened. The actual log file names will be different for each new version by suffixing them with a creation time stamp.

PositiveInteger LogFileRefresh

Logging > Log file refresh unit

"File refresh" and "File refresh unit" are used together to indicate after how long a period open log files should be closed and a new log file should be opened. The actual log file names will be different for each new version by suffixing them with a creation time stamp.

Mandatory LoggingTimeUnit LogFileRefreshUnit

One of: Hour, Day

Logging > Log file refresh on max size

Determines what happens when the maximal log file size is reached. If set, a new log file is created. Else, the translator exits to prevent disk overflow. In such case, the associated translator server is informed and alerts all users of type "system manager" by email.

Mandatory Logical LogFileRefrshOnMaxSize

Logging > Log verbosity

Is either "Low", "Medium" or "High". Use it to set the amount of logging detail. We suggest setting this to "Medium". Info, warning and error messages are always logged, independent from the log verbosity. The log verbosity levels offer the following options:

Low: TCP/IP communication not logged
Medium: TCP/IP communication fully logged
High: byte per byte logging (for debugging purposes only)

Mandatory Verbosity LogVerbosity

One of: Low, Medium, High

Name

Is the identification of the translator. It should not contain any characters other than letters, digits and underscores. The first character must be a letter or an underscore.
