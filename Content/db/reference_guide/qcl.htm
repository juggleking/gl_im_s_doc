This table is a subtype of the Object table and is exclusively used to store QC lots.

Fields

Active

The active toggle box currently activates or deactivates this lot.

Mandatory Invariant Logical Active

Code

The Code is a (bar) code that may be used to identify this QC lot. This code can be very important to store barcode identifications originating from an analyzer. Whenever a bar code is received from a station (e.g. during result upload) GLIMS will check whether this bar code represents a QC Lot where this code equals the bar code. This field has precedence over a Code in the corresponding QC Material record.

Create new lot 

This function makes a copy of this lot and all its related records (QCPopulations and their checks), allowing easy creation of new QC lots. For each existing QC object (an object for which QC populations can be found), the upgrade procedure will create a dedicated QCLot record. Of course, this lot will not contain the results of its predecessor.
