Original coding system

The Original Coding System field allows to define a code set in a coding system which is actually the same as a code set from another coding system. Such a code set is not allowed to have own codes: each time this ocde set is used, the codes from the corresponding code set from the original coding system are used.

Code-based expression for URL

The content of this field is used when requsting the web page for this code on the code system's website.
It is used to build (calculate) the full URL for the specific code record that is selected.
E.g. for a code of a gene record, it will build the URL of the website of that specific gene on the website of that codesystem.
This field should contain a dynamic text expression, with base table 'Code'. It will be evaluated like a dynamic text module.

Examples of such URL expressions in the context of the gene table:
- 'https://www.omim.org/entry/{=.Value}’
- '{<OmimCodeUrlConstructor}’
where text module ‘OmimCodeUrlConstructor’ could be used for multiple tables and contains e.g.:
{: IF .CodeSet.Table.Name = "Gene" OR .CodeSet.Table.Name = "Disorder"
THEN RETURN "https://www.omim.org/entry/" + .Value ;
ELSE RETURN "" ;
ENDIF;
}
- 'https://www.icd10data.com/search?s={=.Value}'
- ...

Copy 

This function automatically fills code set by creating code records for all records of the code set source table. The code values are generated by the code set CodeFunction, a site function based on the source table and returning a string.
If a code record in the code set already exists, one can decide with the function parameter Overwrite if the code should be overwritten or not.
