Object obj glims yes yes Normal

The term 'Object' is used in GLIMS as a common denominator for everything the laboratory assesses properties of. Typical objects for clinical laboratories are patients. Types of objects include animals, persons, lots, QC lots or unknown objects.

AttributePeriod 

Returns the number of days the attribute is/was applicable for the specified object, at the specified reference date.
If no date is specified, the current date (today) is assumed.
If there is no attribute in the object's history that matches the specified date, 0 is returned.
If the attribute is not defined in the configuration, or if it is shielded, then an error will appear.
Example:
IF .AttributePeriod("Pregnancy", Today()) > 21
THEN .Person().SendMail("Congratulations");

DeleteAttribute 

This function should be used with extreme care as it actually deletes the object attribute !!
This function deletes any attribute history for a given object attribute that was active on the specified "EndDate".
It treats the "EndDate" as an inclusive boundary (meaning any attributes that STARTED or ENDED exactly on the specified date are also deleted)

Use the MISPL function .ClearAttribute() if you want to just specify an end date without deleting the attribute.

GetBloodBag 

Blood transfusion history is information about blood bags formerly transfused to the patient. This information can for instance be added to the 'datasheet'. The previous transfusions are important while processing the request of the irregular screening.

This method fetches the current blood bag associated to the object. At success, the record id of the current blood bag is returned. Else, unknown. Remark: this method should be used in combination with blood bag fields in order to get blood bag-related information.

ResultAttribute 

A search (starting from this object) for a result record of this object is carried out.

The result value of the method is a string. At success, the method returns the value of the specified attribute, else an unknown value is returned.

Note: When specifying the external/internal comment as attribute name, GLIMS will not be able to expand possible text modules (due to incompatible start table). However, the example below shows how you can avoid this problem and just display the mnemonic of the text module. The {} are replaced by other signs using the Replace function:

{= Replace(Replace(.ResultAttribute("Natr",?,1,?,"ExternalComment"), chr(123), "["), chr(125), "]")}
