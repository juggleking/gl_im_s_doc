Product usage pdus glims yes yes Normal

A product usage record links a product specification to a work place, and determines the minimum and maximum stock amounts required for the specific work place. If specified and returning a valid number, the fixed amounts may be overruled by a site function, allowing implementation of a dynamic stock.

For general purpose products, a dummy work place may be created. This is however not required. The data model does not impose a ProductUsage. In that case no quantity requirements nor stock location is available however. Hence, automatic order form generation and ordering will not be possible for these products.

Warning threshold

Amount below which a warning will be issued, i.e. the last columns of the product specification browser will be shown with a yellow background. This threshold must be specified as a percentage within the minimum/maximum quantity range. Example:

Minimum quantity = 5

Maximum quantity = 20

Warning threshold = 10

Range => 20 - 5 = 15.
Threshold => 10% of 15 = 1.5
Threshold value = 5 + 1.5 = 6.5 (rounded to 7)
