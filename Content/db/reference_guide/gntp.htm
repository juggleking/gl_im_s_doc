Property classification

This defines a relationship between Properties and GeneticProcedures, so that one can indicate which Properties are used for which genetic exams.
It is especially important when we have more than one exam in the same order, or if the order also contains other requests beside genetics.
This information can be used on all genetic screens where results are shown, to perform a filtering on the properties that must be shown or hidden.

PropertyClassification PropClassification

Report property

This is a link to a special Property record that will be used for triggering reports.
The Property must have data type Genetics.
Each genetic exam gets a report result for the Property indicated in the genteic procedure.

Mandatory Property ReportProperty

Department function

This is a site-function (executed on the GeneticExam) that can be used to define (‘calculate’) at runtime which department will be responsible for the exam, depending on some runtime data, e.g. depending on the order reception department.
It must return the mnemonic of an existing department, or it must return the unknown value (or ‘?’). In that latter case the default department will be used.

gp_SiteFunction RespDptFunction

Responsible department

This defines the department that will be responsible for validating the results and reports for this type of genetic exam.
It can be overruled by the by department MISPL function.

Mandatory Department ResponsibleDepartment

Support family?

Indicates whether the genetic exams for this procedure can refer to a family. If this field is inactive (default active) then the genetics screen will not show the family field and the family pane for this type of genetic exams.

Mandatory Logical SupportFamily

Suspected disorder

This defines which disorder is suspected and will be investigated during the exam.
It can also be left empty, which means that no specific disorder is suspected.

Disorder SuspectedDisorder
