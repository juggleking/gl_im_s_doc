Invoice summary reminder isr glims yes yes None

This table contains the reminders created for an invoice summary. Reminders are reprints of invoice summaries to remind the customer that an invoice summary is due. It is possible to add supplement amounts to reminders. Reminders are created with the "Create Reminders" method of the invoice summary table.
Invoice summary reminders can only be deleted if the corresponding invoice summary does not have the status 'Balanced' yet. If the invoice summary reminder with the highest level is deleted, then the ReminderLevel field of the invoice summary is changed to reflect this.

Level

This is a site enumerated defining the level of the reminder. There are 9 possible levels, which names can be defined in the "Site domains" section. The name of the domain is "ReminderLevel". The level of a new invoice summary must be higher than the highest existing reminder level for this invoice summary. So it is possible to skip ReminderLevels when creating invoice summary reminders, but it is not possible to 'fill the holes' later.

Mandatory Invariant SiteEnumerated Level

Supplement amount

This field indicates the total supplement amount for this reminder. Supplement amounts are never included in any invoice summary amount. If one wants to include the amount on a printed invoice summary, it needs to be done in the layout definition.
The reasoning after not including the supplement amount in the invoice summary amount is that most payers never pay such supplements. Consequently, labs don't like these amounts to appear on their balance sheets. In those rare cases were a supplement is paid, it is booked as a 'Gain'.
