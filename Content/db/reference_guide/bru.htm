Detection order

This field stipulates the order in which the billing rules will be evaluated (low to high).

Mandatory PositiveInteger DetectionOrder

Conditions > Diagnose check

This fields provides access to the billing rules 'eligibility condition' site function. Usually this site function checks patient and diagnostic data. The referred function is based on the table specified by the billing rule's scope and should return a logical value. If the evaluation of this function returns 'true', then the rule is activated. The return values 'false' and '?' are considered as 'not applicable'.

gp_SiteFunction DiagnoseCheck

Actions > Discount code

This field provides access the billing rule's discount code. Whenever the rule is activated, all affected members of the billing rule will be discounted according the specified discount code.

DiscountCode DiscountCode

Conditions > End date

This field indicates the validity period's end date. The billing rule is valid up to and including the specified end date. If there is no end date, the validity period is considered as not ending.

Date EndDate

Conditions > Fund group

This field provides access to the billing rule's fund group. If specified, the rule is only applicable if the related fund is a member (if InFundGroup is set) or is not a member (if InFundGroup is not set) of the specified group.

CorrespondentGroup FundGroup

Id

Primary key. Automatically generated identifier whose value uniquely identifies each record.

Mandatory Unique Automatic BillingRule Id

Conditions > In fund group

This field indicates whether the occurence (yes) or absence (no) of the fund in the specified fund group must be checked.

Mandatory Logical InFundGroup

Conditions > In issuer group

This field indicates whether the occurence (yes) or absence (no) of the issuer in the specified issuer group must be checked.

Mandatory Logical InIssuerGroup

Conditions > Issuer group

This field provides access to the billing rule's issuer group. If specified, the rule is only applicable if the order's issuer is a member (if InIssuerGroup is set) or is not a member (if InIssuerGroup is not set) of the specified group.

CorrespondentGroup IssuerGroup

Actions > Maximum translations

If this field is specified, the line substitutions are affected. During the billing rule line substitution the number in this field is compared with the current number of translations that already took place during this rule evaluation. Depending on whether the maximum is exceeded, the billing rule line outputs marked as such are taken into account. Not specifying a maximum is interpreted as the maximum is not exceeded.

PositiveInteger MaximumTranslations

Conditions > Minimum amount

The billing rule is only activated if the total amount of all the inputs that match the billing rule criteria is higher then the specified minimum. '?' is considered as no condition.

Fractional MinimumAmount

Conditions > Minimum base value

The billing rule is only activated if the total base value of all the inputs that match the billing rule criteria is higher then the specified minimum. '?' is considered as no condition.

Fractional MinimumBaseValue

Conditions > Minimum count

The billing rule is only activated if the total number of the inputs that match the billing rule criteria is higher then the specified minimum. '?' is considered as no condition.

PositiveInteger MinimumCount

Mnemonic

Actions > Reevaluate

This field indicates whether the billing rule should be evaluated more then once, provided all conditions are correct.

Mandatory Logical ReEvaluate

Actions > Resubstitutable

This field indicates whether the outputs of this billing rule can be resubstituted by consecutive billing rules.

Mandatory Logical Resubstitutable

Conditions > Scope

This field where the inputs of the billing rule must be found:
- Root specimen: Billing code related with the same root specimen (= specimen not referencing a parent specimen).
- Sampling time: Billing codes related with root specimen sampled at the same time.
- Order: Billing codes related with the same order.
- Order set: All the billing codes.

Mandatory BillingRuleScope Scope

One of: Root specimen, Order, Order set, Sampling time, Isolation
