Password > Authorization timeout

The Authorization timeout is the timeout in seconds, after which authorization is lost. If no action is performed within the specified timeout period the user is asked for his password, otherwise he eventually will exit the application. It overrides the value specified in the MATE customization screen.

PositiveInteger AuthorizationTimeout

Password > Autologin allowed

If the Autologin allowed toggle box is checked, the user can bypass the application login screen and rely on operating system authentication.

Do not forget to supply startup parameter '-a' (please consult your system manager). On Windows, the application shortcut should contain " -a " (between quotes). E.g. -basekey ini -p gp_prun -T c:\temp -param " -a ".

MISPL functions

HasPrivilege 

This function checks if the user owns the specified privilege. It returns the logical value YES if the privilege is granted to the user, else NO. If the user if of type 'Guest' the return value is always NO, as a guest has no privileges. If the user is of type 'System manager' or 'Developer', the return value is always 'YES'. If an invalid privilege name is specified, the error condition is raised and a run-time error is reported.

Logical HasPrivilege (String Privilege)

String Privilege
Name of the privilege to be checked.

SendMail 

This function sends a mail to a user. The mail will be internal mail if the user has 'Internal' as mail method and external mail if the user has 'External' as mail method. The sender of the mail is the user currently logged in.
