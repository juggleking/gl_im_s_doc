Task tsk glims yes yes None

A task registers that an action has to be undertaken by a specific user (optional), for a specific station (optional), a specific correspondent (optional), a specific tour (optional),... Tasks are also a convenient way of registering incoming calls from customers, assigning work to the customer's tour responsible or a lab biologist.

Standard task query

Standard (String InternalIdPrefix, sc_User User, Correspondent Correspondent, String TourPrefix, Tour Tour, Station Station, Logical Today, Date MinimalDate, Date MaximalDate, TaskStatus MinimalStatus, TaskStatus MaximalStatus, Priority MinimalPriority, String DescriptionPattern)

String InternalIdPrefix
If specified, only tasks with this internal id prefix are selected
sc_User User
If specified, only tasks for this user are selected.
Correspondent Correspondent
If specified, then only tasks for this correspondent will be selected
String TourPrefix
If specified, then only tasks for tours with this prefix will be selected.
Note that the <Tour> parameter has precedence.
Tour Tour
If specified, then only tasks of this tour will be selected.
Station Station
If specified, then only tasks of this station will be selected
Logical Today
If set, then only select tasks scheduled for today.
Note that minimal date and maximal date have precedence.
Date MinimalDate
Only tasks with a schedule date equal to or above this date are selected
Date MaximalDate
Only tasks with a schedule date equal to or below this date are selected.
TaskStatus MinimalStatus
One of: Open, Busy, Closed
Only tasks with a status equal to or above this status are selected
TaskStatus MaximalStatus
One of: Open, Busy, Closed
Only tasks with a status equal to or below this status are selected.
Priority MinimalPriority
One of: Low, Normal, High, Alarm
Only tasks with a priority equal to or above this priority are selected.
String DescriptionPattern
If specified, only tasks where the descriptions matches this pattern are selected.
