Policy clause plcc glims yes yes Normal

A policy clause basically specifies who (reimburser and payer type) reimburses which billing codes (ReimbursementClass) for a specific policy (Policy). Optionally, extra conditions (residence type, subcontracting, etc ...) can be specified.

Conditions > Eligibility

This field provides access to the eligibility site function (order based, logical valued) of the policy clause. The return value indicates whether the policy clause is applicable. Returning '?' is considered as 'no', no site function is considered as 'yes'.

Conditions > Validity > Valid until

This field indicates the validity period's end date. The policy clause is valid up to and including the specified end date. If there is no end date, the validity period is considered as not ending.

Specifications > Charge > Fixed amount

Mandatory Fractional FixedAmount

Conditions > Forwarded

This is an optional validity condition of the policy clause.
Yes: A lab request is forwarded to another (not the receiver) lab for execution.
No: A lab request is executed in the receiving lab.

Specifications > Charge > Multiplier

This field is used for the calculation of the amount to be invoiced to the specified payer type. Suppose the price calculated so far = 'Pt'.
The invoiced amount = Pt * Multiplier + FixedAmount.
This value is mandatory except for a '3th payer' policy where the reimbursement part is defined in the '3th payer payment agreement'.

Fractional Multiplier

Conditions > Residence type

This field specifies the required residence type (? = no requirement). The clause is only applicable in case the order's residence type matches the specified value. An order's residence type is the residence type of the encounter type of the order's encounter. Note that orders, not referencing an encounter are considered as 'ambulatory'.
