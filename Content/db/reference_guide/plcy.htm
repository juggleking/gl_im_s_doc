Payment agreement check

This field provides access to the payment agreement check site function (payment agreement based, string valued). Each time a payment agreement with the checked-box on is updated, this site function is evaluated. If the return value is a non empty string, then that string is considered as an error message and the update fails. Returning '?' also causes an update failure.

gp_SiteFunction PaymentAgreementCheck

Payment agreement update

This field provides access to the payment agreement update site function (payment agreement based, string valued). Each time a payment agreement with the checked flag set is updated, this site function is evaluated. The return value should be a backslash seperated tagged list. The following tags are supported: 'Matriculation' and 'Relation'. Both tags correspond with a payment agreement field. The specified tag values will overwrite the correspondig record fields.
Example: Matriculation=123-456678-12\Relation=Holder
This functionality can be used to (re)format the matriculation number or to set or correct the relation type. e.g. In Germany the relation ('VersichertenArt') is determined by the insurant status ('VersichertenStatus').

Supplement list

This field provides access to the policy's supplement list site function (order set based, string valued). This function is evaluated during the tariffication after the billing rule conversions. It supports the addition of supplement billing codes. Returning '?' causes a tariffication error. Returning the empty string ('') or not specifying a supplement list function causes no supplement addition.
The return value consists of a comma seperated list of billing code specifications. A billing code specification must match the following template:
<BC>[\Comment=<Comment>][\Executor=<Executor>]{\<TagName>=<TagValue>}
<BC> = Billing code code
<Comment> = Comment string
<Executor> = HC-provider mnemonic
<ExecutingLab> mnemonic of the executing lab
<ExecutingDepartment> mnemonic of the executing department
<TagName> = A billing item's site attribute name
Each such specification causes the addition of the corresponding billing code as a supplement. The comment value will be copied to the billing item's comment field. If an executor is specified, that executor will overrule the regular executor determination. The other tag values are saved as a site attribute values. Example:
591673\Executor=GORBY\BV=433016,599955\BV=128030

Tariff time type

This field specifies which date/time to use to retrieve time depending configuration data (price formula, tariff, ...) during the tariffication. If the order set consists of several orders, the earliest date/time is taken into account for 'object time' and 'receipt time', the most recent one for 'completion time'. The object time of an order is the order lowest object time.

Mandatory TariffTimeType TariffTimeType

One of: Object, Completion, Receipt, Actual
