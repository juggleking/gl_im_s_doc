Triggers > Antibiogram confirmation

Isolation-based MISPL program that is executed when:

- an isolation is being confirmed (either explicitly or because the microbiology action status is increased)
- all its antibiogram results have already been confirmed (they have status 'confirmed' or 'validated')
- the isolation has been changed after the confirmation of each of the results in the antibiogram of the isolation.

The following functions can be used in this MISPL:

- SetRIS(AntibioticMnemonic, RISNewReportValue, Create): changes the RISReportValue to RISNewReportValue if

1. RISReportValueChanged = NO (or user agrees after prompt) and the antibiotic was tested, or
2. the antibiotic was not tested and Create =YES.

- ReplaceRIS(AntibioticMnemonic, RISOldRawValue, RISNewReportValue): changes the RISReportValue to RISNewReportValue if RISReportValueChanged = NO (or user agrees after prompt) and the RISRawValue = RISOldRawValue.

- EqualRIS(AntibioticMnemonic, RISRawValueList, MustBeTested): returns YES if the RISRawValue is in of the RISRawValueList or if MustBeTested = NO and the antibiotic was not tested.

- HideRIS(AntibioticMnemonic, RISReportValueList): does not report the RISReportValue of the antibiotic if it is part of the RISReportValueList.

Epidemiology > Maximum number of days > Infection lag before encounter

If an isolation was detected for patient A while he was not hospitalized (so during an ambulatory encounter), and afterwards he is hospitalized and stays in the same ward as patient B (for which the same microorganism is also detected), then Glims is now able to detect this as a cross infection.
This is not the default behaviour however.
To obtain this, this field needs a value:
With this field you can specify how many days Glims should look back in time (before the start of the hospitalized encounter), while searching for isolations of patient A *before* patient A was hospitalized. (default value = 0 days)
This field is also inherited by all microorganisms below it (in the tree).

Fractional InfectionLagBeforeEnc

Statistics > Is group

A microorganism group is a microorganism where the 'Is group' option is enabled. All microorganisms which have this microorganism as parent or grand-parent etc. belong to this group. Can be used in the classification options of microbiology statistics.

If set, no child or parents of this organism may have this field set.
It is meant for statistical purposes. If statistics are done by microorganism group, the system will divide all isolations by their organism group. If an organism is marked as 'group', all underlying child organisms belong to this group.

Technical > Formula to calculate the severity

Isolation-based, integer-valued MISPL function. The severity of a micro-organism can depend on the isolation appraisal, material, material variable...
If an isolation is given a positive appraisal, the severity is calculated with this formula.
The formula is an inherited field. I.e. if the formula is not given, the system searches for the next higher parent's severity formula.

Example function:

IF .Specimen.Material.Mnemonic = "M_Urine"
THEN
IF .Appraisal.Mnemonic = "+++" THEN RETURN 3; ENDIF;
IF .Appraisal.Mnemonic = "++" THEN RETURN 2; ENDIF;
RETURN 1;
ENDIF;
RETURN 0; /* normal */

Statistics > Taxonomic group

Each microorganism can be classified to a certain taxonomic group. Following groups are available: Subspecies, Species, Genus, Family, Order. The taxonomic group can be a criterion in the detection of doubles. Doubles are restricted to isolations of organisms within the same species. Isolations of different species cannot be considered to be identical. You can also use the taxonomic group in the microbiology statistics.

MicroorganismTaxonomy TaxonomicGroup

One of: Subspecies, Species, Genus, Family, Order
