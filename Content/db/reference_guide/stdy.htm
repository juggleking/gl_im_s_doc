Study stdy glims yes yes Normal

A study is issued by an issuer and contains a number of orders. The DefaultReports of the Study's correspondent decide which reports are created for orders of this study. If such DefaultReports exist, the own organization's default reports or the order issuer's default reports are NOT evaluated.

MISPL functions

GetNextOrder

Allows retrieval of the different orders within the study. Returns a reference to the next order within this study. The <PreviousOrder> parameter should be unknown for the first call and contains the id of the last retrieved order for the subsequent calls.

Order GetNextOrder (Order LastOrder)

Order LastOrder
Contains the id of the previously retrieved order

GetNextStudyObject

Allows retrieval of the different members of a study. Returns a reference to the first study object with a seqno greater than the specified seqno and a role matching the imposed role.

StudyObject GetNextStudyObject (PositiveInteger SeqNo, Mnemonic RoleMnemonic, Mnemonic ConclusionMnemonic, PersonRelationType StudyObjectRelation)

PositiveInteger SeqNo
Contains the sequence number of the last retrieved study object
Mnemonic RoleMnemonic
IF specified, only a study object with the specified StudyObject.Role.Mnemonic is retrieved
Mnemonic ConclusionMnemonic
If specified, only a study object with the specified StudyObject.Conclusion.Mnemonic is retrieved
PersonRelationType StudyObjectRelation
One of: parent, child, sibling, halve-sibling via father, halve-sibling via mother, parent of father, parent of mother, child of son, child of daughter, sibling of father, sibling of mother, child of sibling, cousin, other related, partner, candidate unrelated donor, candidate unrelated recipient, organ donor, organ recipient, stem cell donor, stem cell recipient, blood donor, blood recipient, other non-related, patient in needle stick injury, staff in needle stick injury
If specified, only a study object with the specified StudyObject.StudyObjectRelation is retrieved
