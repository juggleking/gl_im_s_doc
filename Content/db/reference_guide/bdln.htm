This table is used for printing billing document; it serves as the start table for all texts that make up the billing document template used for printing. The only exceptions are the texts regarding discounts and VAT detail lines that use another start table.
Several fields only contain a meaningful value for some texts:
+ many fields are invoice item specific and only contain values for 'line data' texts: BillingItem, InvoiceItem, Tariff, Property, Request, ...
+ references to invoice, invoice summary and financial shipment are only filled when printing a document of that level or higher.
+ all amounts and 'number' fields contain the correct data on the 'line data' level and are automatically subtotalled by the printing program for higher levels. So when printing an invoice header, the 'NetAmount' field does not contain a value from the invoice record, but the sum of 'NetAmount' fields of all the invoice items on that invoice.

BillingItem BillingItem

Count

The number of invoice items represented by the current level. So for 'line data' level texts this is 1, except when grouping is in effect: then it represents the number of grouped invoice items. For all higher level texts (invoice, invoice summary, etc...) it is the total number of invoice items represented by that level. So for an invoice header it would be the total number of invoice items on that invoice.

Mandatory Integer Count

Executing lab reference

All labs that occur as 'executing lab' on any of the billing items of the printed document, are given a sequential number starting from 1. For 'line data' texts, this field is filled with that sequential number so that it can be used as a footnote reference.
The actual lab data can then be printed in the page footer (or trailer) using the GetExecutingLab() function. That function returns the RECid of the lab.
