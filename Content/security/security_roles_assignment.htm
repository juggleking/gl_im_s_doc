Introduction

Once all the roles have been created, the next step is to assign the available users to the defined roles.  As a user always has a role within a department (employee assignment), this process involves two steps.

Step 1: Assigning a user to a role using a user role assignment

Starting from the Roles browser, use the contextual ribbon item / menu Show all -> user role assignments to list all users assigned to a specific role. By adding a user in the user role assignments browser, a new user role assignment record is created for that user and that specific role.

You can also assign users to a role in the Security Center.

Step 2: Assigning a user role assignment to an employee assignment

Once a user is assigned to a role, you must now indicate to which employment this role applies.  An employment is a link between a user and a department.

  1. From a User context, open the employee assignments (Settings -> Employments).

  2. For each employment assignment, you can now specify one of the user's roles (user role assignment).

Note

  1. It is possible for a user to have several roles in the same department by creating several employee assignments with a different role.
  2. When logging in, the user will first have to select the department and then the role within the department.
