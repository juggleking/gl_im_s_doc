Fields

The list of roles is available from the main menu via Start -> System management -> Security -> Roles.

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Name

Multilingual field, used for display purposes.

Description

Multilingual field, can be used to supply a description of the role.

Icon

Currently not used by the application.

User type

The user type is important for checking privileges GLIMS.

Guests

Guests have no privileges at all. It is impossible to assign a privilege to a guest as GLIMS will not take this into account. As a consequence, guests only have access to fields, functions or tables which are not linked to a privilege.

Users

Users have access to all functions for which they have the appropriate privilege or for which no privileges have been defined.

System managers

System managers have access to all fields, functions and tables. They also have the authority to assign privileges to users.

Tip

There is 1 exception: system managers will not have access to functions, tables or fields protected by the _SysAdm_NoPriv1 and _SysAdm_NoPriv2 privileges if these privileges are not explicitly assigned to them.

Developers

Warning

This user type is reserved for MIPS employees. Functions with minimal user type Developer should therefore not be executed without the assistance of a MIPS employee!

Ribbon

Ribbon that will be displayed for users with this role. If not specified, the ribbon specified in the general settings (Start -> System management -> Customize -> MATE) will be used.

Side bar

Side bar that will be displayed for users with this role. If not specified, the side bar specified in the general settings (Start -> System management -> Customize -> MATE) will be used.

Home page

Home page that will be displayed for users with this role. If not specified, the home page specified in the general settings (Start -> System management -> Customize -> MATE) will be used.

Conversion from GLIMS 8 to GLIMS 9

When upgrading from GLIMS 8 to GLIMS 9, the conversion procedure will automatically create a number of roles based on your current configuration.

Creation of 4 default roles

4 default roles will be created, one for each user type (Guest, User, System manager and Developer).  The special user sysman will be assigned the default System manager role.

Conversion of user group privileges

This paragraph describes the conversion of existing UserGroupPrivilege records to additional Role and RolePrivilege records.  

If there are no UserGroupPrivilege records, we simply have no RolePrivilege records to create, and will assign each User to the applicable default role, based on his/her UserType.

If there are UserGroupPrivilege records, we need to determine the Role archetypes:

  1. Per UserType
  2. Per unique set of UserGroupAssignment/UserGroupPrivilege

For each of these archetypes, the conversion procedure will create additional Role records of the correct UserType, create the appropriate RolePrivilege records, and create the necessary UserRoleAssignment records.

The Role.Comment field is used to store an explanation of why this role was created during conversion, including which UserGroups its RolePrivileges were derived from.

Note that UserPrivilege records nor UserGroupAssignment records will be changed.
