Users

Contents: User browser - User fields

User browser

The user browser is available from the main menu via Start -> System Management -> Security -> Users or via the Security center. The icon above the list allows to select enabled/disabled/all users.

User fields

Initials

User initials that can be used on external reports

Last name/ first name

Last name and first name of the user 

Login name

Name by which the user must identify himself at login time. This name is not case-sensitive.

Roles

Shows the roles assigned to this user

Enabled

Once a user is created, this record can never be deleted. If you want to deny a user's access to GLIMS then leave this toggle box unchecked.  For example a user who has sick leave or has been dismissed can be disabled for a certain period of time.

Session type

This field determines the type of sessions for this user. There are two options: Interactive or Batch [default value: Interactive]

Interactive

Is meant for humans who work with the application through a user interface

Batch

Is meant for processes such as TranslatorServers and TaskSchedulers.

Note:

A batch user must have the following characteristics:

  • User.Enabled = YES
  • User.AutoLoginAllowed = YES
  • User.SessionType = Batch
  • User is assigned a Role of user type System manager

Auto-login allowed

Check the chapter Login for more information. 

Password verification

This field determines the password verification method for this user.

Internal

Implies that the user password will be verified internally, even if LDAP connection settings were configured.

Tip: This is a good option for Batch process users as the system administrator will no longer have to create an LDAP user for these users

LDAP

Implies that the User password will be verified against LDAP, provided the LDAP connection settings were configured.  If no server connection could be established, the specified password is compared to the password present in the GLIMS user record.  If it matches, access is granted.

Minimum password length and Maximum password age

These are only applicable for users of type guest and user.  For more information please check the chapter "Login -> Passwords".

Authorization time-out / Context time-out:  

For more information, please check the chapter "Login -> Time-outs".

Language

Check the chapter "Languages" for more information

Start tool

Tool that will be activated each time the user logs in.

Printer

Offers the possibility to assign a default printer to a user.

Home page

Check the chapter User interface - Home page

E-mail address

E-mail address of the user, used as the sender identification when the user sends external mail.

Mail method

Indicates how the user wants to receive his mail.

  • Internal: database mail
  • External: internet mail
  • Both: internal and external mail

Phone

User's phone number
