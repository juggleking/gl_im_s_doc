Introduction

The shielding concept in GLIMS is about access rights to routine records of other departments: results, orders, persons, object attributes,...

GLIMS offers two mechanisms for shielding respectively based on:

  • access profiles,
  • department groups.

The shielding mechanism based on profiles is the most recently introduced mechanism and is definitely the most flexible system. Below you will find a short description of both mechanisms with their respective advantages and disadvantages.

Note

Both shielding mechanisms are protected by a license.

Profile shielding

General description

Profile shielding allows explicit specification of the access rights of a user to results, order, persons, object attributes,... other departments. This specification is based on a user-specific profile rather than on the fact that the user works for a specific department.

The profile shielding allows for different settings for technicians, administrative personnel and biologists. It also allows independent setting of the shielding for results, orders, object attributes and persons:

Advantages over department group shielding

  • Easier to understand and configure.
  • Different users of the same department can be given different access rights (for instance, a biologist can see results from another department while a technician cannot).
  • Shielding for results, orders, persons, object attributes,... can be set independently.
  • A user can be given the right to modify a result but not confirm it (or discontinue it), or given the right to request a test and discontinue it again without having the right to modify the result.
  • MISPL calculations and triggers are no longer restricted by shielding, but other MISPL access still is.
  • A batch user can be given full access to all departments.

For more information, please read the chapter Shielding - Access profiles.

Department group shielding

Department group shielding is a mechanism that has been introduced in early versions of GLIMS. It is still supported, but less flexible than the profile shielding mechanism.

For more information, please read the chapter Shielding - Department groups.

Which shielding mechanism is applied?

The general option Shielding method determines which shielding mechanism (Access profile or Department group) will be applied.

From the main menu, choose Start -> System management -> Customize -> GLIMS shielding.

Shielding method

Department group or access profile driven.

Note

These two methods cannot be combined.

Order shielding

For access profile shielding, the configured order shielding will only be applied if this option is activated.
For department group shielding, the orders will only be shielded if this option is activated.

Person shielding

For access profile shielding, the configured person shielding will only be applied if this option is activated.
For department group shielding, the persons will only be shielded if this option is activated.

Switching from Department group to Access profile shielding

When switching from a Department group shielding configuration to an Access profile one, a warning is displayed, indicating that the MA_***Shielding environment variables are no longer to be used.

We strongly advise removing the MA_Shielding settings from the Progress.ini files, since they may conflict with the specific site settings under GLIMS shielding. When using the old (department group) system, the Progress.ini settings still have precedence.
