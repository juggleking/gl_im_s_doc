Introduction

The Users tab page of the Security Center allows to configure users, assign roles to users and assign privileges to users.  The screen is composed of different sections which are documented below.

List of users

The list of users shows all users (enabled and disabled) that are defined in GLIMS.

  1. Select a user to see which roles / privileges are assigned.
  2. Use the Insert key to define a new user.
  3. Users cannot be deleted, but can be disabled by un-checking the Enabled option.

List of roles

The list of roles shows all roles that are linked to the selected user.

  1. Use the Insert key to assign new roles to the selected user.
  2. Use the Delete key to de-assign a role from the selected user.

Tip

Once a role is assigned to a user, this "user role assignment" must also be linked to an "employee assignment".

List of privileges

This section of the screen allows to link privileges to a user, which is mostly used to make exceptions to the role-based privileges.

By selecting a user and a role, you have an immediate overview of all granted privileges. By default, all GLIMS privileges are shown. Enable the option Only show effective privileges to limit the list to those privileges that are assigned to the selected user.

Privilege

Name of the privilege.

Role

The check box in this column indicates whether the role selected in the List of roles (see above) is linked directly to the privilege.  This column is read-only as this screen is designed to define user-based privileges only.  To define role-based privileges, you should switch to the Role tab page of the Security center.

User

The check box in this column allows to assign/de-assign a privilege to/from a user.

Effective

This column indicates whether the selected user/role combination is effectively linked to a privilege (either through the role-based privilege or through the user-based privilege).
