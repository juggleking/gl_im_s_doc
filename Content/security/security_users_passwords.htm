Users: Passwords

Encryption

The passwords in the database are encrypted.

Changing passwords

  • Users can change their own password by selecting the menu option Start -> Tools -> Change password.
  • System managers can change the password of any user by selecting a User and choosing the contextual menu /ribbon item Change password.
  • The maximum password length is 200 characters.

Password rules

Minimum password length / maximum password age

The password must respect the Minimum password length and a Maximum password age.

Both criteria can be set for each individual User or generally for all GLIMS users (via Start -> System management -> Customize -> MATE). If the criteria are not set in the User configuration, the general settings will be applied.

User-specific

General settings

Password history

GLIMS also keeps track of a password history. When changing your password, you must choose one different from your last 6 passwords.

Password strength

In the general options (System management -> Customize -> MATE), you can set the password strength to be applied when choosing a password. Possible values:

Medium

No additional rules.

High

The following rules will be applied:

  • The password must have at least 9 characters (or the minimum password length if it exceeds 9 (see above))
  • The password must contain characters from three of the four following categories
    1. Uppercase letters of European languages (A through Z, without diacritic marks, Greek and Cyrillic characters) A, B, C, … Z
    2. Lowercase letters of European languages (a through z, sharp-s, without diacritic marks, Greek and Cyrillic characters) a, b, c, … z
    3. Base 10 digits (0 through 9) 0, 1, 2, … 9
    4. Non-alphanumeric characters (special characters) $,!,%,^,(){}[];:<>?

Currently Diacritic marks, Greek and Cyrillic characters are NOT taken into account, neither are Unicode characters from Asian languages.

Maximum login failures

In the general settings (Start -> System management -> Customize -> MATE), GLIMS allows to define the maximum number of login failures after which a user will be disabled.

Note

All login failures are logged in GLIMS.
