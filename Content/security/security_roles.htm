Introduction

A role defines how the user interface is presented to the user and, most important of all, what a user is allowed to do.

  • When multiple persons working in a laboratory all perform the same tasks and share the same responsibilities, they can all be assigned to the same role.
  • When one person working in a laboratory has different tasks and responsibilities, he or she can be assigned to multiple roles.

Roles thus allow a task oriented interaction between the user and the system.

By selecting a role during login, the system is started according to the role's settings and optimized for performing the tasks and responsibilities associated with the role.

Implications

Roles have a major impact on the access rights and visualization of the application. Roles can be linked to a:

  1. ribbon
  2. side bar
  3. home page
  4. start menu
  5. set of privileges
