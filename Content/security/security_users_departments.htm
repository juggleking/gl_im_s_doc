Security - Assigning a department to a user

Introduction

A laboratory can be subdivided into several departments. A user can be employed in several of these departments, possibly with a different role. In GLIMS this can be defined as Employee assignments.

Use in GLIMS

Several programs in GLIMS make use of the employee assignment information.

Login

When logging into GLIMS, the user will have to indicate in which department (and in which role) s/he is currently working. If a user is only assigned to a single department, GLIMS will automatically select that department.

The selected department will be displayed in the right corner at the bottom of your screen.

Order entry

The department influences the procedure selection. GLIMS will also link an order created by this user to his current department.

Responsibility

Employee assignments can be used to determine a responsible to be printed on reports.

Fields

To define an employee assignment, select a user and choose the contextual menu / ribbon item Employment.

Role

Select one of the roles assigned to this user.When logging in, the user will first have to select the department and then the role within the department.

Warning

If a user selects a department, and no role at all has been defined for that employee assignment, the user will be able to select any of the roles assigned to him / her.

We therefore strongly recommend to always specify a role for each employee assignment!

Department

Select one of the defined departments.

Sequence number

Used to determine the sequence in which the departments are shown when logging in (if a choice must be made).

Access profile

Please check the chapter Shielding - Access profiles.

Active

If an employee no longer works in a certain department, you can disable this option. By default this field is enabled.

Responsible

Specify the mnemonic of a health care provider which is the responsible. This health care provider can be printed on reports as the responsible.
