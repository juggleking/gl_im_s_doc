Analyzer download status

Contents: Introduction - Setting the download status - Consequences - Clearing the download status

Introduction

This chapter describes how GLIMS marks that an action (and its result outputs / isolation test outputs) have already been downloaded to a given station, how you can clear this download status in order to resend all the actions etc.

Setting the download status

GLIMS has several built-in mechanisms to mark an action (and its result outputs / isolation test outputs):

  1. When a specific specimen query is received from an analyzer, the Queried by station mark will be set automatically. Note that this option is not set in case of an ALL query.
  2. Whenever an action is downloaded to an analyzer, this is stored in the Download status of the action (and its result outputs / isolation test outputs). The download status marks to which station(s) or download group(s) the action has been downloaded.

    You can use the Action pop-up menu function View download status to display a browser with all the statuses on the action's outputs.



    The browser shows the following columns:
    • Record type (*): shows the record type having a download status value, A=Action, R=Result output, I=Isolation test output.
    • Station: the station for which a download took place
    • Group: the station download group for which a download took place. If group is filled in for one record, the station column value is omitted.
    • Time: the time the download took place (only for action outputs)
    • The queried flag (Q): set when the specimen was explicitly queried for download
    • Output
      • When result output record: the related property mnemonic
      • When isolation test record: the related micro-organism test mnemonic and the isolation's internal sequencer value between brackets
  3. For stations grouped in a Download group (stations with the same remote id) you can mark that an action (and its result outputs / isolation test outputs) has been downloaded by one of the stations belonging to the download group. In order to activate this marking mechanism, you must enable the option Mark action by group for all stations that belong to the download group.

Consequences

In the Station configuration, you can set the option Check downloaded action.

  1. If this option is enabled, GLIMS will only select actions (i.e. result outputs / isolation test outputs) that have not yet been downloaded to this station, as indicated by the download status. If the query is an ALL query, GLIMS will also check if these actions have not yet been queried for by another analyzer, as indicated by the Queried by station option. If so, the action will be selected. The last check is not executed in case of a one-specimen query.
  2. If this option is disabled, all actions will always be selected.

Clearing the download status

Manually

You can use the Action/Result contextual menu function Clear download status to clear the download status of an action (and its result outputs / isolation test outputs) for a specified station.

  • Station mark: whenever an action is downloaded to an analyzer, this is stored in the "Download status" of the action (and its result outputs / isolation test outputs. If this option is set, these kind of marks will be cleared. You can clear the mark for all stations or for one specific station.
  • Query flag: when a specific specimen query is received from an analyzer, the "Queried by station" mark is set. If this option is set, these find of marks will be cleared.
  • Download group mark: whenever an action is downloaded to an analyzer belonging to a download group, this is stored in the Download status of the action (and its result outputs / isolation test outputs). If this option is set, these kind of marks will be cleared. You can clear the mark for all download groups or for one specific download group.

Automatically

The option Clear download status upon sampling time updateis available in the GLIMS general settings. When enabled, each update of the sampling time of a specimen will clear the download status of all the actions for which the specimen is the input. This option is disabled by default.

It is the equivalent of manually going over all the actions of a specimen and applying the pop-up menu function Clear download status with all of its options activated.
