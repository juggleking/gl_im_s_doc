Dilutions

Introduction

Only instrument dilution is supported. This assumes that GLIMS can send a dilution factor to the instrument, in which case the instrument takes care of both the dilution and the result conversion.

Notes

  • When diluting a result, GLIMS will not change the current result value. The result is discontinued and a dilution factor is sent to the instrument. The instrument performs the dilution and sends a new result value to GLIMS.

  • When discontinuing a result that is calculated from other results, these other results are also discontinued when they are not requested explicitly and when no other non-discontinued result needs the same result as input.

Dilution codes

For each station, the dilution codes supported by the analyzer must be configured in GLIMS.

  1. Select a Station.
  2. Choose the contextual menu / ribbon item Dilution codes.
  3. Use the Insert key to define a new Dilution code.

Code

Code that is shown in GLIMS screens whenever a dilution code must be selected (see Manual dilution).

Factor

Decimal dilution factor.

Condition

Dilution condition (sometimes called "protocol" or "method", e.g. UP, DOWN, NEAT).

Remote factor

The corresponding remote id to be sent/received to/from the instrument.

Note

In case of dilutions, it is assumed that the outbound id always matches the inbound id.

Remote condition

Remote id for the condition.

Manual dilution

Users can manually indicate that a result should be diluted by:

  1. selecting a Result.
  2. choosing the contextual menu / ribbon item Dilute.

Dilution code id

Select the dilution code to be sent to the instrument.

Dilution reason

Enter a free text or use the Insert key to select a Result-based text module.

The dilution reason can be consulted in the Log tab page of the Result detail screen.

Restrict validation

If enabled, the new result (sent by the instrument after dilution), can only be validated by the user who executed the Dilute function.

Note

A user should have the required privileges for both Result.Validate and Result.RestrictValidation before this option can be used.

Tip

This restricted validation can also be obtained by executing the Restrict validation function on a Result record. From then on, only the current user can validate the result.

In the Order review screen, such results will not be pre-selected and even when selecting them manually, they will not be validated.

Dilution via MISPL

GLIMS can automatically indicate that a result should be diluted by using the Result-based MISPL function Dilute (in e.g. the Property field On result entry).

Syntax

Dilute (DilutionCode DilutionCodeId, String DilutionReason, Logical RestrictValidation)

DilutionCodeId

The record id of the dilution code. To get the record id, use the Result.GetDilutionCode MISPL function.

Example

.Dilute(.GetDilutionCode("ByHalf"), "Result needs dilution", No);

The Dilution code must exist for the Station that assessed the Result.

DilutionReason

The dilution reason can be consulted in the Log tab page of the Result detail screen.

RestrictValidation

This option is intended for manual dilution. In MISPL, we recommend to set this to No.

Default dilution

Some analyzers require a default dilution to be sent regardless whether or not a dilution is requested. To accomplish this, set the Assessment method field Default dilution to the corresponding dilution code.

Diluted results

The requested and effective dilution (which may be different) are stored along with the Result in the fields

  • Requested dilution condition
  • Requested dilution factor
  • Effective dilution condition
  • Effective dilution factor
