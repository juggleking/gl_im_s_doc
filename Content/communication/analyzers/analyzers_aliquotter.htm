Aliquotter station configuration

Basic configuration

The GLIMS internal service supports primary tube queries from aliquotter stations. The configuration for an aliquotter station in GLIMS will be modeled as in the following figure:

As explained in the figure above, an aliquotter procedure must be configured for the aliquotter station. The output of the aliquotter procedure consists of the secondary specimens, which are in turn input of the 'regular' procedures. Note that the final properties must have assessment methods for both the 'assessment stations' and the 'aliquotter station'.

When the aliquotter station does a query for the primary specimen, GLIMS returns the secondary tubes in its reply message. In this message, GLIMS will send the outbound codes as defined in the assessment methods of the aliquotter station. This allows the latter to determine to which stations the secondary tubes must be sent.

Note that GLIMS will reply with a secondary tube as long as one of the corresponding results has a status lower than 'Validated'. For archive tubes, the result status must be 'Validated'.

Download of specimen archive result value to aliquotter station

Definition

Some aliquotter stations want to know if there are secondary tubes used for specimen archiving. The aliquotter station translator will in that case transmit the query specific tag 'ArchiveData' to the internal service to retrieve the archive information.

Configuration

What needs to be configured in Glims? The archive tube is a secondary tube, so the basic configuration as described above is needed.

Moreover, a logical site-attribute must be defined on the table AssessmentMethod (of the aliquotter station !) with name "_IsArchive". The value of this site attribute must be set to true (enabled) on the aliquotter stations' archive assessment method.

When the aliquotter query is received, the internal service will transmit the archive data as a result to the translator. The result should be validated. The outbound id of the assessment method will represent the archive name, the result value will represent the location of the tube in the archive.
