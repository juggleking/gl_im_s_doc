Storing result measurement times from instrument

Description

Glims is capable of storing the following information received in a result message from an instrument:

The time the measurement started:

  1. MIPS ASTM protocol field is R.12.1.1.

  2. GLIMS database field is 'MeasurementStarted'

The time the measurement was completed:

  1. MIPS ASTM protocol field is R.13.1.1.

  2. GLIMS database field is 'MeasurementCompleted'

The result verifier

  1. MIPS ASTM protocol field is R.11.1.2.

  2. In GLIMS, the verifier is shown in the result log, as 'Verified on instrument by...'.

Features

Note that GLIMS also shows the result operator (MIPS ASTM R.11.1.1) in the result log as 'Uploaded by...'.  The user represented in the 'Uploaded by...' will be the operator or verifier, depending if either operator or verifier is available (in that order). Even more, when the operator is available, GLIMS will also add a result log entry 'Performed on instrument by...'

Similar to the field 'store operator id' on the station receiving the results, the field 'store verifier id' enables or disables the storage of the verifier in the result log. When enabled, it also implies that the verifier id received in the result message must be a known GLIMS user with matching login name or initials. If the verifier is unknown, the result will be rejected !

Once result measurement started and/or completed are stored in the database, GLIMS can lookup results based on these values when new result messages arrive. The lookup can be activated on the GLIMS station with the fields 'result start time lookup' and 'result completion time lookup'. If a result is found matching these fields, then that result will be used to update the result value.

Note that for QC results, GLIMS will only lookup results with an object time not older than 2 days (for performance reasons).
