Values

Default is 'NO'.

Logical Value

  • NO

Note

This option can be set by the driver.

MergeCheckList

Syntax

MergeCheckList = <Entry> {',' <Entry>}

The value is a list. Each entry is a keyword, preceded by a '+' or '-'.

<Item> = '+' | '-' <CheckItem>

Values

Enumerated Values CheckItem
  • Relation
  • Sex
  • Mother
  • BeforeBirth

Note

This option can be set by the driver.

Example

Correspondent:MergeCheckList=+Relation,-Mother,+BeforeBirth

AutoMergeByPIN

Syntax

AutoMergeByPIN = <Logical Value>

Values

Default is 'NO'

Logical Value

  • YES
  • NO

Note

This option can be set by the driver. Its default value is No. If set to Yes, persons can be merged automatically using the patient identification list specified in the import message: when more than one patient is found in GLIMS by using the identification codes in the import message, these patients are automatically merged. When a merge is applied, the sort order of the identifications will be respected, i.e. the person identified by the first identification in the list will have the highest priority.

Example

Assume that the import message contains two identifications and GLIMS finds two different patients in the database: when merging these two persons, the first patient found will be the target patient (= contains 'master'-data), the second is considered as source patient.

Each merge operation is logged in GLIMS via the log type "Person Merge".

Example

Correspondent:AutoMergeByPIN=YES
