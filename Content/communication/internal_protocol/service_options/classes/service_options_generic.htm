Generic Server Options

Options

BrokerDebugMode

Syntax

BrokerDebugMode = <Logical Value>

Values

Default is 'No'.

Logical Value
  • No: Deactivate extended logging.
  • Yes: Activate extended logging. Logs every step taken during the server integration process.
Note

The option BrokerDebugMode can only be set in the progress.ini or .glimsrc environment variable file.

Example

BrokerDebugMode=Yes

BrokerActivateMonitoring

Syntax

BrokerActivateMonitoring = <Logical Value>

Values

Default is 'No'.  Please note that this option will be activated automatically when the option "BrokerDebugMode" is enabled.

Logical Value
  • No: Deactivate monitoring in log files.
  • Yes: Activate monitoring in log files.
Note

The option BrokerActivateMonitoring can only be set in the progress.ini or .glimsrc environment variable file.

Example

BrokerActivateMonitoring=Yes

BrokerActivateLeakChecking

Syntax

BrokerActivateLeakChecking = <Logical Value>

Values

Default is 'No'.  Please note that this option will be activated automatically when the option "BrokerDebugMode" is enabled.

Logical Value
  • No: Deactivate leak checking.
  • Yes: Activate leak checking.
Note

The option BrokerActivateLeakChecking can only be set in the progress.ini or .glimsrc environment variable file

Example

BrokerActivateLeakChecking=Yes

ServerTransaction

Syntax

ServerTransaction = <Logical Value>

Attention, this generic option is CLASS specific. You always have to specify for which class you want to activate/deactivate the transaction based processing.

Values

Default is 'No'.

Logical Value
  • No: Activate transaction based processing for the specified class.
  • Yes: Activate extended logging. Logs every step taken during the server integration process.

Example

Order:ServerTransaction=Yes
