Connection with GLP-Systems - Analyzer and test availability messages

Introduction

GLIMS can communicate with GLP-Systems. The GLP system is designed to deliver tubes for diagnosis individually to analyzers.

Analyzer and test availability

Whenever an analyzer is unavailable or cannot execute particular tests, GLIMS can immediately inform the GLP system to avoid that tubes are still transported to that analyzer.

GLIMS can assign a LAS status (Connected / Disconnected) to either an entire station or to a specific assessment method.

Setting the LAS status of an entire station

Manually

To manually indicate that an entire station is connected to or disconnected from the LAS, use the contextual menu / ribbon items

  • Set LAS status connected or
  • Set LAS status disconnected

The LAS status field of the Station status will be set Connected / Disconnected.

Via a station flag

Upon result upload to GLIMS, an analyzer can send station flags. The LAS severity of the Station flag is copied to the LAS severity of the Result.

LAS severity / Station wide event

If the option Station wide event is enabled, the LAS severity of the Result will be compared with the LAS (dis)connect severity limits specified in the Station configuration.

  • If the LAS severity of the Result is above the LAS disconnect severity limit of the Station, the LAS status of the Station status will be set to Disconnected.
  • If the LAS severity of the Result is below the LAS connect severity limit of the Station, the LAS status of the Station status will be set to Connected.

Setting the LAS status of a specific assessment method

When an analyzer can no longer perform a specific test, GLIMS allows to set the LAS status of a specific assessment method to Disconnected.

Manually

Select an assessment method and use the contextual menu / ribbon items Set LAS status connected / Set LAS status Disconnected to set the LAS status of the Assessment method status.

Via a station flag

Upon result upload to GLIMS, an analyzer can send station flags. The LAS severity of the Station flag is copied to the LAS severity of the Result.

LAS severity / Station wide event

If the option Station wide event is disabled, the LAS severity of the Result will be compared with the LAS (dis)connect severity limits specified in the Assessment method configuration.

  • If the LAS severity of the Result is above the LAS disconnect severity limit of the Assessment method, the LAS status of the Assessment method status will be set to Disconnected.

    Note

    If the LAS status of the Assessment method status is set to Disconnected, the Status of the Assessment method will automatically be set to Locked.

    This implies that GLIMS will no longer download this test in work orders to the analyzer.

  • If the LAS severity of the Result is below the LAS connect severity limit of the Assessment method, the LAS status of the Assessment method status will be set to Connected.

    Note

    If the LAS status of the Assessment method status is set to Connected, the Status of the Assessment method will automatically be set to Enabled.

    This implies that GLIMS will download this test in work orders to the analyzer again.

Tip

When changing the LAS status of an assessment method, the LAS status of all other assessment methods of that station having the same Inbound id will be changed as well.

Sending LAS status information to GLP

Introduction

Whenever the LAS status of a Station status or Assessment method status changes from Connected to Disconnected or vice versa, a message is sent to the GLP system.

Note

When linking a station to or unlinking a station from a LAS configuration (i.e. setting or clearing the LAS station field of a Station) directly or via a concentrator,

  • the LAS status of the station and its assessment methods (and of the substations and their assessment methods) is automatically updated, and
  • a message is sent to inform the LAS about the (un)availability of the station and its assessment methods (and of the substations and their assessment methods). Note that the LAS configuration must be such that the LAS can work with a message queue (see Active MQ messaging).

Active MQ

To ensure that the GLP system is informed as soon as possible about a LAS status change, GLIMS uses ActiveMQ to send such notifications.

Read more on how to setup the Active MQ messaging system.

Content of the messages

  • When the LAS status of a Station changes, GLIMS sends the Station in the messages to GLP.

    Note

    When the LAS status of a Station is set to Connected, GLIMS also sends an overview of the LAS statuses of that station's Assessment methods (if their status is NOT Disabled and their LAS status is Disconnected!) in order to avoid that the GLP system assumes that all assessment methods of that station are Connected.

    This means that the assessment method's status is also taken into account when informing the GLP system about the LAS status. Therefore, an assessment method's status should not be manually set to Disabled if its LAS status is still Connected.

  • When the LAS status of an Assessment method changes, GLIMS sends the Station and Test in the messages to GLP.

    Note

    If

    • there is a LAS-status change for an assessment method of a LAS substation,
    • the LAS station itself does not have a corresponding assessment method / assessment method code, i.e. there is no Outbound ID / no Code Out.

    then, GLIMS will not create any message informing about this LAS-status change.

    The reasoning behind this is that, if the LAS is not configured for a certain test, then it is not necessary to send LAS-status change information to it.

Station and test identification

  • Station:
    1. Code in the coding system specified in the Station.CodingSystem of the LAS. Make sure that ALL connected analyzers have a code in this coding system !
    2. If no station code is available in the Station.CodingSystem of the LAS (for none of the connected analyzers), the station's Mnemonic is used.
  • Test:

    1. Code out defined in the assessment method codes browser of the LAS station.

      Tip: The Code out field can easily be filled by means of CTRL-A combined with the context function Copy to code out.

      Warning: As soon as 1 assessment method code is found for the LAS, the Outbound ID of the assessment methods linked to the LAS will no longer be taken into account!

    2. Outbound ID specified in the assessment method linked to the LAS (only if no assessment method codes are used!).

      For all tests that can be executed by the analyzers connected to the GLP system, an Assessment method must also be available for the LAS station.

      Tip: You can use the Assessment method function Copy to copy the assessment methods of the connected analyzers to the LAS station.

Publish laboratory device status overview

In case a communication issue occurred between GLIMS and the GLP-system about the station and test availability, you can synchronize GLIMS and the GLP-systems as follows:

  • Select the station (of type LAS) in GLIMS that represents the GLP system.
  • Choose the contextual menu / ribbon item Publish laboratory device status overview.

The LAS status of all connected Stations and their Assessment methods (unless the assessment method's status is Disabled) will be sent to the GLP system (using the Active MQ messaging system).

Consulting LAS status information in GLIMS

Station browser

The Station browser has a column LAS which indicates the stations' LAS status:

Conn

The station has a LAS connection and is Connected to the LAS.

If the LAS status of all the station's assessment methods is also Connected, the status appears on a green background.

If the LAS status of at least one of the station's assessment methods is Disconnected, the status is displayed on an orange background. Assessment methods that have the status Disabled are not taken into account.

Disc

The station has a LAS connection but is Disconnected from the LAS. The status is displayed on a red background.

?

The station does not have a LAS connection (Station.LASStation is not specified). There is no background colour.

Assessment method browser

The Assessment method browser has a column LAS which indicates the LAS status of the assessment methods:

Conn

The LAS status of the assessment method is Connected. It is displayed on a green background.

Disc

The LAS status of the assessment method is Disconnected. It is displayed on a red background.

?

The station linked to the assessment method does not have a LAS connection (Station.LASStation is not specified). There is no background colour.

 

Note

Note

The colour coding has not been applied to the LAS status values in the browser of the Channel quality monitor.

Station log

Each update of the LAS status of a station (and its assessment methods) is logged in the Station log.
