CyberLab communication -> Resend reports to CyberLab

In certain circumstances, it might be useful to resend a number of reports to CyberLab.   A practical example: in CyberLab, a physician was configured for the wrong department (group).  When sending orders for this physician to CyberLab, the requesting location of the order in CyberLab will be incorrect.  You can assign the physician to the correct group in CyberLab, but this will only apply to new orders being reported from GLIMS to CyberLab.   In order to update the requesting location of the existing orders, you would have to resend the reports from GLIMS to CyberLab.

This is how you can resend a number of reports to GLIMS:

  • Start the report generation program (Start -> Routine -> Report generator)
  • In the Report query, specify:
    • Generator: Online
    • Scope: Order
    • Template: specify the report template that you created for CyberLab reporting.
    • Generated since - until: allows to resend reports that have been generated (sent) within this time range.
  • For more information about the configuration of the report generation screen, please check the chapter 'Result reporting from GLIMS to CyberLab'.
