Creating a property classification for CyberLab

A property classification is used to group and classify a number of properties in a certain sequence.  Among other, it is used in the reporting module.  A property classification must be specified in the report template and will thus define which tests will be reported and in which sequence.  You can create a CyberLab-specific property classification or use an existing one.  Just note that only the tests belonging to the property classification referred in the CyberLab report template will be reported.  For more detailed information, please check the chapter on Property classifications.

Creating a report template for CyberLab

A report template defines the layout and content of a report.  The report templates are directly accessible from the main menu via Start -> Configuration -> Reports - Report templates.  A specific report template must be created for CyberLab.

Important fields:

Main page

  • Property classification: property classification that will be used for reports based on this template.  Only tests belonging to this property classification will be reported to CyberLab.
  • Generator: must be set to 'Online'

Conversion page

  • Coding system: a coding system allows assigning external codes to virtually any template table record in your database.

Microbiology

  • Structured message: must be enabled.

Creating a default report for CyberLab

When an order is entered, GLIMS verifies which reports should be created for this order. This is called the report scheduling.  During report scheduling, GLIMS verifies which default reports are applicable for the created order. 

The default report browser (directly accessible from the main menu via Start -> Configuration -> Reports -> Default reports) displays all default reports for your own organization.  Hit 'Insert' to add a new default report for CyberLab.  The newly created default report will automatically be linked to your own organization (defined in Specific Site).

Below, you will find a typical configuration which could be used for reporting to CyberLab.  Naturally, you can customize the default reports according to you own needs.  For a full description of all fields, please check the chapter on Default reports.

Fields

Main page

  • Scope: must be set to 'Order'
  • Correspondent: will automatically be set to your organization (as specified in Specific site)
  • Active: must be enabled
  • Target type: must be set to 'Default'
  • Explicit target: leave this field blank
  • Eligible: this field allow to specify a MISPL function that will be evaluated during report scheduling (when saving an order).  This MISPL function allows to set criteria that decide if a CyberLab report must be scheduled or not.  

    Examples:

    1. You could write a MISPL expression that checks if the issuer of an order belongs to a CyberLab-specific correspondent group (containing all issuers for which the results should be sent to CyberLab)

    E.g.:    RETURN .Issuer.IsGroupMember("CyberLab");
     
    2. You could write a MISPL expression that checks if a patient is located in a given department.  E.g.  results for patient located in urgency department must be reported to CyberLab.
      
  • Report template: specify the report template that you created for CyberLab reporting
  • Is copy: leave this field disabled
  • Exclusive: leave this field disabled
  • Target specific: must be enabled
  • Add missing pages: leave this field disabled
  • Usage: must be set to 'Standard'

Settings page

  • Previous results: must be set to 'All'
  • Issuer restricted: leave this field blank
  • Additional period: leave this field blank
  • Start date: leave this field blank
  • Report medium: should be set to 'Communication' 
  • Printer: leave this field blank
  • Language: if not specified, the default language will be assumed.
  • Trigger status: should be set to 'Initial'
  • Minimal result status: the actual result values will only be sent to CyberLab if they have reached the minimal result status as defined in this field.  In hospitals, this will typically be 'Confirmed', while private labs can opt for more security (at the cost of an extra delay) by setting this to 'Validated'.
  • Hide cumulative comments: leave this field blank.
  • Partial report hold: must be set to 0.0
  • Report discontinued results: must be enabled.

Configuring a translator for communication with CyberLab

For a full description of services and translator, please check the Service-Translator communication design documentation.

Creating a CyberLab batch user

You need to create a dummy user in GLIMS which has to be specified in the service (see below).  Click Start -> System management -> Security -> Users and create a new user:

  • Initials, last name, first name, login name: free to choose.
  • Type: must be set to 'System manager'.
  • Enabled: This field must be disabled as only batch users can execute services.
  • Autologin allowed: must be enabled.

Creating a service

The services are directly accessible from the main menu via Start -> System management -> Services -> Services.

  • Name: free text
  • Type: must be set to 'Translator server'
  • Login user: specify the CyberLab batch user you created (see above)
  • Port: we advise to specify a free port between 1000 - 20.000

Creating an interface

The interfaces are directly accessible from the main menu via Start -> System management -> Services -> Interfaces.

  • Type: must be set to 'Network'
  • Host: IP address of the CyberLab server
  • Port: Port at which CyberLab listens
  • Time-out: we advise a time-out of 30s.  It is important that this time-out is shorter than the one defined in the translator.

Creating a translator

The translators are directly accessible from the main menu via Start -> System management -> Services -> Translators.  

When communicating with GLIMS, you must define a communication daemon of type "Result daemon with structured, specimen-oriented microbiology (requires Glims!)" in CyberLab !

  • Name: free text
  • Time-out: we advise a time-out of 45s.  It is important that this time-out is longer than the one defined in the interface.
  • Server: specify the service that you created for CyberLab.
  • Interface: specify the interface that you created for CyberLab.
  • Driver: must be set to houl
  • Specific options:  specify the following options: -n 62 -v 2.5 -k NTE,SPM -e SAC -r IERPFX -u R,A,P,S -x InstanceId

Creating a CyberLab correspondent

A CyberLab correspondent (of type institution) must be created which will be specified as Receiver in the report generation screen.

Click Start -> Correspondents -> Institutions -> By Mnemonic and create a new institution.

  • Mnemonic: free-to-choose mnemonic that will be sent as Receiving application in the message header.
  • Name: free-to-choose name

Creating a command/task for report generation to CyberLab

The report generation program consists of a report query and the actual report function.  Both can be automated by means of a command and task.

Command

Commands are available from the main menu via Start -> System management -> Commands -> Commands by description.

  • Table: Report
  • Description: free text
  • Query: Generation
  • Query parameter set: see below for the configuration of the report query
  • Function: Generate
  • Function parameter set: see below for the configuration of the report function

Configuration of the report query

  • Generator: must be set to 'Online'
  • Scope: must be set to 'Order'
  • Template: specify the report template that you created for CyberLab reporting.
  • All the other fields can be left blank.

Configuration of the report function

  • Mark output: must be enabled
  • New results: should be set to 'Ignore'.
  • Output buffering: must be set to 'By Report'
  • Calculate budgets: must be set to 'Never'
  • Translator: translator that will send the HL7 message to CyberLab.  See below for more information about the configuration of a CyberLab translator.
  • PIN provider: correspondent who assigns the patient identifications.
  • HC PIN provider: correspondent who assigns the health care provider codes.
  • Receiver: this correspondent will be sent as receiving application in the message header (MSH 5).

Report items:

The "Report items" button allows indicating which specific data should (not) be exported. For CyberLab this should be configured as follows:

  • The following options should be enabled: Identifications, Encounters, Stays, Copy receivers, classifications and Result details.
  • The isolation details are only supported as of CyberLab v6.2.
  • The other options should be disabled to optimize the performance of the CyberLab reporting.

Task

After having created a command, you should now assign a task to define when the command is to be executed.  Select the CyberLab command in the command browser and choose the contextual menu function 'Tasks'.  Hit the 'Insert' key to create a new task.

  • Table/Command: will be filled automatically
  • From/until: leave these fields blank
  • Recurrence: must be set to 'Instantly'
  • Start - until: enter 00:01 as start time and 23:29 as end time
  • Every x minutes: enter 5
  • Days: leave all options enabled

Scheduler assignments

Introduction

GLIMS allows to link a command to a service of type 'Task scheduler' through 'Scheduler assignments'.  By enabling the specific site option 'Strict scheduling', you can indicate that the 'Task scheduler' service only executes the command assigned through a scheduler assignment.  We strongly recommend to use the strict scheduling mechanism when reporting to CyberLab.  This will avoid reporting delays which are caused by the fact that all services are occupied executing other commands.  We advise to configure a dedicated service which is reserved for the CyberLab reporting command.  Note however that all command will have to be linked to a 'Task scheduler' service when using this mechanism.

Create a service of type 'Task scheduler'

  • Name: free text
  • Type: Task scheduler
  • Login user: specify the CyberLab batch user you created (see above)
  • Port: we advise to specify a free port between 1000 - 20.000

Assign the CyberLab reporting command to the 'Task scheduler' service

Select the CyberLab reporting command in from the Commands browser and choose the contextual menu function 'Scheduler assignments'.  In the 'Task scheduler' field, specify the CyberLab service of type 'Task scheduler' you just created (see above).
