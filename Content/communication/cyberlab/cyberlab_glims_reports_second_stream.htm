CyberLab communication ->  Setting up a second, simultaneous reporting mechanism

Contents: Introduction -  Configuration

Introduction

When upgrading CyberLab to a newer version, this new version may not be taken into routine immediately (e.g. for validation purposes).  So for a given period the two CyberLab versions will be running side by side.  In order to avoid that all orders from within that period have to be sent to the new CyberLab version in one reporting action, you can setup a second reporting mechanism in GLIMS to keep the database of the new CyberLab version up-to-date.

This document merely gives a brief description of the necessary configuration steps.   For more details about each configuration editor, check the chapter 'Result reporting from GLIMS to CyberLab'

Configuration

Report template

  • A new report template must be created.  From the report template browser, you can copy the existing CyberLab report template with the "Copy" function in the contextual "Tools" ribbon tab.  Give it an appropriate mnemonic (e.g. containing the CyberLab version).

Default report

  • A new default report must be created.  From the default report browser, you can copy the existing CyberLab default report with the "Copy" function in the contextual "Tools" ribbon tab..  
  • Enter an appropriate code for this new default report (e.g. containing the CyberLab version).
  • In the new default report, specify the new report template you just created.

Communication setup

  • The existing translator server can be reused
  • Create a new interface.  Specify the port at which the new CyberLab version listens (as defined in CyberLab in ResultDaemon.properties located in %CYBERLABVXX_HOME%\conf).
  • Create a new translator referring to the new interface

Command and task

  • Report query: 
    • Make sure that the 'Generator' field is set to 'Online'
    • In the 'Report template' field, specify the new report template that you created.

What about partial orders ?

  • Orders for which reports have already been scheduled, but which are not yet completed, have to be rescheduled for the new CyberLab version.  

Use the general order query (accessible via Start -> Routine -> Orders -> Query) to obtain a list of all partial orders (Order status range should be set from 'Initial' to 'Partial') which are to be reported to CyberLab (you can use the default report's eligibility MISPL as filter MISPL if necessary).  Make sure that the 'Result status range' is set from 'Initial' to 'Discontinued'.

In the order browser, select all record (CTRL-A) and choose the contextual menu function 'Add report'.  Specify the 'Code' of the new default report.  For more information on this functionality, see 'Adding reports manually or via MISPL'.
