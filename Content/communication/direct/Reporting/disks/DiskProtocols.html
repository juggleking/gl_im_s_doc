Disk reporting

Content: Introduction - Configuration - File layout - Generation - Supported types

Introduction

Glims allows to generate reports in different disk reports.  These disk reports are typically used by general practitioners who can import these disks in dedicated software on their local PC.

Configuration

For each type of disk report, you must define a separate report template.  The configuration of the content of these disk reports can be divided into two major parts:

  1. General info such as lab. order, patient and doctor data: this must be configured with MISPL in the report template fields First page header, Header, Footer, Trailer etc.  The configuration must reflect the official protocol description of the disk report.   This approach allows customers to configure this data to their own specific needs e.g. use a doctor code instead of name.
      
    See the detailed documentation of each disk report type, for more information.
     
  2. Result section:  there are two main concepts for defining the result section of disk reports:
    1. When selecting a specific disk report type (e.g. Medidoc) in the Generator field, Glims will generate a fixed result section.  In this case, you do not need to configure anything manually in GLIMS.
    2. If the fixed result section generated by Glims does not meet your requirements, you can also manually configure the result section with MISPL.  In that case, you must no longer select a specific disk report type (e.g. Medidoc) in the Generator field but choose the option Variable text.   The Body field of the report template should be used to define the content of the result section.

File layout

Each generated disk report has the following file structure:

<FirstPageHeader> e.g. lab information, defined via MISPL
{
    <PageHeader> e.g. order information, defined via MISPL
    {
        <Result Section> e.g. result information (fixed or defined via MISPL)
    }
    <Footer> e.g. order information trailing data, defined via MISPL
}
<Trailer> e.g. message trailing data, defined via MISPL

{} : repeatable

Generation

The configuration of the report generation screen varies according to the type of disk report.  Please check the detailed documentation of each disk report type, for more information (see below for a list of supported types).

Supported disk report types

  • Health One
  • Medicard
  • Medidoc
  • Medigest
  • Socrates
  • Windoc: can be generated through the "wdoc" driver (or via variable text reporting)
