Edifact v1.0 - MEDVRI

Introduction

GLIMS supports electronic reporting according to the Edifact medvri 1.0 specifications.  This can be accomplished by using the ReportTemplate.Generator type 'EDIMDV1_0' (Edifact Medvri 1.0).

The contents of the TXT segments can be built using the ReportTemplate.Body MISPL expression.  Each generated TXT segment data is max 70 chars long. If the body generates a line larger then 70 chars, it will be put into multiple TXT segments.

E.g. just an example with a simple body MISPL for non-microbiology results and without section labels:

{: if .Type = StringToEnumerated("ObservationalResultType", "Result")
	then return
             "Test=" + .Mnemonic +
  	     ", Value="	+ ifknownstring(.Value)	+
 	     ", Unit=" + ifknownstring(.UnitName)	+
	     ",Ref=" + ifknownstring(.Reference);
	endif;

return "";
}
