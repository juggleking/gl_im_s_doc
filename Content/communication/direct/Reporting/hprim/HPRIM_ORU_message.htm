Generating reports in HPRIM format

Introduction

Glims allows to generate result reports directly in HPRIM v1.2 and HPRIM v2.1 format (ORU files).  This can be done by creating a dedicated report template and setting the "Generator" field to "HPRIM1_2".

Click here for a full description of an example implementation in which Glims generates HPRIM-formatted files that can be processed by an external system called "Hexaflux".

GLIMS will always report all results and their corresponding status of an order. The external system always gets the most recent report of the complete order.  The Glims reporting program will create HPRIM/ORU files on a dedicated directory. It is the responsibility of the external system to retrieve the generated files from the GLIMS system.  In general, rcp or ftp is used to transfer the files from one system to the other.

Message layout and segment description of the HPRIM ORU files

H  (Message header segment)
    P (Patient identification segment)
        OBR (Observation request segment)
            {OBX} (Observation result segment)
L (Message trailer segment)
