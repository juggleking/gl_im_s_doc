HL7: OBX - observation/result segment

The OBX segment is used to transmit a single observation or observation fragment. It represents the smallest indivisible unit of a report.

Its principal mission is to carry information about observations in report messages. But the OBX can also be part of an observation order. In this case, the OBX carries clinical information needed by the filler to interpret the observation the filler makes. For example, an OBX is needed to report the inspired oxygen on an order for a blood oxygen to a blood gas lab, or to report the menstrual phase information which should be included on an order for a pap smear to a cytology lab. OBX is also found in other HL7 messages that need to include patient clinical information.

2. Value Type

  • Export:
    • NTE segment use enabled
      • Depending of the Result Datatype - check table -.
    • NTE segment use disabled
      • Result ExternalComment present or Request ExternalComment present or Result Value contains newlines: 'FT'.
      • Otherwise: Depending of the Result Property Datatype.

    Supported Datatypes:

    • Numeric: 'NM'
    • String: 'ST'
    • Image: 'RP'
    • Graph: 'ED'
    • Structured Numeric: 'SN' - from HL7 version 2.3 and +.
    • All other types: 'FT'

In case of 'Online' reporting with HL7 translation, the value type can be "RP", indicating that the actual result value is an image (path of the file containing the image) (RP = Reference pointer: The RP value must be used if the actual observation value is not sent in OBX but exists somewhere else.)  The online report generator will report the absolute path of the specified Image file.  At least if the program is able to determine the full path.

  • If the result.rawvalue contains an absolute path, then that path is returned. (independent whether the program can locate that file)
  • If the result.rawvalue contains a relative path started from the root directory which is specified by the ImageServerDir environment variable, then the absolute path is returned.
  • If the result.rawvalue contains a relative path started from one of the directories specified in the propath, then the absolute path is returned.
  • If the result.rawvalue contains a relative path which can't be found on the server on which the report program is running, then the relative path is returned.
  • Import: -

4. SubId

  • Export:
    When a test is requested multiple times for a specific order (at different object times), a discriminator value will be put into OBX.4.  Note that this applies for
    all tests, not only the tests for which DynamicType <> None.  If a non-dynamic test  is only requested once, no discriminator will be put into OBX.4.
    e.g. 00:30 or 01:00, ...

Note:  if a test is entered, then discontinued and then re-requested at the same object time, no discriminator will be added for either of the tests.

5. Result Value

  • Export:
    • Result export disallowed (Result Status < minimal allowed Result Status) :
      ReportTemplate::EmptyResultText. e.g. 'volgt', 'follows', ... or empty if no EmptyResultText defined.
    • "RejectionReason" when 
      1. The result is discontinued
      2. The report has already been marked as outputted
      3. The option "Report discontinued results" of the corresponding default report is enabled.
           
    • Result export allowed (Result Status >= minimal allowed Result Status):
      • NTE segment use enabled
        • Result Value
      • NTE segment use disabled
        • Result ExternalComment present:
          Value ::= Result Value + <BR> + Result ExternalComment
        • Request ExternalComment present:
          Value ::= Result Value + <BR> + Request ExternalComment
        • Result ExternalComment and Request ExternalComment present :
          Value ::= Result Value + <BR> + Result ExternalComment <BR> Request ExternalComment
        • No Result ExternalComment :
          Result Value

      In case of 'Online' reporting with HL7 translation, the value type can be "RP", indicating that the actual result value is an image.  The result value will then be the path of the file containing the image.  

      The online report generator will report the absolute path of the specified Image file.  At least if the program is able to determine the full path.

      • If the result.rawvalue contains an absolute path, then that path is returned. (independent whether the program can locate that file)
      • If the result.rawvalue contains a relative path started from the root directory which is specified by the ImageServerDir environment variable, then the absolute path is returned.
      • If the result.rawvalue contains a relative path started from one of the directories specified in the propath, then the absolute path is returned.
      • If the result.rawvalue contains a relative path which can't be found on the server on which the report program is running, then the relative path is returned.

      In case of 'Online' reporting with HL7 translation, GLIMS also supports results for graph properties:

      OBX.5.1.2 = "Graph"
      OBX.5.1.3 = "MPGF"
      OBX.5.1.5 = <XML Format of the Graph>.

8. Observation Flags

  • Export:
    • Result Severity 0 : Flags = "N" in case of alpha-numeric result value.  In case of numeric value, nothing is sent.
    • Result Severity <> 0 :
      • Result DeltaNormSeverity > 0 : Flags = "U"
      • Result DeltaNormSeverity < 0 : Flags = "D"
    • Result PropertyTypeNumeric
      • Numeric Result present :
        • NormSeverity > 0 :
          • Flags empty: Flags = "H"
          • Flags not empty: Flags = Flags + "H" (as 2nd repeat)
        • NormSeverity < 0 :
          • Flags empty: Flags = "L"
          • Flags not empty: Flags = Flags + "L" (as 2nd repeat)
      • Conversion to numeric failed :
        • Flags empty: Flags = "A"
        • Flags not empty: Flags = Flags + "A" (as 2nd repeat)
  • Import: -

Remark for export

For all drivers supporting online URL report generation (e.g. hl7.exe, xcare.exe, ...), GLIMS allows configuring the value for OBX.8 via the ReportTemplate.ResultMark field.  This can be done by specifying a dynamic text with the following format:

NormSeverity=<Value>\DeltaNormSeverity=<Value>

Example:

NormSeverity=HH\DeltaNormSeverity=U

would result in an OBX segment with: 

'OBX|1|....|U~HH|...'

meaning that the result is very high and is a significant change up compared with the previously sampled result value.

MISPL example

{: String List; List := ""; IF .Severity <> ? AND .Severity <> 0 THEN IF ((.Property.DataType = StringToEnumerated("PropertyDatatype", "Numeric") OR .Property.DataType = StringToEnumerated("PropertyDatatype", "Titer")) AND (.Mantissa <> ? OR .RawValue = ?)) THEN IF .NormSeverity <> ? AND .NormSeverity <> 0 THEN List := List + "NormSeverity="; IF .NormSeverity > 0 THEN IF .NormSeverity = 1 THEN List := List + "H"; ELSE List := List + "HH"; ENDIF; ELSE IF .NormSeverity = -1 THEN List := List + "L"; ELSE List := List + "LL"; ENDIF; ENDIF; List := List + "\"; ENDIF; ELSE IF .NormSeverity <> ? and .NormSeverity <> 0 THEN List := List + "NormSeverity="; IF .NormSeverity = 0 THEN List := List + "N"; ELSE IF Abs(.NormSeverity) = 1 THEN List := List + "A"; ELSE List := List + "AA"; ENDIF; ENDIF; List := List + "\"; ENDIF; ENDIF; IF .DeltaNormSeverity <> ? and .DeltaNormSeverity <> 0 THEN List := List + "DeltaNormSeverity="; IF .DeltaNormSeverity > 0 THEN IF .DeltaNormSeverity = 1 THEN List := List + "U"; ELSE List := List + "UU"; ENDIF; ELSE IF .DeltaNormSeverity = -1 THEN List := List + "D"; ELSE List := List + "DD"; ENDIF; ENDIF; List := List + "\"; ENDIF; ENDIF; RETURN List; }

    • 'D'. 'X' : Discontinued
