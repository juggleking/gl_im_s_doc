HL7: PV1 - patient visit segment

The PV1 segment is used by Registration/Patient Administration applications to communicate information on a visit-specific basis. This segment can be used to send multiple-visit statistic records to the same patient account or single-visit records to more than one account. Individual sites must determine the use for this segment.

1. Set Id

  • Export:
    PV1 segment counter
  • Import:
    Related Stay ExternalId

2. Patient Class

Configurable GLIMS specific Encounter Type Code.
e.g. 'I', 'O', ...

3. Patient location

  • Export:
    • 3.1: Ward Identification Code or Ward Mnemonic if the Code is not found
    • 3.2: Current Stay Room
  • Import:
    • 3.1: Ward Identification Code or Ward Mnemonic if the Code is not found
    • 3.2: Stay Room
    • 3.4: will be stored in the "Specialism" field of the Stay table.  This will be the case if the patient location nurse unit or ward (PV1.3.1) is specified.

7. Attending doctor (HCProvider)

  • Export:
    • 7.1 : Encounter Physician Identification Code or its InternalId if no Code was found
    • 7.2 : Encounter Physician LastName
    • 7.3 : Encounter Physician FirstName
    • 7.4 : Encounter Physician MiddleName
    • 7.6 : Encounter Physician Prefix
    • 7.7 : Encounter Physician Title
  • Import:
    • 7.1 : Encounter Physician identifier
      (Looked up as Identification code or via InternalId if not found)
    • 7.2 : Encounter Physician LastName

19. Encounter External id

  • PV1.19.1.1.1 = patient visit number

  • PV1.19.1.4.1 = Identification code of institution assigning the visit number.
    If the identification code is not known then the program looks up the institution by internal id.  If still not known then an error is reported and the default institution is taken as assigning institution.  so, if PV1.19.1.4.1 is filled in and valid, we will assign this institution to the created encounter.

24. Contract Code

Configurable GLIMS specific Contract Code.  The translator processes the contract information specified in the PV1.24, PV1.25 and PV1.27 fields.
