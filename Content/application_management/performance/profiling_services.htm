Profiling services

Contents: Introduction - Start the profiler - Stop the profiler

Introduction

The profiling tool allows profiling running GLIMS 'Translator servers' and 'Task schedulers' and generating a Progress '.out' profile info file.  The creation of the '.out' file is done at run-time, so the service does not have to be stopped and restarted.  The '.out' file can be loaded in the Progress profile info viewer for further analysis (available from within GLIMS through the main menu via Start -> Development -> Profiler -> View sessions).

Warning

This profiling tool should only be used by MIPS employees.

Start the profiler

From the gp_Service table, you can use the contextual menu functions 'Start profiler'.  When doing so, the service will gather profile information.  

What does it do:

  • Stops all currently running persistent servers
  • Clears '-q' cache
  • Activates the Progress profiler
  • When receiving the next message(s), the needed persistent server(s) will be restarted with progress profiling active. 

Options:

  • Scope
    • Session:  one profiling info file will be generated for all messages received/tasks ran.  
      File format:  * {logfilename_service}_{Date}_{Time}_S{Counter}.out
    • Detail: the MinimalProcessingTime option will be taken into account.
      File format: * {logfilename_service}_{Date}_{Time}_D{Counter}.out
  • Minimal processing time
  • 0 or ?: one profiling info file will be generated per message received/task ran.
      
  • >0 : one profiling info file will be generated per message received/task ran.  IF the message/task processing time is longer than this parameter itself
    (e.g. 5 seconds -> when message/task processing exceeds 5 seconds, then and only then a profiling info file will be generated).
  • Load/Store buttons: not supported

Stop the profiler

From the gp_Service table, you can use the contextual menu functions 'StopProfiler'.  When doing so, a '.out' file will be saved in the directory of the log file.

What does it do:

  • Stops all currently running persistent servers
  • Clears '-q' cache
  • Deactivates the Progress profiler
