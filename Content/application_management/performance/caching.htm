Menu, privilege and schema caching

Menu caching

In order to speed-up GLIMS as a client/server application, an option has been provided to aggressively cache the menu definitions on the clients. This minimizes database access for menu-building.

Enabling/Disabling menu caching can be done by enabling/disabling the general option Cache Menus (System Management-> Customize -> MATE). When you are modifying the layout of your menus, disable menu caching to see the last updated versions of your menus. At all other times, enable menu caching to improve performance. In general, cache-enabled is recommended.

Privilege caching

Caching privileges is implemented in the same way as it was done for menus.

Schema caching

Schema caching is standard and cannot be turned off.
