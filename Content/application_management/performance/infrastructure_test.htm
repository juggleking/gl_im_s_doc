Infrastructure test

The Infrastructure test tool measures the performance of some basic database operations.

It can be started via

  • Start -> System management -> Tools -> Infrastructure test -> Run Infrastructure test
  • if a command called "Infrastructure test" has been defined beforehand: Start -> System management -> Commands -> Commands -> By description -> search Infrastructure test -> Do

Features

The Infrastructure test program

  • offers tests on synthetic test data (read, write, delete) and indicates the response time of the server.
  • offers both sequential and random read tests.
  • takes the influence of the buffer cache into account.
  • stores the test results in the database.
  • can be run in batch on a regular basis after the creation of a scheduled task in GLIMS.
  • allows uploading the results to a web service, so that they can be compared to those of other sites.

Consulting the test results

Test results are kept in GLIMS for 90 days. They can be consulted via Start -> System management -> Tools -> Infrastructure test -> Infrastructure test history records. The history enables you to

  • get a global idea of the current performance of your system,
  • troubleshoot performance issues,
  • observe performance trends over time and compare with data from other sites.

Logging

After each run of the Infrastructure test, a log file is created in GLIMS. Its name looks as follows: InfrastructureTest.yyyyMMddHHmmss.log. This log file contains the results of the performed tests and, if configured, the URL to which the results will be uploaded.

Log files older than 30 days are purged.
