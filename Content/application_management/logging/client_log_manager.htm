Clientlog manager

Introduction

To obtain detailed information about a specific action performed in GLIMS, you can activate Client logging. This is typically used to provide MIPS with detailed logging about a detected issue.

Warning

Client logging should only be activated while reproducing a specific scenario and should be deactivated afterward.

Activate client logging

  1. Login with a Role of which the User type is set to User, System manager or Developer.

  2. In the upper right corner of your GLIMS screen, click the Expert settings icon and choose Clientlog manager.

  3. Specify where the client Log file should be saved.

  4. The other options allow to indicate which actions should be logged. For most cases, it should be sufficient to enable the 4GL Trace option with Level set to 4.

  5. Click the Start button.
  6. Perform the actions in GLIMS of which you want to log detailed information.
  7. Click the Stop button.
  8. All details will be saved in the specified log file.
