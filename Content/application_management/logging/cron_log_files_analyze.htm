Analyze cron log files

This utility makes it possible to analyze the log files of the GLIMS CRON jobs. The program can be accessed through the main menu via Start > System management > Activity > Analyze Task Scheduler Log Files.

When starting the program, an empty browser will initially be shown. You can add files using the Add and Add group buttons. The Add button allows to manually select one or more cron log files. A list of files matching a pattern may be added using the Add group button. Files can also be added to the browser using "drag and drop".

The browser shows the start and end time of each job, how much database access was caused by it, disk reads, hit ratio and the duration. Pressing the column label in the browser sorts the tasks accordingly. This makes it easy to figure out which task caused the most system/database overhead.

  1. Column "O": if checked, the task is Overdue. The scheduled task started 3 minutes or more later than planned because the cron was busy with another task.

    Too many overdue tasks => schedule tasks on other crons.

  2. Column "C": if checked, Contention has been detected. The scheduled task started at least 10 seconds too late because the cron was busy with another task.

    Too much contention => provide more time for the task execution.

The Clear button empties the browser. The Summary button shows an overview with totals and mean values per task.
