Logging of session reason and menu choices

Logging of session reason

When users login, GLIMS can ask for the reason as to why this "session" is started.  The entered information is stored in a log type called "Maintenance sessions".   This feature has been added to enhance ISO-15189 support.

To activate this feature, the option "Log maintenance session" must be enabled in the "Role" definition.  Note that this option is only available for roles with user type "System manager" or "Developer".

Logging of menu choices

GLIMS allows to track all menu items selected by users.

This feature should only be activated in close consultation with MIPS !!

To activate this feature, the general option "Keep action log days" should be specified.  This option (expressed in days) indicates  how long this log information needs to be kept.  

The information will be stored in the "gp_UserAction" table.
