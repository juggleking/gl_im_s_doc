Export configuration data for SampleNet

Contents: Introduction - Export details - Remarks

Introduction

The menu option System Management -> Database -> Specific export -> SampleNet export allows exporting a lot of configuration tables at once in a format suitable for import in SampleNet (MIPS instrument manager).  The program will create a number of .csv and one samplenet.txt file on your temporary files directory.

To import this configuration data in SampleNet, start SampleNet and choose Start -> System management - Database -> Import.  In the import screen, enable the option "Use script" and select the export file "samplenet.text" as script file.

Remarks

  1. If a Station has assessment methods, their inbound id's will be stored in a coding set with mnemonic = "M_" + station mnemonic and the instrument will refer to this coding system
  2. Since the base tables are different, the 'Eligible', 'Comment' and 'Trigger' fields will not be converted for the moment
  3. The seq no passed for tests will be based on the default property classification in Glims and will start from 90000
  4. Property.DataType will be converted to Test.DataType
  5. Physician.Address will contain entire Correspondent address
  6. For each Glims RequestCode, only 1 non-issuer specific request definition will be converted, giving precedence to property request definitions over panel request definitions
  7. All character fields will be cut to meet SampleNet length restrictions if necessary
