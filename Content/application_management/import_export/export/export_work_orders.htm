Work order export

Contents: Introduction -  Work order selection - Export

Introduction

In case certain tests need to be executed in an external lab, GLIMS offers an export program for electronic work order communication with an external LIS.

Work order selection

GLIMS offers several ways to select what has to be sent out to the external lab.

Action-based

Work list export

Use the work list query to select a number of work lists and choose the contextual function Export orders. This process can be automated via a command and task.

Action export

Create a command with base table Action and select the Export function.

Order-based

Contextual function on Order

You can also add a contextual function on the Order table to export orders:

  1. Choose System management -> Scheme -> Tables.
  2. Select the Order table and choose the contextual function Menu -> Edit.
  3. Insert a new menu option with function RunProcedure, create a new function parameter set and choose the function Export in the setup screen.

Export

Each of the export functions described above share the following output options:

Mark export

If set, the field Sent to station of the result outputs of the exported actions will be enabled. The marked result outputs will not be exported again.

Add derived actions

If enabled, derived actions used for e.g. calculations will also be exported.

Check if exported

If enabled, GLIMS will only export actions that have never been exported before.

Check action status

If enabled, GLIMS will only export actions with status Outputting or Partial.

Check work list template items

If enabled, GLIMS will check if the action's result outputs are defined as work list template items (property outputs). Only applies when using the work list export function.

Output format / Destination

Online

An internal XML message will be generated. Destination should be set to Translator. Specify a Translator to translate this XML message for external communication.

Mipsys

A Mipsys formatted file will be written to disk. Destination should be set to File (a file specification window will be shown).

Coding system

Property, unit, micro-organism, antibiotic code sets for export.

Translator

The only translator which is currently able to translate the order export XML message is 'ldt.exe'. The translator will export an LDT message of type 8218.

PIN provider

Patient PIN provider.

HC PIN provider

Health Care PIN provider.

Sender

Correspondent who sends the order export data stream (used as "Sending application" in HL7 message).

Receiver

Correspondent who will receive the order export data stream.

Export items

This button allows to indicate which data should (not) be exported. This allows to speeding up the order export by excluding information that will never be used by the receiving translator. Items that can be included/excluded:

  • Site attributes
  • Identifications
  • Encounters
  • Stays
  • Payment agreements
  • Studies
  • Referrals
  • Medical records
  • Object attributes
  • Informational results
  • Procedure data results
  • Encounter details
  • Order details
  • Request details
  • Specimen details
  • Report copy receivers
