Export configuration data for CyberLab - Municipalities and postal codes

To export municipalities and postal codes, the option Municipalities of the CyberLab export window should be activated.

Note

CyberLab expects the country to be defined as an ISO two letter-code.

  • GLIMS will try to match the country code, as defined in the Country table, with the ISO 3166 two-letter or three-letter code (see http://en.wikipedia.org/wiki/ISO_3166).
  • If this fails, then GLIMS will try to match the ISO 3166-1 English short name of the country with a translation of the country, as defined in GLIMS.
  • If the GLIMS Country code cannot be mapped with an ISO code, then an error will be shown and the municipality export will fail.

It is recommended to use the ISO two-letter or three-letter code as country code in GLIMS.

Output file: "municipalities.xml".
