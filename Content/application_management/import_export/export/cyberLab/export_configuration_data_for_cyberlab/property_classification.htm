Export configuration data for CyberLab - Property classifications

To export a property classification, the following fields of the CyberLab export window can be used:

  • Classification: select the property classification to be exported.
  • Coding system: optional coding system for export of properties. If not provided or if a code is missing, the property mnemonic will be exported.
  • Export node label: if set, the property classification node's label will be exported instead of the property description.

Output file: "PropertyClassification.xml"

Note

Note

In the .xml file, each node and property is uniquely identified by an id composed of the primary key and creation time of the record. This ensures that existing nodes in CyberLab are updated in case of changes of configuration data (such as a reorganisation of properties or classification nodes) in GLIMS.
