Export configuration data for CyberLab - Funds and policy names

To export funds and policy names, the option Export funds of the CyberLab export window should be enabled.

Output file: "Funds.xml". This file contains:

  • a list of funds
  • a section with policy information
  • a list of policy names
  • a section with policy clause information

Fund section

This section contains the following fields, among others:

  • priceList code: optional, used when the policy clause refers to the fund price list.

  • policyName code: optional, for automatic selection of the default price list upon creation of a new payment agreement entry.

Note

Note

For the funds, the coding system is not taken into account.

Policy information section

This section contains the following fields:

  • code: used as a unique identifier
  • descriptions: default ("EN") description of the policy
  • checkBillingRules: defines whether or not billing rules should be applied for the policy.

PolicyName section

This section contains the following fields, among others:

  • code: used as a unique identifier
  • descriptions: optional, default ("EN") description of the policy name. If the policy name has no description, the policy description is indicated instead.
  • policy code: refers to a policy

Policy clause information section

This section contains the following fields:

  • label: can be empty
  • descriptions: optional, default ("EN") description of the policy clause
  • policy code : refers to a policy
  • seqNo
  • residenceType

  • party: optional, includes the following elements:

    • reimburserType: optional
    • payerType: optional

    • specificInstance: optional, includes the following elements:

      • type
      • code
  • toCharge
  • multiplier: optional
  • fixedAmount: optional
  • priceListType: optional
  • priceList code: optional
  • reimbursementClass: optional
