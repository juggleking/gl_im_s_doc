Export configuration data for CyberLab - Introduction

Introduction

An export tool allows the synchronization of GLIMS configuration data with CyberLab. It is available in GLIMS under Start -> System management -> Database -> CyberLab export.

This tool supports the export of the following data in an XML file:

  • property classifications
  • issuers and group practices
  • funds and policy names
  • request codes and request forms
  • municipalities and postal codes
  • billing rules

The export files can be automatically imported from GLIMS in CyberLab. The procedure is described in your CyberLab manual.

Warnings

Warnings

  • This export function requires a separate license: GLIMS CyberLab synchronization.
  • The full synchronization functionality as described here is only supported for the communication between GLIMS 9.6.3 (and higher) and CyberLab 9.11 (and higher).

Target directory

The field Target directory allows you to indicate the operating system folder in which the export files are to be saved.

If the export directory already contains export files, a confirmation request will be displayed before they are overwritten.

Manual or batch export

The function gp_Site.ExportCyberLab(), with standard menu path System management -> Database -> CyberLab export, is batch-enabled. Hence, it can either be launched manually or configured as a batch task providing periodic export.

Backward compatibility

The field CyberLab version allows backward compatibility with former versions of CyberLab. If you choose the option lower than v9.11, the export will run in compatibility mode. This means that certain data, such as non-explicit and non-active request definitions, will not be exported to CyberLab.
