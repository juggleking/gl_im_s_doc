Specific update

A program is available to support new HLA nomenclature. This tool should only be used after consulting a MIPS employee and should first be tested thoroughly in a test environment !!

The tool allows to update the main keys of the supported tables (initially only for HLAAntigen.Name, but it has been extended to other tables). The tool can be added to the main menu with function "RunProcedure" and parameter "SpecificUpdate".

The user interface will present a list of tables to choose from. The field that can be changed with this function is automatically selected and displayed.

The program is based on the generic import program, regarding display of progress bar, number of records processed, rejected and created. As for generic import, the output file is based on the input file with addition of an extra "csv" extension.

The input file, typically called "TableShortName".csv should have two columns, the first one containing the old values, the second column the new values. The input file may contain a header, but unlike generic import, it is completely ignored.

For each updated record the following validations are done:

  1. each old value should exist in the database

  2. each new value should be specified (non-null value)

  3. if the field is a mnemonic, it should not contain special characters

  4. the value length should not exceed the maximum allowed field length

  5. the new value should not already exist in the database (unique)

The operation is done in 1 transaction only. If one record fails, the whole transaction will be reverted. The user should correct the input file, taking into account info from the output file and retry, until all records can be updated.
