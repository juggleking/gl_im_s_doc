Microorganism codes import

Contents: Prerequisites - Import program - Logging - Example

Prerequisites

The import file should comply with the following rules:

  1. The character used as delimiter between field values in the file is a space.
  2. All character fields must be enclosed in quotes (" ") if they contain any delimiter characters (i.e. spaces).
  3. If you want to import any quotes contained in the data, replace them by two consecutive quotes ("").
  4. If you do not wish to specify a field, you should put an unquoted question mark (?).
  5. Specify numbers with decimal points (no commas).
  6. The import file must always end with a new line.
  7. Each record must meet the following format:
Name Mandatory Contents
Code Yes Micro-organism code
Description Yes Micro-organism description

Import program

The micro-organism code import program can be started from the main menu via Start -> System management -> Database -> Specific import -> Micro-organism codes.

  • File name: name of the import file. You can use the FIND key (TTY) or double-click (PC) to browse through your file system.
  • Character set: If the import file character set doesn’t match the GLIMS character set, you can specify an alternate one. The name that you specify must be a valid code page available in the $DLC/convmap.cp file (a binary file that contains all of the tables that Progress uses for character management). For more information please refer to the Progress ‘System Administration Guide’ that was provided along with your Progress license.
  • Coding system: specify the coding system for which micro-organism codes should be imported. The coding system should have a code set for the table Microorganism
  • Simulate: if set, the import program will not update the coding system but only log messages to file.

Note about finding a record by name: the import program will first search for a perfect matching name. If not found, the import program will try to compare only the words (words are composed of characters a-z, 0-9) of the name.

Logging

For each import, 2 files are generated based on the name of the import file: One with the extension .rej and one with extension '.log'. The .rej file will contain all the import lines from the import file that are not imported. The .log file will contain the line number, import code, import name and a log message.

Log messages:

  • Both code and name required for import: Check the import file
  • Code/name already exists: The import code already exists for a record with the same name as the import name
  • Code exists with different name: The import code already exists but the name of the record differs from the import name. When import is not in simulation mode, the code will be made obsolete (prefixed with the '_' character)
  • No code nor name found: The import code does not exist and no record was found with a matching name
  • No code/name found but mnemonic found: The import code does not exist and no record was found with a matching name. However, a record was found with a mnemonic matching the import code.
  • Created code: The code does not exist but a record was found with a name matching the import name and which has no code record yet. Not in simulation mode (message is 'Code would be created')
  • Updated code: The code does not exist but a record was found with a name matching the import name and which already has a code record. The code record value is updated to the import code. Not in simulation mode (message is 'Code would be updated')
