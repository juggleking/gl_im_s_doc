Import HLA data of patients

Introduction

After having selected the file to be imported, GLIMS will ask if existing data can be replaced by the imported data.

Prerequisites

The file format must be:

  • The import file must always end with a new line.
  • columns separated by '|'
  • all dates: format DD/MM/YYYY
  • empty fields should be left blank
  • each line contains 5 columns (all optional except first):
  1. PIN code of the patient (mandatory): PIN codes are those provided by the PIN code provider that you defined in the order entry screen -> options -> advanced.
  2. HLA type (serological method)
  3. HLA type (DNA method)
  4. HLA antigen list: comma-separated list of antigen names, preceded by '-' sign if absent
  5. HLA antibodylist: comma-separated list of antibody names + blank + date, name is preceded by '-' sign if antibody was absent on that date.

Activation

  • System management -> Import -> HLA data of patients
  • A file dialog will open: select your input file
  • The program runs and shows its output on screen.

Remark: Each person for which HLA data is available will be marked as an 'Object' in Glims.
