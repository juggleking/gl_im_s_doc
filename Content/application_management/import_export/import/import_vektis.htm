Vektis import (Dutch customers only)

Het Vektis import programma kan gestart worden via het hoofdmenu (Start -> Systeembeheer -> Importeren -> Vektis). Het volgende scherm zal getoond worden:

  • Bestandsnaam: dubbelklik of gebruik CTRL-F om het gewenste importbestand te selecteren. Enkel de bestanden FAGBX020 en FAGBX030 (mutaties) bevatten relevante informatie voor GLIMS (actuele NAW en algemene gegevens van zorgverleners).
  • Datum formaat: vanaf 01/01/1998 bevatten Vektis datums de eeuwaanduiding (DDMMEEJJ, "E" staat voor Eeuw). In de user-interface moet het juiste formaat aangegeven worden omdat dit niet automatisch kan gedetecteerd worden.
  • GZ code verstrekker: selecteer de correspondent (via type, interne id of naam) waaraan de arts identificaties gekoppeld zullen worden.

    Als standaard wordt de GZ code verstrekker uit de specifieke instellingen voorgesteld (Systeembeheer > Aanpassen > GLIMS), of bij afwezigheid de VEKTIS organisatie. Indien in het laatste geval de VEKTIS organisatie nog niet bestaat, wordt er voorgesteld deze automatisch aan te maken.

    Alle geïmporteerde artsen krijgen een identificatie voor deze organisatie, nl. de Vektis code. Indien de arts nog niet bestaat, dan wordt die aangemaakt.

    Opgepast: indien u het Vektis import programma reeds gebruikte vòòr dit veld beschikbaar was en nu deze optie gebruikt, ontstaat het risico dat alle artsen dubbel aangemaakt worden (omdat er geen identificatie record voor de opgegeven GZ code verstrekker zal gevonden worden). Daarom zal het programma bij detectie van een zogenaamd 'nieuwe' arts steeds controleren als er misschien een identieke arts-code bestaat gekoppeld aan de "VEKTIS"-organisatie. Desgevallend zal er verondersteld worden dat deze arts toch reeds in de database aanwezig is.

    Om compatibiliteitsredenen worden deze 'oude' VEKTIS-identificaties ongemoeid gelaten. We raden echter ten stelligste aan deze na de import manueel te verwijderen indien niet meer nodig. Indien om een of andere reden behoefte mocht ontstaan om de Vektis-codes van de artsen aan meerdere GZ code verstrekkers te koppelen, dan kan dit door hetzelfde bestand meerdere keren in te lezen. De optie "Vektis code overnemen als arts code ?" moet dan wel aangevinkt zijn, omdat dit de enige mogelijkheid is om te weten als een arts reeds in de database aanwezig is. Indien dit niet gebeurt worden alle artsen dubbel aangemaakt !!

  • Vektis code overnemen als arts code: indien aangevinkt, dan wordt de Vektis code (die normaal wordt opgeslagen in Identification.Code) doorgekopieerd naar HCProvider.Mnemonic en HCProvider.Code.

Opmerkingen

  • De opties (bestandsnaam, tekenset, datumformaat,...) worden opgeslagen in de gebruiker-specifieke voorkeuren (gp_Preference).
  • Zorgverlenernummers zijn niet uniek doorheen de verschillende Vektis tabellen. Hetzelfde nummer kan bv. zowel in FAGBX20.s01 (huisartsen), FAGBX20.s03 (specialisten) als in FAGBX20.s08 (verloskundigen) voorkomen. Daarom gebruikt het Vektis import programma als unieke sleutel zorgverlenersoort + zorgverlenernummer (positie 4 t.e.m. 11), in plaats van enkel zorgverlenernummer (positie 6 t.e.m. 11).
