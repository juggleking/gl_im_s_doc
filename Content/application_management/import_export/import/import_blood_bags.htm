Blood bag import

Introduction

The program for importing transfused blood bags is available from the main menu via Start -> System management -> Database -> Specific import -> Blood bags (administered). A window will be opened that allow to select the import file and specify a coding system.

  • Import file: select the file to be imported
  • Coding system: you can use a coding system to map the external identifications with the Glims identifications for:
    • HC Provider mnemonic
    • Blood product specification mnemonic
    • Department mnemonic
    • Antigen mnemonic

File format

Requirements

  • The import file must always end with a new line.
  • columns separated by '|'
  • empty fields should be left blank
  • all dates + time must have this format: DD/MM/YYYY HH:MM

Columns:

If this program is used for importing historic blood bag info from e.g. an old LIS,at least 17 fields exist in the file (empty or not):

  1. the bag internal id (mandatory)
  2. the bag external id (mandatory)
  3. the bag donor (optional)
  4. the bag product spec. mnemonic (mandatory)
  5. the bag's department mnemonic (optional; if empty: the department in which you're logged in)
  6. the bag's typing mnemonic (optional)
  7. the blood group (readable, untranslated) (mandatory; '?' if unknown)
  8. the rhesus-D (readable, untranslated) (mandatory; '?' if unknown)
  9. the list of present or absent antigens in the format: [+/-]antigenmnemonic , ...
  10. the bag's production date-time (optional)
  11. the bag's receipt date-time (mandatory)
  12. the supplier mnemonic (mandatory)
  13. the expiration date-time (mandatory)
  14. the transfusion date-time (mandatory if transfused)
  15. the receiving patient PIN (if transfused, PIN or internal id must be specified )
  16. the receiving patient's internal id (if transfused, PIN or internal id must be specified )
  17. the transfusion hc provider's mnemonic (if transfused; optional)

Field 18 can also be specified

  1. the bag's status (readable, untranslated) (optional): This should be one of: initial/verification/administered.If empty: 'initial' or 'administered' is assumed according to 'transfusion date-time' being specified or not.

The following lines can contain (optionally): if status is 'administered': the transfusion reactions; else the bag's comment field.

The next line should contain: "---" (to indicate the end of the bag's info)

Import of non-transfused blood bags:

Blood bags that appear in the import file with an empty transfusion date (field 10), an empty PIN code (field 11) and an empty HC provider mnemonic (field 12) are assumed to be non-transfused and will be imported in GLIMS as non-reserved blood bags (free stock) with status 'Initial'.The comment field in the import file can still be used for these blood bags.

When used for BTM synchronization after structural upgrade

This program is also capable of importing a file in the format of "bloodbaglist.txt" file generated by 'ap_exbb'.It tries to update as much as possible: if required,

- all blood bag info is updated if not correct.
- the blood selections are created
- the crossmatch result is created
- the blood selection status is updated
- the blood bag status is updated

Note: the orders, the patients, the HC providers and the supplier organization(s) should already exist.
