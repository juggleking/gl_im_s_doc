Property import

Prerequisites

The import file should comply with the following rules:

  • The character used as delimiter between field values in the file is a space.
  • All character fields must be enclosed in quotes (" ") if they contain any delimiter characters (i.e. spaces).
  • If you want to import any quotes contained in the data, replace them by two consecutive quotes ("").
  • If you do not wish to specify a field, you should put an unquoted question mark (?).
  • The import file must always end with a new line.
  • Each record must meet the following format:

Name Mandatory Contents
Mnemonic

Ö

Mnemonic of the property
ShortName Short name of the property
RequestableMnemonic Allows to specify a default request definition for the property. If left unknown (?), no default request definition will be created.
Code Code of the property
Description

Ö

The property description is a multilingual text field. You can enter several texts in different languages. This can be done by importing the same property several times with a different language code and description (see example below)
DataType

Ö

The value of the DataType field can be specified either via its numeric or its symbolic English representation: e.g. Numeric = 1, String = 2, Enumerated = 3, ... Check the reference manual for the full list of datatypes. The program will reject invalid values.
ReportedDecimals Allows to specify the number of reported decimals.
UnitName If the property data type is numeric, a unit must be specified. (must be an existing unit).
LanguageCode Allows to enter a multilingual description of the property. Possible values are: E (English), N (Nederlands), F (Français), D (Deutsch), ES (Español)
Cumulative

Ö

Is this property to be reported with the previous values ? (YES/NO)
Dynamic

Ö

Results of dynamic properties which were obtained in one day are grouped on reports. (YES/NO)
Visible

Ö

Enumerated fields have to be specified using a number which represents one of the possible values. Check the reference guide for possible values. E.g. 3 = visible
MaterialMnemonic Material mnemonic for this property (must be an existing material mnemonic).
ReportFormat

Ö

The value of the ReportFormat field can be specified either via its numeric or its symbolic English representation: e.g. Columnar = 1, Full width = 2, Text = 3, Two Column Text = 4, Hidden = 5. Check the reference guide for possible values. The program will reject invalid values.

Activation

The property import program can be started from the main menu via Start -> System management -> Import -> Properties.

Parameters

The File name is the name of the patient import file. You can use the FIND key (TTY) or double-click (PC) to browse through your file system.

If the import file character set doesn’t match the GLIMS character set, you can specify an alternate one. The name that you specify must be a valid code page available in the $DLC/convmap.cp file (a binary file that contains all of the tables that Progress uses for character management). For more information please refer to the Progress ‘System Administration Guide’ that was provided along with your Progress license.
