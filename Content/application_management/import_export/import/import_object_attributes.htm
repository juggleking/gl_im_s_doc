Object attribute import

Prequisites

The import file should comply with following rules:

  1. The character used as delimiter between field values in the file is a space.
  2. All character fields must be enclosed in quotes (" ") if they contain any delimiter characters (i.e. spaces).
  3. If you want to import any quotes contained in the data, replace them by two consecutive quotes ("").
  4. If you do not wish to specify a field, you should put an unquoted question mark (?).
  5. Date fields are interpreted according to the date format (-d) and century (-yy) start-up parameters of your GLIMS session (typically "dmy"). However, it is advised to specify years in a 4 digit format (i.e. including the century).(see session.pf)
  6. Specify numbers with decimal points (no commas).
  7. The import file must always end with a new line.
  8. Each record must meet the following format:

Name Mandatory Contents
PIN Ö Patient identification number
AttributeMnemonic Ö Object attribute mnemonic as defined in GLIMS
StartDate   Start date of the attribute
EndDate   End date of the attribute

User interface

The object attribute import program allows importing object attributes for persons identified via their PIN. You can start this import program via Start > System management -> Database > Specific import -> Object attributes.

File name: name of the object attribute import file. You can use the FIND key (TTY) or double-click (PC) to browse through your file system.

Character set: if the import file character set does not match the GLIMS character set, you can specify an alternate one. The name that you specify must be a valid code page available in the $DLC/convmap.cp file (a binary file that contains all of the tables that Progress uses for character management). For more information please refer to the Progress System Administration Guide that was provided along with your Progress license.

PIN provider: is the correspondent that assigned the PIN contained in your import file. Typically this will be the Institution record identifying a hospital. If you have defined an Identification source in your order entry options screen, it will be proposed as default value.

Output

During import some statistics are issued on the screen indicating the progress. Also some problem files are generated:

File name Contents
<ImportFileName>.rej All records rejected for one of the following reasons: the PIN or object attribute mnemonic is empty or unknown, the object attribute doesn't exist in GLIMS, the end date is before the start date, or the start date is in the future.
<ImportFileName>.pin PIN's that were not found in GLIMS. The affected object attributes are of course not imported.
<ImportFileName>.obja Object attribute mnemonics that were not found in GLIMS.

Records with format errors cannot be processed and will cause an immediate error message, with the logical line number on which they occurred. In that case, the operator has the choice to continue or abort the import.
