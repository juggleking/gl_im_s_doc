Import of street-specific postal codes

In some countries (e.g. Luxembourg), the same postal code can be used for a number of streets, while a street may have more than one postal code. In order to support the import of such postal code files, a specific municipality import program is available under System Management -> Database -> Specific import -> Postal codes.

The format of the input files is

"Country code" "Postal code" "Name" "Street" "Box even" "Box minimum" "Box maximum"

For single-word character fields, integers or logical values, the quotes can be omitted. Note that "Box even" should be either ?, "YES" or "NO". No header line is allow, only data.

The program tries to find an existing municipality based on Country, Postal code, Name, Street, Box Minimum and Box Maximum before it decides to create a new record.

Please note that in order to allow municipality import on a specific country, the Country.PostalCodeFormat MUST be specified (e.g. as "X(15)")
