Update existing patients

An import tool is available to UPDATE (not create) existing patients in Glims through an import file.In Glims v8, this tool has to be added manually in the main menu through the "Run procedure" mechanism.In the function parameter setup, choose "ImportPersonAndValidationScreen" as function.This function cannot be automated through a command and task.

The functionality is similar to the already existing person import program (which allows to create persons).Differences:

  1. In the import screen, the PIN provider will by default be set to the National PIN provider (as specified in the general settings)
  2. The patient to be updated should have a PIN (national number) in Glims.However, if the PIN is not found and there is only a single person known in Glims with the specified last name, first name, birth date, sex, spouse last name, the update is allowed but a warning will be produced.The PIN will be created.
  3. A validation screen is shown per patient.It shows:
    1. The original patient data in Glims and the new data from the import file
    2. The changed values are in a blue font
    3. After validation, any update problems are shown in red and the user is offered the possibility to save the remaining data

  4. Old payment agreements are closed: during the import, the user also can enter the start date of the new agreement: default = today
  5. The structure of the import file is similar to the one used by the already existing person import program but has 4 optional fields at the end of the line:

FamilyDoctorLastName FamilyDoctorFirstName FamilyDoctorMunicipalityPostalCode FamilyDoctorMunicipalityName

Note: rejected or skipped patient updates are saved in a file <InputFileName>.rej (and <InputFileName>.mncp, .hcpr, .paya, .lng).
