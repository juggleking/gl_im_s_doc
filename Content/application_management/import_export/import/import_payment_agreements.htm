Payment agreement import

An import program is available for payment agreements.

The program can import CSV files with the following columns:

  1. Party.Person().InternalId

  2. Fund.Mnemonic

  3. PolicyName.Code

  4. Matriculation

  5. Discriminator

  6. StartDate

  7. EndDate

Example:

19940617DEBA04,2304009,DianaPlcn,20050601_01,1,2010/8/1,2020/1/1

Features

  1. Reevaluate order set defaults: will reevaluate the recent order sets (order object time max 7 days < today) of the imported persons

  2. Automatic closure of other payment agreements of the same person but with an other fund or matriculation (end date of old agreement = new agreement's start date -1)

  3. Wild cards are allowed in the file name (e.g. *.csv)

  4. Processed files are automatically renamed to <file name>.read
