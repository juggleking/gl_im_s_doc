Quality control: Import QC configuration from Advia 120

Introduction

GLIMS allows to import QC data provided through an import file provided by the Advia 120 analyzer. The program is available from the main menu via Start -> System management -> Database -> Specific import -> QC Configuration Advia 120.

GLIMS will create the QC lots and the corresponding QC populations based on the file being imported.

Download an example import file.

Parameters

GLIMS will create QC populations based on the following options:

General

Import file

Allows to select the Advia 120 QC configuration file to be imported in GLIMS.

Coding System

Specify the coding system containing a code set for the table "Property". Configure a code value for each property in GLIMS that is used on the Advia station.The code value has to match a strict syntax: TTT,v,s where

  • TTT: the 3 digit code on the instrument

  • v: the length of the target value field in the import file (integer value)

  • s: the length of the 2SD value field in the import file

Material

The target QC material of the QC lot

Station

The target station for the QC lot (QC populations)

Level

The level to be set for the QC population

QC population

Target mean, target deviation, target value, accuracy limit

Allows to define the field values for the created QC populations in GLIMS:

  1. Import: take the value from the import file

  2. Copy: take the value from a previous QC population (with same assessment method, same lot and -if specified- same level).

  3. Ignore: leave the field empty

Origin and Origin info, Accuracy severity, Severity 2S/3S, Limit 1/2 and Severities, Delta SD Limit and Severity, CV Max (%), Ignored deviation count

Allows to define the field values for the QC population fields:

  1. Ignore: leave the field empty (default)

  2. Set: take the value specified in this screen

  3. Copy: take the value from a previous QC population (with same assessment method, same lot and -if specified- same level).

You cannot combine 2SD and 3SD severities with the custom limit severities (severities for Limit 1 and Limit 2). Custom limit severities take priority over 2SD and 3SD severities.
