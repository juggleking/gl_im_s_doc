Automatic cleanup of temporary directory

Glims automatically purges files from the temp directory.When a user leaves the application, the temp-directory is scanned for obsolete files (done randomly, once of four times).Additionally, a running cron will scan the temp directory once a week.The scan is also done when the server application is started (database start up).

Files on the GLIMS TEMP directory matching any of

m*.brp
m*.htm
m*.lbl
m*.ljg
m*.prn
m*.rpt
m*.tfl
m*.tmp
m*.tsf
m*.txt
m*.doc
m*.docx
m*.pdf
m*.rtf

which are older than 30 days will be deleted.Progress tempfiles (lbiXXXX,dbiXXXX,srtXXXX) older than 7 days will also be deleted.However, when a value is specified in the gsit_MaxTmpFileAge field, this value will be used to determine the maximum temp file age when purging log files, instead of the hard-coded value of 30 days.
