Introduction

The license checks in GLIMS are based on the unique system identifier (Server Identifier). When this system identifier changes, you may need new license keys ! This document describes how this unique system identifier is obtained for the different operating systems.

COMPAQ TRU 64

Network card Mac Address (DLPI, first card returned by OS). Changing, removing, adding or switching network cards may cause the server identifier to change

LINUX

Network card Mac Address (SYS, first card returned by OS). Changing, removing, adding or switching network cards may cause the server identifier to change

WINDOWS 32

Network card Mac address (MacAddress.dll ; card for which license already exists taken with priority)

AIX

uname.machine : a unique identifier supplied by the vendor. No 2 ibm machines will return the same value. 

The machine ID number contains 12 characters in the following digit format:
xxyyyyyymmss. The xx positions indicate the system and is always 00. The yyyyyy
positions contain the unique ID number for the entire system. The mm position
represents the model ID. The ss position is the submodel number and is always
00. The model ID describes the ID of the CPU Planar, not the model of the System
as a whole.
(…)

IMPORTANT:

The machine identifier value returned by the uname command may change when new operating system software levels are installed. This change affects applications using this value to access licensed programs. To view this identifier, enter the uname -m command. Installing new os-levels on AIX may cause the server identifier to change.

SUN SOLARIS, HP-UX

The value returned by hostid; very likely to be a CPU serial number.
On hp-ux “hostid” is the value shown by “uname –i” ;
“man hostid” on solaris returns:

DESCRIPTION
     The hostid  command prints the  identifier  of  the  current
     host  in hexadecimal. This numeric value is likely to differ
     when hostid  is run on a different machine.
http://blogs.sun.com/ambiguous/20050620

gethostid() on solaris returns the same value; “man gethostid” says:

DESCRIPTION
     The gethostid() function returns the 32-bit  identifier  for
     the  current  host, which should be unique across all hosts.
     This number is usually taken from the CPU board's ID PROM.

Changing, adding, switching or removing the CPU board is likely to cause the server identifier to change.
