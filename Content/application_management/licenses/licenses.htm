Licenses

Contents: Introduction - Protected programs - License code activation - Obtain a license code - Deactivate licenses

Introduction

A number of programs within GLIMS are protected by a license check. You will need to enter a license key in order to activate the protected programs. For some programs, a demo period is available during which the program can be started without a license key. Once this period has elapsed, you will need a key.

Warning

The environment variable MA_HOST must be set to the server name !On Unix systems, you can check the server name with "uname -a".

The license check is based on the unique system identifier (Server Identifier). When this system identifier changes, you may need new license keys ! Click here to see how this unique system identifier is obtained for the different operating systems.

One of the prime objectives of laboratories is to ensure that laboratory quality control practices are adequate to achieve satisfactory analytical quality.

GLIMS provides internal and external quality control. Internal quality control (IQC) checks whether test results on quality control materials are reproducible.

External quality control (which may be enforced by law) checks whether different laboratories produce comparable results on the same material. In this case, the control samples may not even be

License code activation

Import license activation codes

The best way to activate all necessary license codes is to use an import file generated by MIPS. Contact MIPS to obtain such license activation file for your site. To access the import function, choose Start -> System management -> Licenses -> Import activation codes.

With this function, you can activate multiple product licenses in a single operation. The activation codes don't have to be typed in individually, but are imported from a text file generated by MIPS.

File name is the name of a MIPS-generated file with activation codes for one or more product licenses.

Note

The license activation import allows an optional 4th entry for each line in the input file. If present, it denotes the server identifier. If absent, the active server is assumed. This allows you to import activators for non-active cluster nodes.

Manual activation

In case the license activation file (see above) should not be complete, the following screen will be displayed when starting a program for which the necessary license is not available. This allows manual activation of the license.

Users: Specify the number of GLIMS users. GLIMS counts the number of connections and verifies whether the license user limit is not exceeded. Batch processes are not taken into account for a license usage. The Progress watchdog should be running against the genre database in order for this to work correctly.

If you do not enter the activation code, a message will be displayed, indicating how many days are left of the demo period (if available).

Obtain a license code

Please contact MIPS. You will need to specify the Resource (see figure 1: e.g. PRODUCTION STATISTIC) and the Serialization code.The latter can also be found in the Help -> About window.

Deactivate licenses

From the License browser, one can select specific license records and use right-mouse button menu option Deactivate. This functionality is reserved for users of type "system managers" [and developers] only.

The deactivated license record is NOT deleted (otherwise a new demo period would go into effect). The record is marked as 'invalid license'.
