Consistency check

Introduction

This tool is available from the main menu via Start > System management > Tools > Consistency check.

It performs an application consistency check and allows to repair when necessary. Inconsistent situations will not cause program errors (unlike database integrity errors) but may lead to unexpected behavior. Inconsistencies are typically caused by external factors like e.g. unexpected client disconnect due to network problems, etc...

Inconsistency examples

  1. An order does not reach status Complete although all specimens are confirmed and all results are validated.
  2. An action remains in status Frozen after network problems.

Options

Actions

If enabled, the tool will check inconsistencies in actions and will immediately correct them.

Orders

If enabled, the tool will check inconsistencies in orders and will immediately correct them.

Requests

If enabled, the tool will look for pending requests in orders and will immediately activate them.
Pending requests with a tariffication status higher than Initial will be deleted.

Test mode

If enabled, the tool will check for inconsistencies without immediately correcting them, which gives you the opportunity to first find out what caused the inconsistencies.
