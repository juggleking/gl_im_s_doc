Convert database to test database

By default, all databases are marked as production databases. However GLIMS offers a tool to convert a database to a test database. It is available from the main menu via Start -> System management -> Database -> Convert this database to a test-database.

This function

  • marks the database as test-database by enabling the This is a test database option (not visible in the application).
  • renames service and translators and turns off their auto-start.
  • disables all cron jobs and clears their execution status information.
  • clears the Watchdog port configuration.

Tips

  • It is highly recommended to choose a different user interface theme to more clearly distinguish between the production/test environments, thereby avoiding configuration changes in a live environment.
  • When a user logs into a production system with a Role that has User type set to Developer, a warning message will be displayed to avoid unintended data changes by service engineers.
  • GLIMS allows to customize the login screen to distinguish between production and test environments.
