User monitor

Content: Introduction - User monitor screen - Options

Introduction

The user monitor shows the users currently connected to the database together with some important database activity statistics. Note that this tool is only supported for Progress databases !

User monitor screen

Columns

The /s columns indicate how much the value of the preceding column has changed during the interval period (see the options button) or when you manually provoke a refresh by pushing F5.

In order to analyze this user monitor, the DBAccess and DBRead columns are the most important items.

The DBRead column shows the amount of database blocks that where read by this user which could not be found in the database server cache (physical access to the disks). When this number is high, it may suggest that the user (or the batch program) is performing a very heavy query. If such a situation persists and the overall performance is low, it might be wise to ask this user to stop his activity.

The DBAcces column measures the total access to the database (questions asked to the database). If this is high then CPU usage will be high too.

To facilitate stopping a user's activity, the detail rectangle at the bottom shows some information regarding this user like the computer he is working on, his process id, the time he logged in to the system,... The type of information varies according to the selected user type.

Usage

This tool has been developed to search for users or processes which put a high load on the database. Therefore it is possible to sort the table by clicking in the header.

Example: Click on the DBRead column. The highest will be on top. Now check the value of the according /s column. If the value is 0 and if it stays 0 during some time, this indicates that the process has used a lot of DBRead in the past but is not performing any DBReads anymore. If the /s would be high then we have caught a process red handed putting a high load on the hardware.

  • Processes with a high DBAcces and low DBRead: reporting,...
  • Processes with a high DBAcces and a high DBRead: statistics over a long period, invoicing,...

Options

  • User types: only display users of type
    • Self servicing clients: which do not connect to the DB over the network (cron's, analyzer brokers). These contain all GLIMS progress processes executed directly on the server itself.
    • Remote clients: opposite of self servicing clients. (e.g. GLIMS PC's)
    • System processes: e.g. BIW, APW, WDOG (Watchdog)
    • Servers: Progress servers. Each remote client uses a Progress server to connect to the database.
  • Connection type:
    • Batch: show batch connections
    • Interactive: show interactive connections
  • Server number: only show remote clients using the server with this number.
  • Interval: the user monitor automatically refreshed itself. This field indicates the time in seconds after which the monitor screen should be refreshed.
