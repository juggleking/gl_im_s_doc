User interface: Tools

About tools

A tool refers to a (pre-configured) function.  Tools can be used in:

  • Ribbons
  • Side bars
  • Home pages
  • The Start menu

Access

A tool is always scoped to a specific table.  To access the tools, you should therefore start from a Table context:

  1. Open the list of tables (Start -> System management -> Scheme -> Tables)
  2. Select a table and choose the contextual menu/ribbon item right Tools

Fields

Mnemonic

Short, unique identifier. Cannot contain spaces, quotes nor backward slashes.

Label

Multilingual label used for displaying the tool in a ribbon, side bar, home page, ...

Tooltip

Text that will be shown when hovering over a tool in the ribbon, side bar, home page, ...

Small icon / Large icon

Most functions in GLIMS will have a default icon.  These fields allow to customize the default icons.

  • Small icon: 16x16px
  • Large icon: 32x32px
  • Supported file types: png, gif, jpg, tiff, bmp

In the ribbon, the small or large icon will be used depending on the ribbon item's Preferred size.

Table

Read-only, table from which the tools browser was opened.

Function table / Function mnemonic / Function parameter set

Allows to assign a function and possibly a function parameter set to the tool.

  1. Select the base table (gp_Site for use in Application-wide ribbon, Side bar, Home page and Start menu).

     The Function table may differ from the Table but there must be a direct link between both tables (e.g. Order and Object).  For more information, see Function path (below).

  2. Select a function
  3. Select a function parameter set

Function path

If the Function table differs from the Table, this field should specify the connection path between both tables.

Note

The path consists of 1 or more elements, which are references (r1... rn) separated by a dot. A MISPL-like syntax is used.

For a given base table "t1", the first reference must have "t1" as source table (for outgoing references) or target table (for incoming references).

Subsequent references must connect to the source or target of the previous reference.

  • For outgoing references, the syntax is:

    ReferringField1.ReferringField2... ReferringFieldN

    Where the source table of ReferringField2 is the target table of ReferringField1.

  • For incoming references (inversions), the syntax is:

    <field name of IncomingReference>^<source table name>.<path>

As a whole:

Path == <PathElement>[.Path]

PathElement == IncomingRefField^IncomingRefTable[-RefId] | ReferringField

RefId is optional

Example

In order to add a tool (to the context ribbon or menu) to open the genetic exam screen from an action (in the worklist editor, for instance):

  • Create a tool on the Action table:

  • In the Function path field enter: Action^ApproachActivityAction.ApproachActivity.Approach.GeneticExam

Notes

It is not possible to specify a Function path if the base table of the tool is gp_Site.

Skip setup

If enabled

The setup screen linked to the selected function will not be shown when starting the tool.

If disabled

The setup screen will first be shown on screen (e.g. query options) before actually starting the tool, which allows updating the parameters first.

Notes

Notes

  • The option Skip setup will only be visible when the chosen function actually supports this feature.
  • For functions that change data in the database (as opposed to functions that simply open a browser), the Skip setup option will only be visible when a parameter set is defined.

Start tool

GLIMS allows to select a Start tool that will be executed as soon as users login. The Start tool defined at User-level has precedence over the Start tool defined in the gp_Site level.

You can also specify a Start tool via the command line. The -param command line parameter has an option -T where one can specify the Mnemonic of a Tool record defined on the gp_Site table. If the -T option is specified, then it overrides the Start tool defined at the User and gp_Site level.

Example

This allows e.g. for better integration with third-party lab work flow monitoring dashboards.

Assuming there is a gp_Site-based Tool with Mnemonic UrgencyMonitor, you could start the application with a command-line as follows:

path\to\prowin32.exe -basekey ini -ininame bin\progress.ini -p gp_prun -param "-P be/mips/ablframework/gui/start.p -T UrgencyMonitor" -assemblies assemblies

Tool to launch an executable

Gp_Site based function "LaunchExecutable"

The gp_Site-based function LaunchExecutable is available. Armed with this function, system managers can create the necessary tool records, menu items, ribbon items and home page shortcuts for each of their external tools.

Parameters

The function parameter set for the LaunchExecutable function offers the following parameters:

Executable directory

The directory in which the executable is stored on disk. The directory must exist and the user must have Read access.

Executable name

The full name of the executable file, including extension. The user must have Read access.

Arguments

The command-line arguments to pass to the executable.

Working directory

The directory in which to start the executable. The user must have Read and Write access.
