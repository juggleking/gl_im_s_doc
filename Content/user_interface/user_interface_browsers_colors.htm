Introduction

The color schemes for results, orders and other records can be tailored to the needs of a site using general settings available under System management -> Customize -> Colors. These general settings are presented below.

Note

The defined colors are valid for all the users of the site.

Results

This will bring up a browser where you can see the already defined colors. You can choose a fore- and background color for any combination of Severity and Status.

Defining a color combination

  1. Use the Insert key to add a new line.
  2. Click in the Sev cell and specify a severity.
  3. Double-click the Status cell and choose a result status.
  4. Double-click the Example cell to select a foreground and background color.
  5. The chosen color will be applied to the specified Severity / Status combination.
Note

Every color corresponds to a certain result severity and status, meaning that results with that, or a higher severity and with that, or a higher status will be shown in that color. For instance, if you have not defined a color for status Validated, validated results will be shown in the color defined for Confirmed results.

When a result matches with two colors, the color with the highest severity is used.

Examples

Example 1

Severity Status Foreground color
0 Available Green
1 Available Red

Results with status Available and severity "3" will be shown in red.

Example 2

Severity Status Foreground color
1 Available Green
0 Confirmed Red

Given that the severity has priority, results with status Confirmed and severity "1" will be shown in green.

Removing a color combination

To remove a color combination:

  1. Select a row by clicking the Arrow icon in the first column.
  2. Use the Delete key to remove the row.

Previewing the color scheme

At the bottom of the screen, in Display color scheme for, you can specify a Severity (for results) or an Urgency (for orders) and preview the defined colors.

Reset to default colors

The Defaults button allows the user to reset the color scheme to the application defaults.

Orders

The same goes for Orders, but in this case a color corresponds to an Urgency and Status combination.

Genetic exams

The same goes for Genetic exams, but in this case a color corresponds to a Severity and Status combination. The color scheme is applied in the Genetic exams browsers and grids: Genetic exams query browser, Other genetic exams panes, ... However, it is not applied in dynamic browsers such as Genetic exams of order or Genetic exams of object.

If no color scheme is defined for genetic exams, the scheme defined for normal results is used.

Locus results

The same goes for Locus results, but in this case a color corresponds to a Severity and Status combination. The color scheme is applied in the Locus results pane.

If no color scheme is defined for locus results, the scheme defined for normal results is used.

Variant results

The same goes for Variant results, but in this case a color corresponds to a Severity and Status combination. The color scheme is applied in the Variant results pane and Variants pane.

If no color scheme is defined for variant results, the scheme defined for normal results is used.

Other records

To define color schemes for other records,

  1. go to System management -> Scheme -> Tables -> right-click on a table -> Go to -> Style sheet.
  2. in the Style sheet editor, choose two criteria on which the color scheme should be based.
  3. Restart GLIMS.
  4. Open the Color scheme editor (System management -> Customize -> Colors) : the table you chose appears in the Select a table field.
  5. Define the colors as described above.

Warning

Once you have created a style sheet for a table, you cannot delete it anymore. In other words, the table will always appear in the Select a table field.
