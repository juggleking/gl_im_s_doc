Screen customization

Contents: Introduction - How to customize the screen -  Which screen layout is used

Introduction

The Order entry and Order consultation screens in GLIMS can be customized. This mean that you can:

  1. Change the position of a field within the screen.
  2. Change the label of a field.
  3. Change the label of a tab page.
  4. Hide a field from the screen.

Notes

  • Only users having a role with User type set to System manager or Developer can change the layout of a screen.
  • This generic functionality can have side effects if used in other screens than the order entry and order consultation screens. Its usage in other screens is therefore not recommended!

How to customize the screen

Open the screen to be customized and press the F12 key.

New code

The screen layout is saved using a code. You can either:

Type in a new code

If you want to setup a new layout, type in a new code. The screen will first be reset to the default layout.

Select an existing code

If you want to edit an existing layout, select an existing code.

Tip

  • When customizing several screens, make sure to use the same code as you can only specify one code that GLIMS should use for all screens (see Which screen layout is applied).

Notes

  • The last selected code remains effective until you leave the program completely.
  • Clearing the code will show the default layout of the screen.

Mode

Layout

Layout mode allows to change field labels, field positions, hide fields etc. Select Layout and press Enter to start customizing the screen.

Page

Page mode allows to change tab page labels. Select Page and press Enter to start customizing the tab pages.

Customization features - Layout mode

After having entered/selected a code and Layout mode, press Enter to start customizing the screen.

Message with navigation keys

Screen is opened in customization mode

Customization options

Key Function
Arrows Use the arrows to move the selected field
+ / -

Using the + / - keys, you can specify the distance (RowJump, ColumnJump) which is used when you move the fields. The - key allows to move the selected field more accurately.


Example:

Page Down / Page Up Move the selected field to the previous / next tab page
Delete Hide the selected field from the screen
Insert Makes ALL fields which where made invisible by the customization, visible again
Tab Select the next field
Home Reset to factory defaults
F6 Change label of currently selected field

Notes

  • When customizing the order entry screen, the fields from the Billing tab page cannot be moved to another tab page! It is also not allowed to move fields from another tab page to the Billing tab page!
  • If the window to be customized contains a tab page with name SiteAttr. (containing the defined site attributes), we advise not to change anything to that page !  If you do this anyway, the screen customization can be reset to the factory defaults !
  • Each layout is identified by a code.  GLIMS will automatically create a site attribute on gp_Site (starting with CustW-<editorname>...) for the screen that is customized.  The value of that site attribute stores the actual layout for that screen.

Customization features - Page mode

After having entered/selected a code and Page mode, press Enter to start customizing the screen. Enter your proper label in the Customized column. Click OK to save your changes.

Save customized screen

To save your customization, press the Esc key or the Customize button.

Which screen layout is applied

Introduction

This chapter explains which layout will be used in GLIMS. A distinction must be made between the order entry screen and all other customizable screens.

All screens except order entry

The environment variable MA_CustomizeKey in the progress.ini file allows to specify the layout code to be used.

Note

If system managers define a new layout, this layout will be used during the current GLIMS session.  When logging out and logging back in, the value of the environment variable will be applied.

Order entry screen

The difference between the order entry screen and other customizable screens is that the layout code can be included in the pre-configuration.  So you could define several pre-configured menu or ribbon items that open the order entry screen each with a different layout.

Standard, not pre-configured

  1. Layout code specified in the order entry options of the logged in user.
  2. Value of the environment variable MA_CustomizeKey in the progress.ini file
  3. Default layout of the screen

Pre-configured

  1. Layout code defined in the function parameter set (order entry option).
  2. Value of the environment variable MA_CustomizeKey in the progress.ini file
  3. Default layout of the screen

Note

If system managers define a new layout, this layout will be used during the current GLIMS session.  When logging out and logging back in, the value of the environment variable will be applied.
