User interface - Browsers: Styles

Introduction

Styles can be used to customize how records are displayed in a browser.

Note

In case of result and order browsers, the site specific color scheme has precedence over the styles.

Defining styles

Styles can be defined via Start -> System management -> User interface - > Styles.

Defining site functions

Site functions must be defined that return the Mnemonic of the Style. Moreover, their Description should begin with Style. These site functions serve as a link between the configured styles and the button in the contextual ribbon tab Tools that is used to apply them. The usage of site functions allows for different styles to be applied conditionally to the records in the browser.

Example

This site function applies the "Red" style to user records with a last name beginning with "A" and the "Blue" style to all other records.

Applying styles

A style can be applied to a browser using the Style button in the contextual ribbon tab Tools. Clicking on the Style button displays a list of site functions, where the user can select from all the site functions of data type String for the current browser's base table.

Notes

  • If the list of site functions does not open, then styling is not yet supported by the current browser.
  • Active site functions (which modify the database content) cannot be selected.

Clearing styles

The Clear style menu item of the Style split button allows to clear any active style.

Note

If a style is still active upon closing of the browser, it will still be active the next time the browser is opened.
