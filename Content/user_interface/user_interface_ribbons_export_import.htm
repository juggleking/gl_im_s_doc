User interface - Ribbons: Export / Import

Case description

Customers upgrading from GLIMS 8 to GLIMS 9 will typically first upgrade the test environment in which they can prepare the new user interface in GLIMS 9.

After having upgraded the production environment to GLIMS 9, it is then possible to export the new user interface configuration from the test environment and import it into the production environment.

Note

Although this document focuses on Ribbons, you would probably also export and import the Home page, Side bar, Quick access toolbar, Roles and Privileges.

Export manually

If you only need a subset (e.g. only export Ribbon configuration), then do not use the script and select the tables manually.

Tables

um_Tool, um_Ribbon, um_RibbonTab, um_RibbonTabAssigment, um_RibbonGroup, um_RibbonItem, bt_ActualFunctionParameterSet, bt_ActualFunctionParameter, bt_ActualQueryParameterSet and bt_ActualQueryParameter.

Languages

Select the languages you want to export.

File name style

Select Table name.

Ribbon import

Delete all existing ribbon tabs

To avoid duplicate ribbon items, you need to delete all existing ribbon tabs before starting the import.

  1. Login with a user having a role with Minimal user type set to Developer.
  2. Click Start -> Development -> 4GL -> Run procedure.
  3. File name must be gp_DeleteAllRibbonTabs.p.
  4. Leave the option Pass handle disabled.

Import manually

If you selected the tables to be exported manually, then import these tables in the same order as in the import script, skipping those you have not exported.

Troubleshooting

When importing um_Tool.csv, there might already be a Tool with the same Function, Table, FunctionPathName and FunctionParameterSet.

If the tool you want to import has a different Mnemonic and a RibbonItem is pointing to this Tool, then the RibbonItem will not be imported either.

In the um_RibbonItem.csv.csv file you will find lines with Tool not available. In this case, edit the um_RibbonItem.csv file and rename the Tool.Mnemonic reference.

Restart GLIMS

Restart the application and the imported ribbon(s) should be visible.
