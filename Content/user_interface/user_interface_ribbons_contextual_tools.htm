User interface - Ribbons: Contextual tools

Each contextual ribbon has a fixed Tools tab:

Record

New

This will create a new record.

Duplicate

This will copy the focused record.

Edit

This will edit the focused record.

Save

Allows to save changes to a record when in edit mode.

View

This will view (the details of) the focused record (activate associated editor in view mode).

Delete

This will activate the Delete verification window, prompting for explicit acknowledgment before deleting the focused or selected records.

Navigation

Previous

This will reposition to the previous record (in editors) or the previous set of records (in browsers). Shortcut key: Page Up.

First

This will reposition to the first record in a query.

Next

This will reposition to the next record (in editors) or the next set of records (in browsers). Shortcut key: Page Down.

Last

This will reposition to the last record in a query.

Find

This will produce a Finder dialog, enabling you to specify record search criteria.

Find next

This will reposition to the next record matching the search criteria specified previously in the Finder dialog.

Data

Query / Reopen

From a browser, reopens the current query (after prompting for any selection criteria).

Note

You can also use the SHIFT-F5 shortcut key to activate this feature.

Refresh

Refreshes the content of the focused window.

Auto refresh

When activated, the current browser will auto refresh every 60 seconds.

Note

This allows to continue working and that several auto-refreshing browsers can be put on the screen simultaneously. You can also use the ALT-F5 shortcut key to activate this feature.

Filter

Allows selection of a filter (site function of type Logical on the applicable table) for selective display of records.  Note that active site functions (which modify the database content) cannot be selected.

Clear filter

From a browser, clears any active filter, so that all records contained in the query are again visible.

Style

Allows to apply a color scheme to the current browser. For more information on the required configuration, see Styles.

Clear style

From a browser, clears any active style.

Selection

Select all

Selects all records in the query.

Toggle current

Toggles the selection of the focused record.

First to current

Selects (i.e. enables the check boxes of) the "first to current" rows.

Note

The SHIFT-HOME key combination also selects the "first to current" rows but applies row selection (instead of record selection).

Current to last

Selects (i.e. enables the check boxes of) the "current to last" rows.

Note

The SHIFT-END key combination also selects the "current to last" rows but applies row selection (instead of record selection).

Column

Used in grids.

Row

Used in grids.

Specific criteria

Not supported.

Deselect all

Deselects all records in the query.

First to current

Deselects the "first to current" rows.

Current to last

Deselects the "current to last" rows.

Column

Used in grids.

Row

Used in grids.

Invert selection

Toggles the selection of all records in the query.

Edit

Cut

Moves the selection to the Windows clipboard.

Copy

Copies the selection to the Windows clipboard. From a browser (see below), copies entire query to clipboard.

Paste

Copies the contents of the Windows clipboard to the cursor position.

Undo

Reverts the last edit operation in text editor controls.

Clear

Clears the contents of a fill-in control from the cursor position onward.

Recall

Reverts the contents of a fill-in control to its original value.

Action

Print

The browser content will immediately be sent to the default printer.

Note

The default printer is:

  1. The printer specified in the User record
  2. The printer specified in the general settings (System management -> Customize -> MATE)

Print ...

Allows to set some options before printing the browser content.

Selection only

If enabled, only the selected records will be printed. If records have been selected, this option will be enabled by default.

Device type

Select an output type.

Format

Some browsers do not support all formats. The Device type can also restrict the available formats.

Printer

Set as default

Set the selected printer as default printer.

Directory / File

Only available when Device type is set to File.

Copies

Number of copies to be printed.

Export

Uses the denormalized export program

Export to Excel / Export to Word

Allows to export browser content to an MS Excel / Word file.  Only the columns shown on screen are exported.  You can also use the shortcut key CTRL-O.

Run MISPL

Activates the Evaluate MISPL function dialog, where you can specify a site function that should be evaluated for the focused record.

Report

Allows the generation of a custom report for the browser.

Title page, First page header, Header, Body, Footer

These fields allow to define the actual content of the report. They allow to select a dynamic Text based on the table of the browser for which you would like to generate a report.

Example

If you want to print a report of the Property browser, you can select Property-based Texts.

Language type

Defines the language if you use multilingual texts.

Format

Text

Allows generating a report in plain text, using the content defined in the specified Text modules (see above).

PDF, XML

Allows generating a report with the Report builder technology in GLIMS. Use the External template field to specify the Jasper report template to be applied.

Content

Analogous to other Report Builder output, the application will generate an XML file.

- the root element is always GenericReport

- the root element contains a child element Metadata with sub-elements:

Table (Id, ShortName and Name)

TitlePage (Mnemonic of the chosen TitlePage Text record)

FirstPageHeader (Mnemonic of the chosen FirstPageHeader Text record)

PageHeader (Mnemonic of the chosen PageHeader Text record)

Body (Mnemonic of the chosen Body Text record)

PageFooter (Mnemonic of the chosen PageFooter Text record)

Language (Type and Code)

ExternalTemplate (Mnemonic, WindowsPath and UnixPath of the chosen ExternalTemplate record)

OnePagePerRecord (Yes/No)

SkipEmpty (Yes/No)

XmlEscapeTexts (Yes/No)

PageHeight

NumberOfCopies

ReportUser (LoginName, FirstName, Initials, LastName and Email of the effective User record)

ReportDateTime

- the root element contains a child element Data with sub-elements:

TitlePage

Record (one per selected record) with sub-elements:

FirstPageHeader (the evaluated dynamic text, if set in the user interface)

PageHeader (the evaluated dynamic text, if set in the user interface)

Body (the evaluated dynamic text, if set in the user interface)

PageFooter (the evaluated dynamic text, if set in the user interface)

[Table name (e.g. gp_Holiday)] with sub-elements depending on the actual table

Notes

  • The XML structure does not include the full data of referenced database records. If you need additional related data, you can use the FirstPageHeader, PageHeader, Body or PageFooter dynamic texts to extend the XML structure with this data.
  • Enable the option XML escape texts to ensure valid XML documents when using dynamic Texts.

External template

External templates define the Jasper report template to be used when generating a PDF report. See Format.

Page height

Number of lines on a single page. Mainly used for Text reports.

Number of copies

Number of report copies to be generated.

Printer

Set as default

Set the selected printer as default printer.

Don't generate document when no records available

Preview

Display a preview of the report, before actually printing it.

Output tot file

Allows saving the report to file. A file specification window will be displayed.

One page per record

Allows to put each record on a separate page.

XML escape texts

Ensures valid XML documents when generating PDF or XML reports. See Format.

Lookup

Activates a lookup table for controls with constrained values.

Zoom

From a referencer, activates editor associated with referenced record.

Spell check

Will be enabled whenever you are positioned in a text field, anywhere in GLIMS. Examples: Order.Description field, zoomed editor of text modules, ...   The spell check uses the MS Word "Spelling and Grammar" functionality.

GLIMS will try and use the MS Word dictionary that corresponds to the language of the logged in user.  If not available, a warning message will be displayed.

Hyperlink

Use this to edit a hyperlink.

Reset screen

Allows the user to reset browsers, editors and outliners to their default settings. When clicking on the ribbon item, a message is shown to the user indicating which settings will be reset. If the user decides to continue, the settings will be reset and the screen will be closed.

For browsers, the following settings are reset:

  • Screen size and position
  • Column visibility, order and width (.NET only)
  • Infobox visibility (collapsed or not) and size (when applicable, e.g. in the Orders by internal ID browser) (.NET only)
  • Active sort column (when applicable, e.g. in the Order consultation browser) (.NET only)
  • Active style function (.NET only)

For editors and outliners, only screen size and position are currently reset.

View

Audit

Opens the audit dialog for viewing creation and last update audit stamps associated with the current record.

Log entries

Opens the log entries browser for viewing any log entries that are scoped to the current record.

Field history

Allows to see a history of changes applied to a site function.

Attachments

Show the available attachments for the focused record.

Codes

Produces a list of codes for the selected record (See Coding systems).
