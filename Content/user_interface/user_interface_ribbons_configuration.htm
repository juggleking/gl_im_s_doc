Ribbon structure

Both the site ribbon and the role ribbons have the same basic structure:

  1. A ribbon consists of application-wide and contextual ribbon tabs. In both cases, start from a table-based view: the main application-wide ribbon contains functions from the gp_Site table, the contextual ribbons contain functions based on a specific GLIMS table.
  2. A ribbon tab contains one or more ribbon groups that gather related functions.
  3. A ribbon group contains one or more ribbon items.
  4. A ribbon item refers to a tool.
  5. A tool refers to a (pre-configured) function

Site ribbon versus role ribbon

The concept of a single site ribbon differs from the setup for multiple role ribbons.

Site ribbon

The ribbon to be used as site ribbon is specified in the general settings (Start -> System management -> Customize -> MATE).

Role ribbons

The role-based ribbons are specified for each role:

Before starting the configuration of role-based ribbons, you should clearly define what parts of the ribbon will be common for all ribbons, which parts will be reused among some of the ribbons. Functions that logically belong together should be placed in a reusable ribbon tab.

Start from a table-based view: the main application-wide ribbon contains functions from the gp_Site table, the contextual ribbons contain functions based on a specific GLIMS table.

Take the following concepts in mind:

Inheritance

When for a given table, no ribbon tab is assigned to the role-based ribbon, the ribbon tab from the site ribbon will be used.

Example

  1. For the role-based ribbons, you only want to customize the "application-wide" ribbon tabs.
  2. Define custom ribbon tabs for the gp_Site table and assign them to the role-based ribbon(s).
  3. The "Contextual ribbons" will be inherited from the "site-based" ribbon, unless a role-specific ribbon tab is assigned for a specific table.

Reusable ribbon tabs

A ribbon tab can be reused in different ribbons by adding multiple ribbon tab assignments.

Example

  1. You have two roles "Lab technician" and "System manager"
  2. User with role "Lab technician" should have two ribbon tabs: "Routine" + "Microbiology"
  3. User with role "System manager" should have two ribbon tabs: "Microbiology" + "System management"
  4. Define 3 ribbon tabs and assign them to the correct role-based ribbon. The "Microbiology" ribbon tab should be assigned to both role-based ribbons.
