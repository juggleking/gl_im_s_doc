Introduction

Many browsers have an Info box at the top of the screen: result confirmation, result validation, microbiology action browser, etc.

This document describes the contents of these info boxes and how you can customize them.

Customizing the info boxes

Content

The content of the info boxes can be customized using a text module with mnemonic <Master>Externalization. (e.g. ObjectExternalization).

Note

By using the symbolic reference {DefaultExternalization} in the user-defined text module (on the first level!), the pre-programmed default can be re-used in the user-defined content.

Fonts

The font and color can be specified in the Progress.ini configuration file.  

  • The font used in the info boxes is font 25.
  • The background and foreground color of the info boxes are color 74 and 75.

Size of info boxes

  1. The info boxes will automatically enlarge (if needed) when clicking inside the box.
  2. The original box size is restored when releasing the mouse button.  
  3. Double-clicking on the box opens a large text editor to the screen.

HTML support

When customizing the content of the info boxes, GLIMS allows to use HTML. When the text module contains <HTML>, your internet browser will be used to display the text instead of the built-in text editor.

Warning

GLIMS does not support Frames nor iFrames in the HTML definition !

Notes

  • The height of an HTML box can be defined by placing the comment <!--height=<xx>--> somewhere in your HTML text (where <xx> stands for the desired height in character units).
  • Automatic enlargement by clicking inside the box is not supported when using HTML.

  • To preserve the background/foreground colors defined in the Progress.ini file, the following text is added automatically before the custom text:

    <html> 
    <style type="text/css"> 
    body { 
    margin:2; color: rgb(<br>, <bg>, <bb>); 
    background: rgb(<fr>, <fg>, <fb>);
    </style>

    where <br> ... <fb> define the rgb-values of the box background and foreground colors as defined in Progress.ini

Typographic conventions

Blue italic expands to a component explained in detail below.

Red bold denotes text that will appear literally.

Green italic refers to database members.

Where .Type is one of the list "Animal, Lot, Unknown, QC lot". For person objects, the object type is omitted.
