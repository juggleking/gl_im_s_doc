Application-wide ribbon

The application-wide ribbon is always present in the user interface. It groups the most important GLIMS functions, and serves as a starting point for starting these functions.

In GLIMS terminology, the application-wide ribbon contains functions defined on the gp_Site table.

Contextual ribbons

Contextual ribbons are not always present in the user interface, but conditionally in function of the context. They group the most important contextual functions and are only visualized when a specific context is active.

In GLIMS terminology, the contextual ribbons contain contextual functions defined on a GLIMS table. They are an alternative to the right-mouse click contextual menus.

The contextual ribbon also has a fixed Tools tab with general utilities.

Behavior

When the user logs in to GLIMS and the user interface is displayed, no context is activated within the application. Hence the application-wide ribbon is visualized.

When the user starts a GLIMS program, e.g. Orders by Receipt Time, the order context is activated and the order contextual ribbon is visualized. The application-wide ribbon is still present and accessible within the user interface; however the focus is put automatically on the contextual ribbon.

When Orders by Receipt Time is closed, the order contextual ribbon disappears and the focus is put back to the application-wide ribbon.
