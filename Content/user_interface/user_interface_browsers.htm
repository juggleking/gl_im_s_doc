User interface - Browsers

Introduction

This topic describes the common features that are supported in all redesigned .NET browsers (lists of records) in GLIMS.

Classic browsers (as opposed to redesigned .NET browsers)

Classic browsers (i.e. browsers which have not been redesigned as a .NET browser) offer some but not all of the functionality described in this topic.

Tip

In classic browsers, the record which has focus is indicated with a gray selection bar. In order to use a blue selection bar (which is more easily visible), add OldBrowseSelectionBar=yes in the [Startup] section of the progress.ini file.

Field chooser

Introduction

Each browser has a Field chooser which allows defining which columns should be shown.  It is located in the top left corner of the browser.

Available columns

When clicking the Field chooser, the list with available columns is shown.  You can add a column to the browser by:

  1. Clicking the check box in front of the column.
  2. Dragging the column to the desired position in the browser.

The following items are available:

Database columns

All columns of the browser's base table.

Site attributes

All site attributes defined on the browser's base table.

Coding systems

All coding systems for the browser's base table.  Allows to show the codes directly in the browser.

Custom columns

All dynamic texts based on the browser's base table.  Allows to add dynamic, custom content in the browser.

Note

The Class of the Text should be set to DynamicColumn.

Note

The content of the Field chooser is loaded when opening the browser for the first time and is not refreshed within a GLIMS session.  So when creating a new site attribute, coding system or custom column, you have to re-login before the Field chooser will show it.

Column width

Columns can be resized by dragging the column divider to the desired location.

Column position

To change the position of a column:

  1. Click its column header
  2. Drag and drop it to its new position

Sorting

Browsers can be sorted by clicking the column header.  The sort icon in the column header indicates the current sorting.

For columns that do not support sorting (performance reasons), clicking the column header will have no effect.  

Sorting on multiple columns

It is also possible to sort on multiple columns.

  1. Click the header of the first column that should be used for sorting.
  2. Shift-click the header of the second column that should be used for sorting.
  3. You can shift-click again to change the sorting order (ascending/descending).

Navigation

Fast positioning

You can navigate to a record in the browser by typing the first characters of the value of the current sorting column, e.g. first characters of the property mnemonic.  

A text window will show the characters you typed. To reset the typed in characters, select another record and start typing again.

Note

You can fast-position on a question mark (?) if the question mark is not the first character of the string you search for.

Finder

CTRL-F will open the Finder window.

Selection

GLIMS distinguishes two types of selections:

Row selection

Visualization

The row is highlighted with a blue background, but the check box in the first column is not enabled.  Row selection can be used to prepare the actual selection.  

Note that row selection will not be taken into account when performing a function on these records.  You first need to switch to record selection by using the space bar (check boxes will be enabled for all records in the row selection).

Row selection methods

The standard Windows selection mechanisms can be used for row selection. Examples:
  • SHIFT + Arrow down/up
  • SHIFT + Mouse drag
  • CTRL + Click / Drag to add records to the current row selection
  • SHIFT + HOME: first to current
  • SHIFT + END: current to last

Record selection

Visualization

The check box in the first column is enabled.  Only Record selection is taken into account when performing a function.

Record selection mechanisms

  • The contextual ribbon tab Tools has a ribbon group Selection.  All items in this group can be used for Record selection (e.g. Select all, First to current, Deselect all, Invert selection, ...).
  • CTRL-A: adds all records to the Record selection
  • CTRL-N: removes all records from the Record selection
  • Space bar: adds/removes the records in the row selection to/from the record selection

Refreshing

Full browser refresh

Auto refresh

Activation

The Auto refresh functionality can be activated / deactivated in the Tools tab of the contextual ribbon.

Visualization

When Auto refresh is enabled for a browser, the text Auto refresh is added between brackets to the browser's title.

Manual refresh

To manually refresh an entire browser, press the F5 key or use the Refresh item in the Tools tab of the contextual ribbon.

Note

If the browser offers a query options screen, the specified query criteria will be re-evaluated. Records that do no longer meet the specified criteria (e.g. due to changed data) will no longer be shown in the browser. New records that meet the query criteria will be added.

Specific row refresh

A specific row refresh occurs after:

  • Changing data of a record via its editor (F6).
  • Executing a function on a record via the contextual ribbon / menu.

Only the row of the updated record is refreshed and not the entire browser. This means that the record remains (and maintains its position) in the browser even if it no longer meets the specified query criteria. A full browser refresh will remove it from the browser.

Attachments

If a record has attachments, a paperclip icon will be shown.  

To see the attachments, click the Attachments button in the contextual ribbon tab Tools.

Export

Browser content can be exported to MS Excel / Word using the Export button in the Action group of the contextual ribbon tab Tools.

Print

Browser content can be printed using the Print button in the Action group of the contextual ribbon tab Tools.

Report

A report containing the browser content can be generated using the Report button in the Action group of the contextual ribbon tab Tools.

Styles

Row coloring can be applied using the Style button in the Data group of the contextual ribbon tab Tools. For more information on the required configuration, see Styles.
